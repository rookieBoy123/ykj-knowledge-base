# 药品信息管理流程（后台）

> 文档版本：release/v2.17.2 | 面向读者：运营、管理员

---

## 概述

管理员在后台维护药房的药品数据库，包含中药和成药两大类。支持药品的新增、查看、删除、批量Excel导入导出，以及从标准药品库同步数据。

## 触发条件

- **前置条件**：管理员已登录后台，从药房列表的「管理药品」入口进入
- **入口**：后台 → 药房管理 → 药房列表 → 操作列「管理药品」

---

## 步骤详解

### 第一步：切换药品类型

1. 进入药品管理页后，默认选中「成药类型」选项卡
2. 页面上方有两个选项卡：**成药类型** 和 **中药类型**
3. 切换选项卡后：
   - 搜索条件中的「药品监管id」字段在中药类型下自动隐藏
   - 列表明细中：监管id、药品类型（OTC/RX）、生产厂家、药品规格在中药类型下自动隐藏
   - 计量计价单位在成药类型下自动隐藏
   - 操作按钮的权限标识随类型切换（成药导入权限`med:drug-lib-patent:import`、中药导入权限`med:drug-lib-herbal:import`）

### 第二步：搜索药品

支持以下搜索条件：
- **商品名称**：模糊搜索，最多255字
- **通用名称**：模糊搜索，最多255字
- **药品监管id**：仅成药类型可用

### 第三步：列表浏览

列表展示字段（按类型动态变化）：
- **成药**：序号、监管id、通用名、商品名称、药品剂型、药品类型（OTC=非处方药/RX=处方药）、生产厂家、药品规格、创建时间、操作
- **中药**：序号、通用名、商品名称、药品剂型、计量计价单位、创建时间、操作

每页展示10条记录。当翻页越界时（如删除最后一页的最后一条），自动回退到上一页。

### 第四步：新增药品

1. 点击列表上方「新增」按钮
2. 跳转到药品编辑页面，携带类型参数和模式参数（`type=add`）
3. 填写药品信息后保存

### 第五步：查看/编辑药品

1. 点击操作列「查看详情」
2. 跳转到药品详情页（携带药品ID、类型和模式参数）
3. 详情页可查看药品的完整信息，不可编辑（`type=detail`）

### 第六步：删除药品

1. 点击操作列「删除」按钮
2. 弹出二次确认弹窗：「此操作将永久删除该药品, 是否继续?」
3. 确认删除后根据药品类型调用不同接口：
   - 成药：`chengDrugsDel`
   - 中药：`zhongDrugsDel`
4. 删除成功后提示"操作成功"，自动刷新列表
5. 取消删除则提示"已取消删除"

### 第七步：Excel导入药品

1. 点击「下载模板」按钮，下载对应类型的Excel模板
2. 按模板格式填写药品数据
3. 点击「导入」按钮，选择填好的Excel文件上传
4. 系统处理导入后弹窗显示结果：
   - 总导入N条，成功N条，失败N条
   - 失败记录列表展示序号、通用名、失败原因（便于修正后重新导入）
5. 导入完成后列表自动刷新

### 第八步：导出药品

1. 点击「导出」按钮，根据当前搜索条件导出数据
2. 导出范围：当前搜索条件匹配的所有记录（不限于当前页）
3. 系统显示"加载中"提示，完成后自动下载文件
4. 导出权限随药品类型变化：
   - 成药导出权限：`med:drug-lib-patent:export`
   - 中药导出权限：`med:drug-lib-herbal:export`

### 第九步：同步标准药品数据库

1. 仅成药类型下显示「同步药品数据库」按钮
2. 需要权限：`med:drug-lib-patent:sync`
3. 点击后系统自动从标准药品库拉取最新数据
4. 成功后提示"操作成功"

---

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 删除确认取消 | 提示"已取消删除"，不执行任何操作 |
| Excel导入失败 | 弹窗展示每条失败记录及其失败原因 |
| 翻页越界 | 自动回退到上一页，重新查询 |
| 导出加载中 | 显示loading提示，完成后自动关闭 |
| 权限不足 | 对应操作按钮不显示 |

## 关联功能

- **药房渠道管理**：药品管理的父级页面
- **处方开具**：医生开方时搜索和选择药品
- **标准药品库**：平台级的标准药品数据源
- **药品库存管理**：药房药品的库存数量管理

## 设计要点

1. **类型切换动态展示**：中药和成药的字段差异自动适配，切换选项卡时搜索条件、列表列、操作权限全部联动变化
2. **翻页保护**：删除最后一条导致当前页为空时自动回退
3. **导入结果明细**：Excel导入失败时展示每条记录的失败原因，方便运营人员修正
4. **缓存策略**：从编辑页返回列表页时保持搜索条件不变
