# 角色权限管理流程

> 文档版本：2026-07-01 | 面向读者：管理员、运营人员

## 概述

管理员在后台创建不同的操作角色（如超级管理员、问诊管理员、药师等），为每个角色分配对应的后台菜单权限（功能权限）和数据权限，然后将角色授予系统用户，从而精确控制每个人在后台能看什么、能做什么。

## 触发条件

- **前置条件**：拥有 `sys:roles:write` 权限的管理员
- **入口路径**：系统管理 → 角色管理

## 步骤详解

### 第一步：查看角色列表

进入角色管理页面后，顶部有两个选项卡：「互联网医院」和「线下药房」。这两个选项卡代表两个独立的后台系统（管理后台 vs 药房后台），各自的角色和权限树是分开管理的。

列表展示每个角色的信息：ID、角色名称、角色编码、角色说明、状态（启用/禁用）、创建时间。其中系统内置的角色会标注「系统角色」蓝色标签，这类角色不可编辑和删除。

搜索区可以按角色名称或角色状态筛选。

### 第二步：新增角色

点击「新增」按钮，弹出新增对话框。需要填写：

- **角色名称**（3~20个字符，必填）：如"问诊管理员""药师审核员"等
- **角色编码**（必填，最长255字符）：编程标识，如 `inquiry_mgr`
- **角色说明**（可选，最长100字符）：描述该角色能做什么
- **状态**（启用/禁用）

提交时系统会检查角色名称是否重复（失焦时自动校验），如果名称已存在会提示"角色名已存在，请重新输入"。

### 第三步：设置角色权限

在角色列表中点击某个角色的「权限设置」按钮，弹出权限设置对话框。有两个选项卡：

**功能权限**（最重要）：
- 左侧是完整的后台菜单树（按模块分组：医院管理、问诊治疗、运营管理、财务管理、系统管理等）
- 勾选某个菜单节点，该角色就能看到和使用对应功能
- 父子节点联动：勾选父节点会自动勾选所有子节点
- 右侧表格实时显示已勾选的功能名称列表
- 保存时需确认"确认提交吗？"

**数据权限**（占位，当前版本未实现具体功能）

### 第四步：启用/禁用角色

列表中的「角色状态」列有一个开关按钮，点击可以即时切换启用/禁用。禁用的角色授予用户后，该用户的相关权限将失效。

### 第五步：编辑和删除角色

- **编辑**：点击「编辑」按钮修改角色名称、编码、说明或状态。系统角色（builtIn=true）不可编辑。
- **删除**：单条删除或批量删除。删除前系统会检查该角色是否已被授予用户，如果已被使用则提示"数据已被使用，不能删除"并阻止操作。批量删除时如果选中了系统角色，会提示"勾选中存在不可删除的角色"。

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 角色名称重复 | 失焦时自动校验，提示"角色名已存在，请重新输入" |
| 角色名不足3个字符 | 表单校验提示"长度在 3 到 20 个字符" |
| 删除系统角色 | 编辑和删除按钮对系统角色（builtIn）灰显不可用 |
| 删除正在使用的角色 | 先调用 deleteCheck 检查，提示"数据已被使用，不能删除" |
| 保存权限时未选择任何菜单 | 提示"请至少选择一条数据" |
| 切换互联网医院/药房选项卡 | 重新加载对应系统的菜单树，之前勾选的权限会被清除 |

## 关联功能

- **用户管理**：将角色授予具体用户
- **菜单管理**：管理后台的菜单树结构
- **权限标识**：菜单权限标识如 `sys:roles:write` 控制操作按钮的显隐
