# 不良事件上报与管理流程

> 文档版本：2026-07-01 | 面向读者：运营管理员、医疗安全管理员

## 概述

管理员通过后台记录和管理平台诊疗服务过程中发生的不良事件（医疗相关或服务相关的负面事件），支持新增、编辑、查看详情和多维度筛选。

## 触发条件

- **前置条件**：登录管理后台，有不良事件管理权限
- **入口路径**：运营管理 → 不良事件管理

## 步骤详解

### 第一步：查看不良事件列表

进入页面后，系统自动加载全部不良事件记录，每页显示10条。列表展示以下信息：

| 列名 | 说明 |
|------|------|
| 事件编号 | 系统自动生成的唯一编号 |
| 事件业务类型 | 从字典中配置的分类（字典编码：sjywlx） |
| 就诊人姓名 | 事件涉及的患者姓名 |
| 证件类型 | 固定为「身份证」 |
| 证件号码 | 患者的身份证号码 |
| 手机号 | 患者联系电话 |
| 上报科室 | 上报该事件的医生所属科室 |
| 上报人 | 上报该事件的医生姓名 |
| 上报时间 | 事件录入系统的时间 |
| 最后修改时间 | 最近一次编辑的时间 |

### 第二步：筛选条件

页面提供三个筛选条件：
- **事件业务类型**：下拉选择（如：药品不良反应、医疗器械事件等）
- **就诊人姓名**：输入患者姓名搜索，最多5个字
- **上报时间**：选择日期范围

### 第三步：新增不良事件

点击「新增」按钮，打开表单弹窗。表单分为三部分，共16个字段（均为必填）：

**基本信息区（前5项）：**
- 事件业务类型：下拉选择（来自字典sjywlx）
- 事件发生时间：日期时间选择器（不能超过当前时间）
- 上报医生：从医生列表中搜索选择
- 上报时间：日期时间选择器
- 就诊人：从患者列表中搜索选择，选中后自动带出手机号（通过解密接口获取明文）

**分类信息区（中4项）：**
- 事件标题：文本输入，最多50字
- 事件类型：下拉选择（来自字典eventType）
- 事件等级：下拉选择（来自字典eventLevel）
- 事件影响级别：下拉选择（来自字典eventImpactLevel）

**详情描述区（后7项）：**
- 事件描述：文本域，最多3000字
- 事件发生的主要原因：文本域，最多3000字
- 采取的措施：文本域，最多1000字
- 造成的不良影响：文本域，最多1000字
- 解决措施：文本域，最多3000字
- 事件发生地址：文本域，最多100字

点击「保存」后，系统进行前端必填校验，通过后提交到后端创建记录。

### 第四步：编辑不良事件

点击列表中的「编辑」按钮，打开表单弹窗并回填已有数据（包括就诊人缓存的手机号），修改后点击保存即可更新。

### 第五步：查看详情

点击「详情」按钮，在独立弹窗中查看事件的完整信息。

## 异常处理

| 情况 | 系统处理方式 |
|------|------------|
| 就诊人手机号校验 | 新增时自动带出患者手机号；编辑时若手机号与缓存一致则跳过格式校验（避免已保存的数据因解密问题而提示错误），不一致则校验11位手机号格式 |
| 事件发生时间超过当前 | 日期选择器中未来的时间不可选，防止录入未发生的事件 |
| 就诊人无绑定手机号 | 如果患者未绑定手机号或会员，手机号字段自动清空，需手动输入 |
| 必填字段为空 | 表单有16个必填校验项，任一为空点击保存时会在对应字段下方显示红色提示 |

## 关联功能

- **患者管理**：不良事件关联了具体的就诊人，可从患者详情页进入查看该患者的所有不良事件
- **医生管理**：不良事件关联了上报医生，可追溯到具体医生的诊疗记录
