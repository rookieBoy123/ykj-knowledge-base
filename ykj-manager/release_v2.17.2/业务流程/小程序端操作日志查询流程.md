# 小程序端操作日志查询流程

> 文档版本 2026-07-03 | 面向运维人员、系统管理员

## 概述

小程序端操作日志记录了所有小程序用户的操作行为。与后台管理员的"操作日志"不同，这里的日志专门记录从小程序（App端）发起的操作请求，帮助运营团队了解用户行为、排查用户遇到的错误、以及进行用户行为分析。

## 触发条件

需要查看小程序用户的操作记录、排查用户反馈的问题、或分析用户行为模式时使用。

- **前置条件**：拥有操作日志查询权限
- **入口路径**：后台管理系统 → 系统管理 → 小程序日志

## 步骤详解

### 第一步：进入小程序日志列表

页面自动加载小程序端的操作记录。列表结构与"操作日志"一致，展示用户名、功能模块、操作内容、操作时间、操作结果。

**与后台操作日志的关键区别**：本页面的数据已经自动过滤，只展示来自小程序客户端（App端）的操作——不会混入后台管理员的内部操作。

### 第二步：筛选定位目标操作

支持四种筛选条件：

1. **用户名**：输入小程序用户的账号进行搜索
2. **功能模块**：输入模块名称，支持模糊匹配
3. **操作时间**：选择日期范围
4. **操作内容**：输入关键词搜索操作描述

### 第三步：查看操作详情

点击「查看详情」查看完整的操作信息，包括操作人、IP地址、客户端信息、请求URI、请求参数、成功/失败状态、错误原因和请求耗时。

## 典型使用场景

| 场景 | 操作方式 |
|---|---|
| 用户反馈"提交订单时报错" | 按用户名+操作时间筛选，定位该用户当时的所有操作记录，查找失败的记录并查看错误信息 |
| 分析某功能模块的使用高峰 | 按功能模块+时间范围筛选，观察操作频率和成功率 |
| 排查异常IP访问 | 查看详情中的IP地址和客户端信息，判断是否为正常用户操作 |

## 异常处理

| 情况 | 系统处理方式 |
|---|---|
| 匿名用户 | 与后台操作日志相同，无法获取用户名时显示"匿名用户" |
| 操作失败 | 红色"失败"标签，详情中显示具体错误信息 |
| 查询无结果 | 列表为空——可能是日期范围内无操作记录，或筛选条件过于严格 |

## 关联功能

- **小程序用户登录日志**：交叉验证用户的登录时间和操作时间线
- **后台操作日志**：运维人员自己的操作与用户操作分开管理，互不干扰
