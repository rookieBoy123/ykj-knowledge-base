# 字典管理流程

> 文档版本 2026-07-02 | 面向读者：产品、运营

## 概述

字典管理是系统的基础配置模块，管理员在此维护整个平台使用的下拉选项数据（如费用类别、疾病分类、服务类型、药品单位等）。所有字典数据以"分类+内容"的树形结构组织，前端页面和小程序的下拉选项均依赖此数据。

## 触发条件

- 管理员在后台左侧菜单点击"字典管理"
- 需要新增/修改某个下拉选项的内容

## 步骤详解

### 第一步：浏览字典分类树

页面采用左右分栏布局：

- **左侧**（30%宽度）：树形目录，展示所有字典分类。二级分类右侧以红色小字显示标准编码。
- **搜索**：顶部搜索框支持按分类名称模糊搜索。
- **右侧**（70%宽度）：点击叶子节点后展示该分类下的字典内容列表。

### 第二步：管理字典分类

点击分类树上的节点可执行：

- **新增分类**（需 sys:dict:write 权限）：弹出弹窗，填写：
  - 名称（必填，最多255字）
  - 编码（必填，最多255字）
  - 父级分类（可选，选择上级分类）
  - 标准编码（必填，最多255字）
- **编辑分类**：弹出弹窗，编码字段为只读（不可修改），其余可编辑
- **删除分类**：仅叶子节点可删除（非叶子节点提示"只能删除叶子节点！"），需二次确认

分类新增时自动校验：标准编码和分类编码的联合唯一性，重复时提示"该编码及标准编码已存在目录中，请重新输入"。

### 第三步：管理字典内容

点击叶子分类节点后，右侧展示该分类下的字典内容列表（编码、值、操作三列）。支持：

- **关键字过滤**：列表顶部搜索框按内容名称前端过滤（最多255字）
- **新增字典项**（需 sys:dict:write 权限）：弹出弹窗，填写：
  - 名称（必填，最多255字）
  - 编码（必填，最多255字）
  - 所属分类（自动带入，只读）
- **编辑字典项**：修改名称和编码
- **删除字典项**：单条删除或批量删除，需二次确认

新增字典项时自动校验：同分类下编码唯一性，重复时提示"编码已存在，请重新输入"。

### 第四步：保存与刷新

- 分类新增/编辑保存后 → 自动刷新分类树 + 清空右侧内容区域
- 字典项新增/编辑保存后 → 仅刷新右侧内容列表
- 批量删除：逐条调用删除接口，最后一条删除成功后才提示

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 删除非叶子节点分类 | 提示"只能删除叶子节点！" |
| 编码已存在（分类） | 提示"该编码及标准编码已存在目录中，请重新输入" |
| 编码已存在（字典项） | 提示"编码已存在，请重新输入" |
| 未选择数据就批量删除 | 提示"请至少选择一条数据" |
| 关闭弹窗（未保存） | 弹窗的"关闭前确认"提示"确认关闭？" |

## 关联功能

- 收费项目管理（费用类别字典 fylb）
- 服务项目管理（服务类型字典 fwlx）
- 不良事件管理（业务类型/事件类型/事件等级字典）
- 反馈管理（反馈类型字典 fklx）
- 文章管理（文章类型字典 ysnrfl）
- 疾病知识管理（疾病分类字典 jbzs-jbfl）
- 药品管理（药品单位字典 ypdw、mcyl 等）
- 小程序端所有下拉选择
