# 管理员登录记录查询流程

> 文档版本 2026-07-03 | 面向运维人员、系统管理员

## 概述

管理员登录日志用于记录每一位后台管理员登录系统的历史。每次登录都会记录登录账号、登录IP、操作系统、浏览器类型、登录方式、登录时间和成功/失败状态。这是一个重要的安全审计工具，可帮助发现异常登录行为。

## 触发条件

安全审计、排查账号异常、或追溯某管理员的登录时间线时使用。

- **前置条件**：拥有登录日志查询权限
- **入口路径**：后台管理系统 → 系统管理 → 管理员登录日志

## 步骤详解

### 第一步：进入登录日志列表

页面自动加载最近的登录记录，默认每页10条。列表展示：

| 列 | 说明 |
|---|---|
| ID | 日志记录的唯一编号 |
| 用户账号 | 登录的管理员账号 |
| 登录IP | 登录时的网络IP地址 |
| 操作系统 | 自动解析浏览器UA信息，如 Windows、macOS |
| 浏览器 | 自动解析，如 Chrome、Edge |
| 登录方式 | 账号密码登录、短信验证码登录等 |
| 登录状态 | 正常（蓝色标签）/ 异常（红色标签）|
| 操作结果 | 成功 / 失败 |
| 登录时间 | 登录发生的具体时间 |

### 第二步：筛选登录记录

支持三种筛选条件：

1. **用户搜索**：输入管理员账号进行搜索
2. **登录状态**：下拉选择「全部」「正常」「失败」
3. **时间选择**：选择日期范围

### 第三步：分析登录信息

通过列表可以快速识别：

- **异常登录**：登录状态为"异常"或"失败"的记录，可能是密码错误或账号异常
- **异地登录**：通过登录IP识别不同地理位置的登录
- **异常客户端**：不常见的操作系统或浏览器组合

## 操作系统与浏览器识别

系统会自动从登录请求中解析出操作系统和浏览器信息并展示。不需要管理员手动输入——只要是正常的浏览器登录，系统就能自动识别。如果显示为空，可能是通过非浏览器方式（如API调用）登录。

## 异常处理

| 情况 | 系统处理方式 |
|---|---|
| 登录失败 | 登录状态显示"失败"，红色标签标记。常见原因包括密码错误、账号被禁用、验证码错误等 |
| 登录IP异常 | 系统记录实际的登录IP，管理员可自行判断是否为异常来源 |
| 查询无结果 | 可能该账号在筛选时间内没有登录记录 |

## 关联功能

- **操作日志**：通过登录时间可以关联到该管理员登录后的具体操作
- **小程序用户登录日志**：两个页面分别管理后台管理员和小程序用户的登录记录
