# 角色权限配置与管理流程

> 文档版本 v1.0 | 面向读者：后台系统管理员

## 概述

系统管理员可以创建和管理后台系统的角色，为每个角色分配不同的功能权限（菜单权限）。角色分为"互联网医院"和"线下药房"两个客户端，不同客户端的角色和菜单体系独立。内置角色不可编辑和删除。

## 触发条件

- **前置条件**：已登录管理后台，有角色管理权限（`sys:roles:write`）
- **入口路径**：后台菜单 → 系统管理 → 系统管理 → 角色管理

## 步骤详解

### 第一步：切换客户端

搜索区域顶部有Tab切换：
- **互联网医院**（mgmt）：管理后台端角色
- **线下药房**（phar）：药房端角色

默认选中"互联网医院"。切换后列表数据和菜单树都会重新加载对应客户端的数据。

### 第二步：搜索角色

筛选条件：
- **角色名称**：模糊搜索
- **状态**：启用/禁用

### 第三步：查看角色列表

表格展示：
- ID、角色名称、角色编码、角色说明
- 角色状态（开关：启用/禁用）
- 创建时间、最后修改时间
- 操作按钮：编辑、权限设置、删除

**内置角色**：编辑和删除按钮置灰不可用（`builtIn`标记）

### 第四步：新增/编辑角色

点击"新增"/"编辑"弹出表单：

**字段**：
- **角色名称**（必填，3-20字符）：失焦时自动校验是否已存在，存在则提示"角色名已存在，请重新输入"
- **角色编码**（必填）：角色的唯一标识码
- **角色说明**（选填，最多100字）：textarea输入
- **状态**（必填）：启用/禁用

编辑模式下，修改角色名称时同样校验重名。

### 第五步：配置角色权限

点击"权限设置"弹出权限配置对话框：

**左侧菜单树**：
- 以树形结构展示所有后台菜单（check-strictly模式，父子节点独立勾选）
- 每个节点前有三角箭头展开/折叠
- 勾选/取消勾选某个菜单后，右侧表格实时更新已选菜单列表

**右侧已选菜单列表**：
- 表格展示已勾选的功能名称和功能描述
- 保存时校验至少选择一条，否则提示"请至少选择一条数据"

**数据权限标签页**（预留）：
- 第二个标签页"数据权限"，目前显示"数据权限"文字占位

### 第六步：保存权限

点击"保存"按钮：
1. 校验至少选择了一条数据
2. 弹出确认框"确认提交吗？"
3. 将角色ID和选中的菜单ID列表提交保存
4. 提示"操作成功"并关闭弹窗

### 第七步：启用/禁用角色

列表中的状态开关：
- 切换后会调用`rolesDisable`或`rolesEnable`接口
- 操作成功后提示"操作成功"，列表自动刷新

### 第八步：批量删除

- 勾选多条角色记录
- 点击"批量删除"按钮执行批量删除

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 角色名称已存在（新增时） | 提示"角色名已存在，请重新输入" |
| 角色名称已存在（编辑时） | 仅当名称改变时才校验重名 |
| 保存权限时未选择菜单 | 提示"请至少选择一条数据" |
| 编辑内置角色 | 编辑和删除按钮不可用 |

## 关联功能

- **菜单管理**（`menu/index.vue`）：管理后台的菜单树结构，角色权限的基础
- **用户管理**（`user/index.vue`）：为用户分配角色
- **权限管理**（`perm/index.vue`）：细粒度的权限点配置
