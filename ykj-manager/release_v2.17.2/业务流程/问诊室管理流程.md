# 问诊室管理流程

> 文档版本：v2.17.2 | 面向读者：运营 / 客服

## 概述

管理员在后台以医生视角操作问诊室，包括接诊、结束问诊、写病历、开处方、发起视频通话等完整问诊工作流。

## 触发条件

- 管理员必须以「医生角色」登录后台
- 进入「问诊室」功能页面

## 步骤详解

### 第一步：选择问诊单

页面左侧为问诊单列表区（宽360px），展示所有分配给当前医生的问诊单。管理员点击任意一条问诊单，右侧区域展示问诊详情。

- **未选择任何问诊单时**：中间区域显示「选择患者查看详细信息」的空白提示
- **选择问诊单后**：中间区域加载聊天室，右侧加载问诊资料（问诊详情、病历记录、处方记录）

### 第二步：接诊

对于状态为「待接诊」的问诊单，管理员可以在聊天室顶部点击「接诊」按钮。

- 系统调用接诊接口
- 成功后显示「接诊成功」提示
- 1秒后左侧列表自动切换到「进行中」选项卡
- 1.5秒后自动刷新当前问诊详情

### 第三步：在线沟通（聊天室）

中间区域为聊天室，支持：
- 发送文字消息
- 发送图片
- 查看患者发来的病历资料
- 发起视频通话（向患者发送视频邀请卡片消息）
- 查看历史病历记录

### 第四步：结束问诊

在聊天室中点击「结束问诊」按钮，系统弹出二次确认弹窗「确定结束问诊吗？」。

- 确认后调用结束接口
- 成功后显示「问诊结束!」提示
- 左侧列表自动切换到「已结束」选项卡

### 第五步：写病历

在聊天室工具栏中点击「写病历」按钮，右侧区域切换为病历编辑界面。

管理员可在此：
- 填写患者主诉、现病史、既往史、过敏史等
- 选择中西医诊断
- 上传病历图片资料
- 提交后显示「病历保存成功」，右侧区域恢复为问诊资料视图
- 如果已有病历，点击「更新病历」则会更新而非新建

### 第六步：开处方（医嘱）

在聊天室中点击「开处方」按钮：

- **已有病历时**：系统自动加载病历的诊断信息，打开处方编辑对话框。管理员可选择药品、设置用法用量、签章后提交。
- **没有病历时**：弹出提示「请先写病历再开医嘱!」，只有一个「知道了」按钮。这是因为处方必须先有诊断依据，系统强制先完成病历再开方。

### 第七步：发起视频通话

在聊天室中点击「视频通话」按钮：

1. 系统请求获取实时音视频（TRTC）房间号
2. 向患者发送视频邀请卡片消息（显示"某医生正在邀请您进行视频问诊，点击按钮加入"）
3. 获取TRTC用户签名参数
4. 打开视频通话房间（全屏覆盖）
5. 通话结束后关闭房间

## 右侧问诊资料区

右侧面板（宽360px）包含三个选项卡：

| 选项卡 | 内容 |
|--------|------|
| 问诊详情 | 患者基本信息、问诊编号、服务类型、问诊时间、费用等 |
| 病历记录 | 已创建的病历列表，可查看病历详情 |
| 处方记录 | 已开具的处方列表，可查看处方详情和药品清单 |

## 异常处理

| 情况 | 系统处理方式 |
|------|------------|
| 非医生角色进入此页面 | 显示「请登录医生角色查看!」，隐藏所有问诊室功能 |
| 没有病历就点击开处方 | 提示弹窗「请先写病历再开医嘱!」，阻止操作 |
| 视频通话获取房间号失败 | 提示「响应错误，请联系管理人员」 |
| 结束问诊时用户取消 | 弹出「操作取消」提示 |

## 关联功能

- 左侧问诊单列表中的「问诊中」「待接诊」「已结束」等选项卡切换
- 与「处方管理」联动（开方后处方记录自动出现在右侧面板）
- 与「病历管理」联动（创建病历后可查看和更新）
- 与「视频通话记录」联动（视频回放可在门诊问诊订单中查看）
