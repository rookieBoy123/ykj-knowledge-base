# 调度任务配置与管理流程

> 文档版本：2026-07-03 第5轮 | 面向读者：系统管理员、运维人员

## 概述

调度任务管理功能用于配置和管理系统中的各类自动化定时任务（如定时发送随访消息、定时同步数据等）。管理员可以查看所有任务、编辑任务执行周期、手动触发任务、启用/禁用任务，以及查看任务执行日志。

## 触发条件

- **前置条件**：需要系统管理员角色
- **入口路径**：侧边栏 → 系统管理 → 调度任务 → 定时任务

## 步骤详解

### 第一步：任务列表查看

进入页面后展示所有调度任务列表，每列信息如下：

| 列名 | 说明 |
|------|------|
| 任务类型 | "定时任务"或"一次性任务"（两种类型的显示值相同，均为下拉中的"定时任务"选项值） |
| 任务名称 | 任务的标识名称 |
| 任务描述 | 该任务的功能说明文字 |
| cron表达式 | 任务的执行时间规则（如"0 0 8 * * ?"表示每天8点） |
| 状态 | 启用/禁用开关，可直接在列表中切换 |
| 操作 | 编辑、触发、日志 |

### 第二步：搜索筛选

支持三种查询条件组合：
- **任务类型**：下拉选择（全部/定时任务/一次性任务）
- **任务名称**：输入任务名称搜索，最多50字
- **任务描述**：输入描述关键词搜索，最多255字

此外，从其他页面跳转过来时（如从"任务日志"页携带jobId参数），页面会自动根据传入的参数进行筛选。

### 第三步：编辑任务

点击某条任务的"编辑"按钮，弹出编辑窗口（宽度600px），可修改：

1. **cron表达式**（必填）：最多64字，用于定义任务执行时间规则。例如"0 */2 * * * ?"表示每2分钟执行一次
2. **任务描述**（必填）：最多100字，说明该任务的用途

注意：一次性任务不能编辑（编辑按钮置灰禁用），因为一次性任务执行完即结束，修改其配置没有意义。

### 第四步：手动触发任务

点击"触发"按钮，弹出二次确认："触发后定时任务将立即执行，是否触发任务？"。确认后任务立即执行一次，不受cron表达式的时间规则限制。

同样，一次性任务不支持触发操作。

### 第五步：启用/禁用

每条任务行内有一个开关按钮，可直接切换任务的启用/禁用状态：
- **启用**（开关打开/绿色）：任务按cron表达式正常运行
- **禁用**（开关关闭/灰色）：任务暂停执行，但配置保留

切换后系统提示"修改成功"。

### 第六步：查看执行日志

点击"日志"按钮跳转到执行日志页面（scheduler/logs），并自动携带当前任务的任务ID（jobId），日志页会根据该ID筛选展示该任务的所有历史执行记录。

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| cron表达式为空 | 提示"请输入cron表达式"，阻止提交 |
| 任务描述为空 | 提示"请输入任务描述"，阻止提交 |
| 一次性任务点击编辑 | 编辑按钮置灰，不可点击 |
| 一次性任务点击触发 | 触发按钮置灰，不可点击 |
| 触发操作确认 | 弹出警告框，需用户确认后才执行 |
| 任务状态切换 | 即时生效，无需二次确认，提示"修改成功" |
| 列表翻页到无数据页 | 自动回退到上一页 |

## 关联功能

- 任务执行日志（scheduler/logs）：查看每个任务的执行记录、执行结果、报错信息
- 系统配置：部分业务的开关配置可能影响任务的执行逻辑
