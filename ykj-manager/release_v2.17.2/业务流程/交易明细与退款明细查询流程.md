# 交易明细与退款明细查询流程

> 文档版本 2026-07-02 | 面向读者：产品经理、运营、客服

## 概述

描述管理员在后台财务模块查看所有交易流水（支付明细和退款明细）的功能。支持多维度筛选、汇总统计和数据导出。

## 触发条件

- **管理员**：需要查看平台的收入和支出流水，对账、统计、导出报表

## 步骤详解

### 第一步：切换Tab筛选维度

页面顶部有两个标签页切换：

1. **支付明细**：展示所有收入类交易流水（用户支付成功的订单）
2. **退款明细**：展示所有支出类交易流水（平台退款给用户）

默认进入页面时展示"支付明细"，可通过URL参数 `type` 指定默认Tab（type=0为支付、type=1为退款）。

### 第二步：筛选条件

支持的筛选维度：

1. **交易日期**：日期范围选择器，默认展示最近30天（当天往前推30天）
2. **订单编号**：支持输入最多30个字符
3. **交易流水号**：支持输入最多30个字符
4. **订单类型**：从字典（类别编码 ddlx）动态加载所有订单类型
5. **支付渠道**：微信支付 / 医保统筹（需开启医保支付） / 医保个账（需开启医保支付） / 医保支付-微信支付（需开启医保支付） / 线下支付
6. **医生**：支持远程搜索（输入医生姓名模糊查询，下拉选择）

支付渠道中的医保相关选项仅在系统开启了医保支付功能时展示（通过 `medicalInsurancePay()` 判断）。

### 第三步：查看汇总统计

表格上方展示该筛选条件下的金额汇总：

- **合计收入/支出**：总金额
- **微信支付**：微信渠道金额
- **医保统筹**（仅开启医保时显示）
- **医保个账**（仅开启医保时显示）
- **医保支付-微信支付**（仅开启医保时显示）
- **线下支付**：线下支付金额
- **退款合计**：退款的金额

所有金额保留两位小数展示。

### 第四步：列表查看

根据Tab类型展示不同的列：

**支付明细列**：序号 → 交易流水号 → 订单编号 → 订单类型 → 医生 → 支付渠道 → 交易金额 → 订单支付时间

**退款明细列**：序号 → 交易流水号 → 订单编号 → 订单类型 → 医生 → 退款渠道 → 退款金额 → 退款完成时间

每行有"查看详情"操作按钮，可跳转到对应订单详情页。

### 第五步：数据导出

点击"导出"按钮，需要 `order:flows:export` 权限：

1. 以当前筛选条件为参数，调用导出接口
2. 导出过程中显示 loading 提示"加载中"
3. 导出为 Excel 文件并自动下载

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 无导出权限 | 导出按钮不显示（sysPerm控制） |
| 导出中再次点击 | loading提示"加载中"，不可重复操作 |
| 医保功能关闭 | 不展示医保支付渠道选项和医保金额统计 |

## 关联功能

- 财务看板（宏观财务数据展示）
- 订单管理（查看订单详情）
- 医生管理（医生搜索下拉数据源）
- 字典管理（订单类型 ddlx 字典）
