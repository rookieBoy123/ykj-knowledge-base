# 远程会诊申请与审核管理流程

> 文档版本：2026-07-02 | 面向读者：运营人员、医疗机构管理员

## 概述

远程会诊是医院与外部医疗机构之间的跨院协作功能。本院医生为患者发起会诊申请，选择拟邀请的会诊机构、科室和医生，受邀机构应答后安排会诊时间，双方通过视频/音频完成会诊并出具报告。

管理后台承担两个角色视角：**申请方（远程医疗管理员）**可新增、编辑、撤回、取消会诊申请；**受邀方（受邀机构管理员）**可查看待应答的会诊请求并做出应答或拒绝。

## 触发条件

- 入口：管理后台 → 诊疗管理 → 远程医疗中心
- 权限：远程医疗管理员角色 / 受邀机构管理员角色

## 步骤详解

### 第一步：申请方创建会诊申请（远程医疗管理）

远程医疗管理员在「远程会诊中心」页面点击「新增」按钮，填写会诊信息：
- 申请科室、主治医师
- 患者姓名、性别、年龄
- 拟申请会诊机构、会诊服务、会诊科室、会诊医生
- 拟申请会诊时间
- 会诊需求描述、病历资料上传（支持图片/PDF/Word/Excel，单文件不超过 10MB）

可保存为草稿或直接提交申请。提交后状态变为「待应答」。

### 第二步：受邀方处理会诊请求（受邀机构管理员）

受邀机构管理员在「会诊列表」页查看分配给本机构的待处理请求。针对每条请求可执行：

- **接受会诊**：选择会诊时间、分配会诊医生、填写应答备注
- **拒绝会诊**：必须填写拒绝原因（500 字以内）
- 还需要补充资料时，可要求申请方补充资料

### 第三步：申请方跟进处理

申请方在管理后台可针对不同状态的会诊执行不同操作：

| 会诊状态 | 可执行操作 |
|---------|-----------|
| 草稿 | 编辑、删除 |
| 待应答 | 撤回 |
| 已撤回 | 编辑、删除 |
| 已拒绝 | 编辑、取消 |
| 待收费 | 收费、取消 |
| 待开始 | 开始会诊、取消 |
| 进行中 | 进入会诊、结束会诊 |
| 已结束 | 查看详情 |
| 已归档 | 查看详情 |
| 已取消 | 查看详情 |

### 第四步：收费与开始会诊

受邀方接受后会诊状态变为「待收费」。申请方管理员完成收费后状态变为「待开始」。管理员点击「开始会诊」进入会诊室（视频/音频通讯）。

### 第五步：结束与归档

会诊结束后，状态变为「已结束」。受邀方医生可撰写会诊报告。后续系统自动将已完成较长时间的会诊归档到「已归档」状态。

### 第六步：取消会诊

在「已拒绝」「待收费」「待开始」三种状态下都可以取消会诊。取消时必须填写取消原因（最多 500 字）。

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 非受邀机构管理员访问 | 页面显示"当前页面仅供受邀机构管理员使用" |
| 非远程医疗管理员访问 | 页面显示"当前页面仅供远程医疗管理员使用" |
| 取消原因超限 | 前端限制最多 500 字 |
| 补充资料文件格式不符 | 仅支持 jpg/png/jpeg/pdf/doc/docx/xls/xlsx |
| 补充资料文件过大 | 超过 10MB 拒绝上传 |

## 状态流转

```
草稿 → 待应答 ↔ 已拒绝
待应答 → 已撤回 → 草稿
待应答 → 待收费 → 待开始 → 进行中 → 已结束 → 已归档
待收费/待开始/已拒绝 → 已取消
```

## 关联功能

- 会诊报告撰写
- PDF 报告生成与签章
- 会诊室视频/音频通讯（RTC 实时通信）
- 费用结算与收费管理
- 订阅消息通知
