# 平台交易收款与退款对账流程

> 文档版本 v1.0 | 面向读者：财务管理员

## 概述
平台收款与退款对账功能提供了完整的资金流水查询和统计能力。管理员可查看所有订单的收款明细和退款明细，按支付渠道（微信支付、医保统筹、医保个账、医保微信支付、线下支付）分类汇总统计金额，支持按交易日期、订单编号、交易流水号、订单类型、支付渠道、医生等条件筛选，并可导出报表。

## 触发条件
- **前置条件**：拥有订单流水查看权限的财务管理员
- **入口路径**：财务管理 → 收款清单

## 步骤详解

### 第一步：选择查询标签页
页面顶部提供两个标签页切换：
- **支付明细**：查看所有收款记录
- **退款明细**：查看所有退款记录

也可通过URL参数直接跳转到指定标签页。

### 第二步：设置查询条件
支持以下查询条件：

| 条件项 | 说明 |
|--------|------|
| 交易日期 | 日期范围选择，默认最近30天 |
| 订单编号 | 输入订单编号精确查询 |
| 交易流水号 | 输入微信/医保支付流水号查询 |
| 订单类型 | 下拉选择（药品订单/问诊订单/会诊订单/预约订单），从字典动态加载 |
| 支付渠道 | 全部/微信支付/医保统筹/医保个账/医保支付-微信支付/线下支付 |
| 医生 | 远程搜索选择医生姓名 |

**支付渠道显示规则**：如果医院未开启医保支付功能，支付渠道下拉框中不显示医保相关选项（医保统筹、医保个账、医保支付-微信支付）。

### 第三步：查看汇总统计
列表顶部展示统计汇总栏，根据当前标签页显示：
- **支付明细模式**：合计收入、微信支付金额、医保统筹金额、医保个账金额、医保微信支付金额、线下支付金额、退款合计
- **退款明细模式**：合计支出及各类退款渠道金额

金额以元为单位，保留两位小数。

### 第四步：查看列表详情
列表展示每条交易记录：

**支付明细列表**：
- 序号、交易流水号、订单编号、订单类型、医生、支付渠道、交易金额、订单支付时间
- 每行有"查看详情"按钮

**退款明细列表**：
- 序号、交易流水号、订单编号、订单类型、医生、退款渠道、退款金额、退款完成时间

### 第五步：导出报表
- 点击"导出"按钮，将当前筛选条件下的数据导出为Excel文件
- 导出过程中显示"加载中"提示，导出完成后自动下载
- 需要拥有"订单流水导出"权限

## 异常处理
| 情况 | 系统处理方式 |
|------|-------------|
| 未开启医保支付 | 统计栏和支付渠道筛选中不显示医保相关项 |
| 查询无结果 | 列表显示空，统计数据全部为0 |
| 翻到最后一页数据为空 | 自动回退到上一页 |
| 医生远程搜索无结果 | 医生下拉框显示空列表 |
| 导出权限不足 | "导出"按钮不显示 |
| 缺少查询权限 | 接口可能返回403 |

## 关联功能
- **订单详情**：点击"查看详情"可跳转到对应订单的详情页
- **退款审核**：退款明细中的数据来源与退款审核模块关联
- **财务看板**：统计数据与财务看板中的收支汇总相关联
