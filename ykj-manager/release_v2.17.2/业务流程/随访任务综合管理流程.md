# 随访任务综合管理流程

> 文档版本 2026-07-02 | 面向读者：运营管理人员

## 概述

随访任务管理是后台运营人员监控医生发起的患者随访计划执行情况的管理功能。管理人员可以按多种条件筛选随访任务、查看四类状态分布、查阅任务详情（包含患者信息、医生信息、随访内容完成情况）。

## 触发条件

- **前置条件**：已登录管理后台，拥有随访任务查看权限（fup:tasks:read）
- **入口路径**：后台左侧菜单 → 诊疗管理 → 随访管理 → 随访任务

## 步骤详解

### 第一步：筛选与查询

页面顶部提供六维筛选条件：

| 筛选条件 | 类型 | 说明 |
|---------|------|------|
| 创建时间 | 日期范围 | 按任务创建时间筛选 |
| 随访日期 | 日期范围 | 默认显示当月（月初至今天），按计划随访发送日期筛选 |
| 创建人 | 远程搜索下拉 | 输入医生姓名模糊搜索，最多返回500条 |
| 患者姓名 | 文本输入 | 精确匹配，最多5个字符 |
| 手机号码 | 文本输入 | 精确匹配，最多11位数字 |
| 随访状态 | Tab切换 | 全部/待随访(0)/已随访(1)/已完成(2)/已取消(3) |

"随访日期"默认自动填充当月1号至今天，方便快速查看近期任务。

### 第二步：查看任务列表

列表展示以下信息：
- **患者**：患者姓名
- **手机号**：患者手机号
- **随访任务**：该任务包含的随访内容标题（多个以逗号分隔），如"术后康复指导,满意度问卷"
- **随访状态**：待随访/已随访/已完成/已取消
- **创建人**：发起随访的医生姓名
- **创建时间**：任务创建时间
- **随访日期**：计划发送日期
- **操作**：查看详情按钮

列表支持分页（每页10条），可跳页。

### 第三步：查看任务详情

点击"查看详情"弹出详情弹窗，展示：

**患者信息**：
- 患者姓名、年龄、性别、手机号、头像

**医生信息**：
- 医生姓名、职称、科室、头像

**任务基本信息**：
- 创建人、创建时间、随访日期

**随访内容明细**（任务包含的子项）：

| 内容类型 | 详情展示 |
|---------|---------|
| 患教内容（类型0） | 显示患教文章的详细内容 |
| 问卷内容（类型2） | 显示问卷题目、选项、患者作答答案（单选/多选/填空/图片上传），患者上传的图片通过OSS链接展示 |
| 提醒内容（类型1） | 显示提醒的基本信息 |

**内容禁用状态**：如果随访任务的发送时间还未到（大于当前时间），系统会检查关联的患教内容或问卷内容是否被禁用（status=Published），如果是，则标记为"禁用"状态。

**任务小结**：如果医生填写了小结，显示小结内容和填写时间。

### 第四步：随访状态流转

随访任务有四种状态，由系统自动流转：

```
待随访(0) → 已随访(1) → 已完成(2)
                        ↘ 已取消(3)
```

- **待随访**：任务已创建但到达随访日期前，患者尚未作答
- **已随访**：患者已完成部分或全部随访内容
- **已完成**：所有随访内容均已作答完成，且关联的随访计划下所有任务也全部完成 → 计划状态同步变为"已完成"
- **已取消**：任务被取消

## 异常处理

| 情况 | 系统处理方式 |
|------|------------|
| 无权限访问 | 接口返回403，页面无法加载数据 |
| 查询参数异常 | 返回空列表，不报错 |
| 内容ID不存在（查看详情时） | 后端返回错误"id有误" |
| 问卷已作答再次提交 | 后端返回错误"已答题，请勿重复作答" |
| 关联的患教/问卷被禁用 | 前端列表显示"禁用"标记（发送时间大于当前时间时触发） |

## 关联功能

- 关联**随访计划管理**：每个任务属于一个随访计划，计划完成状态联动更新
- 关联**患教内容管理**：随访任务引用的患教内容（course）
- 关联**问卷管理**：随访任务引用的问卷内容（questionnaire）
- 关联**患者管理**：任务关联的患者信息
