# 互联网机构信息管理流程

> 文档版本 2026-07-03 | 面向运营管理员

## 概述

管理互联网医院在监管平台上的机构备案信息，包括机构基本信息、实体医院信息和相关证照资料。这些信息将用于向省级监管平台上报。

## 触发条件

- 运营管理员登录管理后台
- 首次配置或需要更新互联网医院的机构备案信息
- 菜单路径：医院管理 → 互联网机构

## 步骤详解

### 第一步：查看当前机构信息

进入"互联网机构信息"页面后，系统自动加载当前已保存的机构信息。页面分为两个大区域：上半部分是"互联网医院信息"，下半部分是"实体医院信息"。

### 第二步：填写互联网医院信息

在"互联网医院信息"区域，需要填写以下信息：

- **机构标识**：互联网医院的唯一标识编码（必填）
- **机构名称**：互联网医院的全称（必填）
- **单位隶属关系**：从下拉列表中选择（必填）
- **信息安全等级保护证书编号**：网安备案相关证书编号
- **机构类型**：从下拉列表中选择（必填）
- **机构成立日期**：选择互联网医院的成立日期（必填）
- **机构分类**：从下拉列表中选择（必填）
- **经济类型**：如国有、民营等（必填）
- **信息安全等级保护**：选择安全等级（必填）
- **机构分类管理类别**：从下拉列表中选择（必填）
- **是否分支机构**：选择"是"或"否"（必填）
- **机构所在地民族自治地方标志**：选择是否在民族自治地区（必填）
- **部署平台**：选择系统部署的平台类型（必填）
- **是否具备双路供电或紧急发电设施**：选择是或否（必填）
- **地址**：先通过省市区级联选择器选择地区，再输入详细地址（必填）
- **互联网医院网址**：输入网址（必填）
- **第二机构名称**、**邮政编码**、**E-mail**、**电话号码**、**机房面积**：补充信息
- **第三方机构统一社会信用代码**、**第三机构名称**：合作方信息
- **协议有效期**：选择合作期限
- **机构描述**：对机构的文字描述（最长5000字）
- **备注**：补充说明

### 第三步：填写实体医院信息

在"实体医院信息"区域，填写依托的实体医疗机构信息：

- **统一社会信用代码**：实体医院的信用代码（必填）
- **实体医院名称**：实体医院全称（必填）
- **许可证号码**：医疗机构执业许可证号（必填）
- **实体医院机构等级**：如三甲、二甲等（必填）
- **实体医疗机构级别**：从下拉列表选择（必填）
- **开办资金金额数**：以万元为单位（必填）
- **法人代表/负责人**：姓名（必填）
- **许可项目名称**：许可证上批准的项目（必填）
- **许可证有效期**：选择起始日期和结束日期（必填）

### 第四步：保存

填写完成后，点击页面底部的"保存"按钮。系统会校验所有必填项，校验通过后提交到服务端。保存成功后提示"保存成功"。

如果点击"重置"，系统会弹出确认提示，确认后恢复为上次保存的版本。

## 异常处理

| 情况 | 系统处理方式 |
|------|------------|
| 必填项未填写 | 点击保存时，对应输入框下方显示红色错误提示，无法提交 |
| 地址只填了详细地址但未选择省市区 | 提示"请选择地址" |
| 许可证有效期未选 | 提示"请选择日期" |
| 重置操作误触 | 弹出"是否确认重置?"确认框，可取消 |

## 关联功能

- **医院配置**（hospitalConfig）：互联网医院基础参数配置
- **医院设置**（hospitalSet）：医院业务参数设置
- **字典管理**：管理机构类型、等级等下拉选项的数据来源
