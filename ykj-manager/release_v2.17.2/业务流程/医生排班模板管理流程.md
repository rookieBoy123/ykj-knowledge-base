# 医生排班模板管理流程

> 文档版本 | 面向读者：管理员

## 概述

管理员可以在后台为医生创建排班模板，设定医生每周的出诊班次（上午/下午/晚上），系统会根据模板自动生成未来日期的排班计划，减少重复操作。同时支持手动管理单日排班和号源控制。

## 触发条件

- **前置条件**：管理员拥有排班管理权限
- **入口路径**：管理后台 → 医院管理 → 医生管理 → 排班模板/排班管理

## 步骤详解

### 第一步：查看排班模板列表

- 列表展示所有已创建的排班模板，每条记录显示医生姓名、科室、模板生效日期范围
- 支持按医生姓名搜索
- 每页分页展示

### 第二步：创建排班模板

配置排班模板需要以下信息：

1. **选择医生**：从医生列表中搜索并选择目标医生
2. **设置号源总量**：该班次可预约的总人数
3. **设置出诊时段**：勾选上午/下午/晚上的出诊时间和对应号源数
4. **设定模板生效期**：开始日期和结束日期，模板在此范围内生成排班
5. **选择适用诊室**：关联具体诊室

### 第三步：模板自动生成排班

创建模板后，系统会按模板规则自动生成指定日期范围内的排班记录：

- 按模板的周几规律在日历中标记出诊日
- 每天生成对应的上午/下午/晚上班次
- 每个班次自动分配号源编号（scheduleItemNo），号源总量由模板设定
- **跳过已有排班的日期**：如果某天医生已有手动创建的排班，模板不会覆盖

### 第四步：停诊处理

管理员可以处理医生的临时停诊：

- **全天停诊**：一次性取消医生当天的所有班次，所有号段被释放
- **指定班次停诊**：仅取消某个班次（如仅取消下午班），其他时段不受影响
- 停诊需填写停诊原因（最多100字）
- 停诊会触发事件（ScheduleStopReceptionEvent），系统自动批量取消该号源的所有关联预约，并向受影响患者发送通知
- 已预约患者会收到停诊通知，了解预约被取消的原因

### 第五步：排班查询

- **管理后台端**：可查看所有排班记录（包括已发布和未发布），支持按日期/医生/科室筛选
- **小程序端**：仅显示已发布、已启用且未停诊的排班，确保患者看到的都是有效的可预约时段

## 号源控制

- 患者每预约一个号源，系统扣减对应号段的可用数量
- 扣减前进行二次校验：验证号源仍可用、未超时、未超限
- 取消预约时自动释放号源，恢复可用数量
- 并发预约场景下通过数据库锁保证不会超卖
- 预约截止时间：距离开诊时间不足30分钟时，该号源不可再预约

## 异常处理

| 情况 | 系统处理方式 |
|------|------------|
| 模板时间与已有排班冲突 | 提示冲突信息，需调整日期或先删除手动排班 |
| 停诊原因超过100字 | 前端截断或后端校验限制 |
| 批量停诊单条处理失败 | try-catch保护，单条失败不影响其他预约取消 |
| 号源已满 | 患者端提示"该时段已约满"，无法提交预约 |
| 距离开诊不足30分钟 | 该号源的预约入口关闭，患者无法预约 |
