# 电子发票查询与导出流程

> 文档版本：v1.0 | 面向读者：运营、财务

## 概述

管理员在后台查询平台所有已开具的电子发票，支持按患者姓名、订单编号、开票日期筛选，并可将查询结果导出为Excel文件进行财务对账。

## 触发条件

- **前置条件**：管理员已登录管理后台
- **入口**：左侧菜单 → 财务管理 → 发票查询

## 步骤详解

### 第一步：查询发票

- **搜索条件**：
  - 患者姓名：输入框搜索，最多5个字
  - 订单编号：输入框搜索，最多30个字符
  - 开票日期：日期范围选择器（开始日期 ~ 结束日期）
- **列表展示**：每页10条，展示以下字段：
  - 电子发票号码
  - 交易流水号
  - 订单编号
  - 订单类型（处方/问诊/会诊/预约）
  - 门诊号码（HIS系统的门诊号，无数据显示"/"）
  - 患者姓名（无数据显示"/"）
  - 支付渠道（微信/支付宝/医保等）
  - 交易金额（元）
  - 订单支付时间
  - 开票日期

### 第二步：打印发票

- **操作入口**：每行发票记录的「操作」列有「打印」按钮
- **显示条件**：仅当发票有图片地址时显示打印按钮（`pictureNetUrl` 非空）
- **操作效果**：点击后在新窗口打开发票图片，用户可自行打印

### 第三步：导出发票

- **导出按钮**：列表上方有「导出」按钮，不受行选中限制始终可用
- **导出逻辑**：使用当前搜索条件（排除分页参数），调用导出接口
- **文件格式**：导出为 `.xls` 格式的Excel文件
- **加载提示**：导出过程中显示"加载中"的旋转图标

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 发票无图片 | 「打印」按钮隐藏 |
| 患者姓名空 | 显示"/" |
| 门诊号为空 | 显示"/" |
| 导出失败 | 关闭加载提示，静默处理 |

## 关联功能

- **发票管理流程**（小程序患者端）：患者查看和申请发票
- **服务预约流程**：预约核销时的自动开票
