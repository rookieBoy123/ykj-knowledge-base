# 药品运费多地区配置管理流程

> 文档版本 2026-07-02 | 面向读者：运营管理人员

## 概述

运费配置管理允许运营人员为不同地区设置差异化的运费规则。支持多个运费条件（每个条件对应一组地区），每个条件可独立设置运费金额、满额包邮阈值、是否支持到付。最后一条固定为"其他未选择地区"的兜底规则。医生开处方时可选择是否免除运费（需开启"免除运费"开关）。

## 触发条件

- **前置条件**：已登录管理后台，拥有运费管理权限（order:freight:write）
- **入口路径**：后台左侧菜单 → 运营管理 → 运费设置

## 步骤详解

### 第一步：加载现有配置

进入页面后，系统自动从服务端加载已保存的运费配置，展示所有已设置的地区条件和对应的运费规则。

### 第二步：添加运费条件

点击"新增条件"按钮添加新的地区条件：

**前置校验**：如果上一个条件尚未选择地区（effectiveAreasList为空），提示"请先填写完新条件，再继续新增"，阻止添加。

新条件默认值：
- 运费：0元（即包邮）
- 满额包邮：关闭
- 包邮门槛：0元
- 到付：关闭

新条件插入到已有条件列表的倒数第二位置（最后一条始终是"其他未选择地区"）。

### 第三步：配置地区

点击"选择地区"按钮弹出地区选择弹窗：

- 使用级联选择器，支持省→市两级选择
- 支持多选，可选择多个省或多个市
- 直辖市（北京/天津/上海）只到省级，无子级
- 已选地区以标签形式显示，可单独删除

**地区唯一性校验**：
1. 同一条件内不能同时选择省份和其下辖市（如选了"广东省"就不能再选"广州市"）
2. 不同条件之间不能有重叠地区（"一地区只支持设置一个条件"）
3. 选择时进行数据去重校验

### 第四步：设置运费规则

每个条件卡片内可设置：

**运费金额**：
- 输入框输入数字（单位：元）
- 设为0表示包邮
- 当勾选"到付"时，运费输入框自动禁用

**满额包邮**：
- 勾选后，当订单总价达到指定金额时自动免运费
- 需填写包邮门槛金额
- 当勾选"到付"时，此选项自动禁用

**到付**：
- 勾选后，该地区订单统一到付
- 选择到付后保存时自动清空运费和满额包邮设置（产品要求）

### 第五步："其他未选择地区"兜底规则

最后一条为"其他未选择地区"的兜底配置：
- 不需要选择地区（自动匹配所有未在前序条件中指定的地区）
- 支持两种模式：统一运费 或 统一到付（单选切换）
- 独有功能：**免除运费开关** — 开启后，医生在开处方时可手动修改运费为包邮。提示文字："开启后，医生开处方时可修改运费为包邮。"
- 支持满额包邮设置

### 第六步：保存配置

点击"保存"按钮前的校验：

| 校验项 | 错误提示 |
|--------|---------|
| 有条件的地区为空 | "请添加地区再保存" |
| 未设置运费且非到付 | "请设置运费再保存" |
| 运费为负数 | "运费不能小于零" |
| 开启满额包邮但未填写金额 | "请正确填写满额包邮订单金额" |
| 满额包邮金额为负数 | "请正确填写满额包邮订单金额" |
| 无修改内容 | "无修改内容" |

保存成功后：
- 到付条件自动清空运费、满额包邮相关字段
- 提示"保存成功"
- 重新加载配置确认保存结果

### 第七步：导航离开保护

当用户修改了配置但未保存时，如果点击左侧菜单导航到其他页面，系统会弹窗提醒：
- 标题："有更新内容，请先保存"
- 选项：保存（保存后跳转）/ 不保存（放弃修改直接跳转）

## 异常处理

| 情况 | 系统处理方式 |
|------|------------|
| 无写权限 | 接口返回403，页面无法保存 |
| 配置加载失败 | 列表为空，需手动刷新 |
| 地区数据加载失败 | 地区选择器为空 |

## 关联功能

- 关联**医生开处方**：医生处方页面的运费显示依据此配置计算
- 关联**药品订单**：订单运费根据收件地址匹配对应地区条件
