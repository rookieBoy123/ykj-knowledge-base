# 互联网医院信息配置流程

> 文档版本 1.0 | 面向角色：系统管理员

## 概述

管理员在后台填写和维护互联网医院的机构基础信息，用于向监管平台申报和备案。信息涵盖机构标识、类型、分类、安全等级、隶属关系等多个维度，是与各地卫健委系统对接的数据基础。

## 触发条件

- **前置条件**：具有管理后台超级管理员或医院配置权限
- **入口路径**：管理后台 → 医院管理 → 医院基础配置 → "医院信息"Tab

## 步骤详解

### 第一步：进入医院信息编辑页

用户看到什么：

- "互联网医院信息"标题
- 左右两栏布局的表单，每行两个字段
- 各字段均为输入框/下拉选择器/日期选择器

用户能做什么：

- 在各字段中填写或选择相应数据

---

### 第二步：填写机构基础信息

表单包含以下字段（按出现顺序）：

| 字段名称 | 类型 | 长度限制 | 说明 |
|---------|------|---------|------|
| 机构标识 | 文本输入 | 30字 | 机构在系统中的唯一编码 |
| 机构名称 | 文本输入 | 500字 | 机构全称 |
| 单位隶属关系 | 下拉选择 | - | 如"省属""市属"等 |
| 信息安全等级保护证书编号 | 文本输入 | 18字 | 等保证书编号 |
| 机构类型 | 下拉选择 | - | 机构在监管体系中的类型 |
| 机构成立日期 | 日期选择 | - | 选择具体日期 |
| 机构分类 | 下拉选择 | - | 来自字典配置 |
| 经济类型 | 下拉选择 | - | 来自字典配置 |
| 信息安全等级保护 | 下拉选择 | - | 等保级别 |
| 机构分类管理类别 | 下拉选择 | - | 来自字典配置 |
| 是否分支机构 | 下拉选择 | - | 是/否 |
| 机构所在地民族自治地方标志 | 下拉选择 | - | 是/否 |
| 部署平台 | 下拉选择 | - | 部署在哪个平台 |
| 是否具备双路供电或紧急发电设施 | 下拉选择 | - | 是/否 |
| 地址 | 级联选择 + 详细地址 | - | 省/市/区级联 + 手动输入 |

---

### 第三步：保存配置

用户填写完成后保存：

- 所有下拉选项均来自系统字典，确保数据标准化
- 如"单位隶属关系""机构类型""部署平台"等下拉值预先配置在字典中
- 保存后的数据将用于向监管平台申报

---

## 异常处理

| 情况 | 系统处理方式 |
|------|------------|
| 机构名称超长（500字） | 输入框限制，不可输入更多 |
| 机构标识超长（30字） | 同上 |
| 等保证书编号超长（18字） | 同上 |
| 下拉选项无合适的值 | 需联系管理员在字典管理中补充 |
| 地址未选完整 | 级联选择器引导逐级选择 |

---

## 关联功能

- **字典管理**：所有下拉选项的值来源于字典配置
- **服务点信息**：同页面的另一个Tab
- **科室信息**：同页面的另一个Tab
- **设备信息**：同页面的另一个Tab
- **监管平台申报**：配置的信息用于向卫健委等监管机构申报
