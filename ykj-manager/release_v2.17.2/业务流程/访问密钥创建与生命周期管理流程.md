# 访问密钥创建与生命周期管理流程

> 文档版本 2026-07-03 | 面向管理人员

## 概述

访问密钥（AccessKey）用于外部系统或第三方服务通过API接口访问平台数据。管理员可以为指定的系统用户创建访问密钥对（AccessKey ID + AccessKey Secret），密钥用于接口调用时的身份认证。每个用户最多创建2对密钥，密钥支持启用/停用和删除操作。

## 触发条件

- 需要为第三方系统或合作方开通API访问权限
- 需要管理现有密钥的状态（启用/停用）
- 密钥泄露或不再使用时需要删除
- 需要更新密钥的描述信息

## 步骤详解

### 第一步：进入密钥管理
通过系统用户列表中的某个用户，点击进入该用户的「访问密钥」管理页面。页面必须通过URL参数传递用户ID，否则无法加载数据——系统会提示"缺少用户id"。

### 第二步：查看密钥列表
页面展示该用户下所有的访问密钥列表，表格包含以下信息：
- **访问密钥ID**：系统自动生成的唯一标识（UUID格式，不含连字符）
- **描述**：管理员填写的密钥用途说明
- **状态**：启用（正常）/ 停用（已失效）
- **创建时间**：密钥的生成时间
- **最近使用时间**：该密钥最后一次被用于API调用的时间

支持按「访问密钥ID」进行精确搜索。

### 第三步：新增密钥
点击「新增」按钮，弹出新增对话框：

1. **描述信息**：填写密钥的用途说明，最多500字
2. 点击确定后系统自动生成：
   - **AccessKey ID**：全局唯一的标识符（UUID）
   - **AccessKey Secret**：密钥密码，用于签名认证
   - **盐值**：用于密码加密的随机值
   - **密钥状态**：默认启用

3. **下载密钥文件**：新增成功后，弹出确认框询问是否下载密钥文件（Excel格式的CSV文件）。点击"确定"则触发浏览器下载，点击"取消"则跳过——但此时无法再获取该密钥的Secret值（Secret是加密存储的，只在创建时返回明文）。

> ⚠️ **安全提示**：AccessKey Secret只在创建时以明文形式返回一次。如果关闭了下载窗口，只能删除旧密钥后重新创建。

### 第四步：编辑密钥描述
列表中的每条密钥都可以编辑。点击「编辑」按钮弹出编辑对话框，只能修改「描述」字段。AccessKey ID和创建时间不可修改。

### 第五步：管理密钥状态
每条密钥支持「启用」和「停用」操作：
- **停用**：将密钥状态设为禁用。停用后，使用该密钥的API调用将被拒绝。
- **启用**：将已停用的密钥重新激活。

操作前需要二次确认。

### 第六步：删除密钥
点击「删除」按钮，二次确认后删除该密钥。删除后该密钥立即失效，使用该密钥的API调用将被拒绝。删除操作不可逆。

## 异常处理

| 情况 | 系统处理方式 |
|---|---|
| 页面URL没有用户ID参数 | 提示"缺少用户id"，页面不加载数据 |
| 用户不存在 | 提示"用户不存在"，阻止新增 |
| 用户已有2对密钥 | 提示"一个用户最多只能创建2对密钥"，阻止新增 |
| 编辑时长按不存在的密钥 | 提示"密钥不存在"，操作失败 |
| 密钥描述超过500字 | 输入框阻止继续输入（maxlength限制） |
| 新增密钥后未下载文件 | 无法再次获取Secret明文，只能重新创建 |
| 停用的密钥无法用于API调用 | API端会返回认证失败 |

## 关联功能

- **系统用户管理**：密钥与用户绑定，通过用户进入密钥管理
- **开放平台API**：密钥用于开放平台接口的签名认证
- **登录日志**：使用密钥的API调用记录可在登录日志中审计
