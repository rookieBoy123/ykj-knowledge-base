# 定时任务调度与执行日志管理流程

> 文档版本 2026-07-03 | 面向管理人员

## 概述

定时任务管理用于维护平台在后台自动执行的各种周期性任务，如数据同步、账单结算、消息推送等。管理人员可以查看所有任务的状态、修改任务的执行计划（Cron表达式）、手动触发任务立即执行，以及查看每次任务执行的详细日志。

## 触发条件

- 需要临时调整某个定时任务的执行频率
- 需要手动触发一次任务（如补跑某个数据同步）
- 需要排查某个任务执行失败的原因
- 需要暂停某个不再需要的定时任务

## 步骤详解

### 第一步：进入任务列表
进入管理后台的「系统管理 → 定时任务」页面。页面展示所有已注册的定时任务列表。

**任务列表信息**：
- **任务类型**：分为「定时任务」（按Cron表达式周期性执行）和「一次性任务」（执行一次后自动停止）
- **任务名称**：系统内部的任务标识
- **任务描述**：任务的用途说明
- **Cron表达式**：定时任务的触发规则（如"0 0 2 * * ?"表示每天凌晨2点执行）
- **状态**：启用/禁用（开关形式，直接点击切换）

**搜索筛选**：支持按「任务类型」、「任务名称」、「任务描述」进行筛选。可通过URL传递预筛选参数（如从其他页面跳转时自动带上任务类型）。

### 第二步：编辑任务
点击某条任务的「编辑」按钮，弹出编辑对话框：
- **Cron表达式**：修改任务的执行规则（必填，最多64个字符）
- **任务描述**：修改任务的用途说明（必填，最多100个字符）

> ⚠️ **一次性任务不能编辑**：类型为"一次性任务"的行，编辑按钮为禁用状态。一次性任务执行后自动结束，无需修改计划。

### 第三步：手动触发任务
点击「触发」按钮，弹出确认框，确认后任务将立即执行一次（不改变其原有调度计划）。触发结果提示"已触发"。

> ⚠️ **一次性任务不能触发**：类型为"一次性任务"的触发按钮禁用，因为一次性任务只能由系统自动触发一次。

### 第四步：启用/禁用任务
在任务列表的状态列，直接点击开关即可切换启用/禁用状态：
- **开启**：任务按Cron表达式正常调度执行
- **关闭**：任务暂停，不会自动执行（但仍可手动触发）

切换即时生效，无需额外确认。

### 第五步：查看执行日志
点击某条任务的「日志」按钮，跳转到「执行日志」页面，自动带上该任务的筛选条件。

**执行日志页面**：
- **任务筛选**：下拉选择要查看哪个任务的执行记录（选项从所有任务列表中动态加载，显示为任务描述）
- **执行状态筛选**：成功 / 失败
- **日志表格信息**：
  - 任务类型、任务描述
  - **任务参数**：本次执行时传入的参数内容
  - **计划执行时间**：Cron表达式推算出的理论执行时间
  - **实际执行时间**：任务真正开始执行的时间
  - **耗时（毫秒）**：任务从开始到结束的耗时
  - **执行状态**：成功（绿色标签）/ 失败（红色标签）。失败时鼠标悬停可查看失败原因

## 异常处理

| 情况 | 系统处理方式 |
|---|---|
| 编辑时Cron表达式未填写 | 表单校验提示"请输入cron表达式"，阻止提交 |
| 编辑时任务描述未填写 | 表单校验提示"请输入任务描述"，阻止提交 |
| Cron表达式格式错误 | 后端截获非法表达式，提示错误 |
| 尝试编辑一次性任务 | 编辑按钮置灰，不可点击 |
| 尝试触发一次性任务 | 触发按钮置灰，不可点击 |
| 执行日志无数据 | 显示空列表（正常情况：该任务尚未执行过） |
| 任务执行失败 | 状态列显示红色"失败"标签 + hover显示失败原因 |

## 关联功能

- **系统配置管理**：部分定时任务的执行参数从全局配置中读取
- **执行日志**：从任务列表可直接跳转查看该任务的历史执行记录
- **通知管理**：任务执行失败时可触发系统通知
