# 医保结算上传管理流程

> 文档版本 v1.0 | 面向读者：医保专员、运营管理者

## 概述

医保结算上传管理页用于管理慢病医保患者的结算数据，运营人员可将待上传的结算记录批量申报到医保监管平台。页面按状态分为待上传/已上传/上传失败三个标签页，支持多项筛选和批量操作。

## 触发条件

- 前置条件：已登录管理后台，拥有医保结算上传权限（`medins:declares:upload`）
- 入口路径：左侧菜单 → 医保管理 → 医保结算管理

## 步骤详解

### 第一步：切换状态标签

页面顶部有三个标签页：

- **待上传**（status=1，默认）：新生成的结算记录，等待申报
- **已上传**（status=2）：已成功申报到医保平台的记录
- **上传失败**（status=3）：申报失败的记录，可重新上传

### 第二步：筛选查询

支持以下筛选条件：

- **病人名称**：文本输入
- **结算日期**：日期范围选择
- **人员编号**：文本输入
- **病种**：远程搜索下拉框，动态搜索慢病病种
- **身份证号**：文本输入
- **科室**：树形选择器，选择科室结构
- **申报日期**：日期范围（仅在"已上传"和"上传失败"标签显示）

### 第三步：查看结算列表

列表展示字段：

- 复选框（用于批量选择）
- 人员编号
- 患者姓名
- 慢性诊断科室
- 病种
- 证件类型（默认显示"居民身份证(户口簿)"）
- 证件号
- 性别
- 年龄
- 结算时间
- 申报时间（仅在"已上传"/"上传失败"标签显示）

### 第四步：单条上传

在"待上传"标签页，每条记录有"上传"按钮。点击后弹出日期选择弹窗：

1. 选择申报日期（日历选择器，不能选择未来日期）
2. 点击确定提交
3. 系统将数据上传至医保监管平台

### 第五步：批量上传

在"待上传"标签页：

1. 勾选多条需要上传的记录（列表左侧复选框）
2. 点击列表上方"批量上传"按钮
3. 在弹出的日期选择弹窗中选择申报日期
4. 点击确定，系统批量提交所有选中记录

### 第六步：重新上传

在"上传失败"标签页，点击"重新上传"按钮重新提交失败的记录，操作流程与单条上传相同。

### 第七步：查看上传详情

任意状态的记录都可点击"查看详情"查看结算明细信息。

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 上传失败 | 记录状态变为"上传失败"（status=3），可点击"重新上传"重试 |
| 批量上传操作 | 仅支持一次最多选择当前页可见记录（不支持跨页全选） |
| 上传权限不足 | 列表不显示"上传"/"批量上传"操作按钮 |
| 选择未来日期 | 日期选择器默认禁用未来日期 |
| 身份证号搜索 | 后端自动加密后再查询（数据库中身份证为加密存储） |
| 病种远程搜索无匹配 | 下拉列表为空 |

## 关联功能

- **慢病病种管理**：医保结算关联的慢病病种在另一页面管理
- **医保诊疗类别管理**：结算数据中的诊疗类别对应医保编码体系
- **医保监管平台**：数据最终上传到外部医保监管系统
