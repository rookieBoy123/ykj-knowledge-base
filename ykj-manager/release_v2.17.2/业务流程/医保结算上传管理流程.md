# 医保结算上传管理流程

> 文档版本 v1.0 | 面向读者：后台管理员

## 概述

管理员在后台管理慢病医保患者的结算清单，筛选待上传的结算数据，选择申报日期后批量上传到广州市医保基金结算平台。系统提供三个状态选项卡（待上传/已上传/上传失败），方便管理员跟踪每笔结算的上传进度。

## 触发条件

- **前置条件**：管理员已登录后台、医院已开通医保慢病业务
- **入口路径**：后台左侧菜单 → 财务管理 → 医保结算管理

## 步骤详解

### 第一步：筛选结算数据

管理员进入页面后，系统自动加载最近30天的"待上传"状态结算数据。

**筛选条件（共6项）**：

1. **病人名称**：文本输入框，支持姓名模糊搜索
2. **结算日期**：日期范围选择器，默认最近30天（前端自动计算并填充）
3. **人员编号**：医保系统中的人员编号，文本搜索
4. **病种**：下拉选择框，支持远程搜索。数据来源：慢病管理中的病种列表（最多一次加载500条）
5. **身份证号**：文本搜索
6. **科室**：树形下拉选择器（科室树），支持按名称筛选

**状态选项卡**：三个快捷Tab，过滤不同的上传状态
- **待上传**（默认选中，status=1）：尚未上传到医保平台的结算记录
- **已上传**（status=2）：已成功上传到医保平台的结算记录
- **上传失败**（status=3）：上传过程中出现错误的结算记录

**选项卡联动规则**：
- 选择"已上传"时：列表增加"申报时间"列（处于隐藏/显示切换），多选列和"批量上传"按钮被隐藏
- 选择"待上传"或"上传失败"时：列表显示多选列和"批量上传"按钮

### 第二步：查看结算数据列表

列表展示 **11 列信息**，每页 10 条：

| 列名 | 说明 |
|------|------|
| 多选 | 仅在待上传/上传失败状态显示 |
| 人员编号 | 医保系统的人员唯一编号 |
| 患者姓名 | 就诊人真实姓名 |
| 慢性诊断科室 | 开具慢病诊断的科室名称 |
| 病种 | 慢病病种名称（如高血压、糖尿病） |
| 证件类型 | 固定显示"居民身份证(户口簿)" |
| 证件号 | 身份证号码 |
| 性别 | 患者性别 |
| 年龄 | 患者年龄 |
| 结算时间 | 医保结算的时间 |
| 申报时间 | 仅在"已上传"状态显示，上传到医保平台的时间 |
| 操作 | 上传/重新上传/查看详情 |

### 第三步：单条上传

对于单条待上传或上传失败的记录：

1. **点击"上传"按钮**（待上传状态）或"重新上传"按钮（上传失败状态）
2. 弹出"选择申报时间"对话框
3. 从日期选择器中选择申报日期（不能选择未来日期，已通过 `disabledDate` 限制了 `time > Date.now()` 的日期）
4. 点击"确定"按钮
5. 系统校验：如果未选择申报时间，提示"请选择申报时间"
6. 调用 `declaresUpload` 接口，传入申报时间和结算记录 ID
7. 上传成功后提示"上传成功"并刷新列表

### 第四步：批量上传

对于多条待上传或上传失败的记录：

1. 在列表中勾选多条记录（支持多选）
2. 点击列表顶部"批量上传"按钮
3. 如果未勾选任何记录，提示"请至少选择一条数据"
4. 弹出"选择申报时间"对话框
5. 选择申报日期，点击"确定"
6. 系统将选中的记录 ID 集合 + 申报时间一并提交
7. 上传成功后刷新列表

### 第五步：查看详情

点击任一记录的"查看详情"按钮，弹出详细结算清单窗口：

**弹窗标题**："广州市医疗保障基金结算清单"

**三个信息区域**：

1. **一、基础信息**（17项）：申报时间、清单流水号、病案号、医保编号、人员编号、患者姓名、性别、年龄、结算ID、就诊ID、国籍、职业/联系人/电话/关系/地址均显示"无"

2. **二、门诊慢特病诊疗信息**（9项）：诊断科别、就诊日期、入院时间（取就诊时间）、出院时间（取就诊结束时间）、离院方式、诊断类别、入院病情类型、主诊断标志等

3. **三、医疗收费信息**（4项）：业务流水号、票据代码、票据号码、结算日期

**上传失败特殊展示**：如果记录状态为"上传失败"，在详情页顶部额外显示红色失败原因（`failureReason` 字段）

### 已上传状态下的列表差异

切换到"已上传"选项卡后：

1. "申报日期"列从隐藏变为显示
2. 多选列和"批量上传"按钮被隐藏
3. "申报时间"筛选项（日期范围）由隐藏变为显示 — 可以按申报日期进一步筛选
4. 操作按钮只有"查看详情"（没有"上传"或"重新上传"）

## 异常处理

| 情况 | 系统处理方式 |
|------|------------|
| 未选择申报时间就点确定 | 前端提示"请选择申报时间"（warning级别） |
| 批量上传未勾选记录 | 前端提示"请至少选择一条数据"（warning级别） |
| 接口返回非200 | 前端不做错误处理（仅处理成功分支） |
| 上传失败记录 | 状态变为"上传失败"（status=3），可点击"重新上传" |
| 接口超时或网络异常 | 列表不刷新，需手动重新查询 |
| 第1页无数据但页码>1 | 前端自动回退到上一页重新查询（current!==1时pageNum--） |

## 关联功能

- **慢病病种管理**：病种下拉选项的数据来源（systemmanage/medicalInsurance/chronicdiseasemanage）
- **科室管理**：科室树的数据来源
- **医保结算服务**：`declaresUpload` 接口调用服务端医保结算上传逻辑（ykj-service-yb-declare）
