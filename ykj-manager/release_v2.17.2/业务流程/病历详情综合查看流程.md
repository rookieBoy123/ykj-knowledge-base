# 病历详情综合查看流程

> 文档版本 1.0 | 面向读者：运营、客服、产品

## 概述

后台管理人员在患者详情中综合查看患者的就诊信息、病史、处方建议、会诊记录和问诊对话，一站式掌握患者的完整诊疗轨迹。

## 触发条件

- 后台管理 → 患者管理 → 患者列表 → 点击某患者 → 病历详情
- 需要对应权限

## 步骤详解

### 第一步：查看患者基本信息

页面顶部展示患者信息卡片：
- 就诊卡号、姓名、性别、出生日期、年龄（优先显示岁，不满1岁显示月）
- 证件号码（身份证，标注"已校验"如已验证）
- 监护人信息（如有）：监护人姓名、关系、身份证号
- 慢病医保复诊标签（如有登记慢病病种）

### 第二步：查看病史信息

病史信息卡片展示问诊时填写的完整病史：
- **主诉**：患者的病情自述
- **现病史**：当前病情描述
- **既往史**：过去的病史
- **家族史**、**个人史**、**过敏史**
- **月经史**、**婚育史**（如适用）
- 中医特色（根据医生类型显示）：脉象、舌象、治法
- 西医特色：治疗意见
- **常见体征**、**辅助检查**

所有未填写项统一显示"未填写"。

### 第三步：查看就诊信息

就诊信息卡片展示：
- **就诊方式**：固定显示"线上就诊"
- **首诊机构**：首次就诊的机构名称
- **首诊时间**：首次就诊时间
- **问诊时间**：实际问诊开始时间
- **服务类型**：图文问诊/电话问诊/视频问诊等
- **科室名称**
- **病情描述**：患者的病情描述
- **病历资料**：患者上传的图片资料，点击可预览大图；如未上传显示"记录遗失/不在身边"
- **主治医生**：医生姓名、科室、职称、医院（是否为三甲）

提供"问诊对话记录"按钮（如有问诊记录），点击可查看完整聊天记录。

### 第四步：查看会诊记录（如有）

如果系统启用了会诊功能（consultationEnabled），展示该患者的会诊记录：
- 每次会诊显示：会诊时间、会诊状态
- 点击可跳转会诊详情页

### 第五步：查看处方建议

展示医生为该患者开具的所有处方，区分：
- **成药处方**（type=1）：展示诊断（辨病/辨证）、药品明细（名称+规格+数量+用法用量）、处方医生
- **中药处方**（type=2）：展示诊断（辨病/辨证）、药品明细（药材名称+克数+单位）

每张处方提供两个操作按钮：
- **查看对应订单**（如关联了订单）
- **查看原始处方**（如生成了电子处方PDF/图片）
- 处方按"诊断/辨病/辨证"三种维度分别展示

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 无法获取患者信息 | 展示空或"-"占位 |
| 无会诊记录 | 显示"暂无数据" |
| 无病历资料图片 | 显示"记录遗失/不在身边" |
| 问诊记录不存在 | 不显示"问诊对话记录"按钮 |
| 未填写病史项 | 统一显示"未填写" |

## 关联功能

- 患者列表管理：入口页面
- 会诊管理：查看会诊详情和报告
- 处方管理：查看电子处方和对应订单
- 问诊对话记录：查看医患聊天记录
