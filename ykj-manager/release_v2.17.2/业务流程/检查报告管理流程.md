# 检查报告管理流程

> 文档版本 1.0 | 面向读者：运营、医务、产品

## 概述

后台管理人员查看和管理所有患者的检查报告，支持按检查日期、患者姓名、报告状态筛选，可查看已出报告或填写未出的报告。

## 触发条件

- 后台管理 → 诊疗管理 → 检查报告管理
- 需要对应权限

## 步骤详解

### 第一步：筛选查找

页面顶部提供筛选条件：
- **检查日期**：日期选择器，按检查日期范围筛选
- **姓名**：输入框，最长5个字，按患者姓名搜索
- **状态**：下拉选择——全部/未出报告（status=0）/已出报告（status=1）

点击查询后自动重置到第1页。

### 第二步：浏览报告列表

列表展示以下字段：
- **检查号**（reportNo）
- **姓名**（patientName）
- **性别**（patientGender：1男/2女）
- **年龄**（patientAge）
- **检查项目**（type_dictName，如CT、B超等）
- **科室**（department）
- **检查时间**（examineTime）
- **报告医生**（doctorName，无则显示"-"）
- **报告时间**（reportTime）
- **状态**：未出报告 / 已出报告
- **操作**：根据状态不同显示不同按钮

### 第三步：操作

根据报告状态提供对应操作：
- **未出报告**（status=0）→ "填写报告"按钮
- **已出报告**（status=1）→ "查看报告"按钮

点击任一按钮打开详情弹窗（details子组件）：
- 未出报告：可填写和提交报告内容
- 已出报告：查看完整报告信息

### 第四步：分页浏览

每页10条记录，支持切换每页条数和翻页。

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 无匹配报告 | 列表为空，显示空数据 |
| 报告医生未知 | 显示"-" |
| 查询参数异常 | 自动使用默认参数重新查询 |

## 关联功能

- 患者管理：可查看对应患者的完整信息
- HIS数据同步：检查报告数据来源于HIS系统对接
