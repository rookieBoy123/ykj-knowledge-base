# 用户协议管理流程

> 文档版本：2026-07-03 | 面向读者：运营人员、法务/合规人员

## 概述

用户协议管理用于维护平台中各类用户协议的内容，如患者端协议、医生端协议、隐私政策等。每个协议包含协议标识（唯一编码）、协议类型、协议标题和协议正文（富文本内容）。管理员可以编辑已有协议的内容。

## 触发条件

- **入口路径**：管理后台 → 运营管理 → 文章管理 → 用户协议
- **前置条件**：拥有对应菜单权限的管理员

## 步骤详解

### 第一步：浏览协议列表

进入页面后，系统自动加载协议列表，每页显示10条记录。列表展示以下信息：
- **协议标识**：协议的唯一编码（如 `user_agreement`、`privacy_policy`）
- **协议类型**：协议的类型分类（如患者端协议、医生端协议、其他协议，由字典 `xylx` 维护）
- **标题**：协议的显示标题

列表支持勾选多条记录（但注意：代码中批量删除功能已被注释，实际不可用）。

### 第二步：编辑协议

协议管理**不支持新增**——协议由系统预设好标识和类型，管理员只能编辑已有协议的标题和正文内容。

点击操作列的「编辑」按钮，弹出编辑弹窗（宽度800px），弹窗包含：
- **协议标识**：编辑模式下该字段置灰不可修改（确保标识唯一性和系统识别）
- **协议类型**：编辑模式下该字段置灰不可修改。协议类型选项来源于字典 `xylx`
- **标题**：文本输入，最多255字，可修改
- **协议正文**：通过富文本编辑器编辑，支持文字排版、图片插入等

点击「确定」保存修改。

### 第三步：查看协议标识重复校验（代码中存在但被注释）

代码中预留了协议标识重复校验的逻辑（`validExistsName` 方法），但目前已被注释掉。当前版本不会主动校验标识重复。

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 标题为空 | 保存时校验提示"请输入标题" |
| 标识为空 | 保存时校验提示"请输入标识" |
| 协议标识重复 | （当前版本未启用校验，已注释） |
| 编辑时协议标识和类型 | 置灰不可修改 |

## 关联功能

- 编辑后的协议内容会更新到小程序端的用户协议展示页面
- 协议类型来源于系统字典 `xylx`
- 协议标识是前后端约定的固定编码，不能随意修改
