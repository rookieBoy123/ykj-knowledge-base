# 运费设置管理流程

> 文档版本 v1.0 | 面向运营管理员

## 概述

运费设置让运营人员按地区配置药品配送的运费规则。支持按省份/城市设置不同的运费标准，支持满额包邮和到付模式，最后一条始终为"其他未选择地区"兜底规则。

## 触发条件

- 运营人员需要为新开通地区设置配送费用
- 需要调整现有地区的运费或包邮门槛
- 需要开启或关闭到付模式

## 页面结构

### 运费条件列表
每条运费条件包含：
1. **支持地区**：通过省市选择器选择，已选地区以标签形式展示，可逐个删除
2. **运费金额**：手动输入金额（元），设为0即包邮
3. **满额包邮**：勾选后设置包邮门槛，订单总价达到门槛即免运费
4. **到付**：勾选后无需设置运费（到达付款），此时运费和包邮设置被禁用

### 其他未选择地区（兜底规则）
列表的最后一条始终为"其他未选择地区"，有两种模式：
- **统一运费**：为所有未单独配置的地区设置统一运费，同样支持满额包邮
- **统一到付**：所有未配置地区统一采用到付模式

### 其他设置
兜底规则中还有一个"免除运费"开关：
- 开启后，医生在开具处方时可以手动将运费修改为包邮（0元）
- 关闭后，医生不能修改运费

## 操作详解

### 新增条件
点击"新增条件"按钮：
- 在"其他未选择地区"之前插入一条新条件
- 新增条件前需要先填写完上一个条件（已选择地区），否则提示"请先填写完新条件，再继续新增"
- 列表超过3条时自动滚动到底部

### 选择地区
1. 点击"选择地区"弹出省市选择器
2. 选择器以级联方式展示：省份→城市，支持多选
3. 直辖市（北京/天津/上海）无城市层级

**地区冲突校验**：
- 同一条件内：不能同时选择省和该省下的市（提示"选择省后不能在选择所在市区"）
- 不同条件之间：一个地区只能设置一个条件（提示"一地区只支持设置一个条件，该地区已存在，请确认后再添加"）

### 删除条件
点击删除按钮，弹出确认"确认删除吗？删除后该条件不作为运费设置条件"。

### 保存
点击保存按钮前的校验：
1. 每条已配置的条件必须有选择的地区（提示"请添加地区再保存"）
2. 非到付模式下必须设置运费（提示"请设置运费再保存"）
3. 运费不能为负数（提示"运费不能小于零"）
4. 勾选满额包邮时必须填写包邮门槛金额（提示"请正确填写满额包邮订单金额"）
5. 包邮门槛不能为负数

保存时自动处理：选择到付的条件，其运费、包邮金额自动清空。

### 未保存离开提示
如果修改了运费设置但未保存就尝试离开页面（通过菜单点击切换），会弹出提示"有更新内容，请先保存"，可选择"保存"或"不保存"。

如果没有任何修改则直接跳转，不提示。

## 异常处理

| 情况 | 处理方式 |
|------|---------|
| 未选择地区就新增下一个条件 | 提示"请先填写完新条件，再继续新增" |
| 同一条件内选了省又选了该省下的市 | 提示"选择省后不能在选择所在市区" |
| 不同条件选择了相同地区 | 提示"一地区只支持设置一个条件" |
| 已配置条件无地区 | 提示"请添加地区再保存" |
| 非到付未设运费 | 提示"请设置运费再保存" |
| 满额包邮未填金额 | 提示"请正确填写满额包邮订单金额" |
| 运费为负数 | 提示"运费不能小于零" |
| 未保存离开 | 弹出"有更新内容，请先保存"确认框 |
| 无修改点击保存 | 提示"无修改内容" |

## 关联功能

- 药品订单配送（前端根据收货地址匹配运费规则）
- 医生开方（免除运费开关影响医生端操作）
