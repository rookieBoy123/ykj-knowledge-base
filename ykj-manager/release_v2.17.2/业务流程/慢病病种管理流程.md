# 慢病病种管理流程

> 文档版本 v1.0 | 面向读者：产品、运营

## 概述

管理员在后台维护医保门诊慢性疾病病种的基础数据，包括病种名称、编码、关联科室和启用状态。慢病病种用于医保慢病复诊流程中，患者发起慢病问诊时需匹配对应病种。

## 触发条件

- **前置条件**：拥有"医院管理-慢病管理"权限
- **入口路径**：系统管理 → 医保管理 → 慢病病种管理

## 步骤详解

### 第一步：查看慢病列表

进入页面后看到慢病病种列表，每页显示10条记录。表格列包括：

| 列名 | 说明 |
|------|------|
| 序号 | 自动编号 |
| 病种编码 | 病种的唯一编码 |
| 病种名称 | 病种的中文名称 |
| 关联科室 | 该病种对应可接诊的科室列表 |
| 状态 | 启用/禁用开关 |
| 操作 | 编辑按钮 |

### 第二步：搜索筛选

顶部搜索栏支持：
- **病种名称搜索**：输入病种名称关键词进行模糊搜索
- **病种编码搜索**：输入病种编码进行精确搜索

两个条件可组合使用。

### 第三步：新增慢病病种

列表上方无"新增"按钮（代码中未实现前端新增功能）。慢病病种通过后端接口创建（需 `hosp:chronic:write` 权限）。

### 第四步：编辑慢病病种

点击某条记录的"编辑"按钮，弹出编辑弹窗（500px宽）：

**弹窗内容**：
- **病种名称**：只读显示，不可修改
- **病种编码**：只读显示，不可修改
- **关联科室**：树形多选下拉框，从全院科室树中选择该病种关联的科室。**必填**，未选择时提示"请选择关联科室"
- **状态**：启用/禁用单选（默认选中"禁用"）

点击"确定"提交修改，点击"取消"关闭弹窗。

### 第五步：启用/禁用

列表每行有启用/禁用开关，行内直接操作，即时生效：
- **开启开关**（enabled=1）：调用启用接口，病种生效
- **关闭开关**（enabled=0）：调用禁用接口，病种失效

操作成功提示"修改成功"，无需二次确认。

## 异常处理

| 情况 | 系统处理方式 |
|------|------------|
| 编辑时未选关联科室 | 表单校验不通过，提示"请选择关联科室" |
| 无慢病写入权限 | 接口返回403，编辑/启用禁用操作不可用 |
| 无慢病读取权限 | 接口返回403，列表不加载 |

## 关联功能

- 医生科室管理（关联科室数据来源）
- 医保慢病复诊流程（患者端使用）
- 权限管理（hosp:chronic:write / hosp:chronic:read）
