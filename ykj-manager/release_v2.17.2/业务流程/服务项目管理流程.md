# 服务项目管理流程

> 文档版本 v1.0 | 面向读者：运营、管理员

## 概述

管理员在线下服务预约功能中创建和管理可供患者预约的服务项目。每个服务项目定义服务名称、服务方式（上门/到院）、服务类型、适用人员类型，并可随时启用或禁用。

## 触发条件

- **前置条件**：拥有服务项目管理权限（hosp:service-projects:write）
- **入口路径**：管理后台 → 诊疗管理 → 服务项目管理 → 服务项目列表

## 步骤详解

### 第一步：查看服务项目列表

管理员进入列表页后看到所有已配置的服务项目：

- **列表字段**：服务类型、人员类型、服务名称、服务方式、启用状态
- **每页显示**：10条记录，支持分页
- **开单科室列**：默认隐藏，仅在开启电子发票功能时显示

### 第二步：筛选服务项目

支持以下筛选条件：

- **服务名称**：模糊搜索，最多20字
- **服务方式**：上门服务 / 自行到院 / 两者都支持
- **服务类型**：从字典动态加载（如体检、健康证、核酸检测、检查等）
- **人员类型**：个人 / 团体企业 / 两者都支持
- **服务状态**：启用 / 禁用

筛选条件组合使用，后端查询时内部转换：
- 人员类型"个人" → personalEnabled=1
- 人员类型"团体/企业" → teamEnabled=1
- 服务方式"上门服务" → visitingEnabled=1
- 服务方式"自行到院" → hospitalEnabled=1

### 第三步：新增服务项目

点击"新增"按钮跳转到新增编辑页：

- 填写服务名称
- 选择服务类型（从字典加载）
- 选择服务方式（上门/到院/两者）
- 选择人员类型（个人/团体企业/两者）
- 设置启用状态

### 第四步：编辑与查看详情

每行提供两个操作按钮：

- **查看详情**：跳转到详情页，查看服务项目的完整信息
- **编辑**：跳转到编辑页，修改服务项目配置

### 第五步：启用/禁用服务

列表中可直接切换服务项目的启用/禁用状态：

- **即时生效**：开关切换后立即调用接口更新
- **操作反馈**：成功提示"操作成功"
- **禁用影响**：禁用的服务项目在小程序端不再显示，患者无法预约
- **缓存策略**：从详情/编辑页返回时保留搜索条件，使用 keep-alive 路由缓存

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 操作失败 | 后端返回错误提示，前台显示对应消息 |
| 启用/禁用失败 | 接口报错，开关回弹到原状态 |
