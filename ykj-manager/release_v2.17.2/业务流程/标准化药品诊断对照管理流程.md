# 标准化药品诊断对照管理流程

> 文档版本：2026-07-03 第5轮 | 面向读者：平台管理员、药品管理员

## 概述

标准化药品诊断对照是将"药品通用名"与"标准诊断名称"进行关联绑定，确保医生在开具处方时系统能自动根据诊断匹配对应的可用药品，是实现处方合规审核的基础数据。

## 触发条件

- **前置条件**：需要有系统管理员或药品管理员角色
- **入口路径**：侧边栏 → 医院管理 → 标准化管理 → 标准药品诊断对照

## 步骤详解

### 第一步：查看对照列表

进入页面后显示已有的药品诊断对照列表，每条记录包含：序号、药品通用名、关联的诊断名称列表（多个诊断用顿号"、"分隔）、操作按钮（编辑）。

列表支持分页浏览，每页默认显示10条。

### 第二步：搜索查找

支持两种搜索方式：
- **按药品通用名搜索**：输入药材名称，最多40字
- **按诊断搜索**：输入诊断名称关键词，最多255字

两种条件可组合使用，搜索后列表自动重置到第一页。

### 第三步：新增对照关系

点击"新增"按钮弹出编辑窗口，需填写两个必填项：

1. **药品通用名**（必填）：输入要关联的药材名称，最多40字
2. **诊断**（必填）：从诊断列表中多选要关联的诊断名称。诊断列表下拉框支持远程搜索——输入关键词后系统自动从标准诊断库中检索匹配的诊断（默认加载前999条记录），选择后可多选。

两项都填写后才能点击"确定"提交。

### 第四步：同名药品的智能处理

当新增的药品通用名在系统中已存在时，系统会弹出提示："该药品通用名已存在，是否将本次新增的诊断更新到原数据中？"

- 选择"确定"：本次新增的诊断会追加到已有记录中，合并保存
- 选择"取消"：不做任何操作，窗口保持打开

如果药品通用名尚不存在，则直接创建新记录。

### 第五步：编辑已有对照

点击某条记录的"编辑"按钮，弹出编辑窗口：
- **药品通用名**：灰色不可修改（编辑模式下该字段被禁用）
- **诊断**：显示当前已关联的诊断（多选回显），可添加新的诊断或减少已有诊断。编辑时系统会先查询该药品已有的完整诊断列表，同时自动补全所有诊断选项供选择。

提交后保存更新。

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 药品通用名为空 | 提示"请输入"，阻止提交 |
| 未选择任何诊断 | 提示"请选择"，阻止提交 |
| 药品通用名超过40字 | 输入框自动限制最大长度，无法超过 |
| 诊断搜索无匹配 | 下拉列表为空 |
| 保存后列表无数据 | 自动回退到上一页（如当前页只有一条被删导致无数据） |
| 编辑时药品通用名 | 输入框置灰不可编辑，保护对照关系的一致性 |

## 关联功能

- 标准诊断库管理（stdDeptDiagList）：诊断数据来源，药品对照中的诊断选项从该模块获取
- 标准药品信息库：药品通用名的来源基准
- 处方开具：医生开处方时根据诊断自动筛选可用药品
