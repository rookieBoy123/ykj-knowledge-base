# 反馈管理流程

> 文档版本：v1.0 | 面向读者：管理员

## 概述

管理员在后台查看患者通过小程序提交的意见反馈，按反馈类型和状态筛选，对未处理的反馈进行回复处理。

## 触发条件

- **前置条件**：以管理员角色登录
- **入口路径**：运营管理 → 反馈管理

## 步骤详解

### 第一步：筛选反馈

页面顶部提供两个搜索条件：

1. **反馈类型**：下拉选择，选项从字典`fklx`动态加载，页面加载时通过`dictionaryList({ categoryCode: 'fklx' })`接口获取。选项的value取contentCode，label取contentName
2. **状态**：全部/已处理(status=1)/未处理(status=0)

两个条件可组合使用。

### 第二步：浏览反馈列表

列表支持每页10条分页，展示以下字段：

| 列名 | 说明 |
|------|------|
| 反馈类型 | 从字典fklx获取的类型名称（typeName） |
| 问题和建议 | 患者提交的文字内容（content） |
| 联系电话 | 默认脱敏，点击眼睛图标解密查看 |
| 反馈时间 | createTime |
| 状态 | 蓝色"已处理"或红色"未处理" |
| 发生事件 | happenEventName |
| 处理反馈 | 管理员的回复内容（replyContent） |
| 处理人 | replyUserName |
| 处理时间 | replyTime |

### 第三步：处理/查看反馈

操作栏按钮根据状态动态显示：

- **「处理」按钮**：仅在status=0（未处理）时显示，点击打开处理弹窗（dealFeedback组件）
- **「查看」按钮**：仅在status=1（已处理）时显示，点击打开查看弹窗

处理弹窗中管理员可以：
1. 查看反馈详情（类型、内容、图片截图、联系电话）
2. 填写处理回复（replyContent）
3. 选择处理事件类型
4. 提交后反馈状态变为"已处理"

提交成功后自动刷新列表（通过dealFeedback组件的cb回调触发findPage）。

### 第四步：查看手机号

联系电话默认脱敏，点击眼睛图标后调用`feedbackManage.getCompletePhone(id)`接口解密，逐条查看。点击页面任意处自动恢复脱敏。

## 异常处理

| 情况 | 系统处理方式 |
|------|------------|
| 反馈类型字典加载失败 | 类型筛选下拉为空，其他列正常显示 |
| 手机号解密失败 | 保持脱敏状态 |
| 处理弹窗提交失败 | 弹窗不关闭，可重新提交 |

## 关联功能

- **字典管理**：反馈类型配置（fklx字典）
- **小程序端反馈提交**：用户在小程序"个人中心→意见反馈"提交
