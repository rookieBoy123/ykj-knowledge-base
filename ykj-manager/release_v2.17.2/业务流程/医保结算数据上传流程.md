# 医保结算数据上传流程

> 文档版本：2026-07-02 | 面向读者：产品运营、管理员

## 概述

互联网医院产生的医保结算数据需要定期上传到医保平台。后台运营人员通过此功能管理结算数据的查询、筛选和批量上传。

## 触发条件

- **前置条件**：具有医保结算管理权限的后台账号
- **入口路径**：左侧菜单 → 财务管理 → 医保结算管理

## 步骤详解

### 第一步：按上传状态筛选

页面顶部有三个Tab，按数据上传状态分类：

| Tab | 说明 |
|-----|------|
| 待上传 | 尚未上传到医保平台的数据 |
| 已上传 | 已成功上传的数据（此时显示申报时间列和隐藏批量选择框） |
| 上传失败 | 上传失败需要重新处理的数据 |

### 第二步：筛选查询

支持以下筛选条件：

| 筛选维度 | 说明 |
|---------|------|
| 病人名称 | 文本搜索 |
| 结算日期 | 日期范围（默认最近30天） |
| 人员编号 | 医保人员编号 |
| 病种 | 远程搜索下拉（慢性病种列表） |
| 身份证号 | 身份证号搜索 |
| 科室 | 树形科室选择 |

在「已上传」Tab下，额外显示「申报日期」范围筛选。

### 第三步：查看结算数据列表

列表展示以下信息：

| 列名 | 说明 |
|------|------|
| 选择框 | 多选框（待上传Tab显示，已上传Tab隐藏） |
| 人员编号 | 医保人员编号 |
| 患者姓名 | 患者姓名 |
| 慢性诊断科室 | 诊断科室 |
| 病种 | 慢性病病种名称 |
| 证件类型 | 固定显示「居民身份证(户口簿)」 |
| 证件号 | 身份证号 |
| 性别 | 患者性别 |
| 年龄 | 患者年龄 |
| 结算时间 | 结算日期时间 |
| 申报时间 | 上传申报时间（仅已上传Tab显示） |
| 操作 | 上传/重新上传/查看详情 |

### 第四步：单条上传

点击某条记录的「上传」或「重新上传」按钮：
1. 弹出日期选择弹窗
2. 选择申报时间（不能选择未来日期）
3. 点击「确定」，系统将数据上传到医保平台

### 第五步：批量上传

1. 勾选多条待上传记录
2. 点击列表上方的「批量上传」按钮
3. 如未选择任何记录，提示「请至少选择一条数据」
4. 选择申报时间
5. 确认后批量上传

### 第六步：查看详情

点击「查看详情」按钮，弹出详情弹窗，展示完整的结算明细数据。

## 异常处理

| 情况 | 系统处理方式 |
|------|------------|
| 批量上传时未选择数据 | 提示「请至少选择一条数据」 |
| 未选择申报时间 | 提示「请选择申报时间」 |
| 已上传数据 | 隐藏选择框和批量上传按钮，操作列只显示查看详情 |
| 分页数据为空且不在第一页 | 自动回退到上一页 |
| 申报日期不能选择未来日期 | 日期选择器中未来日期置灰不可选 |
