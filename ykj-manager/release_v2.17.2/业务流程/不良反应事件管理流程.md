# 不良反应事件管理流程

> 文档版本：2026-07-03 | 面向读者：医务管理人员、运营人员

## 概述

不良反应事件管理用于记录和追踪诊疗过程中发生的不良反应事件。管理员可以新增、编辑、查看不良反应事件的详细信息。每条记录包含事件编号、事件业务类型、事件发生时间、患者信息、事件等级、事件影响级别、事件描述、原因分析、采取措施、造成影响、解决措施等内容。

## 触发条件

- **入口路径**：管理后台 → 运营管理 → 不良反应事件
- **前置条件**：拥有对应菜单权限的管理员

## 步骤详解

### 第一步：浏览不良反应事件列表

进入页面后，系统自动加载不良反应事件列表，每页显示10条记录。列表展示以下信息：
- **事件编号**：系统自动生成的唯一编号
- **事件业务类型**：事件的业务分类（如用药不良反应、手术并发症等，由字典 `sjywlx` 维护）
- **就诊人姓名**：发生不良反应的患者姓名（搜索时最多输入5个字）
- **证件类型**：固定显示"身份证"
- **证件号码**：患者的身份证号（脱敏显示）
- **手机号**：患者手机号（脱敏显示）
- **上报科室**：上报该事件的科室名称
- **上报人**：上报该事件的医生姓名
- **上报时间**：事件上报的日期时间
- **最后修改时间**：最近编辑时间

管理员可以通过筛选条件过滤数据：
- **事件业务类型**：下拉选择
- **就诊人姓名**：文本搜索
- **上报时间**：日期范围选择

### 第二步：新增不良反应事件

点击列表上方的「新增」按钮，弹出新增弹窗（宽度850px）。弹窗包含丰富的填写字段：

**基础信息**：
- **事件业务类型**：必填，下拉选择
- **事件发生时间**：必填，日期时间选择器，不能选择未来时间
- **上报医生**：必填，下拉搜索选择医生（加载全部医生列表）
- **上报时间**：必填，日期时间选择器
- **就诊人**：必填，下拉搜索选择患者。选择患者后系统自动带出患者手机号（通过患者绑定的会员信息获取真实手机号）

**事件评级**：
- **事件标题**：必填，最多50字
- **事件类型**：必填，下拉选择（字典 `eventType`）
- **事件等级**：必填，下拉选择（字典 `eventLevel`）
- **事件影响级别**：必填，下拉选择（字典 `eventImpactLevel`）

**详细描述**：
- **事件描述**：必填，文本域，最多3000字
- **事件发生的主要原因**：必填，文本域，最多3000字
- **采取的措施**：必填，文本域，最多1000字
- **造成的不良影响**：必填，文本域，最多1000字
- **解决措施**：必填，文本域，最多3000字
- **事件发生地址**：必填，文本域，最多100字

所有字段填写完成后点击「保存」。

### 第三步：查看详情

点击操作列的「详情」按钮，弹出详情弹窗，以只读方式展示该事件的所有信息，方便管理人员查看完整的事件报告。

### 第四步：编辑事件

点击操作列的「编辑」按钮，弹出编辑弹窗并自动回显已有数据。可修改所有字段后保存。

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 事件业务类型未选择 | 校验提示"请选择事件业务类型" |
| 上报医生未选择 | 校验提示"请选择上报医生" |
| 就诊人未选择 | 校验提示"请选择就诊人" |
| 手机号为空或格式错误 | 校验提示"请输入就诊人手机号"或"就诊人手机号格式错误"（正则校验：1开头的11位数字） |
| 事件发生时间未选择 | 校验提示"请选择事件发生时间" |
| 事件发生时间选择未来时间 | 日期选择器禁止选择未来日期 |
| 事件标题/描述/原因/措施等为空 | 各字段对应校验提示 |
| 选择就诊人后自动带出手机号 | 如果患者未绑定会员或无手机号，手机号字段保持为空 |

## 关联功能

- 事件类型、事件等级、事件影响级别来源于系统字典
- 医生列表从医生管理模块获取
- 患者列表从患者管理模块获取
- 手机号通过会员模块获取真实手机号（非脱敏号码）
