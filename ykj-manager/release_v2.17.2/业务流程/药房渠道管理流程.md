# 药房渠道管理流程

> 文档版本：2026-07-02 | 面向读者：管理员角色

## 概述

管理员在后台管理所有线上/线下药房的基础信息、启用状态、库存管理开关，以及药房的显示排序（排至最前、排至最后、上移、下移）。药房的顺序决定了医生端药房选择列表的展示顺序。

## 触发条件

- **前置条件**：管理员已登录管理后台
- **入口路径**：左侧菜单 → 医院管理 → 药房管理 → 药房渠道

## 步骤详解

### 第一步：筛选药房

页面提供三个筛选条件：

- **药房名称**：输入框模糊搜索，最多20个字
- **状态**：下拉选择"禁用（0）/ 启用（1）"
- **药房类型**：下拉选择"全部 / 线下药房（2）/ 线上药房（1）"
  - 当系统配置为"严格模式"或"未启用线下药房"时，药房类型筛选自动隐藏，仅显示线上药房

点击搜索后列表刷新。

### 第二步：查看药房列表

表格列展示：

| 列名 | 说明 |
|------|------|
| ID | 药房编号 |
| 药房类型 | 线上药房 / 线下药房（严格模式下隐藏此列） |
| 药房名称 | - |
| 负责人 | 药房负责人姓名 |
| 状态 | 开关按钮，可直接切换启用/禁用 |
| 库存管理 | 显示"启用库存"或"禁用库存"的图标+文字状态 |
| 创建时间 | - |

### 第三步：新增药房

点击"新增"按钮（需权限 `med:pharmacies:write`），跳转到药房编辑页面（`/mgmt/hosp/pharmacy/channel/edit`），新增时需要填写：

- 药房名称（不可重复）
- 药房类型（线上/线下）
- 负责人手机号（不可重复绑定）
- 是否启用第三方药房（启用后需配置第三方药房的进货价和来源编码）
- 是否启用库存管理
- 药品类型（成药/中药饮片，含切片加工服务配置）
- 配送方式
- 地址信息

新增时药房的显示序号自动设为当前最大值+1（排在最后）。

### 第四步：编辑药房

点击"编辑"按钮跳转到编辑页面：
- 可修改药房名称（排己校验）、手机号（排己校验）、库存设置等
- 已启用第三方药房的不允许编辑为禁用
- 线下药房修改手机号时，系统自动处理用户与药房的绑定关系（解绑旧号码、注册/绑定新号码）

### 第五步：管理药品

点击"管理药品"按钮，跳转到该药房的药品管理页面（`/mgmt/hosp/pharmacy/channel/drugmanage`），可添加、编辑、导入导出该药房下的药品。

### 第六步：调整药房排序

药房的显示顺序可通过四个按钮操作：

- **排至最前**：将该药房排到所有药房的最前面
- **排至最后**：将该药房排到所有药房的最后面
- **上移**：与该药房的上一个位置交换
- **下移**：与该药房的下一个位置交换

如果已经排在最前/最后，对应按钮操作时会提示"已经排在最前啦"/"已经排在最后啦"。

排序结果实时保存，刷新列表后生效。

### 第七步：管理库存开关

列表中的"库存管理"列显示当前药房的库存管理模式：
- **启用库存**：绿色图标，开方时会校验药品库存是否充足
- **禁用库存**：红色图标，开方时不校验库存（适合线下药房或库存非实时场景）

库存管理开关需在编辑页面中修改。

### 第八步：页面缓存

从药品管理子页面返回药房列表时，列表保持缓存状态不刷新；从其他页面进入时自动刷新。

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 药房名称重复 | 新增/编辑时提示"药房名称重复" |
| 手机号已被绑定 | 提示"该手机号已绑定药房" |
| 已排在最前 | 提示"已经排在最前啦" |
| 已排在最后 | 提示"已经排在最后啦" |
| 编辑时药房不存在 | 提示"药房信息不存在，id=XXX" |
| 已启用第三方药房改为禁用 | 提示"已启用第三方药房的不允许编辑为禁用" |
| 严格模式/未启用线下 | 药房类型筛选和列自动隐藏 |

## 关联功能

- **药品管理**：每个药房下管理自己的药品库存
- **医生药房授权**：医生只能看到被授权药房的药品
- **库存管理**：药房库存开关控制开方时是否校验库存
- **第三方药房同步**：启用第三方药房后定时同步药品和库存数据
- **药房管理员**：线下药房自动注册管理员账号
