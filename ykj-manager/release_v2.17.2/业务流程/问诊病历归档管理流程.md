# 问诊病历归档管理流程

> 文档版本 2026-07-03 | 面向运维人员

## 概述

问诊病历归档管理用于追踪每一次线上问诊结束后，病历是否成功归档到医院的病历系统。管理员可以查看每份问诊记录的归档状态，并对归档失败或进行中的病历手动触发重新归档。

## 触发条件

医院反馈某次问诊的病历没有同步到院内系统、或运维巡检发现归档失败时使用。

- **前置条件**：拥有文件归档日志查询权限
- **入口路径**：后台管理系统 → 系统管理 → 文件归档日志

## 步骤详解

### 第一步：进入归档日志列表

页面展示所有问诊的病历归档记录，列表包含：

| 列 | 说明 |
|---|---|
| 序号 | 记录排序编号 |
| 问诊单号 | 线上问诊的唯一编号 |
| 患者姓名 | 就诊患者的姓名 |
| 就诊科室 | 患者就诊的科室 |
| 就诊医生 | 接诊的医生姓名 |
| 就诊时间 | 问诊发生的时间 |
| 归档时间 | 病历实际归档完成的时间 |
| 归档方式 | 自动归档 / 手动归档 |
| 成功标识 | 进行中 / 成功 / 失败 |

### 第二步：筛选归档记录

支持三种筛选条件：

1. **问诊单号**：输入问诊编号精确定位
2. **患者姓名**：输入患者姓名搜索
3. **成功标识**：下拉选择「全部」「进行中」「成功」「失败」

### 第三步：手动触发归档

对于归档状态为"进行中"的记录（即归档尚未完成），可以点击操作列的「归档」按钮手动触发重新归档。

点击后系统弹出确认提示"确认归档？"，确认后：
- 系统向后端发送归档指令
- 归档成功后提示"处理完成"
- 列表自动刷新，可看到归档状态更新

**注意**：只有状态为"进行中"的记录才能手动归档——成功或失败的记录无法重新归档。

## 归档状态说明

| 状态 | 含义 | 可手动操作 |
|---|---|---|
| 进行中 | 归档流程尚未完成，可能正在等待医院系统响应 | ✅ 可点击「归档」 |
| 成功 | 病历已成功同步到医院系统 | ❌ 无需操作 |
| 失败 | 归档失败，可能由于医院系统不可用或数据格式不匹配 | ❌ 无法手动重试 |

## 异常处理

| 情况 | 系统处理方式 |
|---|---|
| 归档进行中 | 显示黄色标签，点击「归档」按钮可手动触发 |
| 归档失败 | 操作列的归档按钮变为"/"（禁用状态），无法手动重试 |
| 确认弹窗取消 | 不会执行归档操作，返回列表 |

## 关联功能

- **问诊记录管理**：来源数据——问诊结束后系统自动发起病历归档
- **医院系统对接**：归档的目标是医院的电子病历系统（HIS/EMR）
