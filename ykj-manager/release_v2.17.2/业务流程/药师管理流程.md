# 药师管理流程

## 概述
管理后台运营人员对平台药师进行全生命周期管理，包括新增、编辑、删除药师，管理药师的启用/禁用状态，以及CA电子签章认证管理。

## 触发条件
平台需要新增药师或管理现有药师资审状态、联系方式等信息时。

## 前置条件
- 登录账号具有药师管理权限
- 已配置好医院、科室等基础字典数据

## 步骤详解

### 步骤一：查看药师列表
进入「医院管理 → 药师管理」页面，列表展示所有药师的信息：
- **姓名**：药师真实姓名
- **性别**：男/女（通过字典转义显示）
- **联系电话**：默认脱敏显示，点击可查看完整号码（需后端权限校验）
- **邮箱地址**
- **CA认证状态**（仅在系统开启CA签章功能时显示）：未认证 / 认证中 / 已认证 / 认证失败
- **CA用户同步状态**（仅在系统开启CA签章功能时显示）：显示CA系统同步的详细状态，如「身份审核通过」「证书签发」「设置签章」「申请拒绝：XXX原因」等
- **入驻时间**
- **启用/禁用开关**

支持按药师姓名搜索（最多5个字）。每页10条。

### 步骤二：新增药师
点击「新增」按钮，弹出药师信息填写表单。表单需要填写的字段通过后台字典动态加载，包括：
- 姓名、性别、出生日期等基础信息
- 所属医院、科室（树形菜单选择）
- 职称、职务
- 民族、学历
- 药师专业分类
- 联系电话、邮箱
- 从业资格证书等

填写完成后提交，系统自动保存并刷新列表。

### 步骤三：编辑药师信息
点击某位药师所在行的「编辑」按钮，弹出同新增一致的表单，回填当前信息。修改后保存。

### 步骤四：删除药师
点击「删除」按钮，系统弹出二次确认对话框：「此操作将永久删除该药师，是否继续？」。确认后执行删除，不可恢复。取消则不执行任何操作。

### 步骤五：启用/禁用药师
通过列表中的「状态」开关直接切换。启用操作与禁用操作分别调用不同的后端接口。操作成功后提示「操作成功」，列表自动刷新。

禁用后的药师无法正常使用审方功能。如果被禁用的药师尝试登录，系统会提示「当前药师角色已被禁用」。

### 步骤六：CA电子签章管理
仅在系统开启CA签章功能后显示以下操作：
- **同步资料**：将药师资料同步到CA签章系统（authStatus为1（未认证）或4（认证失败）时才能操作）
- **去认证**：引导药师完成CA签章认证流程（authStatus为2（认证中）时才能操作），操作会生成认证二维码供药师扫码认证

### 步骤七：变更手机号
点击「变更手机号」按钮，在弹出的对话框中输入新的11位手机号。系统实时校验：手机号格式（1开头11位）、手机号是否已被其他医生或药师占用。校验通过后才可保存。保存成功后会弹出提示告知操作完成。

## CA认证状态说明

| authStatus | 状态名 | 说明 |
|-----------|-------|------|
| 1 | 未认证 | 资料未同步至CA系统 |
| 2 | 认证中 | 资料已同步，等待药师完成认证 |
| 3 | 已认证 | 资料已同步且认证通过 |
| 4 | 认证失败 | 同步数据有误或不符 |

CA用户同步子状态包括：身份审核通过、证书签发、设置签章、用户注销、申请拒绝（会附带拒绝原因）、用户停用、修改手机号、用户启用、用户删除。

## 权限控制
| 操作 | 所需权限 |
|------|---------|
| 新增药师 | `hosp:pharmacists:write` |
| 查看完整手机号 | 后端权限校验 |
| 启用/禁用药师 | 药师管理编辑权限 |
| CA同步/认证 | CA功能开启 |

## 异常处理
- 手机号已被占用：校验时提示「手机号码已存在，请重新输入」
- 手机号格式错误：提示「手机号格式错误」
- 删除操作：二次确认后方可执行，取消后提示「已取消删除」
- CA认证失败：同步子状态中附带具体 failReason 失败原因

## 关联功能
- 处方管理：药师审核处方时需要有效的药师账号
- CA签章配置：CA功能开关决定认证相关列的显隐
- 系统角色管理：分配给药师的系统权限
