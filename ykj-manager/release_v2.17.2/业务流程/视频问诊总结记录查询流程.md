# 视频问诊总结记录查询流程

> 文档版本 2026-07-03 | 面向产品/运营/客服

## 概述

视频问诊结束后，系统会记录每次视频问诊的总结信息，包括患者手机号、问诊是否有效、医生建议摘要、视频房间ID和回调时间。管理员可通过此功能查询和筛选所有视频问诊总结记录。

## 触发条件

- **入口路径**：管理后台 → 运营管理 → 视频问诊总结
- **使用场景**：运营人员需要查看视频问诊的完成情况和医生给出的建议摘要，判断问诊质量

## 步骤详解

### 第一步：进入视频问诊总结页面

管理员进入页面后，系统自动加载最近10条视频问诊总结记录。

表格展示的列包括：
- **手机号**：问诊患者的手机号码
- **问诊是否有效**：显示"是"或"否"
- **医生建议**：视频问诊结束后医生给出的总结或建议内容
- **视频房间ID**：对应视频通话的房间编号
- **回调时间**：视频问诊服务回调通知系统的时间

### 第二步：按条件筛选记录

管理员可使用以下筛选条件：

- **手机号**：输入患者手机号精确查询某次问诊
- **问诊是否有效**：下拉选择"是"（有效问诊）或"否"（无效问诊）
- **回调时间**：选择日期范围，查看某个时间段内的视频问诊总结

点击"查询"后分页展示符合条件的记录。

### 第三步：查看问诊详情

从视频房间ID可关联到对应的视频问诊记录，结合问诊是否有效和医生建议，运营人员可评估问诊质量。

## 数据说明

| 字段 | 说明 |
|------|------|
| 手机号 | 患者的联系方式 |
| 问诊是否有效 | "是"表示本次视频问诊为有效问诊，"否"表示可能为测试或无效通话 |
| 医生建议 | 医生在视频问诊后做出的诊疗建议或总结内容 |
| 视频房间ID | 唯一标识一次视频通话，可用于关联视频回放 |
| 回调时间 | 视频服务商回调通知的时间，非问诊实际结束时间 |

## 异常处理

| 情况 | 系统处理方式 |
|------|------------|
| 无筛选结果 | 表格显示空数据，提示无匹配记录 |
| 回调时间为空 | 正常显示为空，不影响数据展示 |
| 医生建议为空 | 正常显示为空，可能医生未填写 |
| 数据加载失败 | 表格显示加载状态后提示错误 |

## 关联功能

- **视频问诊**：视频问诊结束后触发总结记录生成
- **门诊问诊室**：医生工作台可发起视频问诊（管理后台-门诊问诊室）
- **视频回放**：通过视频房间ID关联视频回放记录
