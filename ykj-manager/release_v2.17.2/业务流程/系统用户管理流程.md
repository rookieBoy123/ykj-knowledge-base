# 系统用户管理流程

> 文档版本：2026-07-01 | 面向读者：管理员

## 概述

管理员在后台创建和维护登录管理后台的账号（系统用户），为每个用户分配用户名、密码、角色，控制谁能登录后台以及能操作哪些功能。用户自身的密码管理和角色切换也包含在此流程中。

## 触发条件

- **前置条件**：拥有 `sys:users:write` 权限的管理员
- **入口路径**：系统管理 → 用户管理

## 步骤详解

### 第一步：查看用户列表

搜索区支持按「用户名」（最长50字）「角色」「状态（启用/禁用）」进行筛选。列表展示每个用户的以下信息：

- 用户名、姓名、邮箱、手机号（脱敏显示，点击可解密查看明文单条）
- 用户状态（启用/禁用开关）
- 创建时间、最后修改时间
- 状态开关即时生效

### 第二步：新增用户

点击「新增」按钮弹出对话框，需要填写：

- **用户名**（必填，最长50字，新增后不可修改）
- **姓名**（必填，最长50字）
- **邮箱**（可选，用于接收注册通知和密码重置邮件）
- **手机号**（必填，最长100字，新增后不可修改）
- **角色**（多选级联选择器，从已有的角色列表中勾选）
- **地址**（可选，最长1000字）
- **状态**（启用/禁用）

保存时系统做以下校验：
- 用户名是否已存在（调用 `changeUserName` 的同名校验逻辑）
- 手机号是否已注册："该手机号码已经被注册过，请重新填写手机号码"
- 角色合法性：禁止手动分配的系统角色会提示"以下角色不允许管理后台新增"
- 注册成功后自动发送邮件通知（如填写了邮箱）：告知登录地址和默认密码
- 密码设置为系统默认密码（在系统配置中指定），且密码立即过期——登录后会立刻要求用户修改密码

### 第三步：编辑用户

可以修改姓名、邮箱、地址、角色、状态。用户名和手机号在编辑模式下不可修改（灰色禁用）。

角色更新时使用"增量计算"策略：
- 对比现有角色和提交的角色，算出需要新增授予和需要回收的角色清单
- 禁止分配的系统角色（DISALLOW_ASSIGN_ROLES）不会被增减

### 第四步：用户状态管理

- **启用/禁用**：开关即时切换。禁用后该用户无法登录，提示"账号已被锁定，请联系管理员"
- **状态变更通知**：状态变更时会触发 `permVersionNotifier` 通知权限缓存更新

### 第五步：密码管理

- **管理员重置密码**：将用户密码重置为系统默认密码，密码立即过期（`expirationDate = new Date()`），用户下次登录会被强制要求修改密码。重置后自动发送通知邮件。
- **用户自己修改密码**：需要输入旧密码 → 验证旧密码正确 → 输入新密码 → 新密码不能与旧密码相同 → 设置成功，密码过期时间清除

### 第六步：登录安全机制

- **密码错误锁定**：连续登录失败5次（`loginErrorCount >= 4` 时触发设置），账号被锁定 `blockEndTime = 当前时间 + 10分钟`，10分钟后自动解锁
- **登录成功重置**：登录成功后 `loginErrorCount` 重置为0
- **密码过期**：当 `expirationDate` 不为空且已过期时，登录后会强制要求修改密码

### 第七步：角色切换

用户登录后可以在自己拥有的多个角色之间切换（`switchRole`），系统会验证：
- 账号是否被禁用："账号已被锁定，请联系管理员"
- 是否拥有该角色："您没有权限进行当前操作"
- 切换成功后更新 `currentRoleId`，刷新菜单权限

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 用户名已存在 | 提示"用户名已存在" |
| 手机号已被注册 | 提示"该手机号码已经被注册过，请重新填写手机号码" |
| 手机号已绑定其他用户 | 提示"手机号码已有用户" |
| 账号被禁用登录 | 登录时抛出 `BizException.forbidden("账号已被锁定，请联系管理员")` |
| 连续输错密码5次 | 账号锁定10分钟，`blockEndTime = now + 10min` |
| 新密码与旧密码相同 | 提示"新密码不能与旧密码相同" |
| 原密码错误（修改密码时） | 提示"原密码错误" |
| 没有权限的角色切换 | 提示"您没有权限进行当前操作" |
| 切换用户名时重名 | 提示"用户名不能重复" 或 "用户名已存在" |

## 关联功能

- **角色管理**：创建和维护可分配给用户的角色
- **登录日志**：记录用户登录历史
- **菜单管理**：定义角色能看到的功能树
