# 随访问卷创建与发布管理流程

> 文档版本：2026-07-02 | 面向读者：运营人员、医疗机构管理员

## 概述

随访问卷是医生对患者进行随访时可下发的标准化问卷。管理后台负责随访问卷的创建、编辑、发布和禁用管理。已发布的问卷可被医生在小程序中选择加入随访计划，下发给患者填写。

## 触发条件

- 入口：管理后台 → 诊疗管理 → 随访管理 → 问卷
- 权限：拥有随访管理权限的管理员

## 步骤详解

### 第一步：新增问卷

管理员点击「新增」按钮，跳转到问卷编辑页面。在编辑页面中配置：
- 问卷标题
- 问卷题目列表（可添加多道题目）
- 每道题目的类型、选项、是否必答等
- 题目排列顺序

编辑过程中可随时保存草稿，草稿状态的问卷仅管理员可见，不会出现在医生小程序中。

### 第二步：发布问卷

完成所有题目配置后，管理员点击「发布」按钮。系统执行完整性校验：
- 检查问卷是否所有必填内容已填写完整
- 如有未填写的内容，弹窗提示"该问卷有内容未填写，请确认填写完整再发布"，引导管理员返回完善
- 全部内容填写完整后，问卷状态变为「已发布」

已发布的问卷会同步显示在医生小程序端，医生可在创建随访计划时选取。

### 第三步：管理问卷生命周期

管理后台支持问卷的完整生命周期管理：

**草稿 → 发布**：通过发布按钮完成，附带完整性校验
**已发布 → 禁用**：点击禁用按钮，问卷从医生可见列表中移除。**已禁用问卷的随访任务不会立即受影响**——只有在新一轮随访任务发送时，系统才会检测到问卷已禁用并排除该内容
**禁用 → 启用**：重新点击启用按钮（内部同样走发布逻辑），问卷恢复可见
**已发布/禁用 → 编辑**：仅草稿状态的问卷可编辑。已发布或禁用的问卷如需要修改，需先由管理员操作状态变更

### 第四步：查看问卷详情

管理员可通过「查看详情」按钮预览问卷的完整内容和结构，了解有哪些题目、选项类型等信息，但不能修改。

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 问卷内容未填写完整 | 发布/启用时弹窗提示，阻止操作，引导完善 |
| 权限不足 | 新增/编辑/发布等操作按钮不在页面显示（按钮关联权限码） |
| 页面缓存与数据更新 | 从问卷详情页返回时使用缓存数据；从其他页面进入时重新请求最新数据 |

## 页面缓存策略

随访管理模块有独特的页面缓存逻辑：
- 从「题目编辑页」返回到问卷列表时，**使用 keep-alive 缓存**，不重新请求数据（因为题目编辑不会改变问卷列表的排序/标题等）
- 从其他页面（如菜单切换）进入时，**重置搜索条件并重新请求最新数据**
- 通过 `beforeRouteEnter` 路由守卫判断上一页来源决定缓存策略

## 关联功能

- 随访计划：医生在小程序选择已发布的问卷加入随访
- 随访任务发送：系统定时发送时校验问卷启用状态
- 患者问卷填写：患者在小程序填写问卷并提交答案
- 题目库管理：问卷题目可在题目库中复用
