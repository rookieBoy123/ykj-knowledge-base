# 财务看板与发票查询管理流程

> 文档版本：2026-07-02 | 面向读者：财务管理员、运营管理员

## 概述

本流程描述管理后台的财务数据看板（交易趋势和订单来源分析）以及电子发票查询与导出功能。帮助财务人员快速掌握平台收入概况和发票管理状态。

## 触发条件

- **财务管理员**登录后台查看财务看板
- **运营人员**需要查询或导出发票信息

## 步骤详解

### 一、财务看板（数据概览）

财务看板包含两个图表：

#### 交易数据柱状图（按月统计）

1. **默认显示**：打开页面时默认显示当前年份的1-12月交易数据。
2. **年度切换**：可通过年份选择器切换查看其他年份的数据。
3. **数据维度**：显示每月收入总额（sumAmount），单位为元。后端接口为 findEachMonthIncomeSumamountGroupByYear。
4. **展示形式**：每月一根柱子，带渐变色（浅蓝→深蓝）。

#### 订单来源饼图（按类型统计）

1. **默认显示**：当前年份各类型订单的占比。
2. **订单类型映射**：
   - 1 → 药品订单
   - 2 → 问诊订单
   - 3 → 会诊订单
   - 4 → 预约订单
   - 6 → 服务包订单
3. **展示形式**：五种颜色区分的饼图（青/粉/紫/黄/绿），显示百分比。
4. **数据来源**：findIncomeAppointSumamountGroupByYear 接口。

### 二、电子发票查询

#### 筛选条件

1. **患者姓名**：精确搜索，最多输入5个字。
2. **订单编号**：精确搜索，最多输入30字。
3. **开票日期**：日期范围筛选（DateRange类型）。

#### 发票列表

每页10条，展示以下信息：

| 列名 | 说明 |
|------|------|
| 电子发票号码 | 发票唯一编号（billNo） |
| 交易流水号 | 支付流水号（payOrderNo） |
| 订单编号 | 关联的订单号 |
| 订单类型 | 处方订单/问诊订单等 |
| 门诊号码 | 关联的HIS门诊号（无数据显示"/"） |
| 患者姓名 | 脱敏或显示真实姓名（无数据显示"/"） |
| 支付渠道 | 微信/支付宝/医保等 |
| 交易金额（元） | 实际支付金额 |
| 订单支付时间 | 支付完成时间 |
| 开票日期 | 发票开具日期 |

#### 操作功能

1. **打印/查看发票**：点击"打印"按钮，在新窗口中打开电子发票的图片（pictureNetUrl）。仅当发票有图片URL时才显示此按钮。
2. **导出Excel**：点击"导出"按钮，将当前筛选结果导出为Excel文件。导出时显示加载提示，导出完成后提示消失。文件名为"发票信息.xls"。

### 三、数据导出

1. **导出范围**：导出当前筛选条件下的全部数据（不受分页限制）。
2. **导出格式**：Excel（.xls格式）。
3. **导出字段**：与列表展示字段一致。

## 异常处理

| 情况 | 系统处理方式 |
|------|------------|
| 年份选择器清空后 | 自动重置为当前年份，重新加载数据 |
| 发票无电子图片 | 操作列不显示"打印"按钮 |
| 患者姓名或门诊号码为空 | 列表中显示"/" |
| 导出数据量过大 | 显示加载中提示，导出完成后关闭 |

## 关联功能

- **订单管理**：发票关联到具体订单
- **支付管理**：发票记录与支付流水号关联
- **电子发票开具**：发票通过第三方电子发票平台生成
