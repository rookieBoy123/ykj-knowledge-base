# 系统权限与菜单结构查询流程

> 文档版本：2026-07-03 | 面向读者：系统管理员、开发对接人员

## 概述

菜单管理定义了后台管理系统的页面导航结构，权限管理记录了系统所有可控制的细粒度权限点。两者与角色管理共同构成完整的 RBAC（基于角色的访问控制）体系。菜单管理决定"能看到哪些页面"，权限管理决定"能调用哪些接口"。

## 触发条件

- 需要了解系统有哪些菜单和权限
- 排查某个用户看不到某个菜单的问题
- 对接开发时查询权限码

入口：管理后台 → 系统管理 → 菜单管理 / 权限管理

## 步骤详解

### 一、菜单管理

#### 页面布局

左侧为菜单树（可切换"管理端"和"线下药房"），右侧为选中菜单节点的子菜单列表表格。

#### 菜单树切换

顶部两个Tab：
- **管理端**：互联网医院管理后台的菜单结构
- **线下药房**：线下药房管理端的菜单结构

切换Tab后，左侧树和右侧列表均重新加载对应客户端的菜单数据。

#### 菜单列表查看

点击左侧树的某个节点后，右侧表格显示该节点的直接子菜单。每条记录包含：

| 列名 | 说明 |
|------|------|
| 类型 | 目录（0）/ 菜单（1）/ 按钮（2） |
| 父级菜单名称 | 当前菜单的上级 |
| 菜单名称 | 菜单显示名称 |
| 菜单URL | 前端路由路径 |
| 权限代码 | 对应的权限标识（如 `sys:menu:read`） |

菜单的增删改操作通过操作列完成（具体操作界面需要路由到编辑页）。

#### 菜单层级关系

系统菜单分为三级：
- **目录**：导航栏中的一级分组（如"系统管理"）
- **菜单**：目录下的具体页面（如"用户管理"）  
- **按钮**：页面内的操作权限点（如"新增用户按钮"）

### 二、权限管理

#### 权限列表查看

纯查询页面，展示系统所有已注册的权限点（接口级权限）：

| 列名 | 说明 |
|------|------|
| 权限名称 | 权限点的中文描述 |
| 父权限 | 上级权限名称 |
| 请求链接 | 对应的后端API路径 |
| 请求方法 | GET/POST/PUT/DELETE/PATCH |
| 权限码 | 权限标识（如 `sys:users:write`） |
| 创建时间 | 权限注册时间 |
| 修改时间 | 最后修改时间 |

#### 筛选条件

可按四个维度筛选：
- **权限名称**：模糊搜索
- **权限码**：精确搜索权限标识
- **请求链接**：按接口URL搜索
- **请求方法**：按HTTP方法筛选（GET/POST/PUT/DELETE/PATCH）

权限数据由系统自动同步，前端仅提供查看功能（原"同步权限"按钮已注释，表明权限由后端自动维护）。

### 三、用户登录时的菜单加载

用户登录管理后台时，系统根据以下逻辑返回菜单：

1. **超级管理员**：返回对应客户端（管理端/线下药房）的全部菜单
2. **普通用户**：
   - 查询用户当前使用的角色编码
   - 根据角色ID查询该角色已分配的菜单权限
   - 只返回该角色有权限访问的菜单

用户切换角色后，菜单会重新加载，只显示新角色对应的菜单。

## 异常处理

| 情况 | 系统处理方式 |
|------|------------|
| 搜索无结果 | 列表显示空，不会报错 |
| 切换客户端Tab后菜单不刷新 | Tab切换时自动清空右侧表格数据并重新加载 |
| 超级管理员登录后看不到某个菜单 | 检查该菜单是否在当前客户端（管理端/线下药房）下注册 |
| 普通用户看不到菜单 | 检查其当前角色是否分配了该菜单权限 |

## 关联功能

- **角色管理**：在角色权限设置中选择菜单树和权限
- **用户管理**：用户看到的菜单由角色→菜单的关系决定
- **登录流程**：用户登录时调用 `_user-nav-tree` 接口获取个性化菜单
