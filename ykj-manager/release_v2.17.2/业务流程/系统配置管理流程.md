# 系统配置管理流程

> 文档版本 2026-07-02 | 面向读者：产品经理、运营、客服

## 概述

描述管理员在后台管理平台统一管理所有系统开关和配置的功能。涵盖二级标签页展示、配置项修改、保存、历史版本回溯与还原的完整操作流程。

## 触发条件

- **管理员**：需要调整系统功能开关（如启用/禁用医保支付、货到付款、默认药房等）

## 步骤详解

### 第一步：进入配置管理页面

页面展示两级标签页：
- **一级标签**：功能大类（如"支付设置""业务配置""系统设置"等）
- **二级标签**：每个大类下的子配置组（如"支付设置"下分"微信支付""医保支付""货到付款"等）

每个标签的显隐由条件表达式控制（`dependOn` 字段动态计算）。

### 第二步：查看和修改配置

- **配置项类型**：布尔开关（Boolean）、文本输入（String）、选择框（Select）、URL链接、时间范围（TimeRange）、日期范围（DateRange）、小数（Decimal）、JSON配置
- **货到付款配置**：由于产品设计特殊，单独使用 `CashDelivery` 组件展示（需同时配置订单总价门槛和支付定金金额）
- **URL类型**：自动校验URL格式（必须 http/https 开头），不符合格式提示"链接错误"
- **修改记录**：所有修改暂时缓存在前端，点击"保存"时才统一提交。系统通过 `handleCacheModifies` 方法跟踪每一项修改

### 第三步：保存校验

点击保存前执行多重校验：

1. **是否有修改项**：若没有修改项，提示"没有修改项"
2. **格式校验**：每项修改的数据格式是否正确
3. **默认药房关联**：启用了"默认药房开关"但未配置默认药房ID时，提示"默认药房启用后，需要配置默认药房ID"
4. **货到付款关联**：启用了货到付款开关后，订单总价和定金金额为必填，且定金金额必须 ≤ 订单总价，否则提示"订金金额必须少于等于订单总价设置金额"

### 第四步：保存并刷新

1. 提交所有修改项（`allEdit` 接口）
2. 保存成功后刷新全局系统配置缓存（重新拉取 `headInfo` 并写入 `sessionStorage`）
3. 重新加载配置页面（清空临时修改缓存）

### 第五步：修改历史

点击"修改历史"按钮打开侧边抽屉，展示所有历史修改记录：

1. **分页加载**：支持滚动加载更多（无限滚动 `infinite-scroll`）
2. **每条记录展示**：操作时间、操作人、各项修改的前后值对比
3. **数量超过3条折叠**：默认只展示前3项修改，点击展开按钮查看全部
4. **还原操作**：点击"还原"按钮 → 弹出确认框展示当前版本的所有配置项 → 确认后调用 `restore` 接口还原到该历史版本

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 没有任何修改就点保存 | 提示"没有修改项" |
| 修改项格式错误 | 提示具体的错误信息（isError字段） |
| 启用默认药房但未配置ID | 提示"默认药房启用后，需要配置默认药房ID" |
| 货到付款未填必填项 | 提示"货到付款启用配置后，输入框数据为必填" |
| 订金 > 订单总价 | 提示"订金金额必须少于等于订单总价设置金额" |
| URL格式错误 | 提示"链接错误" |
| eval依赖表达式异常 | 返回false，隐藏该标签（避免页面崩溃） |

## 关联功能

- 小程序功能开关（配置修改后影响小程序端功能显示）
- 支付配置（微信支付/医保支付/货到付款等）
- 历史版本追溯（可还原到任意历史配置）
