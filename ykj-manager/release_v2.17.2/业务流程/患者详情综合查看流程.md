# 患者详情综合查看流程

> 文档版本：v1.0 | 面向读者：管理员

## 概述

管理员在后台查看指定患者的完整档案：账号信息、患者信息、慢病病种、标签，以及四个维度的就诊记录（线上/线下病历、量表、随访、报告）。

## 触发条件

- **前置条件**：以管理员角色登录，从患者列表点击进入
- **入口路径**：患者管理 → 患者列表 → 点击某患者 → 患者详情页
- **路由参数**：id（患者唯一标识）

## 步骤详解

### 第一步：加载患者基本信息

页面通过`findPatientByUniqueKey`接口加载患者完整数据。页面分为多个区域：

**1. 账号信息区**：
- 头像（avatar）
- 昵称（nickname），为空显示"-"
- 手机号（phone）：默认脱敏显示，旁边有眼睛图标
  - 点击睁眼图标→调用`memberPhone`接口解密→显示完整手机号
  - 点击闭眼图标→恢复脱敏
  - 点击页面任意空白处→所有已解密的手机号自动恢复脱敏

**2. 患者信息区**：
- 就诊卡号、创建时间
- 姓名（realName）、性别（1男/2女）、出生日期（birthday）、年龄（ageStr）
- 身份证号（idCard）：如果非空显示"已校验"标签
- 问诊过的医生列表（doctorPos），格式如"张医生（主任医师）、李医生（副主任医师）"
- 如有监护人（hadGuardian=1）：显示监护人姓名、关系（1父亲/2母亲/3其他）、身份证号

### 第二步：查看慢病和标签

**慢病病种区**：
- 展示`patientChronicVos`中已登记的慢病标签列表
- 如糖尿病、高血压等
- 无数据时显示"无"

**患者标签区**：
- 展示管理员为患者打的标签（patientLabels）
- 多个标签以tag形式展示
- 无数据时显示"无"

### 第三步：浏览病历记录（默认选项卡）

**线上病历**（medicalTabIdx=1）：
- 列表展示：门诊号、接诊医生、主诉、诊断、病历日期
- 每页10条分页
- 点击记录→跳转病历详情（recordsdetail页面）
- 如有问卷关联→点击查看问卷详情（打开弹窗展示surveys组件）

**线下病历**（medicalTabIdx=2）：
- 从HIS系统同步的线下病历列表
- 使用offlineMedical组件独立加载
- 通常只读查看

两种病历通过单选按钮切换（el-radio-group）。

### 第四步：查看量表记录

量表选项卡（name="2"）仅在系统开启了`scaleEnabled`开关时显示。

展示患者的量表测评记录列表（scaleRecording组件），可点击查看量表详情。

### 第五步：查看随访任务记录

随访选项卡（name="3"）仅在系统开启了`followupEnabled`开关时显示。

展示患者的随访任务执行记录（followupTaskRecord组件），按患者ID查询。

### 第六步：查看报告记录

报告选项卡（name="4"）仅在系统开启了`examineReportEnabled`开关时显示。

展示患者的检查/检验报告记录（ReportRecords组件）。

### 第七步：页面缓存策略

页面使用keep-alive缓存，从不同详情页返回时行为不同：

- 从病历详情/量表详情/线下病历详情返回：**保持缓存**（status='cache'），维持之前打开的选项卡和滚动位置
- 从其他页面进入：**重新加载**（status='update'），重置到病历记录选项卡

## 异常处理

| 情况 | 系统处理方式 |
|------|------------|
| 患者不存在 | 接口返回空数据，页面各区域显示"-" |
| 无慢病病种 | 显示"无" |
| 无患者标签 | 显示"无" |
| 功能开关关闭 | 对应选项卡不显示 |
| 手机号解密失败 | 保持脱敏状态 |

## 关联功能

- **病历详情**：跳转到recordsdetail页面
- **量表记录**：量表功能开关控制
- **随访记录**：随访功能开关控制
- **报告记录**：检查报告开关控制
