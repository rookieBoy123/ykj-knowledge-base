# 系统全局配置管理流程

> 文档版本 2026-07-03 | 面向管理人员

## 概述

系统全局配置管理是管理后台的核心设置模块，运营人员可以在这里集中管理平台的所有业务参数和功能开关，包括支付方式、问诊设置、处方规则、通知模板等各类全局配置项。所有修改都会自动记录历史版本，支持随时查看和还原。

## 触发条件

- 需要调整平台某项业务参数（如开启货到付款、修改问诊时间范围）
- 需要查看某个配置项的历史变更记录
- 新功能上线需要开启对应的全局开关
- 需要在不同角色（如不同医院、不同地区）间切换配置

## 步骤详解

### 第一步：进入配置管理
进入管理后台的「系统管理 → 配置管理」页面。页面采用两级选项卡组织配置项：一级选项卡按大类分组（如"问诊设置"、"支付设置"、"处方设置"等），每个大类下再进行二级选项卡细分。

### 第二步：浏览与定位配置项
配置项以表单形式展示在二级选项卡下，每个配置项包含：
- **配置标题**：说明该配置项的作用
- **输入控件**：根据配置类型自动适配——开关按钮、文本输入框、URL输入框、时间范围选择器、下拉选择、JSON编辑区等
- **当前值**：显示该项当前的生效值

部分配置项的显示是**有条件的**：只有开启了某个前置开关，依赖它的配置项才会显示（如开启"默认药房"后，才会出现"默认药房ID"的填写框）。

### 第三步：修改配置值
直接在页面上修改需要调整的配置项。修改过的项会被记录到未保存列表中。可以一次性修改多个配置项，然后统一保存。

支持以下配置类型：
- **开关（Boolean）**：开启/关闭功能，如"开启货到付款"、"开启默认医院"
- **文本**：输入任意文本，如"平台名称"、"版权声明"
- **URL**：输入网址，系统自动校验格式（必须以http/https开头）
- **时间范围（TimeRange）**：设置开始-结束时间，如"问诊服务时间"
- **日期范围（DateRange）**：设置开始-结束日期
- **下拉选择（Select）**：从预设选项中选择
- **小数（Decimal）**：输入数字，如"医生收益分成比例"
- **JSON/文件**：结构化配置

### 第四步：保存前系统自动校验
点击「保存」按钮后，系统会自动进行以下检查：

1. **是否有修改**：如果没有任何配置项被修改，提示"没有修改项"
2. **格式校验**：每个配置项的输入值是否符合规定的格式（如URL格式、数值范围）
3. **关联必填校验**：
   - 开启「默认药房」后，必须填写「默认药房ID」
   - 开启「货到付款」后，必须填写「订单最低总价」和「支付定金」，且定金金额不能超过订单总价

校验通过后，所有修改项统一提交保存。

### 第五步：保存生效
保存成功后，系统自动刷新配置列表，并更新页面顶部的系统信息缓存。页面上未保存的修改项列表被清空，所有修改即时生效。

### 第六步（可选）：查看修改历史
点击页面右上角的「修改历史」按钮，从右侧弹出抽屉面板。历史记录以时间线形式展示：

- **时间戳**：每次修改的时间
- **变更明细**：每条记录显示修改了哪些配置项、修改前后的值
- **操作人**：显示是谁执行的修改
- **展开/收起**：如果一次修改了超过3个配置项，默认只显示前3条，点击展开箭头查看完整列表

支持**无限滚动加载**更多历史记录，每页加载10条。

### 第七步（可选）：还原历史版本
在修改历史中点击某条记录的「还原」按钮，弹出确认对话框展示该版本的所有配置项和对应的值。确认后，系统将该版本的所有配置值重新写入到当前生效配置中，相当于"回滚"到历史状态。

## 异常处理

| 情况 | 系统处理方式 |
|---|---|
| 没有修改任何配置项就点保存 | 提示"没有修改项"，阻止提交 |
| 输入的配置值格式不正确 | 在输入框旁显示对应的错误提示（如"链接错误"） |
| 开启默认药房但不填药房ID | 提示"默认药房启用后，需要配置默认药房ID"，阻止提交 |
| 货到付款设置中订金大于订单总价 | 提示"订金金额必须少于等于订单总价设置金额"，阻止提交 |
| 修改历史列表为空 | 显示空状态图标和"暂无修改历史"文字 |
| 加载更多历史记录时已到末尾 | 显示"没有更多了" |
| 还原时发现历史数据已不存在 | 提示"回滚历史数据不存在"（后端拦截） |

## 关联功能

- **系统用户管理**：配置管理的操作权限和用户角色关联
- **定时任务管理**：部分配置项会影响定时任务的触发条件和频率
- **消息通知**：配置项中的通知模板会直接影响消息通知功能
- **支付管理**：支付方式、金额限制等配置直接影响用户支付流程
- **医生端/患者端小程序**：问诊时间、价格区间等配置直接影响小程序的展示和限制
