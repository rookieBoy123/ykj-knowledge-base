# 角色管理与权限分配流程

> 文档版本：2026-07-03 | 面向读者：系统管理员

## 概述

角色管理模块定义了后台管理系统的权限骨架。每个角色关联一组菜单权限（决定能看到哪些页面和按钮），管理员可以将角色分配给用户，从而控制用户在系统中"能看什么"和"能做什么"。

系统将角色分为两个客户端：**互联网医院管理端**和**线下药房管理端**，两种客户端的角色相互独立。

## 触发条件

- 新增运营角色（如"财务专员"）
- 调整某个角色的菜单权限
- 禁用不再使用的角色
- 查看某个角色已分配了哪些菜单

入口：管理后台 → 系统管理 → 角色管理

## 步骤详解

### 第一步：角色列表查看与切换客户端

进入角色管理页面，顶部有两个Tab标签页供切换：

- **互联网医院**：管理后台运营人员的角色（默认选中）
- **线下药房**：线下药房管理端的角色

列表显示每条角色信息：

| 列名 | 说明 |
|------|------|
| ID | 角色唯一编号 |
| 角色名称 | 如"超级管理员"、"运营专员"，系统内置角色旁边有蓝色"系统角色"标签 |
| 角色编码 | 如"super_admin"、"operation"，用于系统内部权限判断 |
| 角色说明 | 对角色用途的文字描述 |
| 角色状态 | 开关按钮，可启用/禁用 |
| 创建时间 | 角色创建时间 |
| 最后修改时间 | 最后一次编辑时间 |

**筛选条件**：可按角色名称和状态筛选。

### 第二步：新增角色

点击"新增"按钮，填写：

- **角色名称**：必填，3-20个字符，输入框失焦时自动校验是否已存在（同客户端下不可重名）
- **角色编码**：必填，系统据此判断权限
- **角色说明**：选填，最多100字，描述角色用途
- **状态**：启用/禁用

### 第三步：编辑角色

点击编辑，可修改角色名称、编码、说明、状态。编辑模式下，只有角色名称发生变化时才会触发重名校验。**系统内置角色（builtIn标记）的编辑按钮为灰色不可点击**。

### 第四步：权限设置（核心功能）

点击某个角色的"权限设置"按钮，弹出权限配置对话框。对话框包含两个Tab：

#### 功能权限 Tab

左侧为**菜单树**（从菜单管理模块加载），以树形结构展示所有可访问的页面和按钮。右侧为**已选功能列表**表格。

操作流程：
1. 在左侧树中勾选/取消勾选菜单节点
2. 勾选父节点时，其所有子节点自动被选中
3. 右侧表格同步显示被选中的功能列表
4. 点击"保存"提交权限变更

**父子节点联动**：当某个父节点的所有子节点都被选中时，父节点也会被标记为选中状态，回显时自动勾选。

#### 数据权限 Tab

当前为预留（显示"数据权限"文字），尚未实现功能。

### 第五步：删除角色

删除前系统先调用"删除检查"接口，判断该角色是否已被分配给用户：

- **未被使用**：弹出确认框，确认后删除
- **已被分配**：提示"数据已被使用，不能删除"，不允许删除
- **系统内置角色**（builtIn标记）：删除按钮灰色不可点击，也不可被批量删除选中

**批量删除**（v2.17.2新增）：勾选多条角色记录后，点击"批量删除"。操作前同样会检查选中角色中是否有已分配或内置角色，如有则提示"勾选中存在不可删除的角色"。

### 第六步：启停角色

直接拨动列表中的状态开关即可启用或禁用角色。禁用角色后，拥有该角色的用户在下次请求时将无法通过对应的权限校验。

## 异常处理

| 情况 | 系统处理方式 |
|------|------------|
| 角色名称重复 | 输入框失焦时自动调用校验接口，提示"角色名已存在，请重新输入" |
| 角色名长度不符合 | 提示"长度在 3 到 20 个字符" |
| 保存权限时未选择任何菜单 | 提示"请至少选择一条数据" |
| 删除已被分配的角色 | 提示"数据已被使用，不能删除" |
| 批量删除时选中系统角色 | 提示"勾选中存在不可删除的角色" |
| 未选择任何角色执行批量删除 | 提示"请至少选择一条数据" |

## 关联功能

- **用户管理**：创建/编辑用户时从角色列表中选择角色分配
- **菜单管理**：权限设置的菜单树来自菜单管理模块
- **权限管理**：角色保存的菜单权限最终体现为权限码的集合
- **系统操作日志**：所有角色的增删改操作均有审计日志
