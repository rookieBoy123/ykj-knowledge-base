# 用户协议文档管理流程

> 文档版本：2026-07-03 | 面向读者：运营、法务、管理员

## 概述

用户协议文档管理用于维护小程序和管理后台中需要向用户展示的法律协议文本，如《用户服务协议》《隐私政策》《医生服务协议》等。管理员可以新增、编辑协议文档，使用富文本编辑器编写协议内容。协议通过唯一的"协议标识"（code）来区分，小程序端通过标识获取对应的协议内容。

## 触发条件

- 需要新增一份用户协议
- 法务要求修改现有协议条款
- 需要调整协议类型分类

## 使用角色

- 运营人员、系统管理员（具有操作权限）
- 协议内容通常由法务部门提供，管理员负责录入

## 步骤详解

### 第一步：查看协议列表

进入"协议管理"页面，以表格形式展示所有已创建的协议，每行显示：
- **协议标识**：唯一编码，如 `USER_AGREEMENT`、`PRIVACY_POLICY`
- **协议类型**：从字典"协议类型（xylx）"中选择，如"患者端协议""医生端协议"
- **标题**：协议的显示标题，如"用户服务协议"
- **操作**：编辑按钮

列表支持分页浏览。

### 第二步：新增协议

点击新增按钮，弹出新增窗口，需填写：
- **协议标识**：唯一编码（必填），编辑时不可修改。用于小程序端按标识获取协议内容
- **协议类型**：从下拉菜单中选择（数据来源：字典`xylx`），编辑时不可修改
- **标题**：协议显示名称（必填）
- **协议内容**：使用富文本编辑器（TinyMCE）编写，支持文字排版、图片插入等

提交前校验：标题和协议标识为必填，不填会有红色提示。

### 第三步：编辑协议

点击某条协议的"编辑"按钮，弹窗中回填已有数据。可修改的内容：
- **标题**：可以修改
- **协议内容**：使用富文本编辑器修改

不可修改的内容（编辑时置灰禁用）：
- **协议标识**：创建后不可更改
- **协议类型**：创建后不可更改

### 第四步：保存协议

- **新增**：调用保存接口，成功后关闭弹窗并刷新列表
- **编辑**：调用编辑接口，成功后关闭弹窗并刷新列表

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 未填写标题 | 提示"请输入标题" |
| 未填写协议标识 | 提示"请输入标识" |
| 提交时未选择协议类型 | 默认值为"1"（患者端协议），但如果没有类型数据会提交空值 |
| 新增保存失败 | 弹出错误提示（由接口返回） |
| 编辑保存失败 | 弹出错误提示 |
| 删除协议 | 二次确认"此操作将永久删除该文件，是否继续？"，确认后调用删除接口 |
| 图片上传格式错误 | 提示"上传头像图片只能是 jpg/png 格式" |
| 图片上传大小超限 | 提示"上传头像图片大小不能超过 2MB" |

## 小程序端如何使用

小程序端通过协议标识（code）调用 API 获取协议内容并渲染展示。新的协议创建后，只要小程序端有对应的 code 调用逻辑即可展示。

## 关联功能

- **网络协议内容管理（服务端）**：另一个协议内容管理入口，侧重 App 端显示的协议（如隐私政策）。两者功能相近但面向不同场景
- **字典管理**：协议类型选项来自字典中的"xylx"分类
