# 院内科室单元配置管理流程

> 文档版本：release/v2.17.2 | 面向读者：运营、管理员

---

## 概述

管理员在后台维护本互联网医院的组织架构——科室单元。每个科室需要配置科室编码、科室名称、科室类型、关联的标准科室以及医保科室。科室采用树形结构组织（支持父子层级），支持新增、编辑、删除操作。对于对接了HIS系统的机构，还支持从HIS同步科室数据到平台。

## 触发条件

- **前置条件**：管理员已登录管理后台，拥有科室管理权限
- **入口路径**：管理后台 → 医院管理 → 科室管理（单元配置）

## 步骤详解

### 第一步：浏览科室列表

进入页面后，科室以树形表格展示，每一行显示：

- **科室编码**：院内自定义编码，用于标识科室
- **科室名称**：在患者端和医生端展示的科室名称
- **科室类型**：从字典`kslx`中取值，如"门诊科室""住院科室"等
- **标准科室**：与国家/行业标准科室编码`ksbm`的映射关系
- **医保科室**：与当地医保系统的科室编码`ybks`的映射关系
- **科室简介**：对该科室的文字描述（最多500字）
- **创建时间**：科室的创建日期

如果当前机构启用了互联网诊疗模式（HIS对接），列表上方显示「同步HIS科室」按钮而非「新增」按钮——此时科室数据以HIS推送为准，不允许手动新增。

### 第二步：搜索科室

支持以下筛选条件：

- **科室名称**：输入关键字模糊搜索（最多32字）
- **标准科室**：通过树形下拉框选择标准科室进行过滤
- **科室类型**：从下拉框中选择科室类型筛选

查询逻辑为前端本地过滤，不重新请求接口——系统一次性加载所有科室的树形数据到前端，输入搜索条件后在已有数据中进行过滤。

### 第三步：新增科室

1. 点击列表上方「新增」按钮，弹出新增对话框
2. 填写科室信息：
   - **上级科室**（必填）：树形下拉框选择，表示该科室在组织架构中的父节点。**注意**：一旦选中上级科室，该字段即被禁用，不可再修改
   - **科室编码**（必填）：输入院内科室编码，最多10个字符。仅在新增时可填写，编辑时不允许修改
   - **科室名称**（必填）：输入科室名称，最多32个字符。在HIS对接模式下为只读（不可修改）
   - **科室类型**（必填）：下拉选择科室类型
   - **标准科室**（必填）：下拉搜索选择关联的国家/行业标准科室
   - **医保科室**（选填）：下拉选择关联的医保科室编码
   - **科室简介**（选填）：文本域输入，最多500字
3. 点击「保存」提交

### 第四步：编辑科室

1. 在列表操作列点击「编辑」，弹出编辑对话框
2. 系统自动加载该科室的当前数据
3. 修改需要变更的字段（科室编码不可修改）
4. 点击「保存」提交更新

### 第五步：删除科室

1. 在列表操作列点击「删除」
2. 弹出二次确认对话框："此操作将永久删除该科室, 是否继续?"
3. 点击「确定」执行删除，点击「取消」放弃操作

### 第六步：同步HIS科室（HIS对接机构专用）

当机构启用了互联网诊疗模式时：

1. 列表上方显示「同步HIS科室」按钮
2. 点击后弹出HIS同步对话框（HISDia组件）
3. 系统从HIS拉取最新科室数据，覆盖/同步到平台
4. 同步完成后自动刷新列表

## 异常处理

| 情况 | 系统处理方式 |
|------|------------|
| HIS对接模式下试图手动新增科室 | 列表上方不显示「新增」按钮，仅显示「同步HIS科室」按钮 |
| 编辑时试图修改科室编码 | 编辑模式下科室编码输入框为禁用状态（不可修改） |
| 新增科室时上级科室为空 | 表单校验不通过，提示"请选择关联标准科室" |
| 未选择科室类型 | 表单校验不通过，提示"请选择科室类型" |
| 未填写科室名称 | 表单校验不通过，提示"请输入科室名称" |
| 删除科室时点取消 | 提示"已取消删除"，不执行删除 |
| 无查询条件时点搜索 | 自动恢复为原始完整列表，避免空搜索误过滤 |

## 关联功能

- **标准科室字典（ksbm）**：科室需要映射到标准科室编码，用于跨系统的科室识别
- **医保科室字典（ybks）**：科室需要映射到医保科室编码，用于医保结算
- **科室类型字典（kslx）**：定义科室的分类标签
- **医生管理**：医生注册时需要选择所属科室
- **问诊预约**：患者选择科室进行问诊预约
- **HIS对接**：互联网诊疗模式下，科室数据以HIS为准
