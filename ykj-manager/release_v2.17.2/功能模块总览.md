# 源康健互联网医院 管理后台 — 功能模块总览

> 版本：v2.17.2 | 面向读者：客服 / 运营 / 产品经理

---

## 一、管理端（mgmt）— 平台运营管理中心

管理端是平台超级管理员、财务、药师等人员使用的后台，涵盖患者管理、医生管理、在线问诊、远程会诊、药房、财务、运营配置等全部功能。

### 1. 首页（工作台）
登录后展示的仪表盘页面，根据不同角色展示不同的数据概览：
- **超级管理员/系统管理员**：全平台核心数据概览
- **财务角色**：财务数据看板
- **药师角色**：药房工作数据看板

### 2. 医院管理（hosp）
管理医院相关的全部基础数据，是整个平台的"数据底座"。

| 子模块 | 用途说明 |
|--------|----------|
| 患者管理 | 查看和管理所有患者信息（姓名、性别、标签等），查看患者详情（基本信息、病历、检查报告、随访记录、消息沟通等） |
| 医生管理 | 管理平台注册医生列表，包括医生审核、排班管理、排班模板、停诊设置、AI医生等 |
| 药房管理 | 管理平台合作药房的药品、证照、发药渠道、药师等 |
| 药品信息 | 管理药品基础数据：药品生产厂家、公共药方、中药常规用量、有毒中药限量、中药特殊煎煮方法、药品黑名单、加工服务等 |
| 医院配置 | 配置医院基本信息（上海/天津/云南/福建等地区差异化配置）、业务类型、服务点信息、检查项目等 |
| 标准管理 | 管理标准医疗信息（药品标准、诊断标准）、科室诊断关联等 |
| 其他 | 互联网机构管理、诊疗科目设置、设备信息、通讯录、业务开展信息等 |

### 3. 诊疗服务管理（therapy）
在线诊疗业务的核心操作区。

| 子模块 | 用途说明 |
|--------|----------|
| 问诊工作台 | 医生在线接诊的工作界面：左侧问诊单列表 → 中间聊天室（图文/语音/视频沟通，语音消息直接显示时长）→ 右侧问诊详情（病历记录、处方记录）。医生可在此写病历、开处方（中药/西药） |
| 订单管理 | 查看和管理所有订单（问诊订单、处方订单等），查看订单详情 |
| 门诊服务订单 | 查看门诊预约服务订单列表、问诊查询、评价列表 |
| 处方管理 | 处方列表查看、药师审方、处方核验、药师签名设置 |
| 远程会诊中心 | 管理远程会诊全流程：会诊列表、发起会诊、会诊审核、会诊室（实时音视频+文字沟通）、机构信息、医生管理、服务管理、邀请机构管理等 |
| 远程诊疗 | 远程诊疗列表、审核、群组会诊 |
| 服务项目管理 | 服务项目列表、添加/编辑服务项目、转诊人管理 |
| 量表管理 | 量表分类管理、量表列表（添加/编辑题目、评分标准等） |
| 问卷管理 | 随访问卷的创建和管理（支持单选题、多选题、自由回答、上传题），以及题库管理 |

### 4. 运营管理（operate）
面向运营人员的C端内容配置功能。

| 子模块 | 用途说明 |
|--------|----------|
| 机构管理 | 管理机构信息（支持查看和管理多机构） |
| 会员管理 | 管理平台的会员信息 |
| 医学百科 | 管理C端展示的医学知识内容：疾病知识、经络穴位、药膳食疗、经典名方、中药图鉴 |
| 健康服务包 | 管理服务套餐（添加/编辑套餐和服务项目）、服务项目、套餐订单、会员权益 |
| 专家团队 | 管理专家团队信息，添加/管理团队成员 |
| 健康视频 | 管理健康科普视频内容 |
| 意见反馈 | 查看和处理用户提交的意见反馈 |
| 运费管理 | 设置配送运费规则 |
| 装修管理 | C端小程序/APP首页可视化装修：Banner、医生列表、视频、文章、菜单、底部导航等组件拖拽式配置 |

### 5. 系统管理（systemmanage）
IT管理员使用的系统级配置功能。

| 子模块 | 用途说明 |
|--------|----------|
| 用户管理 | 管理系统后台登录用户（含手写签名管理） |
| 角色管理 | 管理系统角色及角色对应的菜单/按钮权限 |
| 权限管理 | 管理细粒度的功能权限 |
| 菜单管理 | 管理后台左侧菜单的结构和排序 |
| 系统配置 | 平台全局参数配置（问诊、药房、地图、微信、医保、CA签章、物流、功能开关等数十项配置），支持修改历史查看与回滚 |
| 操作日志 | 查看后台用户的操作记录 |
| 登录日志 | 查看后台用户的登录记录 |
| 消息通知 | 管理平台的系统消息推送 |
| 密钥管理 | 管理API访问密钥（AccessKey） |
| APP日志 | 查看移动端APP的操作日志和登录日志 |
| 定时任务 | 管理后台定时任务（cron job）及其执行日志 |
| 医保管理 | 管理医保相关：医保标准、慢性病管理、诊断编码 |
| 字典管理 | 管理系统中使用的数据字典/枚举值 |
| 归档日志 | 文件归档日志管理 |

### 6. 财务管理（financemanage）
财务相关数据的查询和结算管理。

| 子模块 | 用途说明 |
|--------|----------|
| 财务看板 | 财务数据总览看板 |
| 收益管理 | 查看平台的收益流水 |
| 收款列表 | 查看收款记录 |
| 报销列表 | 管理报销申请和审核 |
| 发票管理 | 发票开具和管理，以及发票查询 |
| 医保结算 | 医保结算单管理 |
| 药品销售统计 | 药品维度的销售数据统计 |
| 药品订单统计 | 订单维度的药品销售统计 |

### 7. 其他功能
| 功能 | 用途说明 |
|------|----------|
| 咨询记录 | 查看和追踪问诊咨询流程记录（独立页面，不在菜单中） |
| 病历查看 | 线下就诊时查看线上病历（独立页面，不在菜单中） |
| 人脸认证 | 管理端登录时支持人脸识别二次认证 |

## 二、药房端（phar）— 线下药房管理中心

药房端是线下合作药房的独立管理后台，供药房管理员和药师使用。

### 1. 首页（工作台）
- **药房管理员**：查看所属药房的核心运营数据
- **药师**：查看个人工作数据

### 2. 药房管理（pharManage）

| 子模块 | 用途说明 |
|--------|----------|
| 药品管理 | 管理药房自有药品（添加/编辑药品信息） |
| 药品销售统计 | 查看药房药品的销售统计数据 |
| 订单管理 | 查看和管理药房收到的处方订单，查看订单详情 |
| 处方审核 | 药师审核线上开出的处方，核验用药合理性 |
| 药师签名设置 | 设置药师的电子签名 |
| 药师列表 | 管理药房内的药师人员 |
| 发货员管理 | 管理药房发货员（处理药品出库配送） |

## 三、系统通用能力

| 能力 | 说明 |
|------|------|
| 多角色支持 | 同一用户可拥有多个角色，支持在前端自由切换（系统管理员、财务、药师、药房管理员、药房药师等） |
| 菜单权限 | 菜单和功能按钮按角色动态加载，不同角色看到的后台菜单不同 |
| 人脸识别 | 管理端支持人脸认证（登录和二次验证） |
| 文件上传 | 支持腾讯云COS对象存储上传文件 |
| 地图服务 | 集成高德地图，支持位置相关功能 |
| 实时通信 | 支持TRTC音视频通话和IM即时消息（用于问诊和会诊的实时沟通） |
| 数据导出 | 支持Excel导出各类数据报表 |
| 图形验证码 | 登录时支持图形验证码校验 |
