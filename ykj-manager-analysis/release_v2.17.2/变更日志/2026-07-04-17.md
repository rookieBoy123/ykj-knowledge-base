# 变更日志 2026-07-04 (第17轮)

## 新增业务流程 (4篇)

### 1. 患者线上就诊记录查询流程
- **概述**：管理员通过患者身份证号查询该患者所有线上就诊记录，以时间线展示病历和处方详情，支持电子病历/处方PDF预览
- **来源文件**：`casehistory/index.vue` (550行)
- **关键发现**：
  - 左侧时间线+右侧详情的左右分栏布局
  - 门诊号统一标注"复诊"标签
  - 病历类型区分中医/西医，分别显示不同字段（四诊/治法 vs 治疗意见）
  - 处方信息区分西药处方（表格）和中药处方（逐行+剂数+煎煮方法）
  - 患者年龄<1岁显示"X月"，≥1岁显示"X岁"
  - 病史字段为空时统一显示"未填写"
  - 无电子病历文件时不显示"电子病历"按钮

### 2. 行政管理通讯信息管理流程
- **概述**：管理员维护医院行政部门的负责人和联系电话，支持增删改查，含手机号格式校验和部门重名校验
- **来源文件**：`hosp/addresslist/index.vue` (281行)
- **关键发现**：
  - 仅广东地区医院配置中显示此选项卡
  - 部门选项来自系统字典"部门(bm)"分类
  - 新增/编辑均有部门重名校验（校验接口：verifyXzkstxl）
  - 权限控制：hosp:concats:write
  - 手机号校验：必须1开头11位数字

### 3. 群组会诊费用配置与收益提成管理流程
- **概述**：管理员按医生职称配置群组会诊价格区间，管理平台收益提成比例和启用开关
- **来源文件**：`therapy/telemedicine/groupconsultation/index.vue` (244行)
- **关键发现**：
  - 费用按医生职称区分（主任医师、副主任医师等）
  - 价格区间最低价和最高价相互关联
  - 收益提成仅变更时才提交（比较新值与原始值）
  - 提成比例范围1%-100%
  - 配置存储在系统配置 consultation 分组中

### 4. 远程会诊审核管理流程 + 远程会诊列表与备案管理流程
- **概述**：审核管理——管理员审核会诊申请（通过/拒绝）；列表管理——查看已审核会诊的流程状态和备案管理
- **来源文件**：`therapy/telemedicine/telemedicinecheck/index.vue` (252行), `therapy/telemedicine/telemedicinelist/index.vue` (249行)
- **关键发现**：
  - 审核页三状态标签（待审核/已通过/已拒绝）带颜色区分
  - 左侧列表无限滚动，右侧详情联动
  - 会诊7种流转状态（申请→待备案→待确认→待开始→进行中→待报告→已完成）
  - 审核操作后自动切换标签页并刷新
  - 列表页患者姓名脱敏（首字+星号），同一时间仅一行可看明文
  - 待备案详情中无需再审核，仅做归档查看
