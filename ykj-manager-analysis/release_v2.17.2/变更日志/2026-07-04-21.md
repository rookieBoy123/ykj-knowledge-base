# 变更日志 - 2026-07-04（第21轮）

## 新增业务流程

### 1. 发票查询流程
- 文档：`业务流程/发票查询流程.md`
- 概述：在管理后台查询所有已开具的电子发票，支持按患者姓名、订单编号、开票日期筛选，可打印发票图片和导出 Excel
- 覆盖页面：`financemanage/invoiceQuery/index.vue` (214行)
- 关键发现：
  - 「打印」按钮仅在发票存在电子图片时显示
  - 导出功能自动生成「发票信息.xls」
  - 支持患者姓名（5字限制）、订单编号（30字限制）、开票日期三维筛选
  - 门诊号码和患者姓名为空时显示"/"

### 2. 小程序登录日志查询流程
- 文档：`业务流程/小程序登录日志查询流程.md`
- 概述：查看小程序端用户的登录记录，手机号自动脱敏显示，记录操作系统、浏览器、登录方式等
- 覆盖页面：`systemmanage/system/appLoginLog/index.vue` (154行)
- 关键发现：
  - 与「后台登录日志」使用同一接口，通过 sysClient='app' 区分
  - 手机号中间四位自动脱敏（如 138****1234）
  - OS 和浏览器由服务端从 User-Agent 自动解析
  - 登录状态分「正常」（蓝色标签）和「异常」（红色标签）

### 3. 退款审核详情看板流程
- 文档：`业务流程/退款审核详情看板流程.md`
- 概述：财务人员审核退款申请——查看订单信息、交易流水、退款明细，决定同意/拒绝退款，处理退款失败
- 覆盖页面：`financemanage/reimburselist/financeboard/index.vue` (423行)
- 关键发现：
  - 审核状态五阶段：待审核→已退款/审核不通过/待退款/退款失败
  - 退款方式分「原路返回」和「手动退款」两种
  - 默认选中「同意退款」+「原路返回」
  - 退款失败时提供「人工已退款」和「重新退款」两个入口
  - 「人工已退款」操作不可逆，有二次确认

### 4. 标准药品信息管理流程
- 文档：`业务流程/标准药品信息管理流程.md`
- 概述：管理药品的基础标准信息库——药监ID、本位码、规格、说明书等，支持新增、编辑、查询、图片上传
- 覆盖页面：`hosp/stdManage/stdMedInfoList/index.vue` (176行) + `stdMedInfoDetail/index.vue` (674行)
- 关键发现：
  - 药品规格支持自动生成（拼接剂量+制剂+包装单位）
  - 药监ID 12位字母数字校验、本位码 14位纯数字校验
  - 药品说明书含9个文本字段（品牌/功能主治/成分/性状/贮藏/不良反应/建议用法/禁忌/注意事项）
  - 图片最多 10 张，单张不超 10M
  - 自动生成规格需同时填写制剂数量+制剂单位+包装单位
