# 变更日志 2026-07-03 (第4轮)

## 新增业务流程（6篇）

### 基础设施模块（系统管理）
1. **数据字典管理流程** — 字典分类的新增/编辑/删除/校验，字典项的新增/编辑/批量删除/搜索过滤
   - 代码来源：`systemmanage/dictionarySet/index.vue`（688行）
   - 关键逻辑：分类编码+标准编码唯一性校验（仅名称变化时触发校验），叶子节点才可删除，批量删除逐条调用

2. **定时任务管理流程** — 定时任务列表+执行日志两个页面的完整操作流程
   - 代码来源：`scheduler/jobs/index.vue`（278行）、`scheduler/logs/index.vue`（169行）
   - 关键逻辑：编辑Cron表达式+描述，手动触发，启用/禁用开关，一次性任务(jobType=2)不可编辑和触发，日志页按任务筛选

3. **后台操作日志查询流程** — 管理员所有操作的审计日志查询
   - 代码来源：`systemmanage/system/log/index.vue`（394行）
   - 关键逻辑：多维度筛选（用户名/模块/时间/操作内容），详情弹窗展示IP/客户端/URI/请求参数/错误信息/耗时，用户名为空显示"匿名用户"
   - 关联：`appLog/index.vue`（394行，同结构但固定sysClient='app'过滤）

4. **登录日志查询流程** — 管理员登录日志+小程序登录日志
   - 代码来源：`systemmanage/system/loginLog/index.vue`（153行）、`systemmanage/system/appLoginLog/index.vue`（154行）
   - 关键逻辑：两个页面结构相同，差异仅为appLoginLog查询时额外传`sysClient='app'`，记录IP/OS/浏览器/登录方式/成功状态

### 业务模块
5. **问诊病历归档管理流程** — 追踪病历同步到医院的进度
   - 代码来源：`systemmanage/fileLog/index.vue`（206行）
   - 关键逻辑：三种状态（进行中/成功/失败），仅失败状态(status=2)可手动重归档，点击确认弹窗后执行

6. **量表发布与管理流程** — 医学量表的全生命周期管理
   - 代码来源：`therapy/scaleManage/scaleList/index.vue`（370行）
   - 关键逻辑：草稿→发布→禁用的三态流转，发布前校验题目列表和评分规则非空，草稿态不可排序，上移下移支持跨页，页面缓存策略（从编辑页返回不刷新）
