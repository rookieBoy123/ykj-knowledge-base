# 医保慢性病种管理流程

> 文档版本 1.0 | 面向读者：运营人员、医保业务管理员

## 概述

慢性病种管理用于维护医保系统中支持的慢性病种类字典数据。每种慢性病可关联多个医院科室，用于医保结算时判断就诊科室与病种是否匹配。病种数据由管理员统一维护，支持启用/禁用和编辑关联科室。

## 触发条件

- **入口路径**：系统管理 → 医保字典 → 慢性病种管理
- **前置条件**：需拥有医保字典管理权限

## 步骤详解

### 第一步：查看病种列表

列表展示所有已录入的慢性病种信息：

| 列名 | 说明 |
|------|------|
| 序号 | 列表序号 |
| 病种编码 | 慢性病在医保系统中的标准编码 |
| 病种名称 | 慢性病的标准中文名称 |
| 关联科室 | 该病种关联的医院科室列表，多个科室以逗号分隔显示 |
| 状态 | 启用/禁用的开关控件 |
| 操作 | "编辑"按钮 |

支持按**病种名称**和**病种编码**进行筛选查询。

### 第二步：编辑病种信息

点击某条病种的"编辑"按钮，弹窗展示详细信息：

- **病种名称**（只读）：不可修改，因为需与医保系统保持一致
- **病种编码**（只读）：不可修改
- **关联科室**（必选）：以树形下拉选择器展示全院科室结构，支持多选。可搜索科室名称进行快速定位
- **状态**：启用或禁用，通过单选按钮切换

编辑确认后会保存修改。

### 第三步：管理启用状态

列表中每行有启用/禁用开关：
- **启用**：该慢性病种在医保结算流程中可以被选择和使用
- **禁用**：该病种在系统中不可用，患者就诊时无法选择该病种
- 切换开关即时生效，无需弹窗确认

## 异常处理

| 情况 | 系统处理方式 |
|------|------------|
| 编辑时未选择关联科室 | 提示"请选择关联科室"，阻止保存 |
| 病种名称或编码需修改 | 病种名称和编码字段为只读，不可在编辑界面修改（需联系系统管理员在数据库层面修改） |
| 科室树加载失败 | 关联科室下拉框为空，无法选择科室 |

## 关联功能

- **医保诊断管理**（系统管理 → 医保字典 → 诊断管理）：管理医保诊断代码，与慢性病种配合使用
- **HIS标准列表**（系统管理 → 医保字典 → HIS标准列表）：管理HIS系统对接标准数据
- **数据字典管理**：科室信息来自数据字典中的科室树
- **医保结算流程**：患者使用医保结算时，系统会校验就诊科室是否在所选慢性病的关联科室范围内
