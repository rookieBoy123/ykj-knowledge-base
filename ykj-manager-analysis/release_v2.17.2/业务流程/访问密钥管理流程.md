# 访问密钥管理流程

> 文档版本 2026-07-03 | 面向系统管理员

## 概述

为后台系统用户创建和管理访问密钥（Access Key）。访问密钥是用户通过API或第三方系统访问平台时的身份凭证，每个用户可拥有多个密钥。支持密钥的创建、编辑、启用/停用、删除等全生命周期管理。

⚠️ 访问密钥是敏感的安全凭证，密钥文件下载后无法再次获取，请妥善保管。

## 触发条件

- **前置条件**：从后台用户管理页面跳转（URL中携带用户ID），拥有访问密钥管理写权限
- **入口路径**：系统管理 → 用户管理 → 点击某用户的"访问密钥"

## 步骤详解

### 第一步：进入密钥管理

从用户管理页面点击某用户的"访问密钥"入口，页面会携带该用户的ID参数跳转到密钥管理页面。如果缺少用户ID参数，页面会提示"缺少用户id"。

### 第二步：查看密钥列表

列表中展示该用户的所有访问密钥：

| 列 | 说明 |
|:---|:---|
| 访问密钥ID | 系统生成的唯一密钥标识 |
| 描述 | 管理员添加的描述信息（如用途、使用场景） |
| 状态 | 启用/停用 |
| 创建时间 | 密钥创建的时间 |
| 最近使用时间 | 密钥最后一次被使用的时间 |

### 第三步：新增密钥

1. 点击"新增"按钮，弹出新增对话框
2. 输入描述信息（如"订单查询接口调用"），最多500字
3. 点击"确定"创建
4. ⚠️ 创建成功后，系统会弹出下载提示："用户创建成功。单击'确定'下载访问密钥文件，单击'取消'后将无法获取该访问密钥"
5. 点击"确定"后下载密钥文件（包含密钥ID和密钥Secret），点击"取消"则放弃下载

**重要提醒**：
- 为了保证历史兼容性，系统会将密钥创建时间作为"最近使用时间"的初始值
- 该密钥首次被使用时，最近使用时间会自动更新
- 密钥文件只提供一次下载机会，请务必保存

### 第四步：编辑密钥描述

点击"编辑"按钮，可修改密钥的描述信息。密钥ID和创建时间不可修改。编辑不会影响密钥本身的有效性。

### 第五步：启用/停用密钥

点击"启用"或"停用"按钮：
- 系统弹出确认提示（"是否启用/停用"）
- **停用后**：该密钥将无法用于API调用鉴权，但之前使用该密钥建立的会话不受影响
- **启用**：恢复密钥的使用权限
- 此操作需要"sys:access-keys:write"权限

### 第六步：删除密钥

点击"删除"按钮，确认后永久删除。删除后该密钥将无法恢复，所有依赖该密钥的API调用将立即失效。

## 异常处理

| 情况 | 系统处理方式 |
|:---|:---|
| 缺少用户ID参数 | 页面提示"缺少用户id"，列表无法加载 |
| 密钥文件未下载 | 创建后点击"取消"放弃下载，密钥文件无法再次获取，只能重新创建新密钥 |
| 无写权限 | "停用/启用"按钮不显示，只能查看 |
| 密钥已停用 | API调用会返回鉴权失败，需要重新启用或创建新密钥 |

## 关联功能

- **后台用户管理**：从用户管理页面进入密钥管理，通过URL中的用户ID关联
- **API鉴权**：系统开放API使用AccessKey进行身份验证
- **操作日志**：密钥的创建、删除、启停操作会记录在操作日志中，用于安全审计

## 安全策略

| 策略 | 说明 |
|:---|:---|
| 密钥文件仅下载一次 | 创建后只有一次下载机会，取消后无法再次获取 |
| 描述字段可修改 | 但密钥Secret不可查看和修改 |
| 最近使用时间 | 自动记录密钥的使用情况，便于审计和排查 |
| 权限控制 | 增删改操作需要"sys:access-keys:write"权限，查看需要"sys:access-keys:read"权限，启停也需要写权限 |
