# 推荐医生管理流程

> 文档版本：v1.0 | 面向读者：运营管理员

## 概述

推荐医生管理用于控制小程序患者端首页"推荐医生"板块展示哪些医生及其展示顺序。管理员可以从平台医生库中选择医生加入推荐列表，并调整排序。

## 触发条件

- **前置条件**：已登录管理后台，具备运营管理权限
- **入口路径**：左侧菜单 → 运营管理 → 文章管理 → 推荐医生

## 步骤详解

### 第一步：查看推荐医生列表

进入页面后看到当前推荐医生列表，展示信息：

| 列名 | 说明 |
|------|------|
| 编号 | 医生ID |
| 医生姓名 | - |
| 所属医院 | - |
| 科室 | - |
| 职称 | 通过字典转换为中文职称名称 |
| 电话号码 | - |
| 创建时间 | 加入推荐列表的时间 |
| 操作 | 排至最前、下移、排至最后、上移、医生详情、删除 |

### 第二步：搜索筛选

支持按以下维度搜索：
- **医生姓名**：最多输入5个字
- **医生职位**（职称）：从职称字典加载下拉选项

### 第三步：添加推荐医生

点击"添加"按钮，弹出"选择推荐医生"弹窗。弹窗中展示可选择的医生列表：
- 系统自动查询条件为医生类型=20（接诊+开处方），状态=10（启用）的医生
- 支持在弹窗内搜索医生姓名和按所属医院筛选
- 勾选一个或多个医生，点击"确定"将其加入推荐列表
- 如果不勾选就点确定，提示"请至少选择一条数据"

### 第四步：调整排序

推荐医生列表支持四种排序操作：
- **排至最前**：将选中医生移到列表第一位
- **上移**：与上一条交换位置，如果是第一条则提示"第一条不能上移！"
- **下移**：与下一条交换位置，如果是最后一条则提示"最后一条不能下移！"
- **排至最后**：将选中医生移到列表最后一位

排序结果直接影响患者端推荐医生板块的展示顺序。

### 第五步：查看医生详情

点击"医生详情"跳转到该医生的信息详情页（医生管理模块），查看医生的完整资料。

### 第六步：删除推荐医生

点击"删除"，弹出确认弹窗"是否确认删除？"，确认后将该医生从推荐列表中移除（不影响医生账号本身）。

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 添加时未选择任何医生 | 提示"请至少选择一条数据" |
| 第一条试图上移 | 提示"第一条不能上移！" |
| 最后一条试图下移 | 提示"最后一条不能下移！" |
| 删除操作 | 弹出二次确认弹窗 |
| 从医生详情页返回列表 | 列表保持缓存状态，不从第一页重新加载 |
| 从其他页面进入列表 | 重置搜索条件并重新查询 |

## 关联功能

- **医生信息详情**：点击"医生详情"跳转到 `hosp/doctormanage/doctorlist/info` 页面
- **医生管理**：推荐医生的数据来源于医生列表（doctorType=20的医生）
