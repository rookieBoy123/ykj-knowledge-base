# 退款审核详情看板流程

> 文档版本：2026-07-04 | 面向读者：财务审核人员、运营管理员

## 概述

财务人员对退款申请进行审核操作——查看订单信息、交易流水、退款明细，决定同意或拒绝退款，并处理退款失败后的重新退款或人工退款。

## 触发条件

- **前置条件**：需要拥有退款审核权限（`order:refunds:audit`）
- **入口路径**：财务管理 → 退款审核 → 点击某条退款记录进入详情

## 步骤详解

### 第一步：进入退款详情看板

从退款审核列表点击某条记录后进入详情页。页面顶部显示「退款详情」标题，按以下区域组织信息：

**区域一：订单信息**

以表格形式展示关联订单的核心信息：

| 信息项 | 说明 |
|--------|------|
| 订单编号 | 唯一标识 |
| 订单类型 | 如处方订单、问诊订单等 |
| 支付时间 | 订单支付的时间 |
| 医生 | 接诊医生姓名（预约订单不显示） |
| 金额 | 订单的总金额 |

**区域二：交易流水**

以表格展示该订单的所有交易流水记录，每条包含：
- 交易流水号
- 支付渠道（如微信支付、医保个账等）
- 交易金额

**区域三：申请信息**

展示退款申请的具体内容：
- **退款金额**：按支付渠道分项列出，如「微信支付 ￥100.00」「医保个账 ￥50.00」
- **问题描述**：申请人填写的退款原因，为空时显示「-」
- **申请时间**：退款申请提交的时间

### 第二步：财务审核（仅待审核状态显示）

当退款申请状态为「待审核」（flowStatus=10）时，页面显示「财务审核」区域，包含：

**审批意见**（必选）：
- 「同意退款」— 默认选中
- 「不同意」— 驳回退款申请

**退款方式**（仅在同意退款时显示）：
- 「原路返回」— 默认选中，系统自动发起退款到原支付渠道
- 「手动退款」— 系统仅登记退款记录，不做实际退款操作，需财务人员自行线下退款

**备注**：可选的文本输入，最多 255 个字符。用于记录审核意见或说明。

点击「保存」按钮提交审核：
- 选择「同意退款」+「原路返回」：系统弹出二次确认「确认退款吗？确认后该订单将发起退费」，确认后自动执行退款
- 选择「不同意」：提交驳回申请，退款流程终止

### 第三步：查看退款结果

当退款申请已进入「已退款」（flowStatus=20）、「待退款」（flowStatus=40）或「退款失败」（flowStatus=50）状态时，页面显示「退款结果」区域：

- **退款结果**：显示当前的退款执行状态
- **退款失败原因**：仅在退款失败时显示具体原因

### 第四步：处理退款失败

当退款状态为「退款失败」（refundStatus=40）时，页面底部出现两个操作按钮：

- **人工已退款**：点击后弹出提示「将修改订单状态为退款成功，该操作不可逆，确认已人工退款成功吗？」，确认后将订单标记为已退款
- **重新退款**：点击后打开弹窗，可选择：
  - 「原路返回」— 重新发起自动退款
  - 「手动退款」— 仅登记记录
  - 填写备注后点击确定提交

## 状态说明

### 审核状态（flowStatus）

| 状态码 | 含义 | 说明 |
|--------|------|------|
| 10 | 待审核 | 等待财务人员审核，此时可操作审批 |
| 20 | 已退款 | 退款已成功执行 |
| 30 | 审核不通过 | 退款申请被驳回 |
| 40 | 待退款 | 已同意但退款尚未完成 |
| 50 | 退款失败 | 退款执行失败 |

### 退款执行状态（refundStatus）

| 状态码 | 含义 |
|--------|------|
| 10 | 待退款 |
| 20 | 退款中 |
| 30 | 已退款 |
| 40 | 退款失败 |

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 未选择审批意见 | 点击保存时提示「请选择审批状态」 |
| 退款申请已被处理 | 审批区域的单选框和输入框全部禁用 |
| 退款失败 | 显示失败原因，提供「人工已退款」和「重新退款」两个操作入口 |
| 退款重试弹窗确认 | 点击确定后自动重新发起退款 |

## 关联功能

- **退款审核列表**：查看所有退款申请的列表
- **交易流水查询**：查看完整的交易流水记录
- **订单管理**：查看订单的完整生命周期
