# 检查报告审核流程

> 文档版本：v1.0 | 面向读者：检查科室医生、报告审核员

## 概述

检查报告审核功能用于管理患者的检查报告单。检查科室的医生可以对患者上传的检查报告进行查看和填写，包括录入检查所见、诊断结果意见，并进行手写签名确认。报告填写完成后状态从「未出报告」变为「已出报告」，后续只能查看不能修改。

## 触发条件

- **入口路径**：管理后台 → 诊疗管理 → 报告审核
- **前置条件**：需要患者在系统中存在检查记录

## 步骤详解

### 第一步：查看报告列表

进入页面后，系统自动加载检查报告列表。列表以表格形式展示，每条记录包含：
- **检查号**：报告唯一编号
- **姓名**、**性别**、**年龄**
- **检查项目**：检查类型名称
- **科室**：患者所属科室
- **检查时间**
- **报告医生**：填写报告的医生姓名（未填写时显示「-」）
- **报告时间**：报告填写完成的时间
- **状态**：「未出报告」或「已出报告」

### 第二步：筛选查询

页面提供三个筛选条件：
- **检查日期**：日期选择器，按检查时间筛选
- **姓名**：输入框，模糊搜索患者姓名，最长 5 字
- **状态**：下拉选择，可选「全部」「未出报告」「已出报告」

### 第三步：填写报告（未出报告状态）

对于状态为「未出报告」的记录，操作栏显示「填写报告」按钮。点击后弹出报告填写弹窗。

弹窗顶部显示：
- 医院名称（从系统配置「heart-check-report」中读取）
- 标题「检查报告单」
- 检查时间

弹窗展示患者基本信息和检查报告图片。下方提供三个填写区域：

| 字段 | 类型 | 必填 | 限制 |
|------|------|------|------|
| 检查所见 | 多行文本 | 是 | 最长 265 字，不能为空或全空格 |
| 诊断结果意见 | 多行文本 | 是 | 最长 265 字，不能为空或全空格 |
| 备注 | 多行文本 | 否 | 最长 265 字 |

填写完成后，还需要进行**手写签名**。点击「手写签名」按钮，弹出画布签名区域，医生在画布上签名后点击确认，签名图片会显示在按钮旁边。

三项内容（检查所见、诊断意见、签名）都完成后，点击「确定」提交。提交成功后：
- 报告状态变为「已出报告」
- 弹窗关闭，列表自动刷新

### 第四步：查看已出报告

对于状态为「已出报告」的记录，操作栏显示「查看报告」按钮。点击后以只读模式展示报告内容，包括：
- 检查所见
- 诊断结果意见
- 备注
- 报告医生姓名 + 签名
- 报告时间

已出报告无法修改，只能查看。

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 检查所见未填写 | 提示「检查所见不能为空且不能为空格」 |
| 诊断结果意见未填写 | 提示「诊断结果意见不能为空且不能为空格」 |
| 手写签名未完成 | 提示「签名不能为空」 |
| 报告图片不存在 | 显示「暂无」占位 |

## 关联功能

- 系统配置管理（「heart-check-report」配置项，设置报告单上的医院名称）
- 手写签名组件（画布签名）
