# 医保结算申报管理流程

> 文档版本 1.0 | 面向读者：运营人员、财务人员

## 概述

医保结算申报管理用于将已完成的医保结算订单数据申报上传至医保局系统。系统自动从医保慢病订单中生成申报记录，运营人员可以在管理后台查看、筛选并逐条或批量上报。

## 触发条件

- **前置条件**：患者通过医保支付完成慢病问诊结算后，系统自动生成一条申报记录（状态为"待上传"）
- **入口路径**：财务管理 → 医保结算申报管理

## 步骤详解

### 第一步：查看申报记录列表

进入页面后，默认显示"待上传"状态的申报记录列表。系统自动设置结算日期范围为最近30天。

每行记录展示以下信息：
- 复选框（仅"待上传"和"上传失败"状态可勾选）
- 人员编号、患者姓名、慢性诊断科室
- 病种名称、证件类型（固定为"居民身份证(户口簿)"）
- 证件号、性别、年龄
- 结算时间
- 操作按钮：待上传状态显示"上传"，上传失败状态显示"重新上传"，所有状态显示"查看详情"

### 第二步：筛选申报记录

支持以下筛选条件：
- **病人名称**：支持模糊搜索
- **结算日期**：日期范围选择
- **人员编号**：精确搜索
- **病种**：下拉远程搜索，支持按病种名称筛选
- **身份证号**：精确搜索
- **科室**：树形选择，按医院科室层级筛选

### 第三步：切换状态标签页

页面顶部有三个状态标签页：
- **待上传**：初始状态，等待申报
- **已上传**：已经成功申报到医保局，此时顶部出现"申报时间"列和"申报日期"搜索字段，复选框隐藏
- **上传失败**：申报过程出现错误，可查看失败原因

### 第四步：单独上传一笔结算单

点击待上传记录的"上传"按钮，弹出"选择申报时间"窗口：
1. 选择申报日期（不能选择未来日期）
2. 点击"确定"提交申报
3. 系统校验医保科室配置、医师编码、患者会员信息后，调用医保局接口上传
4. 成功 → 记录变为"已上传"
5. 失败 → 记录变为"上传失败"，保存失败原因

### 第五步：批量上传

1. 勾选多条"待上传"或"上传失败"记录
2. 点击"批量上传"按钮
3. 弹出申报时间选择窗口（统一日期）
4. 系统逐条处理，每条独立调用医保局接口
5. 成功的记录变为"已上传"，失败的记录保持当前状态并记录失败原因

### 第六步：查看详情

点击"查看详情"按钮，弹出弹窗展示完整的"广州市医疗保障基金结算清单"，包含三大信息区：

**一、基础信息**：申报时间、清单流水号、病案号、医保编号、人员编号、患者姓名、性别、年龄（岁）、结算ID、就诊ID

**二、门诊慢特病诊疗信息**：诊断科别、就诊日期、入院时间、出院时间、离院方式、诊断类别、入院病情类型、主诊断标志

**三、医疗收费信息**：业务流水号、票据代码、票据号码、结算日期、医疗机构填报部门、医疗机构填报人

如果该记录状态为"上传失败"，详情顶部会显示失败原因。

## 状态流转

```
订单结算（医保慢病） → 自动生成申报记录（待上传）
   ↓
待上传 ──→ 已上传（上传成功）
   ↓
待上传 ──→ 上传失败（上传失败，记录失败原因）
   ↓
上传失败 ──→ 已上传（重新上传成功）
   ↓
任意状态 ──→ 已取消（手动取消，不在标签页展示）
```

## 异常处理

| 情况 | 系统处理方式 |
|------|------------|
| 未选择任何记录点击"批量上传" | 提示"请至少选择一条数据" |
| 未选择申报时间直接确认 | 提示"请选择申报时间" |
| 患者对应会员不存在 | 提示"患者对应会员不存在"，该条上传失败 |
| 医生未配置医保编码 | 提示"医师编码不能为空"，该条上传失败 |
| 科室未配置医保编码 | 提示"未配置医保科室，医保申报失败" |
| 医保局接口返回失败 | 记录状态变为"上传失败"，保存失败原因 |
| 医保订单结算信息查询为空 | 提示"医保订单结算信息查询为空！" |
| 已上传的记录 | 复选框隐藏，不可再次操作 |

## 关联功能
