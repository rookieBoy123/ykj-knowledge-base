# 角色与权限管理流程

> 文档版本 1.0 | 面向角色：系统管理员

## 概述

角色管理用于定义管理后台的权限体系。管理员可以创建不同的角色（如"运营管理员"、"财务管理员"等），为每个角色分配不同的菜单权限（功能权限），然后将角色分配给后台用户，从而精确控制每个用户能访问和操作哪些功能。

## 触发条件

- **前置条件**：登录管理后台，拥有角色管理权限（`sys:roles:write`）
- **入口路径**：管理后台 → 系统管理 → 系统设置 → 角色管理

## 步骤详解

### 第一步：查看角色列表

进入页面后展示所有角色列表。页面顶部有一个标签切换：「互联网医院」和「线下药房」，两者各自管理独立的角色体系。

角色列表包含以下信息：

| 列名 | 说明 |
|------|------|
| ID | 角色唯一编号 |
| 角色名称 | 系统内置角色会带"系统角色"蓝色标签（如"超级管理员 系统角色"） |
| 角色编码 | 角色的唯一编码标识 |
| 角色说明 | 对角色的文字描述 |
| 角色状态 | 开关按钮，可快速启用/禁用 |
| 创建时间 | 角色创建时间 |
| 最后修改时间 | 角色最后修改时间 |
| 操作 | 编辑、权限设置、删除 |

**筛选条件**：可按角色名称、状态筛选。

### 第二步：新增角色

点击「新增」按钮，弹出新增角色对话框，需要填写：

- **角色名称**（必填）：3-20个字符，失焦时自动校验是否已存在
- **角色编码**（必填）：唯一标识，如 `operation_manager`
- **角色说明**：文字描述
- **状态**：启用或禁用

点击保存后，角色创建成功。

### 第三步：编辑角色

点击操作列的「编辑」按钮。系统内置角色（带"系统角色"标签）的编辑按钮是禁用的，不可编辑。

编辑时修改角色名称会自动校验是否与已有角色重名（排除自身）。

### 第四步：权限设置（核心功能）

点击「权限设置」按钮，弹出权限设置对话框，包含两个标签页：

#### 功能权限

左侧是一个功能菜单树，展示了管理后台所有可用的菜单和功能点。管理员通过勾选树节点来授予角色相应的功能权限。

- **父子关联**：勾选父节点会自动勾选其下所有子节点
- **右侧展示**：被勾选的菜单项会实时显示在右侧的表格中，展示功能名称和描述

操作完成后点击「保存」，角色的功能权限更新生效。

#### 数据权限

数据权限标签页目前显示为占位状态（"数据权限"文字），功能尚未完全实现。

### 第五步：删除角色

单个删除：点击「删除」按钮，系统先检查该角色是否被用户使用中。如果未被使用，弹出确认对话框后删除；如果已被使用，直接提示"数据已被使用，不能删除"。

**批量删除**：勾选列表中的多个角色，点击「批量删除」按钮。如果勾选中包含系统内置角色，提示"勾选中存在不可删除的角色"。

**系统内置角色**：带"系统角色"标签的角色（`builtIn=true`）不可删除，删除按钮会被禁用。

### 第六步：启用/禁用角色

在角色列表的状态列，点击开关按钮可快速切换角色的启用/禁用状态。禁用后，拥有该角色的用户将暂时失去对应的权限。

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 角色名称已存在 | 提示"角色名已存在，请重新输入" |
| 新增时角色名称为空 | 表单校验提示"请输入角色名称" |
| 角色名称长度不在3-20字符 | 提示"长度在3到20个字符" |
| 角色编码为空 | 表单校验提示"请输入角色编码" |
| 删除被使用的角色 | 提示"数据已被使用，不能删除" |
| 删除系统内置角色 | 删除按钮禁用，批量删除时提示"勾选中存在不可删除的角色" |
| 权限设置时未选择任何菜单 | 提示"请至少选择一条数据" |

## 关联功能

- **用户管理**：用户通过分配角色获得权限，用户管理中的角色选择器展示的就是此处定义的角色
- **菜单管理**：权限设置中的菜单树来自菜单管理模块
- **线下药房**：角色分为两个体系——互联网医院和线下药房，通过顶部标签切换
