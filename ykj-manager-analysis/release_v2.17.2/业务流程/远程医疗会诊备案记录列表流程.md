# 远程医疗会诊备案记录列表流程

> 文档版本 2026-07-05 | 面向管理后台运营人员

## 概述

远程医疗会诊完成后，每笔会诊记录需要进入"备案"环节。管理员通过本页面按备案状态查看和跟踪所有远程会诊订单的完成情况。

## 触发条件

- **前置条件**：管理员拥有远程医疗审核相关权限
- **入口路径**：管理后台 → 诊疗管理 → 远程医疗 → 远程医疗列表

## 步骤详解

### 第一步：按备案状态切换查看

页面顶部有两个标签页：

- **待备案**：显示尚未完成备案的会诊记录（默认打开此标签页）
- **已完成**：显示已完成备案的会诊记录

两个标签页互斥，点击切换会自动刷新列表。默认进入页面时显示"待备案"列表。

### 第二步：搜索筛选

在"待备案"标签页下，提供四个搜索条件：
- **申请单编号**：精确搜索，最多20字
- **患者姓名**：模糊搜索，最多5字（注意患者姓名已脱敏，列表中只显示姓+星号）
- **医生姓名**：模糊搜索，最多5字
- **状态**：下拉选择，可选"会诊待开始""会诊进行中""待会诊报告"三个中间状态

切换到"已完成"标签页时，状态筛选条件会自动从搜索区域移除——因为"已完成"本身就是一个终态，不需要再细分。

> ⚠️ 两个标签页切换时会自动重置到第一页，同时清除当前的搜索条件。

### 第三步：查看列表

列表展示核心信息：
- **会诊编号**：每笔会诊的唯一编号
- **患者姓名**：默认脱敏显示（只显示姓，后面用星号替代）。点击脱敏图标可以查看完整姓名（单次有效，离开页面后恢复脱敏）
- **医生姓名**
- **经治科室**
- **当前诊断**
- **会诊时间**
- **状态**：显示会诊的当前流转状态

### 第四步：查看会诊详情

点击某条记录的"查看详情"按钮，弹出会诊详情对话框（900px宽），展示该笔会诊的完整信息。详情由通用会诊详情组件渲染，内容包括参会医生、患者信息、会诊时间、诊断建议等。对话框内容在65vh高度内滚动。

### 会诊状态说明

| 状态码 | 含义 |
|:---|:---|
| 1 | 会诊申请 |
| 2 | 会诊待备案 |
| 3 | 会诊待患者确认 |
| 4 | 会诊待开始 |
| 5 | 会诊进行中 |
| 6 | 待会诊报告 |
| 7 | 已完成 |

> "待备案"标签页显示的是 status=WAIT_BEIAN（对应状态码1-6的会诊），"已完成"标签页显示的是 status=FINISHED（状态码7）。

## 异常处理

| 情况 | 系统处理方式 |
|------|------------|
| 搜索无结果 | 列表显示空数据 |
| 列表为空时翻页减一到负值 | 页码自动退回上一页并重新查询 |
| 患者姓名脱敏切换 | 脱敏/显示仅限当前行、当前次查看，不会影响其他行的显示状态 |
| 无权限 | 页面可能无法访问 |

## 关联功能

- 与"远程医疗审核"页面（telemedicinecheck）共享同一个后端分页查询接口
- 与"远程会诊管理"模块的会诊申请、审核流程联动
