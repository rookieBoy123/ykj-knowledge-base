# 医生排班周视图管理流程

> 文档版本 v1.0 | 面向读者：运营管理员

## 概述

管理员通过周视图表格直观查看和管理医生的每周排班情况，支持按医生姓名、科室和日期范围筛选，可对每周排班进行发布操作。

## 触发条件

- 前置条件：拥有后台管理权限
- 入口路径：诊疗管理 → 医生管理 → 医生排班

## 步骤详解

### 第一步：查看排班周视图

页面默认展示「本周」的排班情况，表格形式呈现，**行是医生、列是日期**。

- 第一列：医生姓名
- 后续7列：周一至周日的日期，每天一个格子
  - 如果当天是今天，列标题显示为「今天 MM/DD」
  - 其他日期显示为「周X MM/DD」

每个日期的格子内展示该医生当天的排班详情（如排班状态、时间段等），通过独立的排班子组件渲染。

页面顶部提示：「排班发布成功后，系统每天将自动排15天后的班」。

### 第二步：切换周视图

通过顶部的三个按钮切换查看周期：

| 按钮 | 行为 |
|------|------|
| 本周 | 显示当前自然周的排班（周一到周日） |
| 上周 | 显示上一周的排班 |
| 下周 | 显示下一周的排班 |

当前选中的按钮以高亮（primary样式）显示。每切换一次，页面根据目标周的周一日期重新查询数据。

### 第三步：筛选排班

支持以下筛选条件：

- **医生姓名**：输入医生姓名模糊搜索（最多5个字）
- **科室**：树形下拉选择科室
- **排班时间**：日期范围选择器，选择时间段后按自定义日期范围查询

当选择自定义日期范围后，「本周/上周/下周」按钮的高亮状态会消失，表示当前不在预设周视图中。

### 第四步：发布排班

页面右上角有「已发布」（绿色勾）和「待发布」（橙色感叹号）的图例说明。点击「发布」按钮，系统将当前排班状态发布生效。发布成功后提示「发布成功」并刷新列表。

**自动排班机制**：发布成功后，系统每天会自动为15天后的日期生成排班，无需管理员手动操作。

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 使用自定义日期范围查询 | 本周/上周/下周按钮高亮消失，切换到预设周时重新启用 |
| 发布操作 | 成功后刷新列表，使发布状态更新 |

## 关联功能

- 医生列表管理 → 排班的医生数据来源
- 人员审核 → 预约模式医生需在此配置排班
- 排班模板管理 → 排班模板可快速应用
