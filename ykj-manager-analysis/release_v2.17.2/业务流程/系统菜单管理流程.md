# 系统菜单管理流程

> 文档版本 1.0 | 面向读者：系统管理员

## 概述

系统菜单管理用于查看和了解管理后台及线下药房端的菜单结构、权限代码配置。菜单是权限体系的基础，决定了不同角色在系统中能看到哪些页面和操作按钮。

## 触发条件

- **入口路径**：系统管理 → 菜单管理
- **前置条件**：需拥有系统管理权限

## 步骤详解

### 第一步：选择菜单端

页面顶部有两个Tab页签：
- **管理端**：展示管理后台（运营/管理人员使用）的菜单树
- **线下药房**：展示线下药房端使用的菜单树

点击Tab页签进行切换，切换后左侧菜单树和右侧表格会刷新。

### 第二步：浏览菜单树

左侧以树形结构展示完整的菜单层级：
- 点击展开/折叠节点查看子菜单
- 树节点以层级缩进展示父子关系
- 未展开的节点前方有箭头图标指示

### 第三步：查看菜单详情

点击左侧树中的某个节点，右侧表格展示该节点下的**直接子菜单**列表：

| 列名 | 说明 |
|------|------|
| 类型 | 目录（可展开的菜单组）、菜单（可点击的页面入口）、按钮（页面上的操作按钮） |
| 父级菜单名称 | 该菜单项所属的上级菜单 |
| 菜单名称 | 菜单在系统中显示的名称 |
| 菜单URL | 菜单对应的页面路径 |
| 权限代码 | 该菜单/按钮关联的权限标识，用于角色权限控制 |

### 第四步：搜索过滤

右侧表格上方提供搜索框，输入菜单名称后按回车键可过滤显示结果：
- 支持模糊匹配
- 清空搜索框后恢复显示全部子菜单
- 搜索仅在当前选中的父级菜单下生效

## 异常处理

| 情况 | 系统处理方式 |
|------|------------|
| Tab切换时 | 重置右侧表格数据，展示新端口的菜单树根节点 |
| 菜单树为空 | 正常展示空树状结构 |
| 父级菜单下无子菜单 | 表格显示为空列表 |

## 关联功能

- **角色管理**：角色配置时会引用这里的菜单和权限代码，决定角色的页面访问范围
- **权限管理**：权限代码在权限管理中进行更细粒度的控制
- **用户管理**：用户通过分配角色间接获得菜单权限
