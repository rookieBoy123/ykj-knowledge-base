# 小程序主题风格配置流程

> 文档版本：2026-07-16 | 面向读者：运营、管理员

## 概述

小程序主题风格切换功能允许管理员在后台一键更换小程序的整体视觉配色方案。系统提供了 6 种预设主题色（如宁静蓝、生命绿等），点击即可预览效果，保存后小程序端即时生效。

## 触发条件

- 需要更换小程序的整体视觉风格
- 节假日/活动期间需要匹配特殊视觉主题
- 新医院上线需要定制品牌色

## 页面入口

管理后台 → 运营管理 → 装修设计 → 主题

## 步骤详解

### 第一步：进入主题配置页

页面展示当前已选的主题风格。页面背景和手机预览图会根据选中的主题自动变化，方便直观对比不同主题的视觉效果。

### 第二步：选择主题

系统提供 6 种预设主题色，每种主题有对应的名称和色块供预览：

| 主题名称 | 主色调 | 
|------|------|
| 宁静蓝 | #3E86FF（蓝色系，默认） |
| 生命绿 | #03C776（绿色系） |
| 疗愈青 | #35D5D4（青色系） |
| 深海蓝 | #0D66D4（深蓝色系） |
| 安宁紫 | #9177FF（紫色系） |
| 活力橙 | #FF5F0C（橙色系） |

点击任意主题色块即可选中，选中的主题会有蓝色边框和勾选标记。页面背景图和手机预览图会同步更新为对应主题的视觉效果。

### 第三步：保存配置

确认主题后，点击底部的「保存修改」按钮。系统通过系统配置接口（`settings.allEdit`）更新 `ykj.wx-mini.default-theme` 配置项的值，保存后弹出「操作成功」提示。

如果未选择主题直接点击保存，系统提示「请先选择主题」。

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 未选择主题就保存 | 前端直接拦截，提示「请先选择主题」 |
| 配置接口调用失败 | 由系统配置模块的统一错误处理返回错误提示 |
| 主题值非法（非0-5） | 后端 `SettingValidator` 校验不通过，提示「配置项不合法」 |

## 关联功能

- **系统全局配置**：主题配置通过系统设置接口（`SysSettingMgmtService`）保存，与全局配置管理（如支付宝支付开关、小程序名称等）共用同一套配置存储和变更历史机制
- **自定义页面管理**：主题色不影响自定义装修页面的设计，装修页面的配色由页面设计师独立控制
- **配置历史回滚**：主题变更会记录在系统配置历史中，可通过配置历史回滚功能恢复到之前的主题

## 技术说明

主题配置通过系统配置项 `ykj.wx-mini.default-theme` 存储，值为 0-5 的字符串。小程序端通过读取该配置项来确定当前使用的主题色方案。
