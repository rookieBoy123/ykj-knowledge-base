# 互联网机构信息管理流程

> 文档版本 2026-07-04 | 面向管理员

## 概述

管理员在后台维护互联网医院的机构信息。页面包含两个区域：**互联网医院信息**和**实体医院信息**，涵盖机构标识、分类、安全等级、地址、许可证等约30个字段。数据为单一记录，不支持多条，上传后覆盖更新。

## 触发条件

- 需要更新互联网医院在监管平台的注册信息时
- 机构信息变更（如地址迁移、许可证续期、法人变更等）
- 入口：后台 → 医院管理 → 互联网机构管理

## 步骤详解

### 第一步：查看/加载现有机构信息

页面加载时自动从后端获取当前已保存的机构信息。如果没有已保存数据，所有字段为空，需要首次填写。信息分为两大区域：

**互联网医院信息区**（18个字段）：
- 机构标识、机构名称、单位隶属关系、信息安全等级保护证书编号
- 机构类型、机构成立日期、机构分类、经济类型
- 信息安全等级保护、机构分类管理类别、是否分支机构
- 机构所在地民族自治地方标志、部署平台、是否具备双路供电或紧急发电设施
- 地址（省市区级联选择 + 详细地址）、互联网医院网址、第二机构名称
- 邮政编码、E-mail、电话号码、机房面积
- 第三方机构统一社会信用代码、第三机构名称、协议有效期
- 机构描述、备注

**实体医院信息区**（9个字段）：
- 统一社会信用代码、实体医院名称、许可证号码
- 实体医院机构等级、实体医疗机构级别、开办资金金额数（万元）
- 法人代表/负责人、许可项目名称
- 许可证有效期（日期范围选择器，开始日期-结束日期）

### 第二步：编辑信息

管理员可以直接修改任意字段。关键字段说明：

- **地址**：使用省市区三级级联选择器 + 手动输入详细地址，两者配合使用
- **许可证有效期**：使用日期范围选择器，选择起止日期
- **机构成立日期**：单个日期选择器
- 选择类字段（如机构类型、经济类型、部署平台等）的数据从系统字典中加载

### 第三步：保存

点击页面底部的「保存」按钮，系统对全部必填字段进行校验。校验证过则提交，否则提示缺失字段。

所有必填字段（21个）：
机构标识、机构名称、单位隶属关系、机构类型、机构成立日期、经济类型、机构分类、机构分类管理类别、信息安全等级保护、是否分支机构、机构所在地民族自治地方标志、部署平台、是否具备双路供电或紧急发电设施、地址、互联网医院网址、统一社会信用代码、实体医院名称、许可证号码、实体医院机构等级、实体医疗机构级别、开办资金金额数、法人代表/负责人、许可项目名称、许可证有效期。

保存时会将所有选择类字段的值和对应中文名称一同提交，方便后续展示。

### 第四步：重置

点击「重置」按钮会弹出确认提示："是否确认重置?"。确认后重新从后端获取已保存的最新数据，覆盖当前所有编辑内容。

## 异常处理

| 情况 | 系统处理方式 |
|------|------------|
| 必填字段未填写 | 点击保存时在对应字段下方显示红色提示，无法提交 |
| 地址只选了省市区但未填详细地址 | 提示"请完善地址信息" |
| 许可证有效期未选择 | 提示"请选择日期" |
| 保存请求失败 | 显示后端返回的错误信息 |

## 关联功能

- **医院基本信息管理**：另一页面维护医院基本配置信息（如医院名称、Logo等）
- 无子页面跳转，为单一表单页
