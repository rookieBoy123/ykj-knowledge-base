# 疾病知识管理流程

> 文档版本 2026-07-05 | 面向读者：运营/内容管理 | 管理后台路径：运营管理 → 医学百科 → 疾病知识

## 概述

疾病知识是「医学百科」中医知识库的又一个子模块，用于维护常见疾病的知识科普内容。运营人员通过管理后台新增/编辑疾病条目，每条包含疾病名称、所属分类和详细的疾病介绍（支持富文本图文混排）。维护后的内容可展示到小程序患者端的健康知识板块，供用户查阅。

## 触发条件

- **前置条件**：拥有「医学百科-疾病知识」模块的查看和编辑权限（权限码：`hosp:wiki:disease-knowledge`）
- **入口路径**：管理后台 → 运营管理 → 医学百科 → 疾病知识

## 步骤详解

### 第一步：浏览疾病知识列表

进入页面后，系统自动加载已维护的疾病条目，默认每页10条。每条显示：

- **疾病名称**：如"糖尿病""高血压""冠心病"等
- **疾病分类**：从数据字典中选择的分类名称，如"内科""外科""妇科"等
- **最后修改时间**：最近一次编辑的时间
- **操作按钮**：编辑、删除

顶部筛选区支持按「疾病名称」输入关键词模糊搜索，按「疾病分类」下拉框精确筛选（下拉选项来自数据字典，字典编码：jbzs-jbfl）。

### 第二步：新增疾病知识

点击列表上方「新增」按钮，弹出新增弹窗（800像素宽）。需要填写：

- **疾病名称**（必填）：最多20个字符，不能与已有疾病重名
- **疾病分类**（必填）：单选下拉框，选项来源于「疾病知识-疾病分类」字典（字典编码：jbzs-jbfl）
- **疾病详情**（必填）：使用富文本编辑器编写，支持图文混排、格式排版，高度200像素

填写完成后点击「确定」提交。成功后弹窗关闭，列表自动刷新（新增时回到第一页）。

### 第三步：编辑已有疾病

列表每行右侧点击「编辑」，弹出编辑弹窗。系统自动回填该条目的疾病名称、分类和详情。修改后点击「确定」提交更新。

- 编辑弹窗标题显示「编辑」，新增弹窗标题显示「新增」
- 编辑时名称可以修改，但修改后不能与已有疾病重名（排除自身）

### 第四步：删除疾病

**单个删除**：列表每行右侧点击「删除」，弹出确认框「数据删除后不可恢复，您确认要删除吗？」，点击「确定」执行。

**批量删除**：勾选列表中多项后，点击「批量删除」按钮，弹出确认框后一次性删除所有选中条目。

> 与中药图谱不同：疾病知识的批量删除在没有勾选条目时同样无提示，需先勾选再操作。

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 疾病名称为空 | 提交时表单校验不通过，提示「请填写疾病名称」 |
| 疾病名称超过20字 | 输入框限制最大20字，无法输入更多 |
| 疾病名称已存在 | 后端返回提示「疾病名称已存在」 |
| 疾病分类未选择 | 提交时校验提示「请选择疾病分类」 |
| 疾病详情为空 | 提交时校验提示「请填写疾病详情」 |
| 编辑时疾病ID不存在 | 后端返回「疾病知识不存在」 |
| 分类编码不在字典中 | 后端校验时提示「字典:jbzs-jbfl, 编码:xxx不存在」 |
| 删除时未选择条目 | 前端无任何提示，需先勾选 |

## 关联功能

- **数据字典**：疾病分类（jbzs-jbfl）的选项从数据字典维护
- **小程序患者端展示**：患者在小程序的健康知识板块中查看疾病知识内容（通过 ApiDiseaseKnowledgeController 的 `findPageForApp` 接口获取）
- **中药图谱**：同属医学百科子模块，共享字典校验机制
- **药膳食疗、经络穴位、经典名方**：同属医学百科的其他子模块，管理模式类似
