# 发票管理流程

> 文档版本 2026-07-03 | 面向运营/财务人员

## 概述

管理所有订单的发票申请与开票操作。系统支持个人发票和企业发票两种类型，运营人员可以在后台查看用户的发票申请记录，手动完成开票确认，并支持批量导出。

## 触发条件

- **前置条件**：用户在订单中申请了发票
- **入口路径**：财务管理 → 发票管理

## 步骤详解

### 第一步：切换发票类型

页面顶部可切换查看两种发票类型：

| 标签 | 说明 |
|:---|:---|
| 个人发票 | 用户以个人名义申请的发票，抬头为个人姓名 |
| 企业发票 | 用户以企业名义申请的发票，抬头为企业名称（含税号） |

切换后列表会动态变化：企业发票标签下，列表会增加"企业名称"和"企业纳税号"两列。

### 第二步：筛选与查询

支持按以下条件筛选：
- **申请时间**：日期范围筛选，可快速查看某个时间段内的发票申请
- **发票状态**：全部 / 已开票 / 待开票

### 第三步：查看发票记录

列表中每条记录显示：

| 列 | 说明 |
|:---|:---|
| 订单号 | 发起发票申请的订单编号 |
| 金额（元） | 订单总金额 |
| 订单类型 | 处方订单/问诊订单/会诊订单/预约订单 |
| 发票抬头 | 个人发票显示姓名，企业发票显示企业名称 |
| 姓名 | 申请人真实姓名 |
| 身份证号 | 申请人身份证号码 |
| 手机号 | 申请人手机号 |
| 邮箱 | 发票接收邮箱 |
| 申请时间 | 用户提交发票申请的时间 |
| 开票状态 | 待开票 / 已开票 / 开票失败 |

### 第四步：开票操作

对于待开票的记录，操作列显示"开票"按钮：

1. 点击"开票"，系统弹出确认对话框
2. 确认后，该记录的开票状态更新为"已开票"
3. 列表自动刷新

对于处方订单（sourceType=1），操作列额外显示"查看详情"按钮，可查看处方的详细信息。

对于已开票或不可开票的记录，操作列的按钮会变为"-"（不可操作）。

### 第五步：导出

点击列表上方的"导出"按钮：
- 个人发票导出为"个人发票.xlsx"
- 企业发票导出为"企业发票.xlsx"
- 导出时会显示"加载中"提示，完成后自动下载

## 异常处理

| 情况 | 系统处理方式 |
|:---|:---|
| 订单不可开票（billingFlag ≠ 1） | 操作列显示"-"，不可点击开票 |
| 已开票的订单 | 操作列显示"-"，不允许重复开票 |
| 确认开票时取消操作 | 不执行任何操作，保持待开票状态 |
| 处方订单的详情查看 | 弹出详情弹窗，展示处方相关内容 |

## 关联功能

- **处方管理**：处方订单的发票可关联查看处方详情
- **订单管理**：所有订单均可在此查看发票状态
- **数据导出**：支持按筛选条件导出完整发票列表
