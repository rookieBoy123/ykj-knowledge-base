# 用户协议内容管理流程

> 文档版本 2026-07-04 | 面向内容运营管理员

## 概述

运营管理员在后台管理用户协议的内容配置，包括患者端协议、医生端协议和其他协议类型的编辑与维护。协议内容支持富文本编辑器，可嵌入图片等多媒体内容。

## 触发条件

- **前置条件**：登录账号具有内容管理相关权限
- **入口路径**：内容/运营管理 → 文章管理 → 用户协议管理

## 步骤详解

### 页面加载

进入页面后展示协议列表，列出已配置的协议条目。列表包含：选择框、协议标识、协议类型、标题和操作按钮。

### 查看协议列表

列表展示每条协议的关键信息：
- **协议标识**：系统内部唯一编码
- **协议类型**：如"患者端协议""医生端协议""其他协议"等（由数据字典 `xylx` 类别提供）
- **标题**：协议的中文标题
- **操作**：仅有「编辑」按钮（不可新增、不可删除）

### 编辑协议内容

1. 点击目标协议的「编辑」按钮，弹出编辑窗口（800px 宽）
2. 编辑窗口中包含以下字段：
   - **协议标识**：显示当前协议的编码，编辑模式下不可修改（置灰）
   - **协议类型**：显示当前协议类型，编辑模式下不可修改（置灰）
   - **标题**：可修改，最多 255 个字符
   - **富文本编辑器**：使用 TinyMCE 富文本编辑器，可编辑协议的详细内容（支持图文混排）
3. 修改完成后点击「确定」提交
4. 系统保存后提示"修改成功"，窗口关闭并刷新列表

### 协议标识和类型的锁定

编辑协议时，协议标识和类型不可修改，这是为了防止误操作导致协议标识与系统其他模块的引用关系断裂。小程序端和医生端会根据协议标识来查找对应的协议内容展示给用户。

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 未填写标题 | 点击保存时前端校验提示"请输入标题" |
| 未填写标识 | 点击保存时前端校验提示"请输入标识" |
| 保存接口失败 | 不提示具体错误（代码中注释掉了错误提示），界面无变化 |
| 列表分页超出范围 | 翻页自动回退到上一页 |

## 关联功能

- **小程序用户端**：注册/使用服务时展示的协议内容来源
- **医生端小程序**：医生入驻/使用服务时展示的协议内容来源
