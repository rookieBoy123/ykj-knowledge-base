# AI医生管理流程

> 文档版本 2026-07-04 | 面向读者：运营管理员

## 概述

管理员管理AI医生（智能问诊机器人医生），基于模板医生创建AI分身，配置AI医生的基本信息和问诊价格，支持启用/禁用、编辑和主页二维码生成。

## 触发条件

- **前置条件**：具有AI医生管理权限
- **入口路径**：管理后台 → 医院管理 → 医生管理 → AI医生列表

## 步骤详解

### 第一步：查看AI医生列表

进入页面后展示所有AI医生，每条记录显示：

- **编号**
- **医生姓名**（AI医生名称）
- **模板医生**（关联的真实医生，作为AI的知识和风格模板）
- **所属医院**
- **科室**
- **职称**
- **联系电话**（脱敏显示，点击可查看完整号码）
- **问诊价格（元）**（显示格式保留小数）
- **状态**：开关切换，绿色=启用，灰色=禁用
- **创建时间**

列表支持按医生姓名和科室筛选。

### 第二步：新增AI医生

点击「新增」按钮，弹出新增弹窗（通过子组件 `doctorDetail` 实现），需要填写：

- AI医生基本信息：姓名、头像
- 模板医生关联：选择真实医生作为AI模板
- 医院和科室信息
- 问诊价格设置

### 第三步：编辑AI医生

点击列表行的「编辑」按钮，弹窗回填现有信息，可修改AI医生的基本信息和问诊价格。

### 第四步：查看/刷新主页二维码

点击列表行的「主页二维码」按钮，弹出二维码弹窗显示AI医生的专属二维码：
- 显示AI医生姓名作为弹窗标题
- 二维码图片居中展示
- 底部显示「刷新二维码」按钮（蓝色加粗文字）
- 如果AI医生尚无二维码，系统自动生成一个

刷新二维码功能：
- 点击「刷新二维码」后系统重新生成新的二维码
- 旧二维码自动失效
- 刷新后弹窗内显示新的二维码图片

### 第五步：启用/禁用AI医生

列表每行的状态列是一个开关控件：
- **启用（绿色）**：AI医生可正常接受问诊
- **禁用（灰色）**：AI医生停止服务

点击开关切换时：
- 启用操作：调用启用接口，成功提示「启用成功」
- 禁用操作：调用禁用接口，成功提示「禁用成功」
- 启用和禁用使用不同的API接口，有独立权限控制

## 异常处理

| 情况 | 系统处理方式 |
|------|------------|
| AI医生尚无二维码 | 查看二维码时自动调用生成接口，生成后展示 |
| 添加/编辑数据不完整 | 子组件 `doctorDetail` 内部表单校验拦截 |
| 电话号码查看 | 点击脱敏字段触发验证，获取完整号码展示；权限验证通过后才返回 |
| 二维码图片加载失败 | 可点击「刷新二维码」重新生成 |

## 关联功能

- **医生列表管理**：AI医生的「模板医生」关联到真实医生
- **问诊管理**：患者通过AI医生主页二维码进入问诊
- **小程序端AI问诊**：患者在微信小程序通过扫码进入AI问诊
