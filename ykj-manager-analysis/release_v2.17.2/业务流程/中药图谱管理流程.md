# 中药图谱管理流程

> 文档版本 2026-07-05 | 面向读者：运营/内容管理 | 管理后台路径：运营管理 → 医学百科 → 中药图谱

## 概述

中药图谱是「医学百科」中医知识库的子模块，用于维护中药的基本信息、图文详情和使用指导。运营人员在管理后台新增/编辑中药条目后，内容会同步展示到小程序患者端的「中药百科」页面，供用户查阅每种中药的功效、用法和注意事项。

## 触发条件

- **前置条件**：拥有「医学百科-中药图解」模块的查看和编辑权限（权限码：`hosp:wiki:herb-illustrations`）
- **入口路径**：管理后台 → 运营管理 → 医学百科 → 中药图谱

## 步骤详解

### 第一步：浏览中药图谱列表

进入页面后，系统自动加载已维护的中药条目列表，默认每页10条。每条显示：

- **封面图**：中药材的缩略图
- **中药名称**：如"人参""黄芪""当归"等
- **药味**：从数据字典中选择的一种或多种药味，如"甘""苦""辛"等，多选时以顿号分隔显示
- **适应症**：从数据字典中选择的一种或多种适应症，多选时以顿号分隔显示
- **最后修改时间**：最近一次编辑的时间
- **操作按钮**：编辑、删除

顶部筛选区支持按「中药名称」输入关键词模糊搜索，按「药味」和「适应症」下拉框精确筛选（下拉选项来自数据字典）。

### 第二步：新增中药条目

点击列表上方「新增」按钮，跳转到新增页面。需要填写以下信息：

**基础信息区：**

- **中药名称**（必填）：最多20个字符，不能与已有中药重名
- **药味**：多选下拉框，选项来源于「中药图解-药味」字典（字典编码：zytj-yw）
- **适应症**：多选下拉框，选项来源于「中药图解-适应症」字典（字典编码：zytj-syz）
- **展示封面图**：上传正方形图片（推荐200×200像素，1:1比例），最大2MB
- **功效作用**（必填）：最多100个字符的简短说明，如"大补元气，复脉固脱"

**中药详情区（四个标签页，使用富文本编辑器）：**

- **基本用法**（必填）：中药的基本使用说明，支持图文混排
- **药物作用**（必填）：药物的药理作用和功效详解
- **用药方法**（必填）：具体的服用方法、用量指导
- **注意事项**（必填）：禁忌人群、不良反应、用药提醒等

> 四个字段均为必填，提交时系统会逐一检查。如果某个标签页未填写，页面会自动切换到对应标签页并弹出警告提示。

填写完成后点击底部「确定」提交，成功后自动返回列表页。

### 第三步：编辑已有中药

列表每行右侧点击「编辑」，跳转到编辑页面。系统自动回填该条目的所有数据（名称、封面图、药味、适应症、功效作用及四个富文本详情）。修改后点击「确定」提交更新。

- 编辑时同样需要填写完整的四个详情标签页
- 名称可以修改，但修改后不能与已有中药重名（不跟自己比）

### 第四步：删除中药

**单个删除**：列表每行右侧点击「删除」，弹出确认框「数据删除后不可恢复，您确认要删除吗？」，点击「确定」执行。

**批量删除**：勾选列表中多项后，点击「批量删除」按钮，弹出确认框后一次性删除所有选中条目。

> 注意：批量删除前如果没有勾选任何条目，点击按钮不会有任何反应（无提示）。务必先勾选至少一条。

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 中药名称为空 | 提交时表单校验不通过，提示「请填写中药名称」 |
| 中药名称超过20字 | 输入框限制最大20字，无法输入更多 |
| 中药名称已存在 | 后端返回提示「中药名称已存在」，表单不关闭 |
| 药味/适应症不在字典中 | 后端校验时提示「字典:xxx, 编码:xxx不存在」 |
| 功效作用为空 | 提交时提示「功效作用不能为空」，限制100字 |
| 基本用法/药物作用/用药方法/注意事项为空 | 提交时切换到对应标签页并提示「请填写xxx」 |
| 编辑时中药ID不存在 | 后端返回「中药图解不存在」 |
| 删除时未选择条目 | 前端无任何提示（按钮无效果），需先勾选 |
| 新增加载时字典数据获取失败 | 药味和适应症下拉为空，无法选择 |

## 关联功能

- **数据字典**：药味（zytj-yw）和适应症（zytj-syz）的选项从数据字典维护
- **小程序患者端展示**：患者在小程序的「中药百科」中查看中药图谱内容（通过 ApiHerbIllustrationsController 的 `findPageForApp` 接口获取）
- **药膳食疗**：同属医学百科子模块，类似的 CRUD 管理模式
