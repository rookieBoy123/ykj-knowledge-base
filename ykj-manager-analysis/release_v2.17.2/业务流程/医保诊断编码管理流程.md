# 医保诊断编码管理流程

> 文档版本 2026-07-04 | 面向运营/管理员

## 概述
医保诊断编码管理用于维护"医保诊疗类别"对应的"医保编码"。每个诊疗类别（如"高血压"、"糖尿病"）在国家标准中都有一个监管编码，而各地医保系统又有自己的医保编码。管理员在这里填写各地医保系统要求的医保编码，实现国家标准编码到各地医保编码的映射。

## 触发条件
- 菜单路径：系统管理 → 医保管理 → 诊断管理
- 使用场景：对接新的医保系统时需要维护诊断编码映射，或医保编码变更时需要更新

## 页面布局
页面为标准的列表+编辑弹窗模式。列表展示监管编码、医保编码、医保诊疗类别三列。每行有"编辑"操作按钮。

## 步骤详解

### 第一步：搜索筛选
1. 在顶部搜索框中输入"医保诊疗类别"名称（支持模糊搜索）
2. 点击查询，列表刷新

### 第二步：编辑医保编码
1. 点击目标行的"编辑"按钮
2. 弹出编辑窗口，展示三项内容：
   - **医保诊疗类别**：只读显示，不可修改
   - **监管编码**：只读显示，来自国家标准
   - **医保编码**：可编辑输入框，最多32个字符
3. 输入新的医保编码（必须是纯数字值）
4. 点击"确定"保存

### 第三步：确认修改结果
1. 保存成功后提示"操作成功"
2. 弹窗关闭，列表自动刷新

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 医保编码输入非数字字符（如字母、中文） | 表单校验失败，提示"医保编码为数字值" |
| 医保编码不填 | 表单校验失败，提示"请输入医保编码" |
| 医保编码超过32个字符 | 输入框自动截断，限制最大32个字符 |

## 关联功能
- **医保结算数据上传**：诊断编码用于识别就诊诊断是否在医保报销范围内
- **问诊订单**：医生填写诊断后，系统根据此映射判断医保报销资格
- **标准科室诊断对照**：科室诊断对照管理中会使用到诊断编码
