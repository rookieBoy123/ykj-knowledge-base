# 量表发布与管理流程

> 文档版本 2026-07-03 | 面向运营/管理员

## 概述
量表管理用于创建和维护医学量表（如抑郁自评量表SDS、焦虑自评量表SAS等）。量表是一种标准化的医学评估工具，包含多道题目和评分规则。管理员可以新增、编辑、发布、禁用量表，还可以调整量表的展示顺序。

## 触发条件
- 菜单路径：诊疗管理 → 量表管理 → 量表列表
- 使用场景：新增一种医学评估量表、修改已有量表的题目、调整量表在前端展示的顺序

## 量表状态说明

量表有三种状态，每种状态下可执行的操作不同：

| 状态 | 值 | 描述 | 可用操作 |
|------|----|------|----------|
| 草稿 | 0 | 量表正在编辑中，暂未对患者可见 | 编辑、发布 |
| 已发布 | 1 | 量表已发布，患者可以使用 | 禁用、上移、下移 |
| 已禁用 | 2 | 量表被管理员主动禁用，不再展示给患者 | 启用、上移、下移 |

## 步骤详解

### 第一步：新增量表
1. 点击"新增量表"按钮，跳转到量表编辑页面
2. 在编辑页面中：
   - 设置量表基本信息（名称、分类、说明等）
   - 添加题目（questionList）
   - 设置评分规则（ratingList）
3. 保存后量表进入"草稿"状态

### 第二步：发布量表
1. 在草稿状态的量表行点击"发布"
2. 系统会先检查量表内容完整性：
   - 如果**题目列表为空**或**评分规则为空**，弹窗提示"该量表有内容未填写，请确认填写完整再发布"
   - 弹窗有两个选项："完善量表"跳转到编辑页面继续填写，"取消"放弃发布
3. 如果内容和评分规则都完整，直接发布
4. 发布成功后量表状态变为"已发布"，患者可在小程序中看到并使用

### 第三步：调整顺序（上移/下移）
- 已发布和已禁用的量表支持上移/下移
- 草稿状态的量表不支持排序
- 上移按钮在列表中第一页第一条时隐藏（已是最高位置）
- 下移按钮在列表中最后一页最后一条时隐藏（已是最低位置）
- 跨页排序也支持：如果当前页第一条点击上移，会与上一页最后一条交换位置

### 第四步：禁用与启用量表
- **禁用**：已发布状态的量表可以禁用，禁用后患者端不再可见此量表
- **启用**：已禁用状态的量表可以重新启用，启用后患者端恢复可见
- 启用实际上是再次调用"发布"接口

### 第五步：编辑量表
- 仅草稿状态可编辑
- 点击"编辑"跳转到量表编辑页面，携带量表ID参数
- 编辑完成后仍需重新发布才能对患者可见

## 页面缓存策略
- 从编辑页面返回列表时，如果`from.name === 'addEdit'`，则使用缓存数据不刷新
- 如果从其他页面进入，则重置搜索条件并重新加载列表

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 量表题目为空就点发布 | 弹窗提示"该量表有内容未填写"，可选择"完善量表"跳转编辑或"取消" |
| 量表评分规则为空就点发布 | 同上弹窗提示 |
| 草稿状态的量表尝试上移/下移 | 上移下移按钮不显示（isShow条件排除status===0） |
| 已发布的量表尝试编辑 | 编辑按钮不显示（isShow条件仅status===0时显示） |

## 关联功能
- **量表分类管理**：量表列表中"量表分类"下拉选项从分类管理页面获取
- **患者端量表测评**：已发布的量表在患者小程序中可用
- **患者测评详情**：后台可查看患者完成的测评记录
- 服务端：`MgmtScaleController`、`MgmtScaleQuestionController`、`MgmtScaleRatingController`（模块：`ykj-service-scale`）
