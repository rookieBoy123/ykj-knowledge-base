# 药品渠道（药房配送渠道）管理流程

> 文档版本 2026-07-04（第5轮） | 面向运营/产品人员

## 概述

药品渠道管理用于维护平台上的药房基础信息（包括线上药房和线下药房），配置药房的库存管理模式、加工服务等。配置完成后，医生开处方时可以选择对应药房进行药品配送。

## 触发条件

- 前置条件：具有药房管理权限（`med:pharmacies:write`）
- 入口路径：后台 → 医院管理 → 药房信息 → 配送渠道

## 步骤详解

### 第一步：查看药房列表

进入页面后，系统展示所有药房的列表，显示以下信息：

| 列名 | 说明 |
|------|------|
| ID | 药房编号 |
| 药房类型 | 线上药房 / 线下药房 |
| 药房名称 | 药房的名称 |
| 负责人 | 药房负责人姓名 |
| 状态 | 启用/禁用（可直接在列表切换） |
| 库存管理 | 启用库存 / 禁用库存 |
| 创建时间 | 药房的创建时间 |

列表支持按**药房名称**、**状态**（启用/禁用）、**药房类型**（全部/线上/线下）进行筛选。

> **重要提示**：如果系统配置为「严格模式」（`system.mode === 'strict'`）或者系统关闭了线下药房功能（`pharmacy.offlineEnabled` 为 false），则药房类型列和筛选条件中的「药房类型」会被自动隐藏，所有药房统一按线上药房处理。

### 第二步：新增药房

点击「新增」按钮进入药房编辑页面，包含两个标签页：

**标签页1：基础信息**
- 选择药房类型（线上/线下），选择线下药房后才会开放加工服务标签页
- 填写药房名称、负责人、联系电话、地址等基本信息
- 配置经营药品类型（如果包含中药饮片，则需要填写加工服务）

**标签页2：加工服务**（仅线下药房且经营中药饮片时有）
- 配置该药房可提供的中药加工服务类型

填写完成后点击「确认新增」保存。系统会校验基础信息是否完整，不完整则提示"请完善基础信息"。

### 第三步：编辑药房

点击列表中的「编辑」进入编辑页面，页面结构与新增相同，预填已有数据。修改后点击「保存」。

### 第四步：管理药房药品

每个药房都有「管理药品」操作按钮，点击后跳转到该药房的药品管理页面，可以管理该药房可售卖的药品清单。

### 第五步：调整药房展示顺序

药房在医生端开方时按顺序展示，管理员可以通过以下操作调整：
- **排至最前**：将该药房移到列表第一位
- **排至最后**：将该药房移到列表最后
- **上移**：向前移动一位
- **下移**：向后移动一位

### 第六步：启用/禁用药房

在列表中通过开关直接切换药房的启用/禁用状态，立即生效。禁用的药房在医生端开方时不可选。

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 保存时基础信息不完整 | 自动切换到基础信息标签页，提示"请完善基础信息" |
| 删除药房 | 弹出确认框"此操作将永久删除该文件, 是否继续?"，确认后删除 |
| 药房类型列被隐藏 | 由系统全局配置控制（严格模式或关闭线下药房时自动隐藏） |

## 关联功能

- **医生药房授权**：管理员可以将药房授权给特定医生使用
- **药房药品管理**：每个药房的药品清单独立管理
- **药房证照管理**：线下药房需要上传经营证照（见药房证照管理流程）
- **医生端开方**：医生开药时从已授权的药房中选择药品
