# 运费规则设置流程

> 文档版本 1.0 | 面向角色：运营管理员

## 概述

运费规则设置用于配置药品配送的运费策略。管理员可以为不同地区设置不同的运费标准，支持统一运费、满额包邮、货到付款等多种模式。未单独配置的地区适用"其他地区"的默认规则。

## 触发条件

- **前置条件**：登录管理后台，拥有运费设置权限
- **入口路径**：管理后台 → 运营管理 → 运费设置

## 步骤详解

### 第一步：进入运费设置页面

页面展示当前的运费规则列表。每个规则包含：
- **支持地区**：该规则适用的省/市/区
- **运费金额**：基础运费（元）
- **满额包邮**：订单总价达到一定金额后免运费
- **到付**：是否支持货到付款

最后一项始终是"其他未选择地区"——这是一条特殊规则，适用于所有没有被单独配置的地区。

### 第二步：新增运费条件

点击「新增条件」按钮，在列表中新增一条运费规则（插入在其他地区规则之前）。

**限制**：如果上一条规则的地区尚未选择，会提示"请先填写完新条件，再继续新增"。

新增的规则默认：运费0元，不启用满额包邮，不启用货到付款。

### 第三步：选择适用地区

点击规则中的「选择地区」按钮，弹出地区选择器（省/市二级级联选择），可以多选。

**地区冲突校验**：
- 同一规则内：选择了一个省后，不能再同时选择该省下的市（提示"选择省后不能在选择所在市区"）
- 不同规则间：一个地区只能属于一条运费规则。如果选择的地区已被其他规则使用，提示"一地区只支持设置一个条件，该地区已存在，请确认后再添加"

选择完成后点击「确定」，所选地区以标签形式展示在规则中，可以点击标签的关闭按钮移除单个地区。

### 第四步：设置运费参数

每个规则支持三种运费模式的组合：

#### 统一运费
直接输入运费金额（元），设置为0即包邮。此设置仅在不勾选"到付"时有效。

#### 满额包邮
勾选"订单总价金额满__元包邮"，输入包邮门槛金额。当订单总价达到该金额时，该地区的订单免运费。

**依赖关系**：
- 只有在"统一运费"模式下才能设置（勾选"到付"后此项自动禁用）
- 勾选满额包邮后，必须填写包邮门槛金额

#### 货到付款（到付）
勾选"到付"，该地区订单采用货到付款方式，无需设置运费金额。勾选后，运费输入框和满额包邮设置自动禁用，保存时运费相关字段自动清空。

### 第五步：设置"其他地区"默认规则

列表最后一项"其他未选择地区"的配置与普通规则略有不同：

- **地区选择**：不需要也不能选择地区（自动适用于所有未单独配置的地区）
- **运费模式**：可以在"统一运费"和"统一到付"之间选择（单选按钮）
- **免除运费开关**：除了上述设置外，还有一个"可设为免除运费"开关。开启后，医生在开具处方时可以手动将运费调整为包邮（0元）。这是一个灵活的运营功能。

### 第六步：删除条件

点击规则的「删除」按钮，弹出确认对话框"确认删除吗？删除后该条件不作为运费设置条件"。确认后该规则被移除，其覆盖的地区将自动归入"其他未选择地区"的规则。

### 第七步：保存

点击「保存」按钮，系统会进行一系列校验：

- 所有有地区的规则必须至少选择一个地区
- 未启用"到付"的规则必须填写运费金额
- 运费不能为负数
- 启用满额包邮的规则必须填写包邮金额且不能为负数

如果页面没有任何修改，提示"无修改内容"。

校验通过后，所有规则一次性提交保存。

**离开提醒**：如果在有未保存修改的情况下尝试离开页面，会弹出提示"有更新内容，请先保存"，提供「保存」和「不保存」两个选项。

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 未选择地区就新增下一条 | 提示"请先填写完新条件，再继续新增" |
| 未选择地区就保存 | 提示"请添加地区再保存" |
| 同一规则内选省又选市 | 提示"选择省后不能在选择所在市区" |
| 选择的地区已被其他规则使用 | 提示"一地区只支持设置一个条件，该地区已存在，请确认后再添加" |
| 未设置运费就保存 | 提示"请设置运费再保存" |
| 运费为负数 | 提示"运费不能小于零" |
| 满额包邮未填写金额 | 提示"请正确填写满额包邮订单金额" |
| 无修改内容 | 提示"无修改内容" |
| 有未保存修改离开页面 | 弹出确认框"有更新内容，请先保存" |

## 关联功能

- **医生开方**：运费规则直接影响处方订单的配送费用计算
- **药品订单**：下单时根据收货地址匹配对应的运费规则
- **地区数据**：省/市数据来自统一的地区数据接口
