# 远程会诊列表与备案管理流程

> 面向管理员 | 管理后台功能（远程医疗模块）

## 概述

管理员查看已审核通过或已完成的远程会诊记录，追踪会诊状态，并查看会诊详情。主要用于会诊的后续跟踪和备案管理。

## 触发条件

- **前置条件**：拥有管理后台权限
- **入口路径**：管理后台 → 诊疗管理 → 远程医疗中心 → 远程会诊列表

## 步骤详解

### 第一步：筛选会诊记录

页面顶部提供两个快速筛选标签页：

| 标签 | 说明 |
|------|------|
| 待备案 | 会诊已结束、待完成备案的会诊记录 |
| 已完成 | 会诊已全部完成 |

左侧提供多条件搜索：
- **申请单编号**：精确查询，最多20字
- **患者姓名**：模糊查询，最多5字
- **医生姓名**：模糊查询，最多5字
- **状态**（仅在"待备案"标签下显示）：下拉选择——会诊待开始/会诊进行中/待会诊报告

### 第二步：查看列表

表格展示会诊记录，列包括：

| 列名 | 说明 |
|------|------|
| 会诊编号 | 唯一标识 |
| 患者姓名 | 默认脱敏显示（首字+星号），点击可查看完整姓名 |
| 医生姓名 | 申请医生 |
| 经治科室 | 医生所属科室 |
| 当前诊断 | 诊断信息 |
| 会诊时间 | 会诊开始时间 |
| 状态 | 会诊流转状态 |
| 操作 | 查看详情按钮 |

**患者姓名脱敏机制**：列表中仅显示"张***"格式的脱敏姓名。点击某行查看完整姓名时，其他行的脱敏状态自动恢复（同一时间只有一行的患者姓名可见明文）。

### 第三步：查看会诊详情

点击"查看详情"按钮，弹出大尺寸弹窗（宽900px），展示该会诊的完整信息，包括患者、医生、诊断、会诊安排、受邀专家等。

---

## 会诊状态说明

会诊从申请到完成共经过7个状态：

| 状态码 | 状态名称 | 说明 |
|--------|---------|------|
| 1 | 会诊申请 | 医生已提交申请，等待审核 |
| 2 | 会诊待备案 | 审核通过，等待备案 |
| 3 | 会诊待患者确认 | 等待患者确认参与 |
| 4 | 会诊待开始 | 患者已确认，等待会诊开始时间 |
| 5 | 会诊进行中 | 会诊正在进行 |
| 6 | 待会诊报告 | 会诊结束，等待出具会诊报告 |
| 7 | 已完成 | 会诊报告已出具，全部完成 |

"待备案"标签页显示状态4-6的会诊，"已完成"标签页显示状态7的会诊。

---

## 异常处理

| 情况 | 系统处理方式 |
|------|------------|
| 无符合条件的会诊记录 | 列表为空 |
| 切换"待备案"→"已完成" | 自动移除状态筛选条件 |
| 切换"已完成"→"待备案" | 自动添加状态筛选条件 |

## 关联功能

- **远程会诊审核管理**：审核通过后的会诊进入此列表
- **远程会诊详情**：弹窗中的详情与审核页面共用同一详情组件
