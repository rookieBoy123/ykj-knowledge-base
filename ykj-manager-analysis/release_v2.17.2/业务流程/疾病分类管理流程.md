# 疾病分类管理流程

> 文档版本 2026-07-04 | 面向读者：运营人员

## 概述

管理员在后台维护疾病症状分类，控制患者端"对症询医"入口显示哪些症状选项。支持症状的增删改查、排序、显示/隐藏切换。

## 触发条件

- 入口路径：管理后台 → 运营管理 → 疾病分类管理
- 适用角色：拥有医院管理权限的管理员

## 步骤详解

### 第一步：查看症状列表

进入页面后，系统展示症状分类列表，每条记录包含：
- **症状名称**：如"感冒""咳嗽""头痛"等
- **创建用户**：哪位管理员创建的
- **创建时间**：创建日期
- **是否显示**：开关按钮，绿色=显示在患者端，灰色=隐藏

列表按排序值从小到大排列，支持按症状名称和显示状态筛选。

### 第二步：新增症状

点击「新增」按钮，弹出表单：
- **症状名称**：必填，最多10个字
- **是否显示**：默认选中"显示"

填好后点击「确定」保存。新增的症状会自动排在列表末尾。

> ⚠️ 症状名称不能重复。如果输入的名称已存在，系统会提示"症状名称已存在"。

### 第三步：编辑症状

点击某条症状的「编辑」按钮，弹出表单，可修改症状名称和显示状态。

### 第四步：控制显示/隐藏

列表中每行有个「是否显示」开关：
- 点击开启：该症状出现在患者端"对症询医"模块
- 点击关闭：该症状从患者端隐藏，不再展示

注意：患者端"对症询医"最多显示 8 个症状。

### 第五步：调整排序

每条症状提供 4 种排序操作：
- **排至最前**：直接跳到第一位
- **上移**：与上一条交换位置
- **下移**：与下一条交换位置
- **排至最后**：直接跳到末尾

排序通过整数值实现：新建时自动获取最大排序值+1，上下移动通过交换相邻两条的排序值完成。

### 第六步：删除症状

支持两种删除方式：
- **单个删除**：点击某条的「删除」按钮，确认后删除
- **批量删除**：勾选多条后点击「批量删除」按钮

删除操作会同时清除已关联该症状的医生配置（如医生设置的擅长病种），且删除后数据无法恢复。

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 症状名称为空 | 前端阻止提交，提示"症状名称必填" |
| 症状名称重复 | 后端校验返回"症状名称已存在"，前端提示错误 |
| 删除时未勾选任何记录 | 提示"请至少选择一条数据" |
| 上移/下移时已是第一个/最后一个 | 操作静默无效（无提示） |
| 编辑时症状名称改为已存在的 | 后端校验排除自身ID后检查，重名则提示"症状名称已存在" |

## 关联功能

- **患者端对症询医**：显示的症状直接对应此处的疾病分类列表（仅显示visible=true的，最多8条）
- **医生擅长病种设置**：医生档案中关联的擅长病种引用此处的疾病分类，删除症状时会同步清除对应关联
