# 通讯录管理流程

> 文档版本：v1.0 | 面向读者：医院行政管理人员、运营人员

## 概述

通讯录管理功能用于维护医院内部各部门的联络人信息。管理人员可以对各部门的负责人和联系电话进行新增和编辑。系统会对部门选择进行唯一性校验，防止同一部门重复录入；对手机号进行格式校验，确保录入的联系电话有效。

## 触发条件

- **入口路径**：管理后台 → 医院管理 → 通讯录（地址库）
- **前置条件**：拥有「hosp:concats:write」权限才能新增联络人

## 步骤详解

### 第一步：查看通讯录列表

进入页面后，系统自动加载通讯录列表，以表格形式展示。每条记录包含：
- **序号**：自动编号
- **部门**：从数据字典「部门」中取值显示（如：医务科、护理部、药剂科等）
- **负责人姓名**
- **联系电话**
- **创建时间**
- **修改时间**

表格支持分页浏览。

### 第二步：新增联络人

点击「新增」按钮（需要权限），弹出新增表单，包含三个字段：

| 字段 | 类型 | 必填 | 校验规则 |
|------|------|------|---------|
| 部门 | 下拉选择 | 是 | 选项来自「部门」数据字典；提交时后台校验该部门是否已存在记录 |
| 负责人姓名 | 文本 | 是 | 不能为空 |
| 联系电话 | 文本 | 是 | 11 位手机号格式（1 开头 + 10 位数字） |

填写完成点击「确定」保存。

**部门唯一性校验**：如果选择的部门在通讯录中已存在记录，提交时系统会提示「当前校验部门记录已存在」，保存失败。这是为了避免同一部门被重复添加。

### 第三步：编辑联络人

在列表操作栏点击「编辑」，弹出编辑表单，预填当前记录的信息。注意：
- 编辑时如果未修改部门，不会触发唯一性校验
- 修改手机号时，如果值与当前值相同则不重复校验格式
- 修改后点击「确定」保存，列表刷新

### 第四步：查看详情

通讯录页面不支持删除操作（仅编辑）。如需删除某条记录，需要从服务端后台处理。

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 部门未选择 | 提示「请选择部门」 |
| 负责人姓名未填写 | 提示「请输入负责人姓名」 |
| 联系电话未填写 | 提示「手机号不能为空」 |
| 联系电话格式错误 | 提示「手机号格式错误」 |
| 选择的部门已存在记录 | 提示「当前校验部门记录已存在」 |
| 无「hosp:concats:write」权限 | 新增按钮不显示 |
