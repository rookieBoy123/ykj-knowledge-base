# 医保结算数据上传管理流程

> 文档版本：v1.0 | 面向读者：医保相关管理员、运营管理员

## 概述

医保结算数据上传管理用于管理慢性病医保结算数据的申报上传。系统将已结算的医保数据按批次上传到医保监管平台。管理员可以查看各数据的申报状态，对"待上传"和"上传失败"的数据进行单条或批量重新上传。

## 触发条件

- **前置条件**：已登录管理后台，具备医保管理权限
- **入口路径**：左侧菜单 → 财务管理 → 医保结算管理

## 步骤详解

### 第一步：选择数据状态

页面顶部有三个页签（tab），按上传状态分类：
- **待上传**（默认）：尚未申报到医保平台的数据
- **已上传**：已成功申报的数据
- **上传失败**：申报失败的数据

### 第二步：筛选查询

支持以下筛选维度：

| 筛选项 | 说明 |
|--------|------|
| 病人名称 | 患者姓名 |
| 结算日期 | 日期范围，默认最近30天 |
| 人员编号 | 医保人员编号 |
| 病种 | 慢性病病种，远程搜索选择 |
| 身份证号 | - |
| 科室 | 慢性诊断科室，树形选择 |
| 申报日期 | 仅在"已上传"页签显示，日期范围筛选 |

### 第三步：查看数据列表

列表展示字段：

| 列名 | 说明 |
|------|------|
| 选择框 | "待上传"页签显示勾选框，"已上传"页签隐藏 |
| 人员编号 | 医保系统的人员编号 |
| 患者姓名 | - |
| 慢性诊断科室 | - |
| 病种 | 慢性病名称 |
| 证件类型 | 固定显示"居民身份证(户口簿)" |
| 证件号 | 身份证号 |
| 性别 | - |
| 年龄 | - |
| 结算时间 | - |
| 申报时间 | 仅在"已上传"页签显示 |
| 操作 | 上传/重新上传/查看详情 |

### 第四步：上传操作

**单条上传**：点击某条数据的"上传"或"重新上传"按钮，弹出日期选择框，选择"申报时间"后点击确定完成上传。

**批量上传**：勾选多条"待上传"状态的数据，点击"批量上传"按钮，选择申报时间后一次性上传勾选的全部数据。如未勾选就点击批量上传，提示"请至少选择一条数据"。

上传成功后提示"上传成功"并刷新列表。

### 第五步：查看详情

点击"查看详情"可查看该条医保结算的完整信息。

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 批量上传时未勾选任何数据 | 提示"请至少选择一条数据" |
| 上传时未选择申报时间 | 提示"请选择申报时间" |
| 申报日期不能选择未来日期 | 日期选择器中未来的日期不可选（disabledDate限制） |
| "已上传"页签 | 不显示勾选框和批量上传按钮，单条也不显示上传按钮 |
| "待上传"页签 | 操作列显示"上传"按钮 |
| "上传失败"页签 | 操作列显示"重新上传"按钮 |

## 关联功能

- **慢性病管理**：病种筛选数据来源于慢性病管理模块
- **科室管理**：科室筛选数据来源于科室树形数据
- **交易流水**：医保结算数据与交易流水中的医保支付记录相互关联
