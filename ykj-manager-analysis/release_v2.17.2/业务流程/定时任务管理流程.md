# 定时任务管理流程

> 文档版本 2026-07-03 | 面向系统管理员

## 概述
定时任务管理用于查看和维护系统中的所有自动任务，包括定时任务（按Cron表达式自动执行）和一次性任务（只执行一次）。管理员可以查看任务列表、编辑Cron表达式、启用/禁用任务、手动触发执行，以及查看每个任务的历史执行日志。

## 触发条件
- 菜单路径：系统管理 → 任务调度 → 定时任务 / 执行日志
- 使用场景：调试某个定时任务不执行的问题、临时触发一次任务、调整任务执行频率

## 定时任务列表页面

### 功能综述
列表展示所有已注册的定时任务，包括任务类型、名称、描述、Cron表达式、当前状态（启用/禁用），以及编辑、触发、查看日志三个操作按钮。

### 筛选条件
- **任务类型**：下拉选择"全部"、"定时任务"(jobType=1)、"一次性任务"(注意代码中两个选项值都是1)
- **任务名称**：文本输入，模糊匹配
- **任务描述**：文本输入，模糊匹配

### 任务状态
每行有一个开关按钮，绿色"启用"表示任务正在按Cron表达式自动执行，灰色"禁用"表示任务已暂停。

### 操作：编辑任务
1. 点击某行的"编辑"按钮，弹出编辑窗口
2. 可修改两个字段：
   - **Cron表达式**：定义任务执行频率（如 `0 0 8 * * ?` 表示每天8点），最长64字符
   - **任务描述**：一句话说明该任务的作用，最长100字符
3. 两项均为必填
4. **限制**：一次性任务（jobType=2）的"编辑"和"触发"按钮为灰色，不可点击

### 操作：手动触发
1. 点击"触发"按钮，系统弹确认框"触发后定时任务将立即执行，是否触发任务？"
2. 确认后任务立即执行一次（不等于修改Cron，只影响本次）
3. 触发后提示"已触发"

### 操作：查看日志
点击"日志"按钮，跳转到执行日志页面，并自动筛选当前任务的记录。

## 执行日志页面

### 功能综述
查看所有定时任务的历史执行记录，包括成功和失败的。

### 筛选条件
- **任务**：下拉选择具体任务（动态从所有任务中加载）
- **执行状态**：成功(1)或失败(2)

### 展示字段
| 字段 | 说明 |
|------|------|
| 类型 | 定时任务/一次性任务 |
| 任务描述 | 任务名称说明 |
| 任务参数 | 本次执行携带的参数 |
| 计划执行时间 | Cron计算出的预定时间 |
| 实际执行时间 | 本次实际触发的时间 |
| 耗时(ms) | 执行所耗毫秒数 |
| 执行状态 | 成功(绿色)或失败(红色)，失败时鼠标悬浮可看到错误原因 |

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 编辑时Cron表达式为空 | 表单校验失败，红色提示"请输入cron表达式" |
| 编辑时任务描述为空 | 表单校验失败，红色提示"请输入任务描述" |
| 禁用定时任务 | 前端直接调用禁用接口，关闭后任务不再自动执行 |
| 启用定时任务 | 直接调用启用接口，任务恢复自动执行 |
| 手动触发一次性任务 | "触发"按钮灰色，不可点击（代码判断jobType===2） |
| 任务执行失败 | 日志记录失败状态(2)，鼠标悬浮可查看错误原因(error字段) |
