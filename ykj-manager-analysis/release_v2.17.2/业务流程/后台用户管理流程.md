# 后台用户管理流程

> 文档版本 1.0 | 面向角色：系统管理员

## 概述

后台用户管理用于维护管理后台的登录账号。管理员可以新增、编辑后台用户，为用户分配角色（决定其能访问哪些功能），启用/禁用账号，重置密码，以及管理用户的访问密钥。

## 触发条件

- **前置条件**：登录管理后台，拥有用户管理权限（`sys:users:write`）
- **入口路径**：管理后台 → 系统管理 → 系统设置 → 用户管理

## 步骤详解

### 第一步：查看用户列表

进入页面后展示所有后台用户列表，包含以下信息：

| 列名 | 说明 |
|------|------|
| ID | 用户唯一编号 |
| 用户名 | 登录账号名 |
| 姓名 | 用户的真实姓名 |
| 手机号码 | 脱敏显示（如 138****1234），可点击"眼睛"图标查看完整号码 |
| 邮箱 | 用户邮箱地址 |
| 角色 | 用户被分配的角色列表（如"管理员、运营"），多个角色用逗号分隔 |
| 状态 | 显示为开关按钮，可快速切换启用/禁用 |
| 创建时间 | 账号创建时间 |
| 操作 | 编辑、修改用户名、重置密码、访问密钥 |

**筛选条件**：可按用户名、角色、状态进行筛选。

**手机号脱敏**：列表中的手机号默认脱敏显示。管理员可以点击手机号旁的"眼睛"图标，经过服务端验证后显示完整的手机号码。同时只能查看一个用户的完整手机号（点击新的会自动隐藏之前显示的）。

### 第二步：新增用户

点击「新增」按钮，弹出新增用户对话框，需要填写：

- **用户名**（必填）：3-20位字母或数字，系统会实时校验是否已被占用
- **姓名**（必填）：最多50个字符
- **邮箱**：最多100个字符
- **手机号**（必填）：11位手机号，格式校验
- **角色**（必填）：通过级联选择器选择一个或多个角色。注意：部分角色（医生、患者、商务、药师、线下药房相关角色）在列表中会被禁用，因为这些角色通过各自的入驻流程管理，不应通过后台用户管理分配
- **地址**：最多1000个字符
- **状态**：启用或禁用

填写完成后点击「保存」，系统会再次校验用户名是否唯一。保存成功后，系统会向用户邮箱发送通知邮件；如果邮件发送失败，会单独提示"邮箱发送失败，请检查邮箱是否正确"。

### 第三步：编辑用户

点击操作列的「编辑」按钮，弹出编辑对话框。编辑时用户名和手机号不可修改，其他字段可以修改。编辑状态下角色为必填项，如果清空角色提交会提示"角色为必填项"。

### 第四步：修改用户名

点击「修改用户名」按钮，弹出输入框要求输入新的用户名。新用户名必须为3-20位的字母或数字。提交后立即生效。

### 第五步：重置密码

点击「重置密码」按钮，弹出确认对话框。确认后系统自动重置用户密码，并通过邮件发送新密码。如果邮件发送失败，会提示"邮箱发送失败，请检查邮箱是否正确"。

### 第六步：访问密钥管理

点击「访问密钥」按钮，跳转到访问密钥管理页面（`/mgmt/systemmanage/system/accessKey`），可以为该用户生成用于API调用的AccessKey和Secret。

### 第七步：启用/禁用用户

在用户列表的状态列，直接点击开关按钮可快速切换用户的启用/禁用状态。禁用的用户将无法登录管理后台。

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 用户名已存在 | 提示"用户已存在，请重新输入" |
| 用户名格式不正确（不是3-20位字母数字） | 提示"用户名必须是由长度为3至20位的字母或者数字组成" |
| 手机号格式不正确 | 提示"手机号格式错误" |
| 编辑时清空角色 | 提示"角色为必填项" |
| 新增或编辑时用户名/手机号为空 | 表单校验提示必填 |
| 重置密码后邮件发送失败 | 提示"邮箱发送失败，请检查邮箱是否正确" |

## 关联功能

- **角色管理**：用户分配的角色来自角色管理模块
- **访问密钥管理**：点击访问密钥会跳转到独立的访问密钥页面
- **手机号脱敏**：查看完整手机号需要通过服务端权限验证
