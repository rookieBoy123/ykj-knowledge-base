# 健康服务包配置管理流程

> 文档版本 2026-07-03 | 面向运营人员

## 概述

管理可供用户购买的健康服务包产品。每个服务包是由多个服务项目组合而成的套餐，可以设置价格、启用/禁用状态，以及是否在患者端首页推荐展示。

## 触发条件

- **前置条件**：已配置好服务项目（在"服务项目管理"中）
- **入口路径**：运营管理 → 健康服务包 → 服务包管理

## 步骤详解

### 第一步：查看服务包列表

进入页面后，展示所有已配置的服务包：

| 列 | 说明 |
|:---|:---|
| 服务包 | 服务包名称 |
| 服务项目 | 该服务包包含的服务项目列表 |
| 销售价格（元） | 用户购买此服务包的价格 |
| 状态 | 启用/禁用，通过开关直接切换 |
| 是否推荐 | 是/否，通过开关直接切换 |
| 最后修改时间 | 最近一次修改的时间 |

可按"服务包名称"和"状态（启用/禁用）"进行筛选。

### 第二步：新增服务包

点击"新增"按钮，进入新增页面：

1. 填写服务包名称
2. 选择包含的服务项目（从已有服务项目列表中勾选）
3. 设置销售价格
4. 保存后服务包默认为"启用"状态

### 第三步：编辑服务包

点击操作列的"编辑"按钮，可修改服务包的名称、包含的服务项目、价格等信息。

### 第四步：启用/禁用服务包

在列表行内点击"状态"列的开关：
- **启用**：服务包在患者端可见，用户可以购买
- **禁用**：服务包在患者端隐藏，已购买的用户不受影响
- 切换时系统会弹出确认提示："禁用服务包后将在患者端不显示，您确认要禁用吗？"

### 第五步：设置首页推荐

在列表行内点击"是否推荐"列的开关：
- **是**：服务包在患者端首页展示
- **否**：服务包不在首页展示（仍可在服务包列表中看到）
- 切换时系统会弹出确认提示："设置是否推荐为否后患者端首页将不显示该服务包，确认要修改吗？"

### 第六步：删除服务包

点击操作列的"删除"按钮，系统弹出确认提示"数据删除后不可恢复"，确认后永久删除。
注意：删除前应确认该服务包下没有待处理订单。

## 异常处理

| 情况 | 系统处理方式 |
|:---|:---|
| 未配置任何服务项目 | 新增服务包时服务项目选择列表为空，需要先去"服务项目管理"配置 |
| 禁用已有人购买的服务包 | 禁用操作可执行；已购买用户的订单不受影响，但新用户无法看到和购买 |
| 取消推荐首页展示 | 仅影响首页展示位，用户在服务包列表页仍可看到和购买 |
| 删除后是否影响已有订单 | 系统不会阻止删除，但建议先确认无待处理订单 |

## 关联功能

- **服务项目管理**：配置可被服务包引用的服务项目
- **服务包订单管理**：查看和管理用户的购买订单
- **患者端首页**：被推荐的服务包会展示在患者端首页
