# 自定义底部菜单管理流程

> 文档版本：2026-07-16 | 面向读者：运营、管理员

## 概述

自定义底部菜单（DiyGroup）是小程序底部导航栏的可视化配置功能。管理员可以在后台的装修设计模块中，通过拖拽、图标替换和链接设置，自定义小程序底部的导航菜单项。每个小程序只有一个底部分组配置，修改保存后小程序端实时生效。

## 触发条件

- 需要调整小程序底部导航栏的图标样式（选中/未选中状态）
- 需要修改导航菜单的跳转目标
- 需要增删底部导航菜单项

## 页面入口

管理后台 → 运营管理 → 装修设计 → 顶部 Tab 切换到「自定义底部菜单」

## 步骤详解

### 第一步：进入底部菜单配置页

页面分为左右两栏：
- **左侧**：手机模拟预览区，实时显示当前配置在手机上的效果。预览图背景仅为示意。
- **右侧**：模块配置区，以卡片形式展示每个导航菜单项的图标、标题、链接和显示状态。

首次进入时系统自动获取当前已保存的底部菜单配置。如果此前从未配置过，会进入「新增」模式；已有配置则进入「编辑」模式。

### 第二步：添加菜单项

点击「添加数据」按钮可以新增一个导航菜单项。系统限制最多添加 **5 个**菜单项，超出时提示「底部菜单最多支持五个」。每个菜单项需要设置以下内容：

| 配置项 | 说明 |
|--------|------|
| **选中图标** | 导航菜单被选中时显示的图标（点击上传） |
| **未选中图标** | 导航菜单未被选中时显示的图标（点击上传） |
| **标题** | 菜单文字，最多 4 个字 |
| **链接** | 点击后跳转的小程序页面路径 |
| **状态** | 显示/隐藏开关，关闭后该菜单不在小程序底部显示 |

图标从素材库中选择，点击图标区域弹出素材选择弹窗。链接通过链接选择弹窗配置，支持选择文章、医生、科室等多种页面类型。

### 第三步：调整菜单顺序

每个菜单卡片的左侧有拖拽手柄图标，可以**拖拽卡片**调整菜单的排列顺序。拖拽到目标位置松开即可完成排序。

### 第四步：删除菜单项

除第一个菜单外，其他菜单卡片右上角有删除按钮（圆形叉号图标），点击即可删除该菜单项。

### 第五步：保存配置

所有菜单配置完成后，点击底部的「保存修改」按钮。系统会做以下校验：
- **至少保存两个菜单**：如果启用的菜单少于 2 个，提示「请保存至少两个菜单」，保存不成功。
- **编码唯一性校验**：底部分组编码固定为 `BOTTOM`，如果保存时分组已存在则走更新逻辑，否则走新增逻辑。

保存成功后弹出「操作成功」提示，小程序端底部导航即时更新。

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 添加菜单超过 5 个 | 前端直接拦截，提示「底部菜单最多支持五个」，不发起请求 |
| 启用的菜单少于 2 个 | 保存时前端校验，提示「请保存至少两个菜单」 |
| 底部分组编码重复 | 后端 `DiyGroupServiceImpl.save()` 检查编码，提示「存在相同编码的分组，请重新输入」 |
| 分组不存在（编辑模式） | 后端 `DiyGroupServiceImpl.update()` 提示「分组不存在，请重新输入」 |
| 菜单项配置不完整 | 空标题/空链接可以保存，但小程序端显示可能异常 |

## 关联功能

- **自定义页面管理**：同页面的另一个 Tab，管理小程序的装修页面（首页、医带患首页等）
- **素材管理**：菜单图标从素材库中选择
- **DiyGroup 数据模型**（服务端）：编码固定 `BOTTOM`，名称固定「底部分组」，数据以 JSON 格式存储每个菜单项的图标 URL、跳转路径等信息
