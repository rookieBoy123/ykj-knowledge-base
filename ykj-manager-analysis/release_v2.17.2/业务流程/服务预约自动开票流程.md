# 服务预约自动开票流程

> 文档版本 1.0 | 面向读者：运营人员、财务人员

## 概述

当服务预约订单核销完成后，如果系统开启了自动开票功能且订单金额不为0，系统会自动生成电子发票。开票过程从预约单、订单、患者、会员、HIS系统中采集数据，组装成完整的票据信息提交到电子发票平台。

## 触发条件

- 前置条件：服务预约单必须处于"待核销"状态，且核销码验证通过
- 前置条件：系统开启了自动开票配置（`autoInvoice` 开关）
- 前置条件：订单金额必须大于0（0元订单不触发开票）
- 前置条件：预约单为个人预约类型（团队预约不支持自动开票）
- 入口：管理员在服务预约核销页面输入正确核销码并完成核销

## 步骤详解

### 第一步：核销验证

管理员输入预约单核销码，系统执行以下校验：
- 预约单是否存在
- 预约单状态是否为"待核销"
- 核销码是否正确

核销成功后，预约单状态变为"已核销"。

### 第二步：自动开票判断

核销成功后，系统检查：
1. 是否开启了自动开票功能
2. 订单金额是否大于0
3. 预约类型是否为个人预约（团队预约不支持自动开票）

全部满足后，系统自动发布开票事件。

### 第三步：采集开票数据

系统从多个数据源采集开票所需信息：
- **预约单**：预约编号、患者ID、服务类型
- **会员信息**：手机号、微信昵称、小程序OpenID
- **患者信息**：姓名、身份证号、性别
- **HIS系统**：患者就诊卡号（院内ID）
- **订单信息**：订单编号、支付金额、支付流水号
- **服务项目**：医院名称、项目名称

### 第四步：组装票据信息

系统将采集的数据组装为电子发票请求：
- 票据业务标识：`02`（表示非医保）
- 付款方：个人类型用患者姓名，企业类型用"企业名称 + 患者姓名"
- 证件类型：居民身份证（`1101`）
- 卡类型：诊疗卡/就诊卡（`3101`）
- 医疗类别：非医保

### 第五步：提交开票

组装完成后提交到电子发票平台（`einvoiceService.invoiced`），平台返回发票结果（含电子发票图片URL）。

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 预约单不存在 | 提示"预约单信息异常" |
| 非个人预约类型 | 提示"只允许个人预约单自动开票" |
| 会员信息不存在 | 提示"会员信息异常" |
| 患者信息不存在 | 提示"患者信息异常" |
| 支付流水不存在 | 提示"流水信息异常" |
| 订单信息不存在 | 提示"订单信息异常" |
| HIS患者信息不存在 | 提示"患者his信息异常" |
| 团队预约单 | 不触发自动开票，需要手动操作 |
