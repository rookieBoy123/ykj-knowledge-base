# 商务审核管理流程

> 文档版本：2026-07-04 | 面向读者：运营管理人员、超级管理员

## 概述

管理员在此页面审核商务人员的注册申请。商务人员提交个人资料后进入"待审核"状态，管理员查看商务信息后决定审核通过或不通过。

## 触发条件

- 前置条件：具有商务审核权限的管理员账号登录
- 入口路径：「运营管理 → 商务管理 → 商务审核列表」

## 页面说明

该页面仅展示**待审核**（flowStatus=10）的商务申请记录。已审核的记录不在此列表显示。

### 列表字段

| 字段 | 说明 |
|------|------|
| 商务姓名 | 商务人员姓名 |
| 性别 | 男/女 |
| 手机号码 | 脱敏显示，点击可查看明文（需身份验证） |
| 提交时间 | 资料提交审核的时间 |
| 审核状态 | 显示中文状态名称（待审核/驳回/审核通过） |
| 操作 | "审核"按钮 |

### 手机号脱敏查看

列表中的手机号码默认脱敏显示。点击脱敏图标后，系统通过 `memberPhone` 接口验证操作者身份后返回完整手机号明文。同一时刻仅显示一条明文。

## 审核操作

1. 点击列表某行的"审核"按钮，弹出审核对话框
2. 对话框上半部分展示**商务信息**（只读）：
   - 商务姓名、性别、手机号码、身份证号码
   - 开户行、银行卡号、联系地址
3. 对话框下半部分为**审核操作区域**：
   - 选择审核结果：通过（flowStatus=20）/ 不通过（flowStatus=30）
   - 填写备注（最长255字，可选）
4. 点击"保存"提交审核决定

### 审核状态流转

```
待审核(10) ──→ 审核通过(30) ──→ 完成
    │
    └──────→ 不通过(20) ──→ 驳回
```

**注意**：代码中 flowStatus 映射与常规理解有所不同：
- `10` = 待审核
- `20` = 驳回（不通过）
- `30` = 审核通过

## 异常处理

| 情况 | 系统处理方式 |
|------|------------|
| 未选择审核结果（通过/不通过） | 可提交，但系统可能要求必须选择（取决于后端校验） |
| 备注超255字符 | 输入框限制255字，前端校验拦截 |
| 查看手机号明文 | 需验证操作者身份后返回，同一时刻仅显示一条明文 |
| 列表无数据 | 正常展示空列表，无需特殊处理 |

## 关联功能

- 商务人员通过商务注册入口提交申请
- 审核通过后商务人员可正常登录使用系统
- 商务管理列表可查看所有状态的商务记录
