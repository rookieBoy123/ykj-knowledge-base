# 商务审核管理流程

> 文档版本 2026-07-05 | 面向系统管理员、审核人员

## 概述

商务审核管理是运营管理后台中用于审核商务人员注册申请的模块。当商务人员通过注册二维码提交注册信息后，其资料进入"待审核"状态，审核人员需要在此审核商务的基本信息，决定是否通过。

## 触发条件

- **前置条件**：有商务人员提交了注册申请（审核状态为"待审核"）
- **入口路径**：运营管理 → 商务管理 → 商务审核列表

## 步骤详解

### 第一步：进入审核列表

页面加载后展示所有待审核的商务申请，列表包含以下信息：

- **商务姓名**：申请人的姓名
- **性别**：显示为"男"或"女"
- **手机号码**：默认脱敏显示（中间四位隐藏），点击可查看完整号码（单次有效）
- **提交时间**：提交审核的时间
- **审核状态**：统一显示为"待审核"（该列表固定只展示待审核的记录）

### 第二步：查看商务详细信息

点击某条记录的"审核"按钮，弹出审核详情窗口，展示：

**商务信息区**：
- 商务姓名、性别、手机号码
- 身份证号码
- 开户行、银行卡号
- 联系地址

### 第三步：做出审核决定

审核详情窗口下方有审核操作区：

- **审核结果**：选择"通过"或"不通过"（单选）
- **备注**：可填写审核备注，最多255个字符

选择"通过"后，商务账号状态变为"审核通过"，同时系统会：
1. 自动向商务人员发送微信订阅消息通知，内容为"审核通过，请重新进入小程序"
2. 为该商务生成专属邀请二维码

选择"不通过"后，需填写不通过原因（备注），系统会向商务人员发送通知，内容为"您的资料审核失败：（备注内容）"。

### 第四步：审核后的状态变化

- **审核通过（flowStatus=20）**：商务账号正式激活，可在商务列表中进行启用/禁用操作，可生成邀请医生二维码
- **审核不通过（flowStatus=30）**：商务列表中的操作按钮变为"修改资料"，商务可修改后重新提交审核

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 未选择审核结果 | 直接提交"通过"或"不通过"即可，默认为"通过" |
| 不通过但未填写备注 | 系统仍会发送通知，备注为空则只显示"您的资料审核失败：" |
| 会员信息获取失败 | 取消推送审核通知，不影响审核操作本身 |

## 状态流转

```
商务注册 → 待审核(10) → 审核通过(20) → 正常使用
                      → 审核不通过(30) → 商务可修改资料后重新提交(重置为10)
```

## 关联功能

- **商务列表管理**：审核通过后的商务在商务列表中管理（启用/禁用、邀请医生等）
- **微信订阅消息**：审核结果通过微信订阅消息通知商务人员
- **商务邀请二维码**：审核通过后自动生成专属邀请二维码
