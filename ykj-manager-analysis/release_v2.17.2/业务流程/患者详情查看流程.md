# 患者详情查看流程

> 文档版本 v1.0 | 面向读者：运营管理员、客服人员

## 概述

患者详情页面集中展示患者的所有信息：账号信息（会员数据）、基本信息（姓名/性别/身份证等）、已登记的慢病病种、患者标签，以及该患者的历史病历记录、量表测评记录、随访任务记录和报告记录。这是一个多Tab的综合信息页面。

## 触发条件

- **前置条件**：管理员已登录管理后台，拥有患者管理权限
- **入口路径**：医院管理 → 患者管理 → 患者列表 → 点击患者进入详情

## 步骤详解

### 第一步：查看账号信息

页面顶部展示患者关联的会员账号信息：
- **头像**：会员头像缩略图
- **用户昵称**
- **手机号**：默认脱敏显示（中间隐藏），右侧有眼睛图标可切换显示/隐藏完整号码。每次只能查看一个账号的完整号码，切换时自动隐藏其他已显示的号码

如果一个患者关联了多个会员账号（`members` 数组），会逐一展示每个账号的信息。

### 第二步：查看患者信息

展示患者的基本身份信息：
- **就诊卡号**：暂显示"-"（预留字段）
- **创建时间**：患者档案创建时间
- **姓名、性别、出生日期、年龄**：年龄以"X岁Y月"格式显示
- **身份证号**：如已验证则显示"已校验"标签
- **问诊过**：列出该患者曾经问诊过的所有医生，格式为"医生姓名（医生职称）"，多个医生用顿号分隔

**监护人信息**（仅当 `hadGuardian === 1` 时显示）：
- 监护人姓名
- 关系（父亲/母亲/其他）
- 身份证号（已验证的显示"已校验"标签）

### 第三步：查看慢病病种和标签

- **已登记慢病病种**：以标签形式展示患者登记的慢性病种名称。无慢病登记时显示"无"
- **患者标签**：以标签形式展示系统为该患者打的标签内容。无标签时显示"无"

### 第四步：查看病历记录（Tab 1）

默认激活"病历记录"标签，显示该患者的线上病历列表，每条记录包含：
- 问诊时间、服务类型、医生姓名、科室、医院
- 病历诊断摘要
- 操作按钮：查看病历详情（跳转到recordsdetail页面）、查看问卷

支持分页，点击"查看病历详情"跳转到病历详情页面。

**线上/线下切换**：病历记录Tab内有一个切换开关，可在"线上"病历和"线下"病历之间切换。线下病历通过`offlineMedical`子组件加载。

### 第五步：查看量表记录（Tab 2）

仅当系统开启了量表功能（`scaleEnabled`）时显示此Tab。

展示患者的量表测评记录列表，包括测评量表名称、测评时间、测评结果等。点击某条记录跳转到量表测评详情页（`fullScaleEvaluateDetail`）。

### 第六步：查看随访任务记录（Tab 3）

仅当系统开启了随访功能（`followupEnabled`）时显示此Tab。

展示患者的随访任务执行记录，通过`followupTaskRecord`子组件加载，传入患者ID作为查询参数。

### 第七步：查看报告记录（Tab 4）

仅当系统开启了检查/检验报告功能（`examineReportEnabled`）时显示此Tab。

展示患者的检查/检验报告记录，通过`ReportRecords`子组件加载。

### 页面缓存策略

患者详情页面使用了路由缓存（`keep-alive`）机制：
- 从病历详情、量表详情、线下就诊详情等子页面返回时，页面保持之前的状态（Tab不重置）
- 从其他页面进入时，Tab重置为默认的"病历记录"
- 功能开关状态在进入页面时从系统配置（sessionStorage）中读取并更新

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 患者无关联会员 | 账号信息区域为空 |
| 手机号未填写 | 显示"-" |
| 无监护人 | 监护人信息区域不显示 |
| 无慢病登记 | 显示"无" |
| 无患者标签 | 显示"无" |
| 无量表/随访/报告功能 | 对应Tab不显示 |
| 病历列表为空 | 显示空列表 |
| 身份证未验证 | 不显示"已校验"标签 |

## 关联功能

- 病历详情页（recordsdetail）—— 查看线上病历完整信息
- 量表测评详情页（fullScaleEvaluateDetail）—— 查看量表答题明细
- 线下就诊详情（offlineMedicalDetail）—— 查看线下病历记录
- 患者列表 —— 返回上级页面
- 问卷详情弹窗 —— 查看问诊附带的问卷内容
