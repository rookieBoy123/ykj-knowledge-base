# 科室单位配置管理流程

> 文档版本 1.0 | 面向读者：医院管理人员

## 概述

这是医院**内部科室的组织架构管理**。与诊疗科目（标准科室）不同，这里的科室是按医院内部架构划分的实际科室，支持树形层级（有上级科室/下级科室），每个科室需要关联标准科室和医保科室编码。

## 触发条件

- 进入「医院管理 → 单位配置」页面
- 特殊模式：如果机构类型为"互联网诊疗"，则「新增」按钮替换为「同步HIS科室」按钮

## 科室属性

每个科室包含以下信息：

| 字段 | 说明 |
|------|------|
| 上级科室 | 树形结构中的父级科室（可为空表示一级科室） |
| 科室编码 | 院内自定义编码，最多10位，新增后不可修改 |
| 科室名称 | 最多32字 |
| 科室类型 | 从"kslx"字典选择（如：临床科室、医技科室等） |
| 标准科室 | 从"ksbm"字典选择，关联国家标准科室 |
| 医保科室 | 从"ybks"字典选择，关联医保结算科室编码 |
| 科室简介 | 最多500字，文本域输入 |

## 步骤详解

### 第一步：查看科室树

科室以**树形表格**展示，支持展开/收起层级：
- 一级科室直接显示
- 子科室缩进展示在父科室下方
- 支持的筛选：科室名称、标准科室（树形选择）、科室类型

**特殊说明**：查询使用前端过滤而非后端分页——数据以树形结构一次性加载，搜索条件在前端对树进行过滤。

### 第二步：新增科室

**普通模式**（非互联网诊疗机构）：

点击「新增」按钮，弹出表单：

| 字段 | 特殊说明 |
|------|---------|
| 上级科室 | 树形下拉选择，选择后将不允许再修改（disabled） |
| 科室编码 | 必填，最多10位；一旦新增不可修改 |
| 科室名称 | 必填，最多32字 |
| 科室类型 | 必填，下拉选择 |
| 标准科室 | 必填，可搜索过滤 |
| 医保科室 | 可选，下拉选择（选择后自动填充医保科室名称） |
| 科室简介 | 可选，最多500字 |

提交时自动将医保科室编码转换为医保科室名称。

**互联网诊疗模式**：

不显示「新增」按钮，而是显示「同步HIS科室」按钮。点击后弹出HIS同步窗口，从医院HIS系统同步科室数据。

### 第三步：编辑科室

点击「编辑」，弹出表单回填所有信息：
- 科室编码不可修改
- 上级科室不可修改（一旦设定）
- 科室名称在互联网诊疗模式下为只读（不可修改）
- 如果上级科室为空，回显时重置为null

### 第四步：删除科室

点击「删除」按钮，弹出二次确认后删除。

## 双模式差异

| 对比维度 | 普通模式 | 互联网诊疗模式 |
|---------|---------|---------------|
| 操作按钮 | 「新增」 | 「同步HIS科室」 |
| 科室新增方式 | 手动填写 | 从HIS同步 |
| 科室名称修改 | 可修改 | 只读 |
| 适用机构类型 | 非互联网诊疗机构 | 互联网诊疗机构 |

## 关联功能

- 诊疗科目配置：标准科室的选择来源
- 医生信息管理：医生关联的科室信息
- HIS数据对接：互联网诊疗机构从HIS同步科室
- 排班管理：按科室配置医生排班
