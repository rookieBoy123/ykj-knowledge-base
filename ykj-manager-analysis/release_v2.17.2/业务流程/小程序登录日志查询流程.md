# 小程序登录日志查询流程

> 文档版本：2026-07-04 | 面向读者：运营管理员、安全审计人员

## 概述

查看小程序端用户的登录记录，包括登录账号（脱敏显示）、登录IP、设备信息、登录方式和登录结果，用于安全审计和异常登录排查。

## 触发条件

- **前置条件**：需要拥有登录日志查询权限（`sys:login-log:read`）
- **入口路径**：系统管理 → 系统管理 → 小程序登录日志

## 步骤详解

### 第一步：进入小程序登录日志页面

页面展示所有小程序用户的登录记录列表。与「后台登录日志」（记录管理后台管理员登录）不同，本页面专门展示 App 端用户的登录行为。

列表包含以下信息：

| 列名 | 说明 |
|------|------|
| ID | 日志记录的唯一编号 |
| 用户账号 | 小程序的登录手机号（中间四位脱敏，如 138****1234） |
| 登录IP | 用户登录时的 IP 地址 |
| 操作系统 | 用户手机的操作系统（如 iOS、Android） |
| 浏览器 | 用户使用的浏览器/微信内置浏览器 |
| 登录方式 | 如微信授权登录、手机号登录等 |
| 登录状态 | 以标签形式展示：蓝色「正常」或红色「异常」 |
| 操作结果 | 以状态点展示：绿色=成功，灰色=失败 |
| 登录时间 | 用户登录发生的具体时间 |

### 第二步：筛选查找

页面提供三个筛选条件：

- **用户搜索**：输入用户账号进行查找，最多 30 个字符
- **登录状态**：下拉选择「全部」「正常」「失败」
- **时间选择**：选择日期范围，筛选该时间段内的登录记录

### 第三步：查看登录详情

列表中每条记录展示登录的完整信息。其中「操作系统」和「浏览器」信息由系统根据登录时的 User-Agent 自动解析得出。登录状态显示为彩色标签，蓝色表示正常登录，红色表示异常（登录失败）。

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 登录失败 | 「登录状态」列显示红色「异常」标签，「操作结果」列显示「失败」 |
| 用户账号为空 | 该列显示为空（数据异常情况） |
| 查询无结果 | 列表显示空数据 |

## 关联功能

- **后台登录日志**：查看管理后台管理员的登录记录
- **后台操作日志**：查看管理员的具体操作记录
- **小程序操作日志**：查看小程序用户的操作行为

## 安全设计

- **手机号脱敏**：列表中的用户账号（手机号）中间四位被替换为星号，保护用户隐私
- **权限控制**：需要 `sys:login-log:read` 权限才能访问
