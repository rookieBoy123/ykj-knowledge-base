# 消息通知模板管理流程

> 文档版本 2026-07-03 | 面向运营/系统管理员

## 概述

消息通知模板管理系统所有自动通知的发送渠道和内容。管理员可以针对每种业务场景（如问诊提醒、处方通知、会诊邀请等），分别配置小程序订阅消息、短信和企微机器人三种通知方式的开关和模板内容。

## 触发条件

- **前置条件**：拥有后台系统管理权限
- **入口路径**：系统管理 → 消息通知模板

## 步骤详解

### 第一步：按通知对象筛选查看

进入页面后，通过顶部的标签页切换通知对象，系统支持四类接收方：

| 标签 | 通知对象 |
|:---|:---|
| 通知患者 | 小程序端的患者用户 |
| 通知医生 | 小程序端的医生用户 |
| 通知商务 | 商务管理人员 |
| 通知药师 | 药房药师 |

页面顶部还显示当前小程序所属的经营类目（如"医疗服务-互联网医院"），这是微信小程序订阅消息的分类归属。

### 第二步：查看通知列表

列表中每一行代表一个通知场景，包含：
- **通知名称**：如"问诊提醒""处方审核通过"
- **场景描述**：说明该通知在什么情况下触发
- **三个渠道的启用开关**：小程序订阅、发送短信、企微机器人通知

每个场景的渠道开关旁边，如果该场景**支持**该渠道，会显示开关按钮；如果不支持（如某些场景不支持短信），则该列不显示开关。

### 第三步：切换通知渠道开关

可以直接在列表行内点击开关，快速启用或禁用某个场景的某个渠道：
- **小程序订阅**：控制是否向用户微信发送订阅消息
- **发送短信**：控制是否向用户手机发送短信
- **企微机器人通知**：控制是否向企业微信群发送机器人消息

### 第四步：配置通知模板

点击某一行操作列的"配置"按钮，弹出配置对话框：

1. **切换通知渠道**：对话框顶部有标签页切换三种渠道（小程序订阅/短信/企微机器人），只显示该场景支持的渠道
2. **是否通知**：控制该渠道的总开关
3. **自定义模板**（仅短信渠道）：可选择是否开启自定义短信模板。开启后可填入自定义的模板ID
4. **模板内容**：显示当前通知的模板文案，其中 `#{xxx}` 格式的是动态参数占位符（如 `#{doctorName}` 在发送时会被替换为实际医生姓名）。模板内容在短信自定义模式下可编辑，其他模式下只读
5. **动态参数**：模板中的 `#{xxx}` 参数会自动解析为输入框，可以预填默认值
6. **跳转URL**（小程序订阅消息）：可配置点击消息后的跳转页面路径，不填则默认跳转到小程序首页。URL中的 `#{uniqueId}` 会被替换为业务唯一ID
7. **企微机器人webhook地址**（企微渠道）：配置企业微信机器人的webhook地址，系统会将通知推送到该地址

### 第五步：保存配置

点击"确定"提交所有修改，配置立即生效。

### 第六步：同步小程序订阅消息

列表上方有"同步小程序订阅消息"按钮。点击后系统会从微信后台拉取最新的订阅消息模板列表，同步到本地。同步不会覆盖已有配置，只新增微信后台新增的模板。如果同步失败，会弹出提示框显示具体原因。

## 异常处理

| 情况 | 系统处理方式 |
|:---|:---|
| 某个通知场景不支持某一渠道 | 配置对话框中该渠道标签页不显示，列表中该渠道开关也不显示 |
| 模板内容中包含 `#{xxx}` 参数但未填写 | 参数输入框为空，发送时这些位置会显示为空白 |
| 同步小程序订阅消息失败 | 弹窗提示失败原因，不中断页面操作 |
| 提交配置时模板ID为空 | 前端提示"请输入模板ID"（仅自定义模板模式下校验） |
| 企微webhook地址过长 | 前端限制最多500字符，超出后无法输入 |

## 关联功能

- **系统配置管理**：小程序首页地址等全局配置会影响订阅消息的默认跳转
- **小程序端用户**：用户可在小程序内管理自己的订阅消息授权
- **企业微信机器人**：需先在企微后台创建机器人并获取webhook地址
