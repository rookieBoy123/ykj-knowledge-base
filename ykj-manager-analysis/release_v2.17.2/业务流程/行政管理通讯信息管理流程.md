# 行政管理通讯信息管理流程

> 面向管理员 | 管理后台功能（医院配置子模块）

## 概述

管理员维护医院的行政管理通讯信息，记录各行政部门的负责人姓名和联系电话，支持新增、编辑和查看。

该功能是"医院配置"页面下的一个选项卡（标签名"行政管理通讯信息"），仅在广东地区的医院配置中显示。

## 触发条件

- **前置条件**：拥有管理后台权限（操作权限：`hosp:concats:write`）
- **入口路径**：管理后台 → 医院管理 → 医院配置 → 选择"行政管理通讯信息"标签页

## 步骤详解

### 第一步：查看通讯信息列表

页面以表格形式展示已录入的行政通讯信息：
- **序号**：自动编号
- **部门**：从系统字典中读取的部门名称（如院办、医务科等）
- **负责人姓名**
- **联系电话**
- **创建时间**、**修改时间**
- **操作**：编辑按钮

### 第二步：新增通讯信息

点击"新增"按钮（需 `hosp:concats:write` 权限），弹出表单对话框：

| 字段 | 必填 | 说明 |
|------|------|------|
| 部门 | 是 | 下拉选择，数据来自系统字典"部门(bm)"分类 |
| 负责人姓名 | 是 | 文本输入 |
| 联系电话 | 是 | 手机号格式校验（必须为11位数字，1开头） |

部门选择时有**重名校验**：
- 新增时：如果选择的部门已有记录，提交时提示"当前校验部门记录已存在"
- 编辑时：仅当部门值发生变化时才触发重名校验

### 第三步：编辑通讯信息

点击表格中的"编辑"按钮，弹出与新增相同的表单对话框，预填当前数据。编辑时也触发部门重名校验。

### 第四步：保存

点击"确定"按钮，系统校验表单：
- 部门必选
- 负责人姓名必填
- 手机号必须为11位数字（1开头）

校验通过后调用新增/编辑接口保存，成功后刷新列表并提示"保存成功"。

---

## 异常处理

| 情况 | 系统处理方式 |
|------|------------|
| 同一部门已有记录 | 新增或部门变更时提示"当前校验部门记录已存在" |
| 手机号格式不正确 | 表单校验失败，提示"手机号格式错误" |
| 手机号为空 | 提示"手机号不能为空" |
| 部门未选择 | 提示"请选择部门" |
| 负责人姓名未填写 | 提示"请输入负责人姓名" |

## 地区差异

该功能仅在**广东**地区（`adaptiveArea === 'guangdong'`）的医院配置中显示。其他地区（云南、天津、福建、上海）的医院配置选项卡中不包含此项。

## 关联功能

- **医院配置总览**：该页面是医院配置的选项卡之一
- **系统字典管理**：部门选项来源于数据字典的"部门(bm)"分类
- **医院信息管理**：同属医院配置模块
