# 用户反馈管理流程

> 文档版本 2026-07-03 | 面向运营/客服人员

## 概述

管理患者端用户提交的反馈和投诉。运营人员可以按反馈类型和状态筛选，查看用户反馈的问题和建议，进行回复处理，并追踪处理结果。

## 触发条件

- **前置条件**：用户在患者端提交了反馈
- **入口路径**：运营管理 → 反馈管理

## 步骤详解

### 第一步：筛选查询

支持两个筛选维度：

1. **反馈类型**：从数据字典（分类编码"fklx"）动态加载。反馈类型种类较多，覆盖范围包括：
   - 机构资质问题
   - 执业人员资质问题
   - 挂号问题
   - 就诊问题
   - 处方开具问题
   - 药品供应问题
   - 药品配送问题
   - 缴费问题
   - 药品/器械质量问题
   - 远程医疗问题
   - 其他

2. **处理状态**：全部 / 已处理 / 未处理

### 第二步：查看反馈列表

每条反馈记录展示：

| 列 | 说明 |
|:---|:---|
| 反馈类型 | 用户选择的反馈分类 |
| 问题和建议 | 用户填写的问题描述或建议内容 |
| 联系电话 | 用户手机号，默认脱敏显示，点击可查看完整号码 |
| 反馈时间 | 用户提交反馈的时间 |
| 状态 | 已处理 / 未处理 |
| 发生事件 | 关联的具体业务事件 |
| 处理反馈 | 运营人员回复的内容 |
| 处理人 | 处理该反馈的运营人员姓名 |
| 处理时间 | 处理完成的时间 |

### 第三步：处理反馈

对于"未处理"状态的反馈，点击操作列的"处理"按钮，弹出处理弹窗：
1. 查看反馈详情（用户提交的内容、联系电话等）
2. 填写处理回复
3. 提交后反馈状态变为"已处理"

对于"已处理"状态的反馈，点击操作列的"查看"按钮可查看处理记录（只能查看不能编辑）。

### 第四步：联系电话脱敏查看

联系电话默认只显示部分数字（中间几位隐藏）。点击可查看完整的手机号码，需通过服务端权限校验。同一时间只能查看一个用户的完整号码，切换查看另一个用户时前一个会自动重新脱敏。

## 异常处理

| 情况 | 系统处理方式 |
|:---|:---|
| 已处理的反馈再次打开 | 操作按钮显示"查看"而非"处理"，只能查看不能修改 |
| 联系电话查看权限 | 需要服务端验证，点击后异步获取完整号码 |
| 反馈类型为"其他" | 用户选择了非标准分类的反馈，需要运营人员根据内容判断处理方式 |
| 无反馈记录 | 列表为空，不做特殊提示 |

## 关联功能

- **患者端反馈入口**：用户在小程序端提交反馈
- **数据字典**：反馈类型从数据字典动态加载，可在字典管理中配置
