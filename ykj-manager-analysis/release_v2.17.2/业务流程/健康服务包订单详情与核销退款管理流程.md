# 健康服务包订单详情与核销退款管理流程

> 文档版本 2026-07-06 | 面向：运营人员、客服人员

## 概述

管理员在健康服务包订单详情页可以查看订单完整信息（用户、支付、服务包内容），并进行核销激活、退款申请、退款重试和人工退款等操作。

## 触发条件

- **前置条件**：用户已下单购买健康服务包
- **入口**：健康服务包订单列表 → 点击查看详情

## 步骤详解

### 一、查看订单信息

页面显示订单的四阶段进度条和详细信息：

**进度条**（按订单状态动态展示）：
- **订单生成**：所有订单的起点
- **用户付款**：已付款/退款中/已完成的订单显示，已取消订单不显示此步骤
- **订单核销**：已付款/退款中/已完成状态显示
- **平台退款**：退款中/退款成功状态显示（替换核销步骤）

在已取消状态下，第二步替换为"取消订单"。

**订单信息区**：
- 订单编号、下单用户（脱敏显示）、联系电话（手机号中间四位加密——点击眼睛图标可查看完整号码，需要调用后端接口验证权限后解密）
- 下单时间、支付时间、支付方式（仅已支付订单显示）
- 完成时间（已完成订单）、取消时间和原因（已取消订单）、退款时间和方式（退款成功订单）

**服务包内容区**：
- 服务包名称
- 包含的服务项目列表：项目名称、服务内容、服务条件、服务次数（不限次数或 XX次/年）

### 二、订单核销

核销操作激活服务包内的服务项目权益。执行前弹出确认提示："订单核销后将为用户激活服务包内的服务项目权益，确认要核销吗？提醒：核销后订单将改为已完成状态，不支持退款操作。"

- 核销成功：订单状态变为"已完成"
- 如果URL带有核销码参数（verifyCode），直接确认核销
- 如果没有核销码，弹出"订单核销"对话框手动输入

### 三、申请退款

仅"已付款"状态的订单可操作。执行前弹出二次确认："申请退款后系统将自动退款给用户，退款成功订单自动关闭，该操作不可逆，确认要申请退款吗？"

退款记录中会标注操作管理员姓名。

### 四、退款失败后的处理

当退款状态为"退款失败"（refundStatus=40）时，显示退款失败原因和两个操作按钮：

1. **退款重试**：系统自动重新发起退款
2. **人工已退款**：如果已通过其他渠道（如线下转账）完成退款，管理员可标记退款已处理。操作前弹出确认："将修改订单状态为退款成功，该操作不可逆，确认已人工退款成功吗？"

### 五、联系电话查看

手机号默认中间四位脱敏。点击眼睛图标后，系统调用后端接口进行解密和权限校验，返回完整手机号供查看。再次点击可重新隐藏。

## 订单状态说明

| 状态值 | 状态名称 | 说明 |
|--------|---------|------|
| 10 | 待支付 | 订单已生成，用户尚未付款 |
| 20 | 已付款 | 用户已付款，可进行核销或退款 |
| 60 | 已完成 | 已核销，服务权益已激活，不可退款 |
| 70 | 已取消 | 订单已取消 |
| 89 | 退款中 | 退款申请已提交，审核处理中 |
| 90 | 退款成功 | 退款已完成 |

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 核销操作 | 二次确认弹窗，提醒不可逆 |
| 退款操作 | 二次确认弹窗，提醒不可逆 |
| 退款失败 | 显示失败原因，提供重试和人工退款两个选项 |
| 人工退款 | 二次确认：确认已人工退款成功 |

## 关联功能

- 服务包配置管理
- 退款审核管理
