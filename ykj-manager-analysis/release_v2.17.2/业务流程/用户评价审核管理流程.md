# 用户评价审核管理流程

> 文档版本 1.0 | 面向读者：运营人员

## 概述

患者在完成问诊后可以对医生进行评价。所有评价提交后会进入**审核队列**，运营人员在这里对评价内容进行审核（通过/不通过），以及控制评价在患者端的**显示/隐藏**。

## 触发条件

- 进入「诊疗管理 → 门诊服务订单 → 评价列表」页面

## 审核状态流转

评价有三个审核状态：

| 状态码 | 状态名称 | 含义 |
|--------|---------|------|
| 10 | 待审核 | 患者刚提交评价，等待运营审核 |
| 20 | 审核通过 | 运营审核通过，评价可公开展示 |
| 30 | 审核不通过 | 运营审核不通过（当前版本未明确展示此状态的操作路径） |

## 步骤详解

### 第一步：筛选评价

可通过以下条件筛选：
- **关键字**：在评价内容中搜索（最多255字）
- **评价时间**：日期范围筛选
- **状态**：待审核/审核通过/审核不通过

### 第二步：查看评价列表

列表展示以下信息：

| 列名 | 说明 |
|------|------|
| 会员名称 | 评价者的姓名 |
| 问诊类型 | 图文问诊/视频问诊等 |
| 接诊医生 | 被评价的医生姓名 |
| 分数 | 患者给出的满意度评分 |
| 评价内容 | 文字评价，过长时悬浮显示完整内容 |
| 评价时间 | 提交评价的时间 |
| 显示开关 | 控制评价在患者端是否显示 |
| 操作 | 通过/详情/删除 |

### 第三步：审核评价

**快速通过**：点击操作栏的「通过」按钮（仅"待审核"状态的评价显示此按钮），一键审核通过。

**查看详情并审核**：点击「详情」按钮，弹出评价详情窗口：

| 详情字段 | 说明 |
|---------|------|
| 会员名称 | 评价者姓名 |
| 问诊类型 | 服务类型的枚举名称 |
| 接诊医生 | 医生姓名 |
| 评价日期 | 提交时间 |
| 分数 | 满意度分值 |
| 所选标签 | 以标签形式展示（逗号分隔转为标签数组） |
| 评价内容 | 完整的文字评价 |
| 是否显示 | 显示/隐藏开关 |

在详情弹窗中：
- 如果评价是"待审核"状态，点击「通过」按钮→审核状态变为20，同时自动设为显示（status=1）
- 如果评价已审核，可修改显示/隐藏开关后点击「确定」

### 第四步：控制评价显示

列表中的显示开关：

- **显示**：评价在患者端（医生主页、评价列表等）可见
- **隐藏**：评价不对外展示，但保留在后台

切换开关时调用不同接口：设为显示调用 evaluateShow，设为隐藏调用 evaluateHide。

### 第五步：删除评价

点击「删除」按钮，弹出二次确认后永久删除该评价记录。

## 关键规则

| 规则 | 说明 |
|------|------|
| 待审核才显示"通过"按钮 | 已审核的评价操作栏只有"详情"和"删除" |
| 审核通过自动设为显示 | flowStatus从10变为20时，status自动设为1 |
| 显示/隐藏独立于审核状态 | 已审核通过的评价仍可被隐藏 |
| 问诊类型枚举转换 | serviceType数字转文字（如1=图文问诊，2=视频问诊） |

## 关联功能

- 患者问诊结束后的评价提交
- 医生主页评价展示
- 门诊服务订单列表（可查看问诊对话记录）
