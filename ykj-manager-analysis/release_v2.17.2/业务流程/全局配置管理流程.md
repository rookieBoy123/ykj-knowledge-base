# 全局配置管理流程

> 文档版本 2026-07-05 | 面向系统管理员/运营人员

## 概述

全局配置是互联网医院管理后台的核心设置页面，集中管理所有系统级参数。管理员通过三级标签页（一级分组 → 二级模块 → 具体配置项）导航，找到需要调整的配置项并修改，点击保存后生效。系统自动记录每一次修改历史，支持随时还原到任意历史版本。

## 触发条件

- 需要开启/关闭某个功能（如支付宝支付、医保支付、货到付款）
- 需要修改支付参数（微信商户号、支付宝应用ID等）
- 需要调整业务规则（医生收益比例、处方自动审核开关等）
- 需要更换密钥或回调地址
- 需要查看某次配置变更的修改人和修改内容

## 页面结构

页面采用三级导航：

1. **一级标签页**：功能大类（如功能设置、支付配置、收益配置等）
2. **二级标签页**（带边框卡片样式）：每个大类下的具体模块
3. **表单项**：具体的配置项，根据类型（布尔开关/文本/密码/URL/JSON/时间范围等）渲染不同的输入控件

## 步骤详解

### 第一步：进入全局配置

从左侧菜单「系统管理」→「全局配置」进入。页面加载所有配置项并按分组展示。

### 第二步：找到要修改的配置

通过一级和二级标签页逐层进入目标配置组。配置项可能设置了**显示条件**——只有当某个前置开关开启后，对应的配置区域才会出现。例如：

- 支付宝的应用ID、密钥等字段，只有在「启用」开关打开后才显示
- 货到付款的订金金额，只有在货到付款开关开启后才显示
- 默认药房ID，只有在「默认药房启用」开关开启后才显示

### 第三步：修改配置值

根据配置项类型输入或选择：

| 类型 | 输入方式 |
|------|---------|
| Boolean（开关） | 开关按钮 |
| String（文本） | 文本输入框 |
| Password（密码） | 密码输入框（显示为圆点） |
| Url（链接） | 文本输入框，自动校验URL格式 |
| Decimal（小数） | 数字输入框 |
| Json（JSON对象） | 文本输入框（需输入合法JSON） |
| File（文件） | 文件选择器 |
| Select（下拉选择） | 下拉框 |
| TimeRange/DateRange（时间/日期范围） | 时间/日期范围选择器 |

每次修改都会被系统**暂存**（不立即生效），页面上修改过的配置项会有视觉标记。

### 第四步：保存前自动校验

点击「保存」按钮前，系统会自动校验：

1. **格式校验**：URL类型是否合法、JSON是否可解析、数字是否在范围内
2. **关联必填校验**：
   - 开启「默认药房启用」后，必须同时填写「默认药房ID」
   - 开启「货到付款」后，必须填写订单总价最低金额和订金金额，且订金不能超过总价
3. **无修改校验**：如果没有任何修改，点击保存会提示「没有修改项」

校验不通过时，会在页面顶部显示红色错误提示，阻止保存。

### 第五步：保存配置

1. 点击「保存」按钮，系统将所有已修改的配置项批量提交
2. 保存成功后刷新全局配置信息到浏览器缓存
3. 页面提示「操作成功」，所有暂存修改被清空

### 第六步：查看修改历史

1. 点击「修改历史」按钮，页面右侧滑出历史记录抽屉
2. 以时间线形式展示每次修改：
   - 修改时间
   - 操作人姓名
   - 每一项配置的「修改前值」和「修改后值」
3. 如果一次修改涉及超过3个配置项，默认只显示前3项，点击展开按钮查看全部
4. 支持无限滚动加载更多历史记录

### 第七步：还原历史版本

1. 在历史记录中找到要还原的版本
2. 点击「还原」按钮
3. 弹出确认对话框，列出该版本的所有配置项和值
4. 确认后恢复该版本的所有配置

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 保存时格式错误 | 红色提示具体哪个配置项格式不对 |
| 开启默认药房但未填药房ID | 提示「默认药房启用后，需要配置默认药房ID」 |
| 货到付款订金大于总价 | 提示「订金金额必须少于等于订单总价设置金额」 |
| 没有任何修改就点保存 | 提示「没有修改项」 |
| 配置列表加载失败 | 页面显示loading状态，超时后保持空白 |
| eval运算异常 | 配置项的显示条件判断失败时，默认隐藏该项（安全策略） |

## 特殊处理

### 配置显示条件依赖

每个配置项/分组可以设置 `dependOn` 表达式，根据其他配置的值决定是否显示。例如支付宝的密钥字段设置了 `dependOn: "settings['ykj.payment.alipay.enabled']"`，只有当支付宝启用开关为 true 时才显示。

### 货到付款独立组件

货到付款模块使用独立的自定义组件（`CashDelivery`），与其他标准表单项分开渲染。这是产品设计的特殊需求。

### 浏览器缓存刷新

保存成功后，系统会重新拉取配置信息并更新浏览器缓存（`sessionStorage`），确保其他页面读取到最新配置。
