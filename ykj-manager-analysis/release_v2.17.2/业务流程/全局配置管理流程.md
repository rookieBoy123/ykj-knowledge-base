# 全局配置管理流程

> 文档版本 1.0 | 面向角色：系统管理员

## 概述

全局配置管理是管理后台的系统参数控制中心，管理员可以在这里调整互联网医院的各项运行参数，包括支付方式、问诊设置、医生收益比例、功能开关等。所有配置按类别分组展示，修改后可保存生效，同时支持查看和还原历史修改记录。

## 触发条件

- **前置条件**：登录管理后台，拥有系统配置管理权限
- **入口路径**：管理后台 → 系统管理 → 系统设置 → 配置管理

## 步骤详解

### 第一步：进入配置管理页面

页面顶部有两个操作按钮：「保存」和「修改历史」。

左侧是一级分类标签页（如"支付设置"、"问诊设置"、"医生设置"等），点击一级标签后，下方会展开二级分类标签页（如"微信支付"、"支付宝支付"等），每个二级标签下是对应的具体配置项。

**注意**：某些分类标签页的显示是受其他配置项控制的。例如，只有当系统功能开关中启用了"专家团队"功能时，"团队设置"相关标签才会显示。如果某个前置配置未开启，对应的标签页会自动隐藏。

### 第二步：修改配置项

每个配置项根据类型不同，显示为不同的输入形式：

- **开关类**（Boolean）：显示为开关按钮，开启/关闭
- **文本类**（String）：显示为输入框，可输入文字
- **数字类**（Decimal/Integer）：显示为数字输入框，输入金额或比例
- **链接类**（Url）：显示为链接输入框，系统会自动校验链接格式
- **下拉选择类**（Select）：显示为下拉选择框，从预设选项中选取
- **时间段类**（TimeRange/DateRange）：显示为时间选择器，选择一个时间范围
- **JSON/文件类**：显示为文件上传组件或JSON编辑器

配置项带有校验规则：URL类型会自动校验链接格式是否正确；部分配置项有关联校验（详见"异常处理"部分）。

**修改暂存机制**：所有的修改不会立即生效，而是暂存在页面的"未保存修改列表"中。只有点击「保存」按钮后，修改才会提交到服务器。

### 第三步：保存配置

点击顶部的「保存」按钮，系统会先进行校验：

- 如果没有任何修改项，提示"没有修改项"
- 如果有配置项格式不正确，提示具体错误信息
- 如果有关联配置校验不通过（如启用了货到付款但未填写必填参数），提示具体错误

校验通过后，所有修改项一次性提交保存。保存成功后，系统会刷新配置信息，页面的暂存修改列表被清空。

### 第四步：查看修改历史

点击顶部的「修改历史」按钮，右侧会滑出一个抽屉面板，以时间线形式展示所有历史修改记录。

每条记录包含：
- **修改时间**：什么时候做的修改
- **修改项列表**：每一项显示"修改前"和"修改后"的值（默认显示前3项，超过3项可点击展开）
- **操作人**：谁做的修改
- **还原按钮**：点击可将配置还原到该历史版本

**还原操作**：点击「还原」按钮后，弹出确认对话框，展示该历史版本中所有修改项的内容。确认后，一键还原到该历史版本的配置。

**无限滚动加载**：历史记录支持滚动加载更多，滚动到底部自动加载下一页。

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 没有任何修改就点保存 | 提示"没有修改项" |
| URL格式不正确 | 提示"链接错误" |
| 启用了默认药房开关但未配置默认药房ID | 提示"默认药房启用后，需要配置默认药房ID" |
| 启用了货到付款但未填写订单总价或支付定金 | 提示"货到付款启用配置后，输入框数据为必填" |
| 货到付款订金金额大于订单总价 | 提示"订金金额必须少于等于订单总价设置金额" |
| 配置项格式不符合要求 | 提示具体的格式错误信息 |

## 关联功能

- **系统功能开关**：部分配置的显示依赖于系统功能开关（如专家团队、AI医生），开关关闭对应配置自动隐藏
- **货到付款**：是一组独立的配置区域，有其专用的配置组件
- **配置依赖链**：某个配置的显示可以依赖另一个配置的状态（如"默认药房ID"只在"默认药房开关"开启后才显示）
