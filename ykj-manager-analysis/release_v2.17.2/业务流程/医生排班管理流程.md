# 医生排班管理流程

> 文档版本 2026-07-04 | 面向读者：运营管理员、排班管理人员

## 概述

管理员以周视图查看和编辑医生的每日排班班次（上午/下午/晚上），支持按医生姓名、科室、排班时间筛选，提供排班的发布和下发表机制。系统发布排班后，小程序端患者可看到医生的可预约时间。

## 触发条件

- **前置条件**：具有医生排班管理权限
- **入口路径**：管理后台 → 医院管理 → 医生管理 → 医生排班

## 步骤详解

### 第一步：查看周视图排班

进入页面后默认显示「本周」的排班周视图。页面顶部信息栏显示：

- **左侧提示文字**：「排班发布成功后，系统每天将自动排15天后的班」
- **中间周切换按钮**：「上周」「本周」「下周」三个按钮，当前选中周高亮为蓝色
- **右侧发布操作区**：
  - 绿色图标「✓」表示已发布班次
  - 橙色图标「i」表示待发布班次
  - 「发布」按钮（蓝色）

表格第一列为「医生」，后续7列分别对应周一到周日的每一天，日期格式为「周X MM/DD」（今天显示为「今天 MM/DD」）。

### 第二步：每个排班格子的操作

点击某个医生在某天的排班格子：

- **无排班时**：显示「请选择」字样，点击可新增排班
- **已有排班时**：显示每个班次的信息卡：
  - 格式：`班次名（开始时间-结束时间）已约X/总Y`
  - 绿色图标「✓」表示该班次已发布
  - 橙色图标「i」表示该班次待发布
  - 红色圆形「停」标识表示该班次已停诊
  - 鼠标悬停格子时显示「编辑」和「停诊」按钮

### 第三步：新增/编辑排班班次

点击「请选择」或「编辑」按钮，弹出排班编辑弹窗（800px宽），其中：

- **新增模式**：
  - 如果该日期已有未勾选的排班模板（之前保存但未启用），系统加载已有数据
  - 否则加载排班模板中已勾选的班次作为默认选项
  - 管理员勾选需要的班次并设置每个班次的号源数

- **编辑模式**：
  - 加载当前日期已有的排班详情
  - 显示已有预约数量的统计信息
  - 对于**今天**的班次：
    - 已经开始的时间段不能取消勾选（灰色不可操作）
    - 已经结束的时间段号源数也不可修改
    - 已过期的班次直接剔除

### 第四步：设置号源数

每个班次可设置号源数量（即可预约的患者人数上限）。已有预约数量的班次，号源数不能修改为小于已预约数。

### 第五步：停诊管理

对于已有已发布班次的格子，可以点击「停诊」按钮弹出停诊设置弹窗：
- 选择停诊范围：全部停诊（整天）或指定班次停诊
- 可以填写停诊备注
- 停诊后，该时间段的班次显示红色「停」标识

### 第六步：发布排班

完成排班编辑后，点击页面右上角的「发布」按钮：
- 系统将当前已排好但未发布的班次发布到小程序端
- 发布成功后患者端可见
- 系统会每天自动排15天的班（基于已有排班规律）

> **关闭排班编辑弹窗提醒**：如果编辑了排班但未保存就关闭弹窗，系统会弹出提示「排班信息未保存，是否保存？」，可选择「保存并退出」或「直接退出」。

## 异常处理

| 情况 | 系统处理方式 |
|------|------------|
| 过去日期的排班格 | 显示「无排班」，不可操作（灰色不可点击） |
| 当天已过的时间段 | 不能取消勾选，号源数不可修改 |
| 同一时间段修改中无变化 | 「确定」按钮禁用，提示数据未变更 |
| 停诊后需恢复 | 需重新编辑添加班次 |
| 排班编辑弹窗关闭时未保存 | 二次确认「排班信息未保存，是否保存？」 |

## 关联功能

- **排班模板管理**：排班模板定义了可用的班次类型和时间段，排班时从模板中选择
- **患者预约**：发布后的排班在小程序端可见，患者可预约
- **医生列表管理**：排班关联到具体医生
- **通知推送**：停诊可能触发已预约患者的通知
