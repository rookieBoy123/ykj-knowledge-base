# 商务审核流程

> 文档版本 v1.0 | 面向读者：运营管理员、审核人员

## 概述

新注册的商务人员需要经过管理员审核才能正常使用平台功能。此页面专门处理商务审核工作，审核人员可以查看商务详细信息并做出通过或驳回的决定。

## 触发条件

- **前置条件**：管理员已登录管理后台，拥有商务审核权限（`hosp:businesses:audit`）
- **入口路径**：运营管理 → 商务管理 → 商务审核

## 步骤详解

### 第一步：查看待审核列表

页面默认展示所有待审核的商务人员，列表字段包括：
- 商务姓名
- 性别（男/女）
- 手机号码（脱敏显示，点击可查看完整号码）
- 提交时间（资料提交审核的时间）
- 审核状态（显示当前状态：待审核、驳回、审核通过）
- 操作按钮（全部显示"审核"按钮）

**注意**：审核列表固定筛选 `flowStatus=10`（待审核），不显示已审核或已通过的商务。如果某个待审核商务被驳回后又修改了资料，其状态会重置为待审核，再次出现在此列表中。

### 第二步：进入审核详情

点击"审核"按钮，弹出审核弹窗，分两部分展示：

**上半部分——商务信息**（只读展示）：
- 商务姓名
- 性别
- 手机号码
- 身份证号码
- 开户行
- 银行卡号
- 联系地址

**下半部分——审核决定**：
- **审核结论**：单选"通过"（flowStatus=20）或"不通过"（flowStatus=30）
- **备注**：可选填，最多255字，用于记录审核意见

### 第三步：审核通过

选择"通过"并点击"保存"：
- 商务审核状态变为"审核通过"（20）
- 系统自动为该商务生成邀请医生二维码（如果之前没有二维码）
- 审核通过的商务可正常使用平台功能，出现在商务列表中
- 该条记录从审核列表移除

### 第四步：审核不通过（驳回）

选择"不通过"并点击"保存"：
- 商务审核状态变为"审核不通过"（30）
- 该商务不能正常使用平台功能
- 在商务列表中，该商务的编辑按钮变为"修改资料"
- 修改资料后提交，审核状态重置为"待审核"，重新进入审核流程

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 商务信息中某字段为空 | 显示"暂无" |
| 审核时未选择通过/不通过 | 默认选中"不通过"（flowStatus='30'），需手动切换 |
| 备注超过255字 | 输入框有字数限制，超出无法输入 |
| 审核列表空数据跳页 | 自动回退到上一页 |

## 关联功能

- 商务列表（审核通过/不通过的商务在列表中管理）
- 邀请医生二维码（审核通过时自动生成）
- 会员管理（商务绑定会员账号）
