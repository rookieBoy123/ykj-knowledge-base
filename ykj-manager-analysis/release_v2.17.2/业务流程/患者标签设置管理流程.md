# 患者标签设置管理流程

> 文档版本：2026-07-04 | 面向读者：运营人员、系统管理员

## 概述

管理员在此页面维护患者标签（又称"病种标签"），用于对患者进行分类标记。标签可在医生端随访、患者管理等场景中使用，帮助医生快速识别患者类型。

## 触发条件

- 前置条件：具有患者标签管理权限的管理员账号登录
- 入口路径：「医院管理 → 患者管理 → 患者标签设置」

## 页面操作

### 查询标签
通过标签名称模糊搜索已有的患者标签。输入标签名称后点击查询，列表更新为匹配结果。

### 新增标签
1. 点击"新增标签"按钮（需权限 `hosp:patient-tags:write`）
2. 弹出新增对话框，输入标签名称（最长30字符）
3. 点击"确定"，系统校验标签名称是否已存在
4. 如标签名重复，提示"病种标签已存在，请重新输入"
5. 如不重复，保存成功并刷新列表

### 编辑标签
1. 点击列表某行"编辑"按钮
2. 弹出编辑对话框，显示当前标签名称
3. 修改后点击"确定"，系统校验新名称是否与已有标签冲突
4. 保存成功后刷新列表

### 删除标签（单个）
1. 点击列表某行"删除"按钮
2. 弹出二次确认对话框"此操作将永久删除该项, 是否继续?"
3. 确认后执行删除，成功提示"操作成功"

### 批量删除
1. 勾选列表中需要删除的标签（多选）
2. 点击"批量删除"按钮（需权限 `hosp:patient-tags:write`）
3. 如未勾选任何项，提示"请至少选择一条数据"
4. 弹出二次确认对话框"此操作将永久删除选中项, 是否继续?"
5. 确认后批量删除，成功提示"操作成功"

## 列表字段

| 字段 | 说明 |
|------|------|
| 复选框 | 用于批量选择 |
| 标签编号 | 系统自动生成的编码 |
| 标签名称 | 用户自定义的标签名 |
| 创建时间 | 标签创建的时间 |
| 操作 | 编辑/删除按钮 |

## 异常处理

| 情况 | 系统处理方式 |
|------|------------|
| 标签名称已存在 | 提示"病种标签已存在，请重新输入" |
| 标签名称为空 | 表单校验拦截，提示"请输入标签名" |
| 批量删除未选择任何项 | 提示"请至少选择一条数据" |
| 删除操作 | 弹出二次确认对话框，防止误操作 |
| 标签名超30字符 | 输入框限制30字，前端校验拦截 |

## 关联功能

- 医生端随访模块中使用标签标记患者状态
- 患者管理列表中可按标签筛选患者
- 标签数据通过 tPatientTag 相关接口与服务端交互
