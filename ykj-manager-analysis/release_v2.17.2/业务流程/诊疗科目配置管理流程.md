# 诊疗科目配置管理流程

> 文档版本 2026-07-05 | 面向运营管理员

## 概述

诊疗科目配置用于管理互联网医院的执业科目信息。每个科目包含科目编码、科目名称，以及执业许可的三个关键日期（发证日期、生效日期、到期日期）。这些科目信息来源于国家规定的科室编码字典（字典分类编码 `ksbm`）。

## 触发条件

- **前置条件**：操作员拥有"诊疗科目"写入权限（`hosp:medical-subjects:write`）
- **入口路径**：管理后台 → 医院管理 → 医院配置 → 诊疗科目信息（云南地区专属标签页）

> **注意**：诊疗科目信息标签页仅在云南地区（`yunnan`）的医院配置中显示。其他地区（广东、天津、福建、上海等）的医院配置不包含此标签页。

## 页面布局

页面分为两个区域：

- **搜索区**：支持按科目名称模糊搜索
- **列表区**：展示已有科目记录，包含科目编码、科目名称、执业许可发证日期、执业许可生效日期、执业许可到期日期、创建时间

## 步骤详解

### 第一步：查看已有科目

打开页面后，系统自动加载已有科目列表，分页展示（每页10条）。可按科目名称搜索筛选。

### 第二步：新增科目

点击"新增"按钮，弹出表单对话框，需填写：

| 字段 | 说明 | 是否必填 |
|------|------|---------|
| 科目 | 从科室编码字典中选择，新增时可选，编辑时不可修改 | 是 |
| 执业许可发证日期 | 许可证的颁发日期 | 是（云南地区） |
| 执业许可生效日期 | 许可证的生效日期 | 是（云南地区） |
| 执业许可到期日期 | 许可证的失效日期 | 是（云南地区） |

**注意事项**：
- 科目来源于数据字典中的"科室编码"（`ksbm`）分类，下拉选择时支持搜索过滤
- 三个执业许可日期均为日期选择器，格式为 `YYYY-MM-DD`
- 同一科目编码不能重复添加，系统会提示"请勿重复添加"

### 第三步：编辑科目

点击列表中的"编辑"按钮，弹出编辑表单。编辑时，"科目"字段不可修改（已锁定），只能修改三个执业许可日期。

### 第四步：删除科目

点击列表中的"删除"按钮，弹出二次确认对话框。

**删除限制**：如果该科目下已有关联的科室，系统会拒绝删除并提示"请删除关联科室再删除科目！"。必须先到科室信息配置中删除该科目下的所有科室，才能删除科目本身。

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 新增时未选择科目 | 提示"科目编码不能为空" |
| 新增时未填写科目名称 | 提示"科目名称不能为空" |
| 新增重复科目编码 | 提示"请勿重复添加" |
| 编辑时科目编码已存在 | 提示"请勿重复添加" |
| 云南地区未填写发证日期 | 提示"执业许可发证日期不能为空" |
| 云南地区未填生效日期 | 提示"执业许可生效日期不能为空" |
| 云南地区未填到期日期 | 提示"执业许可到期日期不能为空" |
| 删除有关联科室的科目 | 提示"请删除关联科室再删除科目！" |
| 删除时取消 | 提示"已取消删除" |

## 地域差异

| 地区 | 是否显示诊疗科目标签页 | 执业许可日期是否必填 |
|------|---------------------|-------------------|
| 云南 | 是 | 是（三个日期全部必填） |
| 广东 | 否 | — |
| 天津 | 是（使用天津专用模板） | 否 |
| 福建 | 否 | — |
| 上海 | 否 | — |
| 其他 | 否 | — |

## 关联功能

- **科室单位配置**：科目下可关联多个科室，删除科目前必须先删除关联科室
- **数据字典**：科目编码来源于数据字典中的"科室编码"分类
- **医院配置**：诊疗科目是医院配置的一个子标签页
