# 诊疗科目配置管理流程

> 文档版本 1.0 | 面向读者：医院管理人员

## 概述

医院在互联网平台上开展诊疗服务，需要**登记备案开展的诊疗科目**。这里管理医院各诊疗科目的执业许可信息，包括科目编码、执业许可证的三个关键日期（发证日期、生效日期、到期日期）。

## 触发条件

- 进入「医院管理 → 专科实践」页面
- 需要权限：`hosp:medical-subjects:write`（新增按钮可见）

## 步骤详解

### 第一步：查看科目列表

列表展示所有已登记的诊疗科目：

| 列名 | 说明 |
|------|------|
| 科目编码 | 国家标准科室编码（从"ksbm"字典选择） |
| 科目名称 | 科目编码对应的中文名称（字典自动转换） |
| 执业许可发证日期 | 许可证颁发日期 |
| 执业许可生效日期 | 许可证开始生效日期 |
| 执业许可到期日期 | 许可证失效日期 |
| 创建时间 | 记录创建时间 |

支持按科目名称模糊搜索。

### 第二步：新增科目

点击「新增」按钮（需权限 `hosp:medical-subjects:write`）：

| 字段 | 说明 | 校验 |
|------|------|------|
| 科目 | 从"ksbm"字典下拉选择（编辑时不可修改） | 必填 |
| 执业许可发证日期 | 日期选择器 | 必填 |
| 执业许可生效日期 | 日期选择器 | 必填 |
| 执业许可到期日期 | 日期选择器 | 必填 |

**重要**：科目编码一旦新增后不可修改。编辑时科目下拉框为禁用状态，只能修改三个日期。

### 第三步：编辑科目

点击操作栏的「编辑」，弹出编辑窗口（科目编码不可修改，三个许可日期可修改）。

### 第四步：删除科目

点击「删除」按钮，弹出二次确认："确认删除此科目吗?"

## 关键规则

| 规则 | 说明 |
|------|------|
| 科目编码新增后不可改 | 编辑时下拉框禁用（disabled），确保数据一致性 |
| 四个字段均必填 | 科目编码和三个日期缺一不可 |
| 科目来源为字典 | 从"ksbm"（科室编码）字典中选取，非自定义输入 |
| 新增按钮有权限控制 | 仅拥有 `hosp:medical-subjects:write` 权限的用户可见新增按钮 |

## 关联功能

- 单位科室配置：每个科室与诊疗科目（标准科室）关联
- 医生执业信息：医生注册时关联的执业科目需在此登记
