# 会诊团队费用与收益提成设置流程

> 文档版本 2026-07-05 | 面向管理后台运营人员

## 概述

管理后台提供团队会诊费用的统一配置功能——按医生职称设置会诊价格区间，并配置医院层面的会诊收益提成比例。这些设置决定了医生发起团队会诊时的收费标准。

## 触发条件

- **前置条件**：管理员拥有会诊费用管理权限（`consul:feesettings`）
- **入口路径**：管理后台 → 诊疗管理 → 远程医疗 → 团队会诊

## 步骤详解

### 第一步：查看费用列表与收益提成

进入页面后看到两个区域：

**① 费用列表区**：展示按医生职称分档的会诊价格区间，每行显示：
- 医生职称名称（如主任医师、副主任医师）
- 价格区间范围（最低价 - 最高价）
- 计价单位
- 创建时间
- 操作按钮（编辑）

**② 收益提成区**：
- **是否启用**：开关按钮，控制是否对医院会诊订单收取收益提成
- **医院会诊收益提成**：百分比数字框，1%-100%，表示医院从每笔会诊订单中抽取的收益比例

收益提成的配置项保存在系统配置中（`ykj.consultation.commissionEnabled` 和 `ykj.consultation.commissionProportion`），修改后立即生效。

### 第二步：编辑某个职称的价格区间

点击某个职称行的"编辑"按钮，弹出编辑对话框，包含三个字段：

- **医生职称**：只读展示，不可修改
- **会诊价格区间**：两个数字框，分别输入最低价和最高价。两个价格互相约束——调整最低价时最高价不能低于它，调整最高价时最低价不能高于它
- **备注**：最多255字的文字说明

修改完成后点击确定保存。数据会实时更新到列表中。

### 第三步：修改收益提成

在收益提成区域调整"是否启用"开关或修改提成比例数字，然后点击下方的"保存"按钮。系统会比较当前值与服务端的旧值，只在有变化时才提交更新（避免无效请求）。

保存成功后页面会提示"操作成功"并刷新配置数据。

## 异常处理

| 情况 | 系统处理方式 |
|------|------------|
| 编辑价格区间时最低价和最高价都为空 | 提示"请输入会诊价格区间" |
| 编辑价格区间时只填了一个价格 | 提示"请输入会诊价格区间" |
| 收益提成"是否启用"和"提成比例"都未修改就点保存 | 不触发任何请求，无提示（因无变更） |
| 无相关权限 | 页面可能无法访问或操作按钮不显示 |

## 关联功能

- 会诊费用配置最终影响医生端和患者端的会诊下单价格
- 对应服务端 `MgmtConsultationFeeSettingsController` 和系统配置 `settings` 接口
