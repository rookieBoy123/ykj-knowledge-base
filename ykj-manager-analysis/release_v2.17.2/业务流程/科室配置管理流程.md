# 科室配置管理流程

> 文档版本 1.0 | 面向读者：医院管理员、系统管理员

## 概述

科室配置管理用于维护医院的科室信息，包括科室编码、名称、类型、标准科室映射和医保科室关联。支持新增、编辑、删除科室，互联网医院模式下还可从HIS系统同步科室数据。

## 触发条件

- 管理员需要新增院内科室
- 需要修改科室信息（名称、类型、医保关联等）
- 互联网医院需要从HIS系统同步最新科室数据

## 步骤详解

### 第一步：科室列表查看

进入"医院管理 → 科室配置"页面，系统展示当前所有科室信息：

- **科室编码**：院内唯一标识
- **科室名称**：院内科室名称
- **科室类型**：来自字典"科室类型（kslx）"的分类，如内科、外科等
- **标准科室**：映射到国家标准的科室编码（ksbm），用于数据上报和医保对接
- **医保科室**：关联的医保科室编码（ybks），用于医保结算
- **科室简介**：可选说明文字，最多500字

支持按科室名称、标准科室、科室类型筛选查询。

### 第二步：新增科室

点击"新增"按钮，弹出新增弹窗：

1. **上级科室**：通过树形选择器选择父级科室（可选），支持建立科室层级关系。注意：选择后不可再修改上级
2. **科室编码**：必填，院内科室的唯一编码，最多10位字符。提交后不可修改
3. **科室名称**：必填，最多32个字符。非HIS模式下可自由编辑
4. **科室类型**：必填，从下拉列表选择（来源于字典kslx），如"内科""外科""妇产科"等
5. **标准科室**：必填，从下拉列表选择国家标准科室（来源于字典ksbm），用于医保对接和数据上报
6. **医保科室**：选填，从下拉列表选择（来源于字典ybks），用于医保结算时的科室映射
7. **科室简介**：选填，最多500字

提交时系统校验：科室名称、编码、标准科室、科室类型四项必填，缺少任何一项都会提示错误。

### 第三步：编辑科室

点击科室行的"编辑"按钮，弹出编辑弹窗：

1. 科室编码不可修改（新增时确定后锁定）
2. 上级科室选定后不可修改
3. 可修改：科室名称、科室类型、标准科室、医保科室、科室简介

### 第四步：删除科室

点击"删除"按钮，系统弹出确认框，确认后删除该科室记录。

### 第五步：同步HIS科室（互联网诊疗模式）

在互联网诊疗模式下，页面不显示新增按钮，取而代之的是"同步HIS科室"按钮：

1. 点击后弹出HIS同步弹窗
2. 系统从HIS系统中拉取最新科室数据
3. 同步完成后刷新科室列表

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 科室名称未填写 | 提示"请输入科室名称" |
| 科室编码未填写 | 提示"请输入科室编码" |
| 标准科室未选择 | 提示"请选择关联标准科室" |
| 科室类型未选择 | 提示"请选择科室类型" |
| 科室编码重复（服务端） | 返回错误，提示编码已被使用 |

## 关联功能

- 医生管理 → 医生关联科室
- 标准科室管理 → 科室与国家标准科室映射
- 医保对接 → 医保科室映射影响医保结算
