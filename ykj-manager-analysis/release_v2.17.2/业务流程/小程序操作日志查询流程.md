# 小程序操作日志查询流程

> 文档版本 1.0 | 面向读者：运营人员、客服人员、系统管理员

## 概述

小程序操作日志记录了小程序端用户（患者/医生等）的所有操作行为，包括访问了哪些页面、执行了什么操作、操作是否成功、耗时多久等。可用于排查用户反馈的问题、安全审计和用户行为分析。

## 触发条件

- **入口路径**：系统管理 → 系统管理 → 小程序操作日志
- **前置条件**：需拥有系统管理权限
- **与后台操作日志的区别**：本页面专门记录小程序端用户的操作，后台操作日志记录的是管理后台管理员的操作

## 步骤详解

### 第一步：筛选查询

页面顶部提供多种筛选条件：

| 筛选项 | 说明 |
|--------|------|
| 用户名 | 输入小程序用户的用户名，最多30字 |
| 功能模块 | 输入功能模块名称，支持模糊匹配，最多100字 |
| 操作时间 | 选择日期范围筛选 |
| 操作内容 | 输入操作内容的关键字，最多255字 |

输入筛选条件后点击查询，列表自动刷新。

### 第二步：查看日志列表

列表展示所有匹配的操作记录，每行包含：

| 列名 | 说明 |
|------|------|
| 用户名 | 操作者的用户名，如果用户未登录或用户名为空则显示"匿名用户" |
| 功能模块 | 操作所属的功能模块名称，如"问诊"、"订单"、"会员"等 |
| 操作内容 | 具体的操作描述文字 |
| 操作时间 | 操作发生的精确时间 |
| 操作结果 | 绿色"成功"标签或红色"失败"标签 |
| 操作 | "查看详情"按钮 |

列表支持分页浏览，默认每页10条。

### 第三步：查看操作详情

点击某条记录的"查看详情"按钮，弹窗展示该次操作的完整信息：

- **用户ID**：操作者的系统唯一标识
- **用户名**：操作者的登录账号
- **姓名**：操作者的真实姓名
- **操作类型**：操作的类型名称
- **操作时间**：精确的操作时间
- **操作内容**：详细的操作描述
- **IP**：操作者的网络IP地址
- **客户端**：操作使用的客户端信息（如小程序版本、手机型号等）
- **请求URI**：操作调用的接口路径
- **请求方法**：接口的请求方式
- **请求参数**：操作时传入的参数内容（如有）
- **请求结果**：成功（绿色勾）或失败（红色叉）
- **错误信息**：操作失败时的具体错误原因（仅失败时显示）
- **请求耗时**：接口响应的耗时，以毫秒（ms）为单位

## 异常处理

| 情况 | 系统处理方式 |
|------|------------|
| 用户名为空 | 显示为"匿名用户"，方便识别未登录或数据缺失的情况 |
| 筛选后无结果 | 列表显示为空 |
| 无请求参数 | 详情弹窗中不显示"请求参数"行 |
| 操作成功 | 不显示"错误信息"行，仅显示成功标识和耗时 |
| 操作失败 | 同时显示红色失败标识和具体错误信息 |

## 关联功能

- **后台操作日志**（系统管理 → 系统管理 → 后台操作日志）：记录管理后台管理员的操作，与本页面结构类似但数据来源不同
- **小程序登录日志**（系统管理 → 系统管理 → 小程序登录日志）：专门记录小程序用户的登录行为
- **管理员登录日志**（系统管理 → 系统管理 → 登录日志）：记录管理后台管理员的登录行为
