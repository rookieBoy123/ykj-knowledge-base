# 患者病历查询流程

> 文档版本 2026-07-04 | 面向读者：运营管理员/客服

## 概述

管理员可以通过患者的身份证号查询该患者在平台上的所有线上就诊记录和病历详情，包括问诊信息、诊断结果、处方详情和电子病历 PDF 查看。

## 触发条件

- **前置条件**：知道患者的身份证号，拥有后台管理权限
- **入口路径**：管理后台 → 病历查询（通过 URL 参数 `?idCard=xxx&token=xxx` 进入）

> 注意：此页面不是通过侧边栏菜单直接进入，而是需要携带患者的身份证号和 token 参数访问。

## 页面布局

页面采用左右分栏布局：
- **左侧**（400px）：时间线列表，展示该患者的所有就诊记录
- **右侧**（自适应）：选中记录的病历详情

## 步骤详解

### 第一步：系统定位患者

页面加载时，系统根据 URL 中的身份证号查询患者信息。如果未找到该患者，显示错误提示"没有该患者相关信息"。同时获取系统配置中的机构名称作为页面标题。

### 第二步：浏览就诊时间线

左侧以时间线（Timeline）形式展示该患者的所有就诊记录，每条记录显示：
- **门诊号**：问诊的唯一编号
- **接诊医生**：医生姓名 + 科室 + 职称
- **主诉**：患者描述的主要症状
- **诊断信息**：西医诊断 / 中医辨病 / 中医辨证（有则显示）
- **就诊时间**
- 每条记录标记为"复诊"

如果患者没有线上就诊记录，显示空状态"暂无线上就诊记录"。

### 第三步：查看病历详情

点击左侧任意一条记录，右侧展示该次就诊的完整病历信息：

**病历基本信息**：
- 就诊人姓名、性别、年龄
- 门诊号、初诊标志（固定显示"复诊"）
- 监护人信息（如有监护人）
- 就诊类型（图文/视频/电话问诊）
- 就诊机构、就诊时间、就诊科室、接诊医生
- 主诉、现病史、既往史、过敏史
- 个人史、家族史、月经史、婚育史

**中医四诊**（中医病历时显示）：
- 脉象、舌象

**诊断信息**：
- 西医诊断
- 中医辨病（中医病历时显示）
- 中医辨证（中医病历时显示）

**其他信息**：
- 常见体征、治法（中医）/ 治疗意见（西医）
- 辅助检查
- 病历资料图片（支持点击放大预览）

### 第四步：查看处方信息

在病历详情下方，展示本次就诊关联的处方列表：

**西药处方**（类型=1）：
- 表格展示每个药品：药品名称、规格、剂型、每次用量、用药途径、用药频次、数量

**中药处方**（类型=2）：
- 药品清单
- 剂数、用药方法、用药说明

每条处方还显示：
- 处方编号、开具时间
- 处方医生、审核药师
- **电子处方链接**：点击可打开 PDF 查看器预览电子处方

### 第五步：查看电子病历/电子处方

点击"电子病历"或"电子处方"按钮，系统使用内置的 PDF 查看器打开对应的 PDF 文件。

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 身份证号对应的患者不存在 | 提示"没有该患者相关信息" |
| 患者无线上就诊记录 | 左侧显示"暂无线上就诊记录" |
| 某字段无数据 | 显示"未填写"或"-" |
| 病历资料图片不存在 | 显示"记录遗失/不在身边" |
| 无电子病历 URL | 不显示"电子病历"按钮 |
| 无审核药师 | 显示"-" |
| 缺少 URL 参数 idCard 或 token | 页面不发起请求，空展示 |

## 关联功能

- **患者管理**：患者列表中可以跳转到此页面
- **处方管理**：处方审核流程产生的处方信息在此展示
- **服务端接口**：`/mgmt/patient/findByIdCard`、`/mgmt/patient/medicalHistoryFindPage`、`/mgmt/patient/findMedicalHistoryByUniqueKey`
