# 运费模板管理流程

> 文档版本 1.0 | 面向读者：运营人员

## 概述

运营人员在这里配置药品订单的**全国运费规则**。支持按地区设置不同的运费方案，支持到付、满额包邮、以及医生开方时灵活修改运费等策略。

## 触发条件

- 进入「运营管理 → 运费设置」页面
- 系统首次使用时，运费规则为空，需要配置

## 规则模型

运费规则由多个"条件项"组成，每个条件项包含：

| 字段 | 说明 |
|------|------|
| 支持地区 | 从省/市选择器中勾选的地区（多个省/市） |
| 运费（元） | 该地区的统一运费金额，设为0即包邮 |
| 满额包邮开关 | 开启后，订单总价达到指定金额可免运费 |
| 满额包邮阈值（元） | 触发满额包邮的最低订单金额 |
| 到付开关 | 开启后该地区采用到付模式 |

**最后一个条件项固定为"其他未选择地区"**，用于处理不在任何已配置条件中的收货地址。

## 步骤详解

### 第一步：新增条件

点击「新增条件」按钮，在"其他未选择地区"之前插入一个新的空白条件行。

- 系统最多同时显示3个条件行时不需滚动，超过3个后自动滚动到底部
- 如果上一个条件尚未选择地区，会提示"请先填写完新条件，再继续新增"

### 第二步：选择支持地区

点击某个条件行中的「选择地区」按钮，弹出省/市选择窗口：
- 使用级联选择器，可选择省或市级别
- 支持多选（勾选多个省/市）
- 已选择的地区以标签形式展示，可点击标签上的 X 单独删除

**地区互斥规则**：
- 选择省后不能再选择该省下的市（提示"选择省后不能在选择所在市区"）
- 同一个地区只能出现在一个条件中（提示"一地区只支持设置一个条件"）

### 第三步：设置运费方案

对每个条件行，可以设置三种运费模式组合：

**模式一：普通运费**
- 关闭到付开关，关闭满额包邮开关
- 填写运费金额（≥0）
- 运费为 0 即包邮

**模式二：满额包邮**
- 关闭到付开关，开启满额包邮开关
- 设置运费金额（≤满额阈值时收取）和包邮阈值（≥阈值时免运费）
- 满额包邮金额必须大于0

**模式三：到付**
- 开启到付开关
- 运费金额和满额包邮设置自动置空（应产品要求，保存时清空无关字段）

### 第四步：配置"其他地区"

页面上最底部的区域为"其他未选择地区"，与上面的条件不同：
- **不可删除**：始终存在
- 有两个单选模式：「统一运费」或「统一到付」
- 可以设置满额包邮

### 第五步：可选 — 开启医生自主包邮

在"其他地区"区域末尾有一个「其他」区域：
- 有一个开关「可设为免除运费」
- 开启后，医生在开具处方时可以**手动将运费改为0（包邮）**
- 提示文案："开启后，医生开处方时可修改运费为包邮。"

## 保存校验

点击保存时，系统会执行以下校验：

| 校验项 | 失败提示 |
|--------|---------|
| 条件行未选择地区 | "请添加地区再保存" |
| 非到付模式下运费未填写 | "请设置运费再保存" |
| 运费为负数 | "运费不能小于零" |
| 满额包邮开启但未填写金额 | "请正确填写满额包邮订单金额" |
| 满额包邮金额为负数 | "请正确填写满额包邮订单金额" |
| 无任何修改 | "无修改内容" |

## 导航离开保护

如果修改了运费规则但未保存，点击侧边栏菜单切换时，系统会弹出提示：
"有更新内容，请先保存"

可选择「保存」后跳转，或「不保存」直接跳转。

## 关联功能

- 药品订单下单：根据收货地址匹配运费规则
- 医生开方：若开启了医生自主包邮，医生可修改运费
- 订单物流：到付订单标记运费收取方式
