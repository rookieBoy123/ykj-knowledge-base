# 问诊病历归档管理流程

> 文档版本 2026-07-03 | 面向运营/系统管理员

## 概述
病历归档管理用于追踪问诊病历同步（归档）到医院内部系统的进度。当医生完成问诊后，患者的病历需要被推送到对应的医院系统中。这个页面展示每次归档的结果——成功、失败还是进行中，并支持对失败的记录手动重新归档。

> 注意：虽然目录名是"fileLog"（文件日志），但实际功能是**问诊病历归档管理**，不是文件操作日志。

## 触发条件
- 菜单路径：系统管理 → 病历归档管理
- 使用场景：发现某次问诊的病历没有同步到医院系统，来排查归档状态或手动触发重归档

## 页面功能

### 筛选条件
| 筛选项 | 类型 | 说明 |
|--------|------|------|
| 问诊单号 | 文本输入 | 精确匹配问诊单号，最长30字符 |
| 患者姓名 | 文本输入 | 精确匹配患者姓名，最长5字符 |
| 成功标识 | 下拉选择 | 全部 / 进行中(HaveInHand) / 成功(Success) / 失败(Fail) |

### 列表字段
| 字段 | 说明 |
|------|------|
| 序号 | 顺序编号 |
| 问诊单号 | 对应的问诊单编号 |
| 患者姓名 | 患者姓名 |
| 就诊科室 | 问诊时挂号的科室 |
| 就诊医生 | 接诊的医生 |
| 就诊时间 | 问诊发生的时间 |
| 归档时间 | 最近一次归档操作的时间 |
| 归档方式 | 系统自动归档 or 手动归档 |
| 成功标识 | 进行中/成功/失败 |

### 手动归档
- 操作列显示"归档"按钮，仅当记录状态为"失败"(status=2)时才可用
- 成功或进行中的记录，操作列显示"/"（无操作）
- 点击"归档"，弹出确认框"确认归档?"
- 确认后触发后台归档流程，处理完成后提示"处理完成"并刷新列表

## 状态说明

| 状态值 | 显示名称 | 说明 |
|--------|----------|------|
| HaveInHand | 进行中 | 归档请求已提交但尚未完成 |
| Success | 成功 | 病历已成功同步到医院系统 |
| Fail | 失败 | 归档失败，可手动重试 |

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 归档失败(status=2) | 操作列显示"归档"按钮，管理员可手动点击重新归档 |
| 归档成功(status=Success) | 操作列显示"/"，不需要也不能手动归档 |
| 归档进行中(status=HaveInHand) | 操作列显示"/"，等待系统自动完成 |
| 手动归档确认 | 弹出确认框二次确认后才执行 |

## 关联功能
- **问诊订单管理**：问诊完成后触发病历归档
- **患者病历详情**：归档成功后可在病历详情查看
- 服务端：`MgmtArchiveLogController`（模块：`ykj-service-archiving`）
