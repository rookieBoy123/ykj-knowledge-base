# 数据字典管理流程

> 文档版本 2026-07-04 | 面向读者：系统管理员

## 概述

数据字典是系统的基础配置数据，用于统一管理系统中所有下拉选项、单选枚举值（如性别、剂型、状态等）。管理员通过左侧的树形字典分类导航，可以查看和管理各类字典内容。

## 触发条件

- **前置条件**：拥有系统管理员权限（`sys:dict:read` 可查看，`sys:dict:write` 可编辑）
- **入口路径**：管理后台 → 系统管理 → 数据字典管理

## 页面布局

页面采用左右分栏布局：
- **左侧**（30%宽度）：树形字典分类导航，支持搜索过滤
- **右侧**（70%宽度）：选中分类下的字典内容列表

## 步骤详解

### 第一步：查看字典分类树

进入页面后，左侧自动加载完整的字典分类树。分类之间支持层级嵌套（如"药品字典"下有"药品剂型""用药途径"等子分类）。

每个叶子分类（没有子分类的）后面会显示该分类的"标准编码"。点击任意叶子分类，右侧会展示该分类下的字典内容列表。

**搜索功能**：左侧分类树上方有搜索框，输入分类名称关键字可快速定位到目标分类。

### 第二步：查看字典内容

点击分类后，右侧表格展示该分类下的所有字典条目，包含三列：
- **编码**：每个字典值的唯一标识（如性别字典中，1 代表男，2 代表女）
- **值**：字典值的显示名称
- **操作**：编辑、删除按钮

右侧上方有搜索框，可按"值"的关键字在表格中进行前端过滤。

### 第三步：新增字典分类

点击左侧分类树上方的"新增"按钮（需要有 `sys:dict:write` 权限），弹出新增表单：

| 字段 | 说明 | 必填 |
|------|------|------|
| 名称 | 分类的中文名称（如"性别代码"） | 是 |
| 编码 | 分类的唯一编码（如 `xbdm`） | 是 |
| 父级分类 | 选择所属的上级分类（不选则为一级分类） | 否 |
| 标准编码 | 对接外部系统的标准编码 | 是 |

保存前，系统会校验"编码+标准编码"是否与已有分类重复。如果重复，提示"该编码及标准编码已存在目录中，请重新输入"。

### 第四步：编辑/删除字典分类

每个分类节点右侧有"编辑"和"删除"操作按钮（都需要 `sys:dict:write` 权限）：
- **编辑**：弹窗修改分类的名称、编码（不可修改）、父级分类、标准编码
- **删除**：只能删除叶子分类（没有子分类的），删除非叶子分类会提示"只能删除叶子节点"

### 第五步：新增字典内容

在右侧表格上方点击"新增字典"按钮（需要 `sys:dict:write` 权限），弹出表单：

| 字段 | 说明 | 必填 |
|------|------|------|
| 名称 | 字典值的中文显示名（如"男"） | 是 |
| 编码 | 字典值的编码（如 `1`） | 是 |
| 所属分类 | 自动带入当前选中的分类，不可修改 | — |

保存前，系统校验编码是否在当前分类下已存在。如果重复，提示"编码已存在，请重新输入"。

### 第六步：编辑/删除字典内容

- **编辑**：修改名称和编码
- **删除**：单个删除或勾选多条后"批量删除"，都需要二次确认

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 新增分类时编码+标准编码重复 | 弹出警告"该编码及标准编码已存在目录中，请重新输入" |
| 新增内容时编码重复 | 弹出警告"编码已存在，请重新输入" |
| 删除非叶子分类 | 提示"只能删除叶子节点"，阻止删除 |
| 没有 `sys:dict:write` 权限 | 新增、编辑、删除按钮不显示 |
| 编辑分类时修改名称 | 如果修改后的名称与已有分类重复，阻止保存 |
| 所有输入字段都有长度限制 | 名称、编码、标准编码均为 255 字上限 |

## 关联功能

- **关联跳转**：无直接跳转，但字典编码被系统中所有使用下拉框的页面引用（如商务管理中的性别选择、病历查询中的药品剂型）
- **服务端接口**：对应 `MgmtDictionaryController`（ykj-service-system）
