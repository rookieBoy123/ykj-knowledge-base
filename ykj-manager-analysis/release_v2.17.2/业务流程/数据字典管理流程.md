# 数据字典管理流程

> 文档版本 2026-07-03 | 面向运营/管理员

## 概述
数据字典是系统中所有下拉选项、类型分类的统一管理入口。管理员在这里定义"字典类型"（如"用户反馈类型fklx"、"支付状态"）和每个类型下的"字典项"（如"功能建议"、"Bug反馈"）。前端页面在需要显示下拉框时，统一从这里读取数据，保证选项的一致性。

## 触发条件
- 菜单路径：系统管理 → 数据字典
- 需要权限：查看字典需 `sys:dict:read`，增删改需 `sys:dict:write`
- 使用场景：新增一个业务分类（如新的反馈类型），或修改某个下拉选项的显示名称

## 页面布局
左侧是树形结构，展示所有字典类型分类；右侧是该类下的字典项列表。左侧分类可以嵌套（如"基础数据"下面有"性别"、"证件类型"等子分类）。

## 步骤详解

### 第一步：新建字典类型（分类）
1. 在左侧树形区域点击"新增"按钮
2. 填写弹窗中的四个字段：
   - **名称**：该分类的显示名（如"用户反馈类型"），最长255字
   - **编码**：该分类的系统标识码（如"fklx"），新建后不可修改
   - **父级分类**：如果是一级分类则不选；二级分类需选择上级分类（编辑时一级分类的此字段不显示）
   - **标准编码**：与国家/行业标准对应的编码
3. 点击"保存"，系统会校验编码和标准编码是否与已有分类重复
4. 如果重复，提示"该编码及标准编码已存在目录中，请重新输入"

### 第二步：新建字典项（内容）
1. 先在左侧树形结构中点击目标分类（必须是叶子节点，即没有子分类的末端分类）
2. 点击右侧列表上方的"新增字典"按钮
3. 填写：
   - **名称**：字典项的显示文字（如"功能建议"），最长255字
   - **编码**：字典项的标识码，新建后不可修改
   - **所属分类**：自动带出当前选中的分类，不可编辑
4. 点击"保存"，系统校验编码在当前分类下是否唯一
5. 如果重复，提示"编码已存在，请重新输入"

### 第三步：编辑/删除字典分类
- 左侧树节点右键菜单提供"编辑"和"删除"操作
- 编辑分类时，编码字段为灰色（不可修改），可修改名称、标准编码等
- **删除限制**：只能删除叶子节点（即没有子分类的末端分类），非叶子节点删除会提示"只能删除叶子节点"
- 编辑时名称修改过才会触发后台重复校验

### 第四步：编辑/删除字典项
- 右侧列表中每行提供"编辑"和"删除"按钮
- 支持**批量删除**：勾选多条记录后点击"批量删除"
- 批量删除时逐条调用删除接口，所有删除完成后刷新列表

### 第五步：过滤字典项
- 右侧列表上方有关键字搜索框，支持实时过滤
- 搜索匹配字典项的"名称"字段，不区分大小写
- 清空搜索框后恢复显示全部

## 状态与权限
- 所有新增/编辑/删除操作均需 `sys:dict:write` 权限
- 无写权限时，新增、编辑、删除、批量删除按钮均不显示
- 查看权限 `sys:dict:read` 仅控制菜单入口

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 新建分类时编码已被占用 | 弹提示"该编码及标准编码已存在目录中，请重新输入"，保存不通过 |
| 编辑分类时只改名称未改编码 | 不触发重复校验（编辑模式只在名称变化时校验） |
| 删除非叶子分类 | 提示"只能删除叶子节点"，不允许删除 |
| 删除时未勾选任何记录 | 提示"请至少选择一条数据" |
| 不填名称或编码点保存 | 表单校验失败，红色提示"请输入名称"/"请输入编码" |
| 不填标准编码 | 表单校验失败，红色提示"请输入标准编码" |

## 关联功能
- **反馈管理**：反馈类型下拉框从字典"fklx"读取
- **文章管理**：文章分类从对应字典类型读取
- **各项业务设置**：几乎所有的下拉选项均来自数据字典
- **菜单管理**和**权限管理**：独立管理的权限体系，不走数据字典
