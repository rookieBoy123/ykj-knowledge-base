# 发票查询流程

> 文档版本：2026-07-04 | 面向读者：财务人员、运营管理员

## 概述

在管理后台查询所有已开具的电子发票信息，支持按患者姓名、订单编号、开票日期筛选，并可导出为 Excel 文件。

## 触发条件

- **前置条件**：需要拥有发票查询权限（`order:invoices:read`）
- **入口路径**：财务管理 → 发票管理 → 发票查询

## 步骤详解

### 第一步：进入发票查询页面

页面展示发票列表，每页显示 10 条记录。列表包含以下信息：

| 列名 | 说明 |
|------|------|
| 电子发票号码 | 发票的唯一编号 |
| 交易流水号 | 该笔交易的支付流水号 |
| 订单编号 | 关联的订单编号 |
| 订单类型 | 如处方订单、问诊订单等 |
| 门诊号码 | 患者的门诊号，为空时显示"/" |
| 患者姓名 | 患者的真实姓名，为空时显示"/" |
| 支付渠道 | 如微信支付、支付宝等 |
| 交易金额（元） | 实际支付的金额 |
| 订单支付时间 | 订单支付完成的时间 |
| 开票日期 | 发票开具的日期 |

### 第二步：筛选查找发票

页面顶部有三个筛选条件：

- **患者姓名**：输入患者姓名进行精确查找，最多输入 5 个字
- **订单编号**：输入订单编号查找，最多 30 个字符
- **开票日期**：选择日期范围，筛选该时间段内开具的发票

输入筛选条件后点击查询，列表自动刷新。

### 第三步：打印发票

点击某条发票记录操作列的「打印」按钮，会在新窗口打开发票图片进行打印预览。

**注意**：只有当发票存在电子图片（`pictureNetUrl`）时，「打印」按钮才会显示。如果发票未生成图片，该按钮会被隐藏。

### 第四步：导出发票数据

点击列表上方的「导出」按钮，系统会加载后将当前筛选条件下的所有发票数据导出为 Excel 文件（`发票信息.xls`）。

导出过程中页面顶部会显示「加载中」的转圈提示，导出完成后提示自动消失。

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 发票没有电子图片 | 操作列的「打印」按钮自动隐藏，无法点击 |
| 导出时数据为空 | 导出文件仍会生成，但内容为空表格 |
| 导出请求失败 | 加载提示关闭，导出不成功 |
| 查询结果超过一页 | 列表底部分页器可翻页查看 |

## 关联功能

- **发票管理**：进行发票申请和开票操作
- **订单详情**：查看订单的完整信息
- **交易流水查询**：查看对应的支付流水记录
