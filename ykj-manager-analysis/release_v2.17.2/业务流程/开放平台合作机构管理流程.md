# 开放平台合作机构管理流程

> 文档版本：v1.0 | 最后更新：2026-07-05 | 面向读者：运营人员、产品经理

## 概述

开放平台合作机构是接入互联网医院平台的外部医疗或健康机构。每个机构拥有唯一标识（机构ID）和访问密钥（机构密钥），用于通过开放平台的API接口调用系统能力。管理员可在此管理页面新增、编辑、启用/禁用合作机构。

## 触发条件

- **前置条件**：系统管理员登录管理后台，拥有开放平台管理权限
- **入口路径**：管理后台 → 运营管理 → 开放平台机构管理

## 步骤详解

### 第一步：查看合作机构列表

进入页面后，展示所有已登记的合作机构列表，列表包含以下信息：

| 列名 | 说明 |
|------|------|
| 机构ID | 系统自动生成的唯一标识，合作机构通过此ID识别身份 |
| 机构名称 | 机构的中文名称，用于后台管理识别 |
| 机构密钥 | 用于API调用的签名密钥（右键点击可自动复制） |
| 回调密钥 | 用于回调通知的签名密钥（右键点击可自动复制） |
| 创建时间 | 机构登记时间 |
| 机构状态 | 启用/禁用开关 |

支持按**机构名称**和**机构状态**进行筛选查询。

### 第二步：新增合作机构

点击"新增机构"按钮，弹出编辑弹窗，管理员需填写以下信息：

| 字段 | 说明 | 必填 |
|------|------|-----|
| 机构ID | 系统自动生成，不可修改 | - |
| 机构名称 | 自定义，最长30字 | 是 |
| 机构密钥 | 系统自动生成GUID，可点击重置，用于API调用签名 | - |
| 回调密钥 | 系统自动生成GUID，可点击重置，用于回调通知签名 | - |

> **便捷操作**：新增时机构密钥和回调密钥由系统自动生成（GUID格式），无需手动填写。如果需要重新生成，可点击重置按钮。

### 第三步：编辑机构信息

点击列表中的"编辑"按钮，可修改机构名称。机构ID、密钥信息在编辑模式下不可直接修改（但可通过重置按钮重新生成密钥）。

### 第四步：启用/禁用机构

列表每行末尾有"机构状态"开关，管理员可以：

- **开启**：机构可正常通过开放平台API访问
- **关闭**：机构API密钥失效，接口调用会被拒绝

切换状态时系统会弹出确认框"是否确认启用/禁用？"，确认后立即生效。

### 第五步：复制密钥

管理员可通过**右键点击**列表中的"机构密钥"或"回调密钥"单元格来快速复制密钥内容：

- 右键点击机构密钥 → 提示"机构密钥复制成功"
- 右键点击回调密钥 → 提示"回调密钥复制成功"

密钥被复制到剪贴板后，可发送给合作机构的技术对接人员。

## 异常处理

| 情况 | 处理方式 |
|------|---------|
| 机构名称为空 | 提交时校验提示"请输入机构名称" |
| 禁用机构后API调用 | 接口返回认证失败 |
| 重新生成密钥后 | 旧密钥立即失效，合作方需及时更新 |
| 机构名称重复 | 接口返回错误（后端校验） |

## 关联功能

- **开放平台处方上传**：合作机构通过机构ID+密钥上传处方数据
- **Webhook回调**：合作机构可使用机构ID+回调密钥注册回调地址接收通知
- **全局配置管理**：机构合作功能本身也受全局功能开关"是否启用机构合作功能"控制
