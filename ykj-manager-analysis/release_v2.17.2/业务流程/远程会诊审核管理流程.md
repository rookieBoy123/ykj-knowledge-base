# 远程会诊审核管理流程

> 文档版本：2026-07-05 | 面向读者：远程医疗管理员、运营管理员

## 概述

管理后台审核合作机构提交的专家会诊申请，支持查看会诊详情和在详情中直接审批（通过/拒绝）。采用左右双栏布局，左侧为会诊申请列表，右侧为当前选中会诊的详情。

## 触发条件

- 前置条件：拥有会诊审核权限（`consul:inst-consultations:audit`）
- 入口路径：管理后台 → 诊疗管理 → 远程医疗中心 → 远程会诊审核

## 步骤详解

### 第一步：筛选会诊申请

页面顶部有三个Tab标签页：
- **待审核**（默认选中，蓝色标签）：显示所有待处理的申请
- **已通过**（绿色标签）
- **已拒绝**（红色标签）

左侧列表还支持按"医生姓名"搜索（输入框限制5个字，带字数统计），点击搜索按钮或回车触发。

### 第二步：浏览会诊列表

左侧列表330px宽，每一条会诊记录卡片包含：
- **审核状态标签**（右上角彩色角标）：
  - 待审核：蓝色背景
  - 待签名：绿色背景（审核通过但未签章，显示"待签名"）
  - 已通过：绿色背景
  - 已拒绝：红色背景
- **申请医生**：提交会诊申请的医生姓名
- **开始时间**：预计会诊开始时间
- **预计时长**：会议计划时长（分钟）
- **申请时间**：提交申请的时间

卡片可点击高亮（蓝色边框），表示当前正在查看。

列表支持**无限滚动加载**（滚动到底部自动加载下一页），共支持翻页查看所有记录。

### 第三步：查看会诊详情

点击左侧任意会诊卡片，右侧面板展示完整的会诊详情（包括患者信息、申请医生信息、会诊需求描述等）。右侧面板可独立滚动。

### 第四步：审核会诊

在右侧详情面板底部进行审核操作：

**通过审核**：
- 点击"通过"按钮，系统自动将该会诊状态更新为"已通过"
- 审核完成后自动切换到"已通过"Tab，并刷新列表

**拒绝审核**：
- 点击"拒绝"按钮，系统自动更新状态为"已拒绝"
- 审核完成后自动切换到"已拒绝"Tab，并刷新列表

### 空状态

如果搜索结果为空或列表未加载数据，左侧显示空状态图标提示。

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 会诊信息不存在 | 详情接口返回错误 |
| 没有审核权限 | 接口返回权限不足，无法操作 |
| 搜索医生姓名输入过长 | 输入框限制5个字 |

## 关联功能

- **机构会诊管理**：后台管理员可直接发起和管理会诊（含审批、收费、开始/结束、归档等全生命周期操作）
- **群组会诊聊天**：审核通过后可查看会诊聊天记录
- **远程会诊配置**：配置受邀机构、会诊医生、会诊服务
