# 远程会诊审核管理流程

> 面向管理员 | 管理后台功能（远程医疗模块）

## 概述

管理员对医生提交的远程会诊申请进行审核，可以查看会诊详情并根据实际情况通过或拒绝申请。页面采用左右分栏布局，左侧为申请列表，右侧为会诊详情。

## 触发条件

- **前置条件**：拥有管理后台审核权限
- **入口路径**：管理后台 → 诊疗管理 → 远程医疗中心 → 远程会诊审核

## 步骤详解

### 第一步：筛选审核列表

页面顶部提供三个快速筛选标签页，对应不同审核状态：

| 标签 | 说明 |
|------|------|
| 待审核 | 医生已提交申请，等待管理员审核 |
| 已通过 | 审核已通过（含待签名和已通过子状态） |
| 已拒绝 | 审核被拒绝 |

左侧还提供医生姓名搜索框，支持5字以内的模糊搜索。

### 第二步：浏览申请列表

左侧列表展示会诊申请卡片，每张卡片右上角有状态标签（不同颜色区分）：

- **待审核**：蓝色标签
- **待签名**：绿色标签（审核通过但会诊报告尚未签名）
- **已通过**：绿色标签（审核通过且流程已完成）
- **已拒绝**：红色标签

每张卡片显示：
- 标题"专家会诊"
- 申请医生姓名
- 会诊开始时间
- 预计时长（分钟）
- 申请时间

列表支持**无限滚动加载**：滚动到底部自动加载更多数据。

点击某张卡片，该卡片高亮（蓝色边框），右侧展示对应的会诊详情。

### 第三步：查看会诊详情

右侧展示完整的会诊详情，包括：
- 患者信息、病情描述
- 申请医生信息、申请科室
- 会诊时间安排
- 受邀专家名单
- 相关病历资料

### 第四步：审核操作

在详情页中，管理员可以进行审核操作：

- **通过**：会诊申请审核通过，列表自动切换到"已通过"标签页，并刷新该申请的最新状态
- **拒绝**：会诊申请被拒绝，列表自动切换到"已拒绝"标签页，并刷新该申请的最新状态

审核操作后，选中的会诊编号会保持高亮状态。

---

## 异常处理

| 情况 | 系统处理方式 |
|------|------------|
| 没有待审核的会诊申请 | 左侧列表显示空状态（el-empty） |
| 搜索医生姓名无匹配结果 | 列表为空 |
| 医生姓名超过5字 | 输入框限制最大5字并显示字数统计 |

## 状态说明

会诊申请的审核状态流转：

```
待审核(approveStatus=1) → 已通过(approveStatus=2) 或 已拒绝(approveStatus=3)
已通过 → 待签名(flowStatus=1) → 已完成(flowStatus≠1)
```

- **approveStatus=1**：管理员尚未审核
- **approveStatus=2**：管理员审核通过，根据 flowStatus 决定是否需要签名
- **approveStatus=3**：管理员审核拒绝

## 关联功能

- **远程会诊列表与备案管理**：审核通过后的会诊在"远程会诊列表"中可见
- **群组会诊费用配置**：费用标准由此配置决定
