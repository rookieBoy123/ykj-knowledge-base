# 交易流水查询流程

> 文档版本：v1.0 | 面向读者：财务管理员、运营管理员

## 概述

交易流水查询用于查看平台所有资金交易的明细，包括收入（用户支付）和退款（用户退款）两大类。支持按多种维度筛选，并汇总各支付渠道的金额统计。

## 触发条件

- **前置条件**：已登录管理后台，具备财务管理权限
- **入口路径**：左侧菜单 → 财务管理 → 交易流水

## 步骤详解

### 第一步：选择查询类型

页面顶部有两个页签可供切换：
- **支付明细**（默认）：显示所有用户支付的记录
- **退款明细**：显示所有退款记录

切换页签后，表格列和统计数据会自动调整。

### 第二步：设置筛选条件

支持以下筛选维度：

| 筛选项 | 说明 |
|--------|------|
| 交易日期 | 日期范围选择，默认最近30天 |
| 订单编号 | 精确搜索 |
| 交易流水号 | 精确搜索 |
| 订单类型 | 下拉选择，选项来自字典"订单类型" |
| 支付渠道 | 微信支付 或（如开启了医保支付则额外显示：医保统筹、医保个账、医保支付-微信支付）或 线下支付 |
| 医生 | 远程搜索选择 |

### 第三步：查看汇总金额

在表格上方，系统自动汇总当前筛选条件下的金额统计：
- **合计收入/支出**：所有交易的总金额
- **微信支付**：微信支付渠道的金额
- **医保统筹**（医保支付开启时）：医保统筹支付部分
- **医保个账**（医保支付开启时）：医保个人账户支付部分
- **医保支付-微信支付**（医保支付开启时）：医保订单中的微信自费部分
- **线下支付**：线下支付渠道的金额
- **退款合计**：退款总金额

### 第四步：查看明细列表

**支付明细列表展示：**
| 列名 | 说明 |
|------|------|
| 序号 | - |
| 交易流水号 | - |
| 订单编号 | - |
| 订单类型 | - |
| 医生 | - |
| 支付渠道 | - |
| 交易金额 | - |
| 订单支付时间 | - |

**退款明细列表展示：**
| 列名 | 说明 |
|------|------|
| 序号 | - |
| 交易流水号 | - |
| 订单编号 | - |
| 订单类型 | - |
| 医生 | - |
| 退款渠道 | - |
| 退款金额 | - |
| 退款完成时间 | - |

### 第五步：导出数据

点击"导出"按钮可将当前筛选条件下的流水数据导出为Excel文件（需具备 `order:flows:export` 权限）。导出过程中显示加载提示。

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 医保支付功能未开启 | 支付渠道筛选中不显示医保相关选项，汇总统计中不显示医保类目 |
| 无权限导出 | "导出"按钮不显示 |
| 导出加载中 | 显示持续的加载提示"加载中" |

## 关联功能

- **医保结算管理**：医保支付数据与医保结算上传管理相关联
- **订单管理**：订单编号可关联到具体的问诊/药品/会诊订单详情
- **系统配置**：医保支付渠道的显示由系统功能开关 `medicalInsurancePay` 控制
