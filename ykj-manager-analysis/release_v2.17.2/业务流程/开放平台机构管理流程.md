# 开放平台机构管理流程

> 文档版本：2026-07-05 | 面向读者：运营管理员、技术运维

## 概述

管理接入平台的合作机构信息，包括机构ID分配、密钥管理、启停控制。机构是外部合作方（如合作医院、第三方平台）接入系统的基础凭证。

## 触发条件

- 前置条件：拥有机构管理权限（`open:org:read` / `open:org:write`）
- 入口路径：管理后台 → 运营管理 → 组织架构管理

## 步骤详解

### 第一步：搜索与筛选机构

页面顶部有搜索表单，支持按机构名称模糊搜索和按状态（启用/禁用）筛选。输入条件后点击查询，列表自动刷新。

### 第二步：查看机构列表

列表展示以下信息：
- **机构ID**：系统自动生成的唯一标识
- **机构名称**：可自定义的名称（最长30字）
- **机构密钥（右键自动复制）**：用于API调用的身份凭证
- **回调密钥（右键自动复制）**：用于接收系统回调的验证凭证
- **创建时间**
- **机构状态**：开关按钮，可一键启用/禁用
- **操作**：编辑按钮

特别地，密钥列和回调密钥列支持右键自动复制，无需手动选中文本，右键即复制到剪贴板并提示"复制成功"。

### 第三步：新增机构

点击"新增机构"按钮，弹出表单填写：

1. **机构名称**：必填，最长30字，用于标识是哪个合作方
2. **机构ID**：系统自动生成，不可修改
3. **机构密钥**：系统自动生成（GUID格式，32位随机字符串），不可手动编辑
4. **回调密钥**：系统自动生成，不可手动编辑
5. **审方设置**：选择"自动审方"或"按平台流程审方"
6. **药师选择**：如选择自动审方，必须指定审方药师（否则提交时提示"请选择药师"）
7. **CA签章**：是否有CA签章功能
8. **需要药品下单**：是否需要药品下单功能（开启后必须选择对应药房）
9. **患者实名认证**：是否需要患者身份认证
10. **回调地址**：系统事件回调的HTTP地址
11. **业务模块**：选择需要回调的业务模块（多选）
12. **状态**：启用/禁用

系统会强制校验：如果选择了"自动审方"却没选药师，提交失败并提示"请选择药师"。如果选择了自动审方同时启用CA签章，也会被拦截（CA签章必须使用平台流程审方）。

### 第四步：编辑机构

点击列表行的"编辑"按钮，弹出表单（已预填现有数据）。机构密钥和回调密钥为空时，系统自动生成新的GUID密钥。可修改机构名称、审方设置等信息。确认后提交更新。

### 第五步：启用/禁用机构

列表每行的"机构状态"列是一个直接可操作的开关。点击切换立即生效（无需二次确认弹窗），成功提示"操作成功"。禁用后该机构的所有API调用将失效。

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 自动审方未选择药师 | 提交时提示"请选择药师" |
| 自动审方+启用CA签章 | 后端拦截，提示"启用CA只能选择按平台流程审方！" |
| 需要药品下单但未选择药房 | 后端拦截，提示"需要药品下单的状态下，药房不能为空！" |
| 机构名称重复（同名） | 允许创建（不校验重名），但后端支持同名模糊搜索 |
| 机构不存在 | 后端拦截，提示"机构信息不存在" |

## 关联功能

- **开放平台处方对接**：机构通过机构ID+密钥调用开放平台API
- **回调通知**：机构绑定的callbackSecret用于回调消息的签名验证
- **组织架构与权限**：管理员需要有`open:org:write`权限才能创建/编辑机构
