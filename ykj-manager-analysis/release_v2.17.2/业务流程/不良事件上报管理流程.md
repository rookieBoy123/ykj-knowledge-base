# 不良事件上报管理流程

> 文档版本 v1.0 | 面向读者：运营人员、医疗质量管理员

## 概述

不良事件管理用于记录和追踪医疗服务过程中发生的各类不良事件。运营人员可以新增、编辑和查看不良事件记录，包括事件的基本信息、影响评估和处理措施。

## 触发入口

管理后台 → 运营管理 → 不良事件管理

---

## 步骤详解

### 第一步：事件列表查看与筛选

页面展示所有不良事件记录列表，支持以下筛选条件：

| 筛选项 | 说明 |
|--------|------|
| 事件业务类型 | 下拉选择，来自字典「sjywlx」（事件业务类型） |
| 就诊人姓名 | 模糊输入，最多5个字符 |
| 上报时间 | 日期范围选择器 |

列表展示：事件编号、事件业务类型、就诊人姓名、证件类型（固定"身份证"）、证件号码、手机号、上报科室、上报人、上报时间、最后修改时间。

### 第二步：新增不良事件

点击「新增」弹出表单，需填写以下信息：

**事件基本信息：**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 事件业务类型 | 下拉选择 | 是 | 从字典「sjywlx」加载选项 |
| 事件标题 | 文本输入 | 是 | 最多50字 |
| 事件类型 | 下拉选择 | 是 | 从字典「eventType」加载 |
| 事件等级 | 下拉选择 | 是 | 从字典「eventLevel」加载 |
| 事件影响级别 | 下拉选择 | 是 | 从字典「eventImpactLevel」加载 |
| 事件发生时间 | 日期时间 | 是 | 不能选择未来时间 |
| 上报医生 | 下拉选择 | 是 | 从医生列表加载，支持搜索 |
| 就诊人 | 下拉选择 | 是 | 选择后自动带出手机号 |
| 就诊人手机号 | 文本输入 | 是 | 11位手机号格式校验 |
| 上报时间 | 日期时间 | 是 | 不能选择未来时间 |
| 事件发生地址 | 文本输入 | 是 | 最多100字 |

**事件详细描述：**

| 字段 | 类型 | 必填 | 最大字数 | 说明 |
|------|------|------|---------|------|
| 事件描述 | 多行文本 | 是 | 3000字 | 事件的详细经过 |
| 事件发生的主要原因 | 多行文本 | 是 | 3000字 | 原因分析 |
| 采取的措施 | 多行文本 | 是 | 1000字 | 已采取的处理措施 |
| 造成的不良影响 | 多行文本 | 是 | 1000字 | 事件造成的后果 |
| 解决措施 | 多行文本 | 是 | 3000字 | 后续解决方案 |

**智能联动：**
- 选择就诊人后，系统自动获取该就诊人的会员信息，并回填手机号（脱敏手机号会通过接口获取明文）

### 第三步：编辑与查看

- **编辑**：点击某条记录的「编辑」按钮，弹出表单回填已有数据，修改后保存
- **查看详情**：点击「详情」按钮，以只读模式查看事件的完整信息

---

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 手机号格式不正确 | 前端校验提示"就诊人手机号格式错误" |
| 手机号不是11位 | 前端 length 限制为11位 |
| 事件发生时间选择未来时间 | 日期选择器禁用未来日期 |
| 必填字段未填写 | 表单校验提示对应的错误信息 |
| 就诊人无会员信息 | 手机号回填空字符串 |

## 关联功能

- **数据字典**：事件业务类型、事件类型、事件等级、事件影响级别均通过字典配置
- **医生管理**：上报医生从医生列表中选取
- **患者管理**：就诊人从患者列表中选取，选择后自动关联会员手机号
