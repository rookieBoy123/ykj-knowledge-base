# 就诊人解绑验证码流程

> 文档版本 1.0 | 面向角色：患者

## 概述

患者需要更换绑定手机号或解绑已认证的就诊人时，通过短信验证码验证身份后完成解绑操作。如果当前手机号已停用无法接收短信，可通过微信客服联系处理。

## 触发条件

- **前置条件**：
  - 已有绑定手机号的已实名就诊人
  - 需要解绑后绑定新手机号
- **入口路径**：就诊人详情 → 解绑操作 → 进入验证码页面

## 步骤详解

### 第一步：进入解绑验证页面

用户看到什么：

- 页面标题显示"就诊人解绑"
- 顶部蓝色提示区域，带图标和说明文字："请获取当前绑定手机号的验证码进行解绑，解绑成功后将绑定新的手机号"
- 两个输入框：
  - 手机号（灰色禁用，自动填入当前绑定手机号，不可修改）
  - 验证码（可输入，最多10位数字）
- "获取验证码"按钮（蓝色边框）
- "解绑"按钮（灰色/蓝色，验证码未输入前为灰色不可点击）
- 底部提示："若该手机号已停用/无法接收短信，请 联系客服处理"（蓝色可点击文字）

用户能做什么：

- 点击"获取验证码"发送短信
- 输入收到的6位短信验证码
- 点击"解绑"完成操作
- 点击底部"联系客服处理"打开微信客服对话

---

### 第二步：获取短信验证码

用户点击"获取验证码"后：

- 系统向后端发送请求，携带就诊人ID和/或身份证号+姓名
- 如果缺少必要参数（patientId和idCard都为空），提示"缺少必要参数"
- 发送成功后：
  - 按钮变为倒计时状态："重新发送60s" → "59s" → ... → "1s"
  - 60秒倒计时结束后恢复为"获取验证码"
  - 倒计时期间按钮不可点击（防止重复发送）
- 验证码发送到当前绑定手机号对应的手机上

---

### 第三步：输入验证码并解绑

用户输入验证码后点击"解绑"：

- 系统校验验证码（patientId + captcha 或 idCard + realName + captcha）
- 如果缺少必要参数，提示"缺少必要参数"
- 解绑过程中显示"加载中"
- 解绑成功后：
  - 提示"解绑成功"
  - 1.5秒后自动返回上一页（就诊人详情页）
- 解绑失败显示对应错误提示

---

## 异常处理

| 情况 | 系统处理方式 |
|------|------------|
| 缺少必要参数 | 提示"缺少必要参数" |
| 验证码输入错误 | 后端校验失败，提示具体错误 |
| 验证码过期 | 提示验证码已过期，需重新获取 |
| 手机号已停用收不到短信 | 底部引导联系微信客服处理 |
| 短信发送过于频繁 | 60秒倒计时锁定，不可重复点击 |
| 解绑过程中网络异常 | 显示加载中后超时 |

---

## 关联功能

- **就诊人信息管理**：解绑后返回就诊人详情页，此时可绑定新手机号
- **微信客服**：如果无法接收短信，通过客服渠道人工处理解绑
