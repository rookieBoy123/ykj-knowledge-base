# 收银台支付流程

> 文档版本 2026-07-02 | 面向产品运营、客服

## 概述

收银台是患者支付订单的集中页面，展示应付金额、可选支付方式、支付倒计时，并支持微信支付和医保统筹支付两种渠道。支付成功后根据不同订单类型跳转到对应的后续页面。

## 触发条件

- 患者点击任意需要支付的订单（药品订单、问诊订单、会诊订单、服务预约订单）
- 前置条件：已生成订单编号
- 入口路径：各处「去支付」按钮跳转至收银台

## 步骤详解

### 第一步：进入收银台，查看支付信息

页面顶部展示：
- **应付金额**：大字显示，格式为 ¥XX.XX
- **支付剩余时间**：倒计时显示（时分秒），到期后自动触发支付超时处理
- **支付方式列表**：列出所有可用支付渠道

系统接收到订单编号后，自动请求后端生成预支付单，获取：
- 可选的支付渠道列表
- 应付金额
- 订单过期时间

### 第二步：选择支付方式

页面展示两种支付方式：
- **微信支付**（channel=1）：默认选中，图标为微信图标，点击切换选中状态
- **医保支付**（channel=3）：仅医保订单展示，需要验证医生是否为医保医师、患者是否已定点

支付方式选择互斥：只能选一种，选中后自动取消前一个选择。

不可用的支付方式图标灰色，显示提示："暂不支持XX支付"并附带原因说明。

### 第三步：医保定点验证（仅医保支付）

当用户选择「医保支付」时，系统自动执行以下检查：
1. 检查是否有患者ID，没有则提示"缺少患者id，请联系客服"并自动切回微信支付
2. 判断是否是慢病订单，慢病流程跳过定点检查
3. 检查患者是否已在该医院定点：
   - 未定点：弹窗提示"该就诊人暂未定点[医院名称]，定点后才可使用医保支付"，提供「暂不定点」和「确认定点」两个选项
   - 已定点：继续支付流程

如果选择「确认定点」，系统调起微信医保小程序进行定点操作。

### 第四步：确认支付

点击「确认支付」按钮后：
- 如果未选择支付方式，提示"请先选择支付方式"
- 微信支付：调起微信支付密码输入，支付完成后返回收银台
- 医保支付：先请求订阅消息授权，然后调起微信医保小程序进行医保结算（含个人账户和统筹基金支付）

支付前做防重复点击保护（noMultipleClicks）。

### 第五步：支付结果处理

**支付成功**：
- 弹出支付成功弹窗，显示勾号图标和"支付成功"文字
- 提示"我们将尽快安排发货，请耐心等待"
- 提供两个按钮：
  - 「查看订单」→ 跳转订单详情页
  - 「返回咨询」→ 跳转聊天室

**支付状态不确定**（如小程序切换前后台后）：
- 弹出"支付结果信息提示"弹窗
- 显示"未查询到支付状态，请根据您的情况点击下面的按钮"
- 提供两个按钮：
  - 「支付未成功」→ 关闭弹窗，可重新支付
  - 「支付已成功」→ 手动查询支付状态

### 第六步：返回处理（点击返回按钮）

返回行为根据订单类型不同：
- **药品订单**（type=drug）：跳回药品订单确认页面，携带门诊编号和预结算标记
- **预约挂号订单**：跳转到订单详情页
- **其他订单**：直接返回上一页

## 异常处理

| 情况 | 系统处理方式 |
|-----|------------|
| 支付方式不可用 | 图标灰色，显示"暂不支持XX支付"和原因 |
| 未选择支付方式 | 提示"请先选择支付方式" |
| 医保支付缺少患者ID | 提示"缺少患者id，请联系客服"，自动切回微信支付 |
| 医保未定点 | 弹窗提供「暂不定点」和「确认定点」选择 |
| 医生非医保医师 | 弹窗提示"当前医生非医保医师，不支持医保支付，只能自费" |
| 支付超时 | 倒计时归零，触发timeup事件 |
| 支付状态查询失败 | 弹窗让用户自行确认是否支付成功 |

## 关联功能

- 药品订单确认页 → 收银台 → 支付成功 → 订单详情 / 聊天室
- 药品订单确认页 → 收银台 → 返回 → 药品订单确认页（携带预结算标记）
