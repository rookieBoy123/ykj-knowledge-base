# 收银台支付流程

> 文档版本：release/v2.17.2 | 面向读者：客服、运营

## 概述

收银台是平台上所有需要付款的环节的统一支付页面。无论是问诊服务费、药品费、会诊费、预约服务费还是健康服务包，最终都汇聚到收银台完成支付。患者在这里可以看到应付金额、选择支付方式（微信支付或医保支付），在倒计时结束前完成付款。

## 触发条件

用户在以下场景会进入收银台：
- 选好医生和服务后，在「服务确认」页点击「立即支付」
- 在预约服务确认页点击「立即支付」
- 在药品订单页点击「去支付」
- 微信好友打开代支付链接后点击支付
- 会诊申请审核通过后支付会诊费

**前置条件**：系统已生成预支付订单（订单编号、金额、可用支付方式均由后端预结算接口返回）。

## 步骤详解

### 第一步：查看支付信息

进入收银台，页面顶部显示：
- **应付金额**：大号数字显示（如 ¥128.00），取自后端预结算接口返回的金额
- **支付剩余时间**：倒计时器（时/分/秒），提醒用户在此时间内完成支付

倒计时由后端返回的订单有效期决定。订单金额固定显示两位小数（`payAmount.toFixed(2)`）。

### 第二步：选择支付方式

页面列出所有可用的支付方式，通常包括：
- **微信支付**（channel=1）：通过微信支付完成付款
- **医保支付**（channel=3）：通过医保电子凭证完成移动支付

每种支付方式显示对应的图标和名称。如果某种支付方式不可用，会显示灰色禁用状态，并附带不可用原因说明（如"暂不支持医保支付"）。

**微信扫码支付已移除**：频道号为4的二维码支付在小程序收银台不再开放。

用户点击某支付方式即可选中，已选中的显示蓝色实心圆圈，未选中的为空心圆。每次只能选一个。

### 第三步：确认支付

选中支付方式后，点击底部蓝色「确认支付」按钮：

**A. 微信支付**：
1. 系统调用微信支付接口（参数包含订单编号、微信用户标识openId）
2. 弹出微信支付密码输入界面
3. 支付成功后自动跳转到对应结果页

**B. 医保支付**：
1. 选中医保支付时，系统首先检查就诊人是否已医保定点：
   - 已定点 → 直接跳转医保小程序授权
   - 未定点 → 弹出提示框「该就诊人暂未定点[XX医院]，定点后才可使用医保支付」，提供两个选项
     - 「暂不定点」→ 自动切换回微信支付
     - 「确认定点」→ 调用定点接口，成功后继续医保支付
2. 定点检查通过后，跳转到医保小程序的授权页面
3. 授权成功后返回小程序，跳转到医保预结算页（hisPrePay）

**医保定点弹窗说明**：
- 弹窗文本中的医院名称从系统全局配置中动态获取
- 如果是慢病入口进入的订单，跳过定点检查（慢病已有备案）

### 第四步：支付结果处理

支付完成后，根据订单类型跳转到不同结果页：

| 订单类型 | 支付成功后跳转 |
|---------|-------------|
| 药品订单(type=drug) | 药品订单详情页 |
| 预约服务(type=appoint) | 预约详情页 |
| 问诊服务(type=addworkflow) | 进入问诊室（IM聊天/TUI-Chat） |
| 会诊(type=consultation) | 会诊成功状态页 |
| 健康服务包(type=healthServicePackage) | 服务包订单详情页 |

问诊类型(type=addworkflow)还有两个特殊分支：
- 如果服务类型是AI医生问诊(serviceType=9) → 进入AI聊天室
- 如果医生开启了AI预问诊 → 先进入AI预问诊页面

如果系统未能确认支付状态（查询返回false），会弹出「支付结果信息提示」弹窗，让用户自行选择：
- 「支付已成功」→ 手动触发结果跳转
- 「支付未成功」→ 关闭弹窗，可重新支付

### 第五步：倒计时超时

如果在倒计时结束前未完成支付，系统弹出提示「订单支付超时，请重新发起」，点击「知道了」自动返回上一页。

## 异常处理

| 情况 | 系统处理方式 |
|------|------------|
| 未选择支付方式就点确认 | 提示「请先选择支付方式」 |
| 医保支付时缺少患者ID | 提示「缺少患者id，请联系客服」，自动切换为微信支付 |
| 医保支付时医生非医保医师 | 弹出提示「当前医生非医保医师，不支持医保支付，只能自费」 |
| 医保授权时用户取消 | 弹窗提示「您已经取消医保电子凭证身份授权，医保移动支付已中止」 |
| 医保配置参数缺失 | 弹窗提示「医保参数有误，是否重新获取？」 |
| 倒计时超时 | 订单失效，弹窗提示返回上一页 |
| 支付后查询不到支付状态 | 弹出「支付结果信息提示」让用户手动确认 |
| 快速开方医生代付 | 支付后跳转药品订单页时额外传入`doctorDfSuccess=1`标识 |
| 支付后进问诊室超时 | 弹窗「服务器开小差了，点击返回消息列表」 |

## 关联功能

- ← **服务确认页**（createOrder）：患者确认服务信息后进入
- ← **药品订单页**（drugOrder）：患者提交药品订单后进入
- ← **预约确认页**（reservation/confirmation）：填写完预约信息后进入
- ← **代支付详情页**（consultDfDetail/ inquiryDfDetail）：代支付朋友点击链接后进入
- → **问诊室**（timIM/chat）：问诊类型支付成功后进入
- → **AI预问诊页**（aiPreConsult）：有AI预问诊的订单支付后进入
- → **医保预结算页**（hisPrePay）：医保支付授权成功后进入
- → **服务包订单详情**（healthServicePackage/order/details）：服务包类型支付后进入
- → **会诊状态页**（groupConsultation/statusDetail）：会诊类型支付后进入

## 特殊说明

**延后付费模式**：当后台开启「诊费与处方费同时支付」（`consultationFeePayLater`）配置时，问诊确认页（createOrder）的合计费用显示为¥0，按钮文字变为「确认问诊」而非「立即支付」。此时不跳转收银台，而是直接创建问诊单进入问诊室。医生开方后统一收取诊费+药费。（AI图文问诊除外——即使开启延后付费，AI问诊仍需先支付）

**代支付**：如果是从微信好友代支付链接进入，支付成功后统一返回上一页（`navigateBack`）。
