# 药品需求开方流程

> 文档版本 v1.0 | 面向读者：产品运营、医生用户

## 概述

患者在问诊过程中向医生提出用药需求，医生根据患者的需求和诊断信息，补充药品的用法用量后开具处方。如果已有处方则显示"已开处方"状态，不可重复开方。

## 触发条件

- 前置条件：医生正在与患者进行问诊，患者已提交用药需求
- 入口路径：医生在问诊对话中点击患者的用药需求卡片进入

## 步骤详解

### 第一步：查看用药人信息

页面展示患者的姓名、性别、年龄等基本信息，帮助医生确认患者身份。

### 第二步：确认诊断信息

医生需要选择或修改临床诊断（西医诊断/中医辨病辨证），诊断信息来自问诊流程中已有的诊断记录。

### 第三步：查看和补充用药需求

页面展示患者提交的药品清单，包含药品名称、规格等信息。医生需要**补充每项药品的用法用量**：
- 用药方式（口服、外用等）
- 每次用量
- 使用频率（每日几次）
- 使用天数

如果某项药品的用法用量全部未填写，系统会标注"待完善"。

### 第四步：填写医嘱（可选）

医生可以填写用药医嘱说明，作为患者用药指导的补充信息。

### 第五步：确认开方

点击底部"确认开方"按钮。系统会进行以下校验：
- 是否已选择诊断（未选择则提示"请选择诊断"）
- 药品用法用量是否全部填写完整（未完整则提示"请将已添加的药品用法用量填写完整"）

校验通过后，如果医院开启了CA电子签章配置，系统会先验证医生的CA签章状态，再引导完成电子签名，最终开出处方。

### 第六步：开方成功

开方成功后跳转到成功页面，展示处方编号供患者后续查看和购药。

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 处方已开具（obj.issued=true） | 页面顶部显示橙色"已开处方"标签，底部"确认开方"按钮消失，医生不可再操作 |
| 未选择诊断 | 弹出提示"请选择诊断"，阻止提交 |
| 药品用法用量填写不完整 | 弹出提示"请将已添加的药品用法用量填写完整"，阻止提交 |
| CA签章未认证 | 根据认证状态分别处理：未认证→引导同步资料并认证；认证中→提示等待认证完成；认证失败→提示重新认证 |
| 医生CA签章账号未同步 | 弹出提示引导医生先完成CA签章账号同步 |
| 严格模式需要人脸识别 | 先跳转人脸识别页面，通过后再进行CA签名 |

## 关联功能

- 问诊对话 → 患者提交用药需求后跳转至此页面
- 处方详情 → 处方开具成功后跳转查看处方详情
- CA签章管理 → 医生首次需要在设置中完成CA签章认证
