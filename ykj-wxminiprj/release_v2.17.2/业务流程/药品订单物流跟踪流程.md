# 药品订单物流跟踪流程

> 文档版本 2026-07-03 | 面向患者 / 医生

## 概述

患者和医生都可以查看药品订单的物流配送进度。页面以时间轴形式展示从下单到签收的完整物流轨迹，包括快递公司、快递单号和每个物流节点的状态。

## 触发条件

- 从处方详情页或订单详情页点击查看物流
- 页面路径携带处方编号（outpatientNo）和快递单号（deliverySn）
- 患者角色和医生角色均可查看

## 步骤详解

### 第一步：进入物流页面

页面加载后自动根据快递单号查询物流信息，同时根据处方编号获取订单信息（包括收货地址和快递公司）。

### 第二步：查看物流概览

页面顶部展示：
- **订单状态**：当前物流状态（如"派件中"、"已签收"等）
- **快递公司**：显示快递公司名称和快递单号

### 第三步：查看物流轨迹

页面主体以时间轴（竖线+图标）形式展示物流节点，每个节点包含：
- **状态图标**：用不同图标区分物流状态（揽收、在途、派件、签收等）
- **状态名称**：如"揽收"、"在途"、"派件"、"签收"
- **详情描述**：物流节点的具体描述（如"您的快件已到达xxx营业点"）
- **时间**：该节点发生的具体时间

时间轴最上方显示收货地址，最下方显示"已支付"和"备货中"两个固定节点。

### 第四步：空状态处理

如果快递单暂无物流信息（刚下单还未揽收），页面显示"暂无物流信息"和快递单号。

## 物流状态说明

| 状态 | 含义 |
|------|------|
| 揽收 | 快递公司已取件 |
| 在途 | 快件正在运输途中 |
| 派件 | 快递员正在派送 |
| 签收 | 收件人已签收 |
| 退回/退签/转投/疑难/清关/拒签 | 特殊情况处理 |

## 异常处理

| 情况 | 系统处理方式 |
|------|------------|
| 快递公司接口无数据 | 仅显示本地订单信息（收货地址+备货节点），不展示快递轨迹 |
| 用户无就诊人角色 | 按医生角色查询物流信息 |
| 缺少快递单号参数 | 页面无法正常加载 |

## 关联功能

- **处方详情**：页面入口之一
- **药品订单详情**：查看订单后跳转物流
- **我的处方列表**（患者端）/ **处方记录**（医生端）：可跳转物流查看
