# 健康档案综合查阅流程（医生端）

> 文档版本：release/v2.17.2 | 面向读者：客服、运营

---

## 触发条件

医生需要全面了解某个患者的健康信息，包括病史、病历、处方、量表、随访计划、检查报告等。

## 前置条件

- 当前角色为医生
- 已从患者列表中选择一个患者进入

## 步骤详解

### 入口

医生端 → 患者管理 → 点击患者 → 进入健康档案页

### 第1步：查看患者基本信息

顶部展示患者信息卡片：
- 头像根据性别和年龄自动切换（儿童<6岁/男性/女性）
- 姓名、性别、年龄
- 有监护人时显示：监护人: 姓名(关系)，关系有父亲/母亲/其他

### 第2步：患者标签管理

- 医生可为患者添加或编辑标签
- 标签变更后自动保存
- 保存失败弹窗提示「就诊人信息更新失败，请稍后重试」

### 第3步：诊疗记录入口区

4个子功能入口（按开关配置可能不全显示）：
1. **线下病历**（始终显示）
2. **量表记录**（需scaleEnabled开关）
3. **随访计划**（需followupEnabled开关）
4. **查报告**（需examineReportEnabled开关）

按sort值排序：线下病历(1) → 量表记录(2) → 随访计划(3) → 查报告(4)

### 第4步：病历记录列表

- 时间线形式展示，每页5条滚动加载
- 每条含处方信息，支持微信分享给患者
- 分享标题格式：「{医生名}医生为「{患者名}」开具了处方，为不耽误用药请尽快支付」

### 第5步：复诊功能（条件展示）

同时满足三个条件才显示「复诊」按钮：
1. 系统配置开启revisitEnabled
2. 当前医生开通了复诊服务（serviceType=7且enable=true）
3. 从患者管理进入（URL带allowDisplayRevisit）

从其他路径查看时不显示复诊按钮。

## 异常处理

- 就诊人信息更新失败：弹窗提示
- 病历记录为空：不刷新分页状态

## 关联功能

- 线下病历详情、量表记录、随访计划、查报告均独立跳转
- 复诊 → 发起对该患者的复诊问诊
