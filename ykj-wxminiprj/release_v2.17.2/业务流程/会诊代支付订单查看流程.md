# 会诊代支付订单查看流程

> 文档版本 1.0 | 面向非技术读者 | 2026-07-02

## 概述

患者可以通过微信分享将会诊订单发给亲友代为支付。收到分享链接的人打开后可以看到会诊详情（患者信息、会诊时间、会诊医生、费用等）和订单支付状态，并完成代付或仅查看。此功能适用于患者不便自行支付时由他人协助完成付款的场景。

## 触发条件

- 患者发起会诊后，通过微信分享「代付」链接给亲友
- 收到分享链接的亲友点击进入
- 入口：微信聊天中的分享卡片 → 代支付订单详情页

## 步骤详解

### 第一步：身份校验

打开代付链接后，系统首先校验访问者身份：
- **未登录**：显示登录引导页，提示"您未登录，请登录后查看"，可点击登录按钮
- **订单已被支付**：显示"订单已被支付，已无需代付"，可点击返回
- **正常访问**：进入订单详情页，自动每分钟刷新一次订单状态

### 第二步：查看订单支付状态

页面顶部根据订单状态显示不同的状态卡片：

| 订单状态 | 显示内容 | 操作 |
|---------|---------|------|
| 待支付（flowStatus=10） | 显示支付金额和有效倒计时 | 可点击支付按钮 |
| 已取消（flowStatus=70） | 显示"订单已取消" | 无操作 |
| 已失效（flowStatus=80） | 显示"订单已失效" | 无操作 |
| 退款中（flowStatus=50） | 显示已付金额、支付人、支付时间 | 无操作 |
| 已退款（flowStatus=90） | 显示已付金额、支付人、支付时间 | 无操作 |
| 已支付（其他状态） | 显示已付金额、支付人、支付时间 | 无操作 |

### 第三步：查看会诊详情

如果是会诊订单且访问者具有业务角色权限，页面展示完整的会诊信息：

**会诊时间**：显示会诊开始时间、持续时长（分钟）、结束时间

**患者信息**：
- 患者姓名
- 性别和年龄
- 病情描述
- 经治科室
- 当前诊断

**会诊信息**：
- 会诊类型（普通会诊等）
- 会诊原因
- 会诊目的
- 会诊科室

**会诊医生**：
- 经治医生（发起会诊的医生）
- 参会医生（受邀参加的医生列表）

**会诊费用**：显示会诊费用金额

### 第四步：代支付

如果订单状态为「待支付」（flowStatus=10），点击支付按钮：
1. 重新检查订单状态（防止已被他人支付）
2. 确认状态仍为待支付后，跳转到收银台页面
3. 收银台支持微信支付等方式完成付款

### 第五步：分享转发

代付人可以将代付链接再次分享给其他人，分享标题为「我有一笔订单需要你帮我支付」，并附带自定义分享图片。

## 异常处理

| 情况 | 系统处理方式 |
|------|------------|
| 未登录 | 显示登录引导页，引导用户先登录 |
| 缺少订单号 | 提示"缺少订单号参数"，无法进入页面 |
| 订单已被他人支付 | 显示"订单已被支付，已无需代付"，只能查看 |
| 分享记录更新失败（401） | 标记为未登录状态 |
| 分享记录更新失败（其他） | 标记为不可访问状态 |
| 订单状态在支付时已变更 | 支付前重新检查，如已变更则不跳转支付页 |
| 非会诊订单类型 | 不展示会诊详情区域，只显示支付状态和底部返回按钮 |

## 关联功能

- **订单支付**：跳转到收银台完成微信支付
- **会诊详情**：查看完整会诊申请信息
- **微信分享**：支持二次分享代付链接
