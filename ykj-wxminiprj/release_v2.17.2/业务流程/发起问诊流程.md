# 发起问诊流程（填写病情信息）

> 文档版本：release/v2.17.2 | 面向读者：客服、运营

---

## 概述

这是患者从医生详情页选择服务后、到正式进入问诊前的核心中间环节。患者在此填写就诊人信息、病情描述、上传病历资料、同意知情书，再根据不同的问诊类型走不同的后续流程。

系统支持多种入口场景，包括：普通问诊、快速问诊、咨询服务、AI问诊、医生复诊（医带患）、慢病医保复诊、导诊进入等。

---

## 1. 触发条件

患者需先完成以下操作之一才能进入此页面：
- 在医生详情页选择了一项问诊服务并点击确认
- 从导诊流程自动带入患者信息进入
- 从慢病医保通道进入
- 医生发起复诊邀请，患者进入填写

---

## 2. 页面入口参数

页面通过 `service` JSON 参数接收关键信息：

| 字段 | 说明 |
|------|------|
| doctorId | 选择的医生ID |
| serviceType | 服务类型：1-图文问诊 2-视频问诊 3-咨询服务 4-快速问诊 5-快速续方 6-会诊服务 7-医生复诊 9-AI图文问诊 |
| uniqueId | 导诊唯一ID（导诊入口特有） |
| chronicCode/chronicName | 慢病医保相关的编码和名称 |
| recordId | 量表记录ID |
| patientId | 指定的患者ID |
| scheduleItemNo | 预约挂号号段编号 |

---

## 3. 页面内容（按场景分支）

页面根据服务类型展示不同内容，存在3种主要布局：

### 3.1 咨询模式（serviceType=3 或 快速问诊禁止开方）

- **咨询人信息**：与就诊人信息相同，但标签文案为「咨询人」
- **咨询内容**：文本输入框，提示词为「请描述您需要咨询的问题...」，最大1000字
- **上传照片**：最多上传8张咨询相关照片（非病历资料）
- **知情同意书**：患者知情同意书（如果是快速问诊禁止开方，则展示"我已阅读并同意《患者知情同意书》"勾选框）
- **底部按钮**：取消 / 确认

### 3.2 医生复诊模式（serviceType=7）

医生从病历中发起复诊，页面自动带入：
- 患者信息和病史
- 复诊疾病（诊断信息，不可修改为只读）
- 病情描述输入区
- 底部按钮：取消 / 发起复诊

### 3.3 普通问诊模式（其他 serviceType）

完整展示以下所有区域：

- **就诊人信息卡片**：支持切换就诊人（除非从特定患者进入），展示姓名、性别、年龄、监护人信息等
- **门慢医保复诊标签**：如果当前就诊人有慢病且走的是慢病通道，则显示蓝色标签
- **病情描述**：文本输入区，提示词为「详细描述您的病情...最少输入2个字」
- **上传病历资料**：单选「上传」或「记录遗失/不在身边」，选择上传时最多8张图片
- **初诊信息**（可选）：线下初诊医院、就诊时间（如果系统开启首诊资料收集且非AI问诊）
- **知情同意书**：倒计时3秒的知情同意书确认，包含勾选框
- **医保费别选择**（可选）：在图文/视频/快速问诊/快速续方且医生支持医保时显示，可选择「自费」或「医保」

---

## 4. 数据缓存机制

页面利用微信本地缓存（`inquirySync`）保存用户填写的信息：
- 切换就诊人、退出页面后重新进入会恢复之前的数据
- 缓存内容包括：就诊人ID、病情描述、上传图片、初诊信息、病史
- 点击"取消"退出时会弹出确认框「是否取消问诊？确认后信息将无法保存」，确认后清除缓存
- 提交成功后自动清除缓存

---

## 5. 提交前校验规则

提交时按顺序执行以下校验（任一不通过则阻止提交）：

| 校验项 | 提示文案 |
|--------|----------|
| 未添加就诊人 | 「尚未添加就诊人，是否前往添加就诊人？」 |
| 就诊人身份未认证 | 「当前就诊人未认证，请先进行身份认证？」 |
| 就诊人年龄<6岁且监护人信息不全 | 「监护人信息不全，先完善您的信息」 |
| 病史必填但未填 | 「请填写病史信息，如果没有请填写"无"」 |
| 病情描述必填但为空 | 「请先填写病情描述」 |
| 病情描述少于2字 | 「病情描述至少输入2个字」 |
| 选择了「上传」但未上传病历 | 「请先上传病历资料」 |
| 配置要求病历必填但未上传 | 「请上传病历资料」 |
| 未填写线下初诊医院 | 「请填写线下初诊医院」 |
| 未填写就诊时间 | 「请填写就诊时间」 |
| 未勾选知情同意书 | 弹出知情同意书弹窗要求确认 |

### 关于年龄<6岁的处理
- **严格模式**：弹窗提示「不满6岁儿童不能在互联网医院进行问诊，请到线下实体医院问诊」，无法继续
- **普通模式**：允许问诊，但需填写监护人姓名、关系、身份证号

---

## 6. 重复问诊检查

切换就诊人或提交前，系统会自动检查该患者是否已有正在进行的问诊：

- **有已支付的问诊/咨询**：弹窗提示「您有正在进行的问诊/咨询，是否前往？」→ 点击「前往问诊/咨询」跳转聊天室
- **有未支付的问诊/咨询**：弹窗提示「您有未支付问诊单/咨询单，是否前往支付？」→ 可选择「去支付」或「重新创建问诊」（会取消原订单）

---

## 7. 快速问诊的特殊处理

快速问诊（serviceType=4）有以下特殊逻辑：

1. **在线医生检查**：进入页面时自动查询是否有可匹配的快速问诊在线医生，如果没有则弹窗提示「当前暂无快速问诊医生在线，若要问诊，可选择其他医生」→ 点击「我知道了」跳转到专家搜索页
2. **服务时间校验**：提交时检查当前时间是否在系统配置的服务时间内，不在则弹窗提示服务时间
3. **直接进入问诊**：提交后直接创建问诊流程并跳转到聊天室，无需进入支付页面

---

## 8. 医保费别选择流程

当支持医保且选择了「医保」时：

1. 检查是否选择了就诊人
2. 检查就诊人是否可用
3. 查询就诊人医保定点状态：
   - **未定点**：弹窗提示「该就诊人暂未定点[XX]，定点后可享用医保统筹报销」→ 可选「暂不定点」（改自费）或「确定定点」
   - **已定点本机构**：正常使用医保
   - **已定点其他机构**：弹窗提示是否修改定点 → 可选「取消」（改自费）或「修改定点」
4. 定点成功后更新患者病史中的费别

---

## 9. 提交后的流程分支

| 服务类型 | 后续流程 |
|----------|----------|
| 快速问诊（serviceType=4） | 直接调用问诊接口创建问诊 → 跳转IM聊天室 |
| 其他类型 | 跳转创建订单页 → 确认订单 → 支付 → 进入问诊室 |
| 咨询服务（serviceType=3） | 与普通问诊相同，走创建订单流程 |

---

## 10. 异常处理

- **医保接口错误**：弹窗显示医保错误信息
- **就诊人未填写联系地址**：弹窗提示「因处方合规要求，需先填写就诊人的联系地址」→ 可选择「前往填写」或「取消」
- **取消问诊**：弹窗二次确认「是否取消问诊？确认后信息将无法保存」
- **知情同意书获取失败**：提示「知情同意书缺少数据返回」

---

## 关联功能

- 就诊人管理 → 添加/编辑就诊人信息
- 创建订单页 → 确认费用并支付
- IM聊天室 → 进入问诊对话
- 专家搜索页 → 快速问诊无医生时跳转
