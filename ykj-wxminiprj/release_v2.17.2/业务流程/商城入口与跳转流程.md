# 商城入口与跳转流程

> 文档版本：release/v2.17.2 | 面向读者：客服、运营

---

## 流程概述

小程序内置了商城（H5页面），通过 WebView 承载第三方商城系统。小程序作为入口层，负责传递用户身份（Token）、登录状态验证，以及处理商城内触发的各种跳转回调（问诊、用药登记、支付等）。

---

## 触发条件

用户通过以下场景进入商城：
1. **首页入口**：在首页点击商城入口，进入商城首页
2. **推荐商品**：医生在问诊中向患者推荐商品，患者点击跳转商城商品详情
3. **商品支付**：患者在商城中选择商品，发起微信支付
4. **用药登记**：商城问诊场景中跳转到用药登记页

---

## 步骤详解

### 第一步：构造商城访问链接

1. 页面加载时从系统配置中获取商城 H5 地址（后端 cmConfig.mall.h5Endpoint 配置项）
2. 拼装完整的商城 URL：`商城H5地址/pages/transfer_station/index?token=ehos-XXX&url=商城内部页面&extend=参数`
3. extend 中包含：用户 openId、来源参数、其他业务参数等

### 第二步：用户身份验证

商城页面分为两种访问状态：
- **未登录状态**：如果用户没有 Token，仍可访问商城但功能受限
- **已登录状态**：传递完整的用户身份信息

如果是从微信分享进入商城（参数中 type=share 或 otherType 有值），但当前角色不是患者，系统会：
1. 强制切换为患者角色
2. 保存当前页面路径到缓存
3. 引导用户登录后再跳回

### 第三步：商城内部跳转回调

商城 H5 通过小程序通信机制向小程序发送消息。根据消息类型执行不同跳转：

| 消息类型 | 触发条件 | 跳转目标 |
|---------|---------|---------|
| 用药登记 | other.type=1 且有用药需求编号 | 跳转到用药登记页 |
| 问诊跳转 | other.type=1 且有问诊单号 | 重定向到问诊聊天室 |
| 推荐商品 | other.type=2 且有分享编号 | 将商品卡片发送到问诊聊天，然后跳回聊天室 |
| 微信支付 | other.type=3 | 跳转到商城支付页（mallPay） |
| Token过期 | status=401 | 清除登录状态，引导重新登录 |

### 第四步：订单状态检查

当从商城跳转到问诊时，系统会先检查订单的当前状态：
- 如果已生成问诊（inquiryStatus=1）：直接进入聊天室
- 否则使用更新后的平台订单号重新构建商城跳转

---

## 异常处理

| 异常场景 | 处理方式 |
|---------|---------|
| 商城 H5 地址未配置 | 无法访问商城 |
| Token 过期（status=401） | 引导用户重新登录 |
| 非患者角色访问分享链接 | 强制切换为患者角色并引导登录 |
| 订单状态已改变 | 使用最新订单号重新跳转 |
| 重复跳转到同一页面 | 直接刷新页面，不重复导航 |
