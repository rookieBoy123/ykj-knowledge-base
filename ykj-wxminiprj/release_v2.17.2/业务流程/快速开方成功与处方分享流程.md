# 快速开方成功与处方分享流程

> 文档版本 v1.0 | 面向读者：医生用户

## 概述

医生在完成快速开方后进入的成功结果页面，可查看处方信息、失效时间，并通过三种方式将处方发送给患者支付。

## 触发条件

- 前置条件：医生已完成快速开方并提交处方
- 入口路径：快速开方页面提交成功后自动跳转至本页，页面参数携带处方编号

## 步骤详解

### 第一步：查看开方成功结果

页面显示"开方成功"提示，并展示：
- 患者姓名、性别、年龄
- 处方失效时间（红字显示，格式化为 MM月dd日 HH:mm）
- "处方已发送给药师"的提示文字
- "请尽快提醒患者支付"的提醒

点击"查看处方"蓝色链接可跳转到处方详情页查看完整处方内容。

### 第二步：选择支付方式（从问诊室直接开方时）

若从问诊室进入（存在问诊单号），页面底部显示两个按钮：
- **继续开方**：跳转到快速开方页面，自动带入当前患者信息和诊断
- **返回问诊室**：返回问诊室聊天页继续沟通

### 第三步：选择支付方式（从快速开方独立进入时）

若从快速开方独立进入（无问诊单号），页面底部显示三种处方分享方式：
- **发送给患者**：弹出确认窗口，展示患者姓名/性别/年龄，并提示"处方将与首次打开的微信好友绑定，其他人无法查看"
- **患者扫码支付**：生成处方专属二维码，患者微信扫码进入药品订单页支付
- **代患者支付**：医生进入药品下单页面，代为完成支付

### 第四步：微信分享（发送给患者）

点击"发送给患者" → 弹出确认窗口 → 点击"确定"触发微信分享 → 分享卡片标题为"XX医生为「患者名」开具了处方，为不耽误用药请尽快支付" → 患者点击分享卡片进入药品订单页

### 第五步：生成处方二维码

点击"患者扫码支付" → 系统生成处方二维码 → 弹窗展示二维码图片 → 提示"处方二维码将与首次扫码的微信好友绑定，绑定后其他人无法查看"

### 第六步：继续开方

点击"继续开方" → 跳转至处方编辑页（addOrEditRp），自动带入：
- 患者ID（patientId）
- 问诊单号（inquiryNo）
- 病历记录ID（medicalRecordId）
- 快速开方标识（fastIssuanceFlag）

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 处方编号缺失 | 提示"缺失处方编号" |
| 处方已选择一种分享方式 | 提示"该处方已选择了【XX方式】，暂不支持其他方式"，阻止重复分享 |
| 代支付方式不匹配 | 仅当shareType为3（医生代付）时允许代支付，否则提示同上 |

## 关键规则

- **分享方式互斥**：处方只能选择一种支付方式（二维码/微信链接/医生代付），一旦选择后其他按钮点击会提示拦截
- **微信好友绑定**：处方与首次打开的微信好友绑定，确保处方安全性
- **失效时间**：处方有有效期，超时后患者无法支付，需医生重新开方
- **返回逻辑**：返回问诊室时先检查页面栈中是否有聊天室页面，有则回退到聊天室；没有则重新加载聊天室

## 关联功能

- 快速开方 → 处方详情页 → 药品下单页（患者端）
- 处方编辑页（继续开方）
- 问诊室IM聊天页（返回问诊室）
