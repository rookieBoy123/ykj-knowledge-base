# 药品订单预结算与创建流程

> 文档版本 v1.0 | 面向读者：产品、运营、客服

## 概述

用户在收到医生的用药建议/处方后，进入药品订单页面确认药品信息、选择配送方式及收货地址，系统进行预结算展示费用明细，用户确认后创建订单并跳转收银台支付。页面支持医生代付模式和患者自付模式。

## 触发条件

- **前置条件**：医生已开具用药建议/处方并审核通过
- **入口路径**：
  - 患者端：从处方详情页点击"付款"进入
  - 医生端：从处方详情页点击"代支付"进入
  - 快速开方分享链接进入

## 步骤详解

### 第一步：进入订单页

页面加载后自动执行：
1. 查询处方详情（患者信息、药品信息、处方状态）
2. 检查是否已有有效订单：
   - **已有订单** → 直接展示订单信息
   - **无订单** → 调用预结算接口计算费用

### 第二步：查看订单信息

页面从上到下展示：

**订单状态区**：显示当前订单状态（待付款、待发货等）

**物流/配送信息区**：
- 快递配送：显示收货地址（如未添加则提示选择）
- 到店自提：显示药房地址
- 用户可切换配送方式（自提/快递），切换后自动重新预结算

**患者信息区**：显示就诊人姓名、性别、年龄、科室信息

**药品信息区**：
- 处方类型（成药处方/中药饮片）
- 药品清单（名称、规格、数量、单价）
- 中药处方可切换加工工艺（如代煎、颗粒等），切换后自动重新预结算
- 如系统配置了"未支付不可查看用量"，中药饮片的用量在支付前隐藏

**费用信息区**：
- 药品费用
- 加工费用（如有）
- 配送费用
- 合计金额
- 如药房支持订金支付，可选择订金/全款

### 第三步：校验下单条件

点击"提交订单"或"去支付"时，系统依次校验：

1. **收货地址**：快递配送模式下必须选择收货地址，否则提示"请先添加收货地址"
2. **患者可用性**：检查就诊人是否可用（未被禁用/删除）
3. **实名认证**：患者需完成身份认证，否则提示"患者需进行身份认证后才可付款"
4. **年龄限制（严格模式）**：不满6岁儿童不能在互联网医院问诊，提示"不满6岁儿童不能在互联网医院进行问诊，请到线下实体医院问诊"
5. **监护人**：不满6岁需绑定监护人信息，弹出监护人填写表单（监护人姓名、身份证号、与患者关系）

### 第四步：绑定会员

如为快速开方处方且患者尚未绑定会员：
- 自动调用会员绑定接口
- 绑定成功 → 继续下单
- 该就诊人已被其他手机号绑定 → 提示"同一就诊人只能绑定在一个手机号，当前就诊人已绑定[手机号]，是否进行解绑？"
  - 点击"去解绑" → 跳转就诊人解绑页面（需短信验证码）
  - 点击"取消" → 取消下单
- 非自己的处方 → 跳转到"非本人处方"提示页

### 第五步：创建订单

校验通过后，调用创建订单接口：
- 成功 → 返回订单号，跳转收银台（hisPay）
- 失败 → 弹窗提示错误信息

### 第六步：医生代付特殊流程

当医生从处方详情页点击"代支付"进入时：
- 页面标题显示"代患者支付"
- 支付按钮允许医生角色操作（通常仅患者可见）
- 代付成功后弹出"指定处方接收人"弹窗
- 医生可将处方发送给指定患者微信

## 异常处理

| 情况 | 系统处理方式 |
|------|------------|
| 预结算失败 | 弹窗提示具体错误信息，确认后返回上一页 |
| 处方审核不通过 | 隐藏支付按钮，显示"审核不通过"状态 |
| 未添加收货地址 | 提示"请先添加收货地址" |
| 患者未实名 | 提示"患者需进行身份认证后才可付款" |
| 不满6岁（严格模式） | 弹窗提示需线下就诊 |
| 不满6岁（普通模式） | 弹出监护人信息表单 |
| 就诊人被禁用 | 不允许下单，提示检查就诊人状态 |
| 就诊人已被绑定 | 提示手机号冲突，引导解绑 |
| 处方不是自己的 | 跳转"非本人处方"提示页 |
| 订单状态已变更 | 提示"订单状态发生改变，请刷新页面" |
| 处方已失效 | 支付按钮不显示 |

## 关联功能

- 处方详情（点击返回）
- HIS收银台（创建订单成功后跳转）
- 收货地址管理（选择/新增地址）
- 就诊人管理（检查可用性、实名认证）
- 就诊人解绑（会员冲突时）
