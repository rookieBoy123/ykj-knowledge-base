# 服务评价提交流程

> 文档版本：release/v2.17.2 | 面向读者：客服、运营

## 概述

患者在问诊结束后可以对本次服务进行评价。评价包含满意度打分（1-5星，附快捷标签）和对医生的文字评价（最少1字，最多100字）。提交后立即生效，通知问诊列表刷新。已评价过的再进入为只读模式。

## 触发条件

### 前置条件
- 患者已完成一次问诊
- 患者未对该问诊评价过（或评价后可再次进入查看）

### 入口路径
- 问诊结束页 → 点击「评价」→ 传入 doctorId 和 uuid
- 问诊历史列表 → 已完成的问诊 → 评价入口

## 步骤详解

### 第一步：进入评价页

1. 检查是否为订阅消息入口（是则跳过）
2. 获取医生信息（姓名、科室、职称、医院）
3. 检查是否已评价——有记录则只读模式，无记录则可编辑

### 第二步：满意度打分（可编辑模式）

5星制，默认5星：

| 星级 | 文字 | 快捷标签 |
|------|------|---------|
| 1星 | 非常不满意 | 没有解决问题、问诊不够详细、回复不够及时 |
| 2星 | 不满意 | 同上 |
| 3星 | 一般 | 解决部分问题、问诊比较详细、回复比较及时 |
| 4星 | 满意 | 解决问题、问诊详细、回复及时 |
| 5星 | 非常满意 | 同上 |

快捷标签可多选，切换星级后标签内容自动切换。

### 第三步：填写文字评价

- 占位提示：「问诊过程还满意吗？说说您的感受和建议吧」
- 最多100字，必填

### 第四步：提交评价

- 文字为空提示「请填写对医生的评价再提交~」
- 提交数据包含：appointNo、doctorId、satisfaction、tagStr、introduction
- 成功后：提示「提交成功」、切换只读模式、通知其他页面刷新、1秒后自动返回

### 第五步：只读模式

- 顶部橙色条：「评价已提交」
- 星级评分只读
- 已选标签橙色显示
- 评价内容可读
- 右下角显示评价时间
- 底部仅「返回」按钮

## 异常处理

| 情况 | 系统处理方式 |
|------|------------|
| 文字评价为空 | 「请填写对医生的评价再提交~」 |
| 超100字 | 自动截断 |
| 参数不全 | 非订阅消息入口且缺参数时，不初始化 |

## 关联功能

- 提交成功自动返回上一页
- 通知问诊历史列表刷新（uni.$emit）
