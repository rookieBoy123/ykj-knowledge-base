# 药品详情查看与用法配置流程

> 文档版本 | 面向读者：医生

## 概述

医生在开具处方或选择药品时，可以查看药品的详细信息（图片、价格、规格、说明书等），并在需要时配置用法用量（用药方式、使用频次、每次用量、药品数量、使用天数）。

## 触发条件

- **前置条件**：医生在开处方或订单模板中添加药品
- **入口路径**：药品搜索结果 → 点击某条药品记录进入详情页
- **参数**：药品ID、药品类型（1=成药/2=中药）、是否可修改（canModify标识）

## 步骤详解

### 第一步：药品基本信息展示

页面加载后自动根据药品类型调用对应接口（成药/中药）获取详情。展示内容：

**1. 轮播图区域（顶部）**
- 如有药品图片：自动轮播展示，带小圆点指示器
- 如无药品图片：显示默认占位图（"暂无药品图片"样式）

**2. 价格与库存信息**
- 价格：红色大号字体"¥XX.XX"，后面跟随价格单位
- 库存：如开启拆零销售，显示库存制剂量单位；未开启则显示库存包装单位。库存为-1时不显示库存信息（隐藏单位文本）
- 编码：药品编码+药品编号（灰色小字）

**3. 药品标识信息**
- 通用名（加粗大字体）
- 商品名（仅非严格模式显示）
- 厂家或产地（仅非严格模式显示）
- 药品规格
- 药品剂型（如有剂型字典值则显示）

### 第二步：药品说明查看

- 药品说明区域标题带文件图标
- 如有说明书内容：以富文本（HTML）格式渲染展示
- 如无说明书：显示空状态图 + "药品说明正在完善中..."提示文字

### 第三步：用法用量配置（仅药品选择模式）

当从订单模板或处方页进入并传了`canModify`参数时，底部显示"选择药品"按钮。点击后弹出配置面板：

**弹出面板内容**（DrugModifyPopup组件）：
1. **用药方式**（specificationlist）：如"口服""外用""静脉注射"等，从字典`ywsytjbm`加载
2. **使用频次**（degreelist）：如"每日1次""每日2次"等，从使用频次接口加载
3. **每次用量**（phruomlist）：用量单位，从字典`mcyl`加载
4. **药品数量**（qty）：通过减号/加号按钮或直接输入数字调整，最小值为1
5. **使用天数**（useDays）：同样通过增减或输入调整

每个选项区域内嵌子选择器，点击后弹出下级选择面板。修改后数据实时更新。

### 第四步：确认选择

配置完所有用法用量参数后：
- 点击面板底部"添加药品"或"选择药品"按钮
- 通过事件通道将配置好的药品数据回传给上一个页面（`eventChannel.emit('modifyDrug', data)`）
- 自动返回上一页面，药品出现在处方或订单模板的药品列表中

## 严格模式差异

| 展示内容 | 普通模式 | 严格模式 |
|---------|---------|---------|
| 商品名 | 显示 | 隐藏 |
| 厂家或产地 | 显示 | 隐藏 |
| 药品说明书 | 正常显示 | 正常显示 |
| 价格/库存 | 正常显示 | 正常显示 |

严格模式通过全局配置`cmConfig.strict`控制，主要防止患者按商品名或厂家去其他渠道比价购药。

## 异常处理

| 情况 | 系统处理方式 |
|------|------------|
| 药品ID或类型参数缺失 | 页面不加载任何内容（无数据显示） |
| 药品详情接口返回失败 | 静默处理，页面保持空白，不显示错误提示 |
| 药品无图片 | 显示默认占位图而非轮播图 |
| 药品无说明书 | 显示"药品说明正在完善中..."的空状态 |
| 药品库存为-1（不显示库存） | 价格后的单位文本为空字符串 |
| 非选择模式（canModify=false） | 不显示底部按钮，不加载用法配置面板 |
| 用法配置数据异常 | 弹出面板中最小增减有下限（数量不能减至0以下），输入非数字自动处理 |
