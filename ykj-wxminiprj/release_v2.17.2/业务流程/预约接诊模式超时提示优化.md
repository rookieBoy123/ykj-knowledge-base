# 预约接诊模式超时提示优化

> 文档版本 2026-07-05 | 面向读者：产品经理、运营、客服

## 概述
医生开启"预约接诊"模式并启用"过号接诊"后，医生主页的超时取消提示从"X小时自动取消"改为固定的"当天23:59自动取消"。这一变更让超时规则更清晰，避免用户困惑。

## 触发条件
- 医生在后台设置了会诊类型为"预约接诊"（consultationType = 1）
- 医生同时启用了"延时签到/过号接诊"（isDelayCheckIn = 1）
- 患者进入该医生的主页查看

## 步骤详解

### 判断逻辑
1. 系统获取医生的 `consultationType` 和 `isDelayCheckIn` 两个字段（服务端 `CurrentDoctorVO` 新增返参）
2. 当两个条件同时满足时（预约接诊 + 过号接诊），判定为预约过号接诊模式
3. 此时超时提示区始终显示（不再依赖 timeoutHours 字段）

### 显示规则
- **普通模式**（在线接诊或未启用过号接诊）：显示"温馨提示：问诊订单超过X小时未接诊，将自动取消"（X 为服务端配置的接诊超时时间）
- **预约过号接诊模式**：显示"温馨提示：问诊订单当天未接诊，将在当天23点59分自动取消"

### 样式优化
- 提示文案去掉了 `white-space: nowrap` 单行限制，支持自动换行
- 文本区设置为 `flex: 1` 自适应宽度，不会因为文案变长而被截断

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 服务端未返回 consultationType | 字段缺失，Number() 转为 NaN，不等于 1，走普通模式 |
| 服务端未返回 isDelayCheckIn | 同上，走普通模式 |
| timeoutHours 为 0 且非预约过号模式 | 不显示超时提示 |
| 网络异常 | 显示网络错误提示（覆盖超时提示） |

## 关联功能
- 医生主页（packagePatient1/pages/doctor/index）
- 医生接诊模式设置（管理后台/服务端）
- 订单超时自动取消
