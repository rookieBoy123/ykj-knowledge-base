# 问诊详情查看流程（患者端）

> 文档版本：release/v2.17.2 | 面向读者：患者、客服

## 概述

患者可以从问诊记录列表中点击某条记录，查看该次问诊的完整详情，包括医生信息、问诊内容、就诊时间、问卷记录、病历、会诊记录等。

## 触发条件

- **前置条件**：患者已登录，至少有一次问诊记录
- **入口路径**：个人中心→"我的问诊"→点击某条问诊记录

## 步骤详解

### 第一步：查看医生信息

进入页面后，顶部显示医生信息卡片：
- **头像**：医生的圆形头像
- **姓名、科室、职称**：例如"张医生 | 中医科 | 主任医师"
- **医院名称**
- 点击医生卡片可以跳转到该医生的详情页（如果是AI医生则跳转AI医生详情，普通医生跳转服务选择页）

### 第二步：查看问诊详情

问诊详情区展示以下信息：
- **问诊人**：就诊人的姓名、性别、年龄（岁/月）
- **服务类型**：如图文问诊、视频问诊、咨询服务、快速问诊等
- **病情描述**：患者填写的病情描述，没填时显示"无"
- **问诊时间**：问诊创建时间
- **就诊时间**（条件显示）：非AI问诊、非咨询服务且存在开始时间时才显示
- **结束时间**（条件显示）：非AI问诊且存在结束时间时才显示

就诊时间的显示规则较复杂：
- 问诊中（flowStatus=20）+ 助理医生待接诊（doctorFlagStatus=10）：不显示任何时间
- 问诊中 + 接诊中（doctorFlagStatus<50）：显示开始时间，不显示结束时间
- 问诊结束（flowStatus=40）：不显示任何时间
- 其它情况：同时显示开始时间和结束时间

### 第三步：查看问诊问卷

如果问诊时有填写问诊问卷，这一区域会展示问卷的题目和患者的回答。由子组件`InquirySurvey`渲染。

### 第四步：查看病历详情

如果服务类型不是咨询服务（serviceType≠3）且不是AI问诊（serviceType≠9），会显示"病历详情"卡片。

点击后先检查是否存在病历：
- **有病历**：跳转到病历详情页（医生端病历页）
- **无病历**：弹窗提示"未生成病历"

### 第五步：查看会诊记录（如有）

如果本次问诊关联了会诊记录，每条会诊单独一张卡片展示：
- **会诊目的**：发起会诊时填写的目的
- **会诊时间**：会诊安排的时间
- 点击卡片可跳转会诊详情页

### 第六步：问诊对话记录和评价

页面底部有两个入口：
- **问诊对话记录**：点击查看问诊室的聊天记录
  - 如果问诊已被取消（flowStatus=40），弹窗提示取消原因（有备注显示备注，无备注显示"该问诊已被取消"）
  - 如果是AI问诊（serviceType=9），跳转AI问诊聊天室
  - 普通问诊：进入已结束的问诊聊天室（只读模式）
- **评价医生**（条件显示）：仅在`flowStatus=30`（问诊结束）且非AI问诊时显示

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 病历未生成 | 弹窗提示"未生成病历" |
| 问诊已被取消 | 弹窗显示取消原因，有备注显示"该问诊已被取消，[xxx]"，无备注显示"该问诊已被取消" |
| 服务类型为咨询服务(3) | 不显示"病历详情"入口 |
| 服务类型为AI问诊(9) | 不显示"病历详情"入口和"评价医生"入口 |
| 无会诊记录 | 不显示会诊记录区域 |
| 问诊未结束（flowStatus≠30） | 不显示"评价医生"入口 |
| 页面缺少必要参数(uuid) | 页面无法初始化，空白 |

## 关联功能

- 医生详情页（selectedDoctorService/doctorDetail）：点击医生卡片跳转
- AI医生详情页（aiDoctorDetail）：AI医生跳转
- 病历详情页（medicalRecord/detail）：点击病历详情跳转
- 会诊详情页（groupConsultation/detail）：点击会诊记录跳转
- 问诊聊天室（chat.vue）：查看对话记录
- AI聊天室（aiChat）：AI问诊对话
- 评价页（evaluate/index）：评价医生
