# 快速续方流程（患者端）

> 文档版本 2026-07-02 | 面向患者

## 概述

快速续方是为"正按方吃药且病情稳定"的患者提供的便捷开方渠道。患者可以从历史有效处方中直接发起续方，无需再次填写病情信息，大大提高慢病复诊效率。

## 触发条件

- 患者登录小程序
- 从首页或相关入口进入"快速续方"页面
- 适用人群：有线上或线下有效处方，正按方吃药，病情稳定的患者

## 页面说明

### 顶部提示区
展示快速续方适用条件提示："快速续方适用有线上或线下有效处方，正按方吃药并且病情稳定的患者"

### 上传处方续方入口
- 显示"+ 上传处方续方"按钮（非特殊患者模式下可见）
- 点击后走快速问诊流程（服务类型=4），上传线下处方照片发起问诊

### 处方列表区
- 标题"处方列表"，副标题"仅展现最近6个月的有效处方"
- 每张处方卡片展示处方基本信息，并提供"续方"和"详情"按钮
- 支持上拉加载更多（每页5条）

## 续方流程详解

### 第一步：点击"续方"
患者点击某张处方卡片上的"续方"按钮。

### 第二步：医生可用性检查
1. **检查医生是否开通快速续方服务**
   - 已开通 → 继续
   - 未开通 → 弹窗提示：
     - 普通患者："原处方医生暂不方便续方，是否为您匹配新的医生？"
     - 特殊患者（配置支持特殊患者入口时）："医生未开通快速续方服务，您可以向医生发起问诊咨询购药。"

2. **检查是否存在未完成的问诊**
   - 存在未支付的问诊单 → 弹窗"您有未支付问诊单，是否前往支付？"→ 选择支付跳转订单详情
   - 存在进行中的问诊 → 弹窗"您有正在进行的问诊，是否前往？"→ 选择前往跳转问诊室
   - 无未完成问诊 → 继续

3. **检查是否需要匹配新医生**
   - 原处方医生在线可接诊 → 直接进入下单流程
   - 原处方医生不可用 → 弹窗"您的医生暂不在线，是否为您匹配新的医生？"

### 第三步：匹配新医生（如需要）
- 系统根据处方的药房信息自动匹配新的在线医生
- 匹配成功 → 进入下单流程
- 匹配失败（无医生在线） → 弹窗"暂无医生在线，请稍后再试"

### 第四步：下单支付
- 系统生成预订单参数（医生ID、患者ID、服务类型=5）
- 如有处方编号，标记为续方（continuedPrescriptionNo）
- 跳转到下单页完成支付

## 特殊患者模式

如果系统配置了特殊患者功能且当前用户为特殊患者：
- 隐藏"上传处方续方"按钮
- 医生未开通快速续方时显示不同的提示文案
- 点击"去问诊"跳转医生详情页挂号问诊

## 异常处理

| 情况 | 系统处理方式 |
|------|------------|
| 无处方记录 | 显示"暂无处方记录"空状态；普通患者提示"可选择上传线下处方进行续方" |
| 医生未开通快速续方 | 弹窗提示，引导患者匹配新医生或去问诊 |
| 无在线医生可匹配 | 弹窗"暂无医生在线，请稍后再试" |
| 有未支付问诊单 | 弹窗提示去支付 |
| 有进行中问诊 | 弹窗提示前往问诊室 |
| 未登录 | 引导跳转登录页 |

## 关联功能

- 跳转【处方详情页】：查看历史处方完整信息
- 跳转【下单页】：完成问诊服务支付
- 跳转【问诊室】：继续进行中的问诊
- 跳转【订单详情】：支付未完成的订单
- 跳转【医生详情页】：普通问诊挂号
- 跳转【快速问诊】：上传线下处方走问诊流程
