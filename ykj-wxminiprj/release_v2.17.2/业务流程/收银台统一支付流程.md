# 收银台统一支付流程

> 文档版本：v1.0 | 最后更新：2026-07-02 | 面向读者：产品经理、运营人员

## 概述

收银台是小程序的统一支付页面，支持多种支付方式（微信支付、医保支付），覆盖问诊、用药、会诊、预约挂号、体检等多种订单类型的支付。页面显示支付金额、倒计时和可用支付方式列表。

## 触发条件

- **前置条件**：已生成订单
- **入口路径**：
  - 问诊下单后 → 收银台
  - 购买药品后 → 收银台
  - 会诊支付 → 收银台
  - 预约挂号 → 收银台
  - 体检/检查/团体企业预约 → 收银台
  - 代付（他人帮忙支付）→ 收银台

## 步骤详解

### 第一步：进入收银台

页面顶部显示：
- **支付金额**：大字体显示，精确到分
- **支付倒计时**：显示支付剩余时间（时:分:秒），超时后订单自动失效

可用支付方式列表展示每种方式的图标和名称，默认选中推荐的支付方式。

### 第二步：选择支付方式

| 支付方式 | 说明 | 适用条件 |
|----------|------|---------|
| 微信支付 | 使用微信零钱或绑定的银行卡支付 | 所有订单 |
| 医保支付 | 使用医保个人账户支付 | 需医保定点、医生为医保医师 |

**不可用的支付方式**：
- 如果某种支付方式不可用，图标显示为灰色禁用状态
- 下方显示提示信息，说明不可用的具体原因

**筛选说明**：小程序收银台不开放微信扫码支付（二维码支付），该选项会被自动过滤。

### 第三步：医保支付前置校验

选择医保支付时，系统会自动进行以下检查：

1. **患者ID检查**：如果缺少患者ID，提示"缺少患者id，请联系客服"，并自动切换回微信支付
2. **慢病判断**：判断该订单是否为慢病订单（慢病订单走特殊流程）
3. **定点检查**（非慢病流程）：检查患者是否已在对应医保机构定点
   - 未定点：弹出提示框，显示"该就诊人暂未定点[医保名称]，定点后才可使用医保支付"，提供"暂不定点"和"确认定点"两个选项
4. **医保医生检查**：确认接诊医生是否为医保医师，非医保医师弹窗提示"当前医生非医保医师，不支持医保支付，只能自费"

### 第四步：确认支付

点击"确认支付"按钮：
- 如果未选择支付方式，提示"请先选择支付方式"
- 根据选中的支付方式执行不同的支付流程

### 微信支付流程：
1. 调用微信支付接口（使用用户OpenId）
2. 拉起微信支付密码输入
3. 支付结果处理

### 医保支付流程：
1. 订阅医保退款消息
2. 跳转到医保小程序进行授权和支付操作
3. 需要医保配置参数（授权AppId、业务类型、城市代码、渠道号、机构编码等）
4. 如果医保参数缺失，提示"医保参数有误，是否重新获取？"并重新加载配置

### 第五步：支付结果处理

支付完成后，根据订单类型跳转不同页面：

| 订单类型 | 支付成功后跳转 |
|----------|---------------|
| 用药订单(drug) | 药品订单详情页，携带处方编号和预结算标记 |
| 预约挂号 | 订单详情页 |
| 问诊(consultation) | 会诊详情页 |
| 预约(appoint) | 预约详情页 |
| 流程创建(addworkflow) | 聊天室页面 |

**支付成功弹窗**：
- 显示"支付成功"图标和文字
- "我们将尽快安排发货，请耐心等待"
- 提供两个按钮："查看订单"和"返回咨询"

**支付状态不确定处理**：
如果从医保小程序返回后无法确定支付状态，弹出"支付结果信息提示"：
- "未查询到支付状态，请根据您的情况点击下面的按钮"
- "支付未成功"：关闭弹窗，留在收银台
- "支付已成功"：跳转到对应详情页

### 第六步：返回处理

点击返回按钮时：
- **药品订单**：跳转到药品订单详情页（预结算状态）
- **预约挂号**：跳转到订单详情页
- **其他类型**：返回上一页

## 代付场景

支持他人代付（isDf=1）：
- 通过生成二维码或微信链接分享给代付人
- 代付人扫码或点击链接进入收银台完成支付
- 传入shareType标记代付方式（二维码/微信链接/医生代付）

## 异常处理

| 情况 | 处理方式 |
|------|---------|
| 支付方式全部不可用或过滤后为空 | 无法确认支付 |
| 支付倒计时归零 | 触发超时处理（timeup事件），订单失效 |
| 选择医保支付但缺少患者ID | 提示"缺少患者id，请联系客服"，自动切回微信支付 |
| 患者未医保定点 | 弹窗提示定点，可选择暂不定点或确认定点 |
| 医生非医保医师 | 弹窗提示只能自费 |
| 医保参数缺失 | 弹窗询问是否重新获取配置 |
| 未选择支付方式就点确认 | 提示"请先选择支付方式" |
| 支付状态查询不确定 | 提供"支付未成功"/"支付已成功"两个选项供用户确认 |

## 关联功能

- **订单详情**：支付成功后跳转查看订单
- **药品订单**：药品购买支付
- **预约挂号**：挂号费支付
- **会诊管理**：会诊费用支付
- **AI预问诊**：支付后可触发AI预问诊流程
- **就诊人管理**：医保定点与就诊人绑定
