# 用药需求开方流程

> 文档版本：2026-07-02 | 面向读者：产品运营、客服、医生

## 概述

患者在问诊中可以提交「用药需求」，列出自己需要的药品。医生查看患者用药需求后，可直接基于需求内容开具处方。这是一个从患者需求到医生开方的闭环流程。

## 触发条件

- **前置条件**：患者在问诊中提交了用药需求，医生角色进入该页面
- **入口路径**：从问诊会话中点击用药需求卡片进入
- **传入参数**：用药需求编号（drugDemandNo）、问诊单号（inquiryNo）、已有诊断信息

## 步骤详解

### 第一步：查看用药人信息

页面顶部卡片展示患者基本信息：
- 姓名、性别、年龄
- 如果处方已开具，顶部显示「已开处方」状态标签，且诊断和药品变为只读

### 第二步：查看/编辑诊断

诊断区域显示患者提交的诊断信息（从问诊传入）。如果处方尚未开具且当前角色为医生，可以修改诊断。

### 第三步：查看/编辑药品清单

「用药需求」卡片显示患者提交的药品列表，每味药品包含：
- 药品名称/通用名
- 规格
- 用法用量（每日次数、每次用量、用药天数、用药方式）
- 如果用法用量未完善，显示「待完善」标记

医生可以对药品列表进行操作：
- 修改数量
- 完善用法用量
- 删除不需要的药品
- 添加新药品

### 第四步：填写医嘱

医生可以在医嘱输入框中填写用药建议和注意事项。

### 第五步：确认开方

点击底部「确认开方」按钮：

1. **诊断校验**：必须选择诊断，否则提示「请选择诊断」
2. **用法用量校验**：所有药品的用法用量必须填写完整，否则提示「请将已添加的药品用法用量填写完整」
3. **CA签名校验**：
   - 如果开启了CA电子签章，会先检查医生的CA认证状态
   - 认证状态有四种：
     - **未认证（1）**：弹出同步资料弹窗，引导医生同步资料到CA平台
     - **认证中（2）**：提示正在认证，引导去完成认证
     - **认证成功（3）**：根据平台类型处理：
       - 医网信/源康健PaaS：检查本地证书，有则弹出「是否确认患者是复诊」确认框；无则提示先完成签章
       - 腾讯电子签：直接弹出「是否确认患者是复诊」确认框
       - 放心签：检查本地签章，有则确认复诊；无则提示完成签章
     - **认证失败（4）**：提示重新认证
4. **复诊确认**：认证通过后，弹窗确认「您是否确认患者是复诊？」
5. **处方签名**：
   - 严格模式下先人脸识别
   - 然后调用CA签名组件（医网信/放心签）
   - 签名成功后开方生效
6. **开方成功**：跳转到开方成功页面

## 异常处理

| 情况 | 系统处理方式 |
|------|------------|
| 诊断未选择 | 提示「请选择诊断」，阻止提交 |
| 药品用法用量不完整 | 提示「请将已添加的药品用法用量填写完整」 |
| 药品列表为空 | 开方按钮灰色禁用状态 |
| CA账号未同步 | 弹出同步资料弹窗，需医生先同步后才能继续 |
| CA认证中 | 提示前往完成认证流程 |
| CA认证失败 | 弹出提示，引导重新认证 |
| 医网信/放心签未设置本地签章 | 提示「未完成CA电子签章不可接诊，请先完成签章」 |
| 非复诊患者 | 取消操作，不进行开方 |
| 签名失败 | 跳转到用药建议详情页 |
| 已开处方 | 页面只读模式，底部按钮隐藏 |
| 签名取消 | 跳转到用药建议详情页 |
