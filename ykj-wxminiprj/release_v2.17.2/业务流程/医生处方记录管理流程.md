# 医生处方记录管理流程

> 文档版本 v1.0 | 面向读者：医生、产品经理

## 概述

医生在小程序端查看和管理自己开具的所有处方记录。支持按8种状态筛选、删除草稿/已驳回处方、撤回待审核处方、驳回处方（审核场景）、CA电子签章、以及快速开方场景下的多微信绑定发送。

## 触发条件

- **前置条件**：医生已登录小程序、已通过医师认证
- **入口路径**：
  - 普通模式：医生端首页 → 处方记录（显示全部8种状态Tab）
  - 已开处方模式：从问诊聊天室点击"查看已开处方"（仅显示该问诊下的处方，无状态Tab）

## 步骤详解

### 第一步：处方列表浏览

进入页面后，系统加载医生的处方记录列表。

1. **两种入口模式**：
   - **普通处方记录**（默认）：显示搜索栏 + 8种状态Tab + 处方列表
   - **已开处方**（`isIssuedRp=true`）：页面标题变为"已开处方"，隐藏搜索栏，只显示该问诊下的所有处方，不按状态过滤

2. **搜索功能**：支持按患者姓名搜索处方（仅在普通模式下显示）

3. **8种状态Tab**（仅在普通模式下显示）：
   - 全部处方 / 审核通过 / 审核中 / 待审核 / 审核不通过 / 开方中 / 已撤回 / 处方草稿
   - 切换Tab时自动重新加载第一页数据

4. **每页15条**，支持滚动到底部自动加载更多（`scrolltolower` 触发 `getMore`）
5. **空状态**：无处方记录时显示"暂无记录"

### 第二步：处方操作（按状态差异化）

每张处方卡片下方显示的操作按钮根据处方状态动态变化：

**① 删除处方**（草稿/审核不通过状态可见）：
- 点击弹出确认弹窗"确认删除该处方吗？"→"删除之后无法恢复"
- 确认后调用 `deleteRecipe` 接口
- 成功后提示"已删除"，从列表中移除该卡片

**② 撤回处方**（待审核状态可见）：
- 点击弹出确认弹窗"确认撤回吗？"
- 确认后调用 `withdrawRecipe` 接口
- 成功提示"已撤回"，1秒后自动刷新列表
- 失败弹窗显示后端返回的错误信息

**③ 驳回处方**（审核场景）：
- 点击"驳回"按钮，从底部弹出输入面板
- 必须输入驳回原因（最多500字）
- 确认后调用 `doctorApprovalReject` 接口
- 成功提示"已驳回"，关闭面板并刷新列表
- 未填写原因：提示"请输入驳回原因"

**④ CA电子签章**（需签章的处方）：
签名流程分4步：
1. 调用 `getRecordSign` 获取签名平台类型和唯一标识
2. **三种签名平台分支**：
   - `platform=1`（医网信）或 `platform=3`（源康健PaaS）：严格模式下需先人脸识别 → 调用医网信 SDK → 显示签名组件
   - `platform=4`（放心签）：弹出放心签组件，传入 uniqueId 和医生 caUserNo
   - `platform=2`（腾讯电子签）：暂不支持
3. 签名成功：刷新处方列表
4. 签名失败：用户取消操作不做处理，签名错误提示"签字失败，请重试"

**⑤ 发送处方微信消息**（快速开方场景，`fastShareMemberFlag` 为 true）：
- 点击"发送"按钮 → 弹出患者绑定微信选择弹窗
- 获取该患者绑定的会员列表（当前版本为1对1关系，自动选中）
- 用户确认后调用 `bindingMember` 接口
- 成功后提示"操作成功"，更新卡片状态

### 第三步：查看处方详情

点击处方卡片 → 跳转到处方详情页（`/packagePatient1/pages/drugsuggest/detail`）：
- 传递参数：`outpatientNo`（处方门诊号）+ 可选的 `fastIssuanceFlag=1`（快速开方标记）
- 处方详情页展示完整的处方信息（药品清单、用法用量、签章状态等，详见"处方详情查看与操作流程"）

### 第四步：分享处方

点击分享按钮：
- 触发 `onShareAppMessage` 微信分享
- 分享标题："{医生姓名}医生为「{患者姓名}」开具的处方"
- 分享路径：指向患者端处方详情页（`shareType=2` 标记为分享来源）
- 分享图片：默认处方分享图

## 异常处理

| 情况 | 系统处理方式 |
|------|------------|
| 网络异常 | 弹窗提示"网络异常，请稍后再试" |
| 撤回处方失败 | 弹窗显示后端返回的具体错误信息 |
| 驳回未填写原因 | 前端 toast 提示"请输入驳回原因" |
| 签名失败（取消） | 不提示，静默处理 |
| 签名失败（错误） | toast 提示"签字失败，请重试" |
| 绑定会员失败 | toast 提示后端返回的错误信息 |
| CA 签名组件加载失败 | 调用 `signFail` 回调，signFailActions 统一处理 |
| 处方列表接口返回错误 | 弹窗显示"网络异常，请稍后再试" |
| 某页无数据（翻页越界） | 前端自动回退 pageNum-1 重新查询 |

## 关联功能

- **处方详情查看**：点击处方卡片跳转（packagePatient1/pages/drugsuggest/detail）
- **快速开方**：快速开方的处方会带有 `fastIssuanceFlag` 标记（packageDoctor1/pages/quickPrescribe）
- **CA签章设置**：医生的签章方式在"My→处方设置"中配置（packageDoctor1/pages/fxqSetting）
- **人脸识别**：严格模式下签章前需人脸验证（packageCommon/pages/faceRecognition）
- **问诊聊天室**：从聊天室入口查看已开处方（timIM/pages/TUI-Chat）
