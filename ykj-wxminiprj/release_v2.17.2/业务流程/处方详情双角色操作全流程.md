# 处方详情双角色操作全流程

> 文档版本 v1.0 | 面向读者：产品、运营、客服

## 概述

处方详情页是医生和患者查看处方信息并执行后续操作的核心页面。医生端支持编辑、签字、代支付、发到患者微信、引用处方等操作；患者端支持确认领取、付款、查看订单详情等操作。页面根据处方状态和用户角色动态显示不同的底部操作按钮。

## 触发条件

- **前置条件**：处方已生成（用药建议已提交）
- **入口路径**：
  - 医生端：问诊室 → 处方记录 → 点击处方卡片
  - 患者端：消息通知 → 点击处方链接 / 我的处方列表 → 点击处方
  - 快速开方分享链接

## 步骤详解

### 第一步：页面加载

页面加载时自动获取处方详情，包含：
- 处方基本信息（处方单号、开具日期、处方类型）
- 就诊人信息（姓名、性别、年龄、科室）
- 诊断信息（西医诊断、中医病名、中医辨证）
- 药品信息（成药处方显示药品清单，中药处方显示饮片组成）
- 医生和药师信息
- 处方状态和订单状态

如该处方关联同一病历的多张处方（快速开方场景），顶部显示处方标签页（Tab）可切换：
- 标签显示"处方1"、"处方2"等
- 已支付的标签标记"已支付"，未支付的标记"未支付"

### 第二步：查看处方信息

页面按区块展示：
1. **处方头**：处方类型（成药处方/中药饮片）、处方单号、开具日期
2. **就诊人信息**：姓名、性别、年龄、科室（已支付处方显示"已支付"水印）
3. **诊断信息**：西医诊断、中医病名、中医辨证
4. **药品信息**：药品名称、规格、用量、数量
   - 中药处方显示加工工艺（如"代煎（水煎服）"）
   - 按重计费的中药处方显示"按重"标识
   - 患者未支付时：若系统配置了"未支付不可查看用量"，中药饮片的克数隐藏
5. **医嘱**：用药方法和注意事项
6. **医生和药师信息**
7. **查看电子处方**：审核通过且处方有效时显示入口，点击查看电子处方PDF
8. **提示信息**：显示处方说明（从系统配置获取）

### 第三步：医生端操作

医生端底部按钮根据处方状态动态变化：

#### 待审核/已撤回状态
- **编辑**按钮：点击进入处方编辑页，修改药品和用量后重新提交
- **指定处方接收人**：处方未指定患者时显示，弹窗选择发送给哪位患者微信
- **代支付**：处方未支付且在有效期内，医生可代患者付款
  - 点击 → 跳转药品订单页（代付模式）
  - 代付成功后自动弹出"指定接收人"弹窗
  - 也可发微信链接/二维码给患者

#### 审核中（待签字）状态
- **签字**按钮：点击触发CA电子签章
  - 支持医网信、源康健PaaS、放心签三种CA平台
  - 签字成功 → 处方状态变为审核通过，跳转成功页
  - 签字失败 → 提示"签字失败，请重新签字"

#### 审核通过/未支付状态
- **发到患者微信**：触发微信分享，将处方发送给患者
- **引用处方**：复制当前处方全部内容，创建新处方（商城问诊和机构处方不可引用）
- **返回问诊室**：如处方关联了问诊单，显示此按钮跳转IM聊天室

### 第四步：患者端操作

患者端底部按钮根据处方状态和支付情况动态变化：

#### 医生已代付、患者未领取
- **确认领取**按钮：患者确认领取处方
  - 点击 → 弹出确认处方弹窗（需勾选同意《互联网诊疗风险告知及知情同意书》）
  - 确认后 → 绑定会员 → 领取成功 → 跳转药品订单页

#### 审核通过、未支付、有效期内
- **确认领取处方并支付**（未绑定会员）或 **付款**（已绑定会员）
  - 点击"付款"前先检查：
    1. 是否已完成实名认证（未认证跳转订单页走认证流程）
    2. 是否为商城问诊（是则跳转商城订单页）
    3. 处方是否已失效（已失效弹窗提示）
    4. 是否已有订单号（有则直接跳转订单详情）
    5. 首次领取需确认《互联网诊疗风险告知及知情同意书》
  - 确认后跳转药品订单页

#### 已支付/有订单
- **订单详情**按钮：查看订单物流和状态

### 第五步：处方状态判断

页面通过多个状态字段综合判断处方当前阶段：

| 状态组合 | 含义 | 影响 |
|---------|------|------|
| createFlowStatus=20 + prescriptionStatus≠40 | 待处方医生审核 | 医生可编辑 |
| createFlowStatus=30 + prescriptionStatus=25 | 审核中 | 医生可签字 |
| createFlowStatus=30 + prescriptionStatus=30 | 审核通过 | 患者可支付 |
| createFlowStatus=40 或 prescriptionStatus=40 | 审核不通过 | 处方无效 |
| createFlowStatus=50 | 已失效 | 处方无效 |
| 有效期 < 当前时间 + flowStatus=10 | 超过有效期未支付 | 处方失效，不可支付 |
| createFlowStatus=20 或 31 | 待审核/已撤回 | 患者看不到药品信息 |

### 第六步：处方失效处理

处方在以下情况被判定为失效：
1. 药师审核不通过（prescriptionStatus=40）
2. 处方医生审核不通过（createFlowStatus=40）
3. 处方已失效（createFlowStatus=50）
4. 超过有效期且未支付（expirationTime < 当前时间 + flowStatus=10）

失效后：
- 患者端：不显示付款按钮，药品信息区域标记为失效
- 点击付款时弹出"处方单已失效，请联系医生重新为您开具处方"
- 医生端仍可正常查看和引用处方

## 异常处理

| 情况 | 系统处理方式 |
|------|------------|
| 处方已失效 | 弹窗提示"处方单已失效，请联系医生重新为您开具处方" |
| 审核不通过 | 处方标记失效，无法支付 |
| 获取签字ID失败 | 弹窗提示"获取处方签字ID失败，请联系管理员"，点击确认后返回 |
| CA签字失败 | 提示"签字失败，请重新签字"或显示重试按钮 |
| 患者未实名 | 隐藏直接付款按钮，引导走"确认领取"流程 |
| 就诊人已绑定其他手机号 | 提示冲突手机号，引导解绑 |
| 当前医生无药房授权 | 提示"当前医生没有该药房授权，请联系管理员"，快速续方不可用 |
| 问诊单号缺失 | 提示"问诊单号缺失"，快速续方不可用 |
| 处方未关联问诊单 | 提示"该处方未关联问诊单，不可返回问诊室" |
| 商城问诊处方 | 不可引用处方（引用按钮不显示） |
| 机构处方 | 不可引用处方（引用按钮不显示） |
