# 医生PC端扫码登录授权流程

> 文档版本 v1.0 | 面向读者：医生、运营人员

## 概述

当医生在PC端医生工作台打开登录页面时，系统展示二维码，医生使用手机小程序扫码后进入授权页面。系统验证医生身份（必须是医生角色），确认后完成PC端登录授权。

## 触发条件

- **前置条件**：医生已登录小程序，且在PC端打开了医生工作台登录页
- **入口路径**：使用微信"扫一扫"扫描PC端工作台二维码 → 小程序解析二维码参数并跳转到授权页面

## 步骤详解

### 第一步：扫码进入授权页面

医生使用微信扫描PC端二维码后，小程序自动跳转到登录授权页面。

页面展示：
- **系统Logo**（由后台配置决定，如互联网医院的品牌Logo）
- **系统标题**（由后台配置决定）
- 授权提示文字：「确认授权登录医生工作台吗?」
- 「确认登录」按钮

### 第二步：身份验证

系统后台自动执行以下检查：

1. **检查登录状态**：确认医生在小程序中已登录。如果未登录，先跳转到登录页面，登录后自动回到授权页面。
2. **检查医生角色**：在用户的所有角色中查找是否存在 `doctor` 角色。
   - 如果有医生角色 → 显示"确认登录"按钮，允许继续
   - 如果没有医生角色 → 弹出提示：「您当前暂无医生角色」，只能点击"返回首页"

### 第三步：确认登录

医生点击「确认登录」按钮后，系统调用绑定登录码接口，将该二维码对应的code与当前医生账号绑定。成功后弹出提示「授权登录成功!」，点击「知道了」后退出小程序。

此时PC端医生工作台自动完成登录。

### 第四步：异常处理

- 如果扫描的二维码缺少场景参数（scene），页面提示「缺少二维码参数」，无法继续。
- 如果医生身份验证失败（非医生角色），弹出提示后只能返回小程序首页。

## 异常处理

| 情况 | 系统处理方式 |
|------|------------|
| 未登录状态下扫码 | 跳转到登录页面，登录后自动回到授权页 |
| 用户无医生角色 | 弹出"您当前暂无医生角色"，只能返回首页 |
| 二维码缺少参数 | 提示"缺少二维码参数" |
| 系统Logo未配置 | 不显示Logo图片 |
| 确认按钮快速重复点击 | 防抖处理，避免重复提交 |

## 关联功能

- PC端医生工作台：授权成功后PC端自动完成登录
- 小程序首页：无医生角色时跳转回首页
- 小程序登录页：未登录时先跳转登录
