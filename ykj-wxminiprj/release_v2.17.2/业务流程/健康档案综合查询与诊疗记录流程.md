# 健康档案综合查询与诊疗记录流程

> 文档版本 2026-07-02 | 面向读者：医生、患者

## 概述

健康档案页面是医生查看患者完整病历信息的综合入口。页面汇集了患者的个人信息、标签、慢病登记、诊疗记录、量表记录、随访计划、线下病历、检查报告等多维度健康数据，帮助医生全面了解患者情况。

## 触发条件

- **前置条件**：已登录，当前角色为医生或患者
- **入口路径**：患者管理 → 患者详情 → 病历信息（携带患者ID）

## 步骤详解

### 第一步：查看患者基本信息

进入页面后，顶部展示患者核心信息：
- **头像**：根据性别和年龄自动选择（6岁以下显示儿童头像，男性显示男士头像，女性显示女士头像）
- **姓名**、**性别**（男/女/未知）、**年龄**
- **监护人信息**（如有）：监护人姓名和关系（父亲/母亲/其他）

### 第二步：查看和编辑患者标签

患者信息下方显示患者标签区域：
- 展示该患者已有的标签列表（从服务端获取）
- 医生可以点击标签进行增删操作
- 修改后自动保存，保存失败弹窗提示"就诊人信息更新失败，请稍后重试"

### 第三步：诊疗记录入口（仅医生角色）

医生角色可以看到"诊疗记录"快捷入口区，显示以下功能入口（按功能开关和角色动态显示）：

| 入口 | 功能 | 显示条件 | 排序 |
|------|------|---------|------|
| 线下病历 | 查看患者线下门诊病历 | 医生角色 | 1 |
| 量表记录 | 查看患者历史量表测评记录 | 医生角色 + scaleEnabled开关 | 2 |
| 随访计划 | 管理患者随访计划 | 医生角色 + followupEnabled开关 | 3 |
| 查报告 | 查看患者检查检验报告 | 医生角色 + examineReportEnabled开关 | 4 |

点击入口跳转到对应功能页面，均携带患者ID。

### 第四步：已登记慢病病种

如果同时满足以下条件，展示慢病病种区域：
- 系统开启了医保支付功能（medicalInsurancePay）
- 患者有已登记的慢病信息（patientChronicVos不为空）

每个慢病病种以蓝色标签形式展示（如"高血压"、"糖尿病"）。

### 第五步：浏览病历记录

病历记录区域采用分页加载（每页5条），支持：
- **下拉滚动加载更多**：滚动到底部自动加载下一页（isLastPage为true时停止加载）
- **处方分享到微信**：医生点击分享按钮可将处方发送给患者微信（标题格式："XX医生为「患者名」开具了处方，为不耽误用药请尽快支付"）
- **复诊按钮**（仅从患者管理进入时显示）：满足条件时显示复诊按钮（需系统开关 revisitEnabled 和医生开通复诊服务 serviceType=7 双重开启）

### 第六步：页面滚动与背景自适应

- 页面背景图高度随患者标签区域高度动态调整（updateImgHeight事件通知）
- 初始背景高度240像素，患者标签加载后自动增大

## 异常处理

| 情况 | 系统处理方式 |
|------|------------|
| 未传入患者ID参数 | 页面不加载数据，患者信息区域空白 |
| 患者信息加载失败 | 患者信息区域显示加载中状态 |
| 病历记录为空 | 病历记录区域不显示任何记录 |
| 标签更新失败 | 弹窗提示"就诊人信息更新失败，请稍后重试"，确认后重新加载患者信息 |
| isShowTag参数为false | 不显示患者标签区域 |
| isShowPatientInfo参数为false | 不显示患者基本信息区域 |
| 复诊功能开关未开启 | 不显示复诊按钮 |
| 当前角色不是医生 | 诊疗记录快捷入口不显示 |

## 关联功能

- 跳转到**线下病历**：healthRecords/offLineRecords/index（携带患者ID）
- 跳转到**量表记录**：scale/scaleRecord/index（携带患者ID）
- 跳转到**随访计划**：followUp/planList（携带患者ID）
- 跳转到**查报告**：report/index（携带患者ID）
- 跳转到**处方详情**：drugsuggest/detail（微信分享目标页，携带门诊号+患者角色）
