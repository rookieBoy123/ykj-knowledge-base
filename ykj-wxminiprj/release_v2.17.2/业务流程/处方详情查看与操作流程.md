# 处方详情查看与操作流程

## 触发条件
- 患者：收到医生开具的处方后，从问诊聊天消息、用药建议列表或推送通知进入
- 医生：从问诊记录、处方记录、患者病历等入口查看自己开具的处方

## 前置条件
- 患者和医生均已登录
- 处方已生成（用药建议单号 `outpatientNo` 有效）
- 支持多处方场景：一个用药建议包含多个子处方时，页面顶部以Tab切换查看

## 步骤详解

### 1. 进入处方详情页
用户从问诊消息中的「查看处方」链接、用药建议列表或快速续方页面点击处方卡片进入。页面加载时自动查询处方详情信息。

### 2. 查看处方基本信息
页面顶部展示：
- 处方单号和开具日期
- 处方类型标签（「成药处方」或「中药饮片」）
- 若有多张处方，顶部以Tab切换，每个Tab显示「处方1」「处方2」及支付状态（已支付/未支付）

### 3. 查看处方关键信息
向下滚动依次查看：
- **患者信息区**：姓名、年龄、性别、身份证号（脱敏显示）
- **诊断信息区**：临床表现、诊断结果、辨证信息
- **药品信息区**：
  - 成药处方：列出药品名称、规格、用量、天数
  - 中药饮片处方：列出每味药材的名称、剂量；根据后台配置可能隐藏未支付处方的剂量信息
- **医嘱信息**：医生填写的用药说明和建议
- **医生信息**：开具医生和审核药师姓名

### 4. 查看电子处方（PDF版）
处方审核通过后，页面底部出现「查看电子处方」入口。仅适用于西医处方，且中药处方如果后台配置了"未支付不可查看"或"不可查看"，则在支付前不显示此入口。

### 5. 底部操作按钮（根据角色和状态动态变化）

**患者端按钮（按优先级排列）：**

| 处方状态 | 底部按钮 | 说明 |
|---------|---------|------|
| 审核通过 + 未支付 + 有效期内 + 未绑定会员 | 「确认领取处方并支付」 | 先领取再支付 |
| 审核通过 + 未支付 + 有效期内 + 已绑定 + 有药房 | 「付款」 | 直接付款 |
| 已支付 + 有订单号 | 「订单详情」 | 查看已购药品订单 |
| 已失效（超时/审核不通过） | 无按钮 | 显示处方失效提示 |
| 处方撤回/草稿 | 无按钮 | 仅查看 |

**医生端按钮（按优先级排列）：**

| 处方状态 | 底部按钮 | 说明 |
|---------|---------|------|
| 待审核（用药建议状态为20或31）| 「编辑」 | 修改处方内容 |
| 审核中 + 有CA签字标识 | 「签字」 | 进行电子签章 |
| 已支付或已失效 | 「复用处方」 | 基于当前处方创建新处方 |

### 6. CA电子签章流程（医生）
点击「签字」按钮后：
1. 弹出签名组件（根据后台配置选择：医网信/源康健PaaS/放心签）
2. 页面显示「等待回调签字信息（10s）」的加载提示
3. 系统每1.5秒检查一次签名状态
4. 签名成功后自动刷新处方状态
5. 若签名失败，显示「签字失败，请重新签字」并可重试

### 7. 快速续方入口（医生）
如果医生是从「快速续方」进入处方详情，页面底部额外显示「快速续方」按钮。点击后跳转到患者选择页面，为该患者创建新的快速续方问诊单。

## 异常处理

| 异常场景 | 系统行为 |
|---------|---------|
| 处方已失效（审核不通过/已过期/超时未支付） | 弹窗提示「处方单已失效，请联系医生重新为您开具处方」 |
| 处方被撤回（草稿状态） | 药品信息区标注「处方已撤回」，不显示支付按钮 |
| 访问不存在的处方 | 自动跳转到「处方代支付」过渡页 |
| CA签名超时（约10秒未完成） | 强制结束等待并刷新页面，签名状态需手动查看 |
| CA签名失败 | 底部错误提示区显示「签字失败，请重新签字」+ 重试按钮 |

## 中药处方剂量显示规则
中药饮片的每味药材剂量是否可见取决于后台配置：
- `medicineUnitHide = 1`：不启用，始终显示
- `medicineUnitHide = 2`：未支付不可查看（患者端支付后可见）
- `medicineUnitHide = 3`：始终不可查看（不显示剂量）

## 关联功能
- 跳转到「药品订单页」→ 支付后查看订单物流
- 跳转到「编辑处方页」→ 医生修改处方
- 跳转到「患者选择页」→ 快速续方选患者
- 跳转到「问诊聊天室」→ 查看完整问诊上下文
- 分享处方给微信好友（患者可分享，标题为"XX医生为「患者名」开具了处方，为不耽误用药请尽快支付"）
