# 医生邀请二维码生成与分享流程

> 文档版本 v1.0 | 面向读者：医生用户

## 概述

医生端生成带有个人信息的邀请二维码海报，支持保存到手机相册或通过微信分享给患者，方便患者扫码后直接进入医生主页进行问诊或复诊。

## 触发条件

- 前置条件：医生已登录且有医生身份
- 入口路径：医生端"我的"页面 → 点击"邀请患者"入口 → 进入二维码页面

## 步骤详解

### 第一步：页面加载

页面加载时从本地存储获取医生信息（doctorInfo），包括：
- 医生姓名、职称
- 所在科室、医院名称
- 患者端二维码链接（patientQrCode）
- 头像图片

同时开始预下载所有需要的图片素材（二维码、背景图、头像、箭头图标）。

### 第二步：查看二维码海报

页面展示完整的邀请海报，包含：
- 顶部：医院背景图 + 医生头像（圆形，130×130rpx）
- 中部：医生姓名+职称、科室名称、医院名称（医院名超20字自动换行）
- 核心区：患者端二维码（380×380rpx）
- 底部：引导语"微信扫码，足不出户即可找我复诊开方"（支持换行，最多3行，每行20字）

### 第三步：保存到手机

点击"保存到手机" → 先请求相册写入权限 → 将二维码、背景图、头像、箭头等素材通过Canvas绘制合成海报图片 → 保存到系统相册

- 允许权限：提示"保存成功"
- 拒绝权限：提示"您没有授权，无法保存到相册"

### 第四步：分享名片

点击"分享名片" → 先请求相册写入权限 → 同样通过Canvas生成海报图片 → 调用微信原生分享图片功能（showShareImageMenu）

- 允许权限：图片生成后拉起微信分享菜单 → 成功提示"分享成功"
- 拒绝权限：提示"您没有授权，无法生成名片"

### 第五步：空状态处理

如果医生没有配置患者端二维码（qrCodeSrc为空），页面显示"暂无医生二维码，请联系管理员"的空状态提示。

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 医生未配置二维码 | 显示空状态提示"暂无医生二维码，请联系管理员" |
| 用户未授权相册权限 | 提示"您没有授权，无法保存到相册"或"您没有授权，无法生成名片" |
| 医院名称过长（>20字） | 自动截断为"前20字..."并换行显示 |
| 引导语过长 | Canvas绘制时按每行20字自动换行，最多3行 |

## 关键规则

- **图片预下载**：进入页面时异步下载所有远程图片到本地，后续Canvas绘制时使用本地路径，避免跨域问题
- **Canvas离屏渲染**：Canvas放在视口外（left: 999999px），不影响页面显示
- **页面卸载清理**：离开页面时清空imageMap缓存，释放内存
- **引导语可配置**：引导语支持后台配置（cmConfig['wx-share']['qr-description']），默认文案为"微信扫码，足不出户即可找我复诊开方"

## 关联功能

- 患者扫码 → 打开小程序进入医生主页
- 医生信息管理（头像、二维码等由管理员在后台配置）
