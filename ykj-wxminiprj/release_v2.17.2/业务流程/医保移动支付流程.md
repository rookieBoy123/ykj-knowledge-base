# 医保移动支付流程

## 触发条件
患者在处方药品下单页面点击「付款」后，系统判断订单需要走医保支付时进入。

## 前置条件
- 医生已开具处方，患者已进入药品订单页面
- 平台后台已配置医保参数（医保小程序appId、渠道编码、区域编码等）
- 就诊人已完成实名认证
- 就诊人已完成医保定点

## 步骤详解

### 1. 获取医保授权码
进入医保支付页面前，系统先跳转到医保小程序获取授权码：
- 携带参数：`openType=getAuthCode`（授权类型）、`bizType`（业务类型）、`cityCode`（城市编码）等
- 用户在医保小程序中完成电子凭证身份授权
- 授权成功后，医保小程序返回`retCode=0`和`authCode`授权码
- 如果用户取消授权，返回`retCode=503608`，提示「您已取消医保电子凭证身份授权，医保移动支付已中止」

### 2. 进入医保结算页
获取授权码后跳转到医保结算确认页（`hisPrePay`），页面展示：

**顶部付款信息**：
- 收款方（医院名称）
- 费用总额

**医保结算明细**：
- 医保基金支付金额（统筹报销部分）
- 个人账户支付金额（医保卡余额支付部分）
- 其他抵扣金额
- 现金支付金额（自付部分）

**个人账户支付开关**（如支持个人账户）：
- 「使用」：优先使用医保个人账户余额支付
- 「不使用」：完全自费支付

**查看明细入口**：
- 点击「查看明细」弹出半屏面板
- 分组展示每项处方的药品明细、单价、数量
- 高亮标注关键金额项

### 3. 选择个人账户支付方式
- 切换「使用/不使用个人账户」会重新发起医保预结算请求
- 每次切换后需要重新获取医保授权码
- 如果获取授权码失败，弹出提示并可重新尝试

### 4. 发起支付
点击「去支付」按钮：
1. 系统调用支付接口获取微信支付参数（appId、path）
2. 显示「准备拉起支付」加载动画
3. 跳转到微信医保支付小程序完成支付
4. 支付完成后返回当前页面

### 5. 支付结果确认
支付返回后系统主动查询订单状态：
- **支付成功**：自动跳转到药品订单详情页
- **无法确认支付状态**：弹出选择弹窗
  - 「支付未成功」：关闭弹窗，停留在当前页面
  - 「支付已成功」：重新查询订单状态，成功则跳转

## 异常处理

| 异常场景 | 系统行为 |
|---------|---------|
| 缺少必要参数（orderNo/authCode/outpatientNo） | Toast提示「缺少必要参数」 |
| 医保信息查询失败 | 弹窗提示错误信息，确认后返回上一页 |
| 用户取消医保授权 | 弹窗提示「医保移动支付已中止」，确认后返回支付选择页 |
| 未获取到授权码 | 弹窗提示「未获取到授权码，请重新医保电子凭证身份授权」 |
| 医保参数配置缺失 | 弹窗提示「医保参数有误，是否重新获取？」 |
| 处方明细加载失败 | Toast提示错误信息 |
| 跳转医保小程序失败 | 标记`isToPayed=false`，不进入支付回调流程 |

### 医保自费混合支付
- 首次进入页面调用`insurancePrepay`接口获取医保结算信息
- 之后如果用户从医保小程序返回（onShow触发的场景值变化），系统自动查询支付状态
- 第二次及以后查询支付状态时使用`paymentStatus`接口（带`payMethod=3`）

## 关联功能
- 跳转到「药品订单详情页」→ 支付成功后查看订单物流
- 返回「支付选择页」→ 切换为微信自费支付
- 跳转到医保小程序 → 医保授权/支付
- 关联「HIS表单填写」→ 医保慢病复诊需要HIS登记

## 重要说明
- 医保支付仅支持：图文问诊、视频问诊、快速问诊、快速续方场景的处方
- 就诊人必须已完成实名认证才能使用医保
- 医保定点是使用统筹报销的前置条件，未定点时仅能自费
- 不同地区的医保小程序配置不同（正式版/体验版/开发版）
