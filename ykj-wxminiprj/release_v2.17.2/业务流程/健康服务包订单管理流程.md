# 健康服务包订单管理流程

> 文档版本：release/v2.17.2 | 面向读者：客服、运营

## 概述

健康服务包（也称特色疗法服务包）是平台推出的打包式健康服务产品，患者购买后享有专属权益。本流程覆盖患者对服务包订单的查看、支付、取消等管理操作，以及订单详情中的核销码、退款等信息展示。

## 触发条件

患者已购买或正在购买健康服务包，需要查看订单状态或继续支付。

**前置条件**：
- 用户已登录小程序
- 已通过服务包购买流程生成了订单

## 步骤详解

### 第一步：查看订单列表

进入服务包订单列表页（healthServicePackage/order/index），顶部有6个状态选项卡：

| 选项卡 | 对应状态码 | 说明 |
|--------|----------|------|
| 全部 | 空 | 不过滤，显示所有订单 |
| 待支付 | flowStatus=10 | 已下单但未付款 |
| 已付款 | flowStatus=20 | 已付款待核销 |
| 已完成 | flowStatus=60 | 已核销，权益生效 |
| 已取消 | flowStatus=70 | 订单已取消 |
| 已退款 | flowStatus=89,90 | 退款中+退款成功 |

点击选项卡切换筛选，每页10条，滚动到底部自动加载下一页。

每个订单卡片展示：
- **订单编号**：带订单图标，左侧显示
- **状态标签**：右上角彩色文字（主题色/退款中为红色），包括：待支付、已付款、已完成、已取消、退款中、退款成功
- **服务包封面图**：160px正方形，若无封面显示默认占位图
- **服务包名称**（加粗大号）
- **包含项目摘要**：灰色小字一行
- **订单金额**：右对齐，¥符号小号+金额大号加粗

**底部操作按钮**（根据状态动态显示）：
| 状态 | 按钮组合 |
|------|---------|
| 待支付(10) | 「取消订单」+「查看详情」+「去支付」 |
| 已付款(20) | 「查看详情」 |
| 已完成(60) | 「查看详情」 |
| 已取消(70) | 「查看详情」 |
| 退款中(89) | 「查看详情」 |
| 退款成功(90) | 「查看详情」 |

### 第二步：查看订单详情

点击「查看详情」进入订单详情页（healthServicePackage/order/details）。

**A. 状态区**（顶部）
- 待支付：显示倒计时「剩余支付时间  MM:SS，超时订单将自动取消」（红色文字）
- 已取消：显示取消原因（红色文字，`cancelRemark`字段）
- 已付款：显示核销码（红色文字，`verifyCode`字段）
- 各状态配文字说明：
  - 已付款：「付款成功，已生成核销码：XXXX」
  - 已完成：「订单已核销，专属权益已生效」
  - 退款中：「订单退款中，预计1~3日到账。」
  - 退款成功：「退款成功，订单已关闭。」

**B. 服务包信息**
- 封面图（140px）+ 名称 + 价格
- 「包含的服务项目」区域：特殊标题背景 + 逐项列出包含的具体服务项目

**C. 联系客服**
底部显示客服图标 + 「联系客服处理」按钮，点击拉起微信客服。

**D. 订单信息**
展示以下字段，不同状态下显示的内容不同：

| 字段 | 显示条件 |
|------|---------|
| 订单编号 | 始终显示，右侧有「复制」按钮 |
| 下单时间 | 始终显示 |
| 订单金额 | 始终显示（红色） |
| 支付时间 | 非待支付(10)且非已取消(70)时显示 |
| 支付方式 | 非待支付(10)且非已取消(70)时显示 |
| 取消时间 | 已取消(70)时显示（`refund.cancelTime`） |
| 完成时间 | 已完成(60)时显示 |
| 退款方式 | 退款成功(90)时显示（`refund.refundType_enumName`） |
| 退款时间 | 退款成功(90)时显示（`refund.refundTime`） |

**E. 复制订单号**
点击订单编号右侧的「复制」按钮，将订单号复制到手机剪贴板，提示「复制成功」。

**F. 倒计时**
待支付状态下显示倒计时。倒计时归零后自动刷新订单详情（订单变为已取消状态）。

### 第三步：去支付

在待支付状态的订单上点击「去支付」，跳转到收银台（hisPay），订单类型为`healthServicePackage`。

### 第四步：取消订单

在待支付状态下可取消订单：
1. 点击「取消订单」弹出确认弹窗
2. 弹窗内容：「您确定要取消当前订单吗？取消后将无法恢复。」
3. 提供「取消」和「确定」两个按钮
4. 点击「确定」调用取消接口
5. 成功后提示「取消成功」，刷新页面

## 异常处理

| 情况 | 系统处理方式 |
|------|------------|
| 订单列表为空 | 显示「暂无记录」空状态 |
| 倒计时结束 | 自动刷新订单详情，状态变为已取消 |
| 复制失败 | 提示「复制失败~，请重试！」 |
| 服务包无封面图 | 显示默认占位图 |
| 取消订单接口失败 | 弹窗关闭，数据不变 |

## 关联功能

- ← **服务包详情页**（healthServicePackage/list/details）：了解服务包内容后购买进入
- → **收银台**（hisPay）：去支付时跳转
- → **专属权益页**（healthServicePackage/memberBenefits）：支付完成核销后查看权益
- → **微信客服**：点击「联系客服处理」

## 特殊说明

**退款中与退款成功的区别**：退款中(89)表示退款已发起但资金尚未到账（预计1~3日），退款成功(90)表示退款已完成。退款中的状态标签为红色以引起注意。

**核销码**：支付完成后生成的核销码是一个字符串标识，到院时出示给工作人员。只有当订单已付款(20)时才显示。

**取消后不可恢复**：取消订单操作不可逆，弹窗中明确提示「取消后将无法恢复」。
