# PC医生工作台授权登录流程

> 文档版本 v1.0 | 面向读者：医生用户

## 概述

当医生在PC端医生工作台扫码登录时，小程序端弹出授权确认页面，医生确认后将PC工作台绑定登录。

## 触发条件

- 前置条件：用户在PC医生工作台点击"扫码登录"，生成了登录二维码
- 入口路径：微信扫描PC工作台的登录二维码 → 跳转到小程序授权页

## 步骤详解

### 第一步：扫码进入授权页

用户扫描PC工作台登录二维码 → 小程序打开登录授权页面 → 页面显示系统Logo和"确认授权登录医生工作台吗?"提示 → 显示"确认登录"按钮

### 第二步：身份校验

点击"确认登录"后，系统检查当前用户身份：
- **有医生角色**：调用bindLoginCode接口绑定二维码code → 提示"授权登录成功!" → 退出小程序（PC端自动登录）
- **无医生角色**：弹出提示"您当前暂无医生角色" → 点击"返回首页"跳转到小程序首页

### 第三步：未登录处理

若扫码时用户未登录小程序：
- 先跳转到登录页
- 登录成功后自动回到授权页继续流程

### 第四步：完成授权

授权成功后弹出确认框 → 点击"知道了" → 退出小程序 → PC端自动完成登录

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 缺少二维码参数（scene） | 提示"缺少二维码参数" |
| 当前用户无医生角色 | 弹窗提示"您当前暂无医生角色"，点击返回首页 |
| 用户未登录 | 跳转登录页，登录后自动回授权页 |
| 系统配置未加载 | 自动初始化获取cmConfig |

## 关键规则

- **角色校验**：只有拥有"doctor"角色的用户才能授权登录PC工作台，其他角色（如患者）会被拦截
- **二维码绑定**：二维码code与用户登录态绑定，绑定后PC端自动完成登录
- **防抖处理**：确认登录按钮做了防抖处理，避免重复请求
- **系统配置**：页面标题和Logo根据后台配置动态显示（cmConfig.system.title / adminLogo）

## 关联功能

- PC医生工作台登录
- 小程序登录页
- 首页（无医生角色时返回）
