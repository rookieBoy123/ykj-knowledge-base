# 随访任务详情查看与小结填写流程

> 文档版本 | 面向读者：医生、患者

## 概述

医生和患者都可以查看某个随访任务的详细内容，了解三类内容（患教资料、提醒、问卷）的完成状态。医生在患者完成随访后，还可以填写随访小结，记录随访结果。

## 触发条件

- **前置条件**：医生已为患者创建随访任务，任务ID有效
- **入口路径**：
  - 医生端：随访计划详情 → 点击某次随访任务
  - 患者端：我的随访列表 → 点击某次随访任务

## 步骤详解

### 第一步：查看患者信息（角色差异化）

**医生角色**：页面顶部显示患者信息卡片（PatientIntro组件），包含患者姓名、性别、年龄等基本信息

**患者角色**：页面顶部显示医生信息卡片，包含医生头像、姓名、科室、职称，帮助患者了解随访来自哪位医生

### 第二步：查看随访任务内容

页面主体展示三类随访内容，每类独立一个区域：

**1. 患教资料**
- 图标：课程图标 + "患教资料"文字
- 状态标签：全部完成显示蓝色"完成"，有未完成显示红色"待办"
- 每一条患教资料的标题文字（可两行截断）
- 右侧有蓝色"详情"按钮（医生端固定蓝色，患者端随主题色变化）
- 点击"详情"跳转到患教内容详情页

**2. 提醒**
- 图标：提醒图标 + "提醒"文字
- 状态标签：与患教同理
- 仅展示提醒标题，无详情按钮（患者端查看本页时，系统自动将所有未完成的提醒标记为"已完成"）

**3. 问卷**
- 图标：问卷图标 + "问卷"文字
- 状态标签：与上述同理
- 右侧有"详情"按钮：
  - **患者端**：点击进入问卷答题页
  - **医生端**：
    - 未完成：点击查看问卷但不可填写（医生不能代替患者答题）
    - 已完成：点击查看患者填写的答案

### 第三步：填写随访小结（仅医生）

当随访任务状态为"已完成"（status=2）时，医生可填写随访小结：

- 页面底部显示白色卡片区域：
  - **未填写时**：展示蓝色虚线按钮"填写小结"，左侧有编辑图标
  - **已填写时**：展示总结时间（summaryTime）和总结内容文本，右对齐显示
- 点击"填写小结"弹出底部面板（SummaryPop）：
  - 标题："填写随访小结"
  - 多行文本输入框，可输入总结内容
  - 点击"确认"调用保存接口，成功后页面自动刷新

### 第四步：患者端信息展示

患者端额外显示：
- 随访时间（sendingTime）：在页面底部白色卡片中展示
- 三种内容类型的完成判定逻辑与医生端一致

## 状态说明

| 状态值 | 状态名 | 说明 |
|--------|--------|------|
| 0 | 待随访 | 随访计划已安排但尚未发送 |
| 1 | 已随访 | 随访内容已发送给患者 |
| 2 | 已完成 | 患者已完成所有随访内容 |
| 3 | 已取消 | 随访计划被取消 |

## 完成状态判定规则

三类内容各自的"完成"标记逻辑相同：只要该类型**所有条目的status都为true**，才显示"完成"（蓝色）；任意一条为false则显示"待办"（红色）。如果该类型条目为空，不显示完成状态标签。

## 异常处理

| 情况 | 系统处理方式 |
|------|------------|
| 缺少taskId参数 | 页面不发起请求，不显示任何内容 |
| 订阅消息入口（分享卡片进入） | 通过`isSubscribeEntry`判断，静默跳过页面加载 |
| 医生填写小结内容为空 | 接口层面不校验，允许上传空字符串 |
| 患者查看时提醒自动标记完成 | 页面加载后自动调用`changeRemindStatus`，批量将未完成提醒标记完成 |
| 切换角色查看同一任务 | 通过`currentRoleCode`判断显示不同的顶部信息区和操作按钮 |
