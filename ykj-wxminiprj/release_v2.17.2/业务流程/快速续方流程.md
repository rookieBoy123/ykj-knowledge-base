# 快速续方流程

## 触发条件
- 患者有线上或线下的有效处方，且病情稳定，正在按方服药
- 从首页的「快速续方」入口、医生处方详情页的「快速续方」按钮或推送通知进入

## 前置条件
- 患者已登录（患者角色）
- 历史处方列表需要至少一张有效处方（近6个月内）

## 步骤详解

### 1. 进入快速续方页
页面顶部展示说明文字：「快速续方适用有线上或线下有效处方，正按方吃药并且病情稳定的患者」

### 2. 查看处方列表
页面展示「处方列表」区域，标题下方注明「仅展现最近6个月的有效处方」。
- 每张处方卡片展示：开具时间、处方类型、医生信息、诊断信息
- 每张处方卡片有两个操作按钮：「查看详情」和「快速续方」
- 列表支持上拉加载更多（每次5条）

### 3. 上传处方续方（线下处方）
如果没有线上历史处方，患者可以点击「上传处方续方」：
1. 跳转到「用药登记」页面（快速问诊模式，serviceType=4）
2. 上传线下处方照片
3. 填写药品需求信息
4. 提交后系统为患者匹配合适的医生进行问诊开方

### 4. 点击「快速续方」
在处方卡片上点击「快速续方」按钮：

#### 4.1 检查医生是否开通快速续方
- 系统查询医生是否已开启快速续方服务
- **未开通**：
  - 普通患者：弹窗「原处方医生暂不方便续方，是否为您匹配新的医生？」→ 「取消」或「匹配新医生」
  - 特殊患者（系统配置`specialEnabled`开启且当前患者标记为特殊患者）：弹窗「医生未开通快速续方服务，您可以向医生发起问诊咨询购药。」→ 「取消」或「去问诊」

#### 4.2 检查是否有未完成问诊
- 查询当前患者与医生之间是否存在未完成的快速续方问诊（serviceType=5）
- **有未支付订单**：弹窗「您有未支付问诊单，是否前往支付？」→ 「取消」或「去支付」
- **有进行中问诊**：弹窗「您有正在进行的问诊，是否前往？」→ 「取消」或「前往问诊」

#### 4.3 匹配医生
- 系统判断是否需要匹配新医生（原始处方医生可能不在线或不接诊）
- **需要匹配**：弹窗「您的医生暂不在线，是否为您匹配新的医生？」→ 「取消」或「匹配新医生」
  - 点击「匹配新医生」后，系统根据药房信息匹配新医生
  - 匹配成功后跳转下单页面
  - 匹配失败弹窗「暂无医生在线，请稍后再试」
- **不需要匹配**：直接跳转下单页面

### 5. 匹配新医生
当原处方医生不在线时：
1. 系统根据处方中的药房ID（`pharmacyId`）匹配新的在线医生
2. 匹配成功后保存快速续方参数（`serviceType: 5`，关联续方处方号 `continuedPrescriptionNo`）
3. 跳转到下单确认页面（`createOrder`），自动填充医生、患者、服务类型信息

### 6. 下单并完成续方
进入下单页面后流程与普通问诊下单一致：确认就诊人 → 填写病情 → 选择支付方式 → 支付 → 等待医生接诊 → 续方开药。

## 异常处理

| 异常场景 | 系统行为 |
|---------|---------|
| 无历史处方 | 显示空状态「暂无处方记录」，提示「可选择上传线下处方进行续方」 |
| 医生未开通快速续方 | 弹窗提示，可选择去问诊或匹配新医生 |
| 匹配新医生失败 | 弹窗「暂无医生在线，请稍后再试」→ 确认返回 |
| 有未支付订单 | 弹窗提示跳转支付 |
| 有进行中问诊 | 弹窗提示前往问诊 |

## 关联功能
- 跳转到「处方详情页」→ 查看处方完整内容
- 跳转到「下单确认页」→ 创建续方问诊单
- 跳转到「问诊聊天室」→ 继续已有问诊
- 跳转到「订单详情页」→ 支付未完成的订单
- 跳转到「用药登记页」→ 上传线下处方填写需求
- 跳转到「医生详情页」→ 特殊患者去问诊

## 重要说明
- 快速续方（serviceType=5）属于一种特殊的问诊服务类型，需要医生在诊费设置中单独开启
- 快速续方依赖图文问诊，医生开启快速续方前必须先开启图文问诊
- 处方列表默认展示近6个月的有效处方
- 分享标题由后台配置（`wx-share.fastcontinuation`）
