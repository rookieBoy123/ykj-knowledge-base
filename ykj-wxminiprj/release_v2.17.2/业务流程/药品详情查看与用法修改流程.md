# 药品详情查看与用法修改流程

> 文档版本 v1.0 | 面向读者：医生

## 概述

医生在开处方过程中，查看药品的详细信息（价格、规格、说明等），并在添加药品时修改用法用量。页面支持两种模式：纯查看模式和可修改模式。

## 触发条件

- **前置条件**：医生在处方/药方编辑页面点击药品进入详情
- **入口路径**：
  - 纯查看：处方详情 → 点击药品 → 药品详情页
  - 可修改：药方模板编辑 → 点击需要修改的药品 → 药品详情页（底部出现"选择药品"按钮）

## 页面结构

- **药品图片轮播**：支持多张药品图片的自动轮播，带圆点指示器，无图片时显示占位图
- **药品价格区**：价格（¥符号+大号数字） + 库存信息（如"库存100粒"）
- **药品基本信息**：通用名（大字加粗）、商品名（仅非严格模式显示）、厂家或产地（仅非严格模式显示）、药品规格、药品剂型
- **药品说明区**：富文本格式的药品详细说明，无说明时显示"药品说明正在完善中…"占位
- **药品编码**：右上角显示药品编码

## 步骤详解

### 第一步：进入药品详情页

页面接收参数：药品ID和药品类型（type=1 成药 / type=2 中药）。

页面加载后自动根据类型调用对应接口查询药品详情：
- type=1：调用成药查询接口（cyDrugDetails）
- type=2：调用中药查询接口（zyDrugDetails）

### 第二步：浏览药品信息

**库存显示**：
- 库存为 -1 时，不显示库存信息
- 库存不为 -1 时，显示格式"（库存XX单位）"
- 如果药品开启了拆零销售（retailFlag），单位显示制剂单位；否则显示包装单位

**严格模式隐藏信息**：在严格模式下，商品名和厂家/产地信息不显示，减少对医生选药的商业引导。

**药品说明**：
- 如果有说明信息，以富文本HTML格式渲染
- 如果没有说明，显示空状态图片和"药品说明正在完善中…"文字

### 第三步：修改药品用法用量（可选）

仅在"可修改模式"下可用（入口来自药方模板的药品修改场景）。底部出现蓝色"选择药品"按钮。

点击"选择药品"后弹出修改面板，医生可以调整：

1. **用法**（服用方式）：从可选列表中选择（如"口服""外用"等），数据来自字典
2. **每日次数**：加减按钮调整，最小为1次
3. **用量**：每次使用数量，加减按钮或直接输入调整，最小为1
4. **每次用量单位**：从可选列表中选择（如"粒""片"等）
5. **数量**：药品总量，加减按钮调整
6. **天数**：用药天数，加减按钮调整

修改后点击确认，修改数据通过页面事件通道（eventChannel）传回上一个页面，自动返回。

## 异常处理

| 情况 | 系统处理方式 |
|:---|:---|
| 药品ID缺失 | 页面不发起查询请求 |
| 药品类型缺失 | 无法确定调用哪个查询接口 |
| 药品无图片 | 显示占位图（drug-nodrugBanner） |
| 药品无说明 | 显示"药品说明正在完善中…" + 空状态图片 |
| 药品接口返回失败 | 页面无内容，静默失败 |
| 用法/每日次数/用量低于1 | 减号按钮无响应，不弹提示 |
| 严格模式下 | 商品名、厂家/产地信息不显示 |

## 关联功能

- 处方开方流程：医生选药→查看药品详情→确认添加
- 药方模板编辑：模板中添加的药品可跳转修改用法用量
- 我的药房：药房药品管理和模板管理
- 药方模板组件（popup.vue）：药品修改弹窗的实际组件
