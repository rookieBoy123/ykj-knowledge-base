# 订单模板药品选择与添加流程

> 文档版本：release/v2.17.2 | 面向读者：医生、运营

## 概述

医生在创建订单模板（成药处方模板的药品清单）时，需要从药房的药品库中选择药品并设置用法用量。本流程描述医生搜索、选择、配置药品的完整操作。

## 触发条件

- **前置条件**：医生已登录，需要有指定的药房ID（pharmacyId参数）
- **入口路径**：订单模板编辑页→点击"添加药品"

## 步骤详解

### 第一步：搜索药品

进入"添加药品"页面后，顶部有搜索框：
- 支持输入药品名称或拼音首字母搜索
- 输入后即时触发搜索（每输入一个字符都会重新搜索）
- 搜索时自动重置到第1页

### 第二步：浏览药品列表

药品列表每页显示10条。每条药品卡片显示：
- **药品基本信息**：由子组件`drug-item`渲染（含药品图片、名称、规格、价格等）
- **操作按钮**（根据药品状态动态显示）：
  - **选择**（蓝色按钮）：药品可用时显示
  - **缺货**（灰色按钮）：药房有该药但库存为0时显示
  - **禁用**（灰色不可点击）：药品状态为"未启用"时显示

**"选择"按钮显示的条件**（需要同时满足）：
- 药品状态不是"禁用"（status ≠ 0）
- 药房库存条件（以下任一满足）：
  - pharmacyStock=0（表示未启用库存管理）
  - pharmacyStock=1 且 drugsStock>0（有库存）
  - pharmacyStock=1 且 drugsStock为null/undefined（库存字段未配置）
  - pharmacyStock=1 且 drugsStock=-1（库存无限）

药品价格为0的会自动设置为0显示。

### 第三步：选择药品并配置

点击"选择"按钮后，底部弹出配置面板，可以设置：

- **用药方式**：从字典`ywsytjbm`中选取（如口服、外用等）
- **使用频次**：从字典`useFrequencies`中选取（如每日1次、每日2次等），对应周期单位（日/周）
- **每次用量**：数字输入
- **用量单位**：从字典`mcyl`中选取（如片、粒、支）
- **药品数量**：默认1，支持增减
- **使用天数**：默认建议天数或1天

配置面板支持数量增减按钮和使用天数增减按钮，最低为1。

### 第四步：确认添加药品

点击配置面板的"添加"按钮：

1. 系统检查是否已选择过该药品：
   - 如果已存在：更新该药品的配置数据（数量、用法、用量等）
   - 如果不存在：将药品加入已选列表
2. 如果药品数量为0，会从已选列表中移除
3. 已选列表中的药品状态会与左侧搜索列表同步更新
4. 通过事件通道将已选列表实时同步回订单模板编辑页

### 第五步：完成选择

页面底部有固定的"选好了"按钮，点击返回上一页。所有选中药物的数据已经通过事件通道传递。

## 关键限制和规则

| 限制 | 说明 |
|------|------|
| 最多5种药品 | 选择第6种药品时提示"最多只能添加五个药品"，配置弹窗不会打开 |
| 药品库存为0 | 显示"缺货"按钮，不可选择 |
| 药品已禁用 | 显示灰色"禁用"按钮，不可选择 |
| 每页10条 | 滚动到底部自动加载下一页 |

## 历史搜索记录

系统会保存用户最近搜索的药品记录：
- 最多保存15条
- 新搜索的药品放到最前面
- 重复搜索会更新位置
- 存储在本地缓存`historySearchYPSS`中
- 目前历史搜索的UI已被注释掉，实际不展示

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 缺少pharmacyId参数 | 提示"缺少关键id"，无法加载数据 |
| 药品列表为空 | 显示"暂无药品"空状态 |
| 超过5种药品 | 提示"最多只能添加五个药品" |
| 已加载全部数据 | 列表底部显示"-已显示全部-" |
| 上传的药品图片加载失败 | 显示默认药品图片 |

## 关联功能

- 订单模板编辑页（addZy-template.vue / addZy.vue）：药品选择的目标页面
- 药品详情页（drugDetail/index.vue）：点击药品卡片跳转，支持修改药品参数
- 订单模板详情（ordertemplate/druglist-template.vue）：查看已配置的模板
