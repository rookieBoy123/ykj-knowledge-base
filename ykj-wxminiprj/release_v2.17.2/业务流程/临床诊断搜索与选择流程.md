# 临床诊断搜索与选择流程

> 文档版本 2026-07-02 | 面向读者：所有用户

## 概述

医生在处理用药需求、开具处方时，需要为患者添加临床诊断。本流程描述医生如何通过关键词搜索、从搜索结果中选择并使用ICD-10标准诊断编码，完成诊断信息的录入。

## 触发条件

- 医生进入「用药需求」页面，看到患者已提交的用药需求
- 医生需要为处方填写临床诊断
- 医生角色且该用药需求尚未开方（已开方后只能查看不能修改）

## 步骤详解

### 第一步：查看当前诊断

进入用药需求页面后，在「临床诊断」区域可以看到当前已有的诊断列表。每个诊断以标签形式显示，可以逐个删除。最多可添加5个诊断。

如果用药需求已经处理完成（已开方），诊断区域变为只读状态，灰色底标签显示已选的诊断名称。

### 第二步：打开诊断搜索弹窗

点击「添加诊断」按钮，从屏幕底部弹出搜索窗口。窗口包含：
- 顶部标题「添加诊断」和关闭按钮
- 搜索输入框，提示文字为「请输入疾病名称查询」
- 搜索历史记录（按最近使用排序）
- 搜索结果列表

### 第三步：搜索诊断

在搜索框中输入疾病名称关键词（如"高血压""糖尿病"等），系统调用ICD-10标准诊断库进行模糊匹配。搜索结果每页10条，支持上拉加载更多。

搜索结果显示内容包括：
- 疾病标准名称（如"原发性高血压"）
- 搜索关键词高亮（蓝色标记）
- 点击任一结果即可添加到诊断列表

### 第四步：添加诊断

点击某个诊断结果后：
1. 系统自动检查该诊断是否已在列表中（按诊断编码code去重）
2. 如已存在则不重复添加
3. 如不存在则添加到当前诊断列表
4. 搜索框自动清空、弹窗自动关闭
5. 返回主页面，诊断标签列表更新

### 第五步：删除诊断

在诊断标签上点击删除按钮（×），即可移除该诊断。删除后不影响其他已选诊断。

> 注意：诊断选择完成后，医生还需要填写药品的用法用量、医嘱等信息，才能点击「确认开方」提交。

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 搜索结果为空 | 列表显示空白，提示无匹配结果 |
| 搜索时网络异常 | 显示加载失败提示，可重试 |
| 添加重复诊断 | 自动忽略，不重复添加也不提示 |
| 诊断数量已达上限（5个） | 「添加诊断」按钮仍可点击，但搜索结果被限制不超过5个 |
| 已开方的用药需求 | 诊断区域变为灰色只读状态，无法新增或删除 |
| 非医生角色查看 | 诊断区域为只读状态 |

## 关联功能

- 跳转来源：医生端「用药需求」页面
- 诊断数据来源：ICD-10标准诊断库（版本ICD-10）
- 诊断用途：随处方提交至服务端，用于电子病历和处方签章
