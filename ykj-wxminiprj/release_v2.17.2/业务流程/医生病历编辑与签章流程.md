# 医生病历编辑与签章流程

> 文档版本：v1.0 | 面向读者：医生、平台运营

## 概述

医生在接诊后为患者编写电子病历，包含病史采集、诊断录入、治疗意见和病历资料上传，最终通过CA电子签章或手写签名确认病历的完整流程。

## 触发条件

- **前置条件**：医生已接诊（flowStatus=20，问诊中），有激活的问诊单号（inquiryNo）
- **入口路径**：
  - 从问诊聊天室→点击「写病历」按钮进入
  - 从处方页面→点击「病历」入口进入
  - 携带参数：inquiryNo（问诊单号）、medicalRecordId（已有病历ID，编辑模式）、serviceType（服务类型）、comeFrom（来源）

## 步骤详解

### 第一步：页面初始化（自动加载数据）

进入编辑病历页面后，系统按照优先级加载数据：

1. **先检查缓存**：如果本地有缓存数据（`medical_record_{inquiryNo}`），直接使用缓存填充所有表单区域，然后清除缓存
2. **如无缓存+有病历ID**：调用`getRecordById`接口获取已有病历数据，回填到患者信息、病史、诊断、治疗意见、病历资料各区
3. **如无缓存+无病历ID**：调用`getInterrogationInfoByNo`获取问诊卡数据，包括患者信息、病情描述（主诉）、就诊时间、慢病名称等，初始化空白病史和诊断区

医生类型（medicalType）决定展示模式：
- **中医医生（TCM）**：展示辨病+辨证两个诊断入口、治法输入框、中医特有的舌像/脉象病史
- **西医医生**：展示诊断入口、治疗意见输入框

### 第二步：查看患者信息和病史

页面上半部分展示患者信息和病史：

1. **患者信息卡片**：显示姓名、性别、年龄（≥1岁显示"岁"，<1岁显示"月龄"）、监护人信息（姓名+关系）、科室、就诊时间。右侧有「历史记录」按钮，可跳转查看患者的历史病历。

2. **慢病标识**：如果问诊关联了慢病，显示慢病名称标签。

3. **病史信息区**（可折叠）：包含7大类病史，每类有"有/无"开关：
   - 过敏史（allergy）
   - 现病史（medicalHistory）
   - 既往史（previousHistory）
   - 个人史（personalHistory）
   - 家族史（familyHistory）
   - 婚育史（obstetricalHistory）
   - 月经史（menstrualHistory）
   中医模式下额外显示：舌像（tongueCondition）、脉象（pulseCondition）、常见体征（commonSigns）

   每项病史可以填写详细描述文本。系统会自动查询患者最新病史数据并填充。

4. **主诉**：从问诊卡或已有病历中带入的病情描述，医生可修改。

### 第三步：填写诊断信息

诊断区包含三种类型，根据医生类型展示：

1. **西医诊断**（diagnosisType=1）：输入疾病名称，可关联标准诊断编码
2. **中医辨病**（diagnosisType=2）：输入中医辨病名称
3. **中医辨证**（diagnosisType=3）：输入中医辨证名称

已有关联处方的诊断可能被标记为"只读"（unalterable），不可修改。

非中医模式下如开启`diagnoseRequireEnabled`配置，西医诊断必须填写。

### 第四步：填写治疗意见/治法

- **西医模式**：输入"治疗意见"，占位提示包含拟采取的治疗方法、药物使用、手术治疗、康复措施等
- **中医模式**：输入"治法"，描述治疗方案和治疗方法

上海地区（adaptiveArea=shanghai）模式下，治疗意见/治法为必填项，为空保存时会提示。

### 第五步：上传病历资料

医生可上传病历相关图片资料（检查报告、影像等）。

### 第六步：AI辅助（可选）

如果医院开启了AI辅助功能（`paas.medicalAiSuggestion=true`）：

1. 页面右下角显示AI悬浮按钮
2. 首次使用时弹出引导"精准生成提问信息，提升诊断效率。快去试试吧！"
3. 点击AI按钮打开底部弹出的AI面板，AI自动提取病历内容供参考：
   - 更新主诉（疾病描述）
   - 补充诊断（去重后追加）
   - 补充病史信息（7大类病史的开关和描述）

### 第七步：保存并签章

底部按钮根据问诊状态动态变化：
- 问诊中（flowStatus=20）：显示「返回问诊」和「保存」两按钮
- 其他状态：显示「返回」和「保存」

点击「保存」后的流程分支：

**严格模式（strict=true）**：
1. 弹出"确认已完整编写病历？"弹窗
2. 选择「否，暂存草稿」→ 保存病历但不触发签名
3. 选择「是，确认签字」→ 保存病历后进入签章流程

**运营模式**：
直接保存后进入签章流程。

**签章流程**：
1. 调用保存接口（新增`addRecord`或更新`updateRecord`）
2. 后端返回`caUniqueId`：
   - **有caUniqueId**：需要CA电子签章 → 调用`getRecordSign`获取签章信息 → 根据平台类型调用签章组件（医网信/源康健PaaS/放心签）
   - **无caUniqueId**：严格模式→打开手写签名板；运营模式→自动用医生姓名签名

**保存成功后的弹窗**：
- 如果医生有开方权限且非快速问诊禁止开方：显示「开具处方」和「返回问诊」
- 否则：显示「返回」

选择「开具处方」会先校验CA认证状态，通过后跳转到处方编辑页。

### 第八步：离开页面时缓存

页面隐藏（onHide）或卸载（onUnload）时，自动将当前填写的内容缓存到本地存储（`medical_record_{inquiryNo}`），下次进入时自动恢复。

## 异常处理

| 情况 | 系统处理方式 |
|------|------------|
| 诊断区全部为空 | 保存时弹出提示"请添加临床诊断信息(诊断、辨病至少填写一项)" |
| 上海地区治法/治疗意见为空 | 保存时弹出提示"请输入治法"或"请输入治疗意见" |
| CA未认证（authStatus=1） | 弹出引导弹窗，提示同步资料到签章平台 |
| CA认证中（authStatus=2） | 弹出提示引导去完成认证 |
| CA认证失败（authStatus=4） | 弹出提示引导重新认证 |
| 医网信/源康健PaaS本地无证书 | 弹出"未完成CA电子签章不可接诊，请先完成签章"，引导下载证书 |
| 放心签无手写签名图片 | 引导跳转设置签章页面（signPassword） |
| 签章失败 | 显示"签字失败，请重试" |
| 病历缓存数据存在但medicalRecordId为空 | 使用缓存数据但需调接口查询诊断是否可修改 |

## 关联功能

- **开处方**：保存成功后可直接跳转到处方编辑页（addOrEditRp）
- **历史病历**：点击「历史记录」跳转健康档案列表页
- **问诊聊天室**：点击「返回问诊」回到IM聊天页
- **AI辅助**：问诊中的AI病历提取功能（medicalAiSuggestion）
