# 线下处方详情查看流程

> 文档版本 2026-07-05 | 面向读者：患者/医生 | 小程序路径：健康档案 → 线下就诊记录 → 处方详情

## 概述

患者或医生通过小程序查看线下医院开具的处方详细信息。处方数据来自医院的HIS系统，通过接口同步到平台。该页面为纯查看模式，不可编辑。支持查看成药处方和中药处方两种类型的处方，以及电子处方的原件预览。

## 触发条件

- **前置条件**：患者已在目标医院有线下就诊记录，且HIS系统已将处方数据同步到平台
- **入口路径**：
  - 患者端：消息 → 健康档案 → 线下就诊记录 → 点击某条记录的处方详情
  - 医生端：问诊聊天室 → 病历卡片 → 点击线下处方
- **页面参数**：`outpatientNo`（处方单号），由上一页通过URL参数传递

## 步骤详解

### 第一步：进入页面

页面加载时，根据URL参数中的「处方单号」调用HIS接口查询处方详情。页面顶部有渐变蓝色背景和导航栏（标题在滚动后吸顶显示）。

### 第二步：查看处方基本信息

页面顶部卡片区展示处方核心信息：

- **标题**：处方详情，右上角标签区分处方类型——
  - 「成药处方」（处方种类为5，即西药/成药）
  - 「中药处方」（处方种类为4，即中草药处方）
- **处方单号**：由医院系统生成的唯一编号
- **开具日期**：处方在医院的开具时间

### 第三步：查看患者信息

标题下方显示患者的基本信息行：

- **姓名**：加粗显示
- **性别**：男/女（根据性别编号1=男、2=女自动转换）
- **年龄**：以「X岁」格式显示
- **科室**：开具处方的医院科室名称

### 第四步：查看临床诊断

显示「临床诊断」区域，包含：
- **诊断**标签：医生给出的诊断结论文字

### 第五步：查看处方药品（RP区域）

根据处方类型展示不同内容：

**成药/西药处方（type=5）：**
- 每味药品独立一行，用灰色分隔线分隔
- 药品名称 + 规格（如无规格则不显示）
- 数量 + 单位
- 剂型（如无剂型则显示「其他」）
- 规格
- 用法用量：「用法，频次，每次用量+单位」
- 使用天数（如有）：底部显示「使用X天」

**中药处方（type=4）：**
- 所有药材以三列网格展示，每列显示：药材名称 + 数量 + 单位
- 底部汇总：
  - 用法说明（如「水煎服」）
  - 用药频次
  - 总剂数：「共X剂」

### 第六步：查看用药医嘱

显示「用药医嘱」区域：
- **用药说明**：即煎煮方法（如「先煎」「后下」「包煎」等），无则显示「-」

### 第七步：查看医生和药师

页面底部显示：
- **医生**：开具处方的医生姓名
- **药师**：审核处方的药师姓名

### 第八步：查看电子处方原件

页面下方有「查看电子处方」按钮（带处方图标）：
- 如果有电子处方文件，点击后在微信中预览原件
- 如果没有电子处方，点击后弹出提示「暂无电子处方」

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 处方单号查询失败（接口无数据） | 页面内容为空（不会崩溃），各字段不显示 |
| 处方类型未知（非4/5） | 不显示处方类型标签，RP区域无内容 |
| 电子处方文件不存在 | 点击「查看电子处方」提示「暂无电子处方」 |
| 患者姓名为空 | 不显示（接口未返回该HIS关联的患者信息） |
| 某种药品无剂型 | 剂型字段显示「其他」 |
| 药品无规格 | 规格字段显示「其他」 |
| 用法/医生/药师等字段为空 | 显示「-」 |
| 性别取值非1非2 | 默认显示「男」 |

## 关联功能

- **患者端-线下就诊记录列表**（`offLineRecords`）：展示患者所有线下就诊记录，点击跳转到本页面
- **医生端-问诊聊天室病历卡片**（`medicalrecord-box` 组件）：医生在聊天室中查看患者病历时，点击线下处方跳转本页面
- **电子处方预览**：通过 `commonMethod.viewEprescription()` 调起文件预览
- **小程序处方详情（在线处方）**（`rpDetail` 其他页面）：与本页面类似但数据来源不同（在线处方来自平台问诊，本页来自HIS）
