# 药品订单下单与配送流程

> 文档版本 1.0 | 面向非技术读者 | 2026-07-01

## 概述

患者（或代付的医生）在处方审核通过后，进入药品订单页面完成下单操作。本页面聚合了配送方式选择、患者信息确认、药品清单查看、订单费用明细、支付渠道选择等功能。支持医生代付、微信分享领取、医保支付三种支付场景。

## 触发条件

- 处方审核通过后，医生点击"发到患者微信"或"代支付"
- 患者从微信聊天中点击处方分享卡片进入
- 患者从处方记录中点击进入

## 步骤详解

### 第一步：页面初始化

进入页面后，系统按顺序执行：

1. **查询当前处方是否存在订单**：如果已存在订单（如已支付），直接加载订单详情
2. **不存在订单则预结算**：调用预结算接口，计算药品费、加工费、运费等

### 第二步：订单状态展示

页面顶部展示订单状态卡片：

| 业务状态 | 含义 | 触发条件 |
|---------|------|---------|
| 待处方医生审核 | 用药建议待审核 | createFlowStatus=20 且处方未被拒绝 |
| 待药师审核 | 处方待药师审核 | createFlowStatus≠40 且 prescriptionStatus=20 |
| 审核通过 | 可以下单 | createFlowStatus=30 且 prescriptionStatus=30 |
| 审核不通过 | 处方被拒绝 | prescriptionStatus=40 |

### 第三步：配送方式选择

根据药房支持的配送模式：

- **快递配送**（pharmacyMode=2）：需要选择收货地址，配送费按运费规则计算
- **自提**（pharmacyMode=1）：不需要地址，到指定药房自取
- **快递+自提（pharmacyMode=3）**：两种方式都支持，用户可切换

切换配送方式后，系统自动重新预结算（费用可能因配送费变化）。

### 第四步：药品信息查看

展示处方中的所有药品，包括药品名称、规格、数量。如果处方类型为中药，还会展示加工工艺选项（如"水煎服"、"代煎"等），选择不同工艺会重新计算加工服务费。

### 第五步：订单费用明细

| 费用项目 | 说明 |
|---------|------|
| 药品费 | 处方中所有药品的单价×数量之和 |
| 加工服务费 | 中药饮片的加工费用（按工艺计费） |
| 配送费 | 按运费规则计算的快递费 |
| 订金 | 如果支持分段支付，先支付订金 |
| 合计 | 以上各项汇总 |

### 第六步：下单校验

点击"提交订单"按钮时，系统执行一系列校验：

1. **快递模式**：必须选择收货地址，否则提示"请先添加收货地址"
2. **实名认证**：患者必须已通过身份认证，否则提示"患者需进行身份认证后才可付款"
3. **年龄限制（严格模式）**：不满6岁儿童不允许在互联网医院问诊，弹窗提示
4. **监护人**：不满6岁需要补充监护人信息（姓名+身份证+关系）
5. **会员绑定**：快速开方的处方需要先绑定会员关系

### 第七步：支付

下单成功后进入支付环节：

- **微信支付**：跳转到收银台页面（hisPay），支持微信JSAPI支付
- **医保支付**：如果药房支持医保结算且有医保授权码，跳转医保小程序完成支付

### 第八步：医生代付

如果医生在处方详情页点击"代支付"：

- 页面标题变为"代患者支付"
- 底部按钮对医生角色可见（正常情况只对患者可见）
- 禁止医生再次代付（`forbiddenDf=true`）
- 代付成功后，弹出"代付成功"弹窗，可选择发送给患者

### 第九步：地址缓存机制

患者从订单页跳转到地址管理页选择地址后，地址信息通过本地缓存（`addressCache`）回传：

- 订单页监听地址缓存变化
- 检测到新地址后自动重新预结算
- 页面销毁时清理地址缓存

### 第十步：医保授权码处理

如果使用医保支付，患者需要先从医保小程序获取授权：

- 页面监听小程序从后台切回前台事件
- 检测医保小程序返回的授权码（authCode）
- 授权成功（retCode=0）：自动调用退款/支付接口
- 取消授权（retCode=503608）：弹窗提示"您已经取消医保电子凭证身份授权，医保退款已中止"

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 无收货地址（快递模式） | 提示"请先添加收货地址" |
| 患者未实名认证 | 提示"患者需进行身份认证后才可付款" |
| 不满6岁（严格模式） | 弹窗"不满6岁儿童不能在互联网医院进行问诊" |
| 就诊人已被其他手机号绑定 | 弹窗显示绑定手机号，提供"去解绑"按钮 |
| 非本人处方（100500） | 跳转到处方记录列表 |
| 预结算失败 | 弹窗显示错误信息，确认后返回上一页 |
| 订单状态已变更 | 提示"订单状态发生改变，请刷新页面" |
| 医保取消授权 | 提示"您已经取消医保电子凭证身份授权" |
| 处方的订单不存在（getOrderInfo失败） | 跳转到处方记录列表 |

## 关联功能

- **收货地址**：选择/添加收货地址（跳转addressList）
- **收银台**：完成支付（跳转hisPay）
- **就诊人解绑**：绑定冲突时跳转解绑页面
- **监护人选填**：不满6岁就诊人需补充监护人信息
