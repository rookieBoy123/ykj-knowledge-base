# 医生处理用药需求开方流程

> 文档版本：2026-07-02 | 面向读者：医生角色

## 概述

患者通过问诊提交了"用药需求"（列出需要的药品及数量）后，医生在诊室中查看到该需求，需要为患者确认药品、填写诊断和医嘱、执行电子签名后完成开方。这是介于"患者需求"与"正式处方"之间的过渡环节。

## 触发条件

- **前置条件**：患者已在问诊中提交了用药需求（含药品清单），该需求已关联到当前问诊单
- **入口路径**：医生在问诊聊天界面 → 点击用药需求卡片 → 进入本页面
- **页面参数**：`drugDemandNo`（需求编号）、`inquiryNo`（问诊单号）、`treatmentDiagnoses`（已有的诊断信息）

## 步骤详解

### 第一步：查看需求基本信息

进入页面后，医生看到三个信息区域：

- **已开处方标记**：如果该用药需求已经被处理过（医生已开方），页面顶部会显示"已开处方"的绿色标记，此时所有编辑操作均不可用
- **用药人信息卡片**：显示患者姓名、性别、年龄（只读）
- **诊断区**：显示问诊中已录入的诊断信息，医生角色且未开方时可以修改
- **用药需求区**：列表展示患者需求的药品（药品名称、数量、备注），医生可以修改药品列表中的用法用量

### 第二步：填写药品用法用量

医生在药品列表中为每种药品补全用法用量信息：

- **使用频率**（每日几次）：从系统预设的使用频率字典中选择
- **用药方式**：口服、外用、静脉注射等
- **使用天数**：默认1天，可修改
- **使用数量**：每次用量

如果所有药品的用法用量均已填写完整，底部的"确认开方"按钮变为蓝色可点击状态；如果有药品未填完整，按钮保持灰色禁用状态。

### 第三步：填写医嘱

在医嘱输入框（药品列表下方）填写给患者的用药注意事项。

### 第四步：确认开方

点击底部"确认开方"按钮后，系统执行两阶段验证：

**阶段1 — 诊断检查**：如果医生删除了所有诊断信息，提示"请选择诊断"，阻止提交。

**阶段2 — 用法用量完整性检查**：如果仍有药品未填写用法用量，提示"请将已添加的药品用法用量填写完整"。

通过验证后，系统提交开方请求，包含：药品列表、诊断、医嘱、总剂数、需求编号等信息。

### 第五步：电子签名

如果医院启用了CA电子签名功能：

1. 系统先检查医生的CA认证状态：
   - **未认证（authStatus=1）**：弹出"CA资料未同步"提示，引导医生同步账号资料
   - **认证中（authStatus=2）**：提示"认证审核中"，引导查看认证进度
   - **认证失败（authStatus=4）**：提示认证失败信息
   - **认证成功（authStatus=3）**：进入签章环节

2. 签章分为三种平台路径：
   - **医网信（platform=1）**：严格模式下需先完成人脸识别，然后调用医网信SDK进行签名
   - **源康健PaaS（platform=3）**：同医网信流程
   - **放心签（platform=4）**：弹出放心签签名弹窗

3. 签名成功后自动跳转到开方成功页，显示处方号
4. 签名失败或取消时，跳转到患者端的处方详情页（让医生可以重新操作）

如果医院未启用CA签章，提交后直接跳转到成功页。

### 第六步：开方完成

跳转到成功页 `/packageDoctor1/pages/myPharmacy/success`，页面显示处方号（`outpatientNo`），医生可点击查看处方详情或返回。

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 用药需求已被开方 | 页面顶部显示"已开处方"标记，所有输入框和按钮不可操作 |
| 诊断列表为空 | 点击确认开方时提示"请选择诊断"，阻止提交 |
| 药品用法用量不完整 | 提示"请将已添加的药品用法用量填写完整" |
| CA账号未同步 | 弹出引导弹窗，提示先同步资料到电子签名平台 |
| CA认证审核中 | 提示等待认证审核通过 |
| CA认证失败 | 显示失败原因，引导重新认证 |
| 签章取消 | 跳转到处方详情页，医生可重新发起开方 |
| 提交按钮重复点击 | 通过 `clickBtnStatus` 状态锁防重复提交 |

## 关联功能

- **问诊聊天室**：用药需求的来源入口
- **处方详情**：开方完成后查看完整处方信息
- **CA签章设置**：医生的电子签名认证管理（`fxqSetting`）
- **药房/药品管理**：药品列表中的数据来源
