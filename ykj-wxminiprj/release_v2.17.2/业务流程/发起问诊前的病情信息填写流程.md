# 发起问诊前的病情信息填写流程

> 文档版本：v1.0 | 面向读者：患者

## 概述

患者在选定医生和服务类型后，需要填写病情信息（就诊人、病情描述、病历资料）才能正式发起问诊。根据不同问诊类型（咨询、问诊、复诊、快速问诊等），填写内容和校验规则有所不同。

## 触发条件

### 前置条件
- 患者已选择医生和服务类型（图文问诊、视频问诊、咨询、快速问诊、快速续方等）
- 通过`service`参数接收服务信息（服务类型、医生ID、金额等）

### 入口路径
选择医生服务后，系统携带服务参数（JSON字符串）跳转到病情填写页。

## 步骤详解

### 第一步：识别问诊类型

系统根据传入的`serviceType`参数将当前流程分为三大模式：

| 模式 | serviceType | 页面标题 |
|------|------------|---------|
| 咨询模式 | 3（咨询服务） | 「填写咨询信息」 |
| 复诊模式 | 7（医生发起复诊） | 「填写病情信息」 |
| 问诊模式 | 1/2/4/5/6等（图文/视频/快速/续方/会诊） | 「填写病情信息」 |

三种模式展示的填写区不同：
- **咨询**：就诊人选择 + 咨询内容文本（提示「请描述您需要咨询的问题」）
- **复诊**：就诊人选择（固定不可切换）+ 复诊疾病选择 + 病情描述
- **问诊**：就诊人选择 + 门慢医保标签 + 病情描述 + 病历资料上传 + 初诊信息 + 知情同意书

### 第二步：选择/确认就诊人

- 就诊人卡支持切换（`showSwitchBtn`控制），但以下场景固定不可切换：
  - 慢病复诊进入（`chronicCode`存在）：固定当前慢病患者
  - 携带指定患者ID进入：锁定该患者
  - 医生发起复诊（serviceType=7）：固定患者
- 再次问诊进入：自动选中上次问诊的就诊人ID
- 导诊进入（`uniqueId`存在）：自动创建或匹配就诊人
- **重要**：如果当前选中的就诊人未完成实名认证，弹出提示「当前就诊人未认证，请先进行身份认证？」并提供「去认证」按钮

### 第三步：填写病情内容

**A. 病情描述（所有问诊模式）**

文本输入框，字数上限1000字：
- 咨询模式提示：「请描述您需要咨询的问题！例：疾病（疾病位置），持续时间，吃过什么药，需要咨询的问题」
- 问诊模式提示：「详细描述您的病情，症状，治疗过程，购药需求，以及想要获得的帮助（最少输入2个字）」
- 复诊模式提示：「请输入病情描述」
- 系统配置可开启「病情描述必填」（`enableDescription`），开启时标题旁显示红色星号

**B. 上传病历资料（问诊模式）**

- 默认选择「记录遗失/不在身边」单选（确认线下就诊但资料不在身边）
- 可切换为「上传」模式，支持最多8张图片上传
- 每个图片有删除按钮，点击可预览
- 系统配置可开启「病历资料必填」（`medRecordRequired`），开启后仅允许上传模式，隐藏「记录遗失/不在身边」选项

**C. 上传照片（咨询模式）**

- 支持最多8张咨询相关照片上传
- 与病历资料上传共用相同组件和限制

**D. 初诊信息（问诊模式，可选显示）**

当系统配置开启「首诊资料收集」（`dataCollection`）时显示：
- **初诊医院**：文本输入框，必填（红色星号）
- **就诊时间**：日期选择器，最大日期为前一天
- 底部提示：「提醒：本服务仅限于为部分常见病、慢性病患者提供复诊服务，请勿提交不真实的信息」

### 第四步：医保费别选择

在图文/视频/快速问诊/快速续方流程中，如果系统开启医保支付功能且医生支持医保，显示费别选择：

- **自费**（默认选中）：医保个账或自费支付
- **医保**：普通门诊医保统筹报销
- 切换为「医保」时触发三重校验：
  1. **就诊人检查**：必须有选中的就诊人且已认证
  2. **就诊人可用性检查**：就诊人状态正常
  3. **定点检查**：
     - 未定点（status=0）：弹窗提示「该就诊人暂未定点[XX]，定点后可享用医保统筹报销」，提供「暂不定点」和「确定定点」
     - 已定点（status=1）：直接记录费别为医保
     - 已定点其他医院（status=2）：弹窗提示「是否确定修改定点为[XX]」，提供「取消」和「修改定点」
- 费别选择后自动更新就诊人历史就诊备注

### 第五步：勾选知情同意书并提交

- 页面底部显示「我已阅读并同意《患者知情同意书》」
- 提交前必须勾选同意（`readProtocal`）
- 未勾选则按钮仍可点击但后端会拦截
- 严格模式下知情同意书类型会根据地区适配（如四川地区使用「sichuan-tys」）

### 第六步：提交后的页面跳转

- 提交成功 → 跳转到服务确认页（`createOrder`）
- 特殊情况处理：
  - 有进行中问诊 → 弹窗「您有正在进行的问诊，是否前往？」选择「前往问诊」
  - 有未支付问诊单 → 弹窗「您有未支付问诊单，是否前往支付？」提供「重新创建问诊」和「去支付」
  - 不满6岁儿童 → 弹窗「不满6岁儿童不能在互联网医院进行问诊，请到线下实体医院问诊」
  - 快速问诊无在线医生 → 弹窗「当前暂无快速问诊医生在线，若要问诊，可选择其他医生」
  - 就诊人未填写联系地址 → 弹窗「因处方合规要求，需先填写就诊人的联系地址」提供「前往填写」

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 服务参数JSON解析失败 | 自动将单引号替换为双引号后重新解析 |
| 未登录时进入 | 跳过就诊人信息获取（token为空时return） |
| 就诊人未认证 | 弹窗提示「当前就诊人未认证，请先进行身份认证？」 |
| 不满6岁儿童+严格模式 | 弹窗禁止继续，无法提交 |
| 选择医保但就诊人不支持 | 强制切回自费（`isInsurance`为false时） |
| 快速问诊无医生 | 弹窗提示，无法继续 |
| 快速问诊禁止开方 | 提交前需额外勾选《患者知情同意书》复选框 |

## 关联功能

- 提交成功 → 服务确认页（`createOrder`）
- 就诊人未认证 → 就诊人详情页（认证流程）
- 就诊人地址缺失 → 地址填写页
- 医保定点 → 定点操作（修改定点/新增定点）
- 知情同意书 → 协议查看弹窗
