# 就诊人新增与个人信息编辑流程

> 文档版本：release/v2.17.2 | 面向读者：患者、客服

## 概述

患者可以为家人添加就诊人，填写基本信息和病史，也可以在个人中心编辑自己的头像和昵称。

## 触发条件

### 新增就诊人
- **前置条件**：已登录
- **入口路径**：个人中心→"我的就诊人"→"添加就诊人"

### 编辑个人信息
- **前置条件**：已登录
- **入口路径**：个人中心→点击头像区域→"编辑资料"

---

## 第一部分：新增就诊人

### 第一步：填写基本信息

进入"就诊人信息"页面后，需要填写以下信息：

- **姓名**（必填）：输入就诊人的真实姓名
- **身份证号**（必填）：用于实名认证，输入后系统自动从身份证号提取性别和出生日期。页面提示为"请输入身份证"
- **性别**：默认"男"，可从身份证号自动解析
- **有无监护人**：默认为"无"。如果就诊人是未成年人，可以切换到"有"，此时需额外填写：
  - **监护人姓名**（必填）
  - **监护人身份证号**（必填）

如果设置为"有监护人"但未填写监护人姓名或身份证号，提交时会提示。

### 第二步：填写病史信息

页面下方是三组病史信息，每组都有一个"有/无"开关：

- **过敏史**：选择"有"时需填写具体过敏信息
- **现病史**：选择"有"时需填写具体病史
- **既往史**：选择"有"时需填写具体既往史

如果某病史开关切到"有"但内容为空，提交时会提示填写。

### 第三步：保存提交

点击底部"保存"按钮，系统依次校验：

1. 姓名不能为空 → "请输入就诊人姓名"
2. 身份证不能为空 → "请输入身份证"
3. 监护人有数据时姓名不能为空 → "请输入监护人姓名"
4. 监护人有数据时身份证不能为空 → "请输入监护人身份证"
5. 过敏史"有"时内容不能为空 → "请输入过敏史"
6. 现病史"有"时内容不能为空 → "请输入现病史"
7. 既往史"有"时内容不能为空 → "请输入既往史"

所有校验通过后显示加载动画"实名信息认证中，请稍等"，提交到后端。成功后自动返回上一页。

### 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 姓名未填写 | 提示"请输入就诊人姓名"，不提交 |
| 身份证未填写 | 提示"请输入身份证"，不提交 |
| 监护人开关开启但姓名为空 | 提示"请输入监护人姓名" |
| 监护人开关开启但身份证为空 | 提示"请输入监护人身份证" |
| 有过敏史但内容为空 | 提示"请输入过敏史" |
| 有现病史但内容为空 | 提示"请输入现病史" |
| 有既往史但内容为空 | 提示"请输入既往史" |
| 实名认证失败 | 后端返回非200状态码时停止加载但不提示具体原因（显示前端的通用错误） |
| 重复点击提交 | 系统自动防重复点击 |

## 第二部分：编辑个人信息（头像和昵称）

### 第一步：进入编辑页面

从个人中心点击头像区域进入"编辑资料"页面，系统自动加载当前的头像和昵称。

### 第二步：修改头像

**新版微信（支持chooseAvatar）**：
- 点击头像区域，微信弹出系统头像选择器
- 选择后自动上传到服务器，上传类型为`avatarIMG`
- 上传成功后头像即时更新
- 上传失败提示"上传图片失败"

**旧版微信（不支持chooseAvatar）**：
- 点击头像区域，底部弹出菜单"从相册选择"和"拍照"
- 选择后上传，流程与新版相同
- 如果微信头像获取失败，提示"使用微信头像失败，请重试！"

### 第三步：修改昵称

- 输入框初始值为当前昵称
- 支持中英文、数字、符号，长度1~40个字符
- 不能为空或纯空格
- 修改时会自动去除首尾空格

### 第四步：保存修改

点击"保存修改"按钮：

1. 昵称为空→弹窗提示"没有输入昵称，请重新填写"
2. 昵称格式不正确→提示"昵称输入格式不正确，请重新输入"
3. 格式正确→提交后端接口
4. 成功后提示"修改成功"，更新本地缓存，通知个人中心刷新，1.5秒后自动返回

### 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 昵称为空 | 弹窗提示"没有输入昵称，请重新填写" |
| 昵称格式不正确（如纯空格/超40字符） | 提示"昵称输入格式不正确，请重新输入" |
| 头像上传失败 | 提示"上传图片失败" |
| 使用微信头像失败 | 提示"使用微信头像失败，请重试！" |
| 保存时后端返回失败 | 静默失败，停留在编辑页 |

## 关联功能

- 个人中心页（index/index.vue）：通过事件通道接收`reFetch`刷新用户信息
- 就诊人列表（patientList.vue）：新增后返回刷新列表
