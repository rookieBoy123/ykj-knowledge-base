# 代支付订单查看与支付流程

> 文档版本：release/v2.17.2 | 面向读者：客服、运营

## 概述

代支付功能允许微信好友帮助患者支付订单。患者可以将待支付的订单分享给微信好友，好友打开链接后查看订单信息并代付。本流程覆盖好友打开代支付链接后的完整交互体验。

## 触发条件

微信好友通过患者分享的代支付链接（或二维码）进入小程序。

**前置条件**：
- 订单已被患者创建且处于待支付状态
- 患者已生成代支付分享链接（shareId）

## 步骤详解

### 第一步：身份验证

打开代支付链接后，系统首先验证访问者的身份：
- **未登录**：显示「您未登录，请登录后查看」+「登录」按钮，点击跳至登录页
- **登录后校验**：通过`updateShareRecord`接口更新分享记录，绑定接收者信息
  - 成功 → 进入详情页
  - 失败（code=401）→ 显示未登录提示
  - 失败（其他）→ 显示「订单已被支付，已无需代付」+「返回」按钮

### 第二步：查看订单状态

验证通过后，页面根据订单当前状态展示不同的视觉效果：

**A. 待支付（flowStatus=10）**
显示待支付头部组件：
- 代付金额（大号数字）
- 支付剩余有效时间
- 蓝色「确认支付」按钮

底部显示完整的业务信息（仅当`hasBusinessRoleFlag`为true时显示，否则只显示「返回」按钮）。

**B. 已取消（flowStatus=70）或已失效（flowStatus=80）**
显示取消/失效头部：
- 灰色图标
- 「订单已取消」或「订单已失效」文字
- 底部「返回」按钮

**C. 退款中（flowStatus=50）或已退款（flowStatus=90）**
显示退款头部：
- 退款金额
- 支付人名称
- 支付时间
- 「订单退款中」或「订单已退款」文字
- 底部「返回」按钮

**D. 已支付（其他flowStatus）**
显示已支付头部：
- 已付金额
- 支付人名称
- 支付时间
- 「返回」按钮

### 第三步：查看业务详情（仅待支付+有权限时）

当订单为会诊类型且访问者具有业务角色时（`hasBusinessRoleFlag`），展示以下业务信息：

**会诊时间区**：
- 会诊开始时间（大号数字显示时分，小号显示日期）
- 会诊时长（分钟数标签）
- 会诊结束时间

**患者信息区**：
- 患者姓名
- 性别标签 + 年龄
- 病情描述
- 经治科室
- 当前诊断

**会诊信息区**：
- 会诊类型（如「普通会诊」）
- 会诊原因（多个原因用逗号拼接）
- 会诊目的
- 会诊科室

**会诊医生区**：
- 经治医生（主治医生）
- 参会医生（多个医生用逗号拼接）

**会诊费用**：
- 红色加粗数字，单位为元

### 第四步：确认支付

在待支付状态下点击「确认支付」：
1. 重新获取订单详情，确认状态仍是待支付（flowStatus=10）
2. 跳转到收银台（hisPay），传入参数：
   - `orderNo`：订单编号
   - `type=consultation`：订单类型为会诊
   - `inquiryNo`：问诊编号
   - `isDf=shareId`：标记为代付入口
3. 收银台完成支付后，代付页面统一返回上一页

如果订单状态已变更（不再待支付），提示「当前订单非待支付状态」。

### 第五步：自动刷新机制

页面每60秒自动刷新订单详情（`setInterval`），确保用户看到最新的订单状态。例如：
- 支付成功后60秒内会自动切换为已支付状态
- 订单过期后会自动切换为已失效状态

用户离开页面时定时器自动清除。

### 第六步：返回

点击「返回」按钮跳转到患者端首页（`patient/index/index?current=home`）。

## 异常处理

| 情况 | 系统处理方式 |
|------|------------|
| 缺少订单号参数 | 提示「缺少订单号参数」 |
| 未登录 | 显示登录引导页 |
| 订单已被他人支付 | 显示「订单已被支付，已无需代付」 |
| 订单非待支付状态点支付 | 提示「当前订单非待支付状态」 |
| 分享记录更新失败(401) | 显示未登录提示 |
| 分享记录更新失败(其他) | 显示已支付/禁止访问页面 |
| 会诊详情查询失败 | 业务信息区域不展示 |

## 关联功能

- ← **微信分享**：患者分享链接，好友点击进入
- → **收银台**（hisPay）：代付人确认支付时跳转
- → **患者首页**（patient/index）：点击返回时跳转
- → **登录页**：未登录时引导登录

## 特殊说明

**分享机制**：代支付链接的生成和追踪依赖于`shareId`。患者在创建代支付时调用`setShareRecord`生成分享记录，好友打开链接时通过`updateShareRecord`更新接收者信息，实现分享链路追踪。

**业务信息可见性**：`hasBusinessRoleFlag`字段控制好友是否能看到会诊的敏感业务信息（患者姓名、病情等）。如果该字段为false，则为保护隐私不展示业务详情。

**60秒刷新**：页面持续的自动刷新确保好友不会因为错过状态更新而误操作。如果订单在好友等待期间被患者取消或被其他人支付，60秒内页面会自动更新。
