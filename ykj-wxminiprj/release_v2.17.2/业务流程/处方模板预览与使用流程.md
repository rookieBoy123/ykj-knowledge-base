# 处方模板预览与使用流程

> 文档版本 v1.0 | 面向读者：医生用户

## 概述

医生在开方时可以查看处方模板的详细内容，包括诊断信息、药品清单、用法用量等。确认无误后点击"使用"按钮，模板内容会自动填充到开方页面中，减少重复录入。

## 触发条件

- **前置条件**：已登录医生端，在开方页面（快速开方或处方添加页）
- **入口路径**：
  - 从处方模板列表点击某个模板进入预览页
  - 从公用处方库选择某个处方进入预览页
  - 还可从处方详情页不显示底部按钮（notGol参数）

## 步骤详解

### 第一步：查看模板名称

页面顶部显示模板名称卡片，只读模式。如果没有名称则显示"暂无"。

### 第二步：查看临床诊断（个人模板）

个人模板显示临床诊断区域的三个维度：
- **诊断**：诊断名称列表（诊断类型=1），逗号分隔
- **辨病**：辨病名称列表（诊断类型=2），逗号分隔
- **辨证**：辨证名称列表（诊断类型=3），逗号分隔

### 第三步：查看药品清单

**中药模板**（type=2）：
- 以"RP："标题展示所有药材
- 每味药材双排显示：药材名、煎煮方法（如有）、用量+单位
- 缺货药材特殊标识：药材名红色，左侧显示缺货图标，下方红色提示"药材缺货，请替换"

**成药模板**（type=1）：
- 以药品卡片形式展示每个药品
- 包含药品图片、药品名+规格、用法用量信息
- 失效药品显示"已失效"遮罩层
- 严格模式下显示药品通用名（drugsAlias），否则显示商品名
- 右侧显示数量×单位和售价

**公用处方**（isGy=1）：只展示药材列表，无诊断和用法用量区域

### 第四步：查看用法用量（中药模板）

中药模板底部展示：
- **用法**：内服/外用
- **剂数**：共X剂，Y日Z剂
- **服用方式**：1剂分N次服用，每次X ml（内服时）
- **煎煮说明**：用药医嘱区域包括煎煮方法、用药时间（必填，从字典加载）、用药禁忌（多选，逗号分隔）

### 第五步：使用模板

点击底部"使用"按钮：
1. 确定返回目标页面：从页面栈中找到`addOrEditRp`（处方添加页）或`quickPrescribe/index`（快速开方页）
2. 通过事件通道（eventChannel）发送`useRp`事件
3. 传递模板类型和ID：
   - 个人模板：`{templateType: 'template', businessId: outpatientNo}`
   - 公用处方：`{templateType: 'public', businessId: id}`
4. 返回开方页面后，开方页监听`useRp`事件，自动加载模板中的药品和诊断信息

### 公用处方模式

当通过公用处方库进入时（isGy=1）：
- 不显示个人模板的诊断和用法用量
- 直接展示药材列表
- 点击"使用"后通过事件通道传递公用处方ID

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 模板中部分药品已失效 | 失效药品显示"已失效"遮罩，但不影响其他药品的使用 |
| 某味药材缺货 | 药材名变红，显示"药材缺货，请替换"提示 |
| 严格模式下药品显示 | 展示药品通用名而非商品名 |
| 没有诊断信息 | 诊断区域不显示（v-if条件不满足） |
| 页面栈中找不到开方页 | 不执行返回操作 |
| 不显示使用按钮（notGol） | 仅预览模式，底部不显示"使用"按钮 |

## 关联功能

- **处方模板列表**（`recipetemplate/index.vue`）：模板管理入口
- **快速开方**（`quickPrescribe/index.vue`）：接收模板数据后自动填充
- **处方添加**（`prescription/addOrEditRp.vue`）：接收模板数据后自动填充
- **公用处方库**：管理后台配置的公共处方
