# 医网信CA证书查看与签章管理流程

> 文档版本：2026-07-05 | 面向读者：医生

## 概述
医生在个人中心查看自己的CA电子签章（医网信/源网信），可以查看签章图片、证书有效期、执业信息，也可以通过手写或上传图片的方式修改签章样式。

## 触发条件
- **前置条件**：医生已完成CA认证（authStatus=3），且使用医网信或源康健PaaS平台（platform=1或3）
- **入口路径**：医生端「我的」→ 个人中心 → CA签章相关入口

## 步骤详解

### 第一步：进入签章查看页面
页面展示两个信息区：
- **医师信息区**：显示医院名称、证书有效期（格式如「2023.01.01 至 2026.01.01」）、科室、执业地点
- **签章图片区**：显示当前签章图片（从本地证书中读取），下方有「修改签章」按钮

证书有效期根据CA认证信息中的证书起始日期和结束日期自动格式化显示。

### 第二步：修改签章
点击「修改签章」按钮，底部弹出选择菜单，提供两种方式：
- **手写签章**：跳转到医网信插件的手写签名页面，在画板上手写签名后保存为签章
- **上传签章照片**：当前提示「暂未开放」，暂不支持从相册上传签章图片

手写签章需要传入 clientId（医网信商户ID，从全局配置中获取）才能正常打开手写插件。

### 第三步：查看结果
修改完成后返回签章页面，页面自动刷新显示新的签章图片。

## 关联功能
- **证书下载**：如果本地没有证书，需要先下载CA证书（跳转到证书下载页面）
- **签章密码设置/重置**：签章密码重置页面（通过短信验证码+新密码完成重置）
- **签章使用**：在处方审核、病历签章等场景中，系统会检查本地证书是否存在，不存在则引导下载

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 本地没有CA证书 | 会被拦截并引导进入证书下载页面，不能直接查看签章 |
| clientId配置丢失 | 提示「初始化配置失败，丢失ywxClientId」，无法打开手写签章 |
| 不是同一用户openId | 会自动清除本地证书，要求重新下载（防止不同医生共用设备时的签章串用） |
| 证书需要更新 | 系统先删除本地过期证书，再引导重新设置密码、下载签章 |
| 上传签章照片 | 当前提示「暂未开放」，该功能尚未启用 |

## 技术要点（供排查）
- 签章图片来源：`ywxUtil.getStampPic()` → base64格式
- 用户信息：`ywxUtil.getUserInfo()` → 含证书起止日期
- 医生信息：`findCurrentDoctor` API → hospitalName, departmentName, practicePlace
- 手写插件路径：`plugin-private://wx046bf83859595dfd/pages/stamp/index`
