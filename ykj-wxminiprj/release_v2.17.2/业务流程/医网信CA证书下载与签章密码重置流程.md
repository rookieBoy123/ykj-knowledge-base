# 医网信CA证书下载与签章密码重置流程

> 文档版本：2026-07-05 | 面向读者：医生

## 概述
医生首次使用CA签章功能或证书过期时，需要下载医网信（源网信）电子签章证书。该流程包括设置签章密码、下载证书、以及后续的密码重置功能。

## 触发条件
- **前置条件**：
  - 医生已完成CA认证注册（authStatus=3）
  - CA平台为医网信或源康健PaaS（platform=1或3）
  - 本地无证书、证书过期需要更新、或切换了微信账号
- **入口路径**：处方审核/病历签章等功能入口自动检测 → 引导进入证书下载页

## 步骤详解

### 场景一：首次下载证书（xz页面）

#### 第一步：自动检测状态
系统根据医生手机号调用CA状态查询接口，检查以下条件：
- 医生是否已注册医网信
- 本地是否存在有效证书
- 是否需要更新证书
- 当前微信openId是否与证书绑定openId一致

#### 第二步：进入证书下载页
若需要下载/更新证书，自动跳转到医网信证书下载页面。页面内嵌医网信插件（`<cert>`组件），需要传入：
- **clientId**：医网信商户ID
- **phone**：医生手机号

系统会自动设置医网信运行环境（测试环境或正式环境）。

#### 第三步：设置签章密码并下载
在医网信插件中完成密码设置和证书下载。下载成功后弹出成功提示框，提供两个选项：
- **设置签章**：跳转到签章管理页面（qz页面），可查看和修改签章样式
- **进入首页**：返回医生工作台首页

#### 第四步：失败处理
如果证书下载失败，弹出失败原因弹窗（显示status和message），需要联系管理员或重试。

### 场景二：签章密码重置（rest页面）

#### 第一步：进入密码重置页
在签章密码管理中点击「忘记密码」或「重置密码」。

#### 第二步：身份验证
页面内嵌医网信插件，根据手机号进行身份验证（通常需要短信验证码）。

#### 第三步：设置新密码
验证通过后设置新的签章密码。

#### 第四步：重置成功
重置成功后弹出「修改成功」提示，点击「返回个人中心」回到上一页。

## 证书生命周期管理

系统中对本地证书的状态判断逻辑（位于 `ywx.js` 的 `isShowCert` 方法）：

| 本地证书状态 | 系统行为 |
|-------------|---------|
| 无本地证书 | 引导进入下载页面 |
| 有证书但需更新（enableUpdateCert=true） | 先删除过期证书，再引导重新下载 |
| 有证书且无需更新，openId匹配 | 直接使用，不重复下载 |
| 有证书且无需更新，openId不匹配 | 删除旧证书（换微信账号了），引导重新下载 |

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| CA账号未注册 | 提示「当前医生未进行CA认证，无法进行处方审核，请联系管理员重新修改医生认证」 |
| 证书下载失败 | 弹窗显示失败原因（含status和message） |
| clientId丢失 | 提示「初始化配置失败，丢失ywxClientId」，无法进入下载页面 |
| 手机号为空 | 从本地缓存`doctorInfo.doctorPhone`获取，若为空则不触发检测流程 |
| caUserNo不存在 | 直接跳过证书检测，不触发下载引导 |

## 关联功能
- **签章查看**：证书下载成功后可跳转到签章管理页面查看签章
- **签章密码设置**：与服务端CA用户密码设置接口联动（`pwdSetting` API）
- **处方审核签章**：审核处方时检测证书状态，无证书则阻断操作
- **病历签章**：病历书写完成后签章时同样检测证书状态
