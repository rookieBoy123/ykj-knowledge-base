# 患者端药品订单下单与支付流程

> 文档版本 2026-07-02 | 面向读者：运营、产品、客服

## 概述

患者通过问诊获得医生开具的处方后，在小程序中查看药品明细、选择配送方式、填写收货地址，完成下单和支付。支持快速开方处方、医生代付处方、医保微信支付等多种场景。

## 触发条件

- **前置条件**：医生已开具处方/用药建议（outpatientNo），处方状态有效
- **入口路径**：通过问诊对话中的处方卡片或分享链接进入药品订单页
- **适用角色**：患者（主要）、医生（代付场景）

## 步骤详解

### 第一步：进入订单页，系统自动加载信息

打开订单页后，系统自动做三件事：
1. **查处方详情**：获取患者信息和药品信息
2. **查是否有已有订单**：判断是新建订单还是已有订单
3. **预结算**：如果没有订单，调用预结算接口计算总金额、获取药房的配送方式

页面从上到下展示四个区域：
- **订单状态**：显示当前处方/订单的状态（待支付/已支付/已失效）
- **物流地址**：快递方式显示收货地址，自提方式不显示
- **患者信息**：姓名、性别、年龄、就诊卡号
- **药品信息**：药品名称、规格、数量、用法用量
- **订单信息**：总金额、配送方式选择、加工工艺选择

### 第二步：选择配送方式

根据药房支持的配送方式，用户可以选择：
- **快递配送**（pharmacyMode=2）：需填写收货地址，药品邮寄到家
- **到店自提**（pharmacyMode=1）：无需地址，到药房自取

切换配送方式后，系统会**重新预结算**计算最新金额（含运费变化）。

### 第三步：填写/选择收货地址（快递模式）

快递模式下，点击地址区域进入地址选择页。选好地址后返回，系统自动重新预结算。

### 第四步：患者身份校验

下单前系统会校验：
- **患者实名认证**：未认证的患者不能付款，提示"患者需进行身份认证后才可付款"
- **年龄限制**：严格模式下，6岁以下儿童不能下单，弹窗提示"不满6岁儿童不能在互联网医院进行问诊"
- **监护人信息**：6岁以下就诊人需要填写监护人信息

### 第五步：创建订单并支付

点击底部"提交订单"按钮：

1. **会员绑定校验**（快速开方处方）：检查就诊人是否已绑定当前微信账号
   - 未绑定：自动绑定
   - 已绑定其他手机号：弹窗提示"同一就诊人只能绑定在一个手机号"，可选择解绑
2. **创建订单**：调用接口生成订单号，获得订单编号
3. **跳转支付**：跳转到支付页面

**支付方式**：
- **微信支付**（默认）：跳转微信支付
- **医保微信支付**：当订单标记为医保订单时，跳转医保小程序完成支付
- **医生代付**（shareType=3）：由医生代为支付，患者无需付款

### 第六步：订单支付后

支付成功后：
- 医生代付场景：展示代付成功弹窗
- 普通支付：订单进入待发货状态
- 患者可查看订单状态变化

## 处方状态说明

药品订单页根据处方状态展示不同界面：

| 处方状态 | 含义 | 界面表现 |
|---------|------|---------|
| 待下单 | 处方已开具，患者还未下单 | 显示"提交订单"按钮 |
| 已下单 | 已有订单但未支付 | 显示"去支付"按钮 |
| 已支付 | 订单已付款 | 不可再支付，显示"确认签收"按钮 |
| 已失效 | 处方已作废/撤回 | 不可支付 |

## 异常处理

| 情况 | 系统处理方式 |
|------|------------|
| 患者未实名认证 | 提示"患者需进行身份认证后才可付款"，阻止下单 |
| 6岁以下儿童（严格模式） | 弹窗提示"不满6岁儿童不能在互联网医院进行问诊" |
| 6岁以下儿童（非严格模式） | 弹出监护人信息填写 |
| 快递模式下未选地址 | 提示"请先添加收货地址" |
| 就诊人已绑定其他手机号 | 弹窗提示手机号，可选择"去解绑" |
| 预结算失败 | 弹窗显示错误信息，引导返回上一页 |
| 订单状态已改变 | 提示"订单状态发生改变，请刷新页面" |
| 处方不是自己的 | 跳转到"处方待支付"列表页 |
| 医保授权取消 | 提示"您已经取消医保电子凭证身份授权，医保退款已中止" |

## 关联功能

- **支付页面**（hisPay）：跳转完成支付
- **地址管理**（addressList）：选择/编辑收货地址
- **解绑页面**（getAuthCode）：就诊人解绑手机号
- **处方列表**（prescribeForPay）：返回待支付处方列表
