# 问诊记录查看流程（医生端）

> 面向医生 | 小程序医生端功能

## 概述

医生查看自己所有患者的问诊记录列表，快速了解每个问诊的状态（问诊中/已结束/已取消）、最新消息内容，并可直接点击进入问诊聊天室继续沟通。

## 触发条件

- **前置条件**：已登录医生账号
- **入口路径**：医生端小程序 → 问诊记录

## 步骤详解

### 第一步：查看问诊列表

页面展示医生名下所有患者的问诊记录，以卡片形式排列。每条记录显示：

- **患者头像**：带有性别和年龄标签（如"男 | 32岁"）
- **患者姓名**
- **服务类型标签**：图文问诊、视频问诊等
- **问诊状态标签**（右对齐）：
  - **问诊中**：问诊正在进行
  - **问诊结束**：问诊已结束（含处方已开具的情况）
  - **已取消**：问诊已被取消
- **最新消息**：
  - 已取消的问诊：显示取消原因
  - 进行中的问诊：根据服务类型显示"咨询问题"或"病情描述"
- **预约时间**（如有）：显示问诊时间段

列表支持**滚动加载**：滚动到底部自动加载20条更多记录。

空状态：如无任何问诊记录，显示"暂无记录"。

### 第二步：解析最新消息

每条记录显示的最新消息经过智能处理：

| 消息类型 | 显示内容 |
|---------|---------|
| 成功预约时间 | "预约时间成功，请提前5分钟进入诊室" |
| 处方订单消息 | "你有一条订单消息" |
| 普通文本消息 | "你有一条未读消息" |
| 语音消息（.mp3） | "[语音]" |
| 图片消息（.jpg/.png/.jpeg） | "[图片]" |
| 视频消息（.mp4） | "[视频]" |
| 视频问诊房间 | "[问诊视频]" |
| 无消息内容 | "暂无消息" |

### 第三步：进入问诊聊天室

点击任意问诊记录卡片，直接跳转到该问诊的聊天室页面，医生可继续与患者沟通、开具处方等操作。

---

## 异常处理

| 情况 | 系统处理方式 |
|------|------------|
| 无任何问诊记录 | 显示"暂无记录"空状态 |
| 无预约时间 | 不显示时间区域 |
| 消息内容为空 | 显示"暂无消息" |
| JSON解析失败 | 回退到文件类型判断逻辑 |

## 业务规则

1. **数据范围**：仅展示当前登录医生的问诊记录（按 `doctorId` 过滤）
2. **分页加载**：每次加载20条，滚动到底部自动加载下一页
3. **问诊状态固定三种**：问诊中（flowStatus=20）、问诊结束（flowStatus=30）、已取消（flowStatus=40）
4. **服务类型名称**：由统一的 `getServiceName` 函数映射，与全局服务类型保持一致

## 关联功能

- **问诊聊天室**：点击记录进入聊天室
- **处方管理**：问诊结束状态通常伴随处方已开具
- **患者管理**：患者信息与此页数据关联
