# 医生二维码名片生成与分享流程

> 文档版本 v1.0 | 面向读者：医生

## 概述

医生生成个人专属的二维码名片，包含医生头像、姓名、职称、科室、医院信息和患者扫码入口二维码。名片可保存到手机相册或通过微信分享给患者。

## 触发条件

- **前置条件**：医生已登录系统，有患者访问二维码
- **入口路径**：医生端 → 个人中心 → 我的二维码

## 页面结构

生成后的名片包含以下视觉元素：

- **顶部背景图**：装饰性渐变背景
- **头像**：圆形头像，叠加在背景图中间
- **医生信息**：医生姓名 + 岗位类别名称、科室名称、医院名称
- **二维码**：患者扫码访问的二维码图片（380rpx正方形，白色背景）
- **引导箭头**：指向二维码的箭头图标
- **底部引导语**：可配置文案（如"微信扫码，足不出户即可找我复诊开方"），支持多行显示（最多三行，每行最多20字）
- **操作按钮区**：保存到手机 + 分享名片

## 步骤详解

### 第一步：进入二维码页面

医生点击进入后，系统自动从本地缓存读取医生信息（包括头像、二维码图片地址等）。同时，系统预下载所有需要绘制到Canvas的图片（头像、背景、箭头、二维码），显示加载遮罩。

### 第二步：查看名片预览

图片预下载完成后，Canvas绘制完整的名片图片并展示在页面上。如果医生没有患者访问二维码，页面显示空状态提示："暂无医生二维码，请联系管理员"。

名片上二维码的引导文字来自平台全局配置（`wx-share.qr-description`），默认文字为"微信扫码，足不出户即可找我复诊开方"。

**医院名称过长处理**：如果医院名称超过20个字符，科室名称和医院名称分两行显示；不超过20字符时，用一个竖线 `|` 分隔显示在同一行。

### 第三步：保存到手机相册

点击"保存到手机"按钮：

1. 系统首先检查是否已授权相册写入权限
2. 如果未授权，弹窗提示"您没有授权，无法保存到相册"
3. 如果已授权，系统用Canvas重新绘制完整名片（600x850像素JPG格式），调用微信API保存到系统相册
4. 保存成功弹窗提示"保存成功"

### 第四步：分享名片

点击"分享名片"按钮：

1. 同样先检查相册写入权限
2. 授权通过后，Canvas绘图生成名片图片
3. 调用微信"分享图片"菜单（`showShareImageMenu`），文件名来自平台配置（如"微信扫码加入XX吧"）
4. 分享成功弹窗提示"分享成功"

### 第五步：页面离开

页面卸载时，清空所有图片缓存（`imageMap`），释放内存。

## 异常处理

| 情况 | 系统处理方式 |
|:---|:---|
| 医生没有二维码 | 页面显示空状态："暂无医生二维码，请联系管理员" |
| 未授权相册写入权限 | "保存到手机"提示"您没有授权，无法保存到相册"；"分享名片"提示"您没有授权，无法生成名片" |
| 图片下载失败 | Promise.allSettled 保证不会因单张图片失败阻塞整体流程 |
| 医院名称超长 | 自动分行显示，超过20字截断并加省略号 |
| 引导文字超长 | Canvas中自动分行，每行20字，最多3行 |
| Canvas绘制失败 | 静默失败，不会弹窗提示 |

## 关联功能

- 患者端扫码：患者扫描此二维码可进入该医生的问诊/复诊页面
- 医生基本信息编辑：名片上的姓名、科室、医院来自医生注册信息
