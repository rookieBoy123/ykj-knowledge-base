# 医生二维码名片生成与分享流程

> 文档版本 2026-07-03 | 面向读者：产品经理、运营、客服

## 概述
医生可以在小程序中生成专属的二维码名片，包含医生头像、姓名、职称、科室、医院以及患者扫码入口二维码。名片可保存到手机相册或通过微信分享给患者，患者扫码后可直接进入该医生的问诊页面。

## 触发条件
- 医生登录小程序后，在「我的」页面或个人中心找到"我的二维码"入口

## 步骤详解

### 第一步：进入二维码页面
- 系统从缓存的医生信息中读取头像、姓名、职称、科室、医院名称
- 从医生信息中获取患者扫码二维码图片地址
- 页面顶部展示医生头像（默认医生头像兜底）、背景图

### 第二步：查看名片预览
页面展示卡片式布局的名片预览，包含：
- **医生信息**：头像（圆形）、姓名+职称、科室+医院（两行排列）
- **二维码**：中央展示扫码入口二维码（正方形区域）
- **引导文字**："微信扫码，足不出户即可找我复诊开方"（可在后台配置）
- 医院名称超过20个字时，分行展示并超长截断

### 第三步：保存到手机
点击"保存到手机"按钮：
- 系统先请求相册写入权限
- 如果用户未授权，提示"您没有授权，无法保存到相册"
- 授权后：通过Canvas将名片各元素合成一张完整图片（头像、背景、文字、二维码）
- 调用微信接口保存图片到系统相册
- 保存成功后提示"保存成功"

### 第四步：分享名片
点击"分享名片"按钮：
- 同样先请求相册写入权限
- 授权后通过Canvas合成完整名片图片
- 调用微信分享图片接口，可以将名片图片发送给微信好友或微信群
- 分享成功后提示"分享成功"

## 异常处理

| 情况 | 系统处理方式 |
|------|------------|
| 医生未配置二维码 | 显示空状态页面，提示"暂无医生二维码，请联系管理员" |
| 用户拒绝相册授权 | 保存：提示"您没有授权，无法保存到相册"；分享：提示"您没有授权，无法生成名片" |
| 图片下载失败 | Canvas合成时该区域显示空白（不会阻断整体流程） |
| 二维码图片地址为空 | 不执行保存/分享操作 |
| 背景图下载失败 | Canvas合成时背景区域为白色 |

## Canvas合成逻辑（内部实现参考）
名片的合成在离屏Canvas上进行（不显示在页面中）：
1. 白色背景填充整个画布（600×850像素）
2. 绘制背景图（顶部200像素高度）
3. 绘制医生头像（圆形裁剪，100×100像素，居中偏上）
4. 绘制医生姓名和职称（32号字体，深色）
5. 绘制科室和医院名称（28号字体，灰色；医院名超20字截断加省略号）
6. 绘制二维码图片（320×320像素，居中）
7. 绘制引导箭头图标
8. 绘制底部引导文字（28号字体；支持2-3行，每行20字）

## 关联功能
- 医生入驻审核（二维码由审核通过后自动生成）
- 医生信息编辑（头像、医院等信息的更新会影响名片内容）
- 患者扫码问诊（患者扫二维码后直接进入该医生的服务页面）
