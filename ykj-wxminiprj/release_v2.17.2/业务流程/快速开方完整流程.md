# 快速开方流程（医生端）

> 文档版本：2026-07-02 | 面向读者：医生

## 概述

快速开方是医生无需关联问诊单、直接给患者开具处方的功能。它简化了传统"接诊→开方"的流程，适用于线下复诊患者或医生已了解病情的场景。支持中药和成药两种类型，可保存为草稿、存为处方模板，并包含签章、黑名单药品过滤、库存校验等功能。

## 触发条件

- **医生**在首页或处方记录页面点击"快速开方"
- **医生**从处方模板或历史处方中"复用"一张处方

## 步骤详解

### 一、页面初始化

1. **自动识别医生的科室类型**：
   - 如果医生是中医科室（medicalType=TCM），默认显示中药处方标签页在前
   - 西医科室默认显示成药处方标签页在前
   - 两个标签可手动切换："中药处方"和"成药处方"
2. **查询医生是否可设包邮**：通过gratisEnabled接口获取医生的免运费权限。
3. **患者信息加载**：
   - 如果从患者列表进入（带patientId参数），自动加载该患者的基本信息和最新病史，患者信息不可修改（只读模式）
   - 如果是"新添加"模式，患者信息需要手动选择或填写

### 二、填写处方内容

#### 中药处方

1. **选药房**：选择有权限的中药药房。
2. **加药品**：通过搜索添加中药饮片或颗粒，支持导入历史处方和模板处方。
3. **用法用量**：设置用药方式、使用频次、每次用量、药品数量、使用天数。
4. **加工工艺**：如选择代煎等加工方式，系统会根据加工标准（按重量计算）要求填写每日使用次数和每次用量。
5. **用法备注**：自由填写的用药备注信息。
6. **医嘱**：服药时间、煎药方法、用药禁忌三部分。
7. **医事服务费**：诊查费、会诊费、总费用的分别展示。

#### 成药处方

1. **选药房**：选择有权限的成药药房。
2. **加药品**：搜索并添加成药，支持导入模板和历史处方。
3. **医嘱**：自由文本的用药建议。
4. **成药诊金**：可选填诊疗费用。

### 三、保存为模板（可选）

1. **开启开关**：底部"保存为模板处方"开关打开后，需要填写模板名称。
2. **名称校验**：点击确认发送时，如果开关打开但名称为空，提示"请输入模板名称"并阻止提交。
3. **保存时机**：模板在处方保存成功前先保存——先调用保存模板接口，再调用保存处方接口。

### 四、表单校验（提交前）

提交时会执行多层校验：

1. **患者信息校验**：checkForm() 检查患者必填信息是否完整。
2. **诊断信息校验**：
   - 中医科室（isTCM=true）：必须填写辨病（diseasesResult）和辨证（dialecticalList）。如果系统配置要求（diagnoseRequireEnabled），西医诊断也必填。
   - 西医科室：必须填写西医诊断（diagnosisList）。
3. **药品列表校验**：至少添加一种药品，否则提示通过validateForm拦截。
4. **成药用法用量完整性校验**（仅对外上传时）：检查已添加的成药药品的用法用量是否填写完整，不完整提示"请将已添加的药品用法用量填写完整"。

### 五、两种提交方式

底部有两个按钮：

#### "存为草稿"

- 提交时传入 isDraft=true
- 调用 apiSaveDraft 保存草稿处方
- 成功后弹出"处方已存为草稿"提示框，显示"可在[我的-处方记录-处方草稿]查看"
- 弹窗提供两个选择：「查看处方记录」→跳转到处方记录页；「返回首页」→回到医生首页

#### "确认发送给患者"

- 提交前弹出确认对话框
- 提交时传入 isDraft=false
- 调用 saveRp 保存正式处方
- 提交中按钮显示加载状态且不可重复点击
- 成功后如果需要签章（authSign=true），触发签章流程

### 六、签章流程

1. **医网信CA签章**：如果系统配置了CA签章（cmConfig.ca.enabled），保存成功后触发签章：
   - 获取clientId（医网信客户端ID）
   - 显示签章组件（Sign组件或FxqSignPop组件）
   - 签章成功：更新处方状态
   - 签章失败：处理失败逻辑（signFailActions）
2. **无需签章**：直接返回成功。

### 七、处方校验与拦截

1. **十八反十九畏校验**：中药处方保存前校验是否存在配伍禁忌，触发时弹出核对弹窗（verifyZyDrugsPopup），医生可选择"继续保存"或"返回编辑"。
2. **有毒中药剂量超量**：同样通过弹窗提示，医生确认后继续。
3. **黑名单药品自动删除**：如果处方中包含已被加入黑名单的药品，系统自动删除并弹出提示"处方中存在黑名单药品系统已自动删除"。
4. **库存不足**：提交时检查药品库存，如果缺货或库存不足，提示"药材缺货或库存不足，请更换药材"。

### 八、草稿编辑

1. **草稿加载**：如果从处方草稿页进入（带editDraftFlag和outpatientNo参数），自动加载草稿数据填充到表单。
2. **缓存恢复**：如果页面从后台切回（onShow），从本地缓存（quickPrescription key）恢复表单数据，避免用户填写的内容丢失。
3. **缓存清除**：处方保存成功后清除缓存。

## 异常处理

| 情况 | 系统处理方式 |
|------|------------|
| 模板名称为空 | 提示"请输入模板名称"，阻止提交 |
| 患者信息不完整 | checkForm校验不通过，阻止提交 |
| 诊断信息缺失 | 提示对应的诊断字段必填 |
| 中药配伍禁忌/超量 | 弹出核对弹窗，医生确认后继续或返回编辑 |
| 药品黑名单命中 | 自动删除黑名单药品并提示 |
| 库存不足 | 提示"药材缺货或库存不足，请更换药材" |
| 提交中再次点击 | isSubmiting=true 拦截重复提交 |
| 药品用法用量不完整 | 提示"请将已添加的药品用法用量填写完整" |

## 关联功能

- **处方记录**：保存后将处方保存到处方记录列表中，可在"我的-处方记录"查看
- **处方模板**：可保存为中药或成药模板供后续复用
- **历史处方复用**：可从历史处方中选择并导入
- **CA签章**：关联医网信或放心签进行电子签章
- **药品库存管理**：关联药房库存系统进行实时库存校验
