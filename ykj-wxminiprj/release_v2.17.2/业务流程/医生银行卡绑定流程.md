# 医生银行卡绑定流程

> 文档版本：release/v2.17.2 | 面向读者：客服、运营

## 概述

医生在查看收益和结算记录时，需要绑定银行卡才能接收结算款项。银行卡绑定页面允许医生填写并提交银行账户信息，支持新增绑定和修改已绑定信息两种模式。

## 触发条件

医生在收益页面（earnings）点击「绑定银行卡」入口进入。

**前置条件**：
- 用户已登录小程序且当前角色为医生
- 平台开启了收益结算功能

## 步骤详解

### 第一步：页面初始化

进入页面后，系统首先检查是否已有绑定的银行卡：
- **无绑定记录**：进入「新增」模式（`isAdd=true`），所有输入框为空，底部按钮显示「提交」
- **有绑定记录**：进入「编辑」模式（`isAdd=false`），自动填充已有的银行卡信息，底部按钮显示「保存」

查询已有记录时，按当前医生的成员ID（`memberId`）和成员类型（`memberType=1`）查询，取第一条记录。

### 第二步：填写银行卡信息

页面包含5个输入字段：

| 字段 | 输入类型 | 说明 |
|------|---------|------|
| 银行卡号 | 数字键盘 | 银行卡号码 |
| 银行名称 | 文本 | 例如：中国工商银行 |
| 开户行名称 | 文本 | 具体到支行名称 |
| 卡主名称 | 文本 | 占位提示「卡主姓名必须与注册医生姓名一致」 |
| 身份证号码 | 文本 | 与银行卡关联的身份证号 |

所有字段右对齐，左侧为标签（宽度固定180rpx）。

### 第三步：提交验证

点击底部「提交」按钮，系统进行前后校验：

**必填校验**：
1. 银行卡号不能为空 → 提示「请填写银行卡号」
2. 卡主名称不能为空 → 提示「请填写卡主名称」

校验通过后调用绑定接口。

### 第四步：保存银行卡

- **新增模式**：调用`bindingcard`接口提交银行卡信息
- **编辑模式**：调用`editcard`接口修改银行卡信息

成功后提示「操作成功」，1.5秒后自动返回上一页（收益页）。

## 异常处理

| 情况 | 系统处理方式 |
|------|------------|
| 银行卡号为空 | 提示「请填写银行卡号」 |
| 卡主名称为空 | 提示「请填写卡主名称」 |
| 银行名称/开户行/身份证号为空 | 不拦截（非必填校验），但可能在后端校验中失败 |
| 接口调用失败 | 后端返回错误信息（具体提示取决于后端） |
| 已有银行卡时自动填充 | 编辑模式直接展示已有信息 |

## 关联功能

- ← **医生收益页**（earnings/index）：查看收益余额后绑定银行卡
- → 操作成功后返回收益页

## 特殊说明

**卡主名称校验**：占位文字提示「卡主姓名必须与注册医生姓名一致」，这意味着后端会校验卡主姓名与医生注册姓名的匹配关系。如果两者不一致，提交可能失败。

**字段非全必填**：前端仅对银行卡号和卡主名称做了非空校验，银行名称、开户行名称、身份证号码可为空。但后端的校验规则可能更严格。

**数据隔离**：银行卡信息按医生的`memberId`隔离存储，每个医生只能绑定自己的银行卡。查询时仅取`records[0]`（第一条记录），说明目前每个医生只能绑定一张银行卡。
