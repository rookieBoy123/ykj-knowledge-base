# 随访计划全生命周期管理流程（医生端+患者端）

> 文档版本：release/v2.17.2 | 面向读者：客服、运营、产品经理

## 概述

医生创建随访计划并发送给患者后，系统按计划自动向患者推送随访任务。医生和患者都可以查看看随访计划的执行进度，每个随访任务包含患教资料、提醒和问卷三种类型的内容。随访计划经过进行中→已完成/已取消的完整生命周期。

## 触发条件

- 医生端：从"随访管理"入口进入查看所有随访计划
- 医生端：从某位患者详情页进入，只查看该患者的随访计划
- 患者端：从"我的随访"进入查看自己的随访计划

## 前置条件

- 医生已创建至少一个随访计划
- 患者已与医生建立就诊关系（医生已对该患者发起过随访）

## 步骤详解

### 第一步：随访计划列表

医生端进入"随访计划"页面，顶部四个选项卡按状态分类：

| 选项卡 | 对应状态码 | 说明 |
|-------|----------|------|
| 全部 | 不传status | 展示所有随访计划，显示总条数 |
| 进行中 | status=0 | 正在执行的随访计划（默认选中） |
| 已完成 | status=1 | 所有任务已完成的计划 |
| 已取消 | status=2 | 医生或系统取消的计划 |

列表特点：
- 每页10条，滚动到底加载更多
- 如果从患者详情页进入（携带 `patientId`），只展示该患者的随访计划
- 每个计划卡片展示：计划名称、患者姓名和头像（头像按性别自动切换）、计划状态标签
- 点击卡片跳转"随访计划详情页"

### 第二步：随访计划详情（医生端）

进入详情页后，从上到下展示：

**患者信息区**：头像、姓名、性别、年龄

**计划信息区**：
- 计划名称
- 就诊时间（关联的就诊日期）
- 每个随访节点（任务卡片），按时间顺序排列，每个节点显示：
  - 计划发送时间（由就诊时间+间隔计算）
  - 状态标签（待随访/已随访/已完成）
  - 包含的内容项（患教资料/提醒/问卷的名称）

**底部操作按钮**（根据状态动态变化）：

| 随访状态 | 医生端按钮 | 说明 |
|---------|-----------|------|
| 进行中（status=0） | 取消随访按钮 | 点击弹出取消原因输入框，必须填写原因 |
| 已完成（status=1）无总结 | "填写随访总结"按钮 | 点击弹出总结输入框 |
| 已完成（status=1）有总结 | "查看随访总结"按钮 | 点击查看已填写的总结 |
| 已取消（status=2）有取消原因 | "查看取消原因"按钮 | 点击查看取消原因 |

**取消随访操作**：
1. 点击"取消随访"→弹出输入框
2. 必须填写取消原因（不限字数）
3. 确认后弹出二次确认："确认取消该患者的随访计划吗？确认后，该随访计划将不再发送给患者。"
4. 确认后调用取消接口，成功后显示"已取消"并1.5秒后自动返回

**填写随访总结**：
1. 点击"填写随访总结"→弹出输入框
2. 填写总结文本
3. 保存后按钮变为"查看随访总结"

### 第三步：随访任务详情

点击某个随访任务节点进入任务详情页：

**患者端视角**：
- 顶部显示医生信息（头像、姓名、科室、职称）
- 三个内容区域（有数据才显示）：
  - **患教资料**：点击"详情"查看学习内容
  - **提醒**：显示提醒文本
  - **问卷**：点击"详情"跳转问卷填写页
- 每个内容区域显示完成状态（红色"待办"或灰色"完成"）
- 患者打开任务详情时，系统自动将所有"提醒"类内容标记为已完成（不需要手动操作）

**医生端视角**：
- 顶部显示患者信息（头像、姓名、性别、年龄）
- 三个内容区域同上
- 每个区域的完成状态
- 底部可能显示"随访小结"区域：
  - 未填写小结：显示蓝色"填写小结"按钮
  - 已填写小结：显示小结内容和填写时间

### 第四步：患者端查看随访任务

患者从"我的随访"进入，可以看到：
- 所有医生发起的随访计划
- 每个计划下的待办任务和已完成任务
- 点击任务查看详情，完成患教学习和问卷填写

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 取消随访未填原因 | 提示"请填写取消原因"，不执行取消 |
| 填写总结未填内容 | 提示"请填写随访总结"，不保存 |
| 随访计划列表为空 | 显示空状态"暂无随访计划" |
| 随访任务列表为空 | 不显示对应内容区域（v-if控制） |
