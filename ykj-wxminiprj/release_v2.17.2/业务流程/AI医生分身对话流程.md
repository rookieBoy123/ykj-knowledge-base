# AI医生分身对话流程

## 触发条件
患者从AI医生详情页点击「AI分身问诊」按钮，或从专家团队的AI分身入口进入。

## 前置条件
- 患者已登录
- 医生已创建AI分身（在AI分身设置中配置了知识库和对话风格）
- 问诊单已生成（有`inquiryNo`作为业务编号）

## 步骤详解

### 1. 进入AI对话页
页面加载时：
1. 根据问诊单号查询问诊基础信息（医生信息、患者信息）
2. 查看当前问诊是否存在AI对话记录
3. **首次进入（无对话记录）**：系统自动创建AI问诊单，然后自动发送第一条消息（内容为`BEGIN`），触发AI开始问诊
4. **有历史对话**：加载之前的对话记录

### 2. AI问诊对话
对话流程如下：
- **AI发起提问**：AI根据医生的知识库和预设的问诊风格，向患者提问（如询问症状、病史等）
- **患者回答**：患者在输入框中输入答案，支持发送文字和图片
- **AI继续追问**：AI根据患者的回答继续深入询问
- **流式响应**：AI的回复以打字机效果逐字显示（流式输出技术）

### 3. 常见问题引导
在AI完成一轮问诊后，系统会自动生成「常见问题」引导选项：
- 页面底部出现3-5个常见问题标签
- 患者可以点击选择，也可以点选多个问题
- 选择后自动发送给AI
- 患者也可以不选，直接在输入框中自由输入

### 4. 问诊状态流转

| 状态值 | 含义 | 页面表现 |
|-------|------|---------|
| 0 | 问诊进行中 | 输入框可用，可以继续对话 |
| 1 | 问诊结束，未生成报告 | 显示「分析中」加载，自动生成预问诊报告 |
| 2 | 问诊结束，已生成报告 | 显示完整报告卡片 + 「问诊已结束」提示 |

### 5. 问诊结束后的处理

**自动生成报告（从状态1到状态2）**：
- 由**创建该问诊单的会员**才能触发生成报告
- 其他访客只能看到对话记录，不会生成报告
- 生成过程中显示全屏加载动画「正在生成报告，请稍候」
- 报告以卡片形式插入到对话记录末尾

**展示专家团队医生**：
- 问诊结束后，在报告卡片下方展示「温馨提示」：如需开药或进一步问诊可选择专家团队的医生问诊
- 列出该AI分身医生的专家团队成员
- 点击团队医生可以跳转到真人医生详情页，发起真人问诊

### 6. 返回逻辑
- 点击左上角返回：正常返回上一页
- 如果从特殊入口进入（`goHome=1`）：返回到患者首页

## 异常处理

| 异常场景 | 系统行为 |
|---------|---------|
| 缺少问诊单号 | 弹窗提示「缺少问诊单」，确认后返回上一页 |
| 查询问诊信息失败 | Promise reject，页面不显示内容 |
| 创建AI问诊失败 | Promise reject，停留在加载状态 |
| 流式响应出错 | 停止loading状态，catch错误日志 |
| 发送消息时缺少问诊ID | Toast提示「缺少问诊ID」 |
| 查询团队医生失败（无团队成员） | 不展示团队医生区域 |

## AI对话特点
- **不中断体验**：对话历史持久保存，退出后再进入可继续
- **智能追问**：AI根据知识库自动引导症状描述
- **人工转接**：AI无法处理时，患者可选择联系真人医生
- **不可开方**：AI分身不能开具处方，仅做预问诊和信息收集
- **全天候**：AI分身24小时可用，不受医生在线状态限制

## 关联功能
- 跳转到「AI医生详情页」→ 返回选择问诊方式
- 跳转到「真人医生主页」→ 查看专家团队成员并发起问诊
- 跳转到「AI问诊记录」→ 查看历史AI问诊
- 关联「AI预问诊流程」→ 预问诊与医生分身的AI模型共享

## 重要说明
- AI分身的功能开关由医生在「AI分身设置」中控制
- AI分身的回答质量取决于医生配置的知识库丰富程度
- AI分身不会给出最终的诊断结论，仅做信息收集和病情初筛
- 创建问诊单的会员才能生成报告，避免多人操作干扰
- 最终诊断和处方必须由真人医生开具
