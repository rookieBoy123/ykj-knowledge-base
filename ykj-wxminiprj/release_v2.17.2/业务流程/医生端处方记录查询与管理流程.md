# 医生端处方记录查询与管理流程

> 文档版本 v1.0 | 面向读者：产品经理、运营人员

## 概述
医生查看自己开具的所有处方记录，支持按状态筛选、按患者姓名搜索，对处方进行删除、撤回、电子签名、驳回、分享等操作。

## 触发条件
- **前置条件**：医生已登录
- **入口路径**：医生工作台 → 处方记录（或问诊中查看"已开处方"）
- **两种模式**：
  - 处方记录：显示该医生的所有处方，带筛选和搜索
  - 已开处方：查看某次问诊中已开具的处方，不显示筛选条

## 步骤详解

### 第一步：查看处方列表
页面展示处方卡片列表，每张卡片显示处方关键信息（通过 prescridetail 子组件渲染）：

- **状态标签**：审核通过、审核中、待审核、审核不通过、开方中、已撤回、处方草稿
- 每次进入页面或切换筛选条件时，从第一页重新加载（每页15条）
- 滚动到底部自动加载更多

### 第二步：按状态筛选
页面顶部有筛选标签栏，支持以下状态筛选：

| 筛选标签 | 对应处方状态 |
|---------|------------|
| 全部处方 | 不筛选，显示所有 |
| 审核通过 | 药师已审核通过 |
| 审核中 | 药师正在审核 |
| 待审核 | 等待药师审核 |
| 审核不通过 | 药师审核驳回 |
| 开方中 | 处方正在创建中 |
| 已撤回 | 医生主动撤回 |
| 处方草稿 | 暂存未提交的处方 |

支持按患者姓名/手机号搜索，切换筛选条件自动回到顶部并重新加载。

### 第三步：处方操作
根据处方状态，医生可执行不同操作：

- **查看详情**：点击处方卡片，跳转到处方详情页（packagePatient1/drugsuggest/detail）
- **删除处方**：弹出二次确认弹窗"确认删除该处方吗？删除之后无法恢复"，确认后删除
- **撤回处方**：弹出二次确认弹窗"确认撤回吗？"，确认后撤回（撤回成功后1秒自动刷新列表）
- **驳回处方**（审批场景）：弹出底部面板，输入驳回原因（限500字），点击确认后驳回
- **发送给会员**：如果患者绑定了多个微信，弹出选择弹窗让医生选择发送到哪个微信

### 第四步：电子签名
部分处方需要医生电子签名，支持三种CA签章方式：

- **医网信**：严格模式下需先通过人脸识别验证，然后再调用医网信SDK签名
- **源康健PaaS**：嵌入第三方小程序签名
- **放心签**：弹出签名图片弹窗，医生手写签名

签名成功后刷新处方列表；签名失败或取消签名时保留当前页面状态。

### 第五步：分享处方
医生可将处方分享给患者：

- 点击分享，生成微信分享卡片
- 分享标题格式："XX医生为「患者名」开具的处方"
- 患者点击后跳转到处方详情页查看

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 处方列表为空 | 显示空状态插图 + "暂无记录" |
| 撤回失败 | 弹窗提示"网络异常，请稍后再试" |
| 驳回时未填原因 | Toast提示"请输入驳回原因" |
| CA未认证 | 引导医生先完成CA认证 |
| 人脸识别未通过 | 不允许调用医网信签名 |
| 放心签签名失败 | Toast提示"签字失败，请重试" |
| 接口异常 | 列表清空，弹窗提示错误信息 |

## 关联功能
- 处方详情（点击处方跳转）
- 快速开方（新建处方）
- CA签章管理（认证设置）
- 问诊接诊（从问诊中查看已开处方）
- 药品下单（患者端从分享链接进入）
