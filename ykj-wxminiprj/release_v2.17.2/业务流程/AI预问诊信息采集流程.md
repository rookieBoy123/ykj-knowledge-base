# AI预问诊信息采集流程

> 文档版本 | 面向读者：患者

## 概述

在正式问诊前，AI助手会与患者进行一轮自动对话，提前采集患者的症状、病史等关键信息，生成预问诊报告。医生接诊时可以直接查看报告，提高问诊效率，减少医生重复询问基础信息的时间。

## 触发条件

- **前置条件**：医生开通了AI预问诊功能，患者提交问诊申请
- **入口路径**：从问诊订单进入预问诊页面，参数携带问诊编号

## 步骤详解

### 第一步：进入预问诊

页面顶部显示"预问诊"标题（如问诊已结束则显示"预问诊对话记录"），顶部有倒计时组件提示当前问诊的剩余时间。

系统自动加载问诊基础信息（医生、患者），并启动AI对话会话：
- 若尚未开始：调用启动接口创建预问诊会话
- 若已有记录：加载历史对话，恢复上次进度

### 第二步：智能问答交互

AI以聊天形式逐步采集患者信息。对话流程与AI医生分身类似，但增加了两个特殊功能：

**1. 常见问题推荐**
AI每次回复后，如对话还在进行中，会自动显示3个与当前对话相关的常见问题供患者选择点击。这些问题按钮可以帮助患者更便捷地描述病情，点击后自动填入输入框。

**2. 选项弹窗（预问诊特有）**
当AI判断需要通过多选或单选来明确信息时，页面底部会弹出一个选项面板，替换掉输入框：
- 面板顶部显示"请选择适合的描述"引导文字
- 列出多个结构化选项（如症状部位、疼痛类型等），患者点击选择
- 选择后自动作为回答发送，AI继续下一轮提问
- 选项面板关闭后，输入框重新显示

### 第三步：问诊结束

AI完成信息采集后，对话状态变更：

1. **状态变为"问诊已结束"**（pcStatus=1）：自动生成预问诊报告，显示AI报告卡片
2. **显示进入问诊室提示**："感谢您的配合，现在您可以进入问诊室等待医生接诊"
3. **倒计时自动进入**：底部出现蓝色按钮"进入问诊室(3s后自动进入)"，3秒倒计时结束后自动跳转到问诊聊天室
4. **手动进入**：患者可主动点击按钮立即进入，无需等待倒计时

### 第四步：查看预问诊报告（已完成状态）

若问诊已完成且报告已生成（pcStatus=2），页面会展示：
- 完整的对话记录
- AI报告卡片
- "进入问诊室"按钮（带动画倒计时引导）

## 异常处理

| 情况 | 系统处理方式 |
|------|------------|
| 缺少问诊编号参数 | 弹出"温馨提示——缺少问诊单"提示，确认后返回上一页 |
| 医生/患者信息获取失败 | 静默处理，界面无异常提示但功能可能受限 |
| 问诊ID缺失时发送消息 | 弹出Toast提示"缺少问诊ID" |
| AI返回选项类型消息 | 底部输入框切换为选项面板，显示可选描述列表 |
| AI返回REPORTED状态 | pcStatus切换为2，结束对话，显示进入问诊室按钮 |
| AI流式响应END | pcStatus切换为1，生成报告，展示提示语和倒计时按钮 |
| 流式传输失败 | 静默处理（catch块仅记录日志），不中断用户操作 |
| 报告生成失败 | 静默处理，不显示任何错误提示 |
