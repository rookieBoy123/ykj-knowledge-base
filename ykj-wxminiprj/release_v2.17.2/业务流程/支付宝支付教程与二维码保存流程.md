# 支付宝支付教程与二维码保存流程

> 文档版本 2026-07-05 | 面向用户：患者

## 概述

当患者在收银台选择"支付宝"支付方式后，系统会生成一张支付二维码，并跳转到支付宝支付教程页面。该页面引导用户逐步完成：保存二维码到相册 → 打开支付宝扫一扫 → 选择相册中的二维码 → 完成支付。

## 触发条件

- **前置条件**：已生成待支付订单，在收银台页面选择了"支付宝"支付渠道
- **入口路径**：收银台页面（hisPay）→ 选择支付宝支付 → 自动跳转

## 步骤详解

### 第一步：查看支付教程

用户进入页面后看到两部分指引：

1. **步骤一：长按保存二维码至相册**
   - 页面顶部展示支付二维码图片（长按可保存）
   - 下方提供"保存二维码到相册"按钮（点击自动下载并保存到手机相册）
   - 保存成功后提示"保存成功"，失败则提示"保存失败"

2. **步骤二：打开支付宝扫一扫**
   - 指导用户打开支付宝App
   - 进入扫一扫功能，从相册中选择刚保存的二维码完成支付

### 第二步：等待支付完成

页面底部有一个蓝色"支付完成"按钮。用户完成支付宝付款后，可点击该按钮手动查询支付结果。

**系统自动轮询**：页面每 3 秒自动查询一次支付状态。当检测到支付成功时，自动返回上一页面并通知收银台刷新订单状态。

### 第三步：返回收银台

支付成功后，系统自动回退到收银台页面。收银台收到支付完成通知后，将跳转到订单详情或问诊页面。

**中途返回**：如果用户点击页面左上角返回按钮，系统会停止轮询并通知收银台页面启动自己的支付状态查询（每3秒轮询），确保不会漏掉支付结果。

## 异常处理

| 情况 | 系统处理方式 |
|------|------------|
| 缺少支付二维码参数 | 提示"缺少支付二维码"，无法进入页面 |
| 二维码保存失败 | 提示"保存失败"，用户可重试或长按手动保存 |
| 支付结果查询超时 | 返回上一页后收银台继续轮询，不会丢失支付结果 |
| 用户在支付宝完成支付但页面未检测到 | 点击"支付完成"按钮手动触发查询，或等待下次自动轮询（3秒间隔） |
| 页面隐藏（切换到其他App） | 停止轮询以节省资源，切回时自动恢复轮询 |
| 用户按返回离开 | 通过事件通道通知收银台启动轮询，不丢失支付状态 |

## 关联功能

- **收银台页面**（hisPay）：支付渠道选择、倒计时、订单确认
- **支付状态查询接口**：`paymentStatus` — 后端主动查询支付宝/微信支付网关
- **订单过期接口**：倒计时归零后触发订单失效
- **支付宝支付后端**：`AlipayPaymentService` + `OrderCommandServiceImpl.preparePayment()`
