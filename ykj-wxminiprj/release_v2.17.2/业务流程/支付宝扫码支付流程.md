# 支付宝扫码支付流程

> 文档版本：v1.0 | 最后更新：2026-07-04 | 面向读者：产品经理、运营人员、客服

## 概述

支付宝扫码支付是收银台新增的支付渠道。用户选择支付宝支付后，系统生成一个支付宝收款二维码，用户用支付宝App扫码完成付款。页面同时提供支付状态自动检测和手动确认两种方式。

## 触发条件

- **前置条件**：已生成订单，且在收银台选择了"支付宝支付"
- **入口路径**：收银台 → 选择支付宝 → 跳转到支付宝教程页
- **页面参数**：`orderNo`（订单号）、`codeUrl`（支付宝收款二维码地址）

## 步骤详解

### 第一步：进入支付宝支付教程页

页面顶部显示标题"支付宝支付流程"，下方以步骤导航条展示三个核心步骤：
1. 保存二维码 → 2. 打开支付扫一扫 → 3. 选择保存的二维码支付

### 第二步：保存收款二维码

页面中央展示一张支付宝收款二维码图片，用户可以：
- **长按保存**二维码图片到手机相册
- **点击"保存二维码到相册"按钮**快速保存

保存成功提示"保存成功"，保存失败提示"保存失败"。

### 第三步：打开支付宝扫码

用户切换到手机上的支付宝App：
1. 打开支付宝"扫一扫"功能
2. 从相册中选择刚才保存的二维码
3. 完成支付

页面上有支付宝扫码的操作示意图，引导用户操作。

### 第四步：支付结果确认

页面底部有"支付完成"按钮，用户完成支付宝付款后点击该按钮：
- 系统自动查询订单支付状态
- 如果支付成功，自动返回上一页并通知收银台支付完成
- 如果暂未查到支付结果，提示"暂未查询到支付结果！"并继续轮询

### 自动轮询机制

页面进入后会自动每 **3 秒**查询一次支付状态：
- 检测到支付成功 → 自动返回并通知收银台
- 未检测到 → 继续轮询
- 用户离开页面（切到后台或返回）→ 暂停轮询
- 用户回到页面 → 自动恢复轮询

### 返回处理

当用户点击返回按钮离开支付宝支付页面时：
- 停止支付状态轮询
- 通知收银台页面开始轮询（由收银台接管支付状态检测）
- 确保用户即使离开了本页面，支付完成也能被检测到

## 异常处理

| 情况 | 处理方式 |
|------|---------|
| 缺少支付二维码参数 | 提示"缺少支付二维码" |
| 二维码图片下载失败 | 提示"保存失败" |
| 二维码保存到相册失败 | 提示"保存失败" |
| 点击"支付完成"但未查到结果 | 提示"暂未查询到支付结果！"，继续轮询 |
| 用户返回收银台 | 停止本页轮询，通知收银台接管轮询 |
| 用户切换到后台 | 暂停轮询（onHide），回来时恢复（onShow） |

## 关联功能

- **收银台统一支付**：选择支付宝后跳转到本页，支付完成后回传结果
- **订单管理**：支付成功后跳转订单详情
- **支付宝配置**：依赖后台配置支付宝应用ID、公私钥等参数
