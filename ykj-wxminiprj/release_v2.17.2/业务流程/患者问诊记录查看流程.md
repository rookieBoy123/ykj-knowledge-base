# 患者问诊记录查看流程

> 文档版本 v1.0 | 面向读者：医生、运营人员

## 概述

医生在小程序端查看自己名下所有患者的问诊记录列表，了解每位患者的问诊状态、病情描述和最近一条消息内容。点击任意记录可进入对应的问诊聊天室继续对话。

## 触发条件

- **前置条件**：医生已登录小程序医生工作台
- **入口路径**：医生工作台 → 问诊记录（底部导航栏或工作台入口）

## 步骤详解

### 第一步：进入问诊记录列表

医生进入页面后，系统自动查询该医生名下的所有问诊记录（每页20条），以列表形式展示。

每条问诊记录包含以下信息：
- **患者头像**（根据性别显示默认男/女头像，如有自定义头像则显示真实头像）
- **患者姓名**、**性别**和**年龄**
- **服务类型标签**（如"图文问诊""视频问诊"等）
- **问诊状态**：右上角显示三种状态之一——「问诊中」「问诊结束」「已取消」
- **病情描述/咨询问题**：展示患者填写的病情主诉（视频问诊显示"咨询问题"，其他类型显示"病情描述"）
- **最近一条消息预览**：自动格式化为可读文本（语音→显示"[语音]"，图片→显示"[图片]"，视频→显示"[视频]"，视频通话→显示"[问诊视频]"）
- **预约时间段**（如有）：显示问诊的开始和结束时间

### 第二步：查看问诊状态

系统支持三种问诊状态，每种状态的处理逻辑不同：

| 状态 | 显示内容 | 可操作 |
|------|---------|--------|
| 问诊中（20） | 病情描述 + 最近消息 | 可点击进入聊天 |
| 问诊结束（30） | 病情描述 + 最近消息 | 可点击进入聊天 |
| 已取消（40） | 取消原因 + 备注 | 可点击进入聊天 |

对于已取消的问诊，系统会显示取消原因（remark字段），如果未填写原因则显示"暂无"。

### 第三步：点击进入问诊聊天

点击任意一条问诊记录，系统会调用公共方法进入对应的问诊聊天室。进入聊天室后可以：
- 查看完整对话记录
- 继续发送消息（问诊中状态）
- 查看已开具的处方、订单等关联信息

### 第四步：滚动加载更多

当列表滚动到底部时，系统自动加载下一页数据（每次20条），直到全部数据加载完毕。底部会显示加载状态指示器。

### 第五步：空状态处理

如果医生名下没有任何问诊记录，页面显示「暂无记录」的空状态提示。

## 消息格式化规则

系统对最后一条消息内容进行智能格式化：

| 原始内容特征 | 显示文案 |
|------------|---------|
| JSON解析后flowStatus=20 | "预约时间成功，请提前5分钟进入诊室" |
| 包含outpatientNo | "你有一条订单消息" |
| 其他JSON内容 | "你有一条未读消息" |
| 以.mp3结尾 | "[语音]" |
| 以.jpg/.png/.jpeg结尾 | "[图片]" |
| 以.mp4结尾 | "[视频]" |
| 包含roomID | "[问诊视频]" |

## 异常处理

| 情况 | 系统处理方式 |
|------|------------|
| 患者头像为空 | 根据性别显示默认头像（男/女） |
| 没有预约时间 | 不显示时间段信息 |
| 已取消但没有取消原因 | 显示"暂无" |
| 最后一条消息为空 | 显示"暂无消息" |
| 下拉刷新 | 列表重新加载第一页数据 |

## 关联功能

- 点击记录 → 进入问诊聊天室（问诊对话功能）
- 问诊聊天室 → 可跳转到处方详情、订单详情等
