# 快速开方流程（医生端）

> 面向医生 | 小程序医生端核心功能

## 概述

医生无需经过完整的问诊流程，直接为患者快速开具中药或成药处方。支持新建处方、编辑草稿处方、引用处方模板和历史处方，以及CA电子签章、保存为模板等功能。

## 触发条件

- **前置条件**：已登录医生账号
- **入口路径**：医生端小程序 → 快速开方

## 步骤详解

### 第一步：选择处方类型

页面顶部提供两个标签页切换：
- 中医科室的医生默认显示"中药处方"标签在前
- 西医科室的医生显示"成药处方"标签（中医科医生才显示双标签）

### 第二步：填写患者信息

页面首先展示患者信息区域：
- **新建开方**：医生需要手动选择或输入患者信息（姓名、性别、年龄等）
- **续方/编辑**：患者信息自动带入且锁定不可修改（只读状态）

当选好患者后，系统自动拉取该患者的**最新病史信息**（过敏史、既往史等），填充到患者信息中。病史字段包括：
- 过敏史、既往病史、既往史

### 第三步：填写诊断信息

诊断区域根据医生科室类型显示不同内容：

**中医科室医生**：
- 中医辨病（必填）
- 中医辨证（必填）
- 西医诊断（根据系统配置决定是否必填，由 `diagnoseRequireEnabled` 控制）

**西医科室医生**：
- 西医诊断（必填）

**诊断数据来源优先级**：
1. 如果从病历或问诊卡带入的数据已有诊断，则使用带入的诊断
2. 如果当前临床诊断区域已有数据，则不覆盖
3. 否则，使用处方模板或历史处方中携带的诊断数据

### 第四步：添加药品（核心步骤）

#### 中药处方：
- 选择药房（需校验医生是否对该药房有授权，以及药房的库存管理是否启用）
- 添加中药药品，设置剂量
- 配置用法用量：用药方式、使用方法、用药天数
- 配置加工工艺（如有）：按剂收费或按重量收费
- 填写医嘱：用药时间、煎煮说明、用药禁忌
- 医生可设置运费免除（需系统配置允许）

#### 成药处方：
- 选择药房（同中药校验逻辑）
- 添加西药/中成药品
- 填写药品用法用量（对外上传场景下必须填写完整）
- 填写医嘱
- 医生可设置运费免除

**药品导入功能**：可通过以下方式快速导入药品：
- **处方模板导入**：从"我的处方模板"中选择
- **历史处方导入**：从患者历史处方中选择
- **公共药方导入**：使用系统公共药方

### 第五步：药品校验

提交前系统自动进行多项校验：

**中药校验**：
- 检查中药数量是否都填写了（不能为0或空）
- 调用"中药合理性校验"接口，检查十八反十九畏、超量、有毒中药剂量等
- 如有问题，弹出校验弹窗展示详情，医生可选择"去修改"或"确认提交"

**成药校验**：
- 检查药房库存是否充足，缺货时提示"药品缺货或库存不足，请更换药品"
- 对外上传场景下检查用法用量完整性
- 自动删除处方中的黑名单药品，并弹出提示

### 第六步：选择保存方式

页面底部有两个操作按钮：

| 按钮 | 说明 |
|------|------|
| 存为草稿 | 保存为草稿处方，不发送给患者，可在"我的-处方记录-处方草稿"中继续编辑 |
| 确认发送给患者 | 正式开方并发送给患者 |

两个操作都会先进行完整的表单校验和药品校验。

### 第七步：处方模板（可选）

在提交前，医生可选择"保存为模版处方"开关，开启后需输入模板名称。提交时系统会同时：
1. 保存/发送处方给患者
2. 将该处方的药品组合保存为个人处方模板

### 第八步：CA电子签章（如需）

如果系统配置了CA签章且本次处方需要签署：
- 系统获取签署信息（平台类型：医网信/源康健PaaS/放心签）
- 根据平台类型调用对应的签章组件：
  - **医网信/源康健PaaS**：小程序内嵌签章页面
  - **放心签**：调用放心签弹窗组件

签名成功后自动完成处方提交。签名失败时：
- 用户取消操作：跳转到处方详情页（处方已保存但未签名）
- 其他错误：提示"CA签名异常，请稍后再试"，保留页面数据供重试

### 第九步：处方提交完成

- **正式开方成功**：跳转到"开方成功"页面，展示处方编号
- **草稿保存成功**：弹出"处方已存为草稿"提示弹窗，提供两个操作：
  - "查看处方记录"：跳转到处方记录页（含草稿列表）
  - "返回首页"：跳转到医生端首页

### 第十步：页面缓存与恢复

快速开方页面支持**表单数据缓存**：
- 离开页面时（`onHide`/`onUnload`），自动将当前表单数据（患者信息、诊断、药品数据等）缓存到本地
- 下次进入时自动恢复缓存数据（仅当医生科室类型未变化时恢复）
- 处方提交成功后清除缓存
- 签名异常时保留缓存，方便下次进入继续操作

## 异常处理

| 情况 | 系统处理方式 |
|------|------------|
| 患者信息未填写完整 | 表单校验不通过，提交按钮状态重置 |
| 诊断信息缺失 | 根据中医/西医类型和配置提示相应缺失字段 |
| 药品数量为空或≤0 | 提示"药材：【XXX】请输入数量！" |
| 药房未授权当前医生 | 不显示该药房选项，无法选择 |
| 药房库存管理未启用 | 视为药房不可用 |
| 药品缺货或库存不足 | 提示"药品缺货或库存不足，请更换药品" |
| 中西药不合理（十八反十九畏/超量/有毒） | 弹出校验弹窗，医生可确认或修改 |
| 处方模板名称未填写 | 提示"请输入模板名称" |
| 对外上传时用法用量不完整 | 提示"请将已添加的药品用法用量填写完整" |
| CA签名异常 | 提示"CA签名异常，请稍后再试"，保留页面数据 |
| 用户取消CA签名 | 跳转到处方详情页（处方已保存） |
| 提交按钮重复点击 | 通过`isSubmiting`标志位防止重复提交 |
| 黑名单药品自动删除 | 弹出提示"处方中存在黑名单药品系统已自动删除" |

**处方编号错误码处理**：
- 正常：开方成功
- `400422`：中药校验不通过，打开校验弹窗展示问题药品
- 其他：弹出"温馨提示"，显示服务端返回的错误信息

## 业务规则

1. **操作类型区分**：`add`（新建）/ `edit`（编辑），影响API调用路径
2. **编辑草稿标识** `editDraftFlag`：为"1"时走"新增处方建议"接口而非"编辑处方"接口
3. **中医科判定**：`medicalType === 'TCM'` 判断当前医生是否为中医科，决定标签页顺序和诊断必填逻辑
4. **运费免除**：系统配置 `gratisEnabled` 控制医生是否可见此选项
5. **CA签章**：处方提交后服务端返回 `authSign` 标识是否需要电子签章
6. **处方模板数据类型**：1=历史处方、2=处方模板、3=公共处方、4=快速续方
7. **草稿撤回再编辑**：草稿状态（flowStatus=20）和已撤回（flowStatus=31）的处方可重新进入快速开方编辑

## 关联功能

- **处方模板管理**：可从快速开方跳转到模板列表选择模板
- **历史处方**：可从快速开方跳转到患者历史处方列表
- **处方记录（含草稿）**：保存草稿后从弹窗跳转
- **药品需求开方**：与快速开方共享药房校验和药品组件
- **CA电子签章**：复用签名组件，支持多平台
