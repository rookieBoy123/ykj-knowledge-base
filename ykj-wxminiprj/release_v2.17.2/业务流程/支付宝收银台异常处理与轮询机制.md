# 支付宝收银台异常处理与轮询机制

> 文档版本：2026-07-05 | 面向读者：产品经理、运营人员

## 概述

用户选择支付宝支付后，系统会跳转到支付宝教程页面展示支付二维码。用户在支付宝中扫码支付后，系统通过定时轮询自动检测支付结果。本文档详细描述这个过程中可能出现的各种异常情况及其处理方式。

## 触发条件

- **前置条件**：用户在收银台页面选择了支付宝支付，并点击了"确认支付"
- **入口路径**：患者端 → 收银台 → 选择支付宝 → 确认支付 → 跳转支付宝教程页

## 步骤详解

### 第一步：收银台生成支付宝二维码

1. 用户在收银台页面看到支付方式列表（微信/支付宝/医保等），**只有后台配置开启了支付宝支付时，支付宝选项才会出现**
2. 用户点击"确认支付"按钮后，系统向后端发起预支付请求
3. 后端返回支付二维码（codeUrl），小程序跳转到支付宝教程页
4. 如果二维码生成失败（codeUrl 为空），收银台弹出提示："支付二维码生成失败，请稍后重试！"

### 第二步：支付宝教程页展示

1. 教程页展示清晰的支付操作指引：
   - 第一步：长按二维码保存到相册
   - 第二步：打开支付宝扫一扫 → 选择相册中的二维码完成支付
2. 页面底部有"支付完成"按钮，供用户手动确认

### 第三步：自动轮询检测支付结果

1. 进入教程页后，系统立即开始定时轮询（每 3 秒一次），主动向后端查询订单支付状态
2. 检测到支付成功后，自动返回到收银台页面，并通过事件通道通知"支付完成"
3. **页面隐藏时（如切换到支付宝 App），轮询自动停止**；返回小程序页面时自动恢复

### 第四步：用户手动确认支付

1. 用户在支付宝完成支付后，可点击底部"支付完成"按钮
2. 系统向后端查询支付状态：
   - 已支付 → 自动返回收银台，触发支付成功流程
   - 未查询到 → 弹出提示"暂未查询到支付结果！"，继续轮询

### 第五步：用户返回收银台

1. 从教程页返回时（点击返回或左滑），轮询停止
2. 系统通知收银台页面**重新开始轮询**，收银台会主动查询支付结果

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 二维码生成失败 | 收银台弹出 Toast："支付二维码生成失败，请稍后重试！" |
| 订单已被其他人支付（错误码 40001） | 弹出提示"该订单已支付成功"，点击"知道了"后返回 |
| 订单已失效（错误码 40002） | 弹出提示"订单已取消关闭"，区分药品订单和普通订单提示文案 |
| 预支付失败（错误码 40003） | 弹出 Toast："支付失败，请稍后重试！" |
| 预支付其他错误 | 弹出支付失败弹窗（含联系客服按钮） |
| 倒计时结束后查询 | 先检查订单是否已支付 → 已支付走成功流程 → 未支付通知后端订单过期 |
| 查询支付结果失败 | 弹出"查询支付结果失败"弹窗，提供"重试"和"联系客服"按钮 |

## 轮询机制精要

| 维度 | 说明 |
|------|------|
| 轮询间隔 | 每 3 秒一次 |
| 停止条件 | 检测到支付成功 / 用户返回 / 页面隐藏 / 页面卸载 |
| 恢复条件 | 页面重新显示时自动恢复；从教程页返回时收银台接管轮询 |
| 切换支付方式 | 从支付宝切换到其他支付方式时，轮询自动停止 |
| 轮询载体 | 教程页独立轮询 + 收银台独立轮询，"返回收银台"时收银台接管 |

## 关联功能

- **收银台支付流程**：支付宝是收银台支付方式之一
- **倒计时机制**：支付倒计时归零后自动触发订单过期
- **订单过期取消**：倒计时结束 → 检查支付状态 → 未及时支付则标记订单失效
