# 医保预缴费支付流程

> 文档版本 2026-07-02 | 面向患者

## 概述

患者在医保问诊模式下开具处方后，进入医保预缴费页面进行费用确认和支付。该页面展示医保基金支付、个人账户支付、其他抵扣和现金支付的明细，支持患者选择是否使用医保个人账户支付，最终通过微信医保小程序完成支付。

## 触发条件

- 患者通过医保问诊流程开具处方
- 从收银台页面跳转进入
- 需要传递三个必要参数：订单编号（orderNo）、授权码（authCode）、处方编号（outpatientNo）

## 页面展示内容

### 付款对象
显示收款医院名称（从系统配置的医保设置中获取）。

### 费用明细
| 项目 | 说明 |
|------|------|
| 费用总额 | 总费用金额 |
| 医保基金支付 | 医保统筹基金报销部分 |
| 个人账户支付 | 医保个人账户支付部分 |
| 其他抵扣金额 | 其他优惠/抵扣 |
| 现金支付 | 患者最终需自付的金额 |

### 个人账户支付开关
- 如果系统支持个人账户支付，显示"使用/不使用"切换按钮
- 默认为"使用"（isIndividualAccount='1'）
- 切换需要重新向医保局获取授权码（跳转医保小程序授权）

### 处方明细
- 点击"查看明细"从底部弹出详情面板
- 显示每张处方的明细项目（名称、单价、数量、金额等）

### 底部统一标识
显示"医保移动支付"图标和文字。

## 支付流程

### 第一步：费用审核
进入页面自动调用医保预结算接口，展示费用分摊明细。

### 第二步：选择支付方式
- 默认使用个人账户
- 可切换为"不使用"个人账户 → 切换时重新跳转医保小程序获取授权

### 第三步：点击"去支付"
1. 显示"准备拉起支付"加载弹窗
2. 系统获取医保支付参数（微信支付AppId+跳转路径）
3. 跳转到微信医保小程序完成支付
4. 支付完成后自动返回当前页面

### 第四步：支付状态确认
返回后自动查询订单支付状态：
- **支付成功** → 自动跳转到药品订单页
- **未查询到支付状态** → 弹出选择框：
  - 点击"支付未成功" → 关闭弹窗，留在当前页可重新支付
  - 点击"支付已成功" → 重新查询状态

### 切换个人账户支付方式
1. 点击"使用/不使用"按钮
2. 跳转微信医保小程序获取新的授权码
3. 授权成功后携带新授权码返回
4. 用户取消授权 → 弹窗提示"您已经取消医保电子凭证身份授权，医保移动支付已中止"→ 跳回收银台页面

## 异常处理

| 情况 | 系统处理方式 |
|------|------------|
| 缺少必要参数（orderNo/authCode/outpatientNo） | Toast提示"缺少必要参数"，无法继续 |
| 医保预结算失败 | 弹窗显示错误信息，点击"知道了"返回上页 |
| 医保授权码获取失败 | 弹窗"未获取到授权码，请重新医保电子凭证身份授权！" |
| 取消医保授权 | 弹窗"您已经取消医保电子凭证身份授权"→ 跳回收银台页面 |
| 处方明细查询失败 | Toast提示错误信息 |
| 医保参数配置异常 | 弹窗"医保参数有误，是否重新获取？"→ 可重新加载系统配置 |
| 跳转支付小程序失败 | 标记isToPayed=false，onShow时不会触发状态查询 |

## 页面生命周期

- **onAppShow监听**：监听从小程序切回前台的事件
  - 如果携带了新的授权码（切换个人账户后）→ 重新初始化费用信息
  - 如果是从医保支付返回 → 检查订单支付状态
  - 如果用户取消了授权 → 弹窗引导回收银台

## 关联功能

- 跳转【微信医保小程序】：完成实际支付
- 跳转【药品订单页】：支付成功后查看订单
- 跳转【收银台页面】：医保授权取消后退回
