# 处方详情查看与操作流程（医生/患者双角色）

> 文档版本 1.0 | 面向非技术读者 | 2026-07-01

## 概述

处方详情页面是平台最核心的页面之一，支持**医生和患者两种角色**查看同一张处方的不同视图和操作。医生端可以编辑处方、签字、代支付、发到患者微信；患者端可以付款、领取处方、查看订单详情。页面还支持一键分享（医生代付/微信链接/二维码三种方式）。

## 触发条件

- 医生或患者从问诊聊天记录/处方记录/订单详情等入口点击进入
- 页面接收参数：`outpatientNo`（处方单号）和 `currentRoleCode`（角色标识）

## 步骤详解

### 一、页面顶部概览

打开页面后，顶部展示：

- **处方类型标签**："成药处方"（西药/中成药）或"中药饮片"
- **处方单号**：唯一编号
- **开具日期**：处方创建时间

如果同一病历下关联了多张处方（如快速开方场景），顶部会出现可横向滚动的**处方选项卡**，每张处方显示"处方1/处方2..."和支付状态（已支付/未支付）。

### 二、患者信息区

展示患者的基本信息（姓名/性别/年龄/科室）、诊断信息（辨病/辩证）。

### 三、药品信息区

**成药处方**：表格展示药品名称、规格、数量、用法用量。
**中药处方**：双列展示中药材名称+克重。

药品区有两个保护的配置策略：

- **未支付隐藏克重**（配置 `medicineUnitHide=2`）：患者未支付时，克重一列显示为"***"而非实际数值（防止患者拿到配方去别处抓药）
- **永久不显示克重**（配置 `medicineUnitHide=3`）：任何情况下都不显示克重

### 四、处方失效判断

患者端会判断处方是否已失效，满足以下任一条件即为失效（页面底部按钮变为"已失效"提示弹窗）：

1. **审核状态失效**：处方审核不通过（status=40）或已失效（status=50）
2. **超过有效期未支付**：处方过期时间 < 当前服务器时间，且订单状态为"待付款"
3. **处方撤回**：医生已撤回处方（createFlowStatus=31）

> 医生端不做失效判断，始终能看到完整处方内容。

### 五、医生端操作按钮

根据处方的不同状态，医生端底部按钮动态变化：

| 处方状态 | 底部按钮 | 说明 |
|---------|---------|------|
| 待医生审核（createFlowStatus=20/31）且问诊结束后24小时内 | **编辑** | 医生可修改处方内容 |
| 审核中（createFlowStatus=25）且有CA签字ID | **签字** | 医生进行电子签名 |
| 审核通过/待审核 + 未支付 + 有效期内 | **发到患者微信**（绿色） | 通过微信分享发给患者 |
| 任何审核通过的状态 | **引用处方**（始终显示，除非商城问诊/机构处方） | 复用处方内容创建新处方 |
| 有关联问诊单号 | **返回问诊室** | 回到聊天界面 |

**医生代支付**：如果处方待审核/审核通过 + 未支付 + 未领取 + 有效期内，医生可以**代患者支付**。

**指定接收人**：如果医生代付后处方还未被患者领取，医生可以点击"指定处方接收人"将处方发送给特定患者。

### 六、患者端操作按钮

| 场景 | 底部按钮 | 说明 |
|------|---------|------|
| 医生代付后，患者未领取 | **确认领取** | 绑定会员关系后领取处方 |
| 审核通过 + 未支付 + 有效期内 + 已绑定 | **付款** | 进入收银台支付 |
| 已支付 + 有订单号 | **订单详情**（白底按钮） | 查看关联订单 |
| 对外机构且无药房ID | 无按钮（隐藏付款） | `showedPay=false`隐藏付款按钮 |

### 七、电子签名（CA签章）流程

医生签字时经历以下步骤：

1. 点击"签字"按钮 → 系统获取签字唯一标识（caUniqueId）
2. 根据配置的签章平台（医网信/放心签）拉起对应签章组件
3. **等待回调（10秒内）**：页面底部显示"等待回调签字信息（10s）"加载动画
4. - **签字成功** → 跳转成功页，可选择发送给患者或查看处方记录
   - **签字失败** → 显示"签字失败，请重新签字"，可重新点击签字

### 八、查看电子处方

审核通过后的处方可在处方详情页查看电子处方PDF（带签章图片叠加的正式处方文件）。

### 九、分享功能

医生可通过"发到患者微信"按钮触发微信分享，分享卡片标题为"XX医生为「患者姓名」开具了处方，为不耽误用药请尽快支付"，分享图片为处方专用封面图。

### 十、快速续方

如果从"快速续方"入口进入，页面底部会出现"快速续方"按钮，医生点击后进入续方流程（自动带入当前处方的药品和诊断信息）。

### 十一、医生代付成功后处理

医生代付成功后，会弹出"代付成功"弹窗，医生可选择将处方发送给患者（触发IM消息通知）。

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 处方已失效（患者端） | 弹窗提示"处方单已失效，请联系医生重新为您开具处方" |
| 签字ID获取失败 | 弹窗"获取处方签字ID失败，请联系管理员"，确认后返回上一页 |
| 处方超过有效期未支付 | 按钮隐藏，显示失效提示 |
| 同一就诊人已被其他手机号绑定 | 弹窗显示绑定手机号，询问"是否进行解绑？"→ 可跳转解绑页面 |
| 非本人处方（100500） | 跳转到处方记录列表页 |
| 商城问诊/机构处方 | "引用处方"按钮不显示（不允许复用） |
| 共享链接参数异常 | 从订阅消息入口不会重复初始化页面 |

## 关联功能

- **快速开方**：引用处方功能可复用当前处方内容创建新处方
- **订单详情**：已支付处方可跳转关联订单
- **问诊室**：有关联问诊单号时可直接返回问诊聊天室
- **就诊人解绑**：绑定冲突时可跳转解绑页面
- **快速续方**：快速续方入口带出处方全部内容
