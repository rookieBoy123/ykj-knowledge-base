# 处方详情查看与操作流程（患者端+医生端双角色）

> 文档版本 2026-07-02 | 面向患者、医生

## 概述

处方详情页是患者和医生查看处方完整信息并执行后续操作的核心页面。同一处方在医生端和患者端展示的底部按钮不同，医生可以看到编辑、签字、代支付等操作，患者则看到付款、领取处方等操作。

## 触发条件

- **患者端入口**：从"我的处方"列表页点击"查看详情"进入
- **医生端入口**：从问诊室、处方记录等页面跳转进入
- 需要获取处方编号（outpatientNo）作为页面参数

## 页面展示内容

### 基本信息区
- 处方类型标签（成药处方/中药饮片）
- 处方编号
- 开具日期
- 患者信息（姓名、性别、年龄、科室）

### 诊断信息区
- 中医诊断（分为三种类型）：
  - **诊断**（诊断类型1）：西医/中医诊断
  - **辨病**（诊断类型2）：中医辨病
  - **辨证**（诊断类型3）：中医辨证

### 药品信息区
- 药品清单（名称+规格+数量+用法）
- 加工工艺（仅中药处方显示，如"水煎服"或药房加工服务）
- **中药饮片保密规则**：有三种配置模式
  - 不启用保密（始终可见）
  - 未支付不可查看（患者付款前隐藏药品明细）
  - 始终不可查看（患者始终看不到明细）

### 医嘱区
- 成药处方：显示医生填写的医嘱
- 中药处方：显示煎服方法和注意事项

### 医生/药师信息区
- 开具医生姓名
- 审方药师姓名（如有）

### 电子处方查看
- 条件：处方审核通过、订单有效、处方未失效
- 点击"查看电子处方"在微信内打开PDF格式的正式处方

## 底部按钮（患者端）

按钮根据处方状态动态变化：

| 条件 | 显示的按钮 | 说明 |
|------|----------|------|
| 未领取+医生已代付 | 【确认领取】 | 患者认领医生代付的处方 |
| 审核通过+未支付+有效期内 | 【付款】 | 进入药品下单支付流程 |
| 审核通过+未支付+未实名 | 【确认领取处方并支付】 | 同时完成领取+支付 |
| 已支付+有订单 | 【订单详情】 | 查看对应订单 |

**付款流程详解**：
1. 点击【付款】→ 先检查是否为商城问诊（跳转到商城订单页）
2. 检查处方是否已失效 → 失效则弹窗提示"处方单已失效"
3. 检查是否已创建订单 → 有订单跳转订单详情；无订单弹出确认弹窗
4. 确认弹窗展示"线上复诊才能为您开处方"提示及《互联网诊疗风险告知及知情同意书》
5. 点击"取消"→ 二次确认"您若取消，医生无法提供处方服务"
6. 点击"确定"→ 确认复诊标记 → 订阅消息 → 跳转药品下单页

## 底部按钮（医生端）

| 条件 | 显示的按钮 | 说明 |
|------|----------|------|
| 待审核(20/31)+问诊结束24h内 | 【编辑】 | 修改处方内容，跳转处方编辑页 |
| 审核中(25)+CA签字 | 【签字】 | 电子签章确认（医网信/放心签） |
| 审核通过/待审核+未支付+有效期内 | 【发到患者微信】 | 触发微信分享卡片给患者 |
| 处方状态>=20+未支付+有效期内+处审核不通过/未撤回 | 【代支付】 | 医生帮患者付款 |
| 未指定接收人+已支付 | 【指定处方接收人】 | 将处方指定给具体患者 |
| 有关联问诊单 | 【返回问诊室】 | 跳转到对应的IM聊天室 |
| 非商城问诊+非机构处方 | 【引用处方】 | 复用当前处方模板开新处方 |

### 签字流程
1. 点击【签字】→ 获取签章信息（CA平台+处方唯一标识）
2. 医网信平台：拉起签名组件
3. 放心签平台：拉起放心签弹窗
4. 签字成功 → 处方被领取→跳转开方成功页；已领取→跳转发药成功页
5. 签字失败 → 提示"签字失败，请重新签字"

### 复用处方
点击【引用处方】→ 跳转快速开方页面，自动带入当前处方的全部药品和诊断信息。

### 快速续方（从快速续方入口进入）
- 医生端底部显示【快速续方】按钮
- 点击前检查当前医生是否有该药房的授权 → 无授权提示"当前医生没有该药房授权"
- 跳转处方编辑页继续开方

## 处方TAB切换

如果同一病历下有多张关联处方（如快速开方产生多张处方），顶部会显示Tab栏：
- 每个Tab显示"处方1/2/3..."标签
- 已支付的Tab标注"已支付"，未支付的标注"未支付"
- 切换Tab重新加载对应处方的全部信息和按钮

## 时效性规则

### 处方失效判断
以下情况处方被判定为"已失效"：
- 创建流程状态为40（审核不通过）
- 创建流程状态为50（已失效）
- 处方状态为40（审核不通过）
- 处方状态为50（已失效）
- 处方有效期已过+订单仍为待付款状态

失效后患者端隐藏付款按钮，显示提示弹窗。

### 处方撤回判断
- 创建流程状态为20（草稿/待审核）或
- 创建流程状态为31（已撤回）
撤回后患者端不显示药品信息和支付按钮。

## 异常处理

| 情况 | 系统处理方式 |
|------|------------|
| 处方已失效 | 弹窗提示"处方单已失效，请联系医生重新为您开具处方" |
| 缺少处方编号 | 不加载页面，需重新进入 |
| CA签字ID获取失败 | 弹窗"获取处方签字ID失败，请联系管理员" |
| 就诊人被其他手机号绑定 | 弹窗提示"同一就诊人只能绑定在一个手机号"+显示已绑定手机号+提供【去解绑】按钮 |
| 处方未关联问诊单 | 点击"返回问诊室"提示"该处方未关联问诊单" |
| 医生无药房授权 | 提示"当前医生没有该药房授权，请联系管理员" |
| 患者未实名 | 提示"该处方患者未实名，不可指定" |

## 关联功能

- 跳到【药品下单页】：进行药品结算和支付
- 跳到【订单详情】：查看支付和物流状态
- 跳到【处方编辑页】：医生修改处方内容
- 跳到【快速开方页】：引用处方开新方
- 跳到【问诊室】：返回IM对话
- 跳到【就诊人解绑页】：解绑已占用的就诊人
