# 医生详情页选服务与下单流程

> 文档版本 1.0 | 面向读者：患者、产品

## 概述

患者在医生详情页查看医生信息、可选服务、患者评价和医生排班，选择问诊/预约服务后进入病情信息填写并下单支付。

## 触发条件

- 患者端从搜索、推荐、专家团队等入口进入医生详情页
- 页面参数包含医生信息（doctorObj）、推荐人ID（uniqueId）、是否线上复诊（isOnlineFollowUp）

## 步骤详解

### 第一步：查看医生基础信息

页面顶部展示医生基本信息（basicInfo组件）：头像、姓名、科室、职称、医院、简介等。

### 第二步：查看专家团队和AI数字分身

如果医生属于专家团队或有AI数字分身，通过extra-service组件展示相关入口。
- 专家团队：展示团队简介和成员头像
- AI数字分身：如果AI医生功能启用且有AI分身，展示入口

### 第三步：查看医生开通的服务

页面根据医生排班和系统配置显示两种模式之一：

**预约问诊模式**（registrationEnabled=true 且 consultationType=1，且非线上复诊）：
- 展示"预约服务"区域（appointService组件）
- 包含医生排班信息（scheduleList）
- 患者可选择时段预约

**即时问诊模式**（其他情况）：
- 展示"问诊服务"区域（inquiryService组件）
- 列出医生已开通的服务项（图文问诊、电话问诊、视频问诊等）
- 每项服务显示：服务名称、时长、价格
- 咨询服务（serviceType=3）自动排到第一位
- 点击"服务说明"可查看服务详情协议

### 第四步：选择服务并检查状态

点击某个服务后：
1. **登录检查**：未登录→引导登录
2. **号段检查**（预约模式下）：检查所选时段号段是否可用，不可用→提示"当前号段不可用"
3. **医生在线状态检查**：
   - 在线（onlineStatus=1）或有AI助手（isDoctorAssisted）→直接进入病情填写页
   - 离线→弹出"医生不在线提醒"弹窗，告知超时取消规则（如"医生在X小时后无接诊将自动取消并退款"），用户可选择"取消"或"继续问诊"

### 第五步：进入病情信息填写

确认后跳转到病情信息填写页（medicalRecord/addRecord），携带参数：
- 医生ID、服务类型、推荐人ID
- 病历ID、就诊人ID（如适用）
- 预约号段（如预约模式）
- 慢病编码（如走慢病医保）
- 是否线上复诊标识

### 第六步：查看患者评价

页面展示该医生的患者评价：
- 评价标签（taglist）：如"态度好(10)"、"专业(8)"等，统计数量
- 评价列表：滚动到底部自动加载更多（每页10条）
- 无评价时显示空状态"还没有评价呢"

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 未登录 | 弹窗引导登录 |
| 预约号段不可用 | 提示"当前号段不可用"，拒绝继续 |
| 获取医生状态失败 | 提示"获取医生状态失败，请稍后重试" |
| 医生离线 | 弹窗提示超时自动取消规则，用户可选择继续 |
| 医生无已开通服务 | 服务列表为空 |
| 评价标签查询无结果 | 标签列表不展示 |

## 关联功能

- 病情信息填写：跳转就诊信息填写页
- 服务预约：通过排班信息选择预约时段
- 服务说明：查看服务协议内容
- 患者评价：查看其他患者的评价和评价标签
