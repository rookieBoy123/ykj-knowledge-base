# 服务预约选与支付流程

> 文档版本：release/v2.17.2 | 面向读者：客服、运营

## 概述

患者可以通过小程序在线预约线下医疗服务，包括体检、核酸、健康证、检查等。整个流程为：浏览服务列表→查看服务详情→填写联系人信息→确认并支付→查看预约状态→到院核销。支持个人预约和团体/企业预约两种模式。

## 触发条件

用户从首页或服务入口进入「服务预约」功能模块。

**前置条件**：
- 用户已登录小程序
- 平台已在管理后台配置了可预约的服务项目

## 步骤详解

### 第一步：浏览服务列表

进入服务列表页（reservation/list），页面显示该类服务下的所有可预约项目。每个项目卡片展示：
- 封面图（左侧112px正方形）
- 服务名称（加粗）
- 个人价格（红色数字，如¥299.00）
- 右侧箭头表示可点击进入

列表支持滚动分页加载，每页10条。滚动到底部自动加载下一页。如果没有任何服务项目，显示空状态插图「暂无数据」。

服务类型由入口参数决定，包括四种：
1. 体检（type=0）
2. 核酸（type=1）
3. 健康证（type=2）
4. 检查（type=3）

同时传入`isTeamType`参数区分个人预约（false）和团体/企业预约（true）。这个参数影响后续页面的表单字段。

### 第二步：查看服务详情

点击某个服务卡片进入服务详情页（reservation/intro）。页面从上到下展示：
- **服务封面大图**（750rpx高，全屏宽）
- **价格区域**：大号红色数字（整数部分大号、小数部分小号叠加显示）
- **服务名称**（加粗大号字体）
- **标签行**：服务方式标签（如「上门服务」蓝色、「自行到院」橙色等），最多3个标签按颜色区分
- **医院信息**：医院名称、地址、电话拨号按钮（底部电话图标）、导航按钮（底部地图图标）
- **选项卡内容区**：两个选项卡切换查看
  - 「包含项目」：富文本展示服务包含的项目详情
  - 「服务须知」：富文本展示注意事项
- **底部固定按钮**：「立即预约」（蓝色/主题色），未登录时点击跳转登录页

导航功能需要获取用户地理位置授权。点击导航图标后：
1. 检查是否已授权地理位置
2. 未授权弹出提示「您未开启地理位置授权，是否前往授权？」
3. 已授权则获取当前位置
4. 跳转到地图页显示从当前位置到医院地址的路线

**页面滚动效果**：顶部导航栏初始透明，向下滚动超过标题栏高度后切换为白色背景+黑色文字。

### 第三步：填写预约信息（reservation/confirmation）

点击「立即预约」进入预约确认页（此页面在之前已经分析过，此处简述关键点）：

- 个人预约vs团体/企业预约的字段差异
- 选择就诊人（个人模式）
- 填写联系人信息（姓名、手机号）
- 选择预约时间
- 是否需要发票
- 费用明细展示

### 第四步：支付

在预约确认页点击「立即支付」后跳转到收银台，支付金额显示为预约总费用。支付完成后跳转回预约详情页。

### 第五步：查看预约详情

支付成功后进入预约详情页（reservation/detail）。页面从上到下展示：

**A. 预约状态区**（顶部）
预约有6种状态（`status`字段）：
- 0 → 待支付
- 1 → 已取消
- 2 → 退款中
- 3 → 已退款
- 4 → 待核销（已付款，等待到院使用）
- 5 → 已核销（已到院使用完毕）

待核销状态（status=4）下会显示条形码，用于到院出示核销。条形码内容为`verifyCode`。

**B. 就诊人信息**（仅个人预约显示）
显示就诊人姓名、性别（男/女/未知）、年龄。

**C. 联系人信息**
- 团体/企业模式：显示企业名称、联系人姓名、手机号
- 个人模式：显示联系人姓名、手机号

**D. 预约信息**
展示以下信息：
- 预约服务名称
- 预约时间（日期 + 时间段）
- 预约人数（仅团体/企业模式）
- 服务方式：自行到院 或 上门服务
- 上门服务额外显示：服务地区、具体地址
- 自行到院额外显示：医院地址 +「导航」按钮（跳转地图）
- 预约单号

**E. 预约费用明细**
逐项列出费用项（名称+金额），底部合计金额（红色加粗）。

**F. 发票区**
显示发票状态：
- 未开票时可申请开票（需在30天内）
- 已开票时显示发票信息（只读）

**G. 查看报告**（仅核酸类型+已核销状态）
核酸预约完成（status=5、projectType=2）后显示「查看报告」入口，点击跳转核酸报告小程序。

**H. 底部操作区**
根据预约状态动态显示：

| 状态 | 底部操作 |
|------|---------|
| 待支付(0) | 合计金额 + 「微信好友代支付」+「立即支付」 |
| 待核销(4) + 已付款 | 「取消预约」+「联系客服」 |
| 其他状态 | 仅「联系客服」按钮 |

### 第六步：取消预约

在待支付(status=0)或待核销(status=4)状态下，可以取消预约：
1. 点击「取消预约」弹出二次确认弹窗
2. 弹窗标题「确认取消预约吗？」
3. 待核销且已付款时，弹窗显示提示「确认后预付款项将在48小时内原路退回」
4. 点击「确认」执行取消
5. 调用取消接口，成功后刷新页面数据

**取消原因**：
- 待支付状态手动取消：`cancelDesc = '订单未支付，已手动取消'`
- 待核销0元取消：`cancelDesc = '预约已取消'`

取消前会触发订阅消息授权，用于后续通知。

### 第七步：代支付

在待支付状态，点击「微信好友代支付」：
1. 检查订单状态（`flowStatus === 10`），非待支付状态提示「当前订单非待支付状态」
2. 创建分享记录（`setShareRecord`），获得分享ID
3. 跳转到代支付详情页（reservationDfDetail），微信好友可通过链接代付

## 异常处理

| 情况 | 系统处理方式 |
|------|------------|
| 缺少服务ID | 提示「缺少服务id」 |
| 地理位置未授权 | 弹窗引导前往系统设置授权 |
| 定位失败 | 提示「请检查是否打开设备定位功能」 |
| 医院缺少联系电话 | 提示「缺少医院联系电话」，不执行拨号 |
| 定位参数不足 | 提示「定位参数不足，无法打开地图」 |
| 导航时缺少当前位置坐标 | 重新获取位置后再导航 |
| 待核销0元订单无退款提示 | 直接取消，不显示48小时退款文案 |
| 取消预约接口失败 | 刷新页面恢复原状态 |
| 打开核酸报告小程序失败 | 提示「打开小程序失败」 |

## 关联功能

- ← **服务列表页**（reservation/list）：选择服务项目进入
- → **预约确认页**（reservation/confirmation）：填写联系人和预约信息
- → **收银台**（hisPay）：进行实际支付
- → **地图导航页**（map）：查看从当前位置到医院的路线
- → **代支付详情页**（reservationDfDetail）：微信好友代付
- → **核酸报告小程序**：查看核酸检测结果（跳转外部小程序）

## 特殊说明

**个人vs团体/企业**：两种模式的主要差异是表单字段。团体模式多了企业名称、预约人数字段，且定价逻辑不同（个人支付全款，团体可能只付定金）。

**发票30天限制**：`allowedBilling`字段控制发票申请是否可用。超时后发票申请按钮不可操作。

**条形码核销**：预约付款后生成的核销码（verifyCode）以条形码形式在详情页展示，用于到院时出示给工作人员扫描核销。
