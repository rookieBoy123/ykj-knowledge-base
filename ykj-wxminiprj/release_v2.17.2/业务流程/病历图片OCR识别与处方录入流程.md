# 病历图片OCR识别与处方录入流程

> 文档版本：2026-07-05 | 面向读者：产品、运营、客服

## 概述

医生在开具中药处方时，可以通过拍摄患者纸质病历/处方照片进行 OCR（光学字符识别）识别，系统自动提取图片中的中药名称，快速录入到处方中。整个流程涉及两个页面：病历资料选择页和图片识别页。

## 触发条件

- **前置条件**：医生已进入中药处方编辑页面（添加中药处方）
- **入口路径**：中药处方编辑页 → 选择"拍照识别" → 进入病历资料选择页

## 步骤详解

### 第一步：选择要识别的病历图片

进入病历资料选择页后，系统自动展示以下图片来源：

1. **问诊病历图片**（medicalRecordImages）：患者在该问诊中上传的病历资料图片
2. **聊天图片**（TIMImageElem）：通过 IM 即时通讯模块拉取问诊聊天室中发送的所有图片消息

两种来源的图片以三列网格形式展示。系统会自动递归拉取聊天记录，直到所有图片消息都被加载完毕（isCompleted=true）。

- 点击任意一张图片将其选中，选中的图片右上角出现蓝色勾选图标
- 未选中的图片显示半透明遮罩，方便区分
- 再次点击已选中图片可取消选择

### 第二步：进入图片识别页

选中图片后，点击底部「确认选择」按钮，系统跳转到图片识别页。选中的图片全屏展示在黑色背景上。

**如果未选中任何图片**，点击「确认选择」按钮无反应（按钮为浅蓝色禁用状态）。

### 第三步：开始OCR识别

进入识别页后，系统自动调用 `api/medicine/drugs-search/_OCRHerb` 接口进行中药识别：

- 如果有图片URL（从聊天记录中获取），使用 `url` 参数
- 如果有图片ID（从病历资料中获取），使用 `objectId` 参数

识别过程中：
- 屏幕中央显示旋转加载图标和提示文字「识别中，请稍候...」
- 图片全屏显示，底部操作栏可见

### 第四步：查看识别结果

**识别成功**：
- 屏幕中央弹出半透明白色提示框，显示识别到的中药名称结果
- 底部出现「确认使用」蓝色按钮（可点击）和「重新上传」按钮（带返回图标）

**识别失败**：
- 屏幕中央弹出半透明白色提示框，左侧显示红色失败图标
- 显示失败原因（服务端返回的错误消息），如未返回则默认显示「未识别到文字，请重新拍摄」
- 底部只有「重新上传」按钮，「确认使用」按钮隐藏

### 第五步：确认使用 / 重新上传

**确认使用**：
点击「确认使用」后，系统将识别结果通过事件通道传回中药处方编辑页（addZy页面），自动填充到处方药品列表中。页面自动退回到处方编辑页（跳过多层中间页面，直接回到 addZy 页面）。

**重新上传**：
点击「重新上传」后，页面返回到病历资料选择页，可以重新选择图片或重新拍摄。

## 异常处理

| 情况 | 系统处理方式 |
|------|------------|
| 未选中图片就点确认 | 按钮无反应，需先点击选中一张图片 |
| 病历资料为空 | 显示"暂无病历资料"空状态 |
| OCR识别失败 | 显示错误提示（红色图标），可重新上传 |
| 照片无文字内容 | 提示"未识别到文字，请重新拍摄" |
| 聊天图片加载未完 | 自动递归拉取直到全部加载完成 |
| 无问诊单号 | 不触发数据加载，页面空白 |

## 关联功能

- 中药处方编辑页（addZy）：接收识别结果，自动填入处方
- 问诊聊天室：提供图片消息来源
- 问诊详情：提供病历资料图片来源

## 页面间通信方式

三个页面通过 eventChannel（事件通道）传递数据：

1. **病历资料选择页 → 识别页**：通过 `recognitionData` 事件传递图片URL
2. **识别页 → 病历资料选择页**：通过 `successRecognition` 事件向上转发识别结果
3. **病历资料选择页 → 处方编辑页**：通过 `successRecognition` 事件传递最终识别结果
