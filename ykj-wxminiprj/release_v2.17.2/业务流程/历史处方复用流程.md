# 历史处方复用流程

> 文档版本：release/v2.17.2 | 面向读者：医生

## 概述

医生在开具处方时，可以查看患者的历史处方记录，将历史处方中的药品清单和诊断信息复用到当前处方中，减少重复填写工作。支持成药和中药两种类型的历史处方各自独立查看。

## 触发条件

- **前置条件**：医生已进入开方页面（普通开方或快速开方），选择了开方患者
- **入口路径**：开方页面 → 「历史处方」按钮 → 跳转到历史处方列表页

## 步骤详解

### 第一步：进入历史处方列表

页面展示所选患者的所有历史处方。每条处方以卡片形式显示：

- **诊断信息**：诊断、辨病、辨证（三项各自独立显示）
- **药品清单**：处方中所有药品的文字摘要
- **处方状态**：右上角显示审核状态标签，包括：
  - 「药师待审核」— 处方已提交等待药师审核
  - 「药师审核中」— 药师正在审核
  - 「药师审核通过」— 处方已通过审核
  - 「已撤回」— 医生主动撤回了处方
  - 「药师审核不通过」— 处方被药师驳回
  - 「已失效」— 处方已超过有效期或被作废
- **基本信息**：开方日期、发药药房名称
- **中药加工服务**（仅中药处方）：显示加工方式，如无则显示「无」

### 第二步：查看处方详情（预览）

医生点击「预览」按钮，跳转到处方详情页查看完整内容。详情页展示：处方单号、患者信息、诊断详情、药品清单（含用法用量）、用药医嘱、医生签名等。

### 第三步：使用处方（复用开方）

医生点击「使用处方」按钮，系统会进行两步校验：

1. **药房授权校验**：检查处方关联的药房是否仍有授权关系。如果药房使用权已失效，弹窗提示「药房使用权已失效，请选择其他药房开方或联系客服处理」，该药房下所有处方的「使用处方」按钮变为灰色且不可点击。
2. **药房启用状态校验**：检查药房是否被禁用。如果药房被禁用，弹窗提示「该药房已被禁用，请选择其他药房开方或联系客服处理」，同理禁用该药房下所有处方的使用按钮。

校验通过后：
- 系统自动返回到开方页面
- 将历史处方的诊断、辨病、辨证、药品清单等信息通过事件通道传输到开方页面
- 开方页面自动填充这些信息，医生可以在填充后继续编辑

### 第四步：药房不可用时的处理

如果处方关联的药房已不可用（失效或禁用），处方卡片底部会有红色提示文字：
- 「药房使用权失效，不可使用处方」
- 「药房已被禁用，不可使用处方」

此时「使用处方」按钮变为灰色不可点击。医生只能查看或预览该处方，无法复用到当前开方中。

## 异常处理

| 情况 | 系统处理方式 |
|------|------------|
| 患者没有历史处方 | 显示空状态「暂无数据」 |
| 药房授权失效 | 弹窗提示，当前处方及其同药房所有处方标记为不可用 |
| 药房被禁用 | 弹窗提示，当前处方及其同药房所有处方标记为不可用 |
| 返回时找不到开方页面 | 重定向到首页 |
| 参数缺少（无patientId） | 直接显示空状态，不发起请求 |

## 关联功能

- **处方开具（addOrEditRp）**：复用历史处方的主要目标页面，接收事件通道传输的处方数据
- **快速开方（quickPrescribe）**：同样支持接收历史处方复用
- **处方详情（detail）**：预览按钮跳转的目标页面
- **药房管理（管理后台）**：药房绑定/禁用状态在后台控制

## 数据加载机制

- 支持下拉刷新：下拉列表重新从第1页开始加载
- 支持滚动到底加载更多：每页10条
- 可同时查看成药处方和中药处方（通过rpType参数区分）
- 每次进入页面（onShow）自动触发刷新
