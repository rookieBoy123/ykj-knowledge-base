## 详细变更

### 1. 新增支付宝扫码支付（packageConsultant/pages/aliPay/index.vue，267行）

新增独立的支付宝支付教程页面，展示完整的支付操作指引：
- 步骤化引导用户"保存二维码→打开支付宝扫一扫→选图支付"
- 支持长按保存和按钮保存两种方式
- 每3秒自动轮询支付状态
- 页面隐藏/返回时通知收银台接管轮询

### 2. 收银台支付改造（packagePatient1/pages/hisPay/index.vue）

- 新增支付宝支付渠道（channel=2），在支付方式选择时走支付宝流程
- 调用预支付接口获取支付宝二维码（codeUrl），跳转到支付宝支付教程页
- 新增支付失败弹窗（pay_fail_popup），区分 40003（toast提示）和其他错误（弹窗展示详情+联系客服入口）
- 新增查询支付结果失败弹窗（query_fail_popup），提供"重试"和"联系客服"选项
- 订单已支付（40001）统一弹窗"该订单已支付成功"
- 订单已过期（40002）统一弹窗"订单已取消关闭"
- 支付倒计时结束：先主动查询订单是否已支付（防跨端支付），再通知后端过期
- 新增轮询机制：页面隐藏时停止轮询，显示时恢复，切换非支付宝渠道时停止
- 移除旧的 payStatusPopup（被新的弹窗体系替代）

### 3. 医生端预约接诊超时提示优化（baseInfo.vue）

- 新增预约接诊+延时签到模式的特殊超时提示文案
- 当医生为预约接诊模式（consultationType=1）且支持延时签到（isDelayCheckIn=1）时
- 提示文案从"问诊订单超过X小时未接诊"改为"问诊订单当天未接诊，将在当天23点59分自动取消"
- 修复超时提示文案不换行的问题（移除 white-space:nowrap）

### 4. 公共方法改造（commonMethod.js）

- `drugCommonPayOrder` 方法重构：支付宝(channel=2)/医保(channel=3)由调用方处理响应数据，微信(channel=1)走原有 JS Bridge 流程
- 新增 `showOrderAlreadyPaidModal(40001)`：统一订单已支付弹窗
- 新增 `showOrderCancelledModal(40002)`：统一订单超时取消弹窗（处方/服务类不同文案）
- 新增 `expireOrder` API：订单过期失效接口
- 新增 `cancelPayment` API：取消支付/作废二维码接口
- 多个预支付接口增加 `notInterceptStatusCode: true`，由调用方自主处理错误码

### 5. API 层改动（http/moudules/base/index.js）

- 新增 `expireOrder`：POST `/api/orders/{orderNo}/expire`
- 新增 `cancelPayment`：POST `/api/orders/_cancel-payment`
- `advancePayment`、`getPayParams`、`verifyAlreadyPay`、`paymentStatus` 增加 `notInterceptStatusCode: true`
