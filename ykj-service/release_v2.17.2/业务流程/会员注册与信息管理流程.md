# 会员注册与信息管理流程（服务端）

> 文档版本：v1.0 | 面向读者：产品、开发

## 概述

平台会员（患者和医生的账号基础）的注册、信息更新、手机号绑定、微信OpenId关联以及积分管理的服务端流程。

## 涉及模块

- **ykj-service-member**：会员模块（MemberServiceImpl 238行）
- **关联模块**：ykj-service-server（对外API聚合）

## 流程详解

### 一、会员注册

**接口入口**：`MemberServiceImpl.register(MemberRegisterCommand)`

状态图：
```
用户输入手机号 → 查询是否已存在 → 不存在则走注册 → 返回会员信息
                   ↓ 存在
                 直接返回已有会员
```

**注册步骤**：
1. 通过手机号加密查询是否已有该会员（findByPhone）
2. 如果不存在，使用分布式锁（FairLock + 手机号key）防止并发重复创建
3. 在事务中执行：
   - 通过SeqService获取会员ID（`dyh_member.id`序列）
   - 设置手机号（明文存储后加密）
   - 绑定微信OpenId（如有）
   - 设置默认头像（OSS域名+配置的默认头像路径）
   - 设置默认昵称（"会员"+ memberId + 手机后4位）
   - 判断是否医生邀请（有doctorId→标记isSpecial=SPECIAL_IS，否则SPECIAL_ON）
   - status设为ENABLE（启用）

**防重复机制**：
- 外层：先查询手机号是否存在
- 内层：分布式锁（key=`SYNC_LOCK_MEMBER_PREFIX + mobile`，等待3秒）
- 锁获取失败返回429错误"请求频率过高，请稍后再试"

### 二、更新会员信息

**接口**：`PUT /api/member/current`

状态图：
```
传入MemberProfile → memberId自动获取 → 更新昵称/头像 → 保存
```

- memberId从登录态自动获取（MemberSubjectUtils.getMemberId()）
- 仅更新非空字段（nickname和avatar）
- 不存在则报错"会员信息不存在"

### 三、更换手机号

**接口**：`MemberServiceImpl.updateMemberPhone(memberId, phone)`

状态图：
```
传入新手机号 → 格式校验（11位手机号）→ 唯一性校验（加密后查是否存在）
  → 更新手机号 → 重置微信OpenId绑定 → 保存
```

**校验规则**：
- 手机号格式：`RegexUtils.isMobile()`（1开头的11位数字）
- 唯一性：新手机号加密后查询已有会员，如存在报错"手机号码已被使用"
- 换号后自动解绑微信（`setXcxOpenId("")`），用户需重新登录小程序

### 四、微信OpenId绑定和解绑

**绑定**：`bindOpenId(memberId, openId)`
1. 先解绑该OpenId的所有已有绑定
2. 将当前会员关联新OpenId

**解绑**：`unbindOpenId(openId)`
1. 查询所有使用该OpenId的会员
2. 逐一清空其xcxOpenId字段

### 五、积分管理

**增加积分**：`increaseGoldNumberMember(memberId, amount)`
- 当前积分 + amount

**减少积分**：`decreaseGoldNumberMember(memberId, amount)`
- 当前积分 - amount（amount取反后add）

两个操作都带`@Transactional`事务保护。

### 六、管理后台查询

**接口**：`mgmtFindPage(PageRequest, PageListSearchDTO)`
- 手机号搜索参数加密后再查询（crypto.encrypt）
- 查询结果中的手机号解密后返回明文（crypto.decrypt）
- 分页返回

## 异常处理

| 错误提示 | 触发条件 | 所属层级 |
|---------|---------|---------|
| "请填写正确的手机号码" | 手机号格式不符合11位 | Service层 |
| "手机号码已被使用" | 新手机号已绑定其他会员 | Service层 |
| "会员信息不存在" | 更新时memberId查不到会员 | Service层 | 
| "请求频率过高，请稍后再试" | 并发注册同一手机号，分布式锁获取失败 | Service层 |
| "获取登录会员失败" | 当前用户未登录或登录态异常 | Controller层 |

## 关联功能

- **患者管理**：患者在platform注册时会创建对应的患者记录（PatientServiceImpl）
- **医生管理**：医生注册也走MemberService，额外关联医生信息
- **微信登录**：通过wx.login获取openId后调用bindOpenId
- **消息推送**：通过memberId关联订阅消息和通知
