# 随访任务发送与状态管理流程

> 文档版本：v2.17.2 | 面向读者：产品经理 / 技术支持

## 概述

医生创建随访计划后，系统通过定时任务自动发送随访任务提醒给患者，并在发送时动态校验任务内容的可用状态。

## 触发条件

- 随访计划已创建（包含患教、问卷、提醒等内容项）
- 到了随访计划的设定时间（由定时任务触发）

## 步骤详解

### 第一步：定时任务触发

系统通过 `sendFollowupTask` 定时任务触发随访任务发送。任务接收随访任务ID作为参数。

### 第二步：校验任务是否存在

定时任务首先根据ID查询随访任务记录：
- **存在**：继续执行
- **不存在**：记录错误日志「ID为xxx的随访记录不存在！」，中止执行

### 第三步：更新任务内容可用状态

这是关键步骤。在发送通知之前，系统需要检查本次随访关联的患教资料和问卷是否仍然可用：

**患教资料状态检查**：
1. 收集所有类型为「患教」的任务内容项
2. 去重后批量查询对应的患教资料
3. 逐一比对：如果患教在后台的状态是「已发布」（Status.Published），则该任务内容的 forbidden 标记置为 true（不禁用）；否则标记为 false（禁用）

**问卷状态检查**：
1. 收集所有类型为「问卷」的任务内容项
2. 批量查询对应的问卷
3. 同理：已发布的问卷 forbidden=true，未发布或已禁用的问卷 forbidden=false

**提醒类内容**：不受此检查影响，始终发送。

### 第四步：判断是否发送

更新完状态后，检查该随访任务下是否有未被禁用的内容项：

- **有未禁用内容**：
  - 随访任务状态更新为「已随访」（FollowUp）
  - 组装推送消息内容「医生给您发了随访消息，请及时查看。」
  - 通过通知服务发送微信订阅消息给患者
  - 发送时校验：医生是否存在、患者会员信息是否存在（失败则记录日志并跳过）

- **全部内容被禁用**：
  - 随访任务状态更新为「已取消」（Canceled）
  - 不发送任何通知给患者

### 第五步：订阅消息推送

推送消息组装步骤：

1. 根据随访计划ID查询医生信息
2. 构建消息内容对象，包含消息文本和医生名称
3. 创建通知发送请求：
   - 通知编码：`PATIENT_FOLLOWUP_REMIND`（患者随访提醒）
   - 目标手机号：患者加密手机号
   - 唯一ID：随访任务ID（用于去重）
   - 微信OpenId：患者小程序的OpenId
   - 订阅消息体：医生姓名 + 提示文案

4. 调用统一通知服务发送

## 状态流转

```
随访计划创建 → 待随访(WaitFollowup) → 定时任务触发
                                      ↓
                          ┌─ 内容全部禁用 → 已取消(Canceled)
                          │
                          └─ 有可用内容 → 已随访(FollowUp) → 发送订阅消息
```

## 任务内容状态检测时机

重要的设计点：任务内容状态（是否禁用）不是创建随访计划时锁定的，而是在**发送随访任务提醒的那一刻**动态检查的。

这意味着：
- 如果管理员在随访计划创建后禁用了某个患教或问卷，该内容在发送时会被自动过滤
- 如果所有内容都被禁用了，整个随访任务变为「已取消」，患者不会收到通知
- 医生在创建随访模板时选择的内容，到实际发送时可能已经被后台管理员下线

## 异常处理

| 情况 | 系统处理方式 |
|------|------------|
| 随访任务记录不存在 | 记录错误日志，静默中止 |
| 查询医生信息失败 | 记录警告日志「医生不存在」，中止发送 |
| 查询患者会员信息失败 | 记录信息日志「取消推送，获取会员信息失败」，中止发送 |
| 任务下所有内容被禁用 | 任务状态变为已取消，不发送通知 |

## 关联功能

- 与「随访计划管理」关联：计划ID关联任务
- 与「患教管理」关联：患教资料的禁用/发布状态影响任务发送
- 与「问卷管理」关联：问卷的禁用/发布状态影响任务发送
- 与「统一通知服务」关联：通过NotificationAppService发送微信订阅消息
- 与「会员服务」关联：获取患者手机号和OpenId
