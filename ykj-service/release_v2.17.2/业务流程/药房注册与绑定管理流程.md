# 药房注册与绑定管理流程

> 文档版本：release/v2.17.2 | 面向读者：产品经理、运营、开发

---

## 概述

平台支持两种类型的药房：线上药房（云端配送）和线下药房（实体药店）。每个药房需要绑定一个手机号作为管理员账号，手机号与药房之间是一对一关系。药房管理涵盖了新增、编辑、启用/禁用、排序、第三方药房对接、加工服务配置、库存数量统计等功能。

## 触发条件

- **前置条件**：管理后台用户具有药房管理权限（`med:pharmacies:write`）
- **入口**：管理后台 → 药房管理 → 药房渠道管理

---

## 步骤详解

### 第一步：新增药房

管理员新增药房时，系统执行以下校验和操作：

1. **手机号非空校验**：手机号不能为空，否则提示"手机号不能为空"
2. **药房名称去重**：检查药房名称是否与已有药房重复，重复则提示"药房名称重复"
3. **手机号唯一绑定**：检查该手机号是否已被其他药房绑定，已绑定则提示"该手机号已绑定药房"
4. **第三方药房对接**：如果开启第三方药房，则先创建第三方药房记录，关联 `thirdPharmacyId`
5. **排序分配**：新药房的排序号 = 当前最大排序号 + 1
6. **加工服务**：如果药房支持中药切片（药品类型包含2），则同步保存加工服务配置
7. **线下药房自动注册**：线下药房类型时，系统自动：
   - 查找手机号对应的系统用户，不存在则自动注册
   - 授予该用户「线下药房管理员」角色
   - 建立用户-药房的绑定关系

### 第二步：编辑药房

1. 先查药房信息是否存在（根据ID），不存在则提示"药房信息不存在"
2. 重新执行名称去重校验（排除自身）
3. 重新执行手机号唯一绑定校验（排除自身）
4. **第三方药房保护**：如果药房之前已启用第三方且编辑时改为禁用，提示"已启用第三方药房的不允许编辑为禁用"
5. 如果启用第三方药房：
   - 已有第三方记录→编辑更新
   - 无第三方记录→新建
6. **线下药房手机号变更**：手机号变更时：
   - 先解除旧手机号用户与药房的绑定（撤销角色+解除关系）
   - 再注册/绑定新手机号用户
7. 如果药品类型含中药切片，更新加工服务配置

### 第三步：药房与用户绑定

**绑定逻辑**（`bind`）：
1. 检查用户是否已有绑定的药房
2. 如果已绑定了其他药房（非目标药房），不允许再绑定，提示"当前用户已绑定别的药房"
3. 一个账号只能绑定一个药房（不支持多药房切换）
4. 线下药房自动绑定关系，避免重复绑定

**解绑逻辑**（`unbind`）：
1. 先检查用户是否还有其他线下药房角色
2. 如果还有其他角色，则不删除绑定关系（保留）
3. 如果没有其他线下药房角色，才真正删除用户-药房关联

### 第四步：启用/禁用药房

1. **启用**：药房状态改为启用（`status=1`），医生可选用该药房开方
2. **禁用**：药房状态改为禁用（`status=0`），被禁用的药房不再出现在医生可选药房列表中
3. 药房不存在时提示"传参错误，没找到对应药房"

### 第五步：药房排序

支持四种操作：
- **排至最前**：将药房排到列表第一位，已是最前时提示"已经排在最前啦"
- **排至最后**：将药房排到列表最末位，已是最后时提示"已经排在最后啦"
- **上移**：与上一个药房交换排序位置
- **下移**：与下一个药房交换排序位置

### 第六步：查询药房

**查询模式**（受系统配置影响）：
1. **默认药房模式**（`defaultEnabled=true`）：所有查询强制限定为指定默认药房ID，忽略其他条件
2. **线下药房开关**（`offlineEnabled=false`）：查询结果排除线下药房，仅返回线上药房
3. **绑定药房限定**：可按用户授权的药房列表过滤

**药品类型查询**（`findEnableDrugsType`）：
1. 查询用户可访问的药房所支持的药品类型
2. 去重后返回，主要返回中药类型（排除药品类型1）
3. 未授权时使用药房ID=0作为占位（查不出任何结果）

### 第七步：药房药品数量统计

独立的药品数量更新方法：
1. 分别更新中药数量和成药数量
2. 两项都为null时记录警告日志并跳过
3. 任一非null时执行更新，写对应字段

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 手机号为空 | 提示"手机号不能为空" |
| 药房名称重复 | 提示"药房名称重复" |
| 手机号已绑定其他药房 | 提示"该手机号已绑定药房" |
| 新增药房插入失败 | 提示"药房新增失败" |
| 编辑时药房不存在 | 提示"药房信息不存在，id=xxx" |
| 编辑时手机号为空 | 提示"手机号码为空"（通过bindingCheck校验抛出） |
| 已启用第三方药房改为禁用 | 提示"已启用第三方药房的不允许编辑为禁用" |
| 修改药房时更新失败 | 提示"药房修改失败" |
| 启用/禁用时药房不存在 | 提示"传参错误，没找到对应药房" |
| 排序已到边界 | 分别提示"已经排在最前啦"/"已经排在最后啦" |
| 用户已绑定其他药房 | 提示"当前用户已绑定别的药房" |
| ID列表为空查询 | 提示"ids不能为空" |
| 药品数量更新都为null | 记录警告日志，跳过更新 |

## 关联功能

- **药品管理**：药房渠道下的药品录入/编辑/删除/导入导出
- **医生药房授权**：配置医生可以开方到哪些药房
- **加工服务管理**：中药切片的加工工艺配置
- **第三方药房对接**：阿康(AKANG)等外部药房系统对接
- **处方开具**：医生开方时选择药房
- **药房管理员登录**：线下药房管理员登录管理后台（药房端）

## 设计要点

1. **手机号一对一绑定**：一个手机号只能绑定一个药房，防止管理混乱
2. **系统配置强控**：`defaultEnabled`和`offlineEnabled`可以全局改变药房查询行为，实现集中管控
3. **线下药房自动账号创建**：添加线下药房时自动创建管理员账号，无需手动注册
4. **第三方药房不可逆**：一旦启用第三方药房就不允许编辑时关闭，防止数据不一致
5. **排序边界保护**：四种排序操作都有边界检查，防止越界操作
