# 问题库存取与校验逻辑（服务端）

> 文档版本：2026-07-05 | 面向读者：产品经理、系统设计人员

## 概述

题库服务的后端核心逻辑，管理可复用的问卷题目。每个题目包含题目编码（自动生成）、题目名称、题目类型（单选/多选/问答/上传图片）、是否必填、选项列表（仅单选/多选题）、启用状态等属性。题目与选项分表存储（QuestionLibrary + QuestionChoices），支持题目和选项的级联管理。

## 触发条件

- 前置条件：管理员拥有`hosp:ql-questions:write`权限
- 系统入口：管理后台调用 `/mgmt/hospital/question-library/questions` 接口

## 核心流程

### 题目创建流程

```
提交题目信息 → 数据校验 → 重名检查 → 自动生成编码+序号 → 插入题目表 → 插入选项表
```

**编码生成规则**：
- 格式：`TM` + 当前时间的纯数字格式（年月日时分秒毫秒），如 `TM20260705143022123`
- 通过 `DateUtil.format(new Date(), DatePattern.PURE_DATETIME_MS_PATTERN)` 生成
- 保证每次生成唯一编码

**序号分配规则**：
- 查询数据库中最大的序号（`seq` 字段）
- 新题目序号 = 最大序号 + 1
- 初始默认值为1

**重名检查**：
- 按 `subject`（题目名称）精确匹配
- 已存在同名题目时抛异常："题目名称XXX已存在"

### 题目编辑流程

```
加载已有题目 → 修改字段 → 数据校验 → [题目名称变更时重名检查] → 更新题目表 → 清除旧选项 → 重新插入新选项
```

**关键设计**：
1. 编辑时题目类型（type）不可修改，强制保持原值
2. 题目名称变更时才做重名检查（名称不变时跳过）
3. 选项更新采用"先删后插"策略：先清除该题目的所有旧选项记录，再逐条插入新选项

### 题目删除流程

```
单个删除: 查询题目是否存在 → 删除关联选项 → 删除题目
批量删除: 遍历ID列表 → 逐个执行单个删除逻辑（事务保护）
```

**级联删除**：删除题目时必须同时删除该题目下的所有选项（`QuestionChoices`），先删选项再删题目，保证数据一致性。

### 题目启用/禁用

通过 `changeUsableState` 接口，只更新 `enabled` 字段，不影响题目和选项的其他数据。

### 选项管理

选项（QuestionChoices）与题目是多对一关系：
- 选项字段：选项内容（content）、选项值（value）、所属题目ID（questionId）
- 创建/编辑题目时，选项随题目一起保存
- 查询题目详情时，通过 `assemblerChoices` 方法批量组装选项数据（先收集所有题目ID，一次性查询所有关联选项，再按题目ID分组映射）

## 校验规则

| 校验项 | 规则 | 异常信息 |
|--------|------|---------|
| 题目名称非空 | subject 不能为null或空 | "题目名称不能为空" |
| 题目名称长度 | 不超过100字 | "题目名称长度不能大于100字" |
| 备注长度 | 不超过300字 | "题目名称长度不能大于100字"（注：异常文案有误，实际校验的是remark） |
| 题目类型非空 | type不能为null | "题目类型不能为空" |
| 单选/多选选项非空 | choices列表不能为空 | "单选题答案选项不能为空" |
| 单选/多选选项数量 | 至少2个 | "单选题选项不能低于2个" |
| 问答/上传图片 | 允许choices为空 | 这两种类型不需要选项 |

`ALLOW_CHOICES_NULL` 常量定义了不需要选项的题目类型：`EQ`（问答题）和 `RBQ`（上传图片）。

## 数据模型

| 字段 | 含义 | 特殊规则 |
|------|------|---------|
| no | 题目编码 | 自动生成，格式TM+时间戳 |
| subject | 题目名称 | 最长100字，不可重名 |
| type | 题目类型 | 1=单选/2=多选/3=问答/4=上传图片，编辑不可修改 |
| required | 是否必填 | 布尔值 |
| enabled | 启用状态 | 默认true |
| seq | 排序序号 | 自增，用于控制题目展示顺序 |
| remark | 备注 | 最长300字 |
| choices | 选项列表 | 子表存储，仅单选/多选有 |

## 关联功能

- **问卷创建**：问卷从题库中选择题目构建
- **API问题库接口**：`ApiQuestionLibraryController` 为小程序端提供题目查询
- **随访问卷**：随访功能也使用题库
