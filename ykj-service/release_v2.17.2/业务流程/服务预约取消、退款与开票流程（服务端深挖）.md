# 服务预约取消、退款与开票流程（服务端深挖）

> 文档版本 2026-07-06 | 面向：产品经理、运营人员

## 概述

患者在服务预约下单后，可以进行取消操作；系统根据预约状态自动处理退款或退款申请，同时支持核销后的自动开票。整个流程通过事件驱动实现了预约单、订单、退款单的状态联动。

## 触发条件

- **取消预约**：患者在小程序主动取消预约
- **过期取消**：订单超时未支付，系统自动取消
- **退款完成**：退款审核通过后触发退款事件，更新预约单状态
- **自动开票**：核销成功后（且开启了自动开票功能），系统自动申请电子发票

## 步骤详解

### 一、患者取消预约（cancelOrderByMember）

患者取消预约时分三种情况处理（第439-487行）：

1. **身份校验**：只有预约人本人才能取消（通过memberId匹配）
2. **状态校验**：仅"待支付"和"待核销"状态的预约可取消，其他状态提示"该预约单不可取消，仅待支付和待核销状态预约订单可取消！"（第451行）

三种取消子场景：
- **待支付状态**：直接改为"已取消"，取消关联订单
- **待核销 + 金额为0**：直接改为"已取消"，取消关联订单，推送取消通知
- **待核销 + 金额大于0**：改为"退款中"，发起退款申请——系统自动创建退款单等待审核

### 二、支付后状态变更（handleReservationAfterPayed）

监听订单支付完成事件（第494-521行）：

- 仅处理来自服务预约的支付事件
- 支付成功后生成8位随机核销码，状态从"待支付"改为"待核销"
- 推送预约成功通知，提醒患者携带身份证到院

### 三、过期自动取消（handleCanceledOrderEvent）

监听订单取消事件（第529-544行）：

- 仅处理服务预约类型的订单
- 自动将预约单状态改为"已取消"
- 取消关联的订单

### 四、核销验证（verify）

工作人员核销预约时（第170-211行）：

1. **查找验证**：根据核销码查找预约单。找不到提示"未查询到预约记录，核销码无效！"（第218行）
2. **状态验证**：只有"待核销"状态才能核销。非待核销状态提示"此核销码已核销"（第222行）
3. **直接核销**：传入预约单ID和核销码，码不一致提示"错误的核销码"（第187行）；状态不是待核销提示"该订单不是待核销状态订单"（第183行）
4. **自动开票**（第193-201行）：如果系统开启自动开票功能且订单金额大于0，核销成功后自动发布开票事件，异步生成电子发票

### 五、退款完成（handleOrderRefund）

监听订单退款完成事件（第547-568行）：

- 仅处理服务预约类型的退款
- 清空核销码，将预约单状态改为"已退款"

### 六、退款申请（handleOrderRefundApply）

监听退款申请事件（第571-581行）：

- 将预约单状态改为"退款中"

## 状态流转

```
待支付 → [支付成功] → 待核销 → [核销] → 已完成
   ↓                   ↓
   └→ 已取消        [取消] → 退款中 → 已退款
```

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 预约单不存在 | 后端拦截，提示"未查询到预约记录，核销码无效！" |
| 核销码错误 | 后端拦截，提示"错误的核销码" |
| 非待核销状态核销 | 后端拦截，提示"该订单不是待核销状态订单" |
| 已核销再次核销 | 后端拦截，提示"此核销码已核销" |
| 非本患者查看预约 | 后端拦截，提示"当前患者无权限查看该预约单信息" |
| 非可取消状态取消 | 后端拦截，提示"该预约单不可取消，仅待支付和待核销状态预约订单可取消！" |
| 项目无可用预约时间 | 后端拦截，提示"当前项目暂无可预约时间"或"当前日期该项目暂无法预约" |
| 自动开票-预约单异常 | 后端拦截，提示"预约单信息异常" |
| 自动开票-非个人预约 | 后端拦截，提示"只允许个人预约单自动开票" |
| 自动开票-HIS信息缺失 | 后端拦截，提示"患者his信息异常" |
