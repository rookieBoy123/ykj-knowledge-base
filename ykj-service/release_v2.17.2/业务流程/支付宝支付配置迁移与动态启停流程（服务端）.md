# 支付宝支付配置迁移与动态启停流程（服务端）

> 文档版本 2026-07-05 | 面向产品/运营/开发

## 概述

支付宝支付开关从「全局功能开关」迁移到「支付宝支付配置」内部，作为一个独立的 `enabled` 字段存在。本次调整后，支付宝支付支持动态刷新（无需重启服务即可生效），且配置结构更加内聚——所有支付宝相关配置集中在一处管理。

## 变更背景

在此次调整前，「是否启用支付宝支付」是一个全局功能开关（`func.alipayEnabled`），与支付宝的具体配置（应用ID、密钥、回调地址等）分属两个不同的配置分组。管理员在系统配置页面需要先到「功能开关」中开启支付宝，再到「支付配置」中填写支付宝参数，操作分两步且关闭开关后支付宝配置区域会整体隐藏。

## 变更内容

### 配置结构变化

- **删除**：`func.alipayEnabled` 全局开关（从全局配置的「功能开关」分组中移除）
- **新增**：`payment.alipay.enabled` 字段，位于「支付配置→支付宝支付」分组下
- **依赖关系**：支付宝的应用ID、私钥、公钥、网关地址、回调地址等字段现在都依赖 `payment.alipay.enabled` 为 `true` 时才显示

### 动态刷新支持

- 支付宝支付服务（`AliPaymentServiceImpl`）新增 `@RefreshScope` 注解
- 开启或关闭支付宝支付后，无需重启服务即可生效

## 配置方式

1. 进入后台管理 → 系统管理 → 全局配置
2. 找到「支付配置」→「支付宝支付」
3. 开启「启用」开关
4. 填写应用ID、应用私钥、支付宝公钥、网关地址、回调地址、退款回调地址
5. 点击「保存」

关闭支付宝支付时：
1. 只需关闭「启用」开关
2. 其他支付宝配置字段会自动隐藏（不需要清空）
3. 保存后支付宝支付方式从前端收银台消失

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 开启支付宝但未填写应用ID | 支付时会因配置缺失而报错（后端拦截，不会展示给用户） |
| 关闭支付宝后用户仍持有未支付订单 | 已生成的支付二维码仍有效（基于支付时的配置快照），但新订单无法选择支付宝 |
| 前端传了支付宝支付方式但后端支付宝未启用 | 后端返回「该订单不支持当前支付方式」 |

## 关联功能

- 系统全局配置管理（管理后台 → 系统管理 → 全局配置）
- 支付宝当面付扫码支付流程
- 收银台支付方式展示（小程序端）
