# 问诊数据归档管理流程

> 文档版本 2026-07-03 | 面向产品/运营/客服

## 概述

问诊结束后，系统自动将问诊产生的全部数据（病历、处方、聊天记录、视频录像）归档到对象存储中。同时支持管理员手动触发归档。归档是问诊数据长期保存和合规管理的关键环节。

## 触发条件

### 自动归档触发
- **问诊完成事件**：问诊状态变更为"已完成"时自动触发
- **药师审方完成事件**：处方审核完成（通过/驳回/过期）时自动触发
- 以上两个事件只要有一个满足条件，系统都会尝试归档

### 手动归档触发
- 管理后台：归档记录列表中，对"归档失败"的记录点击"归档"按钮

## 步骤详解

### 第一步：归档条件校验

系统检查问诊单是否满足归档条件：
- 问诊单必须存在且状态为"已完成"
- 如果问诊单有关联处方，所有处方必须审核完毕（通过/驳回/过期）
- **不满足条件**：系统记录日志后直接跳过，不进行归档

### 第二步：创建归档记录

校验通过后，系统创建一条归档记录：
- 记录问诊单号、患者姓名
- 初始状态标记为"待处理"
- 预分配归档存储目录

### 第三步：自动归档执行

系统定时任务扫描状态为"待处理"的归档记录，逐条执行归档：

#### 3.1 归档病历记录
- 查询问诊对应的病历信息
- 使用HTML模板生成标准化的病历文档
- 将病历文档转为图片（PNG格式）
- 上传至归档目录

#### 3.2 归档电子处方
- 查询问诊关联的所有处方
- 将处方信息转为标准格式文档
- 生成处方图片并上传

#### 3.3 归档图文聊天记录
- 查询问诊过程中的所有聊天消息
- 将消息记录序列化为JSON文件
- 上传至归档目录

#### 3.4 归档视频资料
- 查询问诊中的视频通话记录
- 如有视频录像文件，上传至归档目录

### 第四步：归档结果更新

- **全部成功**：归档记录状态更新为"已归档"
- **任一失败**：归档记录状态更新为"归档失败"，记录错误日志

### 第五步：手动重试（失败时）

对于归档失败的记录，管理员可在后台手动触发重新归档：
- 点击"归档"按钮
- 系统重新执行完整的归档流程
- 只有"归档失败"状态的记录才能手动触发

## 异常处理

| 情况 | 系统处理方式 |
|------|------------|
| 归档记录不存在 | 提示"归档记录不存在"，不执行操作 |
| 归档记录正在处理中或已归档 | 提示"归档记录正在归档中或者已归档, 不能手动归档" |
| 问诊单已结束但处于非完成状态 | 自动归档校验不通过，记录日志后跳过 |
| 有未审核完的处方 | 不触发归档，等待审方完成后再归档 |
| 某类资料上传失败 | 记录异常日志，整条归档标记为"失败"，所有已上传文件保留 |
| 归档执行过程中异常 | 捕获异常后标记失败，不留部分成功状态 |

## 归档内容清单

| 归档内容 | 格式 | 说明 |
|---------|------|------|
| 病历记录 | PNG图片 | 标准化病历模板生成 |
| 电子处方 | PNG图片 | 处方格式化输出 |
| 图文聊天记录 | JSON文件 | 完整问诊对话记录 |
| 视频通话录像 | 视频文件 | 如有录制 |

## 关联功能

- **问诊管理**：问诊完成事件触发归档（ykj-service-inquiry）
- **处方审核**：审方完成事件触发归档（ykj-service-inquiry）
- **文件存储**：归档文件上传至OSS（OSS服务）
- **视频服务**：获取问诊视频录像（ykj-service-rtc）
- **管理后台归档管理**：查看归档记录和手动触发归档（管理后台）
