# 订阅消息模板管理流程（服务端）

> 文档版本 2026-07-01 | 面向读者：系统架构师、后端开发

## 概述

小程序订阅消息是微信提供的能力，允许小程序在用户授权后向其发送服务通知（如问诊状态变更、处方审核通过、药品发货提醒等）。服务端管理订阅消息模板的查询与同步。

## 涉及服务模块

- `ykj-service-subscribe` — 订阅消息管理

## 核心实体

- **SubscribeTemplateDto**：订阅消息模板信息（模板ID、模板标题、模板内容、适用场景等）
- **WxMiniProperties**：小程序配置信息（appId等）

## 业务链路

### 一、模板查询

小程序前端调用接口获取当前可用的订阅消息模板列表：

```
GET /api/subscribe/templates
```

参数：无需传参（系统自动使用当前小程序的appId）

返回：当前小程序已配置的所有订阅消息模板集合

### 二、使用场景

订阅消息模板在以下场景中使用：

1. **患者发起问诊前**：请求用户授权"问诊状态通知"模板，以便医生接诊/回复时提醒
2. **提交会诊申请前**：请求用户授权"会诊进度通知"模板
3. **下单药品前**：请求用户授权"订单状态变更"和"物流发货"模板
4. **加入随访计划**：请求用户授权"随访提醒"模板

### 三、模板同步

- 管理员在微信公众平台配置新的消息模板后，服务端通过此接口将模板信息同步到小程序端
- 模板信息包括：模板ID（微信侧唯一标识）、模板标题、模板内容中的变量占位符
- 小程序端根据模板ID调用微信API请求用户授权

## 关键逻辑

1. **多小程序支持**：通过 WxMiniProperties 的 appId 区分不同小程序的模板
2. **模板获取**：从微信服务端获取已配置模板列表（不区分订阅状态）
3. **前端调用时机**：通常在关键操作前调用（如提交问诊前），先展示授权弹窗再执行业务操作

## 异常处理

| 情况 | 系统处理 |
|------|---------|
| 模板未在微信平台配置 | 返回空集合或缺少该模板 |
| 用户拒绝订阅授权 | 前端跳过订阅流程，不影响核心业务执行 |
| appId 配置错误 | 查询失败，返回错误信息 |
