# 病历管理流程（服务端）

> 文档版本：2026-07-02 | 面向读者：产品运营、技术支持

## 概述

病历是医生在问诊过程中为患者创建的核心医疗文书，记录了患者的病史、诊断、治疗意见等信息。服务端负责病历的创建、更新、签署（电子签章）、撤销等全生命周期管理。

## 触发条件

- **前置条件**：医生已进入问诊会话
- **服务模块**：ykj-service-inquiry（MedicalRecordServiceImpl）

## 步骤详解

### 第一步：创建病历

医生在问诊中首次填写病历时触发创建：

1. **组装医生信息**：根据医生ID查询医生的姓名、医院、科室、职称等
2. **组装患者信息**：根据患者ID查询患者的姓名、性别、年龄、监护人信息
3. **判断病历类型**：根据医生的医疗类型（中医/西医）自动确定病历类型
4. **初始签名状态**：医生签名状态设为「未签」
5. **保存病历**：写入病历主表
6. **保存诊断**：将诊断列表（辨病、辨证、西医诊断）写入诊断子表
7. **创建签署记录**：
   - 判断是否需要电子签章
   - 如果需要签署，创建CA签署任务并返回签署信息
   - 如果不需要签署，自动完成签名流程（运营模式）

### 第二步：更新病历

医生修改已有病历时触发更新：

1. **查询已有病历**：根据病历ID获取已有病历
2. **重置签名状态**：修改后签名状态重置为「未签」，需重新签署
3. **更新主表**：更新主诉、治疗意见、病史等字段
4. **处理诊断变更**：
   - 对比已有诊断和新提交的诊断
   - 找到需要删除的诊断（已在数据库中但不在新提交中）
   - 找到需要新增的诊断（在新提交中但不在数据库中）
   - **删除前检查**：不允许删除已被处方关联的诊断（提示：「不允许删除已经被关联的诊断。诊断名称:【XXX】」）
5. **保存诊断变更**：删除旧诊断、新增新诊断
6. **创建签署记录**：同创建流程

### 第三步：诊断绑定保护

诊断被处方关联后，在病历中变成「不可修改」状态（锁定），防止医生误删导致处方数据不一致。

### 第四步：病历签署（电子签章）

病历创建或更新后需要进行医生签名：

1. **CA电子签章（严格模式）**：
   - 生成唯一签署标识
   - 调用CA服务创建签署任务
   - 医生在小程序端完成手写签名或CA证书签名
   - 签名完成后更新病历签名状态
   
2. **自动签名（运营模式）**：
   - 不需要CA电子签章时
   - 后台自动将医生姓名作为签名
   - 直接标记为已签

3. **病历签章状态**：
   - 未签：初始状态
   - 已签：签名完成

### 第五步：查询病历

支持多维度查询：

| 查询维度 | 说明 |
|---------|------|
| 按患者查询 | 根据患者ID或会员ID查询 |
| 按签名状态查询 | 未签/已签/全部 |
| 筛选有问诊的病历 | 只查询已关联问诊的病历 |
| 分页查询 | 支持分页，按创建时间倒序 |

查询结果包含：
- 病历基本信息（主诉、治疗意见等）
- 诊断列表（辨病、辨证、西医诊断及是否锁定）
- 病史信息（过敏史、现病史等）
- 签名状态
- 患者信息（姓名、性别、年龄、监护人）

### 第六步：病历撤销

管理员或特定条件下可以撤销（逻辑删除）病历：
- 记录撤销原因
- 执行逻辑删除（保留数据但标记为已删除）
- 撤销前会经过拦截器链检查（如检查是否有关联的处方不允许删除）

### 第七步：会员绑定

病历支持绑定到会员：
- 已有绑定：不允许重新绑定（提示：「该病历已绑定会员，不允许重新绑定」）
- 可变更患者：支持更换病历关联的患者

## 诊断类型说明

| 诊断类型 | 编码 | 使用场景 |
|---------|------|---------|
| 西医诊断 | 1 | 西医科室、中医科配置了西医诊断必填时 |
| 辨病 | 2 | 中医科室 |
| 辨证 | 3 | 中医科室 |

## 异常处理

| 情况 | 系统处理方式 |
|------|------------|
| 医生不存在 | 提示「医生不存在」 |
| 删除已被关联的诊断 | 提示「不允许删除已经被关联的诊断。诊断名称:【XXX】」 |
| 病历已绑定会员 | 提示「该病历已绑定会员，不允许重新绑定」 |
| 会员不存在 | 提示「变更病历患者失败，会员不存在」 |
| 病历不存在 | 提示404错误 |

## 关联功能

- 处方开具（诊断来源于病历）
- CA电子签章（签名独立模块）
- 会员服务
- 患者管理
- 问诊服务
