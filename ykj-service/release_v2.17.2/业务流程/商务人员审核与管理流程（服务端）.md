# 商务人员审核与管理流程（服务端）

> 文档版本 v1.0 | 面向读者：开发人员、系统管理员

## 概述

商务人员是平台运营中负责邀请医生入驻的角色。服务端提供完整的商务信息管理接口，包括增删改查、审核流转、邀请二维码生成与刷新、药品销量统计等功能。

## 接口总览

| 接口 | 方法 | 权限 | 说明 |
|------|------|------|------|
| `/mgmt/hospital/business` | POST | `hosp:businesses:write` | 添加商务 |
| `/mgmt/hospital/business/{id}` | PUT | `hosp:businesses:write` | 修改商务 |
| `/mgmt/hospital/business/{id}` | DELETE | `hosp:businesses:write` | 删除商务 |
| `/mgmt/hospital/business` | GET | `hosp:businesses:read` | 商务分页查询 |
| `/mgmt/hospital/business/{id}/_approve` | POST | `hosp:businesses:audit` | 审批商务基本信息 |
| `/mgmt/hospital/business/{id}/_sales-count` | GET | `hosp:businesses:read` | 商务药品销量统计 |
| `/mgmt/hospital/business/wx-mini-code/_create-code` | POST | `hosp:businesses:write` | 生成商务注册二维码 |
| `/mgmt/hospital/business/wx-mini-code` | GET | `hosp:businesses:read` | 查询商务注册二维码 |
| `/mgmt/hospital/business/_refresh-qrcode` | POST | `hosp:businesses:refresh` | 刷新全部商务二维码 |
| `/mgmt/hospital/business/{businessId}/_refresh-qrcode` | POST | `hosp:businesses:refresh` | 刷新指定商务二维码 |

## 业务流程详解

### 商务信息管理

**添加商务**（`save`）：直接保存商务信息到数据库，初始审核状态为"待审核"。

**编辑商务**（`edit`）：
- 先查询该商务的当前信息
- 如果商务已是"审核通过"状态（`BUSINESS_FLOW_STATUS_PASS`）且还没有邀请二维码，则自动调用 `ShareQrCodeAppService` 生成场景二维码
- 二维码 ticket 存储到商务记录的 `doctorQrcide` 字段

**删除商务**：逻辑删除（软删除）

**分页查询**：支持按商务姓名、审核状态、注册时间范围等条件筛选

### 审核流转

审核接口 `/{id}/_approve` 接收审核结果：
- **前置校验**：`businessId` 不能为空，否则抛出"商务不存在"异常
- **审核通过**（`flowStatus = BUSINESS_FLOW_STATUS_PASS`）：
  - 如果商务还没有邀请二维码，自动调用 `ShareQrCodeAppService.generateSceneQrCode()` 生成二维码
  - 二维码包含 `BusinessInviteDoctorPayload`（含 `businessId`），用于医生扫码注册时关联
- **审核不通过**：仅更新审核状态和备注

**状态枚举**（来自 `Business` 模型）：
- 10：待审核
- 20：审核通过（`BUSINESS_FLOW_STATUS_PASS`）
- 30：审核不通过

### 二维码管理

**邀请医生二维码**：
- 审核通过时自动生成，存储为微信场景二维码的 ticket
- 支持单个刷新和批量刷新
- 刷新时只处理已审核通过的商务
- 刷新指定商务时，如果商务不存在（`business == null`），抛出"商务不存在"异常

**注册二维码**：
- 商务注册入口的公共二维码
- 通过 `refreshRegisterQrCode()` 重新生成
- 通过 `getRegisterQrCode()` 获取当前二维码

### 药品销量统计

接口 `/{id}/_sales-count`：
- 接收商务ID和可选的日期范围
- 调用 `OrderQueryService.findBusinessMedicineCounts()` 查询该商务邀请的医生产生的药品销售数量
- 如果 `businessId` 为空，抛出 `BizException.badRequest()`

## 权限控制

| 权限码 | 可执行操作 |
|--------|-----------|
| `hosp:businesses:write` | 添加、编辑、删除商务，生成注册二维码 |
| `hosp:businesses:read` | 分页查询商务、查询销量、查看注册二维码 |
| `hosp:businesses:audit` | 审核商务信息 |
| `hosp:businesses:refresh` | 刷新邀请二维码（批量或单个） |

## 异常分支

| 异常 | 触发条件 | HTTP状态 |
|------|---------|----------|
| 商务不存在 | 审核或刷新时 `businessId` 为空或查询不到 | 400 |
| 请求参数错误 | 销量统计时 `businessId` 为空 | 400 |

## 关联服务

- **ShareQrCodeAppService**（会员模块）：生成微信小程序场景二维码
- **OrderQueryService**（订单模块）：统计商务关联医生的药品销量
- **BusinessService**（医院模块）：商务信息CRUD和审核处理
