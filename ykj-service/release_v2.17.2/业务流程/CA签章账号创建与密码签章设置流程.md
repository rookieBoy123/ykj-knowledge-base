# CA签章账号创建与密码签章设置流程

> 文档版本：release/v2.17.2 | 面向读者：技术运营、管理员

---

## 概述

每个需要在电子处方或电子病历上签名的用户（医生/药师）都必须先在CA（电子认证）平台创建账号，然后设置签章密码和签章图片。本流程覆盖了CA账号的创建、资料同步、认证状态管理、密码/签章设置、手机号变更和账号停用等完整的账号生命周期。

## 触发条件

- **前置条件**：医生/药师已完成平台入驻审核，需要使用电子签章功能
- **入口路径**：
  - 小程序端：医生端 → 设置 → 放心签设置（CA签章设置页面）
  - 服务端：通过 `POST /ca/user/create` 等接口直接调用

## 步骤详解

### 第一步：创建CA账号

当医生/药师首次触发签章功能时，系统自动为其创建CA账号：

1. **信息校验**：系统接收用户的姓名和身份证号
2. **加密存储**：身份证号经过加密后存储到数据库，保证数据安全
3. **去重检查**：先按"姓名+加密后的身份证号"查询是否已存在同名同证用户。如果已存在，复用已有账号编号（userNo）；否则生成新的25位随机账号编号
4. **资料同步**：将用户信息同步到第三方CA平台（如放心签），获取平台返回的openId
5. **创建关联记录**：在用户-平台关联表中创建记录，初始认证状态为"认证中"（Authing）

⚠️ 如果CA平台同步失败（返回openId为空），账号创建失败，系统记录错误日志但不阻塞主流程。

### 第二步：CA认证审核

账号创建后进入"认证中"状态，等待用户在CA平台完成实名认证：

1. CA平台审核通过后，通过回调通知平台
2. 系统异步处理认证结果：更新认证状态为"认证成功"（Succeed）或"认证失败"（Failure）
3. 认证成功时，CA平台返回签章图片（stamp），系统自动保存
4. 认证失败时，记录失败原因（failReason）
5. 认证完成后，系统发布`UserAuthEvent`事件，通知相关业务模块（如处方签章可用性校验）

### 第三步：设置签章密码

用户认证成功后，需要设置签章密码才能进行签名操作：

1. 用户输入密码，系统对其进行MD5加密后存储
2. ⚠️ **前置检查**：系统先验证当前CA平台是否允许用户修改配置（`allowUpdateSetting`），如果不允许则提示"暂不支持用户修改配置"
3. 如果用户还没有签章图片（signImage为空），系统自动生成一个默认的文字签章——根据用户姓名生成"文字转图片"的签章图
4. 密码设置成功后，后续所有签名操作（如处方签章、病历签章）都需要输入此密码进行验证

### 第四步：设置签章图片

用户可以自定义签章图片（如手写签名图片）：

1. 用户上传签章图片
2. **必须验证密码**：系统先校验用户输入的密码是否正确（MD5比对），密码错误则提示"您的密码验证不通过, 请重新输入"
3. 密码验证通过后，更新签章图片到数据库
4. 如果用户上传的签章图片为空，保持原有签章不变

⚠️ 签章图片设置也需要先通过 `allowUpdateSetting` 检查。

### 第五步：变更手机号

当用户需要更换绑定手机号时：

1. 系统更新本地数据库中的手机号（加密存储）
2. **注意**：系统注释掉了"新手机号是否已被其他CA用户绑定"的校验逻辑——这意味着目前允许同一手机号关联多个CA账号
3. 将新手机号同步到CA平台
4. 同步成功后，将用户认证状态重置为"认证中"（Authing），用户需要在新手机号上重新完成实名认证

⚠️ 如果同步到CA平台失败，提示"变更手机号码失败，请稍后重试"。

### 第六步：停用账号

当用户不再需要使用签章功能时：

1. 系统将用户的认证状态设置为"未认证"（NotAuth）
2. 停用后用户无法再进行签名操作
3. 可以通过重新认证恢复使用

## 异常处理

| 情况 | 系统处理方式 |
|------|------------|
| CA服务配置异常 | 系统抛出内部错误"CA服务配置异常"，需运维排查CA平台对接配置 |
| 创建账号时CA平台同步失败（openId为空） | 记录错误日志，账号创建返回null，不影响用户的其他操作 |
| 查找账号时userNo为空 | 提示"参数错误" |
| 认证回调时找不到对应的平台关联记录 | 记录警告日志"用户授权不存在"，不执行后续操作 |
| 认证回调时找不到对应的CA用户 | 记录错误日志"账号不存在" |
| 手机号变更时用户不存在 | 记录警告日志，静默失败 |
| 手机号变更时同步CA平台失败 | 抛出异常"变更手机号码失败，请稍后重试" |
| 设置密码/签章时平台不允许修改配置 | 提示"暂不支持用户修改配置" |
| 设置签章图片时密码验证不通过 | 提示"您的密码验证不通过, 请重新输入" |
| 查询用户认证状态时资料未同步 | 返回认证状态"未认证"（NotAuth），不报错 |

## 关联功能

- **处方签章**：处方提交时需要验证CA签章密码，并通过CA平台完成电子签章
- **病历签章**：电子病历签署时需要调用CA签章服务
- **会诊报告签章**：会诊报告完成后需要多方签名
- **用户认证事件**：认证状态变更后通过事件机制通知相关模块
- **开发者平台**：CA平台对接配置（appId、平台类型）
- **图片工具**：自动生成默认文字签章图片

## 代码来源

- `ykj-service-ca/.../service/impl/CaUserServiceImpl.java` (411行) — CA签章账号核心服务
- CA账号编号生成：19字节安全随机数 → Base64编码 → 25位字符（去除特殊字符）
- 密码加密：MD5不可逆加密存储
- 身份证号/手机号：使用 `Crypto` 组件加密存储
