# 咨询订单创建与状态流转流程

> 文档版本 2026-07-05 | 面向读者：产品经理、运营、客服

## 概述
咨询订单是用户在小程序发起咨询服务时生成的订单类型（订单来源类型编号 7，对应 OrderSourceType.COUNSEL_APPOINT）。与问诊/会诊/药品等订单并行，有独立的状态流转规则和取消逻辑。

## 触发条件
- 用户在小程序内发起咨询服务
- 系统根据咨询服务的定价创建订单

## 步骤详解

### 第一步：订单创建
用户发起咨询后，系统创建咨询订单：
- 金额为 0 元时：订单支付后直接完成（PAYED_STATUS 映射为 PAYED，0元直接进入该状态）
- 金额大于 0 元时：进入待付款状态，需用户完成支付

### 第二步：支付处理
- 支付完成后，咨询订单进入「已付款（待处方审核）」状态（代码值 15）
- 与药品订单相同，咨询订单支付后不会直接完成，而是处于「待处方审核」状态
- 微信支付和支付宝均可使用（取决于后台支付宝开关）

### 第三步：订单完成
- 由后台管理人员或系统事件触发完成
- 完成时会发布 `OrderCompletedEvent` 事件，通知其他业务模块（如咨询记录归档、IM 会话状态更新等）

### 第四步：订单取消
- 已完成的咨询订单允许取消（与问诊订单、预约订单一同列入"已完成可取消"白名单）
- 取消时如果已产生收入流水，系统自动创建退款申请
- 退款审核拒绝后，系统可将订单状态恢复为退款前的状态

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 咨询订单金额为 0 元 | 支付后直接标记为"已付款"，无需实际支付 |
| 已完成状态取消 | 允许取消（与问诊、预约类型相同），生成退款申请 |
| 退款审核拒绝 | 系统可恢复订单到退款前状态（通过 originFlowStatus 字段） |
| 取消时有多笔收入流水 | 逐笔创建退款申请，单笔失败不影响其他流水 |

## 状态流转

```
创建订单 → 待付款(10)
              ├─ 支付成功 → 已付款/待处方审核(15) → 完成(60)
              ├─ 取消(用户主动) → 已取消(70) / 退款中(89) → 已退款(90)
              └─ 超时未付 → 已失效(80)
```

## 关联功能
- 咨询服务管理（小程序患者端）
- 退款审核管理（管理后台 financemanage/reimburselist）
- 订单流水管理（管理后台 financemanage/collectlist）
