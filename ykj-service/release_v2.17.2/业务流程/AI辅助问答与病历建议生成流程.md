# AI辅助问答与病历建议生成流程

> 文档版本 2026-07-02 | 面向读者：产品经理、运营人员

## 概述

在医生与患者进行在线问诊时，系统利用AI能力自动分析当前问诊的上下文信息（患者资料、病史、聊天记录等），为医生推荐后续可以提问的问题，以及辅助生成病历草稿。这是一个医生效率工具，旨在减少医生在问诊过程中的重复思考工作。

## 触发条件

- **前置条件**：医生已接诊，问诊处于进行中状态
- **功能开关**：需要系统管理员在后台开启对应AI功能开关（问答辅助和病历辅助是两个独立的开关）
- **入口**：医生在接诊工作台中使用"智能问答推荐"或"智能病历辅助"功能

## 步骤详解

### 第一步：触发辅助建议

医生在问诊界面选择需要AI辅助的功能类型：
- **问答辅助**：获取AI建议的后续提问问题（最多9条）
- **病历辅助**：生成病历草稿建议（包含主诉、现病史、诊断建议等信息）

### 第二步：上下文信息自动收集

系统自动收集以下信息作为AI分析的素材：

1. **问诊单信息**：患者填写的病情描述
2. **患者基本信息**：根据身份证自动计算年龄和性别
3. **病史信息**（按优先级合并）：
   - 患者档案中的基础病史（过敏史、既往病史）
   - AI预问诊采集的病史（如果已完成AI预问诊并生成报告）
   - 医生已创建的病历中的病史
4. **聊天记录**：问诊室中医生和患者的对话历史

多个来源的病史按"医生优先"策略合并：医生已填写的病史不会被预问诊或病历中的信息覆盖。

### 第三步：AI生成并返回建议

系统将收集到的信息发送给AI引擎，AI返回建议内容：
- **问答辅助**：返回最多9条建议问题，医生可以从中选择发送给患者
- **病历辅助**：返回病历草稿（症状描述、诊断建议等），系统还会将AI建议的诊断名称与标准诊断库匹配，自动补全标准编码

### 第四步：医生审阅采用

AI返回的建议仅供参考，医生需要审阅后决定是否采用。

## 异常处理

| 情况 | 系统处理方式 |
|------|------------|
| AI功能开关未开启 | 提示"系统暂不支持医生问答辅助/病历辅助，请联系管理员" |
| 问诊单不存在 | 提示"没有相关问诊单" |
| 患者信息缺失 | 提示"没有患者信息" |
| 连续3次获取失败 | 自动锁定该问诊的AI辅助功能30分钟，期间提示锁定剩余时间 |
| 聊天记录为空 | 使用患者填写的病情描述作为默认输入，不中断流程 |
| 获取预问诊/病历信息异常 | 静默跳过该来源（不报错），仅使用已成功获取的信息 |
| AI服务返回异常 | 记录错误次数，连续失败3次后将锁定；不会对医生弹窗报错 |

## 关联功能

- **AI预问诊**：预问诊报告中的病史信息是AI辅助建议的数据来源之一
- **病历管理**：已创建的病历中的病史信息会被AI参考
- **即时通讯（IM）**：问诊室聊天记录作为AI分析的对话上下文
- **系统配置**：管理员通过后台开关控制问答辅助和病历辅助的启用
