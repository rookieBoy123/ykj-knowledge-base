# 处方结算与医保支付完整链路

> 文档版本 v1.0 | 面向读者：产品经理、运营人员

## 概述

患者确认用药建议后进行处方结算的完整后端链路，包括预结算（计算费用）和正式结算（创建订单）两个阶段。支持普通支付和医保支付两种模式，医保支付需要先上传费用明细、获取授权码、完成支付下单。

## 结算状态流转

```
医生开方 → 预结算（计算费用）→ 正式结算（创建订单）→ 待支付 → 已支付 → 已发货 → 已完成
                                    ↓ 拒绝/过期
                                  处方失效（需重新开方）
```

## 核心流程

### 一、预结算阶段

1. **校验处方状态**：
   - 处方不存在 → 提示「处方不存在, 请联系管理员」
   - 处方已结算完成 → 提示「处方已结算完成, 请勿重复操作」
   - 医生开方进行中 → 提示「医生开方中, 请稍后再试」
   - 处方审核不通过/已过期 → 提示「处方审核不通过或者已过期, 请联系医生重新开方」
   - 先审方后支付模式：药师未审核 → 提示「药师未审核处方, 请稍后再试」
   - 处方已过期 → 提示「处方已过期, 请联系医生重新开方」

2. **计算费用项**：
   - 药品金额
   - 服务加工费（医生可选服务+患者可选服务）
   - 快递费（快递模式下根据收货地址计算）
   - 定金（货到付款模式下，满足条件时计算）
   - 总金额为负数时置为0

3. **加工服务确认规则**：
   - 医生开方时未选服务 → 患者可自由选择
   - 医生开方时已选服务 → 患者只能选或不选，不能变更为其他服务（提示「开方时已选加工服务, 不可变更成其他服务」）

### 二、正式结算阶段

1. **检查已有订单**：如果已存在有效订单 → 提示「已存在订单信息」
2. **重新预结算**：获取最新的费用计算结果（不合并服务费）
3. **创建订单**：
   - 设置处方过期时间（根据系统配置的订单过期时长）
   - 校验患者是否可用（不可用/不存在时提示「创建处方订单时发生异常, 患者不可用或者不存在」）
   - 判断是否医保订单（根据问诊的费用类型）
   - 快递模式：计算运费，记录收货地址
   - 到店取药模式：不设置收货地址
4. **扣减药品库存**（非开放平台来源）
5. **回填订单号到处方记录**

### 三、医保支付流程

当患者选择医保支付时：

1. **上传费用明细**：收集处方药品、诊断、患者、医生信息，调用医保接口上传
2. **获取授权码**：通过微信医保授权获取支付授权码
3. **支付下单**：调用医保支付下单接口获取预结算结果
4. **诊费处理**：首张处方添加诊费上传（诊金超过10元按10元计算）
5. **个账支持**：根据系统是否支持个人账户，决定是否允许使用医保个人账户支付
6. **历史费用取消**：如果已上传过费用明细，先取消历史上传再重新上传

### 四、订单详情查询

查看已创建订单的详情时：
- 查询订单基本信息
- 查询订单医保扩展信息（是否调用医保支付接口）
- 查询支付渠道组合（多个支付方式用+连接）
- 查询费用项明细
- 查询收货地址（快递模式）
- 判断是否符合定金支付条件

### 五、医保报销明细

- 仅支持医保订单查询
- 非医保订单查询时提示「该订单不支持查询明细」
- 生成HTML模板渲染为图片保存到OSS存储
- 包含患者信息、就诊时间、结算时间等

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 处方不存在 | 后端拦截「处方不存在, 请联系管理员」 |
| 处方已结算 | 后端拦截「处方已结算完成, 请勿重复操作」 |
| 医生开方中 | 后端拦截「医生开方中, 请稍后再试」 |
| 处方审核不通过/过期 | 后端拦截「处方审核不通过或者已过期, 请联系医生重新开方」 |
| 药师未审核 | 后端拦截「药师未审核处方, 请稍后再试」（仅先审方后支付模式） |
| 处方已过期 | 后端拦截「处方已过期, 请联系医生重新开方」 |
| 已存在订单 | 后端拦截「已存在订单信息」 |
| 订单信息不存在 | 后端拦截「订单信息不存在！」 |
| 患者不可用 | 后端拦截「创建处方订单时发生异常, 患者不可用或者不存在」 |
| 处方药品信息为空 | 后端拦截「创建订单失败, 处方药品信息为空, 请联系管理员」 |
| 订单费用项为空 | 后端拦截「订单费用项信息不存在！」 |
| 收货地址不存在 | 后端拦截「该订单收货地址信息不存在」 |
| 找不到药房信息 | 后端拦截「找不到对应药房信息」 |
| 医保授权码为空 | 后端拦截「授权码不能为空」 |
| 非医保订单查明细 | 后端拦截「该订单不支持查询明细」 |
| 加工服务变更 | 后端拦截「开方时已选加工服务, 不可变更成其他服务」 |

## 关联模块

- **订单服务** (ykj-service-order)：订单创建、支付、查询
- **药房服务** (ykj-service-medicine)：药品信息、库存
- **会员服务** (ykj-service-member)：收货地址、微信OpenId
- **医保服务** (ykj-service-yb)：费用上传、支付下单、授权码
- **医生服务** (ykj-service-hospital)：医生信息
- **患者服务** (ykj-service-hospital)：患者信息、医保信息
- **运费服务** (ykj-service-order)：运费计算
