# 即时通讯账号与群组管理流程

> 文档版本：2026-07-03 第5轮 | 面向读者：产品经理、测试人员

## 概述

即时通讯（IM）是问诊和会诊场景中医生与患者沟通的基础能力。系统基于腾讯云TIM（即时通信）服务，管理IM账号的创建、缓存、群组生命周期、成员进退等操作，确保医患双方在问诊过程中可以顺畅发送文字、图片、语音、视频等多种类型的消息。

## 触发条件

- **前置条件**：问诊单已创建、会诊已发起
- **入口**：问诊/会诊进入沟通阶段时，系统自动创建IM账号和群组
- **关联模块**：ykj-service-inquiry（问诊）、ykj-service-consultation（会诊）

## 步骤详解

### 第一步：创建IM账号

当用户（患者或医生）首次进入IM沟通场景时，系统为其创建IM账号：

1. 系统传入本平台用户ID（localAccountId）、昵称（nickname）、头像（avatar）
2. 查询该用户是否已有IM账号：
   - **已存在**：只更新昵称和头像信息，不重新生成账号
   - **不存在**：通过UUID生成全局唯一的16位账号ID（accountId），存入数据库
3. 将账号信息推入Redis缓存（key：`im:account:id`，field：本地用户ID，value：账号JSON），减少后续查询的数据库压力
4. 返回生成的IM账号ID供后续使用

### 第二步：服务启动时预热缓存

系统启动时（@PostConstruct），自动从数据库加载所有已有的IM账号到Redis缓存中，减少服务运行期间的数据库查询。

### 第三步：创建群组

当问诊或会诊进入沟通阶段，系统创建一个IM群组，将医生和患者添加到群中：

1. 创建群组记录（TimGroupEntity），包含群组ID、群组名称等信息
2. 采用防重复机制：如果群组ID已存在（DuplicateKeyException），直接忽略（记录警告日志），不中断流程
3. 群组支持更新（update）：可按ID或群组ID更新群组信息

### 第四步：成员加入群组

就诊开始时，将参与方加入群组：

1. 系统调用 `joinGroup` 接口，传入群组ID、加入类型（joinType）、操作人ID、要加入的用户ID列表
2. 批量写入群组成员记录（TimGroupMemberEntity），包含群组ID、成员用户ID、加入类型

### 第五步：成员退出群组

就诊结束时，成员退出群组：

1. 调用 `exitMembers` 接口，传入群组ID、要退出的用户ID列表、退出类型（exitType）
2. 系统将匹配的成员记录标记为"已退出"（exited=true），并记录退出类型
3. 支持批量操作，一次可退出多个成员

### 第六步：解散群组

当问诊结束或会诊完成后，系统解散群组：

1. 按群组ID查找群组记录
2. 如果群组不存在，记录警告日志
3. 将群组标记为"已解散"（dissolved=true），群组数据保留但不再可用

### 第七步：消息类型转换

系统支持多种消息类型的传输策略：

| 消息类型 | 处理策略 |
|---------|---------|
| 图片消息 | ImageMessageTransferStrategy — 支持图片上传到OSS |
| 语音消息 | SoundMessageTransferStrategy — 语音文件处理 |
| 视频消息 | VideoMessageTransferStrategy — 视频文件处理 |
| 默认文本 | DefaultMessageTransferStrategy — 普通文本消息 |

不同消息类型在转发时会经过相应的策略处理，实现文件的上传和URL转换。

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 群组已存在（重复创建） | 捕获DuplicateKeyException，记录警告日志，不中断流程 |
| 群组不存在（解散时） | 记录警告日志"tim group not exists"，不做其他操作 |
| 更新群组失败 | 记录警告日志"failed to update tim group" |
| 本地账号ID为空 | Redis缓存跳过，不写入 |
| 账号ID重复（UUID碰撞） | 递归调用getAccountId()重新生成，直到不重复 |

## 关联功能

- 问诊聊天（ykj-service-inquiry）：医生患者在线沟通
- 会诊聊天（ykj-service-consultation）：多方会诊沟通
- OSS存储（ykj-core-oss）：图片/语音/视频文件上传存储
- 腾讯云TIM：底层IM能力提供方
