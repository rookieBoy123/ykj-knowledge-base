# 问诊预约与生命周期管理流程

> 文档版本 v1.0 | 面向读者：技术运维、产品经理

## 概述
描述问诊单从创建到结束的完整后端处理逻辑，覆盖问诊预约、激活、接诊、转诊、超时自动处理、会员绑定、病历关联、IM聊天室管理等全生命周期。

## 触发条件
- 患者发起问诊/预约挂号
- 医生接诊/转诊
- 定时任务自动结束超时问诊
- 商城用药需求触发医生匹配

## 核心流程

### 流程一：问诊单查询（多端多维度）
**医生端列表：**
- **待接诊** → 查询该医生为主治医生且状态为待接诊的记录，含医助待接诊状态（主诊医生也可接诊）
- **问诊中** → 主治医生接诊中（自己）或医助接诊中/再次等待（他人）
- **已完成** → 状态为已完成或已取消且结束标记已设置的
- **已预约** → 已预约但未签到的问诊（存在预约记录且未签到）
- 排序：待接诊/问诊中按问诊开始时间和创建时间正序；已完成按倒序

**患者端列表：**
- **已预约** → 问诊中但预约未签到的（wait）
- **问诊中** → 排除预约未签到的问诊中记录（underway）
- **已完成** → 医生结束标记已设置（finish）
- **待支付** → 草稿状态（draft）

### 流程二：问诊冲突检测（findInquiring）
发起问诊前检测是否已有相同类型的进行中问诊：

1. **自己不能问诊自己** → 会员所属医生不能问诊自己账号下的医生
2. **商城问诊** → 只校验是否有相同用药需求编号的问诊
3. **同患者+同会员+同医生** → 同一服务类型不允许同时存在多个进行中问诊（待支付/问诊中）
4. **预约挂号同一天同一医生** → 同一会员同一患者同一医生同一天只能有一个号段

### 流程三：激活问诊单（enableInquiry）
1. **参数校验** → 问诊单号不能为空，问诊单必须存在
2. **重复激活检测** → 问诊单已存在GroupId则不能重复激活
3. **状态变更** → 问诊单状态更新为"问诊中"
4. **创建IM聊天室** → 条件：非禁用聊天室且已绑定会员
   - 复诊类型 → 医生在创建时直接加入聊天室
   - 非复诊 → 医生不进群，需接诊后加入
   - 失败 → 提示"聊天室创建失败，请联系管理员"
5. **更新导诊数据** → 如果关联了导诊记录，同步更新患者信息到导诊系统
6. **更新用药需求数据** → 如果关联了用药需求编号，更新需求状态为"已进入问诊"

### 流程四：激活并接诊（activateInquiry）
会员绑定后调用，一次性完成激活和接诊：
1. 激活问诊单
2. 发送问诊开始通知（系统消息/订阅消息）
3. 如果 reception=true，主动触发医生接诊

### 流程五：超时自动结束（定时任务）
定时任务 finishTimeoutInquiryRecords：
1. **总开关检查** → 配置 autoEnd=false 时直接关闭
2. **状态检查** → 问诊单必须处于草稿或问诊中状态
3. **服务医生检查** → 
   - 有服务医生且主治医生接诊中 → 执行转诊
   - 有服务医生且医助接待中 → 超时不处理，直接取消问诊（原因："超过接诊时间，系统自动取消"）
   - 无服务医生 → 直接结束问诊

### 流程六：会员绑定（bindMember）
1. **机构校验** → 问诊机构与当前机构必须一致
2. **问诊状态校验** → 只允许草稿和问诊中状态绑定
3. **参与医生冲突检测** → 问诊参与医生不能绑定为患者
4. **重复绑定检测** → 
   - 已绑定且memberId相同 → 直接返回
   - 已绑定但memberId不同 → 提示"问诊单已被领取"
5. **执行绑定** → 更新问诊主表和患者表
6. **自动激活** → 非禁用聊天室时自动调用 activateInquiry

### 流程七：病历关联（bindMedicalRecord）
1. 问诊单必须存在
2. 已有病历 → 
   - 相同病历ID → 直接返回
   - 不同病历ID → 提示"问诊单已形成病历，不能变更"
3. 未关联病历 → 绑定病历ID

### 流程八：病历写入（writeMedicalRecord）
1. 问诊单已有病历 → 提示"问诊单已存在病历，请重新进入页面编辑后提交"
2. 组装病历创建命令（医生ID/会员ID/患者信息/病历图片）
3. 调用病历服务创建
4. 回填病历ID到问诊单

### 流程九：量表推送（sendImAboutEvaluateScale）
问诊结束后，通过IM聊天室推送量表测评卡：
- 发送量表卡片消息（医生端）
- 发送结果链接消息（患者端），患者点击跳转到量表测试结果详情页

### 流程十：商品推荐分享（medShare）
1. 校验问诊单号和分享编号
2. 获取推荐商品列表
3. 推送系统消息
4. 组装并发送推荐商品卡片（含图片/名称/规格/价格/购买按钮）

## 状态枚举

| 字段 | 取值 | 含义 |
|------|------|------|
| flowStatus | DRAFT | 草稿（待支付） |
| flowStatus | WAIT_PAY | 待支付 |
| flowStatus | UNDERWAY | 问诊中 |
| flowStatus | INQUIRING | 问诊进行中 |
| flowStatus | FINISH | 已完成 |
| flowStatus | CANCEL | 已取消 |
| doctorFlagStatus | WAIT | 医助待接诊 |
| doctorFlagStatus | UNDERWAY | 医助接诊中 |
| doctorFlagStatus | MAIN_WAIT | 主治医生待接诊 |
| doctorFlagStatus | MAIN_UNDERWAY | 主治医生接诊中 |
| doctorFlagStatus | AGAIN_WAIT | 再次等待 |
| doctorFlagStatus | FINISH | 已结束 |

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 问诊单不存在 | 前端提示"问诊单不存在" |
| 问诊单已激活 | 前端提示"问诊单已经激活，请勿重复操作" |
| 聊天室创建失败 | 前端提示"聊天室创建失败，请联系管理员" |
| 问诊单已被领取 | 前端提示"问诊单已被领取!" |
| 问诊单已形成病历 | 前端提示"问诊单已形成病历，不能变更" |
| 问诊已结束或已取消 | 前端提示"问诊已结束或已取消！" |
| 医生不能问诊自己 | 前端提示"系统不支持问诊自己账号下医生" |
| 参与医生不能作为患者 | 前端提示"问诊单参与医生不能作为问诊单患者" |
| 机构不匹配 | 前端提示"参数异常！" |
| 自动结束开关关闭 | 日志记录"自动结束问诊任务关闭" |
