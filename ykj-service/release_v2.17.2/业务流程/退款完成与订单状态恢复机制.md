# 退款完成与订单状态恢复机制

> 文档版本：v2.17.2 | 面向读者：后端开发、测试、运维

## 概述

当退款申请被审核拒绝或退款失败后，订单需要从"退款中"（REFUNDING）状态恢复到退款前的原始业务状态（如"已支付""已完成"等）。v2.17.2 新增了 `originFlowStatus`（退款前原始状态）字段和 `restoreOrderFlowStatus` 方法来实现这一机制。

## 触发条件

- 退款审核拒绝：管理员驳回退款申请后，系统调用 `restoreOrderFlowStatus` 恢复订单状态
- 退款执行失败：支付渠道退款失败后，系统将订单恢复到退款前状态（而非一直卡在"退款中"）

## 步骤详解

### 第一步：记录原始状态

当订单触发退款流程时（取消订单 + 需要退款），系统在取消订单的同时记录退款前的原始状态：

```
取消订单（触发退款）→ originFlowStatus = 取消前的订单状态
例如：订单当前"已完成" → 取消触发退款 → originFlowStatus = COMPLETED
     订单当前"已支付" → 取消触发退款 → originFlowStatus = PAYED
```

原始状态记录在订单表的 `origin_flow_status` 字段中。

### 第二步：订单进入"退款中"

退款申请创建后，订单状态变更为 REFUNDING（退款中），等待管理员审核。此时 `originFlowStatus` 保留着进入退款前的状态值。

### 第三步：审核或退款结果处理

| 结果 | 动作 |
|------|------|
| 审核通过 + 退款成功 | 订单状态变为 REFUNDED（已退款），originFlowStatus 不再需要 |
| 审核拒绝 | 调用 restoreOrderFlowStatus，将订单恢复为 originFlowStatus 记录的状态 |
| 退款失败 | 同上，恢复到退款前的状态 |

### 第四步：状态恢复（restoreOrderFlowStatus）

恢复方法执行以下校验：

1. **必须处于"退款中"状态**：如果订单不是 REFUNDING，抛出 "当前订单非退款中状态，无需恢复"
2. **必须有原始状态记录**：如果 `originFlowStatus` 为空（null），抛出 "未找到退款前的订单状态，无法恢复"
3. **执行恢复**：通过 `preStatus = OrderFlowStatus.valueOf(originFlowStatus)` 将订单状态设回原始状态，同时将 `originFlowStatus` 清空为 null
4. **乐观锁保护**：使用 `LambdaUpdateWrapper` 更新，如果更新影响行数为0，抛出更新失败异常

## 状态流转图

```
【正常流程】
已支付 ──→ 退款中 ──→ 审核通过+退款成功 ──→ 已退款

【拒绝恢复流程】
已支付 ──→ 退款中 ──→ 审核拒绝 ──→ 恢复到"已支付"
已完成 ──→ 退款中 ──→ 审核拒绝 ──→ 恢复到"已完成"
```

## 与旧版本的区别

| 维度 | 旧版 (updateFlowStatus) | 新版 (restoreOrderFlowStatus) |
|------|------------------------|------------------------------|
| 调用方式 | 手动传入目标状态值 | 自动从 originFlowStatus 读取 |
| 安全校验 | 无状态校验，直接覆盖 | 必须处于"退款中"状态，必须有原始状态记录 |
| 数据一致性 | 无乐观锁保护 | 有乐观锁版本号保护 |
| 原始状态来源 | 依赖调用方记住 | 数据库字段自动记录 |

## 异常处理

| 情况 | 系统处理方式 |
|------|------------|
| 订单未处于"退款中" | 抛出 "当前订单非退款中状态，无需恢复" |
| originFlowStatus 为空 | 抛出 "未找到退款前的订单状态，无法恢复" |
| 乐观锁冲突（并发恢复） | 更新影响0行，抛出 "恢复订单状态失败" |
| 数据库更新异常 | 抛出 "恢复订单状态失败" 内部错误 |

## 关联功能

- **订单取消**：取消时如果触发退款，记录 `originFlowStatus = 取消前状态`
- **退款审核**：管理员审核通过/拒绝退款申请
- **退款执行**：支付渠道实际退款操作
- **服务模块**：`ykj-service-order` 的 `OrderCommandServiceImpl.restoreOrderFlowStatus()`，数据表 `t_order.origin_flow_status`
