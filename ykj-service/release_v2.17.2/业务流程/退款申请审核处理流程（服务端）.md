# 退款申请审核处理流程（服务端）

> 文档版本 v1.0 | 面向读者：开发人员、产品经理

## 概述

退款申请管理模块负责退款申请单的全生命周期管理：创建申请 → 审核（通过/拒绝）→ 执行退款 → 状态同步。该模块依赖订单模块获取支付流水信息，调用微信/支付宝支付服务执行资金退回。

服务模块：`ykj-service-order`

## 触发条件

- 用户在前端（小程序/管理后台）发起退款申请
- 后端创建退款申请单（`createRefundApply`）
- 审核人员在管理后台审核（`agreeRefundApply` / `rejectRefundApply`）

## 业务流程

### 一、创建退款申请

**接口**：`RefundApplyService.createRefundApply()`

**校验规则**：

1. 同一笔支付流水（`wxTransactionId`）只允许创建一条退款申请，防止重复退款
2. 同一订单号（`orderNo`）+ 自定义编号（`customNo`，如有）组合判重
3. 校验通过后生成退款编号（格式：`Refund` + 时间戳 + 随机数）
4. 初始状态：审核状态=待审核（FLOW_STATUS_BE_AUDIT），退款状态=未退款（REFUND_STATUS_NO_REFUND）

**失败情况**：
- "该订单已提交过退款信息"——同一支付流水已存在退款申请

### 二、审核退款申请

#### 同意退款（agreeRefundApply）

**状态流转**：待审核(10) → 审核通过(20)

**处理逻辑**：

1. 校验申请单存在且状态为"待审核"
2. 设置审核通过状态、退款类型、审核备注、审核时间
3. **退款类型判断**：
   - **手动退款**（Manual_Refund）：直接标记退款状态为"已退款"，不调用支付接口（财务线下打款）
   - **原路退回**（Bracktrack）：立即调用 `doRefund()` 向微信/支付宝发起退款
4. 发布 `RefundApplyApprovedEvent` 事件，通知订单模块同步状态

#### 拒绝退款（rejectRefundApply）

**状态流转**：待审核(10) → 审核不通过(30)

**处理逻辑**：

1. 校验申请单存在且状态为"待审核"
2. 拒绝备注必填（remark 不能为空），提示"请填写审核拒绝备注"
3. 设置审核不通过状态

### 三、执行退款（doRefund）

根据支付渠道调用不同服务：

- **微信支付**→ `WxRefundService.refund()`
- **支付宝**→ `AlipayPaymentService.refund()`

退款结果通过回调更新退款状态：

| 退款状态码 | 状态名称 | 说明 |
|-----------|---------|------|
| 未退款 | REFUND_STATUS_NO_REFUND | 初始状态/允许重试 |
| 退款失败 | REFUND_STATUS_REFUND_FAIL | 支付渠道返回失败，允许重试 |
| 退款中 | — | 等待支付渠道回调 |
| 已退款 | REFUND_STATUS_REFUNDED | 退款成功到账 |

**重试机制**：只有"未退款"和"退款失败"状态允许重新发起退款。

### 四、事件驱动联动

| 事件 | 触发时机 | 消费方 |
|------|---------|--------|
| `RefundApplyApprovedEvent` | 审核通过 | 订单模块更新订单退款状态 |
| `RefundApplySucceedRefundedEvent` | 退款成功 | 通知相关方 |
| `RefundCompletedEvent` | 退款流程完结 | 订单完成状态同步 |

## 异常处理

| 情况 | 处理方式 |
|------|---------|
| 申请单不存在或已被审核 | 后端抛出"申请单审核失败, 申请单不存在或状态不允许审核" |
| 同一流水重复提交 | 后端抛出"该订单已提交过退款信息" |
| 拒绝时未填备注 | 后端校验 remark 为空时抛出异常 |
| 退款调用支付接口失败 | 标记 refundStatus=退款失败，允许重试 |
| 跨渠道重复支付场景 | 审核时注入 payChannel 字段区分微信/支付宝流水 |

## 关联功能

- **订单管理**（ykj-service-order）：退款与原订单关联
- **微信支付**（WxRefundService）：微信退款接口调用
- **支付宝**（AlipayPaymentService）：支付宝退款接口调用
- **交易流水**：退款完成后产生退款流水记录
- **管理后台-退款审核页**：前端审核入口
