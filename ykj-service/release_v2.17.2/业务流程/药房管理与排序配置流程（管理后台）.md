# 药房管理与排序配置流程（管理后台）

> 文档版本 v1.0 | 面向读者：运营管理员

## 概述

管理员在后台管理药房信息，包括新增药房、编辑药房信息、启停药房、调整药房展示顺序，以及管理药房关联的第三方药房、加工服务和医生授权。

## 触发条件

- 前置条件：拥有药房管理权限（med:pharmacies:read/write）的管理员账号
- 入口路径：诊疗管理 → 药房管理

## 流程详解

### 一、新增药房

提交药房信息包含：
- 药房名称、类型（线上/线下）、负责人、联系电话
- 库存管理开关（启用/禁用）
- 药房状态（启用/禁用）
- 可关联第三方药房信息

新增操作会被记录到操作审计日志中，记录内容包括药房名称、类型、负责人、电话、库存配置和状态。

**校验规则**：
- 手机号是否已被其他药房绑定（validateMobile）
- 药房名称是否重复（validateName）

### 二、编辑药房

修改已有药房的信息，同样支持更新第三方药房关联。编辑操作也会记录审计日志。

### 三、药房启停

支持独立启用或禁用药房：
- **启用药房**：`POST /{id}/_enable` — 让药房恢复可用状态
- **禁用药房**：`POST /{id}/_disable` — 禁用的药房不会出现在医生的药房选择列表中

### 四、药房展示排序

药房在小程序端的展示顺序可以由管理员在后台调整，提供四种排序操作：
- **移至顶部**（`_top`）：将指定药房排到列表第一位
- **移至底部**（`_bottom`）：将指定药房排到列表最后一位
- **上移**（`_up`）：将指定药房与上一个位置交换
- **下移**（`_down`）：将指定药房与下一个位置交换

### 五、关联第三方药房

如果药房对接了第三方药品供应商，编辑和查询时会自动关联第三方药房信息。在药房列表中，关联了第三方的药房名称会显示为"药房名称(第三方名称)"格式。

### 六、查询药房加工服务

管理员可以查询指定药房的所有加工服务（包括已启用的和已禁用的），以及查询所有系统支持的加工服务列表。这用于配置药房支持哪些中药代煎加工工艺。

### 七、计算加工服务费

管理员端同样提供加工费计算接口，用于在后台模拟计算给定药房和工艺下的加工费用。

### 八、药房绑定校验

管理员可以校验指定药房是否与当前医生账号绑定（checkBinding），用于排查医生端药房不可见的问题。

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 手机号已被其他药房绑定 | validateMobile返回false，前端拦截 |
| 药房名称重复 | validateName返回false，前端提示名称已存在 |
| 禁用的药房 | 医生端不再显示该药房，已关联的处方不受影响 |
| 缺少写权限 | 接口返回403，操作被拒绝 |

## 关联功能

- 医生药房授权 → 医生需要被授权某药房后才能看到和使用该药房
- 药品管理 → 药房中维护的药品目录、库存、价格
- 处方开方 → 医生选择药房后开具处方
- 第三方对接 → 对接外部药品供应商
