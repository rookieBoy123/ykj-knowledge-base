# CA电子签章用户认证与管理流程

> 文档版本 v1.0 | 面向读者：产品与运营人员  
> 服务模块：ykj-service-ca

## 概述

平台对接第三方CA（电子认证）服务，为医生提供电子签章功能。医生在开具处方、签署会诊报告等场景中需要使用CA签章，本流程涵盖CA账号的创建、同步、认证、签章管理和密码设置。

## 状态流转

```
                                密码设置
未认证(NotAuth) → 认证中(Authing) → 认证成功(Succeed)
                                   → 认证失败(Failure)
                                   → 注销(NotAuth)
```

## 核心流程

### 一、CA账号创建

当医生首次需要使用CA签章时，系统自动为其创建CA账号：

1. 根据医生姓名+身份证号（加密存储）查询是否已有CA账号
2. 若不存在，自动生成唯一用户编号（25位Base64编码字符串）
3. 将医生信息同步到第三方CA平台，获取OpenId
4. 创建CA用户平台关联记录，状态设为"认证中(Authing)"
5. 若同步到第三方CA平台失败（返回空OpenId），记录错误日志但不抛异常

### 二、CA账号认证

医生通过第三方CA平台完成身份认证：

1. 调用第三方CA平台发起认证请求，传入OpenId、姓名、身份证、手机号
2. 保留认证OpenId，创建或更新认证有效期
3. 第三方CA平台回调通知认证结果（成功/失败）
4. 异步处理回调：更新认证状态+签章图片+失败原因+子状态
5. 发布用户认证事件（UserAuthEvent），通知其他模块

### 三、密码与签章管理

**密码设置**：
- 首次设置签名密码（密码经过MD5加密存储）
- 若用户未设置签章图片，自动生成默认签章（将医生姓名转为Base64图片）
- 密码设置能力受CA平台策略控制（allowUpdateSetting），不支持时提示"暂不支持用户修改配置"

**签章图片设置**：
- 修改签章图片前需验证密码 → 密码错误提示"您的密码验证不通过, 请重新输入"
- 可自定义上传签章图片
- 默认签章：取医生姓名通过ImageUtils.textToBase64Image生成

### 四、手机号变更

医生更换手机号时：
1. 查询用户信息，检查用户是否存在
2. 加密新手机号后更新本地数据库
3. 将新手机号同步到第三方CA平台
4. 同步成功后，认证状态重置为"认证中(Authing)"，等待重新认证
5. 同步失败提示"变更手机号码失败，请稍后重试"

### 五、账号注销

将用户平台关联记录的状态设为"未认证(NotAuth)"，取消CA签章能力。

## 异常处理

| 情况 | 处理方式 |
|------|---------|
| CA服务配置异常 | 后端抛出"CA服务配置异常"（internalError） |
| 账号不存在 | 提示"账号不存在"（前端提示） |
| CA账号同步失败 | 返回null，记录错误日志 |
| 用户不存在 | 记录警告日志，静默处理 |
| 修改配置不支持 | 提示"暂不支持用户修改配置" |
| 密码验证不通过 | 提示"您的密码验证不通过, 请重新输入" |
| 手机号变更同步失败 | 提示"变更手机号码失败，请稍后重试" |
| 认证回调时用户不存在 | 记录警告日志，静默跳过 |

## 关键设计

- **多平台支持**：通过策略模式（CaUserStrategy/CaSignStrategy）支持多个CA平台（如医网信、云凯健等）
- **身份证加密**：身份证号使用 Crypto 加密后存储
- **密码加密**：签名密码使用 MD5 加密存储，不可逆
- **异步处理**：认证回调处理（handlerAuthed）使用 @Async 异步执行
- **事件通知**：认证成功后发布 UserAuthEvent，其他模块可监听处理
- **账号编号唯一**：使用 SecureRandom 生成19字节随机数，Base64编码后取25位，并检查数据库唯一性
