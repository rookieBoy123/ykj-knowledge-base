# 开放平台订单全生命周期流程（服务端）

> 文档版本 v1.0 | 面向产品/技术人员

## 概述

开放平台提供一套完整的订单API，供外部合作机构（如阿康健康等第三方药房平台）通过接口完成从预结算→锁定库存→创建订单→支付→发货→取货→取消的完整订单流程。整个流程涉及机构校验、药品库存检查、运费计算、第三方药房对接、事件发布和回调通知等多个环节。

## 触发条件

- **前置条件**：外部机构已注册开放平台AppId，机构配置了关联药房且开启了"需要药品下单"开关
- **入口**：外部系统通过HTTP POST调用 `/open/orders/_xxx` 系列接口，每个请求需携带 `X-App-Id` 请求头

## 完整流程

```
预结算 → 锁定库存 → 创建订单 → 支付 → 发货 → 取货完成
   ↓                      ↓
  可取消            也可取消（需原因）
```

## 步骤详解

### 第一步：预结算（`/_preSettlement`）

外部系统提交处方信息进行费用预估，系统返回费用明细但不生成正式订单。

**处理流程**：

1. **机构校验**：通过请求头中的AppId查找机构信息
   - 机构不存在 → 报错"找不到对应的机构信息!"
   - 机构未设置"需要药品下单" → 报错"机构未设置需要药品下单，不能生成订单！"
   - 机构未关联药房 → 报错"机构未设置下单药房，不能生成订单！"

2. **药品校验**：
   - 根据商品ID查询药品信息，药品不存在或数量不匹配 → 报错"不存在对应的药品信息！"
   - 检查所有药品是否属于同一药房 → 报错"同个处方只能选择一个药房的药品"
   - 机构关联的药房必须与药品所属药房一致 → 报错"机构关联的药房和开方药品对应的药房不一致"

3. **库存校验**（两种模式）：
   - **第三方药房模式**（enabledThird=true）：调用第三方药房接口查询实际库存，对比计算是否满足需求。如果平台库存与第三方不一致，自动同步更新平台库存
   - **平台自营药房模式**：直接对比平台库存
   - 库存不足 → 报错"药品库存不足，药品名称：XXX"

4. **费用计算**：
   - 药品金额 = 单剂费用 × 总剂数（向上取整到2位小数）
   - 运费 = 根据收货地址省份和城市 + 药品金额计算
   - 总金额 = 药品金额 + 运费

5. **保存预结算单**：生成预结算单号（presettlementNo），预结算单包含费用明细、收货地址、药品清单等完整属性，但不锁定库存

**返回值**：预结算单号、药品金额、运费、单价、总金额、费用明细列表

### 第二步：锁定库存（`/_lockedInventory`）

预结算完成后，外部系统调用此接口锁定库存，防止超卖。

**处理流程**：

1. 根据预结算单号查找预结算单，不存在 → 报错"找不到对应的预结算单信息"
2. 检查是否已锁定过 → 已锁定则报错"该预约单已锁定过库存！"
3. **再次校验库存**并计算（防止预结算后库存变化）
4. 生成正式订单号（orderNo）
5. **第三方药房**：调用第三方药房下单接口锁定第三方库存（AKANGAddOrderReq）
6. 设置预结算单有效期：默认5分钟（DEFAULT_EFFECTIVE_MINUTES），可通过参数指定
7. 扣减平台库存（updateDrugsStock）

**有效期机制**：锁定库存后，预结算单在有效期内有效。超时后通过定时任务自动取消第三方药房的锁定订单。

### 第三步：创建订单（`/_create`）

外部系统在锁定库存有效期内调用此接口正式创建订单。

**处理流程**：

1. **机构校验**：同预结算，再次校验机构状态
2. **参数校验**：
   - 如果取药方式为"物流快递"（pharmacyMode=2），必须传收货地址，否则 → "取药方式为快递物流，则收货地址不能为空"
   - 校验收货地址各字段合法性
3. 从预结算单中读取费用、药品、地址等属性
4. 调用 orderService 创建正式订单，返回订单号和取货码（pickUpCode）

**订单参数说明**：

| 参数 | 说明 |
|------|------|
| outpatientNo | 处方编号（必填） |
| preSettlementNo | 预结算单号 |
| pharmacyMode | 取药方式：1=到店取货，2=物流快递（默认1） |
| orderReceiverAddress | 收货地址（pharmacyMode=2时必填） |
| payType | 支付方式：1=微信支付 |
| transactionNo | 交易流水号 |
| callBackUrl | 回调地址 |
| ignorePreSettlement | 是否跳过预结算（默认false） |

### 第四步：支付（`/_prepay`）

外部系统调用此接口生成微信支付参数，用户在前端调起支付。

### 第五步：发货（`/_shipment`）

外部系统确认发货后调用此接口，更新订单物流状态。

**处理流程**：
1. 验证AppId权限
2. 调用 openOrderService.shipment 更新订单状态
3. 记录操作日志（@OpenLog）

### 第六步：取货码取货（`/_pick-up`）

到店取货场景下，用户出示取货码，药房工作人员通过取货码完成取货核销。

**参数**：orderNo（订单号）+ pickUpCode（取货码）

### 取消订单（`/_cancel`）

外部系统可在订单未完成前取消订单。

**参数**：orderNo（订单号）+ cancelReason（取消原因，必填）

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 机构未找到 | 后端拦截："找不到对应的机构信息!" |
| 机构未设置下单权限 | 后端拦截："机构未设置需要药品下单，不能生成订单！" |
| 机构未关联药房 | 后端拦截："机构未设置下单药房，不能生成订单！" |
| 药品信息不存在 | 后端拦截："不存在对应的药品信息！" |
| 同一处方选了多个药房的药 | 后端拦截："同个处方只能选择一个药房的药品" |
| 机构关联药房与药品药房不一致 | 后端拦截："机构关联的药房和开方药品对应的药房不一致" |
| 药品库存不足 | 后端拦截："药品库存不足，药品名称：XXX" |
| 第三方药房药品不存在 | 后端拦截："第三方药房药品不存在" |
| 预结算单不存在 | 后端拦截："找不到对应的预结算单信息" |
| 预结算单已锁定 | 后端拦截："该预约单已锁定过库存！" |
| 快递模式下无收货地址 | 后端拦截："取药方式为快递物流，则收货地址不能为空" |
| 收货地址字段校验失败 | 后端拦截：返回具体字段错误 |
| 预结算单过期 | 定时任务自动取消第三方锁定，预结算单失效 |

## 订单状态查询（`/_findOrderStatus`）

外部系统可通过订单号查询订单当前状态，返回 orderNo、flowStatus 和 pickUpCode。

## 事件与回调

订单创建、支付、发货等关键节点会发布领域事件（如 PaidEvent、OfflinePaidEvent），由事件监听器触发后续业务（如发送通知、更新处方状态等）。外部系统可通过 callBackUrl 参数设置回调地址接收订单状态变更通知。

## 关联功能

- **机构管理**：机构需要配置关联药房和下单权限
- **药品管理**：库存校验依赖药品库存数据
- **处方管理**：订单关联处方编号
- **药品运费管理**：运费根据地址和金额计算
- **第三方药房对接**：阿康健康（AKANG）API对接
