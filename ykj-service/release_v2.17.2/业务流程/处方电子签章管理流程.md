# 处方电子签章管理流程

> 文档版本：release/v2.17.2 | 面向读者：产品经理、运营、开发

---

## 概述

平台处方开具后需要进行电子签章才能成为合法有效的处方。签章分为医生签章和药师签章两种角色，支持CA（电子认证）签章和手写文字签名两种方式。CA签章通过对接第三方CA平台（医网信/源康健PaaS/放心签）实现合规的电子签名，手写签名则直接在处方PDF上添加签名文字。

## 触发条件

- **前置条件**：已生成处方PDF文件、医院CA功能配置已开启、医生/药师已注册CA账号
- **入口**：医生开完处方后点击「签章」按钮、药师审核处方后点击「签章」按钮

---

## 步骤详解

### 第一步：创建签章记录

1. 医生/药师点击签章后，后端接收到签章请求（含门诊号、处方PDF的base64编码、签章人信息）
2. 后端生成一个唯一的签章编号（格式：`TR_SIGN` + 日期8位 + 7位随机数，如 `TR_SIGN202607010001234`），确保不重复
3. 保存签章记录到数据库，包含：
   - 签章编号、门诊号
   - 签章人身份（医生/药师）和姓名
   - 签章状态：待签署（PENDING）
   - 处方源文件（PDF上传到OSS）
   - CA是否启用标记

### 第二步：判断是否需要CA签章

系统根据以下规则判断走CA签章还是手写签名：

- **走手写签名**（跳过CA）的情况：
  - 医院未开启CA功能（`enableCA()` 返回false）
  - 外部明确要求不使用CA（`disableCa=true`）
- **走CA签章**的情况：
  - 医院开启了CA功能 AND 未要求禁用CA

### 第三步A：CA签章路径

1. 获取签章人的CA用户编号（医生或药师的`caUserNo`）
2. 组装CA签章创建请求：
   - **业务编号**：签章编号
   - **标题**：来自前端传入
   - **文件**：处方PDF的base64编码
   - **业务类型**：药师签章是「追加签章」模式、医生签章是「创建签章」模式
   - **患者信息**：姓名、身份证号、性别、年龄
   - **签章人信息**：CA用户编号、签章关键词（"医生："或"审方："）、签章位置坐标
3. 调用CA服务创建签章，获取CA唯一标识（`caUniqueId`）
4. 更新签章记录，保存 `caUniqueId`
5. 返回给前端：签章状态、是否启用CA、CA唯一标识

### 第三步B：手写签名路径

1. 跳过CA创建步骤，直接在前端显示确认界面
2. 前端可以显示签名的手写板或文字确认

### 第四步：前往签章

1. 前端拿到 `caUniqueId` 后调用「前往签章」接口
2. 后端校验：
   - 签章记录是否存在（按门诊号 + 签章人ID + 角色查找）
   - 签章状态是否为待签署（`PENDING`），如果已签过则提示"处方已被签署"
3. 返回确认信息，前端跳转到CA平台的签章页面

### 第五步：完成签章

1. 用户在CA平台完成签章操作后，前端回调通知后端
2. 后端根据是否使用CA分支处理：

**CA签章完成**：
1. 调用CA服务查询签章结果（获取已签章的PDF文件base64 + 签章数据）
2. 校验签章记录状态（CA服务返回失败时标记为拒绝REJECT）
3. 签章成功时获取CA用户的手写签名图片（可作为签名预览）
4. 更新签章记录：签章文件OSS存储、签章数据、签章图片、签章时间、状态改为确认CONFIRM

**手写签名完成（无CA）**：
1. 在处方PDF上直接追加文字（"医生：张三" 或 "审方：李四"）
2. 生成带签章文字的PDF并上传OSS
3. 审批不通过时（`pass=false`）状态改为拒绝REJECT

### 第六步：签章撤销

当处方撤回或作废时需要撤销签章：
1. 按门诊号查找所有签章记录
2. 逐一检查：如果已经撤销过则提示"签章记录已撤销，请刷新重试"
3. 状态改为撤销（REVOKE）
4. 如果有CA唯一标识，调用CA服务删除签章远程记录
5. 任何单条撤销失败不影响其他记录的撤销（try-catch保护）

### 第七步：查询签章结果

1. 前端可以轮询查看签章结果
2. 后端找到状态为待签署且有CA唯一标识的最新一条签章记录
3. 主动调用CA服务更新签章状态
4. 返回当前状态给前端

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 创建签章记录插入失败 | 提示"创建签署记录失败，请联系管理员" |
| 处方不需要签字（无签章记录） | 提示"处方不需要签字" |
| 处方已被签署（状态非待签署） | 提示"处方已被签署" |
| 完成签署时未找到签章记录 | 提示"未找到处方签署记录" |
| CA签章时未找到签署记录 | 提示"未找到签署记录" |
| 签章结果查询时未找到待签署记录 | 提示"未找到签署中的处方签署记录" |
| 签章记录已撤销 | 提示"签章记录已撤销，请刷新重试" |
| CA创建签章时更新失败 | 提示"创建签署记录失败，请联系管理员" |
| 处方单号为空 | 分别提示"处方单号不能为空"或"处方不需要签字" |
| 签章完成后更新数据库失败 | 提示"完成签署时发生异常" |
| 审批不通过 | 状态标记为REJECT，仍然生成签章记录并保存 |
| 源文件查找失败 | 按优先级查找：OSS文件ID → 本地文件 → 返回空字符串 |

## 关联功能

- **CA用户管理**：医生/药师CA账号的注册和认证
- **处方开具流程**：处方草稿→提交→签章→生效的完整链路
- **处方记录管理**：查看已签章的处方PDF
- **会诊签章流程**：会诊报告的五种签名类型签章
- **三大CA平台**：医网信、源康健PaaS、放心签

## 设计要点

1. **签章编号唯一性**：`TR_SIGN` + 日期 + 7位随机数，递归检查确保不重复
2. **医生vs药师签章差异**：
   - 医生签章关键词为"医生："、药师为"审方："
   - 药师签章CA业务类型为"追加签章"，医生为"创建签章"
   - 签章位置坐标不同（Y坐标：医生650 vs 药师1950）
3. **非CA审批不通过仍生成记录**：即使审核不通过，也生成带签章文字的PDF（标记为拒绝状态），方便追溯
4. **撤销保护**：已撤销的记录不会再撤销第二次，撤销操作try-catch保护防止部分失败
