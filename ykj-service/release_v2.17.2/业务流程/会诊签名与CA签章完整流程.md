# 会诊签名与CA签章完整流程

> 文档版本 v1.0 | 面向读者：产品经理、运营人员

## 概述

远程会诊各环节（申请、审核、受邀确认、意见提交、报告签署）的电子签名管理。支持两种模式：CA数字签章（调用专业CA厂商接口）和普通手写签名。签名完成后生成会诊记录PDF并触发状态流转。

## 五种签名类型

| 签名类型 | 谁签 | 签名关键词 | 流程影响 |
|---------|------|-----------|---------|
| 申请签名 (APPLICATION) | 主治医师 | 「主管医师签名」 | 提交会诊申请，状态进入待审核 |
| 审核签名 (REVIEW) | 审核医生 | 「审核医师签名」 | 审核通过，状态进入已通过 |
| 受邀签名 (INVITATION) | 受邀医生 | 「确认参加签名」 | 确认参加会诊 |
| 意见签名 (OPINION) | 会诊医师 | 「会诊医师签名」 | 提交会诊意见 |
| 报告签名 (REPORT) | 主治医师 | 「会诊医师签名」 | 完成会诊报告签署 |

## 核心流程

### 一、签名权限校验

签名前校验医生身份：
- **申请签名**：只有会诊的主治医师才能签（非主治医师提示「非主治医生无法签字」）
- **审核签名**：只有指定的审核医生才能签（非审核医生提示「非审核医生无法签字」）
- **受邀签名**：只有受邀医生列表中的医生才能签（非受邀医生提示「非受邀医生无法签字」）
- **意见签名**：只有提交了会诊意见的医生才能签（非意见提交医生提示「非医生意见提交无法签字」）
- **报告签名**：只有主治医师才能签（非主治医师提示「非主治医生无法签字」）

### 二、CA签章模式（启用CA时）

1. **创建签章请求**：
   - 生成唯一签章编号
   - 设置签章状态为「待签」
   - 创建患者信息（姓名、身份证号、性别、年龄）
   - 设置签章位置比例为20%
   - 根据不同签名类型设置签名关键词（如「主管医师签名:张三」）

2. **生成会诊记录PDF**：
   - 组装会诊记录模板数据（患者信息、诊断、会诊目的、受邀医生等）
   - 使用Beetl模板引擎生成HTML
   - 转换为PDF文件（Base64编码）

3. **调用CA厂商签章**：
   - 将PDF和签章人信息发送给CA厂商
   - 获得唯一标识（uniqueId）
   - 保存签章记录到数据库

### 三、普通签名模式（未启用CA时）

1. 使用医生直接绘制的签名图片
2. 在PDF模板上标注各医生的签名位置和时间
3. 直接保存签章记录，状态直接标记为「已接受」

### 四、签名完成回调

签名完成后（completedSign）：
- **CA模式**：从CA厂商获取签名后的PDF和医生签章图片，更新签章记录
- **普通模式**：直接更新会诊状态
- **申请签名完成** → 会诊状态变为「已申请」，审核状态变为「待审核」，发送系统消息通知医生
- **审核签名完成** → 审核状态变为「已通过」，创建超时未审核的定时检查任务
- 签名完成后更新受邀医生的签名信息和报告内容

### 五、受邀/意见医生签名信息设置

系统自动检查每位受邀医生的签名状态：
- 已签名的医生：显示签名图片和签名时间
- 未签名的医生：如果当前操作医生就是该医生，显示其姓名
- 意见签名已签的医生：同时显示其会诊意见内容

### 六、防重复签名

同一医生对同一会诊的同一类型签名，如果已存在未完成的签名记录（状态≤已接受），系统会自动撤销旧签名记录，避免产生多条重复记录。

## 超时机制

| 超时规则 | 时间阈值 | 说明 |
|---------|---------|------|
| 审核超时 | 距离会诊开始前30分钟 | 会诊开始前30分钟仍未审核，触发超时处理 |
| 受邀签名超时 | 会诊创建后12小时 | 受邀医生超过12小时未签名确认 |

## 状态流转

```
签名状态：待签(Pending) → 已接受(Accept) / 已拒绝(Reject)
会诊流程：申请签名完成 → 待审核 → 审核签名完成 → 已通过 → 会诊进行 → 意见/报告签名
```

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 非主治医师申请签名 | 后端拦截「非主治医生无法签字」 |
| 非审核医生审核签名 | 后端拦截「非审核医生无法签字」 |
| 非受邀医生签名 | 后端拦截「非受邀医生无法签字」 |
| 非意见提交医生签名 | 后端拦截「非医生意见提交无法签字」 |
| 非主治医师报告签名 | 后端拦截「非主治医生无法签字」 |
| 未知签名类型 | 后端拦截「未知的会诊签名类型」 |
| 创建签章记录失败 | 后端拦截「创建签署记录失败, 请联系管理员」 |
| 未找到签名记录 | 后端拦截「未找到会诊签名记录记录」 |

## 关联模块

- **CA签章服务** (ykj-service-ca)：CA账户管理、签章创建、签章结果查询
- **会诊服务** (ykj-service-consultation)：会诊申请、审核、流程管理
- **问诊预约服务** (ykj-service-inquiry)：获取问诊单关联信息
- **医生服务** (ykj-service-hospital)：医生基础信息
- **患者服务** (ykj-service-hospital)：患者信息
- **IM服务** (ykj-service-im)：群消息和系统通知
- **订阅消息** (ykj-service-subscribe)：消息模板推送
- **定时任务** (scheduler)：超时检查任务

## 关键设计

- **重复签名保护**：多次点击生成多条签名数据，系统自动取消所有旧的待签记录
- **签名比例**：CA签章位置为页面的20%缩放比例
- **CA/普通双模式**：根据系统CA开关自动切换，无需额外配置
- **PDF模板**：使用Beetl模板引擎生成会诊记录HTML→PDF
