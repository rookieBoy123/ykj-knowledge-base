# 患者实名认证与就诊人绑定流程

> 文档版本 2026-07-02 | 面向读者：产品经理、运营、客服

## 概述

描述患者（就诊人）从创建、实名认证、与会员绑定关系到医保信息同步的完整后端逻辑。这是所有医疗服务（问诊、处方、会诊）的前置基础能力。

## 触发条件

- **患者端**：用户首次添加就诊人、编辑就诊人信息、解绑就诊人
- **管理后台**：查看患者列表、患者详情
- **问诊/处方流程中**：系统自动校验患者身份

## 步骤详解

### 第一步：新增就诊人

1. **证件校验**：根据证件类型分别校验格式
   - 身份证：调用身份证校验算法验证合法性
   - 护照：正则匹配护照号格式
   - 港澳通行证：正则匹配格式
   - 台胞证：正则匹配格式
   - 校验不通过直接提示"请输入正确的证件号码"

2. **非身份证特殊要求**：非大陆居民身份证（护照/港澳/台胞证），必须手动填写性别和生日

3. **分布式锁防重复**：使用"姓名+身份证"组合作为锁键，防止并发重复提交。若获取锁超时提示"请求频率过高，请稍后再试"

4. **患者去重逻辑**：先按"姓名+证件类型+加密身份证"查询是否已存在：
   - **已存在但未实名** → 若开启了身份校验开关，进行实名认证，补全信息后更新
   - **已存在且已实名** → 直接跳到绑定关系步骤
   - **不存在** → 新增患者记录

5. **实名认证**：若系统开启了身份校验开关，调用第三方身份认证服务验证姓名和证件号是否一致。认证失败则提示具体原因（如"身份信息不匹配"）

6. **自动信息填充**：身份证类型则自动从身份证号解析性别和出生日期；非身份证类型用用户手动填写的信息

7. **拼音首字母**：自动生成姓名拼音首字母，用于后台中文难搜时拼音搜索

### 第二步：会员绑定关系

1. **本人关系限制**：一个会员最多只能绑定一个"关系为本人"的就诊人。若已存在，提示"关系为本人的就诊人只能添加一位，若要继续添加请先修改原本人关系"

2. **患者唯一绑定**：一个患者最多只能绑定在一个会员下。若被另一个会员绑定，提示被绑定会员的手机号（脱敏显示）

3. **关系码处理**：未指定关系时默认为"其他"

### 第三步：地址同步

若新增就诊人时填写了省/市/区和详细地址，系统自动将此地址同步保存为会员的收货地址（便于购药配送）。

### 第四步：患者信息编辑

编辑就诊人信息时的特殊处理：

1. **关系变更**：若编辑了关系码（如从"本人"改为"子女"），需要重新校验本人关系的唯一性
2. **身份信息去重合并**：若编辑后产生了与其他已有患者相同的身份信息（同姓名+同身份证），系统自动将旧的会员-患者绑定关系替换为指向已有患者（数据去重合并，避免重复患者记录）
3. **实名认证**：编辑时若患者未实名，系统自动触发身份认证

### 第五步：患者解绑

1. 发送解绑验证码短信到绑定会员的手机
2. 验证码校验通过后执行解绑（删除会员-患者关联记录，不删除患者数据本身）
3. 患者数据保留在系统中，其他会员仍可绑定

### 第六步：医保信息同步

查询患者慢病信息时：

1. 先校验患者已实名认证（"患者未实名认证"）
2. 到医保系统查询人员基本信息（获取人员编号和险种）
3. 同步险种信息到本地（职工医保/城乡居民/新农合）
4. 获取慢病病种列表，与本地比对后增量更新
5. 首次同步时自动将Ⅰ类病种设为默认病种

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 证件号码格式错误 | 提示"请输入正确的证件号码" |
| 非身份证未填性别生日 | 提示"非大陆居民身份证请正确填写性别和生日" |
| 并发重复提交 | 提示"请求频率过高，请稍后再试" |
| 身份信息认证失败 | 提示认证服务返回的具体原因 |
| 本人关系重复 | 提示"关系为本人的就诊人只能添加一位" |
| 患者已被其他会员绑定 | 提示被绑定会员的脱敏手机号 |
| 患者不存在或已解绑 | 提示"患者不存在或者已经被解绑" |
| 患者未实名认证 | 提示"患者未实名认证" |
| 身份校验开关未开启 | 跳过实名认证，直接保存 |

## 关联功能

- 问诊流程（选择就诊人发起问诊）
- 处方开具（处方关联就诊人）
- 会员管理（会员-就诊人绑定关系）
- 医保支付（患者医保资格校验）
- 收货地址管理（就诊人地址→收货地址同步）
