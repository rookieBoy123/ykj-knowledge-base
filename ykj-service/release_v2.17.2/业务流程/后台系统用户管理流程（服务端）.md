# 后台系统用户管理流程（服务端）

> 文档版本 v1.0 | 面向产品/技术人员

## 概述

管理后台的系统用户管理模块负责运维管理人员账号的全生命周期管理，包括创建用户、编辑信息、角色分配、密码管理、启停控制、手机号管理和当前用户信息查询。所有操作均受权限控制并自动记录操作审计日志。

## 触发条件

- **前置条件**：调用方持有有效的管理后台会话，且拥有对应的操作权限
- **入口**：管理后台前端通过 `/mgmt/system/users/*` 接口调用

## 功能详解

### 一、新增用户（POST `/mgmt/system/users`）

创建新的管理后台登录用户。

**权限要求**：`sys:users:write`

**必填参数**：
- userName：用户名，3-32位字母或数字（正则：`^[\w\d]{3,32}$`）
- mobile：11位手机号码（正则：`^\d{11}$`）

**可选参数**：
- realName：姓名
- email：邮箱
- roleCodes：授权的角色编码列表
- address：地址
- status：用户状态（1=启用, 2=禁用）

**审计日志**：`新增用户，用户名【xxx】`

### 二、编辑用户（PUT `/mgmt/system/users/{id}`）

修改用户的姓名、邮箱、角色、地址、状态等信息。

**权限要求**：`sys:users:write`

**注意**：编辑用户不包含用户名的修改，用户名修改有独立接口。

**审计日志**：`编辑用户，用户ID【xxx】`

### 三、修改用户名（PUT `/mgmt/system/users/{userId}/username`）

单独修改用户登录用户名。

**权限要求**：`sys:users:write`

**审计日志**：`修改用户名，用户ID【xxx】`

### 四、用户分页查询（GET `/mgmt/system/users`）

支持按条件分页查询用户列表。

**权限要求**：`sys:users:read`

**查询参数**：支持 SysUserPageParam 中的筛选条件（如用户名、状态等）+ 分页参数

### 五、查询用户角色（GET `/mgmt/system/users/{id}/_user-roles`）

查询指定用户被授予的角色列表。

### 六、检查用户是否存在（POST `/mgmt/system/users/_check-exists`）

新增用户前，检查输入的用户名是否已被注册。

**权限要求**：`sys:users:read`

### 七、重置用户密码（POST `/mgmt/system/users/{id}/_reset-password`）

管理员为指定用户重置密码（密码由系统自动生成）。

**权限要求**：`sys:users:write`

**审计日志**：`重置用户密码，用户ID【xxx】`

### 八、当前用户修改密码（POST `/mgmt/system/users/current/_up-password`）

当前登录用户自行修改密码，需要提供旧密码进行验证。

**参数**：
- oldPassword：旧密码（必填）
- newPassword：新密码（必填，由前端确保两次输入一致）

### 九、获取当前用户信息（GET `/mgmt/system/users/current`）

返回当前登录用户的完整信息，包括：
- 基本信息（用户名、姓名、手机号、邮箱等）
- **用户角色列表**：过滤只返回当前登录端（管理后台/小程序）对应的角色
- **当前角色**：当前会话使用的角色编码和名称
- **权限列表**：当前用户的所有权限标识（如 `sys:users:read`、`sys:users:write`）

**未登录处理**：如果查询不到用户 → 抛出未授权异常

### 十、更换手机号码（PUT `/mgmt/system/users/{userId}/mobile`）

修改指定用户的手机号码。

**权限要求**：`sys:users:write`

**审计日志**：`更换用户手机号码，用户ID【xxx】,手机号码【xxx】`

### 十一、查看手机号码（GET `/mgmt/system/users/{userId}/mobile`）

查看指定用户的手机号码（脱敏场景下单独控制权限）。

**权限要求**：`sys:users:read`

**审计日志**：`查看用户手机号码, 用户ID【xxx】`

### 十二、启用用户（POST `/mgmt/system/users/{id}/_enable`）

将禁用状态的用户恢复为启用状态。

**权限要求**：`sys:users:write`

**审计日志**：`启用用户，用户ID【xxx】`

### 十三、禁用用户（POST `/mgmt/system/users/{id}/_disable`）

将用户设置为禁用状态，禁用后该用户无法登录管理后台。

**权限要求**：`sys:users:write`

**审计日志**：`禁用用户，用户ID【xxx】`

## 权限控制

所有接口均通过 `@PreAuthorize` 注解控制权限：

| 接口 | 所需权限 |
|------|---------|
| 新增用户 | sys:users:write |
| 编辑用户 | sys:users:write |
| 修改用户名 | sys:users:write |
| 分页查询 | sys:users:read |
| 检查用户名 | sys:users:read |
| 重置密码 | sys:users:write |
| 修改密码 | 无需特殊权限（当前用户操作） |
| 获取当前用户 | 无需特殊权限（已登录即可） |
| 更换手机号 | sys:users:write |
| 查看手机号 | sys:users:read |
| 启用/禁用 | sys:users:write |

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 用户名不符合格式 | 参数校验拦截："用户名必须是由字母或者数字组成，且长度为3到32个字符" |
| 手机号格式错误 | 参数校验拦截："请输入有效的手机号码" |
| 手机号为空 | 参数校验拦截："手机号码不能为空" |
| 旧密码错误 | 修改密码时后端验证失败 |
| 新密码为空 | 参数校验拦截："新密码不能为空" |
| 旧密码为空 | 参数校验拦截："旧密码不能为空" |
| 未登录访问current接口 | 抛出未授权异常："未登入" |
| 无权限操作 | Spring Security拦截，返回403 |

## 审计日志

所有关键操作通过 `@Audit` 注解自动记录，日志内容包括：
- 操作类型（新增/编辑/查询/更新）
- 操作模块名称
- 操作对象标识（用户ID、用户名、手机号等）
- 操作人信息

## 关联功能

- **角色管理**：用户授权角色，角色决定权限
- **权限管理**：权限标识控制具体接口访问
- **登录日志**：记录管理后台登录行为
- **双因素认证**：可选TFA短信认证
