# 医生排班管理流程

> 文档版本 v1.0 | 面向系统分析

## 概述

医生排班管理是支撑在线问诊预约功能的核心基础服务。它管理医生的出诊时间安排，包含班次创建、号段管理、排班发布、停诊处理、号源扣减/退还、定时同步等能力。

**对应服务模块**：ykj-service-hospital

## 核心概念

### 班次（Schedule）
- 医生在某个日期的一个出诊时间段
- 包含班次名称、开始时间、结束时间
- 每个班次有唯一的班次编号（scheduleNo）

### 号段/号源（ScheduleItem）
- 班次内细分的可预约时间段
- 包含号源总数（num）、已用数（alreadyUsed）、开始时间、结束时间
- 每个号段有唯一的号段编号（scheduleItemNo）

### 排班关键属性

| 属性 | 说明 |
|------|------|
| isRelease | 是否已发布（未发布的排班对患者不可见） |
| isStopReception | 是否停诊（停诊后不可预约） |
| isEnabled | 是否启用（号段级别控制，未启用的号段不可预约） |

## 排班保存流程

### 第一步：参数校验
1. 校验医生ID不为空（"请指定要保存排班的医生"）
2. 校验班次不为空（"请正确填写班次"）

### 第二步：冲突校验
1. **班次名称冲突**：同一天的班次名称不能重复（"班次名称存在冲突"）
2. **班次时间冲突**：同一天的班次时间段不能重叠（"班次时间存在冲突"）
3. **号段时间范围**：号段的时间必须在所属班次的时间范围内（"号段时间xx-xx, 不在xx班次时间内"）
4. **号段时间冲突**：同一班次内的号段时间不能重叠（"号段时间存在冲突"）
5. **当天排班校验**（仅针对当天）：
   - 新增：结束时间已过的号段不允许新增（"小于当前时间的排班不允许新增"）
   - 编辑：跨当日时间段的号段不允许修改号源数和启用状态（"小于当前时间的排班不允许操作"/"小于当前时间的排班号源数不允许操作"）

### 第三步：数据保存
1. 为每个班次生成唯一编号（UUID，校验数据库不重复）
2. 为每个号段生成唯一编号
3. 先保存号段数据
4. 再保存班次数据
5. 删除旧班次和号段数据（先删号段，再删班次）

### 第四步：状态初始
- isRelease=false（未发布）
- isStopReception=false（未停诊）
- isEnabled：根据号段中是否有启用项计算
- num（号源总数）：已启用号段的号源数之和
- alreadyUsed=0（已用数为0）

## 排班发布

排班保存后默认未发布，需要通过"排班发布"操作让排班对患者可见：

1. 查询所有未发布且启用的排班
2. 如果无待发布排班，提示"当前无待发布排班"
3. 批量更新isRelease=true

## 停诊处理

### 停诊参数
- 医生ID、日期：必填（"请指定停诊的医生和日期"）
- 班次编号：非全天停诊时必填（"请指定停诊的班次"）
- 停诊原因：如果该班次已有患者挂号则必填（"停诊的班次已有患者挂号, 请填写停诊原因"），最长100字

### 停诊操作
1. 查找指定医生、日期、班次的排班记录
2. 查找所有受影响的号段
3. 校验并通过号段+停诊原因
4. 设置isStopReception=true
5. 发布"医生停诊"事件通知，携带受影响号段列表

## 号源管理

### 使用号源（预约成功）
1. 校验号段存在且启用
2. 校验班次：已启用 + 已发布 + 未停诊
3. 校验预约时间：号段结束时间+30分钟前才可预约（"预约不能超过号段结束时间前30分钟"）
4. 扣减前二次校验号源余量（"班次信息发生变更, 请刷新后重试"）
5. 号段已用数+1，班次已用数+1

### 退还号源（取消预约）
1. 取消时不校验停诊状态（停诊的预约也可以取消）
2. 号段已用数-1，班次已用数-1，班次已退数+1

## 定时同步排班

定时任务自动同步医生排班：
1. 查询需要同步排班的医生列表
2. 查询排班模板
3. 如果没有模板，取消同步（"系统暂未配置排班模板, 同步任务取消"）
4. 计算目标日期：当前日期+15天
5. 对每个医生：
   - 检查目标日期是否已有排班（有则跳过）
   - 按模板创建排班数据并保存
6. 异步处理，逐个医生执行

## 排班查询

### 管理后台查询
按医生姓名和科室查询，返回指定日期范围内每天的排班情况（含空排班日期）。

### 患者端查询
只返回有效排班（已发布 + 未停诊 + 启用），过滤不可预约的数据。

### 号源余量校验
`existenceNum()`方法检查号源是否还有余量：
- 条件：记录存在 + 号段启用 + 总数-已用数 ≥ 1

## 异常处理

| 情况 | 后端提示 |
|------|---------|
| 未指定医生 | 请指定要保存排班的医生 |
| 班次为空 | 请正确填写班次 |
| 班次名称重复 | 班次名称存在冲突 |
| 班次时间重叠 | 班次时间存在冲突 |
| 号段不在班次时间范围 | 号段时间xx-xx, 不在xx班次时间内 |
| 号段时间重叠 | 号段时间存在冲突 |
| 班次不存在 | 班次不存在 |
| 排班不存在 | 排班不存在 |
| 预约超时 | 预约不能超过号段结束时间前30分钟 |
| 号源不足 | 班次信息发生变更, 请刷新后重试 |
| 无待发布排班 | 当前无待发布排班 |
| 已挂号未填写停诊原因 | 停诊的班次已有患者挂号, 请填写停诊原因 |
| 停诊原因超长 | 停诊原因不能超过100字 |
| 停诊失败 | 停诊失败, 请联系管理员 |
| 今日过期排班操作 | 小于当前时间的排班不允许操作/新增 |
| 今日过期号源修改 | 小于当前时间的排班号源数不允许操作 |

## 关联功能

- 问诊预约（通过scheduleItemNo扣减号源）
- 排班模板管理（定时同步的数据来源）
- 停诊事件通知（异步通知受影响患者）
- 医生管理（同步排班的医生列表来源）
