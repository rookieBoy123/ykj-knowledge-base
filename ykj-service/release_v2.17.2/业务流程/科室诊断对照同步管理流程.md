# 科室诊断对照同步管理流程

> 文档版本：2026-07-02 | 面向读者：产品经理、系统管理员

## 概述

管理标准科室与诊断编码的对照关系。将医院科室与标准诊断（ICD-10等）建立映射，实现科室诊断的自动同步。当新增科室诊断对照时，会自动向上级父科室同步该诊断数据。涉及模块 `ykj-service-std`，核心类是 `StdDeptDiagnoseServiceImpl`（425行）。

## 核心环节

### 1. 新增科室诊断对照
- 管理员为某个科室配置关联的诊断编码
- 系统自动查找字典中的标准科室信息，如果找不到对应科室会提示"找不到对应的标准科室信息！"
- 自动计算父级科室编码（如科室编码"01.02.03"→父级为"01.02"，顶级科室父级为"0"）
- 新诊断如果与已有诊断不重复则新增，重复的跳过
- 新增成功后，自动同步到父级科室（让父级科室也包含该诊断）

### 2. 编辑科室诊断对照
- 更新已有科室的诊断列表
- 系统对比新旧诊断列表：
  - 新增的诊断：按新增逻辑处理
  - 删除的诊断：从科室移除，同时从父级科室同步删除
  - 如果编辑时未传诊断列表，则清空该科室所有诊断

### 3. 父子科室同步机制
- **向上同步（新增）**：子科室新增诊断→父科室自动获得相同诊断（`syncInsertParentData`）
- **向上同步（删除）**：子科室删除诊断→父科室自动移除该诊断（`syncDelParentData`）
- 这样确保医生在上级科室也能看到下级科室关联的诊断

### 4. 批量导入诊断
- 支持通过 Excel 批量导入科室诊断对照数据
- 使用 `ImportForkJoinPoolListener` 进行大数据量分片处理
- 数据会缓存到 Redis（1小时过期），key 前缀由配置 `spring.cache.redis.key-prefix` 指定

### 5. ICD-10 诊断缓存
- ICD-10 诊断数据缓存在 Redis 中，key 为 `dept_diag_ICD10`
- 科室字典数据缓存，key 为 `std_dept`
- 缓存过期时间：3600秒（1小时）

## 异常处理

| 情况 | 系统处理方式 |
|------|------------|
| 科室编码在字典中找不到 | 后端抛出异常："找不到对应的标准科室信息！"（后端拦截） |
| Excel 导入数据量过大 | 使用分片处理（ForkJoinPool），分批写入 |
| 重复的诊断编码 | 新增时自动跳过已存在的诊断（后端去重） |
| 父级科室同步 | 新增诊断自动向上同步；删除诊断自动向上移除（事务保证一致性） |

## 关联功能

- `ykj-service-std` 模块：标准数据管理（标准科室、标准诊断、标准药品信息）
- `ykj-service-hospital` 模块：医院科室信息字典
- `ykj-service-inquiry` 模块：问诊时调用诊断服务获取诊断列表
- 后台管理标准科室诊断列表页：`mgmt/hosp/stdManage/stdDeptDiagList/index.vue`
