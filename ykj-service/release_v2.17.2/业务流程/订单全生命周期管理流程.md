# 订单全生命周期管理流程

> 文档版本 2026-07-02 | 面向读者：产品、运营、开发

## 概述

订单系统是全平台交易的核心，覆盖从创建订单到支付、发货、签收、退款、完成的完整生命周期。支持多种订单来源（处方/问诊/会诊/预约/服务包），多种支付方式（微信/支付宝/医保），以及订金+尾款、自提/快递等多种场景。

## 订单生命周期

```
创建订单 → 待支付 → 已支付（/已发货）→ 待签收 → 已完成
                ↓         ↓
              已取消    退款中 → 已退款
              （超时过期）
```

不同订单类型的流转路径略有不同：

| 订单来源 | 支付后状态 |
|---------|-----------|
| 处方订单（PRESCRIPTION）| 已支付（PAYED）→ 待发货 → 已发货待签收 → 已完成 |
| 问诊/会诊/预约（INQUIRY/RESERVATION）| 已完成（COMPLETED） |
| 服务包（SERVICE_PACKAGE）| 待发货（PENDING_SHIPMENT）→ 已完成 |

## 一、创建订单

**触发**：用户提交订单时调用 createOrder

**核心逻辑**：
1. 生成订单号（O + 年月日 + 7位随机数）
2. 判断是否0元订单：
   - 0元 + 处方来源 → 状态为"已支付"
   - 0元 + 需发货来源 → 状态为"待发货"
   - 0元 + 其他 → 状态为"已完成"
3. 非0元 → 状态为"待支付"
4. 设置订单过期时间
5. 插入订单主记录、商品明细、收货地址、收费项
6. 发布"订单创建完成"事件，通知关联系统

**过期时间计算**：
- 优先使用业务指定的过期时间
- 否则默认使用24小时后的时间

**测试环境特殊处理**：
- 开启测试支付时，订单金额减0.01元（方便测试小额支付）

## 二、取消订单

**触发**：用户取消、超时自动取消、后台取消

**可取消的状态**：待支付、已支付、已完成（仅问诊和预约类型允许取消已完成）

**取消流程**：
1. 校验订单状态是否允许取消
2. 处方订单通知第三方：发布"第三方订单取消事件"
3. 需要退款时：
   - 状态改为"退款中"
   - 自动生成退款申请单
   - 自动退款：系统自动发起退款
4. 不需要退款时：状态改为"已取消"或"已过期"（超时取消）
5. 发布"订单已取消"事件

## 三、订单支付

### 准备支付（advance）

1. 校验订单是否已支付 → 已支付报错"当前订单已支付！"
2. 校验订单状态是否为"待支付" → 非待支付报错"订单已失效！"
3. **历史渠道查单**：查询是否有"已支付但回调未到达"的情况
   - 查到已支付 → 自动补单 + 拦截支付
4. 构建支付渠道列表：
   - 微信支付（默认选中）
   - 微信扫码支付
   - 支付宝
   - 医保支付（需满足条件：功能开关+处方来源+业务校验）
5. 计算支付金额（订单总额 - 已付金额）

### 发起支付（prepay）

1. **加分布式锁**防止重复支付
2. 校验支付渠道
3. **清理旧预支付单**：关闭旧的第三方支付单、清除缓存
4. 调用对应支付渠道的预支付接口（策略模式：微信/支付宝/医保）
5. 返回支付凭证给前端唤醒支付

## 四、支付完成

**触发**：支付回调通知或主动查单

**completePayment 流程**：
1. **加分布式锁**
2. **幂等判断**：同一笔支付（transactionId/payOrderNo）已处理过则跳过
3. 订单非"待支付"状态 → 自动退款处理（重复支付场景）
4. 保存支付流水
5. 更新订单状态：
   - 处方订单 → 已支付
   - 服务包 → 待发货
   - 其他 → 已完成
6. 生成取货码（处方订单）
7. 发布"订单已支付"事件

## 五、发货与签收

### 确认发货

1. 校验订单状态为"已支付"
2. 校验收货地址存在
3. 订阅物流轨迹
4. 更新快递单号、快递公司
5. 状态改为"已发货待签收"
6. 发布"订单已发货"事件

### 确认签收

1. 校验操作人是订单所属患者
2. 校验订单未重复签收
3. 订金订单：支付尾款、新增尾款流水
4. 状态改为"已完成"
5. 发布"订单已签收"事件

### 自动签收（定时任务）

发货超过15天的订单自动签收，每次处理最多100条。

## 六、退款

### 申请退款

1. 校验订单已付款（非"待支付"）
2. 校验退款金额不大于支付金额
3. 发布退款申请事件

### 医保退款（insuranceRefund）

1. 获取医保授权码
2. 判断是否已进行医保退费
3. 未退费：调医保退款接口 → 退费成功后标记已退费 → 微信医保退款
4. 已退费：直接微信医保退款
5. 退款成功：新增支出流水

### 退款完成（completeRefund）

1. 状态改为"已退款"
2. 记录退款时间
3. 发布"订单退款"事件

## 七、取货方式切换

支持在待付款/待发货状态下切换自提和快递：

- **切到自提**：生成取货码、清除物流信息
- **切到快递**：需填写完整收货信息、清除取货码
- 不能切换到当前相同的取货方式

## 八、取货码取货

运营人员输入取货码完成线下取货：
1. 校验取货码存在 → "取货码有误"
2. 校验订单未取货 → "该订单已取货"
3. 更新状态为已完成、记录取货人信息
4. 发送收货通知

## 异常处理

| 情况 | 系统处理方式 |
|------|------------|
| 订单生成失败 | "订单生成失败，请联系管理员" |
| 当前状态不可取消 | "订单取消失败，当前状态不可取消" |
| 订单已支付（重复支付） | "当前订单已支付！"（错误码40001） |
| 订单已失效 | "订单已失效！"（错误码40002） |
| 支付处理中 | "支付处理中，请稍后重试"（错误码40003） |
| 重复支付已处理（幂等） | 日志记录，静默跳过 |
| 重复支付未处理 | 自动退款（原路退回） |
| 退款金额大于支付金额 | "申请退款金额不能大于实际支付金额" |
| 未找到支付信息 | "未找到订单支付信息，无法退款，请联系管理员" |
| 订单未付款 | "订单未成功付款，不支持申请退还金额" |
| 非患者操作 | "非法操作" |
| 重复收货 | "请勿重复收货" |
| 无收货地址 | "该订单不存在收货地址信息" |
| 取货码有误 | "取货码有误" |
| 订单已取货 | "该订单已取货" |
| 切换取货方式状态不符 | "当前订单状态不允许修改取货方式" |
| 收货地址已变更 | "收货地址信息已变更，请刷新后重试" |
| 会员已绑定 | "订单已绑定会员" |
| 无权限查看订单 | "您暂无权限查看订单" |
