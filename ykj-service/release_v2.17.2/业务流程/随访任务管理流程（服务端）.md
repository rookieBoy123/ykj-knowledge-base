# 随访任务管理流程（服务端）

> 文档版本 v1.0 | 面向读者：产品经理、开发人员

## 概述

随访任务服务端模块负责管理随访计划的执行实例——每个随访任务对应一个患者在一个随访计划节点上的一次执行。服务端提供管理后台分页查询、详情查看、内容详情（患教/提醒/问卷）查询接口，以及小程序端的任务列表查询、任务小结填写、内容状态编辑和问卷答题接口。

## 核心数据模型

### 随访任务（t_followup_task）
| 字段 | 说明 |
|------|------|
| planId | 所属随访计划ID |
| interval | 间隔时间数值 |
| intervalUnit | 间隔时间单位（1=天，7=周，30=月，365=年） |
| sendingTime | 随访发送时间 |
| inquiryTime | 关联的问诊时间 |
| status | 任务状态（见下方状态说明） |
| summary | 任务小结（完成后填写） |

### 任务状态
| 状态码 | 状态名 | 说明 |
|--------|--------|------|
| 0 | 待随访 | 任务已创建，尚未到发送时间或尚未开始 |
| 1 | 已随访 | 随访内容已发送给患者，患者正在进行中 |
| 2 | 已完成 | 所有待办内容均已完成 |
| 3 | 已取消 | 任务被取消，不再执行 |

## 管理后台接口

### 1. 分页查询 (`GET /mgmt/followup/tasks`)

支持按以下条件筛选：患者ID（必填！）、状态、创建时间范围、随访日期范围、患者姓名、手机号、医生ID。

> ⚠️ 注意：管理后台接口的 `FollowupTaskReq` 中 `patientId` 带有 `@NotNull` 校验，但实际上前端可以通过其他条件查询，不走此参数校验——实际分页查询由 DefaultPageEndpoint 框架自动处理。

### 2. 详情查询 (`GET /mgmt/followup/tasks/{id}`)

返回任务完整详情：
- 患者信息（姓名、年龄、性别、电话、头像）：跨模块查询会员服务和患者服务
- 医生信息（姓名、职称、科室、头像）：跨模块查询医生服务
- 任务内容列表：每条内容附加禁用状态判断

**禁用状态判断逻辑**：仅在任务发送时间 > 当前时间（即任务尚未发送）时，检查关联的患教/问卷内容是否被禁用。已发送的任务不判断禁用状态（因为内容已经发送给患者）。

### 3. 问卷详情 (`POST /mgmt/followup/tasks/_item-questionnaire-details`)

查询具体的问卷题目、选项和患者答案。支持图片上传题型的答案URL转换（通过OSS服务获取图片访问链接）。

### 4. 患教详情 (`POST /mgmt/followup/tasks/_item-course-details`)

查询患教资料的标题和正文内容。

### 5. 提醒详情 (`POST /mgmt/followup/tasks/_item-remind-details`)

查询提醒内容的基本信息。

## 小程序端接口

### 1. 列表查询 (`GET /api/followup/tasks`)

- 患者端只能查询「待随访」和「已完成」两种状态的任务
- 如果传入其他状态码（如已取消），服务端直接抛出异常："非法状态查询"

### 2. 任务小结 (`POST /api/followup/tasks/_summary`)

医生在随访完成后填写小结，小结内容必填（`@NotEmpty`）。

### 3. 内容状态编辑 (`POST /api/followup/tasks/_edit-item-status`)

用于标记提醒/患教内容为"已读"。更新对应任务内容项的完成状态，并自动同步更新父级任务状态：

**状态自动流转逻辑**：
1. 单条内容标记为完成
2. 系统检查该任务下所有内容是否全部完成
3. 如果全部完成 → 任务状态从「已随访」变为「已完成」
4. 进一步检查该计划下所有任务是否全部完成
5. 如果全部完成 → 计划状态也变为「已完成」

### 4. 问卷答题 (`POST /api/followup/tasks/_answer-sheet`)

患者提交问卷答案，支持多题目批量提交。服务端执行：
- 校验该任务内容项是否存在
- 校验是否已答题（已答题拒绝重复提交，提示"已答题,请勿重复作答"）
- 保存答案JSON
- 标记该内容为已完成
- 触发状态流转检查

## 跨模块依赖

FollowupTaskAssembler 依赖以下外部服务模块：

| 依赖模块 | 用途 |
|---------|------|
| 会员服务 (member) | 获取患者手机号和头像 |
| 患者服务 (hospital) | 获取患者基本信息（姓名、生日→计算年龄、性别） |
| 医生服务 (hospital) | 获取创建人医生信息 |
| OSS服务 | 问卷图片答案URL转换 |

## 权限控制

| 接口 | 权限标识 |
|------|---------|
| 管理后台查询 | `fup:tasks:read` |
| 小程序端 | 无需特殊权限（通过登录态关联） |
