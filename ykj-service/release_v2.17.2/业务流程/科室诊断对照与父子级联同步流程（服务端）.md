# 科室诊断对照与父子级联同步流程（服务端）

> 文档版本 2026-07-04 | 面向开发/技术支持

## 概述
科室诊断对照功能将"标准科室"与"ICD-10 诊断"建立映射关系。当管理后台为一个科室（如"心血管内科"）关联诊断（如"高血压"、"冠心病"）时，服务端会自动将诊断向上同步到父级科室（如"内科"）——确保在上级科室维度也能查到完整的诊断范围。删除诊断时同样会级联清理父级。

> **与现有文档的关系**：本文档聚焦于"父子级联同步"这一核心机制。科室诊断对照的基础 CRUD 操作可参考已有文档「科室诊断对照与同步管理流程.md」。

## 核心机制：父子级联同步

### 为什么需要级联同步？
标准科室是树形结构（如"内科 → 心血管内科 → 冠心病专科"），一个诊断可能关联到任意层级的科室。如果只在"冠心病专科"关联了诊断，但在"内科"层级也需要能查到所有子科室的诊断汇总，就需要级联同步机制。

### 新增诊断时的向上同步
1. 管理员为科室（如"心血管内科"）新增诊断（如"高血压"）
2. 系统保存科室-诊断对照记录
3. 系统查找该科室的父级科室编码（通过科室编码的点号分隔规则，如 `A.B.C` → 父级为 `A.B`）
4. 如果父级科室存在，系统检查父级是否已有该诊断：
   - 已有 → 跳过
   - 没有 → 在父级科室也插入一条相同的诊断对照
5. 继续向上递归，直到根节点（父级编码为 "0"）停止

### 删除诊断时的向上清理
1. 管理员从科室（如"心血管内科"）删除诊断（如"高血压"）
2. 系统删除该科室的诊断对照记录
3. 系统检查父级科室下的所有子科室，是否还有其他子科室保留了该诊断：
   - 有其他子科室仍有 → 父级保留该诊断（因为仍有子科室在用）
   - 所有子科室都删了 → 父级也删除该诊断
4. 继续向上递归清理

### 更新诊断时
更新操作实际上是"删除 + 新增"的组合：
1. 对比新旧诊断列表，找出被删除的 → 执行删除逻辑（含向上清理）
2. 对比新旧诊断列表，找出新增的 → 执行新增逻辑（含向上同步）

## 科室编码层级规则

科室编码使用点号（`.`）分隔层级，如：
- `01`：一级科室（无父级）
- `01.02`：二级科室，父级编码为 `01`
- `01.02.03`：三级科室，父级编码为 `01.02`

父级编码计算规则：找到最后一个点号，取其左侧部分；如果没有点号，父级编码为 `0`（根节点）。

## 数据来源

本功能依赖两个外部数据源：
1. **标准科室字典**（`DEPT_DICT_CONTENT_CODE`）：来自系统数据字典，由管理员维护
2. **ICD-10 诊断信息**：来自问诊模块的诊断服务（`ApiDiagnosisService`），包含国际疾病分类编码

## Excel 导入机制

支持批量 Excel 导入科室诊断对照，流程如下：
1. 上传 Excel 文件
2. 后台异步读取 Excel（使用线程池）
3. 从 Redis 缓存中获取标准科室映射（科室名称→编码）和 ICD-10 诊断映射（诊断名→编码）
4. 缓存未命中时从数据库加载并写入 Redis（1小时过期）
5. 只导入"科室名和诊断名都能匹配到"的行，匹配不到的自动跳过
6. 导入完成后自动清理 Redis 缓存

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 新增时的科室编码在字典中不存在 | 抛出异常"找不到对应的标准科室信息！" |
| 查询详情时科室编码不存在 | 抛出异常"找不到对应的字典信息" |
| 导入时 ICD-10 诊断信息为空 | 抛出异常"不存在ICD_10诊断信息！" |
| 导入时标准科室信息为空 | 抛出异常"不存在标准科室信息！" |
| 初始化字典错误（科室字典分类不存在） | 日志警告"初始化字典错误"，返回空列表 |
| Excel 中某行的科室名或诊断名匹配不到 | 该行静默跳过，仅导入能匹配到的行 |
| 没有任何能匹配的行 | 返回导入数量 0，不报错 |

## 关联功能
- **数据字典**：科室列表来自字典的标准科室分类
- **问诊模块**：ICD-10 诊断信息来自问诊诊断服务
- **管理后台 - 科室诊断对照**：管理员在此页面操作科室和诊断的对应关系
