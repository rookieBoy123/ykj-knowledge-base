# 处方结算费用明细计算流程

> 文档版本 2026-07-02 | 面向读者：产品、运营

## 概述

当患者对医生开具的处方进行结算支付时，系统自动计算费用明细——包括药品金额、医事服务费、诊金、加工服务费、运费等，并汇总为最终支付总金额。

## 触发条件

- 患者在小程序的处方详情页点击"去结算"
- 开放平台通过接口调用处方预结算

## 步骤详解

### 第一步：处方状态校验

在开始结算前，系统做六重状态校验：

1. **处方存在**：处方编号必须存在，否则提示"处方不存在，请联系管理员"
2. **未重复结算**：若处方已结算完成，提示"处方已结算完成，请勿重复操作"
3. **医生已开方完成**：若医生仍在开方中、审核中或撤销中，提示"医生开方中，请稍后再试"
4. **审核通过**：若医生审核不通过或处方已过期，提示"处方审核不通过或者已过期，请联系医生重新开方"
5. **药师已审方**（可选）：若开启了"先审方后支付"功能，必须药师审核通过后才能支付
6. **未过期**：处方有有效期限，过期后提示"处方已过期，请联系医生重新开方"

### 第二步：确认加工服务

中药处方可能包含加工服务（如代煎、打粉等），患者可以选择是否使用加工服务：

- 医生开方时已选加工服务 → 患者只能接受或取消（不可更换为其他服务）
- 医生未选加工服务 → 患者可以自由选择药房的可用加工服务
- 取消了不允许更改的服务 → 提示"加工服务XXX不支持支付时变更"

加工服务费按药房配置的单价计算。

### 第三步：计算费用明细

系统按固定顺序生成费用清单：

| 费用项 | 计算规则 | 说明 |
|--------|---------|------|
| 药品金额 | 单剂价格 × 总剂数 | 始终排在第一位；若合并显示医事服务费则此项含服务费 |
| 医事服务费 | 医生配置的服务费金额 | 若与药品合并则此项不单独展示 |
| 诊金 | 问诊单中的诊金金额 | 有则展示，无则跳过 |
| 加工费 | 选中的加工服务费 | 括号注明服务名称（如"加工费(代煎)"） |
| 运费 | 根据收货地址和药房运费规则计算 | 仅快递配送时计算，到店取货跳过 |

最终总金额 = 上述五项之和（药品金额保留两位小数、向上取整）。

### 第四步：计算订金（货到付款模式）

若药房支持货到付款（需同时满足：药房启用订金 + 配送方式为快递 + 系统开启了订金功能 + 处方金额达到最低门槛），系统会计算订金金额：

- 订金金额 = 系统配置的固定订金金额
- 若订金大于处方总金额，则订金设为 0
- 尾款 = 处方总金额 - 订金（确认收货时支付）

### 第五步：创建订单

预结算确认后，将费用信息转为订单：

1. 组装订单商品（药品清单、每味药的数量×总剂数）
2. 组装收货地址（用户默认地址或指定地址）
3. 计算运费（根据省/市/区 + 药房运费规则）
4. 扣减药品库存（开放平台订单不扣库存，因为已在预结算阶段扣过）
5. 调用订单创建服务，返回订单编号

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 处方不存在 | 提示"处方不存在，请联系管理员" |
| 已存在订单 | 提示"已存在订单信息" |
| 处方已结算完成 | 提示"处方已结算完成，请勿重复操作" |
| 医生开方中 | 提示"医生开方中，请稍后再试" |
| 处方审核不通过/已过期 | 提示"处方审核不通过或者已过期，请联系医生重新开方" |
| 处方已过期 | 提示"处方已过期，请联系医生重新开方" |
| 药师未审核（先审后付模式） | 提示"药师未审核处方，请稍后再试" |
| 药品总金额为负数 | 自动将总金额置为 0 |
| 患者不可用/不存在 | 提示"创建处方订单时发生异常，患者不可用或者不存在" |
| 加工服务不支持变更 | 提示"加工服务XXX不支持支付时变更" |

## 关联功能

- 药品生命周期管理（库存扣减）
- 运费规则配置（运费计算）
- 订单生命周期管理（订单创建后的支付流程）
- 医保费用上传流程（医保处方需先上传费用明细再支付）
