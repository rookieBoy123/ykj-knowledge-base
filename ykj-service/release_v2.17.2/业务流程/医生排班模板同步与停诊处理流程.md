# 医生排班模板同步与停诊处理流程

> 文档版本 v1.0 | 面向读者：产品、运营、客服

## 概述

描述医生排班数据从模板创建、定时同步到停诊处理的完整后端链路。系统支持管理员手动配置排班模板，由定时任务自动根据模板生成未来15天的排班计划；同时支持医生停诊时的号源锁定与预约取消联动。

## 触发条件

- **排班同步**：定时任务每日自动执行
- **手动配置**：管理员在后台排班模板页创建或修改排班
- **停诊**：管理员在后台医生排班页执行停诊操作

## 步骤详解

### 第一步：排班模板创建

管理员在后台为医生配置排班模板：

- **模板内容**：包含班次时间（如上午/下午/晚上）、每个班次的号段（开始时间/结束时间/号源数）
- **号段编号**：每个号段自动生成全局唯一的UUID编号
- **班次编号**：每个班次自动生成全局唯一的UUID编号

### 第二步：定时同步排班

定时任务每日执行排班同步：

- **同步周期**：自动同步未来第15天的排班
- **跳过策略**：如果医生在目标日期已有排班记录，则跳过不覆盖
- **数据来源**：从排班模板获取所有班次配置，按模板结构生成当天的排班数据
- **状态初始化**：
  - 是否发布：初始为"未发布"（false）
  - 是否停诊：初始为false
  - 是否启用：根据号段是否启用计算
  - 总号源数：所有启用号段的号源数之和

### 第三步：排班发布

管理员手动发布排班：

- **发布条件**：只发布未发布且已启用的排班
- **无待发布排班**：提示"当前无待发布排班"
- **发布后**：小程序端可以查询到该排班

### 第四步：小程序端查询排班

患者在小程序端查看医生排班（apiFindScheduleByDoctorId）：

- **过滤条件**：仅返回同时满足以下条件的排班：
  - 已发布（isRelease = true）
  - 未停诊（isStopReception = false）
  - 已启用（isEnabled = true）
- **返回内容**：包含当天的所有班次和每个班次的号段详情（开始时间、结束时间、总号源、已用号源）

### 第五步：号源使用与释放

患者预约挂号时号源扣减：

- **号源校验**：确认号段存在、有效、有剩余号源（总数-已用数 ≥ 1）
- **预约截止时间**：号段结束时间前30分钟停止预约（提示"预约不能超过号段结束时间前30分钟"）
- **号源扣减**：已用数 +1，同时更新班次的已用数和号段的已用数
- **号源释放**：取消预约时已用数 -1，班次已取消数 +1
- **并发保护**：扣减前再次校验号源可用性

### 第六步：医生停诊处理

管理员执行停诊操作：

- **停诊范围**：
  - 全天停诊：该医生当天全部班次停诊
  - 指定班次停诊：只停诊指定的某个班次
- **停诊原因**：
  - 如果停诊班次已有患者挂号（已用号源 > 0），必须填写停诊原因
  - 停诊原因最长100字
- **状态更新**：所有受影响的班次标记为"已停诊"
- **事件通知**：推送停诊事件（ScheduleStopReceptionEvent），触发下游处理：
  - 号段停诊通知
  - 已预约患者的批量取消与退款处理

### 第七步：当天排班编辑限制

- **当天新增限制**：结束时间已过的号段不允许新增（提示"小于当前时间的排班不允许新增"）
- **当天编辑限制**：
  - 已结束的号段：号源数不可修改（提示"小于当前时间的排班号源数不允许操作"）
  - 进行中的号段：启用/禁用状态不可修改（提示"小于当前时间的排班不允许操作"）

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 未指定医生 | 提示"请指定要保存排班的医生" |
| 班次名称为空 | 提示"请正确填写班次" |
| 班次名称重复 | 提示"班次名称存在冲突，请检查" |
| 班次时间冲突 | 提示"班次时间存在冲突，请检查" |
| 号段时间不在班次范围内 | 提示"号段时间XX-XX，不在XX班次时间内" |
| 号段时间冲突 | 提示"号段时间存在冲突，请检查" |
| 当前无待发布排班 | 提示"当前无待发布排班" |
| 号源已满 | 提示"班次信息发生变更，请刷新后重试" |
| 预约超过截止时间 | 提示"预约不能超过号段结束时间前30分钟" |
| 停诊班次已有挂号未填原因 | 提示"停诊的班次已有患者挂号，请填写停诊原因" |
| 停诊原因超长 | 提示"停诊原因不能超过100字" |
| 班次不存在（停诊时） | 提示"班次不存在" |
| 停诊失败 | 提示"停诊失败，请联系管理员" |
