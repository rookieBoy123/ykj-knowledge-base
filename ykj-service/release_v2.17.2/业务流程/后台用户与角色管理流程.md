# 后台用户与角色管理流程

> 文档版本：2026-07-01 | 面向读者：技术管理员

## 概述

平台的管理后台和药房后台使用统一的用户角色权限体系。系统管理员可以创建用户、分配角色（含功能权限树），管理账号生命周期（启用/禁用/删除），以及密码重置和手机号变更等敏感操作。

## 触发条件

- 管理后台的操作触发对应的服务端接口
- 用户自行修改密码或切换角色

## 步骤详解

### 第一步：注册系统用户

管理员在后台新增用户时，系统执行以下后置处理：

1. 检查用户名是否已存在
2. 检查手机号是否已被注册（加密查询）：同一手机号不能在系统中重复注册
3. 生成密码盐值（Salt）+ 使用系统默认密码（或用户自定义密码）加密存储
4. 密码立即过期：`expirationDate = new Date()`，用户登录后会被强制要求修改密码
5. 默认设为普通用户（非超级管理员）
6. 根据选择的角色确定用户类型：如果仅分配了患者角色 → `userType=1`（小程序账号不能登录管理后台）；否则 `userType=0`（管理账号）
7. 如果填写了邮箱，自动发送注册通知邮件

### 第二步：用户角色变更

编辑用户时修改角色使用「增量计算」策略：

- 对比现状与提交的角色列表
- 新需要授予的角色 = 提交中有但现有中没有的（过滤禁止分配角色）
- 需要回收的角色 = 现有中有但提交中没有的（过滤禁止分配角色）
- 执行 `grantRoles` 和 `revokeRoles` 两步操作
- 角色变更后自动调整用户类型（全患者角色→小程序账号 / 否则→管理账号）

### 第三步：密码安全管理

- **密码加密**：使用随机Salt + 不可逆加密算法存储，数据库中不存明文
- **管理员重置密码**：重置为系统默认密码 + 密码立即过期 + 发送通知邮件
- **用户自行修改密码**：需要验证旧密码 → 新密码不能与旧密码相同 → 重新生成Salt加密 → 清除密码过期标记

### 第四步：登录安全控制

- **失败计数器**：每次登录失败 `loginErrorCount + 1`
- **锁定机制**：连续失败5次 → `blockEndTime = 当前时间 + 10分钟`，10分钟内的登录请求会被拦截
- **自动解锁**：10分钟后 `blockEndTime` 过期，可正常登录
- **成功重置**：登录成功后失败计数归零

### 第五步：账号启用/禁用

- 禁用：`status = DISABLED`，触发权限版本通知，该用户立即无法登录
- 启用：`status = ENABLED`，触发权限版本通知，恢复登录能力
- 权限版本通知（`permVersionNotifier.notifyUserChanged`）会刷新用户缓存的权限信息

### 第六步：手机号变更

管理员在后台变更用户手机号时：
1. 新手机号不能为空
2. 新手机号不能已被其他用户使用
3. 更新用户手机号
4. 发布 `UserChangePhoneEvent` 事件，通知其他模块（如CA签章系统）同步更新

### 第七步：删除用户

直接物理删除（`deleteById`），用户的角色关联关系会被保留在关联表中，但不影响使用。

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 手机号已被注册 | 后端拦截："该手机号码已经被注册过，请重新填写手机号码" |
| 手机号已有其他用户 | 后端拦截："手机号码已有用户" |
| 禁止分配的角色 | 后端拦截："以下角色不允许管理后台新增：XXX" |
| 账号被禁用登录 | 后端拦截："账号已被锁定，请联系管理员" |
| 密码连续输错5次 | 锁定10分钟，`blockEndTime = now + 10min` |
| 新密码与旧密码相同 | 后端拦截："新密码不能与旧密码相同" |
| 原密码错误（修改时） | 后端拦截："原密码错误" |
| 切换角色无权限 | 后端拦截："您没有权限进行当前操作" |
| 用户名已存在 | 后端拦截："用户名已存在" |
| 用户不存在 | 后端拦截："用户不存在" |

## 关联功能

- **角色菜单管理**：角色的功能权限通过菜单树配置
- **权限系统**：`permVersionNotifier` 通知前端缓存刷新
- **CA签章系统**：手机号变更事件通知CA同步
- **邮件通知**：注册和密码重置的邮件通知
