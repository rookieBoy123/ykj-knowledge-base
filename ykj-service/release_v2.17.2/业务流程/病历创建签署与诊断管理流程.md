# 病历创建、签署与诊断管理流程

> 文档版本 2026-07-02 | 面向读者：产品经理 / 技术运营

## 概述

医生在问诊过程中为患者创建病历，填写病史、诊断、治疗意见等信息，并进行电子签名认证。病历创建后可修改、补充诊断，也可在特定条件下撤销。病历签名状态影响后续开处方的权限。

## 触发条件

- **前置条件**：医生在问诊中、已选择患者
- **触发时机**：医生在问诊聊天界面点击「病历」进入编辑

## 步骤详解

### 第一步：创建病历

1. **组装医生信息**：获取医生姓名、医院、科室、职称、是否知名专家、全科标识
2. **判断病历类型**：医生科室是西医 → 西医病历，中医 → 中医病历
3. **设置患者信息**：如果传入了患者 ID，填充患者详细信息
4. **设置签名状态初始值**：医生签名状态初始为「未签署」（`UNSIGNED`）
5. **保存病历**：持久化到数据库
6. **保存诊断**：将传入的诊断列表保存到诊断表
7. **创建签章记录**：
   - 生成诊前问卷 PDF
   - 创建 CA 签章记录
   - 如果需要签章（`authSign=true`）：返回签章信息，前端触发签名流程
   - 如果不需要签章：直接完成签名流程（`completedSign`）
8. **生成分享二维码**（可选）：如果需要分享病历给患者扫码查看

### 第二步：更新病历

1. **获取已有病历**：校验病历存在
2. **校验医生**：医生必须存在
3. **签名状态重置**：修改后医生签名状态重置为「未签署」，需重新签名
4. **诊断增量更新**：
   - 对比已有诊断和本次更新的诊断
   - 找出需要删除的诊断 → 检查是否已被关联（如被处方引用）→ 如果已关联则禁止删除（提示「不允许删除已经被关联的诊断。诊断名称:【xxx】」）
   - 找出需要新增的诊断 → 保存
5. **重新创建签章记录**：同第一步
6. **补充诊断ID**：将本次新增的诊断ID回填

### 第三步：诊断管理

**追加诊断（addDiagnose）**：
- 传入病历 ID 和新诊断列表
- 与已有诊断对比去重：已存在的跳过，新增的保存
- 标记已关联的诊断不可修改

**删除诊断**：
- 更新病历时会自动对比并删除不再需要的诊断
- 已被处方引用的诊断不允许删除

### 第四步：电子签名

签名状态流转：未签署 → 已签署

**发起签名（goSign）**：
- 校验病历存在
- 校验未重复签署（已签署状态抛异常「病历已签署, 不需要重复签署」）
- 检查是否已有签章记录：有则直接返回已有的签章 ID，无需重新创建
- 无则创建新的签章记录

**签名完成（completedSign）**：
- 更新病历签名状态为「已签署」
- 异步处理：生成病历 PDF → 添加医生签章图片 → 水印处理 → 上传 OSS

### 第五步：病历撤销

- 撤销前执行断言链：检查是否有业务约束（如处方已开具、订单已创建等）
- 记录撤销原因
- 软删除病历记录

### 第六步：病历查询

- **按患者查询**：支持多个患者 ID 批量查询
- **按签名状态查询**：筛选未签署/已签署的病历
- **仅查询有问诊的病历**：子查询关联问诊预约表
- **按时间倒序**：最新的病历排最前

## 异常处理

| 情况 | 处理方式 |
|------|---------|
| 病历不存在 | 提示「病历不存在」 |
| 医生不存在 | 提示「医生不存在」 |
| 病历已签署重复签署 | 提示「病历已签署, 不需要重复签署」 |
| 删除已关联的诊断 | 提示「不允许删除已经被关联的诊断。诊断名称:【xxx】」 |
| 病历已绑定会员重新绑定 | 提示「该病历已绑定会员，不允许重新绑定」 |
| 会员不存在 | 提示「变更病历患者失败，会员不存在」 |

## 关联功能

- **CA 电子签章**（ykj-service-ca）：创建签章记录、PDF 签名
- **诊断服务**：诊断编码查询和分类
- **会员服务**（ykj-service-member）：会员绑定
- **处方开具**（ykj-service-inquiry）：诊断被处方引用后不可删除
- **问诊流程**：病历归属问诊记录
