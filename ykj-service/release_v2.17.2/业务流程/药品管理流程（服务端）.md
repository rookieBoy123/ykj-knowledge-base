# 药品管理流程（服务端）

> 文档版本：2026-07-02 | 面向读者：产品运营、技术支持

## 概述

药品管理是平台的核心基础模块，管理药房下的中药和成药药品信息，支持药品的CRUD、导入导出、库存管理、黑名单校验、第三方药品拉取等功能。

## 触发条件

- **服务模块**：ykj-service-medicine（MedicalDrugsServiceImpl）

## 步骤详解

### 第一步：药品分类

平台药品分为两大类：

| 类型 | 说明 | 关键字段 |
|------|------|---------|
| 成药（专利药） | 西药、中成药等标准化药品 | 通用名、批准文号、规格、用法用量 |
| 中药（草药） | 中药材 | 药材名、别名、规格等级、加工工艺 |

每种药品都关联到具体的药房，同一药品在不同药房中可有不同的价格和库存。

### 第二步：添加药品

**中药材添加**：
1. 校验药房+药名是否已存在，重复则提示「药品已存在」
2. 写入药品主表
3. 写入中药扩展信息表（药材属性等）

**成药添加**：
1. 校验药品唯一性（药房+规格+批准文号等组合）
2. 重复则提示「药品已存在」
3. 写入药品主表和成药扩展信息表

### 第三步：药品搜索与查询

**医生端小程序搜索药品时**：

1. **黑名单校验**：搜索关键词先过黑名单，如果命中精麻毒等管制药品，直接提示「该药品为精麻毒等管制类药品，按规定不可线上开方」
2. **药房授权过滤**：只展示当前医生被授权的药房的药品
3. **库存状态**：标注药品的库存情况（有货/缺货）
4. **拼音搜索**：如果输入英文（拼音），使用拼音相似度算法排序结果

查询支持多种参数：
- 药品名称（支持模糊搜索）
- 药房ID（必须条件）
- 药品类型（中药/成药）
- 分页

### 第四步：编辑药品

- 中药：支持修改药材属性、价格、库存等
- 成药：支持修改规格、价格、用法用量、库存等
- 编辑时不可修改药品唯一标识字段

### 第五步：药品图片管理

药品图片存储在OSS（对象存储）中，系统自动组装图片URL返回前端展示。

### 第六步：药品导入（Excel批量导入）

支持从Excel文件批量导入药品：

- **中药导入**：按模板格式导入药材列表
- **成药导入**：按模板格式导入成药列表
- **覆盖更新策略**：
  - 如果药品已存在（同名+同药房），走更新逻辑
  - 如果药品不存在，走新增逻辑
  - 单条导入失败不影响其他药品（记录错误信息）

### 第七步：药品导出

支持将药品数据导出为Excel文件，适用于数据分析、对账等场景。

### 第八步：第三方药品拉取

对接第三方药品供应商（如阿康）：
- 定时从第三方平台拉取药品信息
- 同步到本地药品库
- 支持分页拉取（每页100条）
- 连接超时阈值：7秒

### 第九步：药品库关联

每个药房的药品都关联到标准药品库（药品主数据）：
- 查询药品时同时查询标准库中的库存单位、换算比例
- 标准库信息不存在时抛异常「药品库信息不存在」

### 第十步：支付公开查询

支持面向公众的药品价格公开查询：
- 按药房ID筛选
- 按药品类型筛选
- 无需登录即可查询（费用公示场景）

## 黑名单机制

系统维护药品黑名单，包含精麻毒等管制类药品关键字：
- 搜索时匹配黑名单，命中则拦截
- 开方时校验处方的药品是否在黑名单中，命中自动删除并提示

## 异常处理

| 情况 | 系统处理方式 |
|------|------------|
| 药品已存在 | 提示「药品已存在」 |
| 搜索管制类药品 | 提示「该药品为精麻毒等管制类药品，按规定不可线上开方」 |
| 药品库信息不存在 | 抛内部错误「药品库信息不存在」 |
| 药房ID为空 | 提示「药房ID不能为空」 |
| 导入失败 | 逐条记录错误信息，不影响其他药品导入 |
| 第三方拉取超时 | 7秒超时，记录日志 |

## 关联功能

- 药房管理（药品归属药房）
- 处方开具（药品被处方引用）
- 费用公示（药品价格公开查询）
- 标准药品库（药品主数据）
- 黑名单管理
- 第三方供应商对接
