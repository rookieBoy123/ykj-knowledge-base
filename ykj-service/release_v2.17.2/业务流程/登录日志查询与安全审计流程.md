# 登录日志查询与安全审计流程

> 文档版本 2026-07-03 | 面向运维与安全管理员

## 概述

登录日志记录了所有管理员在后台系统的登录行为，包括登录成功和失败记录。管理员可以通过多种条件查询登录日志，用于安全审计和排查异常登录。系统会自动解析登录设备信息（操作系统、浏览器），并对APP端用户的手机号进行脱敏处理。

## 触发条件

- **前置条件**：管理员已登录管理后台，拥有"登录日志查询"权限
- **入口路径**：管理后台 → 系统管理 → 登录日志

## 步骤详解

### 第一步：查看登录日志列表

管理员进入登录日志页面后，系统展示所有登录记录。每条记录包含：
- **登录账号**：管理后台用户的用户名。如果是APP端用户，显示脱敏后的手机号（中间四位用星号替代，如 138****1234）
- **IP地址**：登录时的网络IP
- **操作系统**：登录设备使用的操作系统（Windows/Mac/iOS/Android等）
- **浏览器**：登录时使用的浏览器（Chrome/Edge/Safari等）
- **登录方式**：账号密码登录、短信验证码登录等
- **登录时间**：精确到分钟
- **登录结果**：成功或失败
- **失败原因**：如果登录失败，会显示具体原因（如"密码错误"、"账号被禁用"等）

系统会自动根据登录请求中的浏览器代理信息（User-Agent）解析出操作系统和浏览器名称。

### 第二步：筛选查询

管理员可以按以下条件筛选登录日志：
- **用户名/手机号**：输入完整或部分账号进行搜索。对于APP端用户，系统支持明文手机号和加密手机号两种搜索方式
- **IP地址**：按IP模糊搜索
- **登录时间范围**：选择开始时间和结束时间
- **登录状态**：筛选成功或失败的记录
- **客户端类型**：管理后台端或APP端。不选择时默认展示管理后台端

### 第三步：安全审计分析

管理员可以利用登录日志进行以下安全分析：
- **异常登录检测**：查看是否有陌生IP在非工作时间登录
- **暴力破解排查**：连续多次登录失败的记录可能是密码暴力破解尝试
- **账号安全验证**：确认特定账号的登录设备和地点是否正常

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| APP端用户手机号展示 | 自动脱敏处理，中间四位显示为星号，保护用户隐私 |
| 搜索账号时输入明文手机号 | 系统同时搜索明文和加密两种格式，确保能找到对应记录 |
| 未选择客户端类型 | 默认只显示管理后台端的登录记录 |
| 登录失败原因显示 | 显示具体失败原因，如密码错误、账号禁用、验证码错误等 |

## 关联功能

- **审计日志**：管理员查看登录日志的操作本身也可能被审计
- **系统用户管理**：可以跳转到对应用户的管理页面
- **会员管理**：APP端用户可关联到会员信息
