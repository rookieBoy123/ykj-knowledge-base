# 处方结算与费用计算流程

> 文档版本：v1.0 | 面向读者：运营、客服、产品

## 概述

患者在医生开具处方后，系统自动计算药品费用、加工服务费、运费和订金，生成完整的费用明细供患者确认。此流程同时支持普通支付和医保支付两种结算方式。

## 触发条件

- **前置条件**：医生已开出电子处方（含药品清单、用法用量、药房选择）
- **入口**：患者在小程序「处方详情」页面点击「去购买」或「下单」，系统执行预结算

## 步骤详解

### 第一步：预结算（不创建订单）

患者点击下单后，系统先做一次预结算，展示费用明细但不创建订单：

- **加载处方信息**：根据门诊号查找对应的处方记录，校验处方状态是否允许结算
- **获取药房信息**：显示药房名称、地址、加工服务是否可用
- **获取收货地址**：如果选择了快递配送，自动加载患者的默认收货地址
- **计算费用项**（分项明细）：
  1. **药品费用**：各药品的单价 × 数量之和
  2. **加工服务费**：中药代煎/颗粒剂等加工费用（按药房配置 × 剂数），可合并到药品金额中
  3. **运费**：根据收货地址和药房运费规则计算，满足包邮条件时显示为0
  4. **订金**：如果药房支持货到付款（COD），且订单金额超过最低限额，展示首付订金金额
- **总金额**：所有费用项加总。如果计算结果为负数（如优惠导致），则置为0

### 第二步：正式结算（创建订单）

患者确认费用并点击支付后，系统执行正式结算：

- **防重复检查**：先查询是否已存在该处方的有效订单，如果已存在则提示"已存在订单信息"
- **重新预结算**：费用明细不合并服务费（药品金额和服务费分开显示）
- **创建订单**：生成订单号，订单来源标记为"处方"，订单金额为预结算总金额
- **订金处理**：如果药房支持货到付款且处方金额超过最低限额，订单仅需支付订金（首付），剩余金额在收货时付清
- **回填关联**：将订单号写到处方记录中，建立关联关系

### 第三步：已创建订单查询

如果处方已生成过订单，患者再次进入时系统展示已有订单的详细信息：

- **订单信息**：显示支付渠道、支付金额、订单状态
- **费用明细**：展示所有费用项（含医保支付信息，如统筹支付/个账支付/自费现金）
- **收货地址**：快递配送时显示收货人、电话、详细地址
- **支付阶段判断**：区分"首次进入"和"已创建订单"两种模式，已创建订单不显示订金首付

### 第四步：医保结算（医保支付专用）

如果患者选择医保支付，系统执行额外的医保结算流程：

- **费用明细上传**：将处方药品、诊断、患者信息、费用总额上传到医保平台
- **取消历史上传**：如果之前已上传过费用明细，先取消旧的再上传新的（防止重复）
- **授权码处理**：如果有授权码，解析授权码获取用户的定位信息用于医保定点校验
- **个账支付判断**：系统判断是否支持个人账户支付。如果不支持，患者只能选统筹+自费现金
- **诊费处理**：如果处方关联了问诊，问诊诊费也会加入医保费用明细中（诊金超过10元按10元计算）

### 第五步：订单医保报销明细查询

支付完成后，患者可查看医保报销明细：

- **条件**：仅医保支付订单且订单信息完整才能查询
- **显示内容**：医保统筹支付金额、个人账户支付金额、自费现金金额、报销比例等详细信息
- **数据来源**：调用医保平台接口实时查询，以图片形式展示结算单

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 处方不存在 | 提示"处方不存在，无法创建订单" |
| 药房信息找不到 | 提示"找不到对应药房信息" |
| 已存在订单 | 提示"已存在订单信息"，禁止重复创建 |
| 医保费用明细已上传过 | 先取消旧的再重新上传，防止重复 |
| 订单非医保支付 | 提示"该订单不支持查询明细"，不显示报销详情 |
| 费用计算出现负数 | 总金额自动置为0，按0元订单处理 |
| 收货地址不存在（快递配送） | 提示"该订单收货地址信息不存在" |

## 关联功能

- **订单生成与支付处理流程**：结算创建订单后进入通用支付流程
- **退款审核与处理流程**：订单支付后如需退款
- **处方开具与审核签章流程**：结算的前置条件
- **医生收益结算流程**：订单支付后触发医生收益计算
