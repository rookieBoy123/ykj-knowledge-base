# 会诊全生命周期管理流程（服务端）

> 文档版本 2026-07-02 | 面向读者：产品、运营、技术

## 概述

会诊是互联网医院的核心业务之一，由经治医生发起，邀请多位专家医生共同为患者进行诊疗。系统管理会诊从申请到完成的完整生命周期，包括申请提交、审核批准、医生签名、患者确认、订单支付、会诊进行、报告生成等环节。

## 触发条件

- **发起方**：医生端发起会诊申请
- **关联模块**：ykj-service-consultation（会诊服务）、ykj-service-order（订单服务）、ykj-service-ca（签章服务）、ykj-service-rtc（实时通讯）

## 状态流转

```
草稿(DRAFT) → 待审核(APPLY) → 审核通过(APPROVED) → 待支付 → 待开始(WAIT_START)
    ↓ 驳回            ↓                              ↑              ↓
  (删除)        审核驳回(REJECTED)              0元自动确认      进行中(HANDING)
                      ↓                                              ↓
                   已完成(FINISH) ← 正常结束 ← 待报告(WAIT_REPORT)
```

## 步骤详解

### 一、会诊申请

1. 医生提交会诊申请，系统生成会诊单号（consultationNo）
2. 记录患者信息、病情描述、诊断、会诊目的、期望时长、科室
3. 会诊初始状态为「草稿」或「待审核」

### 二、审核处理

**审核通过**：
1. 校验当前会诊状态必须是「待审核」
2. 校验所有受邀医生是否已开通会诊功能（医生服务类型为会诊且已启用）
3. 计算会诊费用：累加每位医生的会诊服务费
4. 保存会诊医生信息到 t_consultation_doctor 表
5. 写入操作日志
6. **设置超时未支付定时任务**：距离开会时间15分钟仍未支付则自动取消

**审核驳回**：
1. 校验当前流程状态必须是「待审核」
2. 更新审批状态为「已驳回」，流程状态为「已完成」，结束原因为「驳回」
3. 记录驳回原因
4. 发送审核结果通知（订阅消息）
5. 发布会诊完成事件

### 三、订单创建与支付

1. 患者确认后，系统创建会诊订单（订单来源类型=会诊）
2. 订单包含费用明细（「会诊费用」一项）
3. 订单过期时间设置为会诊开始时间
4. **0元订单特殊处理**：费用为0时自动完成支付，直接进入「待开始」状态
5. 支付成功后：
   - 流程状态更新为「待开始」
   - 写入支付日志（记录患者姓名+「已支付」）
   - 发送系统消息给经治医生（「会诊已生效」）
   - 发送系统消息给患者（「确认并已支付」）
6. **定时任务**：会诊开始前10分钟发送提醒消息
7. **定时任务**：到开始时间自动处理会诊状态

### 四、会诊进行

1. 经治医生和受邀医生进入会诊房间（IM群聊 + 实时音视频）
2. 会诊过程中医生可提交会诊意见
3. 患者可在会诊中进行签名确认

### 五、报告与完成

1. 会诊结束后进入「待报告」状态
2. 医生提交会诊报告（含诊断意见、治疗方案）
3. 所有医生签名确认后，会诊流程结束（正常结束）
4. 发布会诊完成事件，触发收益结算

## 系统消息与通知

| 时机 | 接收方 | 消息内容 |
|------|--------|---------|
| 支付成功 | 经治医生 | 会诊已生效 |
| 支付成功 | 患者 | 确认并已支付 |
| 开始前10分钟 | 经治医生+患者 | 会诊即将开始提醒 |
| 审核结果 | 申请医生 | 审核通过/驳回通知 |

## 关键常量

| 常量 | 值 | 说明 |
|------|-----|------|
| 发送准备通知提前时间 | 10分钟 | 会诊前10分钟提醒 |
| 超时未支付自动取消 | 开始前15分钟 | 仍未支付则自动取消 |
| 超时未审核 | 开始前30分钟 | 仍未审核处理 |
| 会诊结束后 | 12小时 | 未完成则自动处理 |

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 审核时不是待审核状态 | 抛异常「当前会诊不是待审核状态」 |
| 受邀医生未开通会诊 | 抛异常「XX医生暂未开通会诊功能」 |
| 无效会诊单号 | 抛异常「无效的会诊单号」 |
| 会诊费用为0 | 创建订单后自动确认支付 |
| 非主治医生签字 | 抛异常「非主治医生无法签字」 |
| 未支付超时 | 定时任务自动取消会诊 |

## 关联功能

- **订单模块**（ykj-service-order）：会诊订单的创建与支付
- **签章模块**（ykj-service-ca）：会诊各环节的电子签名
- **实时通讯**（ykj-service-rtc）：会诊过程中的音视频通话
- **IM群聊**（ykj-service-im）：医生群聊讨论
- **医生收益**（ykj-service-hospital）：会诊完成后的医生收益结算
- **订阅消息**（ykj-service-subscribe）：会诊状态变更通知
