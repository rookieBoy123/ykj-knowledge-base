# AI辅助问答错误锁定与重试机制（服务端）

> 文档版本 2026-07-05 | 面向后端开发者/运维人员

## 概述

医生在问诊聊天室中使用AI辅助功能（提问建议 / 病历辅助）时，后端采用「失败计数 + 锁定」机制来防止AI服务异常时持续消耗资源。连续失败3次后，该功能对当前问诊单锁定30分钟。

## 触发条件

- 医生在问诊聊天界面点击「AI提问建议」或「AI病历辅助」按钮
- 后端需要先通过配置开关校验：`questionAiSuggestion`（提问建议）和 `medicalAiSuggestion`（病历辅助）
- 两个AI功能独立计数、独立锁定，互不影响

## 功能开关

系统有两个独立的配置项控制AI功能是否启用：

| 配置项 | 控制功能 | 关闭时提示 |
|--------|----------|-----------|
| `questionAiSuggestion` | 医生问答辅助（提问建议） | "系统暂不支持医生问答辅助, 请联系管理员" |
| `medicalAiSuggestion` | 病历辅助 | "系统暂不支持病历辅助, 请联系管理员" |

两个开关都在 `PaasProperties` 中配置，属于平台级全局开关。

## 步骤详解

### 第一步：锁定状态检查

每次医生请求AI辅助时，后端先检查Redis中是否有锁定标记：

```
缓存Key格式：ai-lock:{类型}:{问诊单号}
  类型 = "question"（提问建议）或 "medical-record"（病历辅助）

如果存在锁定标记 → 直接返回失败，携带 `lockingTime`（解锁时间戳）
前端收到 `lockingTime` 后可展示倒计时
```

### 第二步：错误次数检查

检查Redis中的错误计数：

```
缓存Key格式：ai-error-quantity:{类型}:{问诊单号}

如果计数 ≥ 3 → 设置锁定标记（30分钟有效期），返回锁定状态
如果计数 < 3 → 进入第三步，尝试调用AI服务
```

### 第三步：调用AI能力服务

构建请求参数后调用 PaaS 平台的 `SuggesterService`：

- **提问建议**：调用 `suggesterService.questions(reqDTO)`，返回最多9条建议问题
- **病历辅助**：调用 `suggesterService.medicalRecord(reqDTO)`，返回病历建议（含诊断编码映射）

请求参数包含：
- 患者年龄（通过身份证号计算）
- 患者性别（通过身份证号判断）
- 病情描述（优先取病历中的描述，其次取问诊单中的描述）
- 六项病史（过敏史/现病史/个人史/家族史/婚育史/月经史）
- 聊天记录（取群组中的Text类型消息，过滤非医患消息）
- 默认9条记录

### 第四步：处理结果

- **成功**：清除该类型的错误计数（Redis delete），返回建议内容
- **失败**（接口返回非200、数据为空、或异常抛出）：
  - 错误计数 +1（首次设为1，后续递增）
  - 缓存有效期30分钟（与锁定时间一致）
  - 返回 `success=false`（无建议内容）

## 病史信息的三个数据来源（优先级从低到高）

| 来源 | 数据项 | 说明 |
|------|--------|------|
| 患者档案 | 过敏史、现病史、个人史 | 患者注册时填写的基础信息 |
| 预问诊信息 | 全部六项病史 | 通过AI预问诊Agent采集，需状态为"已上报"才生效 |
| 病历信息 | 全部六项病史 | 医生已填写的病历，覆盖前两级数据 |

后一级数据会覆盖前一级的空值，不会覆盖已有的非空值（即首次非空优先，后续来源不覆盖）。

## 聊天记录的筛选规则

发送给AI的聊天记录需要经过以下过滤：

1. 只取 `fromUserId` 包含 `@doctor` 或 `@patient` 后缀的消息（排除系统消息）
2. 只取 `bodies` 非空的消息
3. 只取消息体类型为 `Text` 的消息（忽略图片/语音/视频等）
4. 如果没有任何符合条件的聊天记录，用问诊单的病情描述构造一条默认消息（角色="患者"）

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 功能开关关闭 | 抛出异常："系统暂不支持XXX, 请联系管理员"（前端展示提示） |
| 当前已被锁定 | 返回 `lockingTime` 时间戳（前端展示倒计时） |
| 连续失败3次 | 自动设置30分钟锁定，返回锁定状态 |
| AI服务调用异常 | 捕获异常，记录错误日志，错误计数+1 |
| 预问诊信息获取失败 | 记录warn日志，跳过该来源，不影响主流程 |
| 病历信息获取失败 | 记录warn日志，跳过该来源，不影响主流程 |
| 聊天记录获取失败 | 记录warn日志，使用默认消息（病情描述） |
| 诊断编码映射失败 | 病历辅助中的诊断列表返回空列表（不中断主流程） |

## 关联功能

- **问诊管理**：锁定/计数以问诊单号（inquiryNo）为维度
- **患者管理**：患者年龄和性别通过身份证号自动计算
- **预问诊Agent**：预问诊信息作为病史的补充来源
- **病历管理**：病历信息作为病史的最高优先级来源
- **IM即时通讯**：聊天记录从群组消息中提取
