# 开放平台处方对接流程

> 文档版本：2026-07-01 | 面向读者：非技术读者（业务视角）

## 概述

第三方医疗机构通过开放平台接口将处方提交到平台进行审核、结算和药品下单。平台负责处方审核流转、费用计算、订单生成和状态回调。

## 触发条件

- 第三方机构系统调用开放平台接口，提交医生开具的处方数据

## 业务步骤

### 第一步：处方接收与校验

第三方机构提交处方时，系统进行多层校验：
- **机构身份验证**：确认机构已注册且启用
- **医生匹配**：根据机构传入的医生信息精确匹配平台医生，匹配不到则提示"医生不存在"
- **药品校验**：处方中的药品需与平台药品库匹配，匹配不到则报错
- **患者信息校验**：必填项（姓名、证件号等）不能为空

### 第二步：处方状态流转

处方提交后在平台中经历以下状态：
1. **草稿/待提交**：初始接收状态
2. **医生审批**：平台对应医生审核处方内容
3. **药师审批**：平台药师审核处方合理性
4. **审核通过**：可进入结算和下单
5. **审核不通过**：返回拒绝原因给机构

双重审核（医生+药师）确保处方安全合规。

### 第三步：处方关联订单

审核通过后，机构可基于该处方创建药品订单：
- 处方必须在有效期内（超过有效期的不允许创建订单）
- 处方必须审核通过（未审核或审核不通过的不允许创建订单）
- 已关联订单的处方不可重复下单

### 第四步：订单状态回调

订单处理过程中，关键状态变更通过回调通知机构：
- 支付成功/失败
- 发货完成（含物流单号）
- 订单取消/退款

机构可主动查询处方和订单的当前状态。

## 异常处理

| 情况 | 系统处理方式 |
|------|------------|
| 医生不存在 | 返回错误："医生不存在"，需机构确认医生已在平台注册 |
| 药品不匹配 | 返回错误，需机构与平台对齐药品编码 |
| 处方未审核通过 | 提示"处方未审核通过不允许创建订单" |
| 处方已过期 | 不允许创建订单，机构需重新开具处方 |
| 回调地址不可达 | 记录失败日志，机构需主动查询状态 |

## 关联功能

- 机构管理（后台配置机构密钥和回调地址）
- 处方审核流程（医生审批+药师审批）
- 药品订单处理（计算费用、生成订单、通知发货）
- 回调管理（状态变更回调、回调日志查看）
