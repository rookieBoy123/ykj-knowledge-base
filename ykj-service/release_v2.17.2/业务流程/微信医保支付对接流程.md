# 微信医保支付对接流程

> 文档版本 v1.0 | 面向读者：产品经理、后端开发

## 概述

平台通过微信医保支付接口，支持患者使用医保（统筹+个账）支付处方药品费用。整个流程涉及预支付下单、医保小程序授权、支付回调、退款等环节。适用于开通了微信医保支付的城市和医院。

## 触发条件

- **前置条件**：医院/平台已开通微信医保支付（配置了 AppId、MchId、CityId 等参数）、患者已在医保小程序中激活医保电子凭证、医生是该医院的医保医师
- **入口**：患者在小程序的收银台页面选择"医保支付"→跳转医保小程序授权→返回平台完成支付

## 步骤详解

### 第一步：预支付下单（统一下单）

患者在小程序收银台点击"医保支付"后，系统向微信医保接口发起统一下单。

1. **判断消费类型**（ConsumeType，从配置读取）：
   - `Account`：纯个账支付（用户使用医保个人账户余额支付）
   - `Insurance`：统筹+现金（医保统筹报销一部分，剩余用现金支付）
   - `InsuranceAndAccount`：统筹+个账（医保统筹报销一部分，剩余用医保个账支付）

2. **组装请求参数**：
   - 基础信息：AppId、MchId、随机字符串、签名（MD5）
   - 支付信息：订单金额（元转分）、医保统筹金额、现金补足金额、自费金额
   - 患者信息：姓名、身份证号（MD5加密后传输）、就诊类型（医保电子凭证兼容社保卡）
   - 回调地址：支付结果通知地址 + 支付完成跳转地址（`returnUrl` 带门诊号参数）
   - 医院信息：医院名称（作为订单标题"XX医院--处方订单"）
   - 是否使用沙箱环境：配置项 `useSandbox=1` 使用测试环境

3. **支付类型判断**：
   - 如果 `cashFee > 0`（有自费运费）：支付类型为 `CashAndInsurance`（现金+医保混合支付）
   - 如果 `cashFee = 0`：支付类型为 `Insurance`（纯医保支付）

4. **保存预支付记录**：记录订单号、平台支付单号（订单号+4位随机字符）、金额、OpenId、回调状态初始为"等待中"（Waiting）

5. **调用微信医保统一下单**：POST 请求到微信医保接口，获取支付链接（`payUrl`）和小程序 AppId（`payAppId`）

### 第二步：授权码获取

在医保支付前，系统需要获取医保授权码。

1. 患者在小程序扫描医保电子凭证二维码，系统调用 `getPayAuthNo` 接口
2. 请求参数：二维码内容（`qrcode`）+ 微信 OpenId
3. 使用合作方密钥（PartnerSecret + PartnerId）生成 HmacSHA256 签名
4. 向微信医保授权查询接口发送 POST 请求
5. 返回授权码（`payAuthNo`），供后续支付使用

### 第三步：支付结果处理

医保支付是异步的，系统通过两种方式获取支付结果：

1. **主动查询**（`paymentResult` 方法）：
   - 先查预支付记录表，检查是否已收到过回调
   - 如果回调状态为"成功"（Success），直接返回支付成功
   - 如果还在"等待中"（Waiting），调用微信医保订单查询接口主动查询
   - 查询到支付成功后，发布 `WXYBPayedEvent` 事件，触发后续业务（订单状态更新、库存扣减等）

2. **被动回调**：
   - 微信医保平台回调 `WXYBPaymentCallbackController`
   - 回调内容包括支付单号、医保统筹金额、个账金额、自费金额等
   - 更新预支付记录的回调状态为"成功"

### 第四步：医保退款

当需要退款时，系统调用微信医保退款接口。

1. **调用退款申请接口**（`applyForRefund`）：
   - 传入平台退款单号和原支付单号
   - 传输退款金额（分）
   - 微信医保接口返回退款结果

2. **退款查询**（`refundQuery`）：
   - 根据平台支付单号查询退款状态
   - 确认退款是否成功

3. **重要规则**：医保退款执行"先退医保再退自费"的分步逻辑，确保医保统筹金额先退回医保基金池，再退自费部分。

### 第五步：其他辅助功能

1. **订单关闭**（`closeOrder`）：如果订单在有效期内未支付，可以调用关单接口释放医保支付额度。

2. **对账单下载**（`downloadBill`）：管理员可以在后台下载医保结算对账单。

3. **实名核验**（`verifyName`）：
   - 校验身份证号是否为当前微信的实名信息
   - 如果不是本人身份证（错误码 80005），返回特殊错误码 100500 给前端
   - 其他异常统一返回 400 错误

## 异常处理

| 情况 | 系统处理方式 |
|------|------------|
| 微信医保返回失败 | 后端抛出异常，错误信息直接透传微信医保返回的 `returnMsg` |
| 预支付信息保存失败 | 后端返回"保存预支付信息失败" |
| 找不到预支付记录 | 后端返回"找不到预支付记录" |
| 身份证不是当前微信实名 | 后端返回特殊错误码 100500，前端可据此引导用户 |
| 支付查询时预支付单已处理过 | 记录日志警告，直接返回成功（防重复处理） |
| 支付未成功（订单查询返回非SUCCESS） | 返回 `success=false`，不触发后续业务 |
| 不支持的支付类型 | 后端抛异常"不支持的支付类型" |
| 获取授权码失败 | 后端返回"获取授权码失败" |
| 身份证 MD5 加密失败 | 后端返回"身份证MD5加密失败" |
| Token 获取或签名生成异常 | 后端返回"组装参数异常" |

## 关联功能

- **订单支付**：收银台跳转医保支付入口（ykj-service-order）
- **退款审核**：医保退款的先退医保再退自费逻辑（ykj-service-order/RefundApplyServiceImpl）
- **患者医保定点**：医保支付前需检查患者是否已定点（ykj-service-hospital）
- **医保结算上传**：管理后台上传医保结算清单（ykj-service-yb-declare）
- **系统配置**：医保支付相关的 AppId/MchId/CityId 等参数配置（ykj-service-system）
