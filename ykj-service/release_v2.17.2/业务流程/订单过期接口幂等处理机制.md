# 订单过期接口幂等处理机制

> 文档版本：v2.17.2 | 面向读者：后端开发、测试、运维

## 概述

当用户在支付倒计时归零后，小程序调用订单过期接口（`POST /api/orders/{orderNo}/expire`）将订单标记为过期。新版本的过期接口增加了**幂等保护**：即使多次调用或订单已被其他途径取消，接口也不会报错，而是直接返回成功。

## 触发条件

- 用户在小程序收银台页面停留到支付倒计时归零
- 倒计时归零后，前端先调用 `verifyAlreadyPay` 检查订单是否已被支付（处理跨端支付的场景）
- 如果订单确实未支付，再调用 `expireOrder` 过期接口
- 过期接口也可被后台定时任务触发

## 步骤详解

### 第一步：查询订单

系统通过 `orderNo` 查询订单。如果订单不存在，直接抛出"订单不存在"错误。

### 第二步：幂等检查

系统检查订单当前状态是否属于**异常状态**（`EXCEPTION_ORDER_STATUS`）：

| 异常状态 | 含义 | 幂等返回 |
|---------|------|---------|
| PENDING_REFUND（待退款） | 订单正在等待发起退款 | 返回成功 |
| CANCELED（已取消） | 订单已被用户或系统取消 | 返回成功 |
| EXPIRED（已过期） | 订单已因超时被标记为过期 | 返回成功 |
| REFUNDING（退款中） | 订单正在退款处理中 | 返回成功 |
| REFUNDED（已退款） | 订单已完成退款 | 返回成功 |

如果订单已经是以上任一状态，接口**直接返回成功**，不执行任何操作。这避免了重复调用导致的异常。

### 第三步：状态校验

如果不是异常状态，系统检查订单是否处于**待支付**（PENDING_PAYMENT）状态：
- 是 → 继续执行过期操作
- 否 → 抛出错误 "当前订单状态不允许过期操作"（说明订单已支付或其他不可过期状态）

### 第四步：执行过期

系统调用 `cancelOrder` 方法：
- `timeoutCancel = true`：标记为超时取消（订单状态变为 EXPIRED 而非 CANCELED）
- `refundEnabled = false`：不触发退款流程（仅待支付超时取消，无退款需求）
- 取消备注固定为 "订单超时未支付自动取消。"

## 状态流转

```
待支付 ──[倒计时归零]──→ 已过期
   │
   └──[已被取消]──→ 已取消 ──[再次调用过期接口]──→ 幂等返回成功（不重复执行）
```

## 与前端协作的完整流程

```
用户端收银台  ──→  倒计时归零
                      │
              ┌───────┴───────┐
              │ verifyAlreadyPay │  ← 先检查是否已支付（跨端支付场景）
              └───────┬───────┘
                      │
          ┌───────────┼───────────┐
          │ 已支付    │           │ 未支付
          │           │           │
     跳转支付成功页   │      POST /{orderNo}/expire
                      │           │
                  弹窗提示        │
                (订单已取消)      订单状态 → EXPIRED
```

## 异常处理

| 情况 | 系统处理方式 |
|------|------------|
| 订单不存在 | 抛出 "订单不存在" 错误 |
| 订单已在异常状态 | 幂等返回成功（不报错） |
| 订单已支付（非待支付） | 抛出 "当前订单状态不允许过期操作" |
| 前端倒计时归零但订单实际已支付 | 前端先调用 verifyAlreadyPay 确认为未支付后才调过期接口 |

## 关联功能

- **订单取消接口**：`POST /api/orders/_cancel` — 用户主动取消，状态变为 CANCELED（可能有退款流程）
- **verifyAlreadyPay**：确认支付状态，防止误将已支付订单过期
- **订单支付倒计时**：前端收银台显示的支付有效期限
- **服务模块**：`ykj-service-server` 的 `ApiOrderController`（Controller），`ykj-service-order` 的 `OrderCommandServiceImpl`（Service）
