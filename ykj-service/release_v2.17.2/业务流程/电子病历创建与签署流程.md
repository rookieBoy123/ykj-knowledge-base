# 电子病历创建与签署流程

> 文档版本 v1.0 | 面向系统分析

## 概述

电子病历是问诊结束后医生为患者创建的就诊记录，包含患者基本信息、病史、中医四诊、诊断结果、治疗意见等内容。病历创建后需要通过CA电子签名或手写签名完成签署，签署完成后生成带医院印章的PDF文件。

**对应服务模块**：ykj-service-inquiry

## 触发条件

- 问诊结束后，医生在接诊页面提交病历
- 病历编辑后需要重新签名
- 开放平台通过接口创建或查询病历

## 病历创建流程

### 第一步：组装病历数据
1. 根据医生ID查询医生信息（姓名、医院、科室、职称、中西医保类型）
2. 根据患者ID查询患者信息（姓名、性别、身份证、年龄、月龄、监护人等）
3. 根据医生岗位类型判断病历类型：
   - 西医岗位 → 西医病历（type=1）
   - 中医岗位 → 中医病历（type=2）
4. 初始医生签名状态设为"未签"

### 第二步：保存诊断
1. 校验诊断名称不能为空（"诊断名称不能为空"）
2. 根据诊断编码查询标准诊断名称，存在则覆盖
3. 西医诊断（disease类型）需调用诊断服务校验合法性
4. 保存病历和诊断记录

### 第三步：创建签署记录
1. 生成签署业务编号（格式：BL + yyyyMMdd + 7位随机数）
2. 签署状态设为"签署中"
3. 判断是否启用CA签名：
   - CA全局开关开启 + 病历CA功能开关开启 + 未明确禁用CA → 走CA签名流程
   - 否则 → 走普通签名（仅附加文字"医生签名："）

### 第四步：CA签名处理
如果启用CA签名：
1. 生成待签PDF文件（含医院印章）
2. 调用CA服务创建签署，传入患者信息、医生CA账号、签名定位坐标
3. 返回CA唯一标识，前端引导医生完成CA认证和签署

如果不启用CA：
1. 直接完成签署（不走CA流程）

## 病历更新流程

1. 检查病历是否存在（"查询病历失败，病历不存在"）
2. 修改后签名状态重置为"未签"（需要重新签名）
3. 处理诊断变更：
   - 找出需要删除的诊断（旧有但新数据中不存在的）
   - 找出需要新增的诊断（新数据中有但旧数据中不存在的）
   - 删除诊断前校验：已被关联的诊断不允许删除（"不允许删除已经被关联的诊断。诊断名称：【xxx】"）
4. 重新创建签署记录

## 完成签署流程

1. 校验病历和签署记录存在（"参数错误，未找到对应的病历签署记录"）
2. 校验签署状态必须是"签署中"（"非签署中病历, 无法完成签署"）
3. 获取医生签名：
   - 有CA签署记录：从CA服务查询签署完成后的PDF文件并解码
   - 无CA签署记录：使用医生手写签名图片
4. 生成带签名的完整PDF
5. 上传PDF到OSS存储
6. 更新病历状态为"已签"
7. 更新签署记录状态为"已签"

## 签名状态

| 状态 | 说明 |
|------|------|
| 未签 | 病历创建后的初始状态 |
| 签署中 | 已创建签署记录，等待医生完成签名 |
| 已签 | 签名已完成，PDF已生成并存储 |

## 诊断关联保护

病历中的诊断可能被其他业务（如处方）关联引用。被关联的诊断：
- 不允许删除（修改病历时会校验）
- 标记为"不可修改"（unalterable=true）
- 前端据此控制编辑权限

## 病历与会员绑定

- 病历可绑定会员ID
- 如已绑定会员，不允许重新绑定到其他会员（"该病历已绑定会员，不允许重新绑定"）
- 如绑定的会员不存在，抛出"变更病历患者失败，会员不存在"

## 病历撤销

- 撤销前会执行所有注册的撤销断言校验
- 记录撤销原因
- 软删除病历记录

## 异常处理

| 情况 | 后端提示 |
|------|---------|
| 病历不存在 | 查询病历失败，病历不存在 |
| 医生不存在 | 医生不存在 |
| 诊断名称为空 | 诊断名称不能为空 |
| 病历已签署重复签署 | 病历已签署, 不需要重复签署 |
| 非签署中状态完成签署 | 非签署中病历, 无法完成签署 |
| 未找到签署记录 | 参数错误，未找到对应的病历签署记录 |
| 删除已关联的诊断 | 不允许删除已经被关联的诊断。诊断名称：【xxx】 |
| 病历已绑定会员 | 该病历已绑定会员，不允许重新绑定 |
| 会员不存在 | 变更病历患者失败，会员不存在 |

## 关联功能

- 问诊预约（通过inquiryNo关联）
- 处方开具（诊断关联）
- CA电子签名服务（ykj-service-ca）
- 会员服务（ykj-service-member）
- 患者服务（ykj-service-hospital）
- OSS文件存储
