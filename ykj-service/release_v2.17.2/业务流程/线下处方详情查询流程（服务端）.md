# 线下处方详情查询流程（服务端）

> 文档版本 2026-07-05 | 面向读者：产品/技术 | 服务模块：ykj-service-his | 表：his_prescription / his_prescription_item

## 概述

服务端提供线下HIS处方的查询接口，通过处方单号查询患者在医院开具的处方完整信息，包括基本信息、患者信息、药品明细（成药/中药分别处理）、以及电子处方原件下载链接。该接口供小程序医生端和患者端共同调用。

## 触发条件

- **调用端**：小程序（医生端问诊聊天室、患者端健康档案）
- **接口路径**：GET `/api/patient/his-treatment-record/{outpatientNo}`
- **无需权限校验**：该接口为公开API接口，通过处方单号即可查询

## 步骤详解

### 第一步：接收请求

接口接收URL路径参数 `outpatientNo`（处方单号），调用 `HisPrescriptionService.findByOutpatientNo()` 查询处方记录。

### 第二步：查询处方主记录

从 `his_prescription` 表中查询处方主体信息，包含：

| 字段 | 说明 |
|------|------|
| outpatientNo | 处方编号（唯一标识） |
| type | 处方种类：1=治疗项目 / 2=理疗项目 / 3=体检项目 / **4=中草药处方** / **5=西药/成药** / 6=附加项目 / 7=材料 |
| recipeType | 处方类型：0=普通 / 1=急诊 / 2=儿科 / 3=麻醉精一 / 4=精二 / 5=其他 |
| recipeDoctorName | 处方医生姓名 |
| pharmacistName | 药师姓名 |
| decoctingMethod | 煎煮方法（中药处方专用） |
| totalDosage | 总剂数（中药处方专用） |
| usage | 中药用法 |
| usageFrequency | 用药频次 |
| medInstruct | 用药时间说明 |
| taboo | 用药禁忌 |

### 第三步：查询处方药品明细

通过 `outpatientNo` 关联查询 `his_prescription_item` 表，获取处方中所有药品的明细列表。每条明细包含：

| 字段 | 说明 |
|------|------|
| medicineName | 药品名称 |
| useQty | 用量（数值） |
| phruom | 用量单位 |
| usage | 用法说明 |
| usageFrequency | 用药频次 |
| dosageForm | 剂型 |
| specification | 规格 |
| useDays | 使用天数 |
| qty | 开药数量 |
| diagUnit | 门诊单位 |
| decoctingRemark | 煎药备注 |

### 第四步：组装患者信息

处方记录本身不直接存储患者信息（患者信息由HIS系统关联），通过数据库关联查询获取：

- **patientName**：患者姓名
- **gender**：性别（1=男 / 2=女）
- **age**：年龄
- **departmentName**：科室名称
- **diagnosis**：诊断结论

### 第五步：生成电子处方下载链接

如果处方有对应的电子处方文件（`treatmentObjectId` 有值），系统生成一个可下载的文件URL（`treatmentUrl`），供小程序端预览电子处方原件。

### 第六步：返回完整数据

将所有字段（主记录 + 药品明细列表 + 患者信息 + 电子处方URL）组装为 `HisPrescription` 对象返回给小程序。

## 异常处理

| 异常 | 处理方式 |
|------|----------|
| 处方单号不存在 | 由Service层处理，返回空或抛出业务异常 |
| 处方无药品明细 | items为空数组，小程序端RP区域不显示任何药品 |
| 无电子处方文件 | treatmentUrl为空，小程序端点击提示「暂无电子处方」 |
| 患者信息关联查询失败 | patientName/gender/age等字段为空，小程序端不显示或显示默认值 |
| 处方种类非4/5 | 小程序端仍可查看基本信息，但RP区域模板不会匹配 |

## 数据流

```
小程序（医生端/患者端）
  → GET /api/patient/his-treatment-record/{outpatientNo}
    → ApiHisTreatmentOrderController.findByOutpatientNo()
      → HisPrescriptionService.findByOutpatientNo()
        → his_prescription 表（处方主记录）
        → his_prescription_item 表（药品明细）
        → 关联查询患者信息
        → 生成电子处方URL
      → 组装 HisPrescription 对象
    → 返回完整处方数据
  ← 小程序展示处方详情页
```

## 关联功能

- **小程序-线下处方详情**：`packageDoctor1/pages/rpDetail/offline.vue`，调用本接口展示全量数据
- **患者端-线下就诊记录**：`packagePatient2/pages/healthRecords/offLineRecords`，列表页点击后跳转详情
- **医生端-聊天室病历卡片**：`timIM/pages/video/components/medicalrecord-box`，医生通过卡片跳转
- **HIS系统数据同步**：处方数据由HIS系统通过数据同步机制写入本系统的 `his_prescription` 和 `his_prescription_item` 表
- **电子处方文件存储**：`treatmentObjectId` 指向文件服务的Object ID，用于文件预览
