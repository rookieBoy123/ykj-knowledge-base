# 订单创建与支付准备流程

> 文档版本 2026-07-02 | 面向产品运营、客服

## 概述

订单模块负责管理平台所有交易订单的完整生命周期。本流程覆盖订单从创建、支付准备、取消到自动签收的核心后端逻辑，包括0元订单特殊处理、多渠道支付准备、重复支付防护、医保退款等关键机制。

## 触发条件

- 患者下单购买药品/服务/会诊/预约等
- 前置条件：订单来源类型明确、订单金额已计算
- 所属模块：`ykj-service-order`

## 步骤详解

### 第一步：创建订单

系统接收创建订单指令后，执行以下操作：

**订单号生成**：
- 格式：O + yyyyMMdd + 7位随机数
- 例如：O202607021234567

**0元订单特殊处理**：
- 总金额为0时，订单自动跳过支付环节
- 处方订单（来源类型=PRESCRIPTION）→ 状态直接设为「已支付」
- 待发货订单（来源类型=待发货）→ 状态直接设为「待发货」
- 其他0元订单 → 状态直接设为「已完成」

**测试环境保护**：
- 测试模式下，非0元非医保订单自动将实付金额调整为0.01元

**订单过期时间**：
- 自动计算订单过期时间（默认24小时）
- 过期后订单将自动失效

**关联数据写入**：
- 订单商品明细（OrderItems）
- 收货地址（ReceiverAddress）
- 收费项目明细（OrderCostItems）

最后发布「订单创建完成事件」，通知其他模块（如推送通知、超时监控）。

### 第二步：支付准备（收银台）

患者进入收银台时，系统准备支付信息：

**支付前校验**：
1. 订单状态检查：已支付的订单返回错误"当前订单已支付！"
2. 订单有效性检查：非待支付状态返回错误"订单已失效！"
3. **历史渠道查单（防重复支付）**：查询该订单历史上所有等待回调的预支付单，按渠道逐个向第三方（微信/支付宝）查询真实支付状态，已支付则自动补单并拦截本次支付

**构建支付渠道**：
- 微信支付：默认支持，默认选中，排第一位
- 支付宝支付：默认支持
- 医保支付：需同时满足以下条件才显示
  - 非订金订单（deposit为空）
  - 系统开启医保支付功能
  - 订单来源类型支持医保
  - 通过医保支付校验（医生医保资质、患者定点等）

**计算支付金额**：
- 支付金额 = 订单总金额 - 已支付金额
- 如果已有部分支付（如订金），只需支付剩余部分

### 第三步：订单取消

**取消前校验**：
- 只有特定状态允许取消（待支付、已支付、已完成等）
- 已完成状态仅限问诊和预约类订单可取消
- 处方订单可选择通知第三方取消

**退款处理**：
- 待支付订单取消 → 直接取消，无需退款
- 已支付订单取消 → 自动创建退款申请单，订单状态变为「退款中」
- 支持自动退款（autoRefund）和手动退款两种模式
- 退款金额不能大于实际支付金额

**超时取消**：
- timeoutCancel=true时，状态设为「已过期」

取消完成后发布「订单取消事件」，通知关联模块同步更新状态。

### 第四步：自动签收

**15天自动签收**：
- 定时任务每天扫描发货超过15天且未签收的处方订单
- 自动将订单状态改为「已完成」
- 记录签收时间
- 每次处理最多100条

**取货码核销**：
- 支持线下通过取货码核销
- 校验取货码正确性和订单状态
- 核销后记录核销人信息

## 状态流转

```
待支付(PENDING_PAYMENT) ──支付成功→ 已支付(PAYED) ──发货→ 待收货 ──签收→ 已完成(COMPLETED)
    │                       │
    ├──取消→ 已取消(CANCELED)    ├──申请退款→ 退款中(REFUNDING)
    ├──超时→ 已过期(EXPIRED)     │
    └──0元→ 跳过支付 → 已完成    └──退款成功→ 已退款
```

## 异常处理

| 情况 | 系统处理方式 |
|-----|------------|
| 重复支付 | 历史预支付单查单拦截，抛出错误"当前订单已支付！" |
| 订单已失效 | 返回错误"订单已失效！" |
| 订单状态不可取消 | 返回错误"订单取消失败，当前状态不可取消" |
| 退款金额超过支付金额 | 返回错误"申请退款金额不能大于实际支付金额" |
| 医保退款失败 | 记录失败信息，支持重新退款 |
| 订单生成失败 | 抛出内部错误"订单生成失败,请联系管理员" |

## 关联功能

- 订单创建 → 支付（收银台）
- 订单创建 → 超时监控（定时任务）
- 订单取消 → 退款申请 → 退款审核
- 订单发货 → 物流跟踪 → 自动签收
- 订单创建 → 事件通知（IM消息、订阅消息）
