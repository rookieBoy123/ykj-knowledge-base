# 会诊电子签章服务流程

> 文档版本 2026-07-02 | 面向读者：产品、技术

## 概述

会诊各环节需要医生进行电子签名以确保诊疗意见的真实性和法律效力。系统支持五种签名类型和两种签名方式（CA数字证书签名和普通手写签名），在会诊的不同阶段由不同角色的医生完成签名操作。

## 触发条件

- **业务场景**：会诊申请、审核、受邀确认、意见提交、报告生成等环节
- **关联模块**：ykj-service-consultation（会诊）、ykj-service-ca（CA签章服务）

## 签名类型与权限

| 签名类型 | 标识 | 签名人 | 签名时机 |
|---------|------|--------|---------|
| 申请签名 | APPLICATION | 经治医生（主治医师） | 提交会诊申请时 |
| 审核签名 | REVIEW | 审核医生 | 审核会诊申请时 |
| 受邀签名 | INVITATION | 受邀医生 | 确认参加会诊时 |
| 意见签名 | OPINION | 受邀医生 | 提交会诊意见时 |
| 报告签名 | REPORT | 经治医生 | 会诊报告完成时 |

> **权限校验**：每种签名类型都有严格的医生身份校验。非主治医生无法签署申请签名和报告签名；非受邀医生无法签署受邀签名和意见签名。

## 步骤详解

### 一、发起签名

1. 医生端触发签名请求，传入会诊单号、医生ID、签名类型、签名图片
2. 系统校验该医生是否有权进行当前类型的签名
3. 查询已有的未完成签名记录（多次点击会生成多条），自动撤销旧签名
4. 组装会诊记录数据（患者信息、诊断、会诊目的、医生列表等）

### 二、CA数字证书签名

如果医院启用了CA（电子认证服务）：

1. **创建签章记录**：生成签章编号（signNo），状态为「待签署」
2. **生成PDF**：将患者信息、诊断、医生签名等数据用模板渲染为PDF文件（Base64编码）
3. **创建CA签章对象**：
   - 患者信息：姓名、身份证号、性别、年龄
   - 签章人信息：医生CA用户编号
   - 签名位置关键词（根据签名类型不同）：
     - 申请签名 → 「主管医师签名:XXX」
     - 审核签名 → 「审核医师签名:XXX」
     - 受邀签名 → 「确认参加签名:XXX」
     - 意见签名 → 「会诊医师签名:XXX」
     - 报告签名 → 「会诊医师签名:XXX」
4. **调用CA服务**：将PDF和签章参数发送给CA平台创建签署
5. **返回结果**：包含uniqueId（CA唯一标识）和签名状态

### 三、普通手写签名

如果医院未启用CA，使用普通手写签名：

1. 校验签名图片是否为空（为空则跳过签名）
2. 生成PDF模板（包含所有医生签名图片和时间）
3. 直接标记签名为「已接受」状态
4. 保存签章记录（含签名图片、签署时间）

### 四、PDF生成

签名过程中自动生成会诊记录PDF，包含以下信息：
- 患者基本信息（姓名、性别、年龄）
- 病情诊断和会诊目的
- 经治医生签名及时间
- 审核医生签名及时间
- 受邀医生列表及各自签名
- 会诊意见签名
- 治疗方案

### 五、签名完成回调

签名完成后：
1. 更新签章记录状态为「已完成」
2. 根据签名类型触发对应的会诊流程推进
3. 写入会诊日志

## 超时与时效

| 场景 | 时限 | 说明 |
|------|------|------|
| 受邀医生超时未签署 | 12小时 | 会诊创建后12小时内未签署则超时 |
| 超时未审核 | 开始前30分钟 | 会诊开始前30分钟仍未审核则超时 |

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 非主治医生签申请签名 | 抛异常「非主治医生无法签字」 |
| 非受邀医生签受邀签名 | 抛异常「非受邀医生无法签字」 |
| 非受邀医生签意见签名 | 抛异常「非医生意见提交无法签字」 |
| 非主治医生签报告签名 | 抛异常「非主治医生无法签字」 |
| 创建签署记录失败 | 抛异常「创建签署记录失败，请联系管理员」 |
| 未知签名类型 | 抛异常「未知的会诊签名类型」 |
| 多次点击生成多条签名 | 自动撤销旧签名记录再创建新的 |

## 关联功能

- **会诊服务**（ConsultationServiceImpl）：会诊流程管理
- **CA服务**（ykj-service-ca）：提供数字证书签章能力
- **医生服务**（ykj-service-hospital）：获取医生信息和CA认证状态
- **患者服务**：获取患者身份信息用于签章
