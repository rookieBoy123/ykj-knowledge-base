# 疾病知识管理流程（服务端）

> 文档版本 2026-07-05 | 面向读者：产品/技术 | 服务模块：ykj-service-hospital | 表：wiki_disease_knowledge

## 概述

服务端提供疾病知识条目（文章中医学疾病科普内容）的增删改查接口，分为管理后台接口（CRUD+批量删除）和小程序接口（按分类分页查询）。管理后台侧受权限控制，所有操作经过字典分类有效性校验和疾病名称重名校验。

## 触发条件

- **管理后台接口**：需要 `hosp:wiki:disease-knowledge:read`（查看）或 `hosp:wiki:disease-knowledge:write`（增删改）权限
- **小程序接口**：患者端无需登录即可查阅（公开接口）

## 步骤详解

### 第一步：管理后台新增疾病知识

管理后台提交新增请求（POST `/mgmt/wiki/disease-knowledge`），服务端执行：

1. **名称重名校验**：查询是否已有同名疾病，如果存在则返回提示「疾病名称已存在」
2. **字典校验**：检查提交的「疾病分类」编码是否在字典 `jbzs-jbfl` 中存在，如果不存在则提示「字典:jbzs-jbfl, 编码:xxx不存在」
3. 数据写入 `wiki_disease_knowledge` 表
4. 返回新增的疾病知识记录（含自增ID）

### 第二步：管理后台编辑疾病知识

管理后台提交更新请求（PUT `/mgmt/wiki/disease-knowledge/{id}`），服务端执行：

1. 根据ID查询现有记录，不存在则返回「疾病知识不存在」
2. **名称重名校验**：检查修改后的名称是否与其他记录重复（排除自身ID）
3. **字典校验**：验证分类编码有效性
4. 更新数据库记录并返回更新后的数据

### 第三步：管理后台删除

- **单个删除**（DELETE `/mgmt/wiki/disease-knowledge/{id}`）：直接按ID删除
- **批量删除**（POST `/mgmt/wiki/disease-knowledge/_batch-delete`）：接收ID数组，调用 `removeByIds` 批量删除。如果传入空数组，返回提示「请正确选择要删除的药膳食疗」（注：此错误提示文案不匹配，实际应为「疾病知识」）

### 第五步：查询

**分页查询**（管理后台，POST `/mgmt/wiki/disease-knowledge/_page`）：
- 支持按「疾病名称」模糊搜索
- 支持按「疾病分类」精确筛选
- 按创建时间倒序排列

**详情查询**（管理后台，GET `/mgmt/wiki/disease-knowledge/{id}`）：按ID返回完整疾病知识。

**小程序端分页查询**（`findPageForApp`）：返回精简版数据（`DiseaseKnowledgeAppResponse`），仅包含小程序需要展示的字段。

## 异常处理

| 异常 | 抛出类型 | 说明 |
|------|---------|------|
| 疾病名称已存在 | `BizException.badRequest("疾病名称已存在")` | 新增/编辑时名称重复 |
| 疾病分类编码无效 | `BizException.badRequest("字典:jbzs-jbfl, 编码:xxx不存在")` | 分类不在字典范围内 |
| 疾病知识不存在 | `BizException.badRequest("疾病知识不存在")` | 编辑或查询时ID无效 |
| 新增失败 | `BizException.badRequest("新增疾病知识失败")` | 数据库写入失败 |
| 更新失败 | `BizException.badRequest("更新疾病知识失败")` | 数据库更新失败 |
| 批量删除ID为空 | `BizException.badRequest("请正确选择要删除的药膳食疗")` | 前端未勾选任何记录 |

## 数据模型

**wiki_disease_knowledge 表（核心字段）：**

| 字段 | 说明 |
|------|------|
| id | 自增主键 |
| name | 疾病名称（最长20字，不可重名） |
| classification | 疾病分类编码（字典：jbzs-jbfl） |
| details | 疾病详情（富文本HTML） |
| create_time | 创建时间 |
| update_time | 更新时间 |

## 关联功能

- **中药图谱（HerbIllustrations）**：同属医学百科体系，共享字典校验 + 重名校验模式
- **数据字典**：疾病分类依赖 `jbzs-jbfl` 字典维护
- **小程序展示**：`ApiDiseaseKnowledgeController` 将数据转换为 `DiseaseKnowledgeAppResponse` 返回给患者端
- **管理后台**：`MgmtDiseaseKnowledgeController` 对应管理后台的疾病知识页面
