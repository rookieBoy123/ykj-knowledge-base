# 退款审核与执行流程（服务端）

> 文档版本：v1.0 | 面向读者：产品经理、运营、客服

## 概述

退款系统负责处理所有订单（问诊、药品、会诊等）的退款申请。当患者申请退款后，系统自动创建退款申请单进入审核流程；管理员审核通过后系统调用支付渠道（微信/支付宝）执行原路退款，退款成功后自动通知关联模块更新订单状态。

## 触发条件

- **触发来源**：患者前端申请退款、系统自动退款（如超时取消）或管理后台手动创建退款
- **前置条件**：订单必须处于可退款的状态

## 核心流程

### 第一步：创建退款申请

当需要退款时（患者申请或系统自动触发），系统发布退款申请事件，由 `RefundApplyServiceImpl` 监听处理：

1. **防重复检查**：同一笔订单的同一支付流水（微信交易号）只允许创建一条退款申请。如果已存在则拒绝："该订单已提交过退款信息"
2. **生成退款单号**：格式为 `Refund` + 日期（yyyymmdd）+ 7位随机数，如重复则递归重新生成
3. **初始状态**：
   - 审核状态 = 待审核
   - 退款状态 = 未退款
4. **自动退款**：如果退款事件标记了"自动退款"，创建申请后立即自动审核通过并执行退款

### 第二步：审核退款申请

管理后台通过 `/mgmt/order/refunds/_agree` 和 `/mgmt/order/refunds/_reject` 接口审核（需 `order:refunds:audit` 权限）：

**同意退款**：
- 校验申请单必须处于"待审核"状态，否则拒绝："申请单审核失败, 申请单不存在或状态不允许审核, 请联系管理员"
- 选择退款方式：原路退回 或 人工退款
  - **原路退回**：审核通过后立即调用支付渠道执行退款（微信WxRefundService / 支付宝AlipayPaymentService）
  - **人工退款**：审核通过后标记为"已退款"，不调用支付渠道（适用于线下退款场景）
- 发布退款审核通过事件，通知其他模块（如订单模块更新订单状态）

**拒绝退款**：
- 必须填写备注（拒绝原因），否则拒绝操作："请填写备注"
- 审核状态变为"审核不通过"
- 发布退款审核不通过事件

### 第三步：执行原路退款

审核通过且选择"原路退回"时，系统自动执行退款：

1. **状态校验**：
   - 审核状态必须是"审核通过"
   - 退款状态必须是"未退款"或"退款失败"（允许重试）
   - 退款方式必须是"原路退回"
2. **渠道识别**：优先使用退款申请单中记录的支付渠道，兼容跨渠道重复支付场景；如未记录则查询最新收入流水确定渠道
3. **调用支付接口**：根据渠道调用微信退款或支付宝退款
4. **更新状态**：
   - 调用成功后：退款状态 → "退款中"
   - 调用失败：退款状态 → "退款失败"，记录失败原因

### 第四步：退款重试

**自动重试**：退款失败后可重新发起（支持退款状态为"未退款"和"退款失败"的申请单）：

- 管理后台通过 `/mgmt/order/refunds/_retry` 接口触发
- 系统重新调用 doRefund 流程

**人工退款**：对于多次原路退款失败的情况，管理员可通过 `/mgmt/order/refunds/_manual-refund` 接口进行人工退款：
- 要求申请单审核状态为"审核通过"且退款状态为"退款失败"
- 直接标记为"已退款"，不调支付渠道
- 发布退款方式变更事件

### 第五步：退款完成回调

微信/支付宝退款成功后回调通知：

1. 根据退款单号找到退款申请单，更新退款状态为"已退款"，记录退款时间
2. 解析支付渠道和退款金额（分转元）
3. 发布退款成功事件，通知订单模块等关联模块更新订单状态

## 状态流转

```
退款申请创建 → 待审核 → 审核通过 → 执行退款 → 退款中 → 退款成功(已退款)
                    ↘ 审核不通过          ↘ 退款失败 → 重试/人工退款
```

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 同一订单+流水号重复申请退款 | 提示"该订单已提交过退款信息" |
| 申请单不存在或不在待审核状态时审核 | 提示"申请单审核失败, 申请单不存在或状态不允许审核" |
| 拒绝退款时未填备注 | 提示"请填写备注" |
| 同意的申请单非原路退回方式却执行了doRefund | 提示"当前退款申请单非原路退回申请单" |
| 退款调用失败 | 退款状态变为"退款失败"，记录失败原因 |
| 人工退款时状态不符合要求 | 提示"未找到退款申请单或申请单状态异常, 无法退款" |
| 自动退款失败 | 记录错误日志"自动退款失败, 请管理员关注" |
| 退款回调收到无效退款单号 | 提示"无效的退款单号" |
| 退款申请单并发更新（乐观锁） | 记录警告日志，不阻塞流程 |

## 关联功能

- **订单模块**（ykj-service-order）：退款与订单状态联动，退款完成后更新订单
- **支付服务**：微信退款（WxRefundService）和支付宝退款（AlipayPaymentService）
- **事件总线**：退款各环节通过事件通知其他模块（订单、流水等）
