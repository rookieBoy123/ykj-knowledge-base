# 会诊全生命周期流程

> 文档版本：2026-07-02 | 面向读者：产品/运营/实施

## 概述

会诊是多个医生围绕同一个患者共同讨论病情并给出诊疗建议的协作功能。从医生提交申请、管理员审核、患者确认支付、医生会诊讨论，到最终提交报告和诊疗方案，形成一个完整的会诊生命周期。系统通过多种定时任务自动处理超时场景。本流程覆盖 ykj-service-consultation 模块的后端核心逻辑。

## 状态机

```
草稿(DRAFT)→申请(APPLY)→已通过(WAIT_PAITIENT_CONFIRM)→待开始(WAIT_START)→进行中(HANDING)→待报告(WAIT_REPORT)→已完成(FINISH)
                 ↓拒绝              ↓超时未支付              ↓会诊开始时间到达          ↓关闭会议室
              已完成(REJECTED)    已完成(PAY_TIMEOUT)       (系统自动推进)           12小时后提醒写报告
                 ↓取消申请
              已完成(CANCEL)
                 ↓审核超时
              已完成(APPROVE_TIMEOUT)
                 ↓签名超时
              已完成(UNSIGNED_TIMEOUT)
```

## 步骤详解

### 一、创建与提交申请

医生在问诊过程中为患者发起会诊申请：

1. **保存草稿**（`saveInfo`）：生成会诊单号（格式：`HZ + 日期时间 + 3位随机数`），状态为"草稿"（DRAFT）
2. **提交申请**（`updateInfo`）：状态变为"申请"（APPLY），进入待审核状态
3. **草稿删除**（`delete`）：仅草稿状态可删除，非草稿状态提示"非草稿状态无法删除!"
4. **撤销申请**（`cancelApply`）：仅未生效的会诊可撤销（`effectStatus != EFFECTED`），已撤销的不能重复操作

### 二、会诊管理员审核

会诊管理员在管理后台审核会诊申请：

**通过审核**（`approved`）：
- 校验审核人必须是会诊管理员（`isConsultAdmin=true`），否则提示"您不是会诊管理员, 无法审核会诊"
- 校验当前会诊必须是"待审核"状态
- 校验所有参会医生均已开通会诊服务（`ServiceType.Consultation`），未开通则提示"XX医生暂未开通会诊功能"
- 累加所有医生的会诊费得到总会诊费
- 保存会诊医生信息（医生ID、姓名、职称、科室、医院、头像）
- 设置审核时间、审核人签名
- 记录流程日志
- **启动超时未支付定时任务**：在会诊开始时间前15分钟触发

**拒绝审核**（`reject`）：
- 必须填写拒绝原因（`rejectReason`）
- 状态变为"已完成"（FINISH），结束标记为"已拒绝"（REJECTED）
- 发送订阅消息通知申请医生审核结果
- 发布会诊完成事件

### 三、患者确认与支付

审核通过后，系统为患者创建会诊订单（`saveOrder`）：
- 订单来源类型：`CONSULTATION`
- 费用明细：会诊费用（金额 = 所有参会医生会诊费之和）
- 订单到期时间：会诊开始时间
- 如果会诊费为0元，直接进入"待开始"状态（跳过支付）
- 支付成功后状态变为"待开始"（WAIT_START），发送系统消息给医生和患者

### 四、会诊前提醒

支付成功后系统设置两个定时任务：

1. **会诊前10分钟提醒**（`sendReminderBeforeConsultation`）：给经治医生和患者发送系统消息 + 订阅消息提醒
2. **会诊开始处理**（`handleStartMeeting`）：到达会诊开始时间时，自动将状态从"待开始"推进为"进行中"（HANDING）

### 五、会诊进行中

- 医生可以**手动开始会诊**（`startConsultation`）：只能由经治医生操作，校验状态必须是"待开始"
- **邀请患者**（`invitePatient`）：设置患者可进入会诊室标识，发送订阅消息通知患者
- **移除患者**（`removePatient`）：将会诊患者移出音视频房间和IM群聊，保存移除状态，发送系统消息

### 六、结束会诊与会诊报告

**关闭会议室**（`closeMeetingRoom`）：
- 校验状态必须是"进行中"
- 记录会议结束时间和持续时长
- 解散音视频房间和IM群聊
- 状态变为"待报告"（WAIT_REPORT）
- 设置12小时后提醒未提交报告的医生

**提交会诊报告**（`submitReport`）：
- 已结束的会诊不允许提交报告

**提交诊疗方案**（`submitTreatmentPlan`）：
- 仅"待报告"状态允许提交
- 由经治医生汇总各方意见后填写

### 七、超时自动处理

系统通过定时任务自动处理三种超时场景：

| 超时类型 | 触发条件 | 自动处理 |
|---------|---------|---------|
| 支付超时 | 会诊开始前15分钟仍未支付 | 状态→已完成，结束标记→PAY_TIMEOUT，取消关联订单 |
| 审核超时 | 会诊开始前30分钟仍为"申请"状态 | 状态→已完成，结束标记→APPROVE_TIMEOUT |
| 签名超时 | 待审核状态下医生超时未签名 | 状态→已完成，结束标记→UNSIGNED_TIMEOUT，发送消息通知 |

### 八、收益计算

会诊完成后（`calculationIncome`）：
- 需开启佣金功能（`consultationProperties.commissionEnabled`）
- 每位参会医生的收益 = 医生会诊费 × (100 - 提成比例) / 100
- 收益记录类型标记为"服务收入"（IncomeType.Service）
- 收入记录描述为"会诊费"

## 异常处理

| 错误提示 | 触发条件 |
|---------|---------|
| "当前会诊不是待审核状态!" | 审核时状态不是WAIT_APPROVE |
| "当前会诊不是待审核状态!"（拒绝时） | 拒绝时状态不是APPLY |
| "XX医生暂未开通会诊功能" | 参会医生未开通会诊服务 |
| "您不是会诊管理员, 无法审核会诊" | 审核人没有会诊管理员权限 |
| "会诊已生效，不能撤销!" | 患者已支付的会诊不能撤销 |
| "会诊已撤销，不能重复操作!" | 重复撤销已撤销的会诊 |
| "非草稿状态无法删除!" | 非草稿状态删除操作 |
| "当前会诊不是进行中状态!" | 关闭非进行中的会诊 |
| "当前会诊已结束，不能提交会诊报告!" | 已完成状态下提交报告 |
| "当前会诊状态不是待会诊报告状态!" | 非待报告状态提交诊疗方案 |
| "当前会诊状态不是待开始状态!" | 非待开始状态点击开始会诊 |
| "当前医生无权限开始会诊!" | 非经治医生尝试开始会诊 |

## 关联功能

- **问诊模块**（ykj-service-inquiry）：会诊依附于问诊单
- **订单模块**（ykj-service-order）：会诊费用走订单支付
- **IM/音视频**：会诊使用IM群聊和实时音视频房间
- **通知服务**：微信订阅消息推送
- **收益模块**：会诊收益计入医生收入
