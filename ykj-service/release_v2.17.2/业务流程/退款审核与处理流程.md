# 退款审核与处理流程

> 文档版本：v1.0 | 最后更新：2026-07-01 | 面向读者：产品/运营

## 概述

当患者需要取消订单或申请退款时，系统自动生成退款申请单，交由管理员审核后执行退款操作。退款支持两种方式：原路退回（调用支付平台接口自动退款）和手动退款（管理员线下处理）。

---

## 触发条件

退款申请由以下场景自动触发：

- 患者主动取消订单（已支付状态）
- 医生取消问诊/拒绝接诊
- 问诊/会诊超时未处理自动取消
- 订单超时自动取消（需退款的情况）
- 预约服务取消（已付款）
- 健康服务包订单取消

---

## 退款申请单状态流转

```
待审核 → 审核通过（自动原路退回/手动退款）
       ↘ 审核拒绝
```

退款执行结果：
- 自动退款成功 → 退款完成
- 自动退款失败 → 退款失败（可重试）
- 手动退款 → 退款完成（管理员确认）

---

## 步骤详解

### 第一步：退款申请创建

当订单取消时，系统检查是否需要生成退款申请：

1. 订单状态不是「待支付」（未付款不需要退款）
2. 订单存在支付流水记录
3. 同一笔支付流水只允许创建一条退款申请（防重复）

满足条件后，系统自动创建退款申请单，状态为「待审核」，并记录：
- 退款金额（不超过实际支付金额）
- 退款原因
- 关联的订单号和微信交易单号
- 退款类型（暂未确定，审核时指定）

### 第二步：管理员审核

管理员在后台看到待审核的退款申请后，可选择：

**同意退款**：
- 选择退款类型：
  - **原路退回**：系统调用微信/支付宝退款接口，自动退回原支付账户
  - **手动退款**：管理员线下转账后手动标记退款完成
- 填写审核备注
- 对于原路退回，审核通过后立即执行退款动作

**拒绝退款**：
- 填写拒绝原因（必填）
- 申请单状态变为「审核拒绝」
- 系统不再执行退款

### 第三步：退款执行

**原路退回（自动）**：
- 系统调用支付平台退款接口
- 退款金额可小于或等于原支付金额（支持部分退款）
- 退款成功后更新申请单状态和退款时间
- 插入支出流水记录（标记退款流水）

**手动退款**：
- 审核通过时直接标记退款状态为「已退款」
- 系统不调用支付接口

### 第四步：退款结果通知

- 退款成功：系统发布退款完成事件，通知前端更新订单状态
- 退款失败：系统记录失败原因，允许管理员重新发起退款

### 第五步：医保退款（特殊流程）

如果订单是医保支付，退款流程有所不同：

1. 先调用医保退款接口，退回医保统筹和个人账户部分
2. 医保退款成功后，再调用微信退款接口退回自费部分
3. 两步都成功后，才标记退款完成
4. 医保已退款的订单，再次退款时跳过医保退款步骤

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 同一订单重复提交退款 | 提示"该订单已提交过退款信息" |
| 退款金额大于支付金额 | 提示"申请退款金额不能大于实际支付金额" |
| 申请单状态不允许审核 | 提示"申请单不存在或状态不允许审核" |
| 拒绝退款时未填备注 | 提示"请填写备注" |
| 微信退款接口失败 | 记录失败原因，申请单标记为退款失败，允许重试 |
| 医保退款失败 | 不继续微信退款，返回失败状态 |
| 订单未找到支付流水 | 提示"未找到订单支付信息，无法退款" |
| 待支付订单申请退款 | 提示"订单未成功付款，不支持申请退还金额" |

## 关联功能

- **订单管理**：退款是订单取消流程的一部分
- **财务管理**：退款金额在收款管理中体现为支出
- **微信/支付宝对接**：退款接口调用依赖支付平台
- **医保系统对接**：医保退款需与医保平台交互
