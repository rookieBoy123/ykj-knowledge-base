# 小程序页面装修管理流程（服务端）

> 文档版本 2026-07-01 | 面向读者：系统架构师、后端开发

## 概述

管理后台的"页面装修"功能允许运营人员通过拖拽组件的方式自定义小程序页面布局。服务端提供页面分组管理、页面创建/编辑/发布/预览等核心能力。患者端通过API获取已发布使用的页面数据进行渲染。

## 涉及服务模块

- `ykj-service-system` — 页面装修核心逻辑（DiyPage / DiyGroup）

## 核心实体

- **DiyPage**：页面实体，存储页面标题、类型（首页/微页面/个人中心等）、背景配置、JSON格式的组件配置数据、预览二维码、使用状态
- **DiyGroup**：页面分组实体，管理底部导航栏的分组配置

## 业务链路

### 一、页面分组管理（底部分组）

底部分组（BOTTOM_GROUP）是管理底部导航栏的核心数据：

- **新增底部分组**：管理员创建默认的底部分组，系统自动填充名称"底部分组"和分组编码
- **查询底部分组**：按编码（GROUP_CODE_BOTTOM）查询当前使用的底部分组配置
- **更新底部分组**：修改底部导航栏的菜单项、图标、跳转链接等配置
- **权限控制**：查询需 `diy:group:read` 权限，修改需 `diy:group:write` 权限

### 二、页面管理（CRUD）

页面管理的完整接口能力：

| 操作 | 接口 | 权限 | 说明 |
|------|------|------|------|
| 新增页面 | POST /mgmt/diy/page | diy:page:write | 创建自定义页面 |
| 编辑页面 | PUT /mgmt/diy/page/{id} | diy:page:write | 更新页面配置 |
| 查询页面 | GET /mgmt/diy/page/{id} | diy:page:read | 查看页面详情 |
| 分页列表 | GET /mgmt/diy/page | diy:page:read | 按类型等条件分页查询 |
| 删除页面 | DELETE /mgmt/diy/page/{id} | diy:page:write | 删除页面 |
| 使用页面 | POST /mgmt/diy/page/{id}/_use-page | diy:page:write | 将页面设为"当前使用" |
| 重置预览码 | POST /mgmt/diy/page/{id}/_reset-preview-qr-code | diy:page:write | 重新生成预览二维码 |

### 三、页面发布与使用

- **"使用页面"操作**：将某个页面标记为当前使用中（DiyPageType区分首页/微页面/个人中心等），同一类型只能有一个页面在使用状态
- **预览二维码**：每个页面可生成微信小程序预览二维码，运营人员扫码可在手机端预览效果，支持重置（重新生成）
- **类型枚举**：DiyPageType 定义页面类型的分类（如首页、微页面、个人中心等）

### 四、患者端查询

患者端API（`/api/diy/page`）：

- **按类型查询**：`GET /api/diy/page?type={type}` — 获取指定类型当前正在使用的页面（如首页、个人中心）
- **按ID查询**：`GET /api/diy/page/{id}` — 通过微页面ID获取指定页面的完整配置（如扫码进入微页面时使用）
- 返回的页面数据包含完整的JSON格式组件配置（value字段），前端按照组件类型逐一渲染

## 关键逻辑

1. **页面JSON组件排序**：页面中组件按key字母排序后渲染（formatJSON函数），保证展示顺序稳定
2. **页面类型唯一性**：同一类型的页面只能有一个设为"使用中"
3. **预览二维码生成**：调用微信小程序码生成接口，生成带页面ID场景值的太阳码

## 状态流转

```
创建页面 → 编辑完善 → 预览（扫码确认效果）→ 发布使用（设为当前使用）
                                                    ↓
                                              可随时替换为其他页面
```

## 异常处理

| 情况 | 系统处理 |
|------|---------|
| 页面类型不存在 | DiyPageType.valueOf() 抛出异常 |
| 无权限操作 | @PreAuthorize 拦截返回403 |
| 页面已被删除 | 查询返回null |
| 预览二维码生成失败 | 返回错误信息 |
