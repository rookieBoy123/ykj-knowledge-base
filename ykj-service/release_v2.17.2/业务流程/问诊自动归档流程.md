# 问诊自动归档流程

> 文档版本 v1.0 | 面向读者：产品经理

## 概述

问诊结束后，系统自动将问诊过程中产生的所有医疗资料（病历、处方、聊天记录、视频）统一打包归档到云存储（OSS）。管理员可在后台查看归档记录，归档失败的记录可手动重试。

## 触发条件

- **自动触发**：
  - 问诊结束时（`InquiryFinishedEvent` 事件）
  - 药师审方完成时（`PrescriptionAuditEvent` 事件）
- **手动触发**：管理员在后台对归档失败记录点击"手动归档"
- **前置条件**：问诊单状态为"已完成"（`FLOW_STATUS_FINISH`）、同一问诊下无未审核完成的处方

## 步骤详解

### 第一步：事件触发与可归档校验

当问诊结束或药师审方完成时，系统接收到事件通知后执行校验：

1. **查询问诊单**：获取问诊单完整信息
2. **状态校验**（`canArchiveVerify` 方法）：
   - 问诊单必须存在且状态不为空
   - 问诊单状态必须为"已完成"（`FLOW_STATUS_FINISH`）
   - 如果有处方：同一问诊下所有处方必须已完成审核（通过了/被驳回/已过期，即不存在"审核中"状态的处方）
3. **校验不通过**：记录日志"问诊单可归档校验不通过，不处理归档"，不创建归档记录
4. **校验通过**：进入下一步

### 第二步：创建归档记录

1. **组装归档信息**：
   - 查询医生信息（所属科室、医生姓名）
   - 查询问诊患者信息（姓名、身份证号）
2. **生成归档目录路径**：
   - 格式：`系统配置的归档根目录 / 患者姓名(证件号后4位) / 问诊单号(问诊日期) /`
   - 例如：`/archive/张三(1234)/INQ20231201001(2023-12-01)/`
3. **设置归档类型**：自动归档（Automatic）
4. **设置初始状态**：待处理（HaveInHand）
5. **保存归档记录**

### 第三步：自动归档执行

定时任务 `autoArchive` 每隔一段时间扫描"待处理"状态的归档记录（每次取100条），逐一处理：

#### 子步骤 3.1：归档病历记录

1. 查询该问诊的病历详情（含患者信息、主诉、病史、诊断等）
2. 生成病历图片（格式为 JPG，1200×1024 像素）：
   - 使用 Beetl 模板引擎，将病历数据渲染为 HTML，再转换为图片
   - 图片内容包括：问诊单号、就诊时间、患者姓名/性别/年龄、监护人信息、科室、服务类型、诊断（包括辨病/辨证）
3. 上传图片到 OSS：文件名为 `门诊病历-问诊单号.jpg`

#### 子步骤 3.2：归档电子处方

1. 查询该问诊下所有药师审核通过的处方
2. 逐张处方转储：
   - 从原存储位置获取处方 PDF 文件
   - 中药处方命名为 `中药处方-门诊号.pdf`
   - 西药处方命名为 `西药处方-门诊号.pdf`
   - 上传到 OSS 归档目录
3. 如果没有审核通过的处方：记录日志"不处理处方归档"，跳过

#### 子步骤 3.3：归档图文聊天记录

1. 查询该问诊 IM 群组的所有历史消息
2. 将全部聊天记录序列化为 JSON 格式
3. 文件命名为 `图文资料-问诊单号.json`
4. 上传到 OSS 归档目录

#### 子步骤 3.4：归档视频资料

1. 根据问诊单号查询所有视频记录（TRTC 视频房间录制的回放文件）
2. 逐条转储视频文件：
   - 文件命名为 `视频资料-医生ID@序号.mp4`
   - 从原存储位置转移到 OSS 归档目录
3. 如果没有视频记录：记录日志"不处理视频归档"，跳过

#### 子步骤 3.5：更新归档状态

所有子步骤完成后：
- **成功**：归档状态更新为"已归档"（archiveSuccess）
- **任一子步骤异常**：归档状态更新为"失败"（archiveFail），记录异常日志

### 第四步：管理员查看与手动重试

1. **归档列表查询**（MgmtArchiveLogController）：
   - 支持按问诊单号模糊搜索（右模糊）
   - 支持按患者姓名模糊搜索（右模糊）
   - 支持按归档状态筛选
   - 按归档时间倒序排列

2. **手动重试归档**（manualUpdateArchiving）：
   - 点击"手动归档"按钮触发
   - 状态校验：只有"失败"状态（ArchiveStatus.Fail）的记录才允许手动重试
   - 如果是"正在归档"或"已归档"状态，提示"归档记录正在归档中或者已归档, 不能手动归档"
   - 重试后会重新进入待处理队列，由定时任务再次执行

## 异常处理

| 情况 | 系统处理方式 |
|------|------------|
| 问诊单不存在或状态异常 | 记录日志，不创建归档记录 |
| 问诊单未结束 | 记录日志，不创建归档记录 |
| 同一问诊下有未审核完的处方 | 等待审方完成事件触发后再归档 |
| 归档过程中任一文件上传失败 | 记录为"失败"状态，不中断整个流程，后续可手动重试 |
| 问诊无病历记录 | 跳过病历归档，日志记录"不处理病历归档" |
| 问诊无处方记录 | 跳过处方归档，日志记录"不处理处方归档" |
| 问诊无视频记录 | 跳过视频归档，日志记录"不处理视频归档" |
| 归档记录不存在 | 返回"归档记录不存在" |
| 非失败状态的记录手动归档 | 返回"归档记录正在归档中或者已归档, 不能手动归档" |
| 定时任务每次取100条 | 超过100条的分批次处理，下次定时触发时继续 |

## 关联功能

- **问诊生命周期管理**：问诊结束触发归档事件（ykj-service-inquiry）
- **处方审核**：审方完成触发归档事件（ykj-service-inquiry）
- **电子病历管理**：归档病历记录（ykj-service-inquiry）
- **即时通讯（IM）**：归档聊天记录（ykj-service-im）
- **音视频通话（TRTC）**：归档视频回放文件（ykj-service-rtc）
- **云存储（OSS）**：归档文件统一存储（ykj-oss-core）
