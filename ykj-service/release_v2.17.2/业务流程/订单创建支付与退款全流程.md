# 订单创建、支付与退款全流程

> 文档版本 v1.0 | 面向读者：产品/技术

## 概述

订单系统是整个平台的核心交易模块，负责从订单创建、支付、取消到退款的全生命周期管理。支持微信支付、支付宝、医保支付等多种支付渠道，并包含防重复支付、过期自动取消、自动收货等安全机制。

## 触发条件

- **触发来源**：患者在小程序端发起购买（药品下单、预约服务、问诊支付等）
- **订单来源类型**：处方订单(PRESCRIPTION)、问诊订单(INQUIRY)、预约订单(RESERVATION)等

## 流程详解

### 一、订单创建

**订单号生成**：格式为"O" + 日期(yyyyMMdd) + 7位随机数

**初始状态判定**：
- 正常订单 → `PENDING_PAYMENT`（待支付）
- 0元订单 → `COMPLETED`（已完成），同时设置支付时间
- 0元处方订单 → `PAYED`（已支付，不直接完成）
- 0元待发货订单 → `PENDING_SHIPMENT`（待发货）

**过期时间计算**：默认24小时过期，可通过配置调整。

**订单数据写入**：
1. 写入订单主表（order）
2. 写入订单商品表（order_item）
3. 写入收货地址表（如有）
4. 写入订单费用项（如有）
5. 发布"订单创建完成事件"，触发后续异步流程（如发货、通知等）

**测试环境处理**：开启测试支付时，订单金额改为0.01元。

### 二、支付准备（advance/prepay）

**支付前校验**：
- 如果已支付 → 返回错误码40001"当前订单已支付！"
- 如果不是待支付状态 → 返回错误码40002"订单已失效！"

**历史渠道防重复支付**：查询历史预支付单，按渠道逐一向微信/支付宝查询真实支付状态：
- 如已支付 → 补单（同步推进订单状态）+ 返回40001拦截
- 如查单失败（网络抖动）→ 按未支付放行（体验优先）

**支付渠道构建**：
- 默认支持微信支付（选中）、微信扫码支付、支付宝
- 如果开启医保支付且订单来源支持 → 增加医保支付选项

**支付金额计算**：订单总金额 - 已付金额（订金模式）

### 三、订单取消

**可取消状态**：仅特定状态可取消（CANCEL_ORDER_STATUS集合）

**取消分支**：
1. **第三方通知取消**（处方订单 + 启用第三方通知）→ 发布ThirdOrderCancelEvent事件，订单状态暂不变
2. **超时取消** → 状态变为 `EXPIRED`
3. **普通取消** → 状态变为 `CANCELED`

**退款申请生成**：取消时如果订单已支付且启用退款：
- 订单状态先变为 `REFUNDING`（退款中）
- 发布退款申请事件，异步处理退款流程

**取消后事件**：发布OrderCanceledEvent事件，通知关联模块（问诊、预约等）

### 四、退款申请

**前置校验**：
- 订单未成功付款 → "订单未成功付款，不支持申请退还金额"
- 未找到支付流水 → "未找到订单支付信息，无法退款，请联系管理员"
- 申请退款金额 > 实际支付金额 → "申请退款金额不能大于实际支付金额"

**退款流程**：发布订单退款申请事件，由退款模块异步处理。

### 五、自动签收（定时任务）

处方订单发货超过15天且未确认收货 → 自动签收：
- 设置确认收货标识
- 订单状态变更为 `COMPLETED`
- 发布收货通知事件

### 六、到店取货

**取货码验证**：
- 取货码有误 → "取货码有误"
- 订单已取货 → "该订单已取货"
- 取货成功 → 状态变更为 `COMPLETED`，记录取货人信息

### 七、医保退款

**流程**：
1. 获取微信医保授权码
2. 判断是否已医保退款
3. 未退款 → 调医保退款接口 → 退款成功更新标识 → 微信医保退款 → 插入退款流水
4. 已退款 → 直接微信医保退款 → 插入退款流水

### 八、会员绑定

- 订单未绑定会员 → 直接绑定
- 订单已绑定其他会员 → "订单已绑定会员"

### 订单状态枚举

```
PENDING_PAYMENT（待支付）
  ├→ PAYED（已支付）
  │   ├→ PENDING_SHIPMENT（待发货）
  │   │   ├→ SHIPPED_PENDING_RECEIPT（已发货待收货）
  │   │   │   └→ COMPLETED（已完成）
  │   │   └→ (自动签收15天)
  │   └→ REFUNDING（退款中）
  │       └→ REFUNDED（已退款）
  ├→ CANCELED（已取消）
  ├→ EXPIRED（已过期）
  └→ COMPLETED（0元订单直接完成）
```

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 订单生成失败 | 抛出"订单生成失败，请联系管理员"异常 |
| 重复支付检测 | 历史渠道查单+分布式锁，双重保障 |
| 查单放行（网络异常） | 按未支付放行，仅记录日志 |
| 订单号冲突 | 乐观锁更新失败会重试生成新订单号 |

## 关联模块

- **支付模块**（微信支付/支付宝）：支付回调处理
- **退款模块**（RefundApplyServiceImpl）：退款申请审核和退款执行
- **医保模块**（WXYBService）：微信医保支付和退款
- **物流模块**（LogisticsService）：发货和物流跟踪
- **处方模块**：处方订单来源，取消时通知第三方
