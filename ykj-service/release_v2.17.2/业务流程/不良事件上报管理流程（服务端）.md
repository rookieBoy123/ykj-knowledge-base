# 不良事件上报管理流程（服务端）

> 文档版本：2026-07-04 | 面向读者：产品经理、开发人员

## 概述

不良事件上报功能用于记录和追踪诊疗过程中出现的医疗不良事件。管理后台可对不良事件进行增删改查，每条不良事件关联固定患者和医生，并自动生成唯一业务编号以便追溯。

## 涉及模块

- `ykj-service-hospital`：医院模块 — 不良事件管理

## 系统架构

```
管理后台前端 → MgmtBadEventController
    POST   /mgmt/hospital/bad-event         新增不良事件
    PUT    /mgmt/hospital/bad-event/{id}     编辑不良事件
    GET    /mgmt/hospital/bad-event/{id}     查看不良事件详情
    GET    /mgmt/hospital/bad-event          分页查询不良事件
```

所有接口统一权限前缀：`hosp:bad-event`，具体权限码为：
- 新增：`hosp:bad-event:add`
- 编辑：`hosp:bad-event:edit`
- 查看详情：`hosp:bad-event:read`
- 分页查询：`hosp:bad-event:page`

## 核心流程

### 新增不良事件

1. 管理后台提交不良事件信息（患者、医生、事件描述、上报时间等）
2. 服务端校验必填字段：
   - **医生ID**：不能为空，否则返回"请正确选择医生"
   - **患者ID**：不能为空，否则返回"请正确选择患者"
3. 根据患者ID查询患者信息，自动填充患者姓名和身份证号
4. 根据医生ID查询医生信息，自动填充医生姓名、科室编号、科室名称
5. 自动生成业务编号（格式：`E + yyyyMMddHH + 两位随机数`，例如 `E202607041523`）
   - 生成规则：取上报时间的年月日时 + 01-99随机数
   - 如果编号已存在（极低概率），递归重新生成直到唯一
6. 保存到数据库

### 编辑不良事件

1. 管理后台提交编辑内容
2. 根据ID查找原记录，不存在则返回"编辑保存失败，记录不存在"
3. 只更新提交的字段（非全量更新），保留未修改的字段
4. 重新查询并更新关联的医生和患者信息（姓名、科室、身份证号等）

### 查看不良事件详情

1. 根据ID查询不良事件
2. ID为空时直接返回 null（不报错）

### 分页查询不良事件

1. 传入分页参数（页码、每页条数）和搜索条件
2. 返回分页结果

## 数据模型关键字段

| 字段 | 说明 |
|------|------|
| businessNo | 业务编号，自动生成，格式 E+年月日时+随机数 |
| doctorId | 关联医生ID |
| patientId | 关联患者ID |
| patientName | 患者姓名（自动填充） |
| idCard | 患者身份证号（自动填充） |
| doctorName | 医生姓名（自动填充） |
| departmentNo | 科室编号（自动填充） |
| departmentName | 科室名称（自动填充） |
| reportTime | 上报时间 |

## 异常处理

| 情况 | 处理方式 | 类型 |
|------|---------|------|
| 新增时未选择医生 | 返回"请正确选择医生" | 后端拦截 |
| 新增时未选择患者 | 返回"请正确选择患者" | 后端拦截 |
| 编辑时记录不存在 | 返回"编辑保存失败，记录不存在" | 后端拦截 |
| 查看详情时传入空ID | 返回 null（不报错） | 后端静默处理 |
| 业务编号冲突 | 递归重新生成直到唯一 | 后端自动处理 |
