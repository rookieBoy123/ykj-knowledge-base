# 管理员账号注册与密码管理流程

> 文档版本 2026-07-06 | 面向读者：系统管理员

## 概述

系统管理员通过管理后台创建和编辑后台操作用户账号，系统自动分配初始密码并在首次登录时强制修改。管理员可随时重置他人密码、修改自己的密码、变更绑定的手机号码，以及启用/禁用用户状态。

## 触发条件

- **入口路径**：管理后台 → 系统管理 → 用户管理 → 新增用户 / 编辑用户
- **前置条件**：操作者需拥有系统用户管理权限

## 步骤详解

### 第一步：注册新管理员账号

管理员在「用户管理」页面点击「新增用户」，填写用户名、真实姓名、手机号、邮箱等信息，并为该用户分配角色。

系统在注册时做以下校验：
- 检查手机号是否已被其他管理员注册。如果已存在，提示「该手机号码已经被注册过，请重新填写手机号码」
- 验证分配的角色是否合法
- 自动设置初始密码（系统默认密码或自定义密码）
- **密码立即过期**：注册后用户的密码过期时间设为当前时间，登录时系统会提示修改密码

注册成功后系统会向该用户的邮箱发送注册通知邮件。

### 第二步：首次登录修改密码

新注册的管理员首次登录时，由于密码已标记为过期，系统会引导进入密码修改页面：

- 输入旧密码（即初始密码）和新密码
- 如果**新密码与旧密码相同**，系统提示「新密码不能与旧密码相同」
- 如果**旧密码输入错误**，系统提示「原密码错误」
- 修改成功后，密码过期标记被清除，后续登录不再强制修改

### 第三步：管理员自行修改密码

管理员登录后可通过个人设置页面修改密码：

- 同样需要验证旧密码
- 旧密码错误时提示「原密码错误」
- 新密码不能与旧密码相同

### 第四步：管理员重置他人密码

拥有用户管理权限的管理员可以重置其他用户的密码：

- 在用户列表中找到目标用户，点击「重置密码」
- 系统将密码重置为系统默认密码
- 重置后密码标记为已过期（有效期3天），用户下次登录时强制修改
- 如果用户有邮箱，系统会发送密码重置通知邮件

### 第五步：变更绑定手机号

管理员可以变更用户绑定的手机号码：

- 输入新手机号码
- 如果手机号为空，提示「新手机号码不能为空」
- 如果新手机号已被其他用户绑定，提示「手机号码已有用户」
- 变更成功后，系统发布 `UserChangePhoneEvent` 事件，通知下游业务同步手机号变更

### 第六步：启用/禁用用户

- **禁用**：将用户状态设为「已禁用」，被禁用的用户无法登录
- **启用**：将用户状态恢复为「已启用」，用户可正常登录
- 状态变更后通知权限系统刷新缓存

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 手机号已被注册 | 提示「该手机号码已经被注册过」 |
| 新密码与旧密码相同 | 提示「新密码不能与旧密码相同」 |
| 旧密码输入错误 | 提示「原密码错误」 |
| 新手机号为空 | 提示「新手机号码不能为空」 |
| 新手机号已被其他用户绑定 | 提示「手机号码已有用户」 |
| 账号被锁定 | 角色切换时提示「账号已被锁定，请联系管理员」 |
| 无权限操作 | 角色切换时提示「您没有权限进行当前操作」 |

## 关联功能

- **角色管理**：用户注册和编辑时分配角色
- **权限管理**：角色决定用户可访问的功能模块
- **登录日志**：记录每次登录的IP、操作系统、浏览器等信息
- **双因素认证**：部分操作需通过短信验证码二次确认
