# 订单取货方式切换与发货管理流程（服务端深挖）

> 文档版本 1.0 | 面向读者：产品、运营、测试

## 概述

本流程描述药品订单的取货方式管理，包括自提和物流配送两种方式的切换规则、发货确认、快递单号修改和确认收货的完整后端逻辑。

## 触发条件

- 用户在订单详情页切换取货方式
- 管理员在后台确认发货
- 用户在小程序/管理后台确认收货
- 药房管理员通过取货码核销自提订单

## 步骤详解

### 第一步：取货方式切换（switchPickupMethod）

用户可在特定状态下在"自提"和"物流配送"之间切换取货方式：

1. **状态校验**：只允许在"待支付"（flowStatus=10）或"已支付待发货"（flowStatus=20）状态下切换
2. **相同方式校验**：如果目标方式与当前方式相同，提示"取货方式未变化"
3. **切到自提**：
   - 自动生成唯一取货码（递归生成确保不重复）
   - 清除物流公司和快递单号
4. **切到物流配送**：
   - 必须填写完整的收货信息（姓名、手机号、省份、城市、区/县、详细地址六项全部必填）
   - 清除取货码及发货人信息
   - 保存或更新收货地址（乐观锁保护）

### 第二步：准备发货（prepareShipment）

管理员确认准备发货时：

1. **状态校验**：只有"已付款"（flowStatus=30）状态的订单才能进入待发货状态
2. **状态更新**：订单状态从"已付款"变更为"待发货"（flowStatus=35）
3. **事件发布**：发布 OrderPendingShipmentEvent，通知相关系统

### 第三步：确认发货（confirmShipment）

管理员填写快递信息并确认发货：

1. **状态校验**：
   - 订单状态低于"待发货"（35）提示"订单未付款"
   - 订单状态高于"待发货"（35）提示"该订单已发货或者已失效"
2. **收货地址校验**：必须存在收货地址信息，否则提示"该订单不存在收货地址信息"
3. **物流订阅**：调用物流服务订阅快递单号跟踪
4. **状态更新**：订单状态变更为"已发货待收货"（flowStatus=40），记录发货时间
5. **事件发布**：发布 OrderDeliveryEvent，通知患者发货信息

### 第四步：修改快递单号（updateExpressNumber）

如果填错快递单号需要修改：

1. **状态校验**：只有"已发货待收货"状态的订单才能修改
2. **地址校验**：必须有收货地址信息
3. **重新订阅**：对新的快递单号重新订阅物流轨迹

### 第五步：确认收货（confirmReceipt）

用户收到药品后确认收货：

1. **身份校验**：只有订单的下单人本人才能确认收货，非本人操作提示"非法操作"
2. **状态校验**：已完成状态的订单提示"请勿重复收货"
3. **订金尾款处理**：如果订单有订金（货到付款模式），确认收货时自动记录尾款收入流水
4. **状态更新**：订单状态变更为"已完成"

### 第六步：取货码核销（pickUp）

药房通过取货码核销自提订单：

1. **取货码校验**：取货码不存在提示"取货码有误"
2. **状态校验**：已收货的订单提示"该订单已取货"
3. **状态更新**：订单直接进入"已完成"状态，记录核销人和核销时间

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 取货方式不在待支付/待发货状态切换 | 提示"当前订单状态不允许修改取货方式" |
| 目标取货方式与当前相同 | 提示"取货方式未变化" |
| 切物流配送时收货信息不完整 | 提示"切换到物流配送需要填写完整的收货信息" |
| 非已付款状态发货 | 提示"只有已付款订单才能去发货" |
| 已发货状态再次发货 | 提示"该订单已发货或者已失效" |
| 无收货地址信息 | 提示"该订单不存在收货地址信息" |
| 非下单人确认收货 | 提示"非法操作" |
| 取货码错误 | 提示"取货码有误" |
| 订单已取货 | 提示"该订单已取货" |
| 乐观锁更新失败 | 提示"订单信息已变更，请刷新后重试" |

## 关联功能

- 15天自动收货：定时任务 `automaticReceiptFor15Days`，发货超过15天的处方订单自动确认收货
- 订单完成事件 → OrderCompletedEvent / OrderReceivedEvent 触发后续流程
