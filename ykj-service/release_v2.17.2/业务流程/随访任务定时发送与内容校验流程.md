# 随访任务定时发送与内容校验流程

> 文档版本：2026-07-02 | 面向读者：产品经理、运营人员、客服

## 概述

医生创建随访计划后，系统定期自动将随访内容（患教文章、问卷、提醒）发送给患者。发送前系统会实时校验每一项内容的当前状态 —— 如果某项内容已被管理员禁用，该内容不会出现在患者的任务列表中。这是确保患者只收到有效内容的实时保护机制。

## 触发条件

- 执行方：系统定时任务（自动触发）
- 前置条件：医生已创建随访计划并关联了患者、随访计划状态为「进行中」

## 步骤详解

### 第一步：定时任务触发

系统定时检查待发送的随访任务。每个随访任务有独立的发送时间，到期后系统自动执行发送逻辑。

### 第二步：实时校验内容状态（核心机制）

在发送提醒之前，系统对任务下的每一项内容逐一校验：

1. **患教文章**：查询文章当前的发布状态。如果文章已被禁用（Status ≠ Published），该内容的发送标记设为「禁止」。
2. **问卷**：查询问卷当前的发布状态。如果问卷已被禁用，同样标记为「禁止」。

这条规则的含义是：即使医生创建任务时选择了某篇文章/问卷，如果在发送时刻该内容已被管理员禁用，患者也不会收到。

### 第三步：判断是否发送

系统统计任务下所有「未被禁用」的内容数量：

- **至少有一项可发送内容**：任务状态更新为「随访中」，向患者推送随访提醒消息（微信订阅消息，内容为"医生给您发了随访消息，请及时查看。"）
- **所有内容均被禁用**：任务状态更新为「已取消」，不发送任何消息。这意味着该次随访自动作废。

### 第四步：患者接收与处理

患者收到随访提醒后，进入小程序查看任务详情，可逐一完成：
- 阅读患教文章（系统记录已读状态）
- 填写问卷并提交答案
- 查看随访提醒内容

### 第五步：任务完成与小结

当任务下所有内容的状态都变为「已完成」时，该任务完成。医生可为完成的随访计划填写总结摘要，但前提是计划状态必须为「已完成」—— 进行中的计划不能添加总结。

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 随访记录不存在 | 定时任务跳过，记录错误日志 |
| 任务下所有内容均被禁用 | 任务自动取消，不发送消息 |
| 获取医生信息失败 | 跳过消息推送，记录日志 |
| 获取会员信息失败 | 取消推送，记录日志（无法获取 openId） |
| 查询状态不在允许的范围内 | 返回"非法状态查询"错误 |
| 随访计划未完成时添加总结 | 提示"已完成的计划才能添加总结" |
| 无问诊记录却尝试添加随访 | 提示"未有问诊记录无法添加随访" |

## 关联功能

- 随访计划：医生创建和管理随访计划
- 随访问卷：问卷的创建、发布、禁用状态管理
- 患教课程：患教文章的创建和状态管理
- 订阅消息通知：微信小程序订阅消息推送
- 医生端小程序：随访模板选择、任务详情查看、小结填写
