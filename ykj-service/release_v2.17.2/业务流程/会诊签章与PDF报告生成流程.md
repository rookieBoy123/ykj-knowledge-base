# 会诊签章与PDF报告生成流程

> 文档版本：2026-07-01 | 面向读者：产品经理、技术支持

## 概述

会诊结束后，系统支持对会诊过程中的各类签名进行电子签章（CA认证或手写签名），并将签名后的会诊记录生成PDF报告。

## 触发条件

- **前置条件**：会诊已创建且处于可签章状态
- **触发方式**：医生在小程序端执行签名操作

## 签名类型

系统支持五种会诊签名类型，每种有不同的身份校验：

| 签名类型 | 说明 | 签名人身份要求 |
|---------|------|--------------|
| 会诊申请书签名 | 申请会诊的主治医生签署会诊申请书 | 必须是会诊申请医生（consultation.doctorId） |
| 审核医生签名 | 会诊审核通过后审核医生签署 | 已移除校验逻辑，当前无身份限制 |
| 受邀医生签名 | 被邀请参加会诊的医生签署 | 必须是会诊的受邀医生之一 |
| 会诊意见签名 | 医生提交会诊意见时签署 | 必须是会诊的受邀医生之一 |
| 会诊报告签名 | 会诊结束后主治医生签署最终报告 | 必须是会诊申请医生（consultation.doctorId） |

## 步骤详解

### 第一步：身份校验

医生签章前，系统先验证签名人身份：
- 主治医生签名（申请书/报告）：校验当前医生ID是否等于会诊的主治医生ID，不匹配则提示"非主治医生无法签字"
- 受邀医生签名（邀请/意见）：检查当前医生是否在会诊的受邀医生列表中，不匹配则提示"非受邀医生无法签字"或"非医生意见提交无法签字"

### 第二步：处理重复签名

如果医生之前已发起过同类型的签名（状态为进行中），系统会自动撤销之前的签名记录，避免同一份文档被多次签名产生冲突。

### 第三步：判断签章方式

系统检查CA签章功能是否已启用（enableCA）：
- **CA已启用** → 进入电子签章流程（调用第三方CA平台）
- **CA未启用或降级** → 使用手写签名图片嵌入PDF

### 第四步（CA签章路径）：创建CA签章记录

1. 系统组装会诊数据（患者信息、诊断、会诊目的、受邀医生列表等）
2. 调用CA服务创建签章记录
3. 生成带CA签章的PDF报告文件
4. 记录签章状态和签章时间

### 第五步（手写签名路径）：手写签名嵌入PDF

1. 如果医生提供了手写签名图片（base64格式），系统将其嵌入到PDF中
2. 系统为不同签名类型设置不同的签名位置和署名信息：
   - 主治医生签名：显示在会诊报告左上角
   - 受邀医生签名：显示在对应医生信息的右侧
   - 会诊意见签名：附带医生的会诊意见文本
3. 使用Beetl模板引擎生成PDF，签名图片以约20%（0.2倍）的缩放比例嵌入
4. 签章完成后更新状态为"已签收"（Accept）
5. 触发完成签名的后续处理（记录日志、发送通知）

### 第六步：PDF报告内容

生成的PDF报告包含以下信息：
- 会诊基本信息（会诊编号、会诊时间、申请原因、会诊目的）
- 患者信息（姓名、性别、年龄/出生日期）
- 主治医生信息（姓名、科室、职称、签名）
- 受邀医生列表（姓名、科室、签名、会诊意见）
- 诊断信息（中医疾病、中医证候、西医诊断）
- 治疗方案
- 患者知情同意签名（如有）

### 超时处理

- **审核超时**：会诊开始前30分钟仍未审核 → 系统自动取消会诊
- **受邀医生签章超时**：会诊创建后12小时内受邀医生未签署 → 触发超时提醒

## 异常处理

| 情况 | 系统处理方式 |
|------|------------|
| 非主治医生尝试签申请书 | 返回错误"非主治医生无法签字" |
| 非受邀医生尝试签邀请/意见 | 返回错误"非受邀医生无法签字"或"非医生意见提交无法签字" |
| 手写签名图片为空 | 前端不传图时后端直接返回 result（caEnabled=false），不执行签章 |
| 创建签署记录失败 | 抛出异常"创建签署记录失败, 请联系管理员" |
| 医生多次点击签名 | 系统自动撤销之前的旧签名记录，再创建新的，避免重复 |

## 关联功能

- **电子签章（CA服务）**：第三方CA平台负责电子签章的创建和验证
- **会诊生命周期管理**：签章是会诊流程中的重要节点
- **PDF报告查看**：签章后的PDF可供医生和患者查看下载
- **问诊/处方签章**：共用同一套CA签章服务体系
