# 问诊归档全流程

> 文档版本 2026-07-02 | 面向读者：产品、运营

## 概述

当一次线上问诊完成并全部审核通过后，系统会自动将这次问诊的全部资料（病历、处方、聊天记录、视频录像）打包归档到对象存储（OSS）中，形成长期保存的就诊档案。运营人员也可以对归档失败的记录进行手动重新归档。

## 触发条件

问诊归档通过两种事件触发：

1. **问诊完成事件**：医生结束问诊、填写病历后，系统发布"问诊完成"事件，触发归档
2. **审方完成事件**：药师完成处方审核后（通过/拒绝/过期），系统发布"审方完成"事件，触发归档

两种事件触发后，归档服务校验是否可以归档，满足条件后才执行。

## 归档前置校验

系统在收到归档事件后先做校验，以下情况**不归档**：

1. **问诊单不存在**或数据异常
2. **问诊状态不是"已完成"**（flowStatus != FINISH）：正在进行中的问诊不归档
3. **处方审核未完成**：如果该问诊下有处方、但处方状态不是"已通过""已拒绝""已过期"中的一种，说明还有处方在审核中，不归档

只有上述条件全部满足时才允许归档。

## 步骤详解

### 第一步：创建归档记录

满足归档条件后，系统创建一条归档记录，填入以下信息：
- 问诊单号、患者姓名、科室、医生信息
- 归档类型：自动归档
- 归档状态：待处理
- 归档目标目录：`/系统配置的归档根目录/患者姓名(证件后4位)/问诊单号(问诊日期)/`

归档目录的结构：按患者→按问诊单号分层，方便人工查找。

### 第二步：定时任务扫描

系统的定时任务扫描状态为"待处理"的归档记录，每次取前100条进行处理。

### 第三步：归档四类资料

对每条待归档记录，依次上传四类资料到OSS：

#### 3.1 归档病历记录

- 查询该问诊的病历信息
- 使用模板引擎将病历数据渲染为HTML，再转为图片
- 包含：患者信息（姓名、性别、年龄/月龄、监护人信息）、问诊类型、主诉、现病史、既往史、过敏史、所有诊断（疾病诊断/辩病/辩证）
- 文件名格式：`门诊病历-{问诊单号}.jpg`
- 若查找不到病历，跳过此项并记录警告

#### 3.2 归档电子处方

- 查询该问诊下所有"药师审核通过"的处方
- 根据处方类型区分文件名：中药处方→"中药处方-{门诊号}"，西药处方→"西药处方-{门诊号}"
- 将处方PDF文件从原存储位置转存到归档目录
- 若没有已通过的处方，跳过此项

#### 3.3 归档图文聊天资料

- 查询问诊的IM群组聊天记录（全部消息）
- 将聊天记录序列化为JSON文件
- 文件名格式：`图文资料-{问诊单号}.json`
- 若无聊天记录，跳过此项

#### 3.4 归档视频资料

- 查询问诊关联的视频录制文件（视频问诊的录像）
- 将每个视频从原存储位置转存到归档目录
- 文件名格式：`视频资料-{医生ID}@{序号}.mp4`
- 若有多个视频，按序号依次存储
- 若无视频，跳过此项

### 第四步：更新归档状态

- 全部归档成功 → 状态变为"已归档"
- 归档过程中发生异常 → 状态变为"归档失败"，记录错误日志

### 第五步：手动重新归档（运营操作）

如果某条记录归档失败，后台运营人员可以手动触发重新归档：
- 在后台管理页面找到失败的记录
- 点击手动归档按钮
- 系统重新执行第三步的归档流程
- 仅"归档失败"状态的记录可以手动归档，已归档和待处理的不可手动操作

---

## 状态流转

```
[问诊完成/审方完成] → 待处理(HavedInHand) → 已归档(Success)
                                         ↘ 归档失败(Fail) → [手动重新归档] → 待处理 → 已归档
```

---

## 异常处理

| 情况 | 处理方式 |
|------|---------|
| 问诊有处方仍在审核中 | 不归档，等待审方完成后再次触发 |
| 归档过程中OSS上传失败 | 该记录标记为"归档失败"，记录错误日志；下次手动重试 |
| 无病历信息 | 跳过病历归档，其他三类资料正常归档 |
| 无处方信息 | 跳过处方归档，其他三类资料正常归档 |
| 无聊天记录 | 跳过聊天归档，其他资料正常归档 |
| 无视频录像 | 跳过视频归档，其他资料正常归档 |
| 手动归档非"失败"状态 | 后端拦截，提示"归档记录正在归档中或者已归档，不能手动归档" |

## 关联功能

- 问诊管理：问诊完成是归档的触发条件
- 处方审核：审方完成也是归档的触发条件
- 后台归档管理：运营人员查看归档列表，手动重新归档失败的记录
