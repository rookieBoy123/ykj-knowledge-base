# 电子病历创建与签章管理流程

> 文档版本：v1.0 | 面向读者：运营、客服、产品

## 概述

医生在问诊过程中为患者创建电子病历，记录病情诊断信息，并通过CA电子签章完成法律效力的签署。病历支持创建、修改、追加诊断、更换患者、作废撤回等全生命周期管理。

## 触发条件

- **前置条件**：医生已接诊，进入了问诊流程
- **入口**：医生端问诊聊天页面 → 填写病历信息 → 提交（触发病历创建）

## 步骤详解

### 第一步：创建病历

医生提交病历时，系统执行以下操作：

- **记录医生信息**：自动获取并保存医生的姓名、医院、科室、职称、执业类别（中医/西医）
- **记录患者信息**：如果指定了就诊人，自动关联患者姓名、性别、年龄等
- **保存诊断信息**：诊断名称、诊断编码等诊断明细同时存入诊断表
- **初始签名状态**：病历创建后签名状态为「未签」
- **创建电子签章**：同步创建CA签章记录（用于后续医生签名），如果签章服务需要授权，返回认证信息让医生完成CA认证
- **生成分享二维码**：如果启用了分享功能，生成病历的CA二维码供患者扫码查看
- **严格模式兼容**：v2.4.4后已移除"病历必须签名"的强制要求，部分场景病历可以不签名

### 第二步：修改病历

医生可以修改已创建的病历：

- **签名状态重置**：修改后病历签名状态回到「未签」，需要重新签名
- **诊断差分更新**：系统智能对比新旧诊断列表，自动识别哪些诊断被删除、哪些是新增的，只执行最小变更
- **删除保护**：已被处方关联的诊断不允许删除（提示"不允许删除已经被关联的诊断"）
- **重新创建签章**：修改后重新创建签章记录，让医生再次签署

### 第三步：追加诊断

医生可以在已有病历中追加新的诊断：

- **去重处理**：系统自动判断新诊断中哪些已存在（跳过）、哪些是真正新增的（插入），避免重复
- **不可修改标记**：已关联到处方或医保上传的诊断会被标记为"不可修改"，医生可以看到但不能删除

### 第四步：更换患者

如果发现病历绑定了错误的患者：

- **患者信息更新**：重新设置患者姓名、性别、年龄等基本信息
- **病历ID不变**：更换患者不会生成新病历，只在原病历上修改

### 第五步：绑定会员

病历创建时可以绑定特定会员（小程序用户）：

- **唯一绑定**：每个病历只能绑定一个会员，如果已绑定则不允许重新绑定到其他会员
- **会员存在性校验**：绑定前检查会员是否存在，不存在则提示"变更病历患者失败，会员不存在"

### 第六步：主动签署

医生可以随时对已创建但未签名的病历发起签署：

- **重复签章检查**：如果已有签章记录（历史创建过），直接复用不重建
- **新签章创建**：如果还没有签章记录，创建新签章让医生完成签名流程
- **已签名提示**：病历已签署时提示"病历已签署，不需要重复签署"

### 第七步：作废/撤回病历

特殊情况下可以撤回病历：

- **授权检查**：系统调用所有注册的撤回断言器检查是否可以撤回（如：处方已关联的病历不允许撤回）
- **记录撤回原因**：保存撤回原因文本
- **逻辑删除**：病历从查询列表中移除但不物理删除

### 第八步：病历查询

- **多维度筛选**：支持按患者ID列表、会员ID列表、签名状态列表筛选
- **仅查有问诊的病历**：支持只查询已关联到问诊的病历（排除孤立病历）
- **诊断信息自动组装**：查询结果自动附带诊断列表

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 修改病历时医生不存在 | 提示"医生不存在" |
| 删除已被处方关联的诊断 | 提示"不允许删除已经被关联的诊断。诊断名称：【XXX】" |
| 病历已绑定会员但尝试重新绑定 | 提示"该病历已绑定会员，不允许重新绑定" |
| 绑定病历时会员不存在 | 提示"变更病历患者失败，会员不存在" |
| 签署病历时病历不存在 | 提示"病历不存在" |
| 病历已签署但再次发起签署 | 提示"病历已签署，不需要重复签署" |
| 撤回病历时关联检查不通过 | 拒绝撤回（具体提示由断言器返回） |

## 关联功能

- **问诊生命周期管理流程**：病历依附于问诊创建
- **处方开具与审核签章流程**：处方需要引用病历中的诊断信息
- **医保结算流程**：病历诊断用于医保费用明细上传
