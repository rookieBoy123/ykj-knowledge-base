# 药品生命周期管理与第三方同步流程

> 文档版本：2026-07-02 | 面向读者：运营管理员、药房管理员

## 概述

本流程描述平台中药品从新增、编辑、导入、库存扣减到第三方药房自动同步的全生命周期管理。

## 触发条件

- **后台管理员**在药品管理页面新增或编辑药品
- **定时任务**自动拉取第三方药房（如阿康）的药品信息和库存
- **医生开方时**自动扣减药品库存
- **后台管理员**批量导入/导出药品或禁用药品

## 步骤详解

### 一、药品新增（后台管理）

1. **药品编号自动生成**：系统自动生成唯一药品编号（编码+序号），无需手动填写。若未指定编号，系统会自动分配。
2. **第三方药房药品必填校验**：如果该药房是对接的第三方药房（如阿康），则必须填写「进货价」和「来源编码（sourceCode）」，否则系统会提示"进货价不能为空"或"来源编码不能为空"。
3. **药品类型自动设定**：
   - 中药颗粒：药品剂型自动固定为"颗粒剂(冲剂)"
   - 成药：费用类型默认设为1
   - 不同类型采用不同的唯一性规则：成药按「编号+药房ID」去重，中药按「通用名（drugsAlias）」去重
4. **黑名单检查**：新增药品时，系统检查该药品是否在黑名单中，若命中则提示"药品已被加入黑名单"，不予新增。
5. **福建地区特殊校验**：在福建地区（adaptiveArea=fujian），批准文号（approvalNo）为必填项，否则提示"批准文号不能为空"。
6. **药房药品数量更新**：新增成功后，自动重新统计该药房的成药数量和中药数量并更新。

### 二、药品编辑

1. **不可修改字段保护**：编辑时药品编号（drugsNo）、所属药房（pharmacyId）、药品药房联合标识（drugsPharmacyId）均不可修改，系统以数据库原值为准。
2. **中药通用名唯一性校验**：仅中药类型需要校验通用名是否重复——在同一个药房内，不能出现相同通用名的中药。如果已存在，提示"存在相同通用名，不允许修改"。
3. **第三方药房药品必填保护**：编辑时间样需要校验进货价和来源编码。

### 三、库存扣减（医生开方时触发）

1. **前提条件**：药房必须开启库存管理功能（isOpenStock），未开启则不触发库存校验。
2. **无限库存标记**：如果药品库存设为 -1，代表该药品无限库存，不进行扣减。
3. **库存不足提示**：扣减库存时如果当前库存小于需要的数量，记录错误日志并返回失败，医生端会提示"药材缺货或库存不足"。
4. **准确扣减**：库存充足时，执行减法运算（当前库存 - 消耗量），更新数据库。

### 四、第三方药品同步（定时任务）

系统支持三种第三方药房相关的定时同步：

#### A. 药品信息拉取（pullThirdDrugs）
- 遍历所有已启用的第三方药房
- 每次请求间隔7秒（防止第三方接口过载）
- 分页获取第三方药品列表（每页100条）
- 按sourceCode匹配已有药品，更新：药品名称、通用名、规格、产地、用法、库存
- **注意**：进货价不会在同步时被覆盖，以导入时手动填写的为准
- 某次请求失败时记录日志并跳过该药房，不影响其他药房

#### B. 药品图片拉取（pullThirdDrugsImg）
- 只拉取那些还没有图片但有sourceCode的药品
- 通过第三方接口获取药品图片URL，转存到OSS存储
- 单条图片拉取失败不影响其他药品

#### C. 库存同步（pullStoreFromThird）
- 只同步库存在0以下的药品（有sourceCode且stock≤0的）
- 分批获取库存信息（每批100个）
- 每次请求间隔3秒
- 更新库存数量

### 五、药品导入（Excel批量导入）

1. **三种模板**：成药（PatentMedicineExport）、中药饮片（ChineseHerbalMedicineExport）、中药颗粒（ChineseMedicineGranulesExport），分别对应不同的Excel模板格式。
2. **逐行处理策略**：每条记录先尝试新增，如果新增失败（通常是重复），则自动转为编辑更新。
3. **自动填充默认值**：
   - 拼音首字母：从通用名自动生成
   - 药品剂型：成药→"其它剂型"，饮片→"原料"
   - 成药监管分类：默认"其他(其他)"
4. **使用频次匹配**：导入的使用频次字段会匹配到系统的使用频次字典，匹配不上则不设置。
5. **导入结果**：返回成功条数和失败条数，失败的记录可查看详细原因。

### 六、药品删除与禁用

1. **单个删除**：删除后自动更新所属药房的药品数量统计。
2. **批量删除**：同时更新所有涉及药房的药品数量。
3. **批量禁用**：将选中药品状态设为「禁用」，医生端将看不到这些药品。

### 七、医生端药品查询

1. **药房授权校验**：医生查询药品前，系统检查该医生是否被授权访问对应药房，未授权提示"您暂未授权该药房"。
2. **模糊搜索**：支持拼音首字母搜索中药，结果按字符串相似度算法排序（更匹配的排在前面）。
3. **医保关联**：药品列表自动关联医保信息（medicareMedicine）。
4. **图片自动组装**：查询结果自动从OSS获取药品图片和说明书图片。

## 异常处理

| 情况 | 系统处理方式 |
|------|------------|
| 新增药品时药品已存在 | 提示"药品已存在"，不同药品类型有不同唯一性判断 |
| 药品在黑名单中 | 提示"药品已被加入黑名单"，不予新增 |
| 第三方药房未填写进货价 | 提示"第三方药房药品进货价不能为空" |
| 福建地区未填写批准文号 | 提示"批准文号不能为空" |
| 中药编辑时通用名重复 | 提示"存在相同通用名，不允许修改" |
| 第三方同步时网络异常 | 记录错误日志，跳过该药房继续处理其他药房 |
| Excel导入时格式错误 | 返回失败记录清单，可修正后重新导入 |

## 关联功能

- **药房管理**：药品归属于药房，药房禁用会影响该药房下所有药品的可见性
- **处方开具**：医生开方时查询药品列表、扣减库存
- **药品黑名单**：被加入黑名单的药品名称/别名无法新增
- **医保药品关联**：药品列表展示时关联医保药品信息
- **使用频次字典**：导入时匹配，影响处方中的用法用量选项
