# 标准药品诊断对照管理流程（服务端）

> 文档版本：2026-07-03 | 面向读者：产品经理、技术运营

## 概述

标准药品诊断对照是将系统中的标准药品（通用名）与标准诊断（疾病名称）建立关联关系的功能。通过这个对照关系，系统可以在医生开具处方时根据诊断信息推荐或校验对应的药品，确保用药与诊断的匹配性。管理后台提供新增、修改、查询、导入功能。

## 触发条件

- **管理入口**：管理后台 → 医院管理 → 标准管理 → 标准药品诊断对照
- **服务模块**：`ykj-service-std`
- **权限要求**：需要 `std:med-diag:write`（增/改权限）或 `std:med-diag:read`（查询权限）、`std:med-diag:import`（导入权限）

## 接口说明

### 1. 分页查询列表

**接口**：`GET /mgmt/std/stdMedDiag`

支持分页查询和条件筛选（药品通用名、诊断名称等）。返回分页列表供管理后台展示。

### 2. 新增对照关系

**接口**：`POST /mgmt/std/stdMedDiag`

管理员选择一个药品通用名和一个或多个诊断进行绑定。同一个药品通用名可以关联多个诊断。

**重要行为**：如果新增时系统检测到该药品通用名已经存在对照关系，管理后台会弹出提示"该药品通用名已存在，是否将诊断更新到原数据？"——选择"确定"则追加诊断到已有记录，选择"取消"则放弃新增。

### 3. 修改对照关系

**接口**：`PUT /mgmt/std/stdMedDiag`

管理员可修改已有对照关系的诊断列表。编辑时药品通用名字段置灰不可修改，只能修改关联的诊断。

### 4. 查询详情

**接口**：`GET /mgmt/std/stdMedDiag/detail?medCommonName=xxx`

按药品通用名查询对照详情。

### 5. 批量导入

**接口**：`POST /mgmt/std/stdMedDiag/import`

支持通过 Excel 文件批量导入药品诊断对照关系。需要 `std:med-diag:import` 权限。

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 无写入权限 | 后端拦截，返回权限错误 |
| 无读取权限 | 后端拦截，返回权限错误 |
| 药品通用名已存在 | 前端弹出智能合并提示（不是报错），用户可选择追加或取消 |
| 批量导入文件格式错误 | 后端解析失败，返回错误信息 |
