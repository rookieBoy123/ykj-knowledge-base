# 发票创建与查询流程（服务端）

> 文档版本：v1.0 | 面向读者：产品、开发

## 概述

平台发票管理模块支持订单发票的创建、查询和管理后台的发票导出。关联订单模块、预约模块，支持电子发票的红冲、图片查看等操作。

## 涉及模块

- **ykj-service-order**：订单发票（ApiOrderInvoicesController + OrderInvoicesService）
- **ykj-service-reservation**：服务预约发票（EInvoice相关Controller）
- **ykj-service-server**：管理后台发票管理（MgmtOrderInvoicesController）

## 流程详解

### 一、订单发票创建

**接口**：`POST /api/order/invoices`

状态图：
```
用户提交发票信息(OrderInvoicesDetail) → 自动获取memberId → 数据校验 → 创建 → 返回发票VO
```

**请求参数**（OrderInvoicesDetail）：
- memberId：自动从登录态获取
- 发票抬头、税号、发票类型等信息
- 关联的订单号

**校验**：
- 通过`@Validated({InsertValid.class})`进行插入分组校验
- 必填字段检查

**返回**：`OrderInvoicesVo`（发票视图对象）

### 二、订单发票查询

**接口**：`GET /api/order/invoices/{orderNo}`

状态图：
```
输入订单号 → 查询订单关联的发票信息 → 返回OrderInvoices完整对象
```

- 通过订单号直接查询
- 返回完整的发票信息（包含发票图片URL、开票状态等）

### 三、服务预约发票

预约模块有独立的发票管理：

**相关接口**：
- `EInvoiceReq`：发票请求参数
- `EInvoicePicUrlQueryReq`：发票图片URL查询
- `EInvoiceScarletQueryReq`：红冲发票查询
- `EInvoiceVoidReq`：发票作废

服务预约创建时会自动生成发票记录。

### 四、管理后台发票查询与导出

**接口位置**：`MgmtOrderInvoicesController`（ykj-service-server模块）

管理员可以：
1. **多条件查询**：患者姓名（5字限制）、订单编号（30字限制）、开票日期范围
2. **列表展示**：9项字段（含门诊号、患者姓名的"/"空值处理）
3. **打印按钮**：仅在pictureNetUrl非空时显示
4. **导出Excel**：导出为.xls格式

### 五、发票数据结构

**OrderInvoices（订单发票）**：
- 订单号关联
- 发票抬头
- 税号
- 发票金额
- 发票状态
- 发票图片URL
- 创建时间

**InvoiceExportVo（导出数据结构）**：
- 用于Excel导出的平铺字段
- 包含患者信息、订单信息、发票信息

## 异常处理

| 情况 | 处理方式 |
|------|---------|
| 数据校验不通过 | 返回400错误，指出具体字段问题 |
| 订单号不存在 | 返回空结果或null |
| 发票图片不存在 | 不显示打印按钮 |
| 导出数据为空 | 导出空的Excel文件 |

## 关联功能

- **订单支付**：支付完成后的发票申请
- **服务预约**：预约完成后的电子发票
- **管理后台发票查询**：发票查询页面（invoiceQuery）
- **微信电子发票**：对接微信发票平台
