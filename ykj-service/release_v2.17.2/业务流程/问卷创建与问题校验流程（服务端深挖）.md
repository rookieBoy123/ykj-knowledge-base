# 问卷创建与问题校验流程（服务端深挖）

> 文档版本：2026-07-05 | 面向读者：后台开发/产品

## 概述
本文档补充说明服务端问卷管理中的深层校验逻辑和数据处理细节，包括重名校验策略、问题选项约束、级联删除设计、以及问题库联动机制。

## 核心校验规则

### 1. 问卷标题重名校验

**校验策略有区分**：

| 场景 | 重名判断范围 |
|------|-------------|
| 医生创建私有问卷 | 同一医生（doctorId相同）+ 相同标题 = 重名 |
| 创建公共问卷 | 所有公共问卷（doctorId为null）+ 相同标题 = 重名 |
| 编辑问卷 | 排除自身ID后，同范围内不可重名 |

后端提示：「您当前已有相同标题的问卷」

### 2. 问卷基本信息校验

| 校验项 | 规则 | 提示 |
|--------|------|------|
| 标题 | 必填，最长300字 | 「问卷标题不能为空」/「问卷标题最大字数不能超过：300字」 |
| 副标题 | 选填，最长300字 | 「问卷副标题最大字数不能超过：300字」 |
| 问题列表 | 至少一道题 | 「问卷问题不能为空」 |

### 3. 问题校验

| 校验项 | 规则 | 提示 |
|--------|------|------|
| 题目名称(subject) | 必填 | 「问卷题目名称不能为空」 |
| 题目类型(type) | 必填 | 「问卷题目类型不能为空」 |
| 题目序号(seq) | 必填 | 「问卷题目序号不能为空」 |
| 答案选项 | 选择题/多选题至少2个选项 | 「XX题答案选项不能为空」/「XX题选项不能低于2个」 |

**免选项题型**：问答题（EQ）和日标评估题（RBQ）允许答案选项为空，不需要2个选项的约束。

### 4. 问题保存：先删后插策略

编辑问卷时的问题保存采用**先删除所有旧问题、再插入新问题**的策略：
1. 删除该问卷下所有旧问题（含答案选项）
2. 逐条插入新问题及其答案选项
3. 如果问题标记了「保存到问题库」，异步保存到公共问题库

这种设计避免了复杂的增量对比逻辑，但也意味着每次编辑都会重新生成问题ID。

### 5. 答案选项级联删除

删除问卷 → 删除所有问题 → 删除所有问题的答案选项（三级级联）

## 问题库联动

问题在保存时可以勾选「同时保存到问题库」，系统会：
- 将问题转换为问题库格式（`QuestionLibraryDomain`）
- 调用问题库服务的 `saveQuestion` 方法
- 如果保存失败（如公共库中已存在同名问题），在返回结果中提示「以下问题保存至问题库失败：XXX」
- 保存失败不影响问卷本身的保存

## 私有问卷特殊处理

| 操作 | 私有问卷行为 |
|------|-------------|
| 创建 | doctorId不为空，仅创建者可见 |
| 编辑（管理后台） | 任意管理员可编辑 |
| 编辑（医生端） | 校验 doctorId 是否匹配，不匹配提示「您无法修改当前问卷」 |
| 查询列表 | 私有：仅显示自己的私有问卷+所有公共问卷；非私有：仅显示公共问卷 |
| 保存时 | 如果isPrivate=false，强制清除doctorId |

## 异常处理汇总

| 情况 | 处理方式 |
|------|---------|
| 问卷标题重复 | 后端拦截，返回「您当前已有相同标题的问卷」 |
| 查询私有问卷未指定医生 | 返回「查询私有问卷时, 未指定要查询的医生」 |
| 问卷不存在 | 返回「问卷不存在」 |
| 未选择要删除的问卷 | 返回「批量删除失败, 未选择要删除的问题」 |
| 医生端修改他人问卷 | 返回「您无法修改当前问卷」 |
