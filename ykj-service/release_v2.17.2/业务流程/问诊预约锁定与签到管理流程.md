# 问诊预约锁定与签到管理流程

> 文档版本 2026-07-02 | 面向产品/开发

## 概述

问诊预约是患者购买问诊服务后，与医生约好具体就诊时间的过程。系统通过预约锁定（防止超额）、签到确认（确保按时就诊）、超时自动取消等机制保障预约的公平性和有效性。

## 触发条件

- 患者支付问诊订单成功后，系统自动创建预约记录
- 或患者手动选择就诊时间段时触发预约

## 预约创建流程

### 创建预约
1. 根据预约类型选择对应的预约策略（ReservationStrategy）
2. 策略计算预约的开始时间和结束时间
3. 创建预约记录 → 状态：待锁定
4. **启动锁定超时定时任务**：如果预约在规定时间内未被锁定使用，系统自动取消（handleReservationLockTimeout）
   - 超时原因："预约号锁定时间结束，自动取消"

## 预约锁定流程

### lock - 患者锁定预约
1. 查找预约记录 → 不存在则报错"未找到预约信息"
2. 预约策略计算最终签到截止时间
3. 将预约标记为"已使用"→ 记录锁定人和到期时间
4. **启动签到截止定时任务**：如果预约在截止时间前未签到，系统自动取消（checkInCutoffCancelReservation）
   - 超时原因："未在规定时间内完成签到"

## 签到流程（checkIn）

### 签到时间窗口
- 允许在预约开始前10分钟开始签到
- 超过签到截止时间不允许签到

### 签到操作
1. 查询预约记录
2. 校验是否已签到 → 已签到则跳过
3. 校验签到时间是否在允许窗口内
4. 标记已签到 → 记录签到时间
5. **发送签到成功通知** → 通知医生患者已签到就位
6. **发布签到成功事件** → 其他模块可监听处理（如更新问诊状态）

## 取消预约流程

### 患者主动取消（memberCancel）
1. 校验取消人与预约人是同一人 → 不是则提示"您无权取消该预约"
2. 校验当前状态是否允许取消 → 策略取消校验
3. 取消预约 + 取消关联问诊单 + 发起退款
4. 发送取消通知给相关方

### 问诊取消联动（inquiryCancelHandler）
- 监听问诊取消事件（InquiryCanceledEvent）
- 自动取消关联的预约单 → 备注"订单被取消"

### 班次停诊批量取消（pushReservationCancel）
- 监听班次停诊事件（ScheduleStopReceptionEvent）
- 批量取消该班次下的所有未签到预约
- 取消类型：CREATE_ERROR_AUTO_CANCEL（排班异常自动取消）

### 系统自动取消（定时任务）
两种自动取消场景：

| 定时任务 | 触发条件 | 备注 |
|---------|---------|------|
| handleReservationLockTimeout | 创建后未在规定时间内锁定 | "预约号锁定时间结束，自动取消" |
| checkInCutoffCancelReservation | 锁定后未在截止时间前签到 | "未在规定时间内完成签到" |

## 预约通知

### 预约前提醒（pushReservationsNoticeMessageTask）
- 在预约开始前10分钟发送提醒
- 同时通知患者和医生
- 检查问诊状态必须是"问诊中"且医生处于"待接诊"才发送

### 通知内容
- 患者端：收到"您的问诊即将开始"提醒（含医生姓名+预约号）
- 医生端：收到"有患者预约了您的问诊"提醒（含患者姓名）

## 策略模式（ReservationStrategy）

不同服务类型使用不同的预约策略：
- 策略决定：预约时间的计算方式、签到窗口、取消校验规则、取消后处理逻辑

## 状态流转

```
创建预约 → 待锁定
   ↓ (lock)
已锁定 → 待签到
   ↓ (checkIn)
已签到 → 问诊中
   ↓ (问诊结束)
已完成

取消路径：
待锁定 → (超时) → 已取消
已锁定 → (超时/主动) → 已取消
已签到 → (问诊取消) → 已取消
```

## 异常处理

| 情况 | 处理方式 |
|------|---------|
| 预约不存在 | "未找到预约信息" |
| 无权取消 | "您无权取消该预约，请联系管理员" |
| 定时任务处理时预约已不存在 | 记录警告日志，不报错 |
| 定时任务处理时预约已使用/已取消 | 跳过不处理 |
| 批量取消某条失败 | 记录警告日志，继续处理下一跳 |
| 预约提醒失败 | 记录错误日志，不阻塞主流程 |
