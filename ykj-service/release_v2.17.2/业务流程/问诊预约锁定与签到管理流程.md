# 问诊预约锁定与签到管理流程

> 文档版本 | 面向读者：产品经理、运营人员、技术支持

## 概述

当患者进行预约挂号式问诊时，系统会先为患者锁定一个号源（临时预约），患者需在规定时间内完成支付（"锁号"），支付后在预约开始前10分钟可以签到，规定时间内未签到则自动取消预约。

## 参与角色

- 患者：操作主体，进行预约、支付、签到
- 医生：接收签到通知，按时接诊
- 系统定时任务：自动处理超时取消、签到截止、预约前提醒

## 业务流程

### 阶段一：预约锁号（reservation）

1. 患者选择预约时间和医生后，系统创建预约记录
2. 预约状态：**未使用**（isUsed=false）、**未签到**（isCheckIn=false）、**未取消**（isCancel=false）
3. 同时创建一个定时任务（jobName=`handleReservationLockTimeout`），到期时间=预约的过期时间（expirationTime）
4. **锁号超时处理**：如果在过期时间内患者未支付（预约未被使用），定时任务触发 → 执行自动取消：状态变更为"已取消"，备注"预约号锁定时间结束, 自动取消"

### 阶段二：使用预约（lock）

1. 患者完成支付后，系统调用`lock`方法将预约标记为"已使用"（isUsed=true）
2. 同时创建一个新的定时任务（jobName=`checkInCutoffCancelReservation`），到期时间=预约的最终签到截止时间
3. **签到截止处理**：如果在截止时间前患者未签到，定时任务触发 → 自动取消预约，备注"未在规定时间内完成签到"
4. 任务执行前会二次校验：已签到(跳过)、已取消(跳过)

### 阶段三：签到（checkIn）

1. 签到开始时间 = 预约开始时间 - 10分钟（BEFORE_RESERVATION_TIME_ALLOW_CHECK_IN_MINUTES）
2. 患者在此时间后可以签到：
   - 校验预约记录是否存在和非空
   - 执行`checkIn`操作，标记isCheckIn=true
   - 向医生发送IM单聊消息："您的患者已签到，请按时接诊。"
   - 向医生推送订阅通知："您的患者已签到，请按时接诊。若超时接诊，问诊将被取消。"
   - 发布`CheckInSucceedEvent`事件，供其他模块监听处理

### 阶段四：预约前提醒

系统在问诊预约开始前10分钟，自动向患者和医生发送提醒：

1. 触发时机：问诊单创建时调用`pushReservationsNoticeMessageTask`注册定时任务
2. 定时任务执行时二次校验：
   - 问诊单状态必须为"问诊中"（FLOW_STATUS_UNDERWAY）
   - 医生未接诊状态（DOCTOR_FLAG_STATUS_MAIN_WAIT）
3. 患者通知内容：医生姓名 + 预约号 + 预约时间
4. 医生通知内容：患者姓名 + 开始时间

### 阶段五：取消预约

**患者主动取消**（memberCancel）：
- 校验操作权限（预约的memberId必须匹配当前会员）
- 取消备注："患者主动取消。"
- 若需要退款（isRefund=true），触发退款流程

**问诊取消联动**（inquiryCancelHandler）：
- 监听`InquiryCanceledEvent`事件，问诊取消时自动取消关联的预约
- 取消备注："订单被取消"

**医生停诊联动**（ScheduleStopReceptionEvent）：
- 医生停诊时批量取消所有关联号源的预约
- 取消类型：CREATE_ERROR_AUTO_CANCEL
- 单条失败不影响其他预约的取消（try-catch保护）

## 状态流转

```
创建预约 → 未使用/未签到
    ↓ (支付完成)
  已使用/未签到
    ↓ (签到)            ↓ (超时)
  已签到               已取消（锁号超时）
    ↓                   
    ↓ (超时未签到)      
  已取消（未在规定时间内完成签到）
```

## 异常处理

| 情况 | 系统处理方式 |
|------|------------|
| 预约记录不存在 | 抛出异常"未找到预约信息" |
| 患者无权取消他人预约 | 抛出异常"您无权取消该预约, 请联系管理员" |
| 锁号超时但预约已使用或已取消 | 跳过处理，记录日志 |
| 签到截止但已签到或已取消 | 跳过处理，不重复操作 |
| 定时任务注册失败 | 记录error日志，不影响预约创建主流程 |
| 停诊批量取消时单条失败 | try-catch保护，记录warn日志，继续处理下一条 |
| 问诊取消但预约已取消 | 判断isCancel后直接返回，不重复取消 |
| 取消时问诊状态已结束或已取消 | 跳过取消问诊单操作，不处理 |
| 医生或会员信息不存在 | 不发送通知消息，静默跳过 |
