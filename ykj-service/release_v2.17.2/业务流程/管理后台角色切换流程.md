# 管理后台角色切换流程

> 文档版本：2026-07-03 | 面向读者：管理员、产品

## 概述

管理后台支持一个用户拥有多个角色（如"管理员"和"医生"）。当用户以某个角色登录后，可以切换到另一个角色，切换后系统返回新的登录令牌（AccessToken），前端使用新令牌重新渲染菜单和权限。这实现了在同一会话中快速切换身份，无需重新登录。

## 触发条件

- 管理员需要从"系统管理员"身份切换到"医生"身份以使用门诊问诊室功能
- 运营人员需要切换到不同角色查看对应的管理菜单和功能
- 开发/测试人员需要验证不同角色的权限隔离

## 步骤详解

### 第一步：发起角色切换

用户在管理后台选择要切换的目标角色，系统调用接口 `POST /mgmt/user/changeRole`，传入目标角色编码（roleCode）。

### 第二步：服务端处理

服务端处理流程：
1. 获取当前登录用户的认证信息（AdminAuthenticationToken）
2. 根据当前认证信息、目标角色编码、登录客户端类型，调用 `switchRole` 方法
3. 切换成功返回新的 AccessToken
4. 前端收到新 Token 后，重新拉取该角色对应的菜单和权限

### 第三步：前端更新

管理后台前端收到新 Token 后：
- 使用新 Token 替换本地存储的旧 Token
- 重新请求菜单接口，获取新角色对应的菜单项
- 重新渲染左侧导航菜单
- 刷新页面权限，不可访问的功能将不可见或置灰

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 未传角色编码 | 提示"切换的角色编码不能为空" |
| 角色编码无效 | 切换失败，服务端返回错误（具体提示由 loginService.switchRole 控制） |
| 用户没有该角色 | 切换失败，权限不足 |

## 关联功能

- **角色管理与权限分配**：角色决定用户可访问哪些菜单和功能
- **系统用户管理**：为用户分配多个角色后，角色切换功能才可用
- **门诊问诊室**：需要切换到医生角色才能使用
