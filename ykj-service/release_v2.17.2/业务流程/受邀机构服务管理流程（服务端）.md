# 受邀机构服务管理流程（服务端）

> 文档版本 2026-07-04 | 面向系统对接人员

## 概述

受邀机构管理员登录后管理系统提供的远程会诊服务项目，包含服务的新增、编辑、删除和查询。与医生管理一样，系统自动识别当前管理员所属机构，所有操作限定在本机构范围内。

## 接口清单

| 接口 | 方法 | 路径 | 权限 | 说明 |
|------|------|------|------|------|
| 分页查询 | GET | `/mgmt/consultation/invited-inst-serve-services` | `consul:invited-inst-serve-services:read` | 查询当前机构下服务列表 |
| 新增 | POST | `/mgmt/consultation/invited-inst-serve-services` | `consul:invited-inst-serve-services:write` | 新增服务，自动填入机构ID |
| 编辑 | PUT | `/mgmt/consultation/invited-inst-serve-services/{id}` | `consul:invited-inst-serve-services:write` | 编辑服务 |
| 查询详情 | GET | `/mgmt/consultation/invited-inst-serve-services/{id}` | `consul:invited-inst-serve-services:read` | 查询单个服务 |
| 删除 | DELETE | `/mgmt/consultation/invited-inst-serve-services/{id}` | `consul:invited-inst-serve-services:write` | 删除服务（带机构ID校验） |

## 服务数据结构

| 字段 | 类型 | 说明 |
|------|------|------|
| name | String | 服务名称 |
| type | String | 服务类型（关联数据字典 `jgfwlx`） |
| callRoll | Integer | 是否点名：0=否，1=是，2=不涉及 |
| jobTitle | Integer | 职称等级要求：1=正高，2=副高，3=不涉及 |
| unitPrice | BigDecimal | 单价 |
| unit | String | 单位（关联数据字典 `jgfwlxdw`） |
| status | Integer | 状态：0=禁用，1=启用 |

## 机构数据隔离

与医生管理相同，新增和编辑操作会自动从登录账号反查机构ID并填入数据。查询详情和删除时使用 `findByIdAndInstId` / `deleteByIdAndInstId` 方法，确保跨机构数据不会泄漏。

## 删除校验：申请记录保护

删除服务时，服务端会查询该服务是否已被会诊申请记录引用（`InstConsultationEntity` 表）。如果存在申请记录，拒绝删除并提示："服务【名称】存在申请记录，不可删除！"

这与医生删除的会诊关联保护逻辑类似，都是为了维护数据完整性。

## 关联模块

- **ykj-service-consultation**：会诊模块
- **InstConsultationService**：会诊服务（用于删除校验）
