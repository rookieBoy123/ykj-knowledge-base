# 有毒药品限制管理流程（服务端）

> 文档版本 | 面向读者：开发人员 / 产品经理

## 概述
有毒药品（或称毒性药品）是中药中需要严格用量控制的药品类别。系统提供有毒药品的独立管理模块，与普通的中药剂量管理分开，专门用于管控含有毒性成分的中药饮片和颗粒剂的上限用量。

## 触发条件
- **数据来源**：管理后台"有毒药品管理"页面
- **影响范围**：医生端处方开具
- **服务模块**：`ykj-service-medicine`

## 核心流程

### 1. 有毒药品数据管理

**数据结构**（`HazardousDrugLimits` 表）：
| 字段 | 说明 |
|------|------|
| name | 药品名称（对应药品库通用名） |
| type | 药品类型（1=中药饮片，2=中药颗粒） |
| upperLimit | 上限剂量（精确数值） |
| createTime | 创建时间 |
| lastUpdateTime | 更新时间 |

**新增有毒药品**（`POST /mgmt/medicine/hazardous-drug-limits`）：
- 提交药品名称、类型和上限剂量
- 系统自动校验药品名称是否重复（同一名称不可重复添加）
- 重复时返回错误"药品名称重复"

**编辑有毒药品**（`PUT /mgmt/medicine/hazardous-drug-limits/{id}`）：
- 同样校验名称重复（排除自身ID）
- 修改后立即生效

**删除有毒药品**（`DELETE /mgmt/medicine/hazardous-drug-limits/{id}`）：
- 单条删除
- 删除后该药品不再受有毒药品限制

### 2. 分页查询

**查询接口**（`GET /mgmt/medicine/hazardous-drug-limits`）：
- 支持按药品名称模糊搜索
- 支持按药品类型筛选
- 按更新时间降序排列
- 权限前缀：`med:hazardous-drugs`

### 3. 业务应用场景

有毒药品限制与普通中药剂量管理的区别：
- **有毒药品**：针对特定毒性中药，上限更严格
- **普通剂量**：针对一般中药，上限相对宽松
- 两种限制同时生效，取更严格的值进行处方拦截

## 异常处理

| 情况 | 系统处理方式 | 异常来源 |
|------|-------------|---------|
| 药品名称重复 | 返回错误"药品名称重复" | 后端拦截（BizException） |
| 编辑时药品名称与其他条目重复 | 返回错误"药品名称重复" | 后端拦截（BizException） |
| 未找到指定ID | 返回空/异常 | 数据库查询结果为空 |

## 关联功能
- **中药剂量管理**：普通中药的剂量限制
- **处方开具**：医生开处方时的双重校验
- **药品主数据**：药品库中的药品信息
