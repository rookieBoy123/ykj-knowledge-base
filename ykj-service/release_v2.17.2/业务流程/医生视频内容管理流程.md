# 医生视频内容管理流程

> 文档版本：2026-07-02 | 面向读者：运营管理员

## 概述

本流程描述管理后台创建、编辑和管理医生科普视频内容的完整链路。视频可绑定到医生，在患者端以短视频信息流形式展示，支持分类管理、启用/禁用、推荐置顶和播放量统计。

## 触发条件

- **运营管理员**在后台「视频管理」中新增或编辑视频
- **患者端**浏览医生视频时自动上报播放量
- **管理员**需要推荐某个视频到列表顶部

## 步骤详解

### 一、新增医生视频

1. **视频必须已上传**：系统先检查视频在视频库（Video表）中是否存在，不存在则提示"视频不存在"。
2. **分类必须存在**：选择的视频分类必须在分类表中存在，否则提示"分类不存在"。
3. **医生为可选关联**：可以选择性地将视频关联到某个医生——关联后患者端会显示该医生的信息卡片（头像、姓名、职称、科室、医院）。
4. **医生存在性校验**：如果选择了医生，系统会检查该医生是否存在，不存在提示"医生不存在"。
5. **视频名称不可重复**：同一名称的视频不能重复创建，提示"视频名称已存在!"。编辑时可以排除当前视频自身。
6. **名称同步**：新增成功后，视频库中的视频名称也会同步更新。

### 二、编辑视频

1. **三重存在性检查**：视频记录存在→视频库视频存在→分类存在→医生存在（如有）。
2. **名称重复校验**：编辑时检查名称是否与其他视频重复（排除自身ID）。
3. **允许修改的内容**：分类、名称、封面图、视频ID、关联医生。
4. **视频名称同步更新**：编辑名称时同步更新视频库中的name字段。

### 三、视频列表查询（管理后台分页）

1. **多维筛选**：支持按分类、名称（模糊搜索）、启用/禁用状态筛选。
2. **数据组装**：查询结果自动关联分类名称、医生姓名，避免多次查询。
3. **排序**：按创建时间倒序排列。

### 四、视频详情

1. **基本信息**：返回视频的名称、封面、分类名称。
2. **关联医生信息**：如果有关联医生，返回：医生姓名、头像、科室编号/名称、医院编号/名称、职称、医生状态（启用/禁用）。
3. **视频播放信息**：返回视频URL（原始和转码后）、视频时长。

### 五、启用/禁用

1. **启用**：设置状态为1，患者端可见。
2. **禁用**：设置状态为0，患者端不可见。禁用前检查视频是否存在。
3. **即时生效**：状态变更后立即反映到患者端查询结果。

### 六、推荐管理

1. **设置推荐**：设置isRecommend=1，该视频在列表中的排序会优先。
2. **取消推荐**：设置isRecommend=0，恢复正常排序。
3. **不限推荐数量**：没有限制最多可推荐多少个视频。

### 七、播放量统计

1. **自动计数**：患者端每次播放视频时调用play方法，播放量（playCount）自动加1。
2. **原子递增**：直接在当前播放量基础上+1后更新。
3. **视频存在性检查**：播放前检查视频是否存在，不存在则提示"视频不存在"。

### 八、患者端视频展示

1. **findShowList**：患者端查询视频列表的方法，只返回状态为「启用」（status=1）的视频。
2. **可选筛选条件**：
   - categoryId：按分类筛选（可选）
   - isRecommend：只看推荐视频（可选）
3. **数据组装**：每条视频自动关联分类名称、医生完整信息（姓名/头像/科室/医院/职称/状态）、视频URL/时长。
4. **分类列表**：findCategoryShowList 返回所有视频分类的基本信息（ID、名称）。

### 九、医生选择列表

1. **医生查询**：findDoctorList 返回所有可以关联视频的医生列表。
2. **筛选条件**：只返回状态为「启用」且工作类型为「医生」的账号。
3. **返回信息**：医生ID、姓名、科室编号/名称、医院编号/名称、职称。

## 异常处理

| 情况 | 系统处理方式 |
|------|------------|
| 新增时视频在视频库不存在 | 提示"视频不存在"，新增失败 |
| 新增时分类不存在 | 提示"分类不存在"，新增失败 |
| 新增时选择的医生不存在 | 提示"医生不存在"，新增失败 |
| 视频名称重复 | 提示"视频名称已存在!"，新增/编辑失败 |
| 编辑/删除时视频不存在 | 提示"视频不存在"，操作失败 |
| 患者端查询时无满足条件的视频 | 返回空列表，不报错 |

## 关联功能

- **视频库管理**：医生视频依赖基础视频库，先上传视频才能创建医生视频
- **视频分类管理**：视频必须归属于某个分类
- **医生管理**：可将视频关联到医生，患者端显示对应医生信息
- **患者端视频浏览**：小程序中以短视频信息流形式展示
