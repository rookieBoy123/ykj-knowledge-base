# 素材管理流程（服务端）

> 文档版本 2026-07-03 | 面向读者：技术运营、产品经理

## 概述
管理后台的图片素材管理模块，支持将OSS上已上传的图片文件批量登记为系统素材，按分类（文件夹）组织管理，支持素材在分类之间移动、批量保存和批量删除。

## 接口一览

| 接口路径 | 方法 | 说明 | 权限 |
|---------|------|------|------|
| `/mgmt/system/materials/page` | GET | 分页查询素材 | sys:material |
| `/mgmt/system/materials/{id}` | DELETE | 删除单个素材 | sys:material |
| `/mgmt/system/materials/_batch-save` | POST | 批量保存素材 | sys:material:write |
| `/mgmt/system/materials/_batch-delete` | POST | 批量删除素材 | sys:material:write |
| `/mgmt/system/materials/_move` | POST | 移动素材到指定分类 | sys:material:write |

## 素材数据结构

| 字段 | 说明 |
|------|------|
| categoryId | 所属分类ID（即文件夹） |
| type | 素材类型（图片/文件等） |
| name | 素材名称（从URL中自动提取文件名） |
| postfix | 文件后缀名（自动提取，如 jpg、png） |
| url | 素材在OSS上的存储路径（保存时去除域名前缀，读取时自动拼接完整URL） |

## 核心流程

### 批量保存素材
1. 接收一批素材信息（分类ID + 类型 + URL地址）
2. 校验素材列表不能为空
3. 对每个素材URL进行处理：
   - 从URL中提取文件名（FileUtil.getName）
   - 提取文件后缀名（FileUtil.extName）
   - 将URL中的OSS域名去除，只保留相对路径存储
4. 批量插入数据库（事务保护，全部成功或全部回滚）
5. 保存失败返回"保存素材失败"

### 分页查询素材
1. 接收分页参数和可选的分类ID、类型筛选
2. 如果指定了分类ID：
   - 查询该分类下的所有子分类
   - 将当前分类ID和子分类ID合并为查询条件
   - 一次查询出该"文件夹"及其子文件夹内的所有素材
3. 如果指定了类型，按类型精确筛选
4. 按创建时间倒序排列
5. 查询结果中自动拼接OSS域名前缀，返回完整URL

### 删除素材
- 支持单个删除和批量删除
- 物理删除记录（不可恢复）
- 事务保护

### 移动素材
1. 接收素材ID列表和目标分类ID
2. 批量更新这些素材的分类ID为目标值
3. 素材列表为空时直接返回（不报错）
4. 移动失败返回"移动素材失败"

## 异常处理

| 异常情况 | 错误信息 |
|---------|---------|
| 批量保存空列表 | "要保存的素材不能为空" |
| 数据库保存失败 | "保存素材失败" |
| 批量删除失败 | "删除素材失败" |
| 移动素材失败 | "移动素材失败" |

## 权限模型
- 权限前缀：sys:material
- 写入权限：sys:material:write（控制新增、删除、移动）
- 读取权限：sys:material（控制列表查询）

## 关联服务模块
- `ykj-service-system` — 系统管理模块，包含本功能
- 分类管理（SysCategory） — 素材按分类组织，查询时自动包含子分类
- OSS服务 — 存储素材文件的云存储服务，负责URL域名拼接
