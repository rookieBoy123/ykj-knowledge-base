# 外部机构药品订单对接流程

> 文档版本 v1.0 | 面向读者：产品经理、对接机构技术负责人

## 概述

第三方医疗机构通过开放平台（Open API）接入平台药品下单功能。机构可以先进行"预结算"确认费用明细，然后锁定库存生成订单，最后完成支付和发货。适用于已接入平台的合作医院/诊所，通过自有系统发起药品开方和下单。

> ⚠️ 这是一个纯后端对接流程，小程序端和管理后台均不涉及。机构通过开放平台 API 调用接口完成整个订单生命周期。

## 触发条件

- **前置条件**：机构已在开放平台注册（拥有 AppId）、机构设置了需要药品下单（`neededDrugOrder=true`）、机构配置了下单药房（`pharmacyId`）
- **入口**：机构自有系统通过 Open API 调用

## 步骤详解

### 第一步：预结算（确认费用）

机构提交药品清单后，系统进行预结算，让机构确认费用明细。

1. **校验机构信息**：检查 AppId 对应的机构是否存在、是否开启了"需要药品下单"开关、是否配置了下单药房。三项任一不满足都会提示错误，无法继续。
2. **校验库存并计算费用**：查询每个药品的信息（价格、库存），分两种情况：
   - 如果机构药房开启了**第三方药房**（阿康），从第三方查询实时库存和价格，对比后计算。同时检查平台库存与第三方库存是否一致，不一致则自动同步。
   - 如果使用**平台自有药房**，直接查平台库存和价格。
3. **计算费用明细**：
   - **药品金额**：单剂费用 × 总剂数，金额向上取整保留2位小数
   - **运费**：根据收货地址省市和药品总金额，调用运费规则计算
   - **总金额**：药品金额 + 运费
4. **生成预结算单**：保存预结算记录，返回预结算单号、各项费用明细（药品费、运费、总金额、单价）

**结果**：机构得到一份费用明细预览，可以在自己的系统中展示给患者确认。

### 第二步：锁定库存

患者确认费用后，机构调用接口锁定库存。

1. **校验预结算单**：检查预结算单是否存在、是否已经锁定过（不可重复锁定）
2. **二次库存校验**：重新查询库存，防止预结算后被别人买走
3. **若使用第三方药房**：调用阿康下单接口锁定第三方库存（同步传输订单号、收货地址、商品清单）
4. **设置有效期**：默认 5 分钟有效（可通过参数自定义）。过期后定时任务自动取消第三方订单并释放库存。
5. **扣减平台库存**：扣减对应药品的平台库存数量
6. **生成订单号**：为后续正式下单预留订单号

### 第三步：创建订单（正式下单）

机构在有效期内提交正式订单。

1. **校验机构权限**：检查订单所属机构是否匹配（订单不能跨机构操作）
2. **校验处方状态**：
   - 处方必须已通过**医生审核**（DoctorApproval=Pass）
   - 处方必须已通过**药师审核**（PharmacistApproval=Pass）
   - 未审核通过的处方不能创建订单
3. **校验处方有效期**：
   - 如果机构未上传订单创建时间，系统检查处方是否已过期
   - 如果机构上传了订单创建时间，系统检查该时间是否在处方有效期内
4. **校验订单创建时间合法性**：订单创建时间不能早于处方创建时间
5. **校验配送方式**：如果选择自提，必须有药房信息；否则提示"不存在药房信息，不允许自取方式取药"
6. **两种创建方式**：
   - **通过预结算单创建**：校验预结算单有效性（是否已锁定库存、是否已过期、是否已结算）→校验商品信息一致性（预结算商品和处方商品必须完全一致）→调用处方结算服务生成正式订单
   - **忽略预结算直接创建**：跳过预结算校验，直接用处方信息生成订单

### 第四步：支付流水处理

订单创建后，系统根据机构传入的参数处理支付流水。

1. **生成支付流水号**：优先使用机构传入的 `transactionNo`，其次使用 `customOrderNo`，都没有则用订单号+4位随机字符
2. **两种支付渠道处理**：
   - **微信支付**（payType=1）：发布 `PaidEvent` 支付完成事件，通知下游业务（扣库存、更新订单状态等）
   - **线下支付**（payType 其他值）：发布 `OfflinePaidEvent` 线下支付完成事件
3. **回调注册**：如果机构提供了回调地址（`callBackUrl`），系统保存回调配置，后续发货状态变更时会主动回调通知机构

### 第五步：发货与物流同步

机构可以进行发货操作，系统也定时拉取第三方物流信息。

1. **机构主动发货**：
   - 校验机构权限（order归属校验）
   - **自提模式**：输入取件码核销（`pickUpCode` 不能为空）
   - **快递模式**：输入快递公司编码和快递单号（编码从字典"kdgs"中校验有效性）
   - 确认发货后更新订单状态

2. **自动拉取物流**（定时任务 `pullOrderShippintStatus`）：
   - 查询所有第三方药房订单（每次最多20条，分批次拉取）
   - 调用阿康物流查询接口获取快递信息
   - 自动执行确认发货操作

### 第六步：取消订单

机构可以取消药品订单。

1. **校验机构权限**：订单是否属于当前机构
2. **如果是处方订单**：
   - 查询处方对应的药房
   - 如果药房启用了第三方（阿康），先调用阿康取消订单接口
   - 还原平台药品库存（将之前扣减的数量加回来）
3. **处理退款**：
   - 如果订单有过支付流水记录，自动同意退款（退款类型为"手动退款"）
   - 如果订单创建时没有生成支付流水，直接修改订单状态为已取消

### 过期订单自动取消

系统有一个定时任务（`cancelThirdPharmacyExpireOrder`），定期检查已过期的预结算单：
- 找到状态为"未结算"且有效期已过的预结算单
- 如果关联了第三方药房，取消第三方订单并恢复库存
- 标记预结算单为"已取消"

## 异常处理

| 情况 | 系统处理方式 |
|------|------------|
| 机构未开启"需要药品下单" | 后端直接返回错误"机构未设置需要药品下单，不能生成订单！" |
| 机构未配置下单药房 | 后端直接返回错误"机构未设置下单药房，不能生成订单！" |
| 处方药品与药房不一致 | 后端校验药品所在的药房必须是同一个，且与机构配置的药房一致 |
| 第三方库存不足 | 返回"药品库存不足，药品名称：XXX" |
| 同一处方包含多个药房的药品 | 返回"同个处方只能选择一个药房的药品" |
| 预结算单已锁定库存 | 返回"该预约单已锁定过库存！" |
| 预结算单已过期 | 返回"该预结算订单已超过支付有效期！" |
| 预结算单已结算 | 返回"该预结算订单已结算" |
| 处方未审核通过 | 返回"处方未审核通过不允许创建订单" |
| 处方已过期 | 返回"处方已过期,需上传订单创建时间" |
| 订单创建时间不在有效范围 | 返回"订单创建时间不在有效范围内,请修改订单创建时间在处方失效时间之前" |
| 订单创建时间早于处方创建 | 返回"订单创建时间不能在处方创建时间之前" |
| 自提取药却没有药房 | 返回"不存在药房信息，不允许自取方式取药" |
| 预结算商品与处方商品不一致 | 返回"处方和预约单药品不一致！"或"处方和预约单药品数量不一致！" |

## 关联功能

- **处方开具与审核**：处方必须经过医生审核和药师审核后才可下单（ykj-service-inquiry）
- **药品库管理**：药品库存扣减和恢复（ykj-service-medicine）
- **运费规则**：运费计算依赖管理员配置的运费规则（ykj-service-order）
- **第三方药房对接**：阿康药房的订单创建/取消/物流查询接口（ykj-service-open）
- **支付处理**：微信支付/线下支付的事件发布（ykj-service-order）
