# AI医生问诊代理服务流程

> 文档版本：release/v2.17.2 | 面向读者：产品经理、运营、开发

---

## 概述

平台通过独立的AI代理服务实现了智能问诊能力。当患者选择了AI医生进行问诊时，平台后端作为"中间人"把患者请求转发给AI代理服务，再把AI返回的结果呈现给患者。AI代理服务是一个独立部署的智能系统，与平台通过标准HTTP接口通信，支持文字对话、语音转写、处方OCR识别、问诊报告生成等能力。

## 触发条件

- **前置条件**：医院需要开通AI医生功能（`aiDoctorEnabled`开关）、医院创建了AI医生账号
- **入口**：患者在小程序端选择AI医生并发起问诊、管理后台AI问诊记录查看

---

## 步骤详解

### 第一步：开始问诊

1. 患者选择AI医生后，小程序前端调用「开始问诊」接口
2. 后端检查AI功能是否启用（`aiDoctorEnabled`开关），未启用则直接返回空值，患者端会表现为"功能暂不可用"
3. 启用时，后端将患者信息（姓名、性别、年龄、主诉等）组装后发送到AI代理服务的开始问诊接口
4. AI代理服务返回一个问诊会话ID（`pcId`），后续所有对话都围绕这个会话进行

### 第二步：问诊对话（流式响应）

1. 患者在小程序端输入病情描述、症状等信息后发送消息
2. 后端将消息内容、会话ID转发给AI代理服务的对话接口
3. AI代理通过**流式传输**（SSE，Server-Sent Events）逐字返回AI生成的回复
4. 小程序端实时展示逐字打出的AI回复，类似真人打字效果
5. 对话内容包含：
   - AI的追问（了解病情细节）
   - AI的初步分析
   - AI的就诊建议

### 第三步：AI辅助功能

**处方OCR识别**：
1. 医生上传处方照片（可拍照或相册选择）
2. 后端将图片文件转发给AI代理服务的OCR接口
3. AI代理识别并返回处方上的文字内容（药品名、用法用量等）
4. 识别失败时返回空对象

**语音转写**：
1. 患者长按发送语音消息
2. 后端获取语音转写的WebSocket连接地址
3. 通过WebSocket实时将语音流转写给AI代理
4. AI代理返回转写后的文字，继续对话流程

**问诊报告生成**：
1. 问诊对话积累到一定程度后，可以触发报告生成
2. 后端将完整对话记录发送给AI代理的报告生成接口
3. AI代理生成一份结构化的问诊报告（含主诉、现病史、初步诊断等）

### 第四步：问诊记录查询

1. 管理员在后台查看AI问诊记录列表
2. 后端支持按以下条件分页查询：
   - 问诊编号列表（`bizNos`）
   - 日期范围（`startDate` ~ `endDate`）
   - 年龄范围（`minAge` ~ `maxAge`）
   - 患者姓名
   - 业务类型（`bizType`）
3. 支持查看某次问诊的完整对话历史记录
4. 支持通过业务编号（`bizNo`）查找对应的AI问诊详情

### 第五步：结束问诊

1. 当AI问诊流程走完或患者主动结束时
2. 后端调用AI代理服务的状态更新接口，将状态设为已完成（`status=1`）
3. 问诊结束后，AI代理服务不再接受新的对话消息

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| AI医生功能未启用 | 所有AI相关接口静默返回默认空值，不报错，患者端不展示AI功能 |
| AI代理服务调用失败 | 返回默认空值（空字符串/空列表/空对象），记录错误日志，不中断主流程 |
| AI代理返回非200响应 | 解析为业务异常，抛出"agent服务异常"错误 |
| AI代理响应数据解析失败 | 记录详细日志（含原始响应体），抛出"agent服务异常"错误 |
| OCR识别时文件读取失败 | 抛出"文件读取失败"错误，提示具体的IO异常原因 |
| 对话历史为空 | 返回空列表，不做特殊处理 |
| 分页查询结果为空 | 返回空的分页对象，保留total=0 |

## 关联功能

- **AI医生分身管理**：后台创建/管理AI医生的基本信息
- **AI问诊聊天室**：小程序端的AI问诊聊天界面
- **AI预问诊**：患者在正式问诊前通过AI预问诊收集病情信息
- **问诊记录查询**：后台查看AI和真人医生的问诊记录

## 设计要点

1. **开关保护**：所有AI服务调用前都经过 `aiDoctorEnabled` 总开关的统一拦截，关闭后所有AI功能完全静默，不影响平台其他功能正常运行
2. **独立部署**：AI代理服务与平台主服务完全解耦，AI服务的故障不会拖垮平台主服务
3. **SSE流式传输**：对话接口使用SSE（Server-Sent Events）流式传输，实现逐字输出效果，提升患者等待体验
4. **统一异常兜底**：出现任何异常都返回默认空值而非崩溃，保证平台稳定性
