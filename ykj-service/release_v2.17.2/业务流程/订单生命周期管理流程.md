# 订单生命周期管理流程

> 文档版本 v1.0 | 面向读者：产品、运营、客服

## 概述

管理患者在小程序中每一笔订单从创建、支付、发货到完成或退款的完整生命周期。涵盖处方药品订单、问诊服务订单、服务包订单、服务预约订单等多种类型，每种类型的订单在支付后流转路径有所不同。

## 触发条件

- **前置条件**：患者已登录、已选择就诊人、已完成问诊/处方/服务预约
- **入口路径**：处方结算页点击"提交订单"、服务预约页点击"确认预约"、服务包购买页点击"立即购买"等

## 订单类型与流转差异

| 订单类型 | 支付后流转 |
|---------|----------|
| 处方药品订单 | 待支付 → 已支付 → 待发货 → 已发货（待收货） → 已完成 |
| 问诊服务订单 | 待支付 → 已完成 |
| 服务预约订单 | 待支付 → 已完成 |
| 服务包订单 | 待支付 → 待发货 → 已发货（待收货） → 已完成 |

## 步骤详解

### 第一步：订单创建

系统在用户确认购买后自动生成订单：

- **订单号生成**：格式为 O + 日期（8位）+ 7位随机数，全局唯一
- **初始状态**：默认为"待支付"（PENDING_PAYMENT）
- **0元订单特殊处理**：如果订单总金额为 0 元（如免费服务），则跳过支付直接标记为"已完成"（处方类0元订单标记为"已支付"）
- **过期时间计算**：根据订单类型设置不同过期时间，超时未支付自动失效
- **数据记录**：同时创建订单、订单商品明细、收货地址（配送订单）、收费项目明细

### 第二步：支付前准备

用户点击"去支付"时系统进行一系列校验：

- **状态校验**：只有"待支付"状态的订单才能支付；已支付或已失效订单会提示"当前订单已支付"或"订单已失效"
- **历史渠道防重复查单**：系统自动查询之前是否有过支付记录但回调未到达的情况，发现已支付则补单并阻止重复支付
- **支付渠道组装**：
  - 默认支持：微信支付（首选）、微信扫码支付、支付宝
  - 医保支付：当医保支付开关开启、订单类型为处方订单且非定金订单时，增加医保支付选项
- **支付金额**：总金额减去已支付金额（如定金已付）

### 第三步：执行支付

- **并发保护**：使用分布式锁（Redisson），同一订单同时只能有一个支付请求在处理；并发请求会提示"支付处理中，请稍后重试"
- **预支付前置清理**：关闭旧的未完成预支付单、清除跨渠道缓存
- **策略分发**：按选择的支付渠道（微信/支付宝/医保）调用对应的支付服务

### 第四步：支付完成（回调处理）

用户完成支付后，支付平台回调通知系统：

- **幂等保护**：同一笔支付交易（通过transactionId或payOrderNo判断）只处理一次，重复回调自动跳过
- **重复支付自动退款**：如果订单已支付但收到第二笔支付（如跨渠道重复支付），系统自动发起退款，不保留多余资金
- **状态更新**：
  - 处方订单 → 已支付（PAYED），等待后续发货流程
  - 服务包订单 → 待发货（PENDING_SHIPMENT）
  - 其他类型 → 已完成（COMPLETED）
- **流水记录**：记录收入流水（含支付渠道、交易号、支付金额、支付时间等）

### 第五步：订单取消

订单取消分为多种场景：

- **可取消状态**：待支付、已支付、待发货（不同订单类型取消条件不同）
- **已完成状态取消限制**：只有问诊类和服务预约类订单在已完成状态可以取消，其他类型不可取消
- **超时取消**：标记为"已过期"（EXPIRED）状态
- **退款处理**：已支付订单取消时自动生成退款申请单，走退款审核流程
- **第三方通知**：处方订单取消时选择通知第三方，推送取消事件给外部系统

### 第六步：发货与收货（处方订单）

- **发货**：药师审核通过后，药房发货，状态变为"已发货（待收货）"
- **自动确认收货**：定时任务每日检查，发货超过15天且未确认的订单自动标记为"已完成"
- **线下取货**：患者凭取货码到药房取货，校验取货码正确且订单未取货后完成收货

### 第七步：退款

- **退款申请**：退款金额不能大于实际支付金额
- **医保退款**：
  - 先查询是否已做过医保退费
  - 如未退过，先调用医保平台退还医保部分，再退微信支付部分
  - 如已退过医保部分，直接退微信支付部分
  - 退款成功添加支出流水记录

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 订单生成失败 | 提示"订单生成失败，请联系管理员" |
| 订单状态不可支付（已支付） | 提示"当前订单已支付！" |
| 订单已失效 | 提示"订单已失效！" |
| 支付处理中（并发） | 提示"支付处理中，请稍后重试" |
| 取货码错误 | 提示"取货码有误" |
| 订单已取货 | 提示"该订单已取货" |
| 订单未付款申请退款 | 提示"订单未成功付款，不支持申请退还金额" |
| 退款金额超过支付金额 | 提示"申请退款金额不能大于实际支付金额" |
| 订单已绑定不同会员 | 提示"订单已绑定会员" |

## 关联功能

- 处方结算流程（处方 → 订单创建）
- 支付模块（微信/支付宝/医保）
- 物流管理（发货/收货/物流信息回调）
- 退款审核与处理流程
- 发票管理（订单完成后的发票申请）
- 定时任务（过期订单处理、自动确认收货）
