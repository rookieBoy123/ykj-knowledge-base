# 处方预结算与费用计算流程

> 文档版本 2026-07-02 | 面向读者：产品经理 / 技术运营

## 概述

当患者查看处方并准备下单时，系统需要进行预结算计算药品费用、加工费、运费等各项费用，生成最终应付金额。预结算结果在前端展示给患者确认，确认后调用结算接口创建订单。

## 触发条件

- **前置条件**：处方已开具（状态可结算）、患者已选择配送方式
- **触发时机**：患者进入处方下单页面时自动触发

## 步骤详解

### 第一步：预结算（preSettlement）

系统接收到预结算请求后依次执行：

1. **查找处方**：根据处方编号（outpatientNo）查询处方信息
2. **校验处方状态**：通过 `verifyRecordSettlementStatus` 确保处方处于可结算状态
3. **获取药房信息**：根据处方绑定的药房 ID 查询药房，包含代煎服务是否启用和价格
4. **获取收货地址**：根据配送方式查询会员的收货地址（快递方式需要地址信息）
5. **确认加工服务费**：检查中药是否有选定的加工工艺（颗粒剂等），有则计算加工费
6. **计算费用项**（`computationalCosts`）：
   - 药品费用：根据药品单价 × 数量计算
   - 服务费：是否合并到药品金额
   - 加工费：中药加工工艺费用
   - 代煎费：如果药房启用代煎服务且处方为中药类型
   - 运费：根据收货地址计算快递费用
   - 优惠减免：各项费用的调整
7. **汇总金额**：所有费用项的总金额求和，如果出现负数则置为 0
8. **订金判断**：如果药房启用订金功能且订单金额达到最低门槛，标记支持订金支付

返回完整的预结算结果包含：药房信息、收货地址、费用明细列表、总金额、订金标识、配送方式

### 第二步：结算并创建订单（settlement）

患者确认预结算结果后触发：

1. **防重复校验**：检查该处方是否已存在有效订单，重复则提示「已存在订单信息」
2. **重新预结算**：以不合并服务费的方式重新计算费用项
3. **确认加工服务费**：再次确认加工工艺和费用
4. **创建订单**：
   - 订单来源类型设为 `PRESCRIPTION`（处方来源）
   - 订单状态初始为待支付
   - 支持自定义订单号（第三方系统对接时使用）
   - 生成订单编号
5. **回填处方订单编号**：将订单编号记录到处方中

### 第三步：费用项构成

费用项（CostItems）包含：
- **药品费**：各药品的数量 × 单价汇总
- **医事服务费**：医生的诊查服务费
- **加工费**：中药加工工艺费用（如制成颗粒剂）
- **代煎费**：药房代煎服务的费用（仅中药处方且药房启用代煎时计算）
- **运费**：快递配送的物流费用
- **优惠**：如有平台优惠活动
- **医保支付明细**：医保统筹支付、个人账户支付、微信自费等分项

### 第四步：订单详情查询

创建订单后可通过订单详情接口查看：
- 订单基本信息（编号、状态、金额）
- 费用明细列表（各分项的金额）
- 支付渠道信息（微信支付/医保支付/混合支付）
- 收货地址信息（快递配送方式）
- 药房信息（含代煎服务）
- 订金信息（是否支持订金支付）
- 医保扩展信息（是否跳转医保小程序）

## 异常处理

| 情况 | 处理方式 |
|------|---------|
| 处方状态不可结算 | 后端拦截，返回错误 |
| 已存在有效订单 | 提示「已存在订单信息」 |
| 处方不存在 | 提示「处方不存在, 无法创建订单」 |
| 药房信息不存在 | 提示「找不到对应药房信息」 |
| 订单费用项信息缺失 | 提示「订单费用项信息不存在！」 |
| 收货地址不存在（快递） | 提示「该订单收货地址信息不存在」 |
| 金额为负数 | 自动置为 0 |

## 关联功能

- **订单创建与支付**（ykj-service-order）：订单全生命周期
- **药品管理**（ykj-service-medicine）：药品信息查询
- **药房管理**（ykj-service-medicine）：药房配置查询
- **会员服务**（ykj-service-member）：收货地址管理
- **医保支付**（ykj-service-yb）：微信医保支付和结算上报
