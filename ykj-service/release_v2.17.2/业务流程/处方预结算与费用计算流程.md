# 处方预结算与费用计算流程

> 文档版本 2026-07-02 | 面向产品/开发

## 概述

当患者从处方详情页进入下单，或医生代患者支付时，系统会触发处方结算流程。该流程负责：校验处方状态、计算药品费用/加工费/运费/定金、创建订单。医保场景下还会涉及费用明细上传、个账支付判断和支付下单。

## 触发条件

- 患者在处方详情页点击"付款"或"确认领取处方并支付"
- 医生在处方详情页点击"代支付"
- 第三方机构调用结算接口

## 预结算流程（preSettlement）

### 第一步：校验处方状态
- 查询处方记录（按处方编号outpatientNo）
- 校验处方结算状态是否合法
  - 不能重复结算（已有该处方的有效订单时拒绝，"已存在订单信息"）
  - 处方必须存在且有效

### 第二步：组装结算信息
逐项组装预结算结果内容：

| 结算项 | 来源 | 说明 |
|--------|------|------|
| 药房信息 | pharmacyService | 药房名称、配送方式、代煎服务 |
| 收货地址 | receiveAddressService | 用户默认地址 |
| 加工服务费 | processing确认 | 中药饮片的炮制/代煎费用 |
| 运费 | freightService | 按收货地址+药房计算 |
| 定金 | 配置项 | 药房启用定金+系统金额门槛 |

### 第三步：费用计算
系统将费用拆分为明细项（CostItems）：
1. **药品金额**：药品单价 × 数量，按药品条目逐项计算
2. **加工服务费**：中药配方的煎药费/制粉费等
3. **运费**：按地区+药房的运费规则计算
4. **定金**：满足条件时（药房启用定金 + 订单金额 >= 系统最低定金门槛），先付定金

总金额 = 所有费用项相加。如果出现负数则置为0。

## 正式结算流程（settlement）

1. 检查是否已有该处方的有效订单 → 有则拒绝"已存在订单信息"
2. 执行预结算，获取完整费用明细
3. 订单费用项中药品金额不合并服务费（便于后续对账）
4. 确认加工服务费
5. 创建订单：组装订单信息（订单号/来源类型/费用项/收货地址/支付渠道）
6. 将订单号回填到处方记录中

## 医保预结算流程（insurancePrepay）

在普通结算基础上，增加医保特有的处理：

### 第一步：费用明细上传
- 获取处方药品列表 → 查询国家药品编码
- 获取诊断信息（中医诊断/辨病/辨证）
- 获取患者医保信息（个人编号、医保区划）
- 组装上传费用明细（ExpenseInfo）

### 第二步：诊费处理
- 如果问诊产生了诊费，将诊费也加入费用明细上传
- 诊金超过10元按10元计算（医保规定）

### 第三步：个人账户判断
- 检查系统是否支持个人账户支付（isSupportIndividualAccounts）
- 用户选择"不使用个人账户" → isIndividualAccount设为"0"
- 用户选择"使用" → 保持为"1"

### 第四步：处理历史费用
- 如果之前上传过费用明细 → 先取消历史上传 → 删除旧记录
- 确保每次只有一个有效的费用上传记录

### 第五步：授权码解析
- 如果提供了授权码（authCode）→ 调用微信医保接口解析获取支付授权号和坐标
- 如果无授权码但有历史支付扩展信息 → 复用旧的授权号（切换个账场景）
- 两者都无 → 拒绝"授权码不能为空"

### 第六步：费用明细上传 + 支付下单
- 调用医保平台接口上传费用明细 → 获取支付令牌和订单ID
- 调用支付下单接口 → 获取微信跳转参数（AppId+路径）
- 持久化医保订单预结算信息
- 返回支付结果（现金支付金额 = 订单总金额 - 医保支付金额 + 附加金额）

## 订单详情查询（getTreatmentOrderDetails）

系统支持通过处方编号查询已创建订单的完整信息：
1. 查询订单基本信息（金额、支付渠道）
2. 查询费用明细（各费用项的金额和说明）
3. 查询收货地址（配送场景）
4. 查询支付流水（医保支付渠道单独列出）
5. 判断是否支持定金支付

## 医保报销明细查询（findReimbursementDetails）

- 仅医保订单支持查询
- 查询订单 → 处方 → 问诊 → 患者 → 医保结算单
- 调用医保平台查询结算明细
- 将明细数据生成PDF图片返回

## 状态校验规则

| 校验场景 | 规则 | 失败提示 |
|---------|------|---------|
| 重复结算 | 同处方的有效订单只能有一个 | "已存在订单信息" |
| 处方不存在 | 结算前必须存在有效处方 | "处方不存在，无法创建订单" |
| 药房不存在 | 处方关联的药房必须存在 | "找不到对应药房信息" |
| 订单不存在 | 查询订单详情时必须存在 | "订单信息不存在！" |
| 费用项不存在 | 订单必须有费用明细 | "订单费用项信息不存在！" |
| 地址不存在 | 配送模式必须有地址 | "该订单收货地址信息不存在" |
| 医保非医保订单 | 非医保订单不支持查报销明细 | "该订单不支持查询明细" |
