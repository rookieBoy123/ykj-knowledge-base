# 系统配置批量更新与权限控制流程（服务端）

> 文档版本：2026-07-04 | 面向读者：系统管理员

## 概述

系统配置管理提供了全局参数的查询和批量更新功能。管理员可一次性修改多项配置（如功能开关、业务参数等），系统自动记录配置变更历史，支持回滚到历史版本。

对应的管理后台页面：全局配置管理（`systemmanage/system/settings`）。

## 核心接口

| 接口 | 方法 | 说明 |
|------|------|------|
| `/mgmt/settings` | GET | 获取全部管理端配置项列表 |
| `/mgmt/settings` (batch_update) | POST | 批量更新配置项 |

配置列表接口同时挂载在 `/mgmt/setting/groups` 路径下，支持按分组查询。

## 关键设计

### 1. 权限控制

批量更新接口需要 `sys:settings:write` 权限，通过 `@PreAuthorize` 注解控制：
```java
@PreAuthorize("hasAuthority('sys:settings:write')")
```

无此权限的管理员角色无法修改任何配置项。查询接口不需要特殊权限。

### 2. 批量更新机制

- 接口接收一个 `UpdateItem` 列表，每项包含 `property`（配置属性名）和 `value`（修改值）
- 两个字段均使用 `@NotNull` 校验，不能为空
- 如果整个更新列表为空，返回 "更新配置项不能为空" 错误
- 更新操作由 `SysSettingMgmtService.updateSettings()` 执行

### 3. 配置变更历史

每次批量更新通过 `SysSettingMgmtService` 处理，该 Service 负责：
- 将新配置值写入数据库
- 自动记录变更历史到 `sys_setting_history` 表
- 支持管理后台查看历史版本并还原

### 4. 配置读取方式

查询接口返回的是 `SettingService.getManageSettings()` 方法的结果，这个方法从配置中心读取实时的全局配置，确保管理后台展示的始终是最新值。

## 异常处理

| 异常场景 | 后端处理方式 |
|----------|-------------|
| 更新列表为空 | 抛出 BizException("更新配置项不能为空") |
| 单个配置项的 property 为空 | @NotNull 校验拦截，返回参数校验错误 |
| 单个配置项的 value 为空 | @NotNull 校验拦截，返回参数校验错误 |
| 无 sys:settings:write 权限 | @PreAuthorize 拦截，返回403权限不足 |

## 关联模块

- `ykj-service-system` — 系统配置模块
- `SysSettingMgmtService` — 配置管理业务逻辑（含历史记录）
- `SettingService` — 配置读取服务
- 管理后台「全局配置管理」页面
