# 会诊生命周期管理流程

> 文档版本：v1.0 | 最后更新：2026-07-01 | 面向读者：产品/运营

## 概述

会诊功能支持多位医生围绕同一患者进行协作诊疗，分为两种模式：群会诊（医生发起邀请其他医生加入）和机构间远程会诊（机构A发起邀请机构B参与）。整个流程由服务端的会诊模块统一管理，包括会诊创建、医生邀请、视频通话、会诊报告和电子签章。

---

## 会诊状态流转

```
待审核 → 待开始 → 进行中 → 待填写报告 → 已结束（正常完成）
                                            ↘ 已结束（取消/拒绝）
```

---

## 步骤详解

### 第一步：会诊发起

医生在问诊过程中发起会诊：

- 系统创建会诊记录，分配会诊单号
- 记录申请人（主治医生）、患者信息、病情描述
- 指定会诊时间（会诊开始时间）
- 邀请其他医生加入（通过医生搜索或从机构中选择）
- 生成会诊订单并引导支付（如有费用）

### 第二步：审核与确认

会诊发起后有审核等待期：

- **参会医生审核**：受邀医生审核会诊申请，决定是否参加
- **患者确认**：患者确认会诊安排
- **超时处理**：
  - 距会诊开始30分钟仍未审核：系统自动取消（超时未审核）
  - 距会诊开始15分钟仍未支付：系统自动取消（超时未支付）
- 审核通过后，会诊状态变为「待开始」

### 第三步：会诊进行

会诊时间到达时进入「进行中」：

- **会诊前提醒**：系统在会诊开始前10分钟向所有参与方发送准备通知
- **进入视频房间**：各方通过腾讯TRTC加入视频/音频通话
- **即时通讯群组**：系统为会诊创建专属IM群组，支持文字/图片交流
- **影像文件查看**：可上传和查看DICOM等医学影像文件

### 第四步：填写会诊报告

会诊结束后进入报告填写阶段：

- **主治医生**：填写会诊报告（完整诊断意见）
- **参会医生**：填写会诊意见（个人意见和建议）
- 报告录入采用大文本输入框，空白状态禁用提交按钮
- 主治医生提交时收到二次确认："提交后参会医生不能再填写"
- 所有参会医生填写完毕后，报告完整

### 第五步：电子签章

会诊报告完成后进行电子签名：

- 医生通过CA认证完成电子签章
- 签章平台支持：医网信、源康健PaaS、放心签
- 签章完成后报告PDF加盖电子印章
- 患者可在小程序端查看签章后的会诊报告

### 第六步：会诊结束

- 全部流程完成后，会诊标记为「已结束」
- 系统计算会诊相关的医生收益
- 会诊记录归档保存

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 会诊未支付 | 超时自动取消，订单作废 |
| 会诊未审核 | 超时自动取消 |
| 会诊进行中被取消 | 记录取消原因，终止视频房间 |
| 患者未确认 | 会诊无法开始，等待确认 |
| 报告未填写 | 会诊保持在「待填写报告」状态 |
| 签章失败 | 记录失败原因，允许重试 |
| 影像文件过大 | 可能需要压缩或分片上传 |

---

## 关联功能

- **问诊管理**：会诊通常在问诊过程中发起
- **即时通讯（IM）**：会诊的群聊功能
- **实时音视频（TRTC）**：视频会诊的通话服务
- **CA签章**：会诊报告的电子签名
- **订单管理**：会诊费用的订单处理
- **收益结算**：参会医生的会诊收益计算
- **订阅消息**：会诊通知的消息推送
