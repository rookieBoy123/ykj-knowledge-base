# 线下药房订单管理流程（服务端）

> 文档版本 2026-07-04 | 面向技术人员/产品经理

## 概述

管理线下药房的药品订单全生命周期：订单查询、发货确认、收货确认、快递单号更新、收件地址修改、取货码查询与取货核销。

- 代码来源：`MgmtOfflineOrderController.java` (163行) + `OrderCommandServiceImpl.java` (1308行，核心方法) + `OrderQueryServiceImpl.java` (352行)
- 服务模块：`ykj-service-server` → `ykj-service-order`

## 核心接口

| 接口 | 方法 | 权限 | 说明 |
|------|------|------|------|
| `/mgmt/offline/orders` | GET | `order:offline-orders:read` | 分页查询线下药房订单列表 |
| `/mgmt/offline/orders/confirmShipment` | POST | `order:offline-orders:write` | 确认发货 |
| `/mgmt/offline/orders/confirmReceipt` | POST | `order:offline-orders:write` | 确认收货 |
| `/mgmt/offline/orders/{orderNo}/expressInfo` | PUT | `order:offline-orders:write` | 更新快递单号 |
| `/mgmt/offline/orders/{orderNo}/receiverAddress` | PUT | `order:offline-orders:write` | 修改收货地址 |
| `/mgmt/offline/orders?pickUpCode=` | GET | `order:offline-orders:read` | 根据取货码查询订单 |
| `/mgmt/offline/orders/pickUp` | POST | `order:offline-orders:write` | 线下取货核销 |

## 步骤详解

### 1. 订单列表查询

查询线下药房订单，支持以下筛选维度：
- 收货人电话、收货人姓名
- 订单编号、物流单号
- 开方医生姓名
- 日期范围
- 药房名称、物流公司（配送方式）
- 类型：到店取货 / 物流快递
- 付款方式：全额 / 订金

**数据隔离**：自动过滤为当前登录管理员所属药房的订单（`AdminSubjectUtils.getPharmacyId()`）。

### 2. 确认发货

管理员确认订单已发货。系统：
1. 先校验订单状态（`orderValidation.isApproved()`），确保订单审核通过
2. 设置发货人ID和姓名（从当前管理员获取）
3. 调用 `OrderCommandService.confirmShipment()` 执行发货逻辑

### 3. 确认收货

管理员确认患者已收到药品。调用 `OrderCommandService.confirmReceipt(orderNo, null)`。

### 4. 更新快递单号

修改订单的快递单号和快递公司信息。操作记录到审计日志：
"订单修改快递单号，订单编号【xxx】，快递单号【xxx】"

### 5. 修改收货地址

修改订单的收货地址信息（收货人、电话、详细地址等）。操作记录到审计日志：
"修改订单收货地址，订单编号【xxx】"

### 6. 取货码管理

**根据取货码查询**：输入取货码查询对应订单，用于线下取货确认前的订单查找。

**线下取货**：确认患者到店取货。系统：
1. 接收取货码（`pickUpCode`，不能为空）
2. 调用 `OrderCommandService.pickUp()` 执行取货核销

## 权限体系

所有接口均有 `@PreAuthorize` 权限校验：
- **读取权限**：`order:offline-orders:read`
- **写入权限**：`order:offline-orders:write`

药房数据隔离通过 `AdminSubjectUtils.getPharmacyId()` 自动实现，管理员只能看到本药房的订单。

## 异常处理

| 情况 | 处理方式 | 位置 |
|------|---------|------|
| 确认发货时订单未审核 | 校验失败返回错误（后端拦截） | Controller L56 |
| 取货码为空 | 参数校验失败，提示"取件码不能为空"（后端拦截） | PickUpReq @NotEmpty |
| 订单号为空 | 参数校验失败，提示"订单号不能为空"（后端拦截） | OrderReceiptReq @NotEmpty |
| 取货码不存在 | 查询返回空或执行失败 | Service层 |

## 关联功能

- **药品库存**：发货后扣减库存
- **物流跟踪**：更新快递单号后可通过物流接口查询轨迹
- **订单审核**：确认发货前需订单审核通过
- **审计日志**：快递单号修改和地址修改记录操作审计
