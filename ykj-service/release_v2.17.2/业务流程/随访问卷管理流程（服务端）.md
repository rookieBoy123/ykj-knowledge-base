# 随访问卷管理流程（服务端）

> 文档版本 2026-07-04 | 面向读者：产品经理、后端开发对接方

## 概述

随访问卷管理提供问卷的创建、编辑、题目管理、状态变更和查询功能。对应服务模块 `ykj-service-followup`，接口前缀 `/mgmt/followup/questionnaires`。管理后台通过此接口完成随访问卷的全生命周期管理。

## 接口清单

| 接口 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 分页查询 | GET | `/mgmt/followup/questionnaires` | 按标题/状态/时间范围筛选 |
| 查看详情 | GET | `/mgmt/followup/questionnaires/{id}` | 返回问卷+题目完整数据 |
| 保存问卷 | POST | `/mgmt/followup/questionnaires` | 新建问卷（标题+说明） |
| 修改问卷 | PUT | `/mgmt/followup/questionnaires/{id}` | 编辑问卷基本信息 |
| 编辑状态 | PUT | `/mgmt/followup/questionnaires/{id}/edit-status` | 发布/禁用 |
| 保存题目 | POST | `/mgmt/followup/questionnaires/_save-item` | 新增题目+选项 |
| 修改题目 | PUT | `/mgmt/followup/questionnaires/_edit-item` | 编辑题目内容 |
| 删除题目 | DELETE | `/mgmt/followup/questionnaires/_delete-item` | 删除题目 |
| 修改题目序号 | PUT | `/mgmt/followup/questionnaires/_edit-item-number` | 上移/下移 |
| 查看题目详情 | GET | `/mgmt/followup/questionnaires/_findItemByItemId` | 按题目ID查看 |

## 核心流程

### 1. 问卷创建

1. 前端提交问卷标题（title）和说明（remarks）
2. 权限校验：`fup:questionnaires:write`
3. 保存到 `followup_questionnaire` 表，状态默认草稿（status=0）
4. 返回问卷ID，前端用此ID进入题目编辑

### 2. 题目管理

每个题目（FollowupQuestionnaireItem）包含：
- **基础属性**：名称（name）、题目编号（questionNo）
- **选项内容**：以JSON数组形式存储在 contentJson 字段，如 `[{label: "选项A", value: "A"}, ...]`
- **排序**：通过 editItemNumber 接口调整，参数为 -1（上移）或 1（下移）

题目与问卷之间是一对多关系（questionnaire_id 外键）。

### 3. 状态流转

状态采用三态枚举（Status）：
```
草稿(0) → 已发布(1) ↔ 禁用(2)
```

- **草稿→已发布**：调用 editStatus，后端返回检查结果。如果问卷内容不完整，返回 status=false，前端弹窗提示
- **已发布→禁用**：直接变更，无需内容检查
- **禁用→已发布**：同草稿→已发布，需内容完整性检查

### 4. 权限控制

所有写操作（新增/编辑/删除/状态变更）需要权限 `fup:questionnaires:write`，读操作需要 `fup:questionnaires:read`。分页查询复用 DefaultPageEndpoint 的 `:page` 权限后缀。

## 异常处理

| 情况 | 处理方式 | 错误来源 |
|------|---------|---------|
| 删除题目时未传ID | 抛出 BizException "删除题目id不能为空" | 后端校验 |
| 编辑题目时未传ID | 抛出 BizException "编辑题目id不能为空" | 后端校验 |
| 序号参数不合法 | @Pattern 校验，upOrDown 只能是 -1 或 1 | 参数校验 |
| 问卷内容不完整 | 不抛异常，返回 status=false 由前端展示 | 业务逻辑 |
| 权限不足 | Spring Security 拦截，返回 403 | 框架层 |

## 关联模块

- **随访任务**（followupTask）：使用已发布的问卷发给患者
- **随访提醒**（followupReminder）：关联问卷模板定时发送
