# 订单发票自动开具与管理流程（服务端）

> 文档版本 v1.0 | 面向读者：后台开发/财务管理员

## 概述
订单发票管理是平台财务体系的重要组成部分。当患者完成药品订单支付后，可申请开具电子发票。服务端负责发票的生成、存储、查询和导出，支持发票的开具、红冲（作废），并与第三方发票服务商对接完成税控发票的生成。

## 触发条件
- **发票申请**：患者在小程序端提交发票申请
- **发票开具**：服务端调用第三方发票接口生成电子发票
- **发票查询**：管理员在后台或患者在小程序端查询发票
- **权限要求**：管理端需要发票管理权限

## 核心步骤

### 1. 发票申请
- 患者完成药品订单支付后，在订单详情页点击"申请发票"
- 填写发票抬头（个人/企业）、税号（企业必填）、邮箱等信息
- 提交发票申请后，系统生成发票申请记录

### 2. 发票开具
- 服务端接收发票申请后，组装发票数据（订单金额、药品明细、税率等）
- 调用第三方电子发票服务商接口开具电子发票
- 开具成功后，发票文件（PDF）存储到云存储，发票信息记录到数据库
- 同时发送邮件通知患者（附发票下载链接）

### 3. 发票查询
**管理端查询**：
- 按发票号码、订单编号、发票状态（已开具/已红冲）、开具时间等条件查询
- 支持发票数据导出为Excel
- 查看发票详细信息

**患者端查询**：
- 在"我的发票"页面查看所有已开具的发票列表
- 点击可预览PDF发票

### 4. 发票红冲（作废）
- 当订单发生退款或发票信息有误时，可进行发票红冲操作
- 红冲后原发票失效，如需重新开具需重新申请
- 红冲操作需在税务规定的时间窗口内完成

### 5. 发票导出
- 管理员可按条件导出所有发票数据
- 导出格式为Excel，包含发票号码、订单编号、金额、开具时间、状态等

## 数据字段说明
- **发票状态**：已开具、已红冲、开具中、开具失败
- **发票类型**：增值税电子普通发票、增值税电子专用发票
- **发票抬头**：个人姓名或企业全称
- **发票金额**：与订单实付金额一致

## 异常处理
| 情况 | 处理方式 |
|------|---------|
| 第三方发票接口调用失败 | 发票状态标记为"开具失败"，系统自动重试 |
| 订单已全额退款 | 不允许申请发票，或已开发票自动红冲 |
| 发票红冲超过时限 | 提示无法红冲，需联系财务处理 |
| 税号格式错误 | 前端校验拦截，企业发票必须填写正确税号 |
| 邮箱格式错误 | 前端校验，格式不正确无法提交 |

## 关联功能
- **订单管理**：发票与药品订单关联
- **退款管理**：退款触发发票红冲流程
- **收款清单**：发票金额与收款金额对账
