# 医保结算数据上传流程（服务端）

> 文档版本 v1.0 | 面向读者：开发人员、产品经理

## 概述

医保结算数据上传模块负责将慢病医保患者的结算记录批量上传到医保监管平台。系统通过加密存储身份证号、支持分页查询和批量上传操作来管理结算数据的申报生命周期。

服务模块：`ykj-service-yb`（医保模块）

## 触发条件

- 处方结算完成后生成医保结算记录
- 医保专员在管理后台发起上传操作
- 单条上传或批量上传触发后端接口

## 业务流程

### 一、数据生成

处方结算流程中，系统将患者的医保结算信息存储到 `UploadInsuranceSettlementPo` 表，初始状态为"待上传"（status=1）。

关键字段：人员编号、患者姓名、科室、病种、身份证号（加密存储）、结算时间、性别、年龄等。

### 二、分页查询

**接口**：`POST /mgmt/gzyb/declares/_page`

**权限**：`medins:declares:get`

**查询参数**：

| 参数 | 说明 |
|------|------|
| realName | 患者名称（模糊搜索） |
| startTime/endTime | 结算日期范围 |
| inquiryPersonNo | 人员编号 |
| chronicCode | 病种编码 |
| idCard | 身份证号（前端输入 → 后端加密再查询） |
| departmentNo | 科室编码（树形结构） |
| declareStartTime/declareEndTime | 申报日期范围 |
| status | 上传状态：1-待上传, 2-已上传, 3-上传失败 |

**特殊处理**：身份证号搜索 → 后端使用 `crypto.encrypt()` 加密后再查询数据库（因数据库中身份证为加密存储）。

### 三、批量上传

**接口**：`POST /mgmt/gzyb/declares/_batch-upload`

**权限**：`medins:declares:upload`

**请求参数**：
- `ids`：选中记录 ID 列表
- `declareTime`：申报日期

**处理逻辑**（在 Assembler 中实现）：
1. 根据 ID 列表查询所有待上传记录
2. 逐条或批量调用医保监管平台申报接口
3. 申报成功 → 更新状态为"已上传"（status=2）
4. 申报失败 → 更新状态为"上传失败"（status=3）

### 四、查看详情

**接口**：`POST /mgmt/gzyb/declares/_detail`

**权限**：`medins:declares:get`

返回单条结算记录的完整明细（DeclareDetailsVo），包括病人信息、诊断信息、费用明细等。

### 五、状态说明

| 状态码 | 状态名称 | 说明 | 可执行操作 |
|--------|---------|------|-----------|
| 1 | 待上传 | 新生成的结算记录 | 上传/批量上传 |
| 2 | 已上传 | 成功申报到医保平台 | 查看详情 |
| 3 | 上传失败 | 申报过程出错 | 重新上传/查看详情 |

## 异常处理

| 情况 | 处理方式 |
|------|---------|
| 上传失败 | 记录状态变为"上传失败"，允许重新上传 |
| 权限不足 | `@PreAuthorize` 拦截，返回 403 |
| 身份证号查询 | 加密后再数据库匹配（前端明文 → 后端加密 → 数据库密文比对） |
| 批量上传部分失败 | 单条失败不影响其他记录上传 |

## 关联功能

- **处方结算**（ykj-service-inquiry）：处方结算时生成医保数据
- **慢病病种管理**：关联的病种在管理后台独立管理
- **医保诊疗类别**：结算中的诊疗类别对应 `diagnosis` 字典表
- **管理后台-医保结算管理页**：前端操作入口
