# 外部问诊HIS事件容错与患者信息新增机制

> 文档版本 2026-07-06 | 面向产品经理、运营人员

## 概述

外部机构（如合作医院、第三方平台）通过开放API发起问诊时，系统涉及患者信息创建、HIS系统数据同步、问诊单创建等多个环节。本次更新（v2.17.2）优化了两个关键机制：**患者信息从"查找已有"改为"直接新增"**，以及**HIS事件处理从"失败中止"改为"异常容错"**。

## 触发条件

- 外部机构通过API调用问诊发起接口
- 传入患者姓名、身份证号、性别、年龄等基本信息
- 传入病情描述、病历图片等问诊资料

## 核心变更

### 变更一：患者信息处理逻辑（`ApiInquiryInitiateServiceImpl`）

**旧逻辑**：外部问诊发起时，系统先按"姓名+身份证号"查找是否已存在该患者。如果找到，使用已有患者信息；如果找不到，再创建新患者。

**新逻辑**：外部问诊发起时，如果患者提供了身份证号，系统**始终直接新增**一条患者记录，不再查找已有记录。新增成功后将系统生成的`patientId`回填到问诊单中。

**例外情况**：如果新增患者失败（如因手机号格式问题抛异常），系统仅记录警告日志，不会阻断问诊流程。此时`patientId`为空，但问诊单仍然创建。

### 变更二：HIS事件异常容错（`HisHandler`）

**旧逻辑**：患者注册后，系统向HIS发送建档指令；问诊开始后，系统向HIS拉取数据。如果HIS操作失败，异常会向上传播，影响患者注册或问诊开始的正常流程。

**新逻辑**：两个HIS操作（患者HIS建档、问诊开始后HIS数据同步）均被`try-catch`包裹。即使HIS返回错误或超时，系统仅记录一条告警日志（例如"患者注册成功后HIS建档失败"），**不会影响问诊的正常发起和进行**。

## 影响范围

| 影响点 | 说明 |
|--------|------|
| 患者数据重复 | 同一外部机构多次发起问诊时，即使患者姓名+身份证号相同，系统也会创建多条患者记录（而非复用已有记录） |
| HIS数据缺失 | 当HIS系统不可用时，外部问诊仍可正常进行，但患者档案中可能缺少HIS同步数据 |
| 问诊成功率 | 提升了外部问诊在HIS异常场景下的成功率，不会因HIS故障导致问诊发起失败 |
| 患者归属 | 外部机构发起的患者记录与小程序端自行添加的患者记录是独立的 |

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 新增患者失败 | 记录警告日志，`patientId`置为空，继续创建问诊单 |
| HIS患者建档失败 | 记录告警日志"患者注册成功后HIS建档失败"，不影响问诊流程 |
| HIS数据同步失败 | 记录告警日志"问诊开始后同步到HIS失败"，不影响问诊进行 |
| 查询不到机构信息 | 终止流程，返回错误"找不到对应的机构信息"（非容错，属于前置校验） |
| 患者信息参数缺失 | 终止流程，返回错误"参数缺失, 请正确填写患者信息"（非容错） |

## 关联功能

- 开放平台创建问诊与信息补充流程
- 外部机构对接问诊流程
- HIS数据同步与黔康码获取流程
