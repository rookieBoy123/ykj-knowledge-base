# 商务管理流程（服务端）

> 文档版本 2026-07-04 | 面向技术人员/产品经理

## 概述

商务人员是连接平台与医生的纽带——他们通过分享专属二维码邀请医生入驻平台。本流程覆盖商务的创建、编辑、审核、二维码管理及审核后的微信通知推送。

- 代码来源：`MgmtBusinessController.java` (149行) + `BusinessServiceImpl.java` (222行)
- 服务模块：`ykj-service-server` → `ykj-service-hospital`

## 步骤详解

### 1. 新增商务

管理员填写商务基本信息（姓名、性别、手机号、身份证、银行卡、地址等）。系统：
- 自动生成唯一商务编号，格式为 `BUSI` + 自增序号
- 如有个人照片附件，自动从OSS获取URL
- 通过会员服务获取手机号明文关联到商务记录

### 2. 商务审核

商务在手机上提交资料后（flowStatus = 待审核），管理员进行审批：

**审批通过**：
- flowStatus设为"审核通过"（`BUSINESS_FLOW_STATUS_PASS`）
- 记录审核时间
- 如果该商务尚未生成邀请二维码，自动生成场景二维码
- 发送微信订阅消息通知商务："审核通过，请重新进入小程序"

**审批驳回**：
- flowStatus设为驳回状态
- 通知内容："您的资料审核失败：" + 管理员填写的驳回备注

**关键代码逻辑**：
- 审核通过时检查 `doctorQrcide` 是否为空，为空则调用 `ShareQrCodeAppService.generateSceneQrCode()` 生成二维码
- 发送通知调用 `NotificationAppService.send()`，通知编码 `BUSINESS_SYSTEM_REGISTER`
- 通过 `memberService.findByMemberId()` 获取会员的 openId 和 phone 用于推送

### 3. 二维码管理

商务二维码分两种：

**注册二维码**（让新商务扫码注册）：
- 存储在 Redis 中（key: `business-register-qrcode`）
- 可手动刷新生成新的注册二维码
- 查询时如果缓存不存在，自动生成

**邀请医生二维码**（让医生扫码入驻）：
- 每个已审核商务有专属二维码
- 通过 `ShareQrCodeAppService.generateSceneQrCode()` 生成，携带有 `businessId`
- 支持全量刷新（所有已审核商务）和单个刷新
- 刷新只处理已审核通过的商务

### 4. 修改商务信息

修改时特殊处理：
- 如果商务已审核通过，且修改时仍未生成二维码，自动补生成
- 处理个人照片的OSS URL转换

### 5. 删除商务

软删除（通过ID批量删除）。无级联删除保护——删除商务不会影响已邀请的医生归属。

### 6. 查询商务药品销量

根据商务ID和时间范围查询该商务邀请的医生所开处方带来的药品销售数量。

## 状态流转

```
新增 → 待审核 → 审核通过（生成二维码，发通知）
              → 驳回（发驳回通知）
```

## 异常处理

| 情况 | 处理方式 | 位置 |
|------|---------|------|
| 审批时 businessId 为空 | 抛出业务异常"商务不存在"（后端拦截） | Controller L76-78 |
| 查询销量参数错误 | 返回400错误码（后端拦截） | Controller L104-106 |
| 查询不存在的商务ID | 返回空数据 | Controller L48 |
| 二维码生成失败 | 捕获异常，记录错误日志，不中断流程 | ServiceImpl L166-169 |
| 审核通知推送失败（会员信息获取失败） | 记录日志"取消推送"，不抛异常 | ServiceImpl L134-137 |
| 刷新不存在的商务二维码 | 抛出业务异常"商务不存在" | Controller L143-145 |
| 非本机号码获取失败 | 通过会员服务获取手机号，如果会员不存在则phone为空 | ServiceImpl L209-220 |

## 关联功能

- **邀请医生管理**：审核通过的商务可生成邀请二维码，医生扫码后关联到该商务
- **医生管理**：医生入驻时记录邀请商务ID
- **药品销售统计**：可统计每个商务的药品销量
- **微信订阅通知**：审核结果通过微信订阅消息推送
