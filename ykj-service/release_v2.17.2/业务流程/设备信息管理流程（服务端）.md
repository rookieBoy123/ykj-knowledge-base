# 设备信息管理流程（服务端）

> 文档版本 2026-07-03 | 面向读者：技术运营、产品经理

## 概述
互联网医院机构需要记录和维护其拥有的硬件设备信息，用于监管上报和资产管理。本模块提供设备信息的增删改查接口，支持设备编号自动生成和机构关联。

## 接口一览

| 接口路径 | 方法 | 说明 | 权限 |
|---------|------|------|------|
| `/mgmt/hospital/devices` | POST | 新增设备 | hosp:devices:write |
| `/mgmt/hospital/devices/{id}` | PUT | 修改设备 | hosp:devices:write |
| `/mgmt/hospital/devices/{id}` | DELETE | 删除设备 | hosp:devices |
| `/mgmt/hospital/devices/page` | GET | 分页查询 | hosp:devices |
| `/mgmt/hospital/devices/{id}` | GET | 查询详情 | hosp:devices |

## 设备信息字段

| 字段编码 | 字段名称 | 类型 | 说明 |
|---------|---------|------|------|

| sbmc | 设备名称 | 字符串 | 必填，最多20字 |
| sblx | 设备类型 | 编码 | 字典sblx，如：X光机、CT、B超等 |
| sbxh | 设备型号 | 字符串 | 最多200字 |
| cd | 产地 | 编码 | 字典sbcd |
| syqk | 使用情况 | 编码 | 字典sbsyqk，如：在用、停用、报废等 |
| gmrq | 购买日期 | 日期 | 必填 |
| sccj | 生产厂家 | 字符串 | 最多120字 |
| sbjzje | 设备价值金额 | 数值 | — |
| llsjsm | 理论设计寿命 | 字符串 | 单位：年 |

## 核心流程

### 新增设备
1. 接收设备信息，校验必填字段
2. 系统自动生成设备编号（格式：SBBH + 年月日时分秒 + 3位随机数）
3. 查询当前登录管理员所属的机构信息，获取机构代码
4. 如果机构信息不存在，返回错误"机构信息不存在"
5. 保存设备记录到数据库

### 分页查询
1. 接收分页参数和筛选条件
2. 支持按设备名称模糊匹配、设备型号模糊匹配
3. 支持按产地、使用情况精确匹配
4. 返回分页结果

### 编辑设备
1. 接收设备ID和更新数据
2. 更新对应记录
3. 设备编号不可修改

### 删除设备
1. 接收设备ID
2. 物理删除对应记录（不可恢复）

## 异常处理

| 异常情况 | 错误信息 |
|---------|---------|
| 必填字段缺失 | 表单校验拦截（前端），服务端依赖数据库约束 |
| 机构信息不存在 | "机构信息不存在。" |
| 设备编号重复 | 概率极低（时间戳+随机数），依赖数据库唯一约束 |

## 权限模型
- 权限前缀：hosp:devices
- 写入权限：hosp:devices:write（控制新增/编辑）
- 读取权限：hosp:devices（控制查询/列表）
- 删除权限：继承自 hosp:devices

## 关联服务模块
- `ykj-service-hospital` — 医院管理模块，包含本功能
- 字典服务 — 设备类型、产地、使用情况、新旧况的选项数据
- 机构信息服务 — 新增时获取机构代码
