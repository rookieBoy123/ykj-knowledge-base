# 处方结算流程（服务端）

> 文档版本：2026-07-02 | 面向读者：产品运营、技术支持

## 概述

处方结算是将医生开具的处方转化为可支付的订单的核心流程。包括预结算（费用预览）和正式结算（创建订单）两个阶段，涉及药品费用计算、运费计算、加工服务费、医保支付、订金模式等多个环节。

## 触发条件

- **前置条件**：医生已开具处方（处方状态可结算），患者在小程序端发起结算
- **服务模块**：ykj-service-inquiry（PrescriptionSettlementServiceImpl）

## 步骤详解

### 第一步：预结算（费用预览）

患者在小程序看到处方后点击「去结算」时触发预结算：

1. **查询处方信息**：根据处方编号查询处方详情
2. **校验处方状态**：处方必须处于可结算状态，否则拦截
3. **加载药房信息**：查询处方关联的药房，获取药品价格
4. **加载收货地址**：查询患者的收货地址（快递配送模式需要）
5. **确认加工服务费**：如果选择了中药加工工艺（代煎、打粉等），确认加工费用
6. **计算费用项**：逐项计算：
   - 药品费用（每味药的单价×数量）
   - 加工服务费（中药）
   - 医事服务费（诊金）
   - 运费（根据药房配送规则和收货地址计算）
   - 医保报销金额
7. **计算总金额**：汇总所有费用项
   - 负数保护：如果总金额为负数，自动置为0
8. **订金判断**：
   - 如果药房开启了订金模式 + 系统全局开启了货到付款 + 订单金额满足最低订金门槛 → 启用订金
   - 此时患者只需支付订金部分

返回预结算结果供前端展示费用明细。

### 第二步：正式结算（创建订单）

患者确认费用后点击「确认支付」时触发正式结算：

1. **检查已有订单**：查询是否已存在该处方的有效订单，有则提示「已存在订单信息」
2. **重新预结算**：再次执行预结算，确保费用准确
3. **创建订单**：调用订单服务创建订单
   - 订单来源标记为「处方」
   - 支持第三方调用时自定义订单号
4. **回填处方订单编号**：将生成的订单号写回处方记录
5. **返回订单号**：供前端跳转支付页面

### 第三步：支付方式判断

结算时根据患者选择和处方类型确定支付方式：

| 支付方式 | 说明 |
|---------|------|
| 微信支付 | 微信支付的处方 |
| 支付宝 | 支付宝支付的处方 |
| 医保统筹支付 | 医保统筹基金支付部分 |
| 医保个账支付 | 医保个人账户支付部分 |
| 医保微信支付 | 医保自费部分微信支付 |

**医保支付流程**：
- 需要先获取医保授权码
- 调用医保支付接口
- 结算完成后上传医保明细

### 第四步：处方订单详情查询

患者可以查看处方订单的完整信息：
- 订单基本信息（单号、金额、状态）
- 费用明细（药品费、加工费、服务费、运费、医保报销）
- 支付渠道（如「微信+医保个账+医保统筹」）
- 收货地址（快递配送模式）
- 药房信息（是否支持代煎、代煎费用）

### 第五步：医保报销明细

支持查看订单的医保报销明细：
- 上传到医保平台的费用明细
- 医保基金支付金额、个人账户支付金额、个人现金支付金额
- 可生成医保报销明细PDF

## 费用计算规则

### 加工服务费
- 仅中药处方有加工服务费
- 按工艺类型计算（代煎、打粉等不同收费标准）
- 按剂数计算：代煎单价 × 总剂数

### 运费
- 快递配送模式计算运费
- 药房取药模式不计算运费
- 医生可设置免运费（后台配置权限）

### 订金模式
- 条件：药房开启订金 + 系统货到付款开关开启 + 订单总金额 ≥ 最低订金门槛
- 订金一般为药品金额的一部分
- 支付订金后进入待发货状态
- 其余金额货到付款

## 状态流转

```
处方已开具 → 预结算(费用预览) → 正式结算(创建订单) → 待支付 → 已支付 → 待发货/待取药 → ...
```

## 异常处理

| 情况 | 系统处理方式 |
|------|------------|
| 处方状态不可结算 | 拦截并提示 |
| 已存在有效订单 | 提示「已存在订单信息」 |
| 药房信息不存在 | 提示「找不到对应药房信息」 |
| 收货地址不存在 | 提示「该订单收货地址信息不存在」 |
| 费用项信息异常 | 提示「订单费用项信息不存在」 |
| 总金额为负数 | 自动将总金额置为0 |
| 订单信息不存在 | 提示「订单信息不存在！」 |
| 处方不存在 | 提示「处方不存在, 无法创建订单」 |
