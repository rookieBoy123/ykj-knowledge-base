# AI辅助问诊建议生成流程（服务端）

> 文档版本 2026-07-05 | 面向系统管理员和产品经理

## 概述

AI辅助建议是嵌入在医生问诊过程中的智能工具。医生在问诊界面点击"AI建议"按钮后，系统会根据当前问诊的患者信息、病情描述、聊天记录、病史等上下文，调用AI能力生成**提问建议**和**病历辅助**两种智能输出，帮助医生更快地完成诊疗决策。

## 触发条件

- 医生正在问诊中，点击"AI建议"按钮
- 两种类型独立触发：
  - **提问建议**：医生想获取下一步应该问什么问题
  - **病历辅助**：医生想获取AI生成的初步病历内容

## 前置条件

1. 当前问诊必须有效存在（有问诊单号）
2. 患者信息必须完整（需要从身份证号中提取年龄和性别）
3. 系统已配置AI建议功能开关（`questionAiSuggestion` 和 `medicalAiSuggestion`）
4. 该问诊单未被"AI锁定"（详见错误锁定机制）

## 步骤详解

### 第一步：功能开关校验

1. 医生请求AI建议时，系统首先检查功能开关：
   - 提问建议开关：`questionAiSuggestion` 配置项
   - 病历辅助开关：`medicalAiSuggestion` 配置项
2. 如果开关关闭，直接提示"系统暂不支持医生问答辅助, 请联系管理员"或"系统暂不支持病历辅助, 请联系管理员"

### 第二步：AI锁定检查

1. 系统为该问诊单和请求类型（提问/病历）、生成一个锁定缓存Key
2. 如果该Key存在（说明AI在前30分钟内已经连续失败3次），直接返回锁定状态和剩余锁定时间
3. 医生在前端可以看到"AI建议暂时不可用，请在xx分钟后重试"的提示

### 第三步：收集AI上下文数据

系统从五个维度收集信息：

1. **问诊信息**：从 `InquiryAppoint` 获取问诊单详情（病情描述、分组ID、病历ID等）
2. **患者信息**：从患者服务获取患者基础信息——通过身份证号解析出**年龄**和**性别**传给AI
3. **病史信息**：聚合三个来源的病史数据：
   - 患者档案中的：过敏史、病史、个人史
   - 预问诊记录中的：过敏史、现病史、个人史、家族史、婚育史、月经史
   - 已有病历中的：同预问诊的六个病史维度
   - **每个字段优先使用患者档案的数据**，档案没有的才用预问诊/病历的数据
4. **聊天记录**：从IM服务拉取当前问诊群的全部历史消息，过滤出医生和患者发送的**纯文本消息**
5. **病情描述**：优先取病历中的病情描述，没有则取问诊单中的病情描述，都没有则为"无"

### 第四步：调用AI服务

1. 将上下文数据封装成请求，调用AI建议服务（`SuggesterService`）
2. 对于提问建议：返回最多9条建议问题（通过 `DEFAULT_NUM=9` 控制上限）
3. 对于病历辅助：返回拟诊疾病列表、建议检查项目、建议用药等结构化内容
4. 病历辅助的拟诊疾病会与系统内部的诊断词库进行匹配，只返回系统中已有的疾病名称

### 第五步：AI调用结果处理

**成功时**：
- 清除该问诊单+建议类型的错误计数器
- 返回建议内容给医生前端展示

**失败时**：
- 记录失败次数（使用Redis缓存，30分钟有效期）
- 失败次数从0开始累加：
  - 第1次失败：计数设为1
  - 第2次失败：计数+1
  - 第3次失败：计数+1 → 达到3 → **触发AI锁定**
- 触发锁定后：Redis写入锁定Key，有效期30分钟，锁定期间所有请求直接拒绝
- 返回空建议（success=false）

## 错误锁定机制

| 阶段 | 行为 |
|------|------|
| 正常使用 | AI建议正常返回，失败计数器为0 |
| 第1次调用失败 | 失败计数设为1（Redis，30min TTL） |
| 第2次调用失败 | 失败计数+1（Redis，30min TTL） |
| 第3次调用失败 | 失败计数+1→达到3，**写入锁定Key**（30min TTL），后续30分钟所有请求直接拒绝 |
| 锁定期间 | 返回 locked=true + 剩余秒数的lockingTime |
| 锁定过期后 | 锁定Key自动消失，可以重新请求（从0开始计数） |

**注意**：提问建议和病历辅助的锁定是**独立**的——提问建议锁了不影响病历辅助，反之亦然。

## 异常处理

| 情况 | 系统处理方式 |
|------|------------|
| AI问答辅助功能未开启 | 后端拦截，提示"系统暂不支持医生问答辅助, 请联系管理员" |
| AI病历辅助功能未开启 | 后端拦截，提示"系统暂不支持病历辅助, 请联系管理员" |
| AI锁定中（30分钟冷却期） | 返回锁定状态+剩余秒数，前端显示倒计时 |
| 连续失败3次触发锁定 | 自动锁定30分钟，日志记录异常信息 |
| 问诊单号查不到问诊 | 后端拦截，提示"没有相关问诊单" |
| 患者信息查不到 | 后端拦截，提示"没有患者信息" |
| 聊天记录获取失败 | 不影响主流程，降级使用病情描述作为默认消息 |
| 预问诊信息获取失败 | 不影响主流程，只使用患者档案和病历的病史数据 |
| 病历信息获取失败 | 不影响主流程，只使用患者档案和预问诊的病史数据 |
| AI服务返回非200状态码 | 计入失败次数 |
| AI服务抛异常（超时等） | 计入失败次数，日志记录完整异常信息 |

## 数据拼装规则

### 病史数据优先级

对于每个病史维度（过敏史、现病史、个人史、家族史、婚育史、月经史），按以下优先级逐级取第一个非空值：

```
患者档案 → 预问诊记录 → 已有病历 → "无"
```

### 聊天消息过滤

只提取以下类型的消息：
- `TimMsgTypeConstants.TEXT`（纯文本类型）
- 发送者ID包含 `@doctor` 或 `@patient` 后缀（排除系统消息）

## 关联功能

- **问诊管理**：依赖 `InquiryAppointService` 获取问诊信息
- **病历管理**：依赖 `MedicalRecordService` 获取病历数据
- **AI预问诊**：依赖 `AiAgentService` 获取预问诊记录
- **即时通讯**：依赖 `ImGroupMessageService` 获取聊天记录
- **患者管理**：依赖 `ApiPatientService` 获取患者信息
- **诊断词库**：依赖 `DiagnosisService` 匹配AI返回的疾病名称
