# 医生排班当天号段编辑限制与校验流程（服务端深挖）

> 文档版本：2026-07-05 | 面向读者：产品经理、测试人员、运维人员

## 概述

医生排班系统对"当天"的排班号段编辑有严格的时效限制。出于业务合理性考虑，已经过去的号段、正在进行的号段、以及当天不同时间段的号段，其编辑权限各不相同。系统通过多层次的校验逻辑来防止不合规的排班修改操作。

## 触发条件

- 前置条件：管理后台在排班周视图页面编辑医生的当天排班
- 入口：管理员在排班管理页面修改号段时间、号源数量或启用/禁用状态

## 核心规则

### 规则一：只校验当天排班

系统只在排班日期等于"今天"时才执行当天号段校验。编辑历史或未来的排班不受此限制。

### 规则二：新增号段限制

如果当天之前没有排班（首次新增），则：
- 结束时间已经过去的号段（结束时间 < 当前时间）**不允许新增**，提示"小于当前时间的排班不允许新增"

### 规则三：编辑已有号段限制

如果当天已有排班记录（编辑现有号段），则分为两种情况：

| 号段时间状态 | 可以修改什么 | 不能修改什么 |
|------------|------------|------------|
| 开始时间已过（已开始） | 号源数量（如果结束时间未过） | 号段起止时间、启用/禁用状态 |
| 结束时间已过（已结束） | 无 | 号源数量、启用/禁用状态、任何字段 |
| 开始时间未到（未开始） | 所有字段 | 无 |

- 对于新增的号段（历史中找不到对应编号），如果在当前时间之前，提示"小于当前时间的排班不允许操作"
- 对于已结束的号段（结束时间已过），修改号源数量时提示"小于当前时间的排班号源数不允许操作"
- 修改已过号段的启用/禁用状态时，提示"小于当前时间的排班不允许操作"

## 步骤详解

### 第一步：编辑排班

管理员在排班周视图页面修改医生当天的排班信息（如号段时间、号源数量等）。

### 第二步：班次级别校验

系统先校验班次层面：
- 班次名称是否重复（同一天不能有两个同名班次）
- 班次时间是否有重叠冲突
- 号段时间是否在所属班次的时间范围内

### 第三步：当天时效校验

如果排班日期等于今天，系统执行当天号段时效校验：
1. 查询该医生当天已有的历史排班记录
2. 对比当前编辑的号段与历史记录
3. 根据号段时间与当前时间的关系，判断哪些字段可编辑

### 第四步：号段冲突校验

系统校验同一班次内的号段时间是否有重叠：
- 号段A 08:00-10:00 和号段B 09:00-11:00 会冲突
- 提示"号段时间存在冲突，请检查"

### 第五步：保存排班

所有校验通过后，系统：
1. 为新增号段生成唯一编号
2. 保存号段详情
3. 保存班次信息
4. 删除旧的历史记录

## 异常处理

| 情况 | 系统处理方式 |
|------|------------|
| 新增已过号段 | 提示"小于当前时间的排班不允许新增" |
| 修改已过号段的号源数 | 提示"小于当前时间的排班号源数不允许操作" |
| 修改已过号段的启用状态 | 提示"小于当前时间的排班不允许操作" |
| 号段时间不在班次范围内 | 提示"号段时间XX-XX不在XX班次时间内" |
| 号段时间冲突 | 提示"号段时间存在冲突，请检查" |
| 班次名称冲突 | 提示"班次名称存在冲突，请检查" |
| 班次时间冲突 | 提示"班次时间存在冲突，请检查" |
| 班次保存错误 | 提示"班次保存错误" |

## 关联功能

- **排班模板同步**：定时任务自动为医生创建15天后的排班
- **停诊处理**：手动停诊某班次，通知已挂号患者
- **号源预约**：患者预约时扣减号源数量
- **排班发布**：管理员一键发布所有待发布排班
