# 量表测评答案处理与评分流程

> 文档版本：v2.17.2 | 面向读者：产品经理 / 技术支持

## 概述

患者逐题作答量表时，后端每保存一道题的答案即进行校验；答完最后一题时自动触发评分计算，生成测评结果。

## 触发条件

- 患者在量表答题页选择答案并提交（每题提交一次）
- 测评记录已创建（recordId已生成）

## 步骤详解

### 第一步：逐题保存答案

患者每答完一题，前端调用保存答案接口。后端执行以下校验：

1. **题目存在性校验**：通过 questionId 查找对应题目，找不到则提示「找不到对应的量表题目！」
2. **单选题限制**：如果题目类型为单选（type=0），但答案内容包含多个选项，则提示「单选题不可以有多个答案！」
3. **选项合法性校验**：答案中的每个选项编号必须在题目定义的选项列表中，否则提示「该答案不存在于问题选项中！」
4. **已完成保护**：如果该测评记录（recordId）已经生成了测评结果（result），提示「已测评完成，不能再次修改答案！」
5. **答案已存在处理**：同一题重复提交时，校验通过后更新（update）而非新增

### 第二步：判断是否为最后一题

每次保存答案后，系统检查该题目是否为量表的最后一题。

- **不是最后一题**：保存答案即完成，继续等待下一题
- **是最后一题**：进入第三步，自动触发生成测评结果

### 第三步：组装测评结果

当最后一题答案保存成功后，系统自动计算测评结果：

1. **收集所有题目**：按scaleId查询该量表的所有题目
2. **收集所有答案**：按recordId查询该测评记录的所有已保存答案
3. **建立数据结构**：
   - 题目选项映射表（questionId → 选项列表，含分数值）
   - 题目维度映射表（questionId → 维度名称，如"躯体化""焦虑"等）

### 第四步：计算总分与维度分

逐题遍历所有答案，进行分数累加：

| 计算项 | 计算方式 |
|--------|---------|
| 每题得分 | 从该题的「选项→分值」映射表中查找考生选择的选项编号对应的分值，累加（支持多选题各选项分值累加） |
| 总得分 | 所有题目的每题得分之和 |
| 维度得分 | 按题目所属的维度分组，各维度内的每题得分之和 |

例如：一道多选题选择了"A"和"B"两个选项，A对应1分、B对应2分，则该题得3分。

### 第五步：保存测评结果

将计算好的总得分和维度得分（JSON格式存储）写入结果表，关联recordId和scaleId。

此时前端可调用结果查询接口获取测评报告，包括：
- 总得分
- 各维度得分（如焦虑维度35分、抑郁维度28分）
- 根据分数区间匹配的评语/建议

## 状态流转

```
创建测评记录(recordId) → 逐题答题(每题保存时校验) → 最后一题触发评分 → 测评完成(status=已结束)
```

- 答题过程中可以中断（保存到哪题算哪题）
- 测评完成后答案不可修改（由已完成保护机制拦截）
- 「重新测评」会创建新的测评记录而非修改旧记录

## 异常处理

| 情况 | 系统处理方式 |
|------|------------|
| 题目id找不到对应题目 | 返回错误「找不到对应的量表题目！」 |
| 单选题提交了多个答案 | 返回错误「单选题不可以有多个答案！」 |
| 答案选项不在题目定义中 | 返回错误「该答案不存在于问题选项中！」 |
| 测评已完成后再提交答案 | 返回错误「已测评完成，不能再次修改答案！」 |
| 保存答案时数据库操作失败 | 返回错误「保存量表答案操作失败！」 |
| 修改答案时数据库操作失败 | 返回错误「修改量表答案操作失败！」 |

## 关联功能

- 与「量表测评记录管理」关联：创建测评记录、查询测评状态
- 与「量表管理」关联：量表的题目定义、选项分值配置
- 与「测评结果查询」关联：得分计算后生成可视化报告
- 与「问诊」关联：可从问诊聊天室中发送量表给患者
