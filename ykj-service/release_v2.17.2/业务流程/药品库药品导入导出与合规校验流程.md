# 药品库药品导入导出与合规校验流程

> 文档版本 2026-07-02 | 面向读者：运营管理人员、开发人员

## 概述

药品库是平台的基础数据，包含中药（Herbal）和成药（Patent）两大类。本流程描述药品库中药品的添加、导入、导出、去重校验、监管ID同步等功能。所有药品操作都经过严格的重复校验和计量单位合规检查。

## 触发条件

- 管理人员在后台「药品信息管理」中手动添加药品
- 通过Excel批量导入药品数据
- 导出药品库数据用于审计或统计
- 定时任务同步药品监管ID

## 核心概念

- **中药（Herbal）**：中药材，按通用名（DrugsAlias）去重，计量单位固定为「克」
- **成药（Patent）**：中成药/西药，按药监ID（SupervisionId）去重
- **药品库主表（MedicalDrugsLib）**：存储药品通用信息（ID、类型、通用名）
- **中药子表（MedicalDrugsLibHerbal）**：存储中药特有属性
- **成药子表（MedicalDrugsLibPatent）**：存储成药特有属性（药监ID、生产厂家、规格等）

## 步骤详解

### 第一步：添加中药

添加中药时系统执行以下校验和操作：
1. **重复检查**：按药品通用名（DrugsAlias）和类型（Herbal）查询是否已存在 → 如存在则提示"药品已存在"
2. **插入主表**：写入MedicalDrugsLib，类型标记为Herbal
3. **插入子表**：写入MedicalDrugsLibHerbal，关联主表ID
4. **计量单位校验**：检查药品单位是否在字典中存在 → 不存在则提示"药品计量单位不正确"；检查单位是否为"克" → 不是则提示"药品计量单位应为：克"
5. **事件发布**：发布DrugsLibAddedEvent事件，通知其他模块药品库有新增

### 第二步：添加成药

添加成药时系统执行：
1. **监管ID必填校验**：药监ID未填写 → 提示"药品监管id未填写"
2. **重复检查**：按药监ID查重 → 已存在则提示"药品已存在"
3. **拼音自动生成**：如果未填写药品拼音，自动根据通用名生成拼音首字母
4. **插入主表+子表**
5. **事件发布**

### 第三步：Excel导入药品

系统支持通过Excel文件批量导入中药或成药：

**导入流程**：
1. 上传Excel文件，根据药品类型选择不同的导入模板（中药/成药）
2. 逐行解析Excel数据，跳过标题行
3. 对每行数据执行与手动添加相同的校验（重复检查、计量单位校验等）
4. 成功的行批量插入数据库，失败的行记录错误原因（含行号）
5. 导入完成后统一发布一次事件通知
6. 返回导入结果（成功数、失败数、失败详情）

**导入失败常见原因**：
- 药品已存在（同名或同药监ID）
- 计量单位不正确（中药必须为"克"）
- 药监ID未填写（成药必填）

### 第四步：Excel导出药品

导出分为中药导出和成药导出：
- 中药导出：按筛选条件查询中药列表，导出为Excel（文件名为"药品库中药"）
- 成药导出：按筛选条件查询成药列表，导出为Excel（文件名为"药品库成药"）
- 导出格式包含序号列，Sheet名称为对应类型名

### 第五步：监管ID同步

系统提供定时同步功能（syncSupervisionIdFromStd方法）：
1. 查询成药子表中监管ID为空的药品
2. 对每条记录，调用标准药品服务（ApiStdMedService）查询药监ID
3. 查询参数包括：通用名（commonName）、生产厂家（manufacturers）、规格（specification）、包装单位等
4. 如果查到药监ID，更新到成药子表中
5. 如果查不到，跳过该条记录

### 第六步：药监ID查重

系统提供药监ID查重功能：
- 输入药监ID，检查是否已存在（返回true/false）
- 通过药监ID批量查询药品信息
- 通过药监ID查询单个药品详情

### 第七步：删除药品

- 删除中药：先删主表记录，再删中药子表记录
- 删除成药：先删主表记录，再删成药子表记录
- 两个操作在同一事务中执行

## 异常处理

| 情况 | 系统处理方式 | 提示信息 |
|------|-------------|---------|
| 中药通用名重复 | 后端拦截 | 「药品已存在」 |
| 成药药监ID重复 | 后端拦截 | 「药品已存在」 |
| 成药药监ID未填写 | 后端拦截 | 「药品监管id未填写」 |
| 药监ID参数为空（查询时） | 后端拦截 | 「药品药监ID不存在, 请联系管理员」 |
| 中药计量单位不正确 | 后端拦截 | 「药品计量单位不正确」 |
| 中药计量单位不是"克" | 后端拦截 | 「药品计量单位应为：克」 |
| 药品通用名存在重复记录 | 后端拦截 | 「药品库存在重复通用名药品:xxx」 |
| 导入的药品类型未知 | 后端拦截 | 「未知的药品类型」 |
| 导出时发生异常 | 捕获后抛出 | 「药品库药品导出失败」 |
| 添加失败（插入返回0） | 后端拦截 | 「添加失败」 |

## 关联功能

- 服务模块：ykj-service-medicine（药品管理模块）
- 事件通知：DrugsLibAddedEvent（药品库新增事件，通知处方模块等）
- 标准药品服务：ApiStdMedService（药品监管ID查询）
- 字典服务：DictionaryAppService（计量单位字典）
- 拼音工具：ChinesePinyinUtil（自动生成药品拼音首字母）
- Excel服务：ExcelService（导入导出功能）
