# 处方开具与审核签章流程

> 文档版本：v1.0 | 最后更新：2026-07-01 | 面向读者：产品/运营

## 概述

医生在问诊过程中为患者开具电子处方，处方需要经过内容校验、药师审核和电子签章三个环节后才能生效。系统支持中药处方、西药/中成药处方和公共处方模板。

---

## 处方状态流转

```
草稿 → 已提交 → 待药师审核 → 审核通过 → (待签章) → 已签章 → 已生效
                ↘ 审核拒绝（退回修改）
       ↘ 已删除（仅草稿状态可删）
```

---

## 步骤详解

### 第一步：创建处方草稿

医生开始开具处方：

- **新建处方**：选择处方类型（中药/西药），系统分配门诊号
- **添加药品**：从药品库搜索药品（名称/拼音首字母），选择用法用量
- **诊断填写**：填写中医/西医诊断信息
- **自动保存草稿**：系统自动保存当前编辑内容

如果同一门诊号已有草稿：
- 系统自动加载已有草稿供继续编辑
- 之前的药品选择和诊断信息保留

### 第二步：提交处方

医生完成处方内容后点击"提交"：

- **药品校验**：检查药品是否缺货/禁用，用法用量是否合规
- **诊断校验**：诊断信息是否完整
- **患者信息**：确认患者身份和就诊人信息
- 处方状态从「草稿」变为「已提交」

### 第三步：药师审核

处方提交后进入药师审核环节：

- 药师在后台查看处方详情（药品清单、诊断、患者信息）
- **审核通过**：处方可继续进入签章环节
- **审核拒绝**：处方退回给医生，医生可修改后重新提交
- 审核拒绝时需填写拒绝原因

### 第四步：医生电子签章

审核通过的处方需要医生进行电子签名：

- 系统调用CA签章服务
- 支持的签章平台：医网信、源康健PaaS、放心签
- 医生通过短信验证码或手写签名完成签章
- 签章完成后，处方PDF自动加盖医生电子印章
- 药师也可配置是否需要签章（药师签名设置）

### 第五步：处方生效与使用

签章完成后处方正式生效：

- 系统生成带签章的处方PDF
- 处方推送到关联药房进行配药
- 患者可在小程序查看处方详情和药品清单
- 处方关联的药品订单自动进入发货流程

## 特殊场景

### 处方模板复用
- 医生可将常用处方保存为模板（中药/成药/公用三类）
- 下次开方时直接选择模板，系统自动填充药品和诊断
- 模板中的药品如已缺货会标记提醒

### 公共药方
- 平台可维护公共处方模板
- 所有医生均可使用公共药方
- 公共药方与个人模板分别存储

### 续方
- 患者可使用历史处方发起续方
- 系统加载历史处方内容供医生确认和修改
- 快速续方：基于历史处方一键生成新处方

---

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 药品已缺货 | 处方中标记为"缺货"，提示医生更换 |
| 药品已禁用/下架 | 标记为"已失效"，不允许使用 |
| 医师审核拒绝 | 处方退回草稿状态，医生可修改重提 |
| CA签章服务异常 | 签章失败，记录失败原因，支持重试 |
| 同一门诊号已存在处方 | 加载已有处方供编辑（不新建） |
| 处方超时未审核 | 处方保持已提交状态，等待药师处理 |

---

## 关联功能

- **问诊管理**：处方在问诊过程中开具
- **药品库**：处方的药品来自统一药品库
- **CA签章**：电子签章依赖CA认证服务
- **药师管理**：审核处方的药师需在后台注册
- **订单管理**：处方生效后生成药品订单
- **处方模板**：常用处方的模板化管理
