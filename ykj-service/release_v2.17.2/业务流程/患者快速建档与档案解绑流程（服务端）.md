# 患者快速建档与档案解绑流程（服务端）

> 文档版本 2026-07-04 | 面向读者：产品经理、后端开发对接方

## 概述

管理后台支持通过身份证号快速为患者建档。当平台系统中不存在该患者时，系统会自动从HIS（医院信息系统）拉取患者信息，同步会员数据，并在平台创建患者档案。同时支持解绑档案操作。对应服务模块 `ykj-service-hospital`，接口前缀 `/mgmt/patient/patients`。

## 接口清单

| 接口 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 分页查询 | GET | `/mgmt/patient/patients` | 患者列表 |
| 患者详情 | GET | `/mgmt/patient/patients/{id}` | 查看患者完整信息 |
| 按身份证查ID | GET | `/mgmt/patient/patients/_findIdsByIdCard` | 批量查询患者ID |
| 按身份证查患者 | GET | `/mgmt/patient/patients/_findByIdCard` | 查询单个患者信息 |
| 快速建档 | POST | `/mgmt/patient/patients/_quick-filing` | 从HIS拉取并建档 |
| 解绑档案 | POST | `/mgmt/patient/patients/_unbind` | 解除档案绑定 |

## 核心流程

### 快速建档流程

快速建档涉及三个子系统联动：管理后台 → 互联网医院系统 → HIS系统 → 会员系统。

**流程详解**：

1. 前端提交身份证号（idCard），后端先在本系统查询该身份证号是否已有患者档案
2. **已有档案**：直接返回已有患者信息（PatientVo），流程结束
3. **无档案**：调用HIS系统接口 `apiHisPatientService.getPatInfo(idCard)` 查询HIS中的患者信息
   - 从HIS获取的信息包括：患者姓名、手机号、性别、出生日期等
   - 如果HIS也查不到，抛出异常 "从HIS查询档案失败，请联系管理员！"
4. **会员关联**：使用HIS返回的手机号调用会员服务 `apiMemberService.findByPhone(phone)`
   - 如果会员不存在，自动注册新会员（根据手机号）
   - 如果会员已存在，直接使用
5. **创建患者档案**：将HIS患者信息转换为平台患者记录
   - 绑定会员ID（memberId）
   - 设置默认成员关系为"本人"（relationshipCode=0）
6. **绑定HIS关系**：调用 `apiHisPatientService.addRelation()` 建立平台患者ID与HIS患者ID的对应关系
7. 返回新建的患者档案信息

**关键依赖关系**：
```
管理后台 → 互联网医院(PatientService) → HIS系统(ApiHisPatientService)
                                       → 会员系统(ApiMemberService)
```

### 档案解绑流程

1. 前端提交患者ID（patientId）
2. 权限校验：`hosp:patients:write`
3. 调用 `patientService.unbind(patientId)` 解除档案与HIS的绑定关系
4. 返回成功

### 患者查询

**按身份证号查询**：
- `/mgmt/patient/patients/_findByIdCard` 返回单个患者信息，包含：ID、姓名、性别（1-男 2-女）、出生日期、年龄
- 年龄由 `AgeUtils.getAgeStr(birthday)` 自动计算：周岁以下显示"月"，否则显示"岁"

**批量查询ID**：
- `/mgmt/patient/patients/_findIdsByIdCard` 支持根据身份证号列表批量返回患者ID

## 异常处理

| 情况 | 处理方式 | 错误来源 |
|------|---------|---------|
| 身份证号为空 | 参数校验失败，"身份证号不能为空" | 后端校验 |
| HIS查询患者失败 | 抛出 BizException "从HIS查询档案失败，请联系管理员！" | 后端业务逻辑 |
| HIS患者无手机号 | 可能导致会员注册失败或查找异常 | 数据依赖 |
| 会员注册失败 | apiMemberService.register() 内部处理 | 会员模块 |
| 解绑档案 | patientService.unbind() 处理，不返回异常详情 | 业务逻辑 |

## 权限控制

- 读操作：`hosp:patients:read`
- 写操作（建档/解绑）：`hosp:patients:write`
- 分页查询：组合权限 `hosp:patients:page`

## 关联模块

- **会员系统（ykj-service-member）**：患者必须有对应会员账号
- **HIS对接（ykj-service-his）**：患者信息来源和关系绑定
- **患者列表管理（管理后台）**：使用分页接口展示和搜索患者
