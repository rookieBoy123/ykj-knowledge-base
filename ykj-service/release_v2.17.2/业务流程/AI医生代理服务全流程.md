# AI医生代理服务全流程

> 文档版本 2026-07-02 | 面向读者：产品经理、运营人员

## 概述

AI医生代理是一个独立的智能问诊服务模块，可以在医生不直接参与的情况下，由AI与患者进行对话问诊。AI医生能够理解患者描述的症状、提出问题、记录信息，并在问诊结束时自动生成问诊报告。AI医生代理与人工医生问诊是互补关系——AI先进行预问诊和信息采集，为后续人工医生接诊提供结构化的病情信息。

## 触发条件

- **前置条件**：系统管理员在配置中开启AI医生功能（`aiDoctorEnabled`开关）
- **适用场景**：
  - 患者发起问诊后，先由AI医生进行预问诊采集信息
  - 医生在接诊前查看AI已采集的预问诊报告
- **功能关闭时**：所有AI医生相关接口返回空结果，不影响正常问诊流程

## 步骤详解

### 第一步：开始AI问诊

系统创建AI问诊会话，传入患者的基本信息和问诊上下文。

### 第二步：AI对话问诊

AI医生与患者进行多轮对话（流式传输，类似ChatGPT的逐字输出效果）：
- AI根据患者回复动态调整问题
- AI询问症状、病史、用药情况等信息
- 支持**处方图片OCR识别**：患者上传处方照片后，AI自动识别图片中的文字信息
- 支持**语音转写**：患者通过语音描述症状，系统获取WebSocket地址进行实时语音转写

### 第三步：生成问诊报告

AI问诊结束后，系统调用AI引擎自动生成问诊报告：
- 报告包含：主诉摘要、病史采集结果、初步评估建议
- 报告状态标记为"已报告"后，可以被后续人工接诊时参考

### 第四步：查看与管理

- **问诊记录分页查询**：可按患者姓名、年龄范围、日期范围筛选历史记录
- **对话历史查看**：完整回顾AI与患者的每轮对话
- **问诊详情查看**：查看AI问诊的基本信息、状态和报告

### 第五步：结束AI问诊

标记AI问诊完成，更新状态为"已完成"。

## 异常处理

| 情况 | 系统处理方式 |
|------|------------|
| AI医生功能未开启 | 所有接口返回空结果，不影响正常业务流程 |
| AI服务HTTP请求失败（网络错误） | 记录日志后返回默认值或空结果，不阻断业务流程 |
| AI服务返回异常状态码 | 日志记录状态码和响应内容，返回默认值 |
| AI服务响应内容解析失败 | 日志记录原始响应，抛出"agent服务异常" |
| OCR图片文件读取失败 | 提示"文件读取失败" |
| 对话历史为空 | 返回空列表，不报错 |
| 单个请求执行异常 | 错误被捕获，返回该方法的默认值，不中断调用方 |

## 关联功能

- **人工接诊流程**：AI预问诊报告作为医生接诊时的参考信息
- **AI辅助问答**：SuggestionServiceImpl利用AI预问诊报告中的病史来辅助生成问答建议
- **处方便捷录入**：OCR识别处方图片文字，辅助医生快速创建处方
- **系统配置**：管理员通过`aiDoctorEnabled`配置开关控制整个AI医生代理的启用/停用
