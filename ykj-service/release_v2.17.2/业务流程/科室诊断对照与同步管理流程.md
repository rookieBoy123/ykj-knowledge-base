# 科室诊断对照与同步管理流程

> 文档版本 2026-07-02 | 面向读者：运营管理人员、开发人员

## 概述

为了帮助医生快速选择与科室匹配的诊断，平台维护了一套「标准科室—ICD-10诊断」的对照关系。管理员可以给每个科室配置常用诊断，支持父子科室的自动同步。本流程描述科室诊断的增删改查、父子同步、批量导入导出等功能。

## 触发条件

- 管理人员在后台进入「标准管理 → 科室诊断对照」模块
- 医生开方时需要根据科室过滤可选的诊断列表
- 新增科室或诊断标准更新时需要维护对照关系

## 核心概念

- **标准科室（StdDept）**：遵循国家标准编码的科室分类体系，科室编码支持层级结构（如"01.02"表示01下的子科室02）
- **ICD-10诊断**：国际疾病分类第10版（ICD-10），是疾病诊断的标准化编码
- **科室诊断对照**：将科室与诊断建立关联，一个科室可对应多个诊断
- **父子同步**：当子科室新增/删除诊断时，自动同步到父科室（向上递归到根节点）

## 步骤详解

### 第一步：查看科室诊断列表

列表按标准科室分页展示，支持筛选：
- 科室名称：模糊搜索
- 诊断名称：模糊搜索（通过诊断名称反查对应科室）

每个科室显示其下配置的诊断名称列表。

### 第二步：为科室新增诊断对照

新增时系统执行以下操作：
1. 校验标准科室是否存在（通过字典服务查询科室编码）
2. 提取科室的父级编码（如"01.02"的父级为"01"）
3. 过滤掉该科室已有的诊断（按诊断编码去重）
4. 批量插入新增的诊断对照记录
5. **自动同步父科室**：递归向上，将新增的诊断也添加到父科室的对照列表中

### 第三步：编辑科室诊断对照

编辑操作支持两种场景：
- **有新增诊断**：与新增流程相同，执行commonInsert
- **删除诊断**：对比当前数据库记录与提交的新列表，找出被移除的诊断，执行删除
- **自动同步删除父科室**：只有当父科室下所有子科室都不再有该诊断时，才从父科室中移除

### 第四步：父子同步删除逻辑（关键）

当从子科室删除诊断时，系统会：
1. 查询父科室下所有子科室的诊断对照
2. 检查被删除的诊断是否还存在其他子科室
3. 如果没有任何子科室包含该诊断，才从父科室中删除
4. 递归向上处理到根节点

这确保了只有在所有子科室都不再需要某个诊断时，父科室才删除它。

### 第五步：批量导入科室诊断

系统支持通过Excel文件批量导入科室诊断对照：
1. 上传Excel文件（包含科室名称和诊断名称列）
2. 系统从Redis缓存中加载标准科室字典和ICD-10诊断字典
3. 匹配科室名称 → 科室编码、诊断名称 → 诊断编码
4. 过滤掉无法匹配的行（科室或诊断不存在）
5. 批量插入对照数据
6. 触发父子同步：三级以上科室的诊断自动向上同步
7. 导入完成后清除Redis缓存

### 第六步：导出科室诊断

无导出功能，仅支持页面查看和Excel导入。

## 异常处理

| 情况 | 系统处理方式 | 提示信息 |
|------|-------------|---------|
| 标准科室不存在 | 后端拦截 | 「找不到对应的标准科室信息！」 |
| ICD-10诊断数据为空 | 后端拦截 | 「不存在ICD_10诊断信息！」 |
| 标准科室字典为空 | 后端拦截 | 「不存在标准科室信息！」 |
| 导入时科室名称匹配失败 | 跳过该行 | 不在数据库中插入 |
| 导入时诊断名称匹配失败 | 跳过该行 | 不在数据库中插入 |
| 诊断编码重复（新增时） | 自动过滤 | 已存在的诊断不再重复插入 |

## 关联功能

- 服务模块：ykj-service-std（标准管理模块）
- 字典服务：DictionaryAppService（科室字典、诊断字典）
- 标准药品服务：ApiStdMedService（药品监管ID查询）
- Redis缓存：科室字典缓存（dept_diag_ICD10）、标准科室缓存（std_dept），过期时间3600秒
- 数据表：std_dept_diagnose（科室诊断对照表）
