# 慢病病种管理流程（服务端）

> 文档版本 v1.0 | 面向读者：产品、运营

## 概述

慢病病种是医保门诊慢性疾病的基础字典数据，用于标识哪些疾病属于医保慢病范畴。每个慢病病种拥有唯一编码和名称，可关联多个科室，并支持启用/禁用控制。患者发起医保慢病复诊时，系统根据病种匹配对应科室和医生。

## 涉及接口

| 接口 | 方法 | 路径 | 权限 | 说明 |
|------|------|------|------|------|
| 分页查询 | GET | `/mgmt/chronic` | `hosp:chronic:read` | 按名称/编码/别名筛选 |
| 新增 | POST | `/mgmt/chronic` | `hosp:chronic:write` | 创建新病种 |
| 修改 | PUT | `/mgmt/chronic` | `hosp:chronic:write` | 修改病种（id必传） |
| 详情 | GET | `/mgmt/chronic/detailById/{id}` | `hosp:chronic:read` | 获取单个病种详情 |
| 启用 | POST | `/mgmt/chronic/{id}/_enable` | `hosp:chronic:write` | 启用病种 |
| 禁用 | POST | `/mgmt/chronic/{id}/_disable` | `hosp:chronic:write` | 禁用病种 |
| 删除 | DELETE | `/mgmt/chronic/{id}` | `hosp:chronic:write` | 删除病种 |

## 数据模型

慢病病种的核心字段：
- **名称**：病种中文名称（如"高血压"、"糖尿病"）
- **编码**：唯一编码标识
- **关联科室列表**：该病种可接诊的科室编码列表
- **启用状态**：0-禁用 / 1-启用

## 核心流程

### 新增慢病

1. 管理后台提交病种名称、编码、关联科室列表
2. 服务端校验必填字段
3. 保存病种基本信息
4. 批量保存病种-科室关联关系
5. 返回新增病种ID

### 修改慢病

1. 管理后台提交病种ID、新的关联科室列表
2. 服务端校验ID必传（否则抛出"id必传"异常）
3. 更新病种基本信息
4. 同步更新关联科室列表（先删后增）
5. 返回更新结果

### 启用/禁用

- **启用**：病种状态设为1，在医保慢病复诊中可用
- **禁用**：病种状态设为0，已禁用的病种在患者端不显示

### 删除

删除病种时同时删除关联的科室关系。

## 业务规则

| 规则 | 说明 |
|------|------|
| 修改必传ID | 修改接口强制要求传入ID参数，缺失时后端直接返回"id必传" |
| 权限分级 | 读权限和写权限分离，读权限可查看列表和详情，写权限可新增/修改/启用/禁用/删除 |
| 科室关联 | 病种与科室是多对多关系，一个病种可关联多个科室 |

## 状态流转

```
[新增] → 禁用(默认) ↔ 启用 → [删除]
```

## 关联模块

- **管理后台**：慢病病种管理页面（增删改查+启用禁用）
- **患者端**：医保慢病复诊流程（根据病种匹配科室和医生）
- **科室管理**：关联科室的数据来源
- **服务模块**：`ykj-service-hospital`
