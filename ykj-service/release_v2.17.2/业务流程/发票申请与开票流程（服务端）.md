# 发票申请与开票流程（服务端）

> 文档版本：v1.0 | 面向读者：产品经理、运营、财务

## 概述

发票模块负责处理患者对已完成订单的发票申请。患者可在支付完成后30天内提交开票信息，系统支持个人和企业两种抬头类型。根据系统配置，开票可以是人工模式（管理后台手动操作）或自动模式（提交后自动触发开票事件）。

## 触发条件

- **触发来源**：患者在前端提交开票申请
- **前置条件**：订单必须是已支付的有效订单（支付金额 > 0元）

## 核心流程

### 第一步：提交发票申请

患者在前端填写发票信息提交：

1. **订单校验**：
   - 订单必须存在，且属于当前申请患者（订单的会员ID必须与申请人ID一致），否则提示"订单信息不存在！"
   - 同一订单不能重复提交："已提交过发票信息！"
   - 订单状态必须是"已完成"（flowStatus = COMPLETED）
   - 订单支付金额必须大于0元："免费订单不可申请开票!"
   - 订单支付时间不超过30天："该订单支付已超过30天，不可申请开票"

2. **发票信息校验（非自动开票模式下）**：
   - 手机号码格式校验
   - 邮箱格式校验
   - 选择"企业"抬头时，税号为必填："税号信息必传"
   - 选择"个人"抬头时，个人名称为必填："个人名称必传"
   - 税号格式校验（正则：15/18/20位数字和大写字母）

3. **创建发票记录**：
   - 开票状态初始化为"未开票"
   - 更新订单的开票标记字段为"已申请开票"

### 第二步：执行开票

管理员在管理后台对某条发票记录点击"开票"：

1. **再次校验**：
   - 发票记录必须存在
   - 订单必须是"已完成"状态

2. **开票方式分支**：
   - **自动开票模式**（系统配置 `autoInvoice=true`）：发布自动开票事件，由第三方开票系统异步处理
   - **手动开票模式**（默认）：直接更新发票状态为"已开票"，记录开票时间

### 第三步：查询与管理

管理后台通过分页查询接口查看发票列表：
- 可按订单编号、患者信息、开票状态、抬头类型等筛选
- 导出功能：支持将发票数据导出为Excel
- 票种标记：列表中自动计算"是否可开票"——仅订单状态为"已完成"的才标记为可开票

### 第四步：修改发票信息

支持更新发票的抬头类型和企业名称。

## 状态说明

| 开票状态 | 含义 |
|----------|------|
| 未开票 | 已提交申请，尚未开具发票 |
| 已开票 | 已开具（手动模式：管理后台操作完成；自动模式：事件已发布） |

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 同一订单重复申请 | 提示"已提交过发票信息！" |
| 订单不属于该患者 | 提示"订单信息不存在！" |
| 支付超过30天 | 提示"该订单支付已超过30天，不可申请开票" |
| 订单金额为0 | 提示"免费订单不可申请开票!" |
| 订单未完成 | 管理后台开票时校验，不放行 |
| 手机号格式错误 | 提示"手机号码格式错误" |
| 邮箱格式错误 | 提示"邮箱格式错误" |
| 企业抬头未填税号 | 提示"税号信息必传" |
| 个人抬头未填个人名称 | 提示"个人名称必传" |
| 税号格式不符合要求 | 提示"税号格式错误" |

## 关联功能

- **订单模块**：发票申请依赖订单的支付状态和完成状态
- **自动开票系统**：通过事件总线对接第三方发票系统
- **系统配置**：`autoInvoice` 开关控制是否启用自动开票
