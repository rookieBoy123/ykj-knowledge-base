# 运费规则设置与实时计算流程（服务端）

> 文档版本 1.0 | 面向读者：产品/技术

## 概述

服务端提供运费规则的**管理端设置**和**患者端实时计算**两套接口。管理端设置运费模板后，数据存入数据库并同步到Redis缓存；患者端下单时根据收货地址实时匹配运费规则，计算最终运费。

## 涉及服务模块

- `ykj-service-order`：运费数据存储、计算、管理接口
- Redis缓存：运费规则缓存（key: `freight:items`）

## 数据结构

运费规则由多个 `FreightItems` 组成：

| 字段 | 类型 | 说明 |
|------|------|------|
| effectiveAreasList | List\<Areas\> | 有效地区列表，为空表示"其他地区" |
| freight | BigDecimal | 运费金额（≥0） |
| freeEnabled | Integer | 满额包邮开关：0=关闭, 1=开启 |
| minAmount | BigDecimal | 包邮阈值（≥0） |
| collectEnabled | Integer | 到付开关：0=关闭, 1=开启 |
| gratisEnabled | Integer | 医生免运费开关：0=关闭, 1=开启 |

每个 `Areas` 包含：code（编码）、name（名称）、parentCode（父级编码）、parentName（父级名称）

## 管理端接口

### 1. 获取运费规则 `GET /mgmt/freight`
- 权限：`order:freight:read`
- 逻辑：优先从Redis缓存读取，缓存不存在则从数据库加载后刷新缓存

### 2. 设置运费规则 `POST /mgmt/freight/_setting`
- 权限：`order:freight:write`
- 逻辑：校验→写入数据库→刷新Redis缓存

**校验规则（verifyFreightItems）**：

| 校验项 | 异常提示 |
|--------|---------|
| 运费项列表为空 | "费用项不能未空" |
| "其他地区"（effectiveAreasList为空）不唯一 | "其他未选择地区配置有且仅有一个!" |
| 满额包邮开启但minAmount为null | "满额包邮订单金额不能为空!" |
| 未选择到付但运费为null | "未选择到付则运费不能未空！" |
| 地区编码重复（同条件内） | "一个地区只能设置一个条件!" |
| 地区编码跨条件重复 | "一个地区只能设置一个条件!" |

**缓存刷新（refreshCache）**：
- 方法加 `synchronized` 锁，保证并发安全
- 先删除旧缓存，再从数据库读取最新数据写入Redis

## 患者端运费计算

### 接口：`calculateFreight(地址, 订单总金额, 是否免运费)`

**计算逻辑（优先级从高到低）**：

1. **医生免运费（gratisFreight=true）**：直接返回"包邮"，运费为0
2. **运费规则为空**：返回"包邮"，运费为0
3. **地址匹配**：按城市→省份→默认（"其他地区"）的优先级匹配
   - 先匹配城市名称（calculateFreightAddress.city）
   - 未匹配到再匹配省份名称（calculateFreightAddress.province）
   - 仍未匹配到使用 effectiveAreasList 为空的"其他地区"规则
4. **匹配到规则后的计算**：
   - 如果开启到付（collectEnabled=1）：返回"到付"
   - 如果不满足包邮条件（未开启满额包邮 或 开启但订单金额未达到阈值）：返回对应运费金额
   - 否则：返回"包邮"

### 接口：`isGratisEnabled()` — 查询是否允许医生免运费
- 从"其他地区"（effectiveAreasList为空）的规则中检查gratisEnabled字段
- 返回true/false

## 状态图

```
患者下单
    │
    ├─ 医生免运费=true ──→ 包邮（运费0）
    │
    ├─ 运费规则为空 ──→ 包邮（运费0）
    │
    └─ 正常计算
         │
         ├─ 匹配到城市规则
         ├─ 未匹配→匹配省份规则
         └─ 仍未匹配→使用"其他地区"
              │
              ├─ 到付模式 → 到付
              ├─ 满额包邮达标 → 包邮
              └─ 不满足包邮 → 收取运费
```

## 关联功能

- 管理后台运费设置页 → 调用 GET/POST `/mgmt/freight`
- 药品订单创建 → 调用 calculateFreight 计算运费
- 医生开处方 → 调用 isGratisEnabled 判断是否显示免运费选项
