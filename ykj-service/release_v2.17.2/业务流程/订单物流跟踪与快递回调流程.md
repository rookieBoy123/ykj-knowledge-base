# 订单物流跟踪与快递回调流程

> 文档版本 1.0 | 面向角色：系统管理员、运营人员

## 概述

当药品订单发货后，系统通过接入快递100（kd100）物流平台，自动订阅物流轨迹信息。快递平台在有物流状态变化时主动推送更新，系统接收并存储物流轨迹，供患者和医生在小程序端实时查看包裹的运输状态。

## 参与角色

- **患者**：查看自己订单的物流轨迹
- **医生**：查看自己开具处方的订单物流轨迹
- **快递100平台**：物流轨迹数据提供方，主动推送轨迹更新
- **系统管理员**：配置物流订阅的开关和参数

## 流程步骤

### 第一步：发货时订阅物流

当订单发货后（获得快递单号和快递公司编码），系统调用快递100的订阅接口：

1. 检查物流功能是否开启（`logistics.enabled` 配置开关）。如果未开启，直接跳过订阅，记录警告日志
2. 根据快递单号检查是否已存在订阅记录，防止重复订阅（同一快递单号不会重复订阅）
3. 向快递100发送订阅请求，携带以下参数：
   - 快递公司编码
   - 快递单号
   - 收货人手机号（用于快递100的隐私保护）
   - 回调地址（快递100推送物流更新的URL）
   - 签名盐值（用于回调验签）

4. 快递100返回订阅结果：
   - 200：订阅成功
   - 501：重复订阅（不视为错误，正常返回）
   - 其他：订阅失败，抛出业务异常

### 第二步：快递100推送物流更新

订阅成功后，快递100会根据物流轨迹变化情况主动推送更新：

- **首次推送**：订阅后15分钟到4小时内，推送首次轨迹信息
- **后续跟踪**：之后每4小时跟踪一次，有轨迹变化就推送
- **三天无结果**：如果3天内查无结果（录错单号/快递公司错误/揽收延迟），推送"3天查无结果"通知

推送回调通过 `/callback/logistics/kd100` 或 `/callback/logistics/paas` 接口接收。

### 第三步：接收和处理物流回调

系统接收到快递100的回调推送后：

1. **验签**：使用盐值校验请求签名的合法性，防止伪造推送
2. **解析轨迹数据**：从推送参数中提取物流轨迹信息，包括：
   - 快递单当前状态（0在途、1揽收、2疑难、3签收、4退签、5派件、6退回、7转投）
   - 是否已签收标记
   - 快递公司编码
   - 快递单号
   - 轨迹明细列表（每条包含时间、地点、状态描述）

3. **存储或更新**：
   - 如果该快递单号在数据库中还不存在，新增物流记录
   - 如果已存在，更新物流记录（覆盖旧的轨迹数据）
   - 轨迹明细以JSON格式存储在 details 字段中

4. **返回确认**：返回成功响应给快递100，告知已收到推送；如果返回失败，快递100会重试3次，间隔35分钟

### 第四步：患者端查看物流

患者在小程序端通过 `/api/patient/logistics` 接口查询物流信息：

- 传入快递单号
- 系统校验该快递单号对应的订单是否属于当前患者（不是当前患者的订单会提示"参数错误"）
- 返回物流轨迹详情，包括快递状态和完整的轨迹明细列表

### 第五步：医生端查看物流

医生在小程序端通过 `/api/doctor/logistics` 接口查询物流信息：

- 传入快递单号
- 系统校验该快递单号对应的订单是否属于当前医生
- 返回物流轨迹详情

## 物流状态说明

| 状态码 | 含义 | 说明 |
|--------|------|------|
| 0 | 在途 | 包裹正在运输途中 |
| 1 | 揽收 | 快递员已取件 |
| 2 | 疑难 | 配送出现异常，需要人工处理 |
| 3 | 签收 | 收件人已签收 |
| 4 | 退签 | 收件人拒收或退回 |
| 5 | 派件 | 快递员正在派送中 |
| 6 | 退回 | 包裹退回寄件方 |
| 7 | 转投 | 转投其他快递公司或地址 |

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 物流功能开关未开启 | 跳过订阅，记录警告日志 |
| 同一快递单号重复订阅 | 直接跳过，记录警告日志 |
| 快递100订阅接口返回失败 | 抛出异常"订阅物流信息失败，请联系管理员处理" |
| 快递100返回重复订阅（501） | 不视为错误，正常返回 |
| 快递100回调验签失败 | 返回失败响应，快递100会重试推送 |
| 患者查看非自己的物流 | 提示"参数错误" |
| 医生查看非自己的物流 | 提示"参数错误" |
| 三天查无物流结果 | 快递100会推送通知，可以重新订阅 |

## 关联功能

- **订单发货**（`ykj-service-order`）：发货操作触发物流订阅
- **快递100平台**：第三方物流轨迹数据源
- **患者端小程序**：药品订单 → 查看物流
- **医生端小程序**：处方记录 → 查看物流
