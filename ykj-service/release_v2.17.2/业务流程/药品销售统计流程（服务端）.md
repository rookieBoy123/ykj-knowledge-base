# 药品销售统计流程（服务端）

> 文档版本：2026-07-04 | 面向读者：产品经理、开发人员

## 概述

服务端为管理后台的「药品订单统计」页面提供数据查询和导出能力，支持按处方类型（成药/中药）分别统计销售数据，并提供财务视角的聚合统计和导出功能。系统有两个独立入口：药房端（药房管理员视角）和财务端（平台财务视角），两个入口共用相同的统计逻辑但权限和默认状态不同。

## 涉及模块

- `ykj-service-medicine`：药品模块 — 药房端统计入口
- `ykj-service-inquiry`：问诊模块 — 财务端统计入口
- `ykj-service-order`：订单模块 — 提供药品统计数据模型

## 系统架构

```
管理后台前端
    ├── 药房药品统计 → MgmtDrugStatisticsController（药房端）
    │     接口: GET /mgmt/medicine/drug-statistics
    │     导出: POST /mgmt/medicine/drug-statistics/_export
    │
    └── 财务药品统计 → FinanceDrugStatisticsController（财务端）
          接口: GET /mgmt/finance/drug-statistics
          导出: POST /mgmt/finance/drug-statistics/_export
```

## 核心流程

### 查询药品销售统计

1. 前端传入筛选条件：支付日期范围、订单编号、订单类型、医生ID、所属团队ID
2. Controller 层校验权限（药房端：`med:drug-statistics:read` / 财务端：`fin:drug-statistics:read`）
3. 时间范围自动标准化（DateRange 对起止时间做日级精度校准）
4. **默认状态处理**：如果前端未指定订单状态，系统自动设置默认状态过滤：
   - 药房端默认：已支付(20)、已发货(30)、已完成(60)、已结算(80)
   - 财务端默认：已支付(20)、已发货(30)、已完成(60)
5. **药房端特殊逻辑**：如果当前登录用户是药房管理员，自动限定只查询本药房的药品数据
6. 查询数据库（处方表），按药品分组合并计算
7. 对成药和中药分别处理：
   - **成药**：销售数量 × 单价 = 小计金额
   - **中药**：按剂数计算销售数量，小计金额 = 单价 × 数量

### 导出药品销售统计

1. 校验导出日期范围不超过 **92天**（约3个月），超出则返回错误"只能导出三个月之内的数据"
2. 财务端导出额外关联医生信息（医生姓名、专家团队名称、所属商务人员、科室名称）
3. 生成 Excel 文件，文件名：
   - 药房端：药品销售统计.xls
   - 财务端：财务-药品销售统计.xls
4. 导出含序号列，分页统一处理

## 接口权限汇总

| 接口路径 | HTTP方法 | 权限码 | 用途 |
|---------|---------|--------|------|
| `/mgmt/medicine/drug-statistics` | GET | `med:drug-statistics:read` | 药房端查询药品统计 |
| `/mgmt/medicine/drug-statistics/_export` | POST | `med:drug-statistics:export` | 药房端导出药品统计 |
| `/mgmt/finance/drug-statistics` | GET | `fin:drug-statistics:read` | 财务端查询药品统计 |
| `/mgmt/finance/drug-statistics/_export` | POST | `fin:drug-statistics:export` | 财务端导出药品统计 |

## 异常处理

| 情况 | 处理方式 | 类型 |
|------|---------|------|
| 导出日期范围超过92天 | 返回错误"只能导出三个月之内的数据" | 后端拦截 |
| 药房管理员登录 | 自动限定本药房数据，无需前端传递 pharmacyId | 后端自动处理 |
| 未指定订单状态 | 自动使用默认状态列表过滤 | 后端自动处理 |

## 关联功能

- **交易流水查询**（OrderFlowController）：查询所有订单的收入/支出流水
- **药品管理**：药品的基础信息维护
- **处方管理**：处方的审核和下发
