# 标准药品信息管理流程（服务端）

> 文档版本：2026-07-04 | 面向读者：后端开发、系统对接

## 概述

标准药品信息是系统的基础数据模块，维护药品的药监ID、本位码、通用名、规格、说明书等标准数据。管理后台通过 `MgmtStdMedController` 提供增删改查接口，供药品管理员维护标准化药品库。

## 接口清单

| 方法 | 路径 | 功能 | 权限 |
|------|------|------|------|
| POST | `/mgmt/std/stdMed` | 新增药品 | `std:med:write` |
| PUT | `/mgmt/std/stdMed` | 编辑药品 | `std:med:write` |
| GET | `/mgmt/std/stdMed/{id}` | 查看药品详情 | `std:med:read` |
| GET | `/mgmt/std/stdMed/info` | 分页查询药品列表 | `std:med:read` |
| POST | `/mgmt/std/stdMed/info/import` | Excel 导入药品信息 | `std:med:import` |
| POST | `/mgmt/std/stdMed/guide/import` | Excel 导入药品说明书 | `std:med:import` |

## 数据模型

标准药品管理涉及三张数据表，通过 `StdMedAssembleService` 组装：

- **药品基础信息（StdMedInfo）**：药监ID、本位码、通用名、商品名、生产企业、批准文号、规格、剂量等
- **药品扩展信息（StdMedExtra）**：建议用法用量（每次用量、使用频次、使用方式、建议天数）
- **药品说明书（StdMedGuide）**：品牌、功能主治、成分、性状、贮藏、不良反应、禁忌、注意事项

前端提交时使用 `StdMedAssembleDTO` 一次性打包三部分数据，服务端通过 `StdMedAssembleService` 拆解后分别写入三张表。

## 核心逻辑

### 新增流程

1. 接收 `StdMedAssembleDTO` 请求体
2. 校验必填字段（药监ID 12位字母数字、通用名、商品名、生产企业、剂量、剂量单位、制剂数量、制剂单位、包装单位、药品规格）
3. `StdMedAssembleService.save()` 分别写入三张表
4. 返回成功

### 编辑流程

1. 根据 ID 查询现有记录
2. 合并新数据更新三张表
3. 返回成功

### 查询详情

- `detailsByMedId(Long id)` 同时从三张表查询，组装为 `StdMedAssembleDTO` 返回

### 分页查询

- 支持按「药品通用名」和「商品名称」搜索
- 返回 `StdMedInfoDTO` 的基础信息分页（不含扩展信息和说明书）

### Excel 导入

- **药品信息导入**（`/info/import`）：接收 Excel 文件流，通过 `StdMedInfoService.stdMedInfoImport()` 批量导入药品基础信息
- **说明书导入**（`/guide/import`）：批量导入药品说明书信息

## 异常处理

| 异常场景 | 处理方式 |
|----------|----------|
| 缺少必填字段 | 前端校验拦截（表单校验规则），后端 `@Validated` 再次校验 |
| 药监ID格式不符 | 前端校验：必须 12 位字母数字组合 |
| 药品本位码格式不符 | 前端校验：14 位纯数字 |
| 编辑时 ID 不存在 | 数据库更新 0 行，需前端处理 |
| 导入 Excel 格式错误 | `StdMedInfoService` 内部解析失败时抛出异常 |

## 关联模块
