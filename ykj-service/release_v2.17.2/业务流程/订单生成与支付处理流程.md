# 订单生成与支付处理流程

> 文档版本：v1.0 | 最后更新：2026-07-01 | 面向读者：产品/运营

## 概述

当患者在平台上发起任何需要付费的操作（问诊、开药、预约服务、购买健康服务包等），系统会生成一笔统一订单，然后引导患者通过收银台完成支付。服务端统一管理订单的创建、支付、发货、签收和退款全生命周期。

---

## 触发条件

- 患者发起问诊并确认问诊费用
- 医生开具处方后患者确认购买药品
- 患者预约线下体检/核酸/健康证等服务
- 患者购买健康服务包
- 患者发起群会诊
- 他人通过分享链接代支付

---

## 订单状态流转

系统为订单定义了以下核心状态，按顺序流转：

```
待支付 → 已支付 → (待发货) → 已发货/待收货 → 已完成
                                         ↘ 已取消（支付前）
                                         ↘ 已过期（超时未付）
已支付 → 退款中 → (退款完成/退款失败)
```

**特殊处理**：
- 0 元订单：创建后直接进入「已完成」状态，跳过支付环节
- 处方药品订单：支付后进入「待发货」，发货后进入「已发货/待收货」
- 订金订单：订单金额分为订金和尾款两部分
- 医保订单：标记 `isMedicalInsurance=true`，支付渠道独立处理

## 步骤详解

### 第一步：订单创建

系统根据业务场景生成订单号（格式：`O` + 年月日 + 7位随机数），并记录以下信息：

- **订单来源**：区分问诊、处方、预约、会诊、服务包等类型
- **订单金额**：总金额、运费、订金（如有）
- **支付对象**：收款医生/药房信息
- **患者信息**：患者姓名、手机号
- **过期时间**：订单默认有效期，超时未支付自动作废
- **收货地址**：药品配送地址（仅处方订单）

如果订单金额为 0（如免费问诊），系统直接将订单状态设为「已完成」，并记录支付时间。

### 第二步：收银台支付准备

患者进入收银台时，系统执行以下检查：

1. **状态校验**：确认订单为「待支付」状态，已支付或已失效的订单直接拦截
2. **防重复支付**：主动查询历史支付渠道，如果发现已支付但回调未到的订单，系统自动补单并拦截本次支付
3. **支付金额计算**：订单总金额 - 已付金额 = 本次应付金额
4. **支付渠道组装**：
   - 默认支持微信支付（优先）和支付宝
   - 如果系统开启医保支付且当前业务允许，额外显示医保支付选项
   - 医保支付需校验：医生是否为医保医师、患者是否已医保定点

### 第三步：发起支付

患者选择支付方式后：

- **微信支付**：系统调用微信统一下单接口，返回预支付参数给小程序调起微信支付
- **支付宝支付**：系统调用支付宝下单接口，返回支付链接
- **医保支付**：系统跳转医保授权页面，获取授权码后调用医保支付接口

系统记录预支付信息（prepayOrder），用于后续对账和回调验证。

### 第四步：支付结果处理

支付完成后，支付平台通过回调通知系统：

1. **回调验签**：验证支付平台回调的真实性
2. **更新订单状态**：将订单从「待支付」改为「已支付」
3. **记录支付流水**：记录收入流水（包括支付渠道、交易单号、金额）
4. **后续业务触发**：
   - 处方订单：通知药房备货发货
   - 问诊订单：通知医生开始接诊
   - 预约订单：生成核销条形码
   - 会诊订单：启动会诊流程

### 第五步：订单完成

- **药品订单**：药房发货→物流运输→患者签收（或发货15天后自动签收）→订单完成
- **服务订单**：服务完成/核销→订单完成
- **问诊订单**：问诊结束→订单完成

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 订单已支付但用户再次进入收银台 | 提示"当前订单已支付"，拦截重复支付 |
| 订单已过期 | 提示"订单已失效"，引导返回 |
| 支付回调未到达（用户已付款但状态未更新） | 系统支持历史渠道查询，发现后自动补单 |
| 支付中用户取消 | 预支付单保留，用户可重新发起支付 |
| 0元订单 | 跳过支付，直接完成 |
| 订金订单 | 先付订金，尾款后续支付 |
| 医保支付不可用 | 隐藏医保选项，仅显示微信/支付宝 |
| 微信支付失败 | 系统记录失败原因，前端提示用户重试 |

## 关联功能

- **退款管理**：已支付订单可通过退款申请流程退回款项（详见退款流程文档）
- **发票管理**：已支付订单可申请电子发票
- **物流查询**：药品订单可查看物流轨迹
- **代支付**：订单可分享给他人代付
