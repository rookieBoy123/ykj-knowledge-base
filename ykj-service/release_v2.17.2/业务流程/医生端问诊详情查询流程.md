# 医生端问诊详情查询流程（服务端）

> 文档版本 v1.0 | 面向读者：产品/技术

## 概述

本流程描述医生在小程序端查看某个患者的问诊详情时，服务端的数据组装和处理逻辑。问诊详情页聚合了患者信息、问卷答案、病历记录、处方列表、量表评估等多维度数据。

## 触发条件

- 医生在问诊列表点击某个问诊记录进入详情页
- 服务端接口：`ApiDoctorInquiryController` -> `inquiryDetail`

## 流程详解

### 一、问诊基本信息查询

加载问诊单数据：
- 查询问诊单主表 → 获取问诊状态(flowStatus)、医生服务状态(doctorFlagStatus)、患者信息、服务类型等
- doctorFlagStatus 状态含义：
  - `医助待接诊`：分配给医助但尚未接诊
  - `医助接诊中`：医助正在接诊
  - `主治待接诊`：转诊给主治医生等待接诊
  - `主治接诊中`：主治医生正在接诊
  - `再次等待`：等待再次分配
  - `已结束`：问诊已结束

### 二、患者信息聚合

查询患者就诊人信息，包括：姓名、性别、年龄、身份证号（脱敏）、成员关系等

### 三、问卷答案查询

根据问诊单关联的问卷版本，查询问卷题目和患者的作答：
- 从问卷模板查询题目列表
- 从问诊记录查询患者的每道题答案
- 按题目类型组装展示（单选、多选、文本、图片等）

### 四、病历记录查询

查询本次问诊关联的病历记录：
- 病历ID、病历状态
- 主诉、现病史、既往史、过敏史
- 临床诊断信息
- 电子签章状态

### 五、处方列表查询

查询本次问诊开具的所有处方：
- 处方编号、处方状态（审核中/审核通过/审核不通过等）
- 药品清单（药品名+规格+数量+用法用量）
- 处方总金额
- 签章状态

### 六、关联模块数据

根据配置开关，可能还会加载：
- **量表评估结果**：患者完成的量表测评得分和结果分析
- **检查报告**：HIS系统同步的检查检验报告
- **导诊记录**：AI预问诊或导诊环节的记录

### 七、权限校验

- 验证当前医生是否为该问诊的接诊医生
- 医助只能查看自己接诊的问诊
- 主治医生可以查看已转诊给自己的问诊

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 问诊单不存在 | 返回404或"问诊单不存在" |
| 无权限查看 | 返回403或"无权访问该问诊" |
| 问诊已结束 | 部分操作按钮隐藏，数据只读展示 |
| 病历未形成 | 病历区域显示"暂未形成病历" |

## 关联模块

- **问诊预约服务**（InquiryAppointServiceImpl）：问诊生命周期管理
- **病历服务**（MedicalRecordServiceImpl）：病历创建、编辑、签章
- **处方服务**（PrescriptionCommandServiceImpl）：处方数据查询
- **问卷服务**（SurveyServiceImpl）：问卷模板和答案数据
