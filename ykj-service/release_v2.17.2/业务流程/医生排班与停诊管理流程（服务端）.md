# 医生排班与停诊管理流程（服务端）

> 文档版本：2026-07-06 | 面向读者：产品/运营人员

## 概述

系统为医生提供排班管理功能，支持按星期设置班次（如上午、下午、晚上），每个班次可细分为多个号段（时间段）。支持排班发布、停诊、以及从排班模板自动生成排班。

## 触发条件

- **前置条件**：医生已经过审核，排班模板已配置
- **入口**：管理后台医生排班页 / 自动定时同步

## 步骤详解

### 第一步：保存医生排班

管理后台提交某天（如周一）的排班数据，系统执行以下校验：

1. **医生校验**：必须指定医生ID，否则提示"请指定要保存排班的医生"
2. **班次校验**：必须至少有一个班次（scheduleClasses），否则提示"请指定要保存排班的医生"
3. **班次内时间冲突校验**（`verifyClassConflict`）：检查同一天各时段之间是否有时间重叠
4. **历史班次冲突校验**（`verifyTimeForNowConflict`）：检查新班次与已保存的班次是否有时间冲突
5. **号段时间冲突校验**（`verifyScheduleItemsDateConflict`）：检查班次内的号段时间是否有效

保存流程：
- 为每个班次生成唯一编号（scheduleNo）
- 先保存班次详情（号段/时间段）
- 再保存班次主记录
- 删除旧的班次记录（覆盖更新）

### 第二步：发布排班

通过定时任务或手动触发，将未发布的排班统一发布：
- 查找所有"未发布+启用"状态的排班记录
- 如果无待发布排班，提示"当前无待发布排班"
- 批量更新 `isRelease` 为 `true`

### 第三步：停诊

管理人员可以对医生某天的某个班次执行停诊操作：

**停诊参数**：
- 医生ID、日期、班次编号
- 是否全天停诊：如果为true，该天所有班次全部停诊
- 停诊原因：必填，最大100个字符

**停诊校验**：
- 医生ID、日期不能为空
- 如果非全天停诊，必须指定班次编号
- 检查班次是否存在，不存在则提示"班次不存在"
- 检查受影响的号段是否已有患者预约（有无预约列表）

**停诊执行**：
- 将班次的 `isStopReception` 设置为 `true`
- 记录停诊原因（remark）
- 如果更新失败，提示"停诊失败, 请联系管理员"
- 发布停诊事件通知（`ScheduleStopReceptionEvent`），用于通知预约模块取消相关预约

### 第四步：排班查询

**管理端查询**：
- 先查医生列表，再组装每个医生的周排班数据
- 支持按医生姓名、科室筛选
- 支持按日期范围查询

**小程序端查询**：
- 必须指定医生ID和日期，否则提示"请指定医生和班次日期"
- 查询可问诊的班次：已发布 + 未停诊 + 已启用

### 第五步：自动同步排班

系统定期从排班模板自动同步未来15天的排班数据：
- 默认同步天数：15天
- 仅同步启用状态的排班模板

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 未指定医生 | 提示"请指定要保存排班的医生" |
| 未填写班次 | 提示"请正确填写班次" |
| 班次详情保存错误 | 提示"班次详情保存错误" |
| 班次保存错误 | 提示"班次保存错误" |
| 当前无待发布排班 | 提示"当前无待发布排班" |
| 班次不存在 | 提示"班次不存在" |
| 停诊失败 | 提示"停诊失败, 请联系管理员" |
| 查询未指定医生和日期 | 提示"请指定医生和班次日期" |

## 关联功能

- **排班模板**：通过 `ScheduleTemplateService` 自动生成排班
- **医生管理**：通过 `DoctorQueryService` 查询医生信息
- **预约管理**：停诊事件通知预约模块取消相关预约
- **问诊端展示**：小程序调用 `apiFindScheduleByDoctorId` 获取可问诊班次
