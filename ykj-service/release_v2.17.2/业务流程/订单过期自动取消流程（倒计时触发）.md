# 订单过期自动取消流程（倒计时触发）

> 文档版本 2026-07-05 | 面向读者：产品经理、客服、运营

## 概述
小程序订单支付页面展示倒计时，倒计时归零后由小程序主动调用服务端 `/expire` 端点，服务端将订单标记为"已失效"。服务端也会在预支付环节检查订单是否已过有效期，双重保障防止过期订单被支付。

## 触发条件
- 订单处于"待付款"（代码值 10）状态
- 小程序端倒计时归零，用户未完成支付
- 小程序主动调用服务器过期接口

## 步骤详解

### 第一步：小程序倒计时
- 用户在支付页面看到倒计时（默认 XX 小时）
- 倒计时归零时，小程序触发过期操作

### 第二步：调用过期端点
小程序调用 `POST /api/order/{orderNo}/expire`：
- 服务端先查询订单是否存在
- 如果订单已经是异常状态（已取消、已失效、退款中、已退款），幂等返回成功（不报错）
- 如果订单不是待付款状态，拒绝操作（提示"当前订单状态不允许过期操作"）

### 第三步：执行取消
服务端创建取消命令：
- 取消原因标记为"订单超时未支付自动取消"
- `timeoutCancel` 参数设为 true（超时取消标记）
- `refundEnabled` 设为 false（不用退款，因为还没付款）
- 订单状态流转为"已失效（80）"

### 第四步：预支付二次校验
当用户扫描二维码进入支付时，系统额外检查：
- 订单的到期时间（expirationTime）是否已过
- 如果已过期，直接返回"订单已失效"错误（错误码 40001），阻止支付

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 重复调用过期 | 幂等返回成功，不会重复操作 |
| 订单非待付款状态 | 返回错误"当前订单状态不允许过期操作" |
| 预支付时订单已过期 | 返回 40001 错误码"订单已失效" |
| 订单不存在 | 返回错误"订单不存在" |

## 关联功能
- 订单支付流程（预支付接口 `/prepay` 中的过期检查）
- 待付款订单定时清理（定时任务）
- 小程序支付倒计时组件
