# 经典名方增删改查与字典校验流程（服务端）

> 文档版本 2026-07-04 | 面向读者：产品/技术 | 服务模块：ykj-service-hospital

## 概述

经典名方是中医百科的核心内容之一。服务端提供完整的增删改查接口，并在新增/编辑时执行严格的数据校验：名称唯一性、字典编码有效性。同时自动生成拼音字段供患者端检索排序使用。

| 操作 | 接口路径 | 方法 | 权限 |
|------|---------|------|------|
| 分页查询 | `/mgmt/wiki/TCM-classic-formulas` | GET | `hosp:wiki:TCM-classic-formulas:read` |
| 详情查询 | `/mgmt/wiki/TCM-classic-formulas/{id}` | GET | `hosp:wiki:TCM-classic-formulas:read` |

## 核心规则

### 1. 名称唯一性校验

- **新增时**：检查数据库中是否已存在同名处方
- **编辑时**：检查是否存在其他同名处方（排除自身ID）
- 重名时抛出："该处方名称已存在"

### 2. 字典有效性校验

新增/编辑时会校验以下三个字典：

| 校验内容 | 字典编码 | 说明 |
|---------|---------|------|
| 处方类型 | `jtmf-lx` | 如汤剂、丸剂、散剂等，单选 |
| 书籍出处 | `jtmf-sj` | 如《伤寒论》《金匮要略》等，单选 |
| 适应症 | `jtmf-syz` | 如感冒、咳嗽等，可多选 |

如果传入的编码在字典中不存在，返回错误："字典:{编码}, 编码:{值}不存在"。

适应症仅在有值时校验（可为空），处方类型和书籍出处每次必校验。

### 3. 拼音自动生成

新增/编辑时自动调用 `ChinesePinyinUtil.getPinYin()` 生成处方名称的拼音，存入 `pinyin` 字段。该字段供患者端 App 按拼音排序和搜索使用。

### 4. 查询时字典转译

分页查询返回结果时，自动将适应症的编码值转译为中文名称（通过 `assemblerDictionary` 方法）。例如：编码 `gm` → 展示为"感冒"。处方类型和书籍出处的名称转换由前端自行处理。

### 5. 删除规则

- 单个删除和批量删除共用同一个 `delete(List<Long> ids)` 方法
- 删除前检查ID列表是否为空
- 批量删除没有数量限制

## 异常处理

| 异常情况 | 返回信息 |
|---------|---------|
| 处方名称重复（新增） | "该处方名称已存在" |
| 处方名称重复（编辑） | "该处方名称已存在" |
| 处方类型编码不存在 | "字典:jtmf-lx, 编码:xxx不存在" |
| 书籍出处编码不存在 | "字典:jtmf-sj, 编码:xxx不存在" |
| 适应症编码不存在 | "字典:jtmf-syz, 编码:xxx不存在" |
| 删除ID为空 | "请正确选择要删除的经典名方" |
| 数据库操作失败 | 对应操作失败信息 |

## 关联功能

- **患者端经典名方浏览**：`ApiTCMClassicFormulasController` → `findPageForApp()` 提供 App 端分页查询
- **数据字典系统**：处方类型、适应症、书籍出处由字典服务统一管理
