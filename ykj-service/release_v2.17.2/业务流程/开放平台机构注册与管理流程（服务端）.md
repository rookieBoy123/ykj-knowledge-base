# 开放平台机构注册与管理流程（服务端）

> 文档版本：2026-07-05 | 面向读者：产品经理、技术运维

## 概述

开放平台合作机构的注册、信息管理和启停控制。机构是外部合作方接入系统的核心实体，拥有独立的机构ID（AppId）、接口密钥（Secret）和回调密钥（CallbackSecret），用于API鉴权和回调消息签名验证。

## 触发条件

- 前置条件：管理员拥有`open:org:write`权限
- 系统入口：管理后台调用 `/mgmt/hospital/organization` 接口

## 核心流程

### 机构创建流程

```
管理员填写机构信息 → 校验审方配置 → 系统自动生成AppId → 入库存储
                                              ↓
                                   AppId通过序号服务生成（自增数字）
```

**校验规则**（`verifyOrganization`）：
1. 自动审方必须配置审方药师（否则抛异常："自动审方必须配置审方药师！"）
2. 启用CA签章只能选择"按平台流程审方"（否则抛异常："启用CA只能选择按平台流程审方！"）
3. 需要药品下单时药房不能为空（否则抛异常："需要药品下单的状态下，药房不能为空！"）

**AppId生成**：通过`SeqService`获取自增序号（基于数据库序列表`t_organization`的`appId`字段），不是一个随机UUID，保证了机构ID的规律性和可追溯性。

### 机构编辑流程

```
管理员编辑信息 → 校验ID非空 → 执行同上的审方配置校验 → 更新数据库
```

### 机构启停控制

通过 `_enabled` 接口实现一键启用/禁用：
- 传入机构ID和新状态
- 校验`id`和`enabled`两个字段都必传
- 禁用后，通过`findByAppId`查询该机构会返回null（只查`enabled=true`的机构），该机构的API调用全部失效

### 机构查询

**分页列表**：支持按机构名称模糊搜索（`like`）、按启用状态筛选，按增序排列。

**按AppId查询**：`findByAppId`方法只查询已启用的机构（`enabled=true`），这是API鉴权时的核心查询逻辑。

**按ID列表批量查询**：`findByIds`方法也只查询已启用的机构（`enabled=true`）。

## 数据模型

| 字段 | 含义 | 特殊规则 |
|------|------|---------|
| appId | 机构唯一标识 | 系统自动生成，从序列表获取自增值 |
| name | 机构名称 | 最长30字，支持同名（不校验唯一） |
| secret | 接口密钥 | 用于API调用签名验证 |
| callbackSecret | 回调密钥 | 用于回调消息的HMAC签名验证 |
| auditType | 审方设置 | 0=平台审方，其他=自动审方 |
| pharmacistIds | 审方药师ID列表 | 自动审方时必填，多个用逗号分隔 |
| enabledCA | 是否启用CA签章 | 启用时不可用自动审方 |
| neededDrugOrder | 是否需要药品下单 | 启用时必须选择药房 |
| pharmacyId | 关联药房ID | neededDrugOrder=true时必填 |
| enabled | 启用状态 | 默认true |
| neededRealNameAuth | 是否需要患者实名认证 | 控制患者端行为 |

## 异常处理

| 异常码/场景 | 系统响应 |
|------------|---------|
| 自动审方未配药师 | BizException: "自动审方必须配置审方药师!" |
| CA+自动审方冲突 | BizException: "启用CA只能选择按平台流程审方！" |
| 需药品下单未给药房 | BizException: "需要药品下单的状态下，药房不能为空!" |
| 创建/更新数据库失败 | 返回国际化错误信息 |
| 机构ID不存在 | BizException: "机构信息不存在，id=xxx" |
| AppId为空查机构 | BizException: "appid不能为空！" |
| ID列表为空批量查 | BizException: "ids不能为空" |

## 关联功能

- **开放平台API鉴权**：所有开放平台接口调用前先通过appId查询机构信息和密钥
- **回调消息签名**：callbackSecret用于生成回调消息的签名
- **审方流程**：auditType决定该机构下的处方走自动审方还是平台流程审方
- **权限管理**：接口受`@PreAuthorize`保护，需要`open:org:read`/`open:org:write`权限
