# 系统通知中心流程（服务端）

> 文档版本：2026-07-04 | 面向读者：系统管理员、开发运维

## 概述

系统通知中心为管理后台的所有管理员用户提供实时消息通知功能。支持分页查询历史通知、按角色隔离通知内容、单条/全部已读标记，并通过长轮询机制实时更新未读通知数量。

对应的管理后台页面：消息通知管理（可配置通知模板和触发规则）。

## 核心接口

| 接口 | 方法 | 说明 |
|------|------|------|
| `/mgmt/system/notifications` | GET | 分页查询当前用户的通知列表 |
| `/mgmt/system/notifications/message-count` | GET | 长轮询获取未读通知数量 |
| `/mgmt/system/notifications/{id}/_read` | POST | 标记单条通知已读 |
| `/mgmt/system/notifications/_all-read` | POST | 标记全部通知已读 |

## 关键设计

### 1. 角色隔离

通知查询时，系统根据当前管理员的用户ID和当前角色编码（roleCode）进行过滤。每个管理员在不同角色下看到的通知是独立的 — 切换管理后台角色后，通知列表也会随之变化。

### 2. 长轮询机制

未读通知数量使用长轮询方式获取：
- 客户端发起请求后，服务端延迟5秒后返回结果
- 使用 `DeferredResult` 异步处理，不会阻塞请求线程
- 最多重试2次（通过 `AtomicInteger` 计数），超过次数返回 null
- 使用 `ScheduledExecutorService` 调度延迟执行

这种方式避免了前端频繁轮询，减少服务器压力。

### 3. 已读标记

- **单条已读**：传入通知ID，校验是否为当前用户的未读通知后标记已读
- **全部已读**：将当前用户所有未读通知一次性标记为已读
- 已读操作由 Service 层执行，包含用户ID校验防止越权

### 4. 登录校验

所有接口调用前检查用户ID是否存在，未登录用户访问直接返回 "请先登录" 错误。

## 异常处理

| 异常场景 | 后端处理方式 |
|----------|-------------|
| 未登录用户访问通知接口 | 抛出 BizException("请先登录") |
| 标记他人通知为已读 | Service层校验用户ID，操作不生效 |
| 长轮询超时（超过重试次数） | 返回 HttpResult.ok(null) |
| 分页参数异常 | MyBatis Plus 分页插件自动处理 |
