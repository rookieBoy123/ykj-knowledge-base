# 药品库基础数据管理流程（服务端）

> 文档版本 2026-07-05 | 面向系统管理员和产品经理

## 概述

药品库是整个系统的"药品主数据"来源，定义了所有可以在药房中引用的药品基础信息。药房管理药品时，必须先从药品库中选取已入库的药品，无法凭空创建——这保证了药品信息的标准化和一致性。药品库分为**中药**和**成药**两种类型，使用完全不同的校验规则和数据字段。

## 触发条件

- 管理后台"药品管理 → 药品库"菜单进入
- 运营人员需要新增/编辑/删除/导入药品主数据
- 第三方药房需要引用标准药品信息

## 数据结构

药品库使用主从表设计：

| 表名 | 用途 |
|------|------|
| `medical_drugs_lib` | 主表，存储中药和成药共有的基础信息（类型、通用名、拼音、状态等） |
| `medical_drugs_lib_herbal` | 中药扩展表，存储中药特有的字段（用法用量、药性等） |
| `medical_drugs_lib_patent` | 成药扩展表，存储成药特有的字段（药监ID、批准文号、规格等） |

## 步骤详解

### 新增中药

1. 管理员填写中药信息（通用名、药名、药品形态、计量单位等）
2. 系统校验通用名（别名）是否已存在——同一药品库中不能有重名的中药
3. 如果校验通过，先在主表 `medical_drugs_lib` 插入一条记录（类型=Herbal）
4. 再在中药扩展表 `medical_drugs_lib_herbal` 插入扩展信息，关联主表ID
5. 新增成功后，发布 `DrugsLibAddedEvent` 事件通知药房同步更新
6. 中药的计量单位固定为"克"，系统强制校验——如果字典中该单位的名称不是"克"，会提示"药品计量单位不正确"

### 新增成药

1. 管理员填写成药信息，关键字段是**药监ID**（监管码）
2. 系统首先校验药监ID是否已填写——未填会提示"药品监管id未填写"
3. 然后校验药监ID是否重复——同一药监ID不能重复录入
4. 拼音（药品首字母）为非必填：如果管理员没填，系统自动根据通用名生成拼音首字母
5. 校验通过后，先写入主表，再写入成药扩展表
6. 同样发布事件通知药房

### 删除药品库记录

1. 管理员在列表中选择要删除的药品，点击删除
2. 系统**先删除主表**记录，再根据药品类型删除对应的扩展表记录（中药或成药）
3. 删除操作在同一个事务中执行（@Transactional），保证数据一致性
4. **注意**：删除药品库记录不会自动删除已引用该药品的药房药品——药房药品有独立的删除逻辑

### 药监ID查重与查询

1. 管理员可以输入药监ID查询是否已存在
2. 系统返回布尔值：true表示重复，false表示未使用
3. 药监ID为空时会提示"药品药监ID不存在, 请联系管理员"（这是防御性校验）

### 标准药监ID同步

1. 系统定时检测成药扩展表中**缺失药监ID**的记录
2. 调用标准药品服务 `ApiStdMedService.findYpId()`，传入通用名、厂家、规格等信息
3. 如果标准库返回了匹配的药监ID，自动回填到药品库
4. 无匹配则不更新（不报错，静默跳过）

### Excel导入

1. 管理员上传Excel文件（中药或成药模板）
2. 系统逐行解析，每行数据执行校验：
   - **中药**：校验通用名重名、药品计量单位正确性
   - **成药**：校验药监ID重名
3. 拼音为空时自动生成首字母
4. 成功导入的数据发布批量事件 `publishEvent(null, type, true)`（id为null表示批量新增）
5. 失败的行记录错误原因，汇总返回到前台展示

### Excel导出

- 中药导出字段包括：通用名、药名、药品形态、计量单位、拼音等
- 成药导出字段包括：通用名、药名、药监ID、批准文号、厂家、规格、剂型单位等
- 导出文件名为"药品库中药"或"药品库成药"

## 异常处理

| 情况 | 系统处理方式 |
|------|------------|
| 新增中药时通用名重复 | 后端拦截，提示"药品已存在" |
| 新增成药时药监ID重复 | 后端拦截，提示"药品已存在" |
| 药监ID未填写 | 后端拦截，提示"药品监管id未填写" |
| 中药计量单位不是"克" | 后端拦截，提示"药品计量单位应为：克" |
| 计量单位编码不在字典中 | 后端拦截，提示"药品计量单位不正确" |
| 药品通用名重复（查询单个时出现多条） | 后端拦截，提示"药品库存在重复通用名药品:xxx" |
| 药监ID为空时查询 | 后端拦截，提示"药品药监ID不存在, 请联系管理员" |
| Excel导入药品类型未知（非成药非中药） | 后端拦截，提示"未知的药品类型, 导出错误" |
| 药品库信息不存在（药房引用时） | 后端拦截，提示"药品库信息不存在" |

## 关联功能

- **药房药品管理**：药房引用药品库中的药品，再补充价格、库存、图片等药房特有信息
- **药品黑名单**：导入时通过 `ImportDrugsLibListener` 校验药品是否在黑名单中
- **标准药品服务**：通过 `ApiStdMedService` 同步标准药监ID
- **开放平台**：提供 `findDrugsLibPatentBySupervisionId` 和 `findDrugsLibHerbalByDrugsAlias` 供外部调用
