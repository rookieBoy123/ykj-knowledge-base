# 开放平台创建问诊与信息补充流程

> 文档版本：2026-07-02 | 面向读者：产品/运营/技术对接

## 概述

外部合作机构（如HIS系统、第三方医疗平台）通过开放平台接口在系统中创建问诊记录。机构可以上传问诊信息（患者、诊断、病历图片），系统自动完成会员注册、患者建档、问诊单生成、IM群聊创建、病历生成等一系列操作。创建后机构还可以补充修改病史和诊断信息。本流程覆盖 ykj-service-open 模块的后端核心逻辑。

## 步骤详解

### 一、问诊发起（initiate）

外部机构通过 `/open/inquiry/initiate` 接口发起问诊：

**机构校验**：根据 AppId 查找机构信息，找不到则提示"找不到对应的机构信息"

**预约问诊参数校验**：如果是预约问诊（method=Reservation），必须提供预约开始时间和结束时间

**参数组装**：
- 处理病历图片：外部图片URL通过OSS转储到系统内部存储
- 患者信息：姓名、性别、年龄必须提供
- 病史信息（可选）：过敏史、既往史、现病史
- 扩展信息：来源标识、预约时间、是否禁用聊天室

**调用内部问诊初始化服务**（`inquiryInitiateService.initiate`）完成问诊单创建

### 二、完整问诊上传（upload）

适用于需要一次上传完整问诊信息（含聊天记录）的场景：

**第一步 — 注册会员**：根据患者手机号调用会员注册服务（`apiMemberService.register`）

**第二步 — 保存患者信息**：
- 外部传参未指定患者关系时，默认设为"其他"关系
- 调用 `apiPatientService.save` 保存患者档案

**第三步 — 创建问诊**（`create`）：
- 机构校验（根据AppId查找）
- 查找或匹配医生（根据机构类型和医生姓名/身份证号）
- 生成问诊单（`inquiryAppointDto`）：
  - 服务类型：图文问诊（ImageText）
  - 业务类型：普通问诊（General）
  - 问诊状态：进行中（FLOW_STATUS_UNDERWAY）
  - 医生状态：主治进行中（DOCTOR_FLAG_STATUS_MAIN_UNDERWAY）
  - 生成UUID作为群聊ID
- 保存问诊医生和问诊患者关联记录
- 创建IM群聊（群名：医生姓名+工作室，加入医生和患者）
- 创建病历记录（含诊断信息）
- 处理订单创建（支持线上/线下支付及0元订单）
- 异步转储病历图片

**第四步 — 生成聊天记录**：
- 生成问诊过程中的聊天消息（医生接诊时间、最后消息时间等）
- 将问诊状态直接置为"已完成"（以便对外机构同步数据）

**第五步 — 完成问诊**：
- 更新问诊开始时间、结束时间
- 发送问诊结束事件

### 三、补充修改问诊信息（updateBase）

已创建的对外问诊可以通过 `/open/inquiry/updateBase` 接口补充修改：

**可修改的内容**：
- 病情描述（同时更新问诊单和病历）
- 病历图片（先保存URL，异步转储）
- 患者病史信息（过敏史、既往史、现病史、个人史、家族史、婚育史、月经史、舌诊、脉诊、体征、治疗史）
- 诊断信息（编码+名称+诊断类型）

**自动补创建订单**：
- 如果问诊还没有关联订单，系统自动创建（金额从问诊单读取）
- 支持指定支付方式（0=未支付/1=微信/2=支付宝/3=线下）

**病历修改特殊处理**：
- 对外同步的病历默认不触发CA签名流程（`disableCa=false`）
- 允许修改诊断信息（`disableValidDelBindingDiagnosis=true`）

### 四、取消问诊（cancel）

外部机构可通过 `/open/inquiry/cancel` 取消问诊：

- 机构校验（根据AppId）
- 问诊所有权校验（问诊的机构ID必须与请求机构一致，否则提示"问诊不属于请求机构,无权限取消"）
- 状态校验：仅"待接诊"和"进行中"状态允许取消，其他状态提示"该问诊状态不允许取消"
- 支持设置是否退款（`noRefund`参数）

### 五、获取问诊视频记录（findInquiryVideoRecords）

外部机构可通过 `/open/inquiry/video/records` 获取问诊过程中的音视频通话记录URL：
- 同样需要机构校验和问诊归属校验

## 异常处理

| 错误提示 | 触发条件 |
|---------|---------|
| "找不到对应的机构信息" | AppId未匹配到机构 |
| "参数缺失, 预约问诊需要提供预约开始、结束时间" | 预约问诊缺少时间参数 |
| "参数缺失, 请正确填写患者信息" | 患者姓名/性别/年龄未提供 |
| "找不到对应的问诊信息" | 问诊单号不存在 |
| "问诊不属于请求机构,无权限取消" | 机构无权操作该问诊 |
| "该问诊状态不允许取消" | 问诊状态不符合取消条件 |
| "该问诊单不属于机构数据" | 问诊归属校验不通过 |
| "获取医生im账号信息失败" | IM账号同步失败 |
| "获取患者im账号信息失败" | IM账号同步失败 |
| "图片转存操作失败" | 外部图片URL无法访问或OSS异常 |
| "找不到对应的病历信息" | 更新时病历不存在 |
| "找不到对应的问诊患者信息" | 更新时患者关联不存在 |
| "找不到对应的患者信息" | 补创建订单时患者信息缺失 |
| "找不到对应的医生信息" | 医生不存在 |
| "找不到对应的医生会员信息" | 医生会员不存在 |
| "找不到对应的患者会员信息" | 患者会员不存在 |

## 关联功能

- **开放平台处方上传**（OpenPrescriptionServiceImpl）：机构上传处方
- **开放平台订单**（OpenOrderServiceImpl）：机构创建药品订单
- **问诊预约模块**（InquiryAppointServiceImpl）：问诊单的激活和状态管理
- **病历模块**（MedicalRecordServiceImpl）：病历的创建和诊断关联
- **IM模块**：群聊创建和聊天记录
- **订单模块**：问诊费用订单创建
