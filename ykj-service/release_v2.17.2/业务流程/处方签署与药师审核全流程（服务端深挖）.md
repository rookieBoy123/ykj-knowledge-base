# 处方签署与药师审核全流程（服务端深挖）

> 文档版本 2026-07-06 | 面向：产品经理、运营人员

## 概述

医生开具处方后，系统进入电子签章与药师审核流程。该流程涉及医生签署处方PDF、药师专业校验、审方通过/驳回、处方状态流转以及处方撤回、删除、变更等操作。

## 触发条件

- **前置条件**：医生已完成处方开具，处方处于待审核状态
- **入口**：医生提交处方后自动触发签署流程；药师端收到审方任务后手动审核

## 步骤详解

### 一、处方签署与提交（submit）

医生提交处方时，系统进入签署流程（第334-384行）：

1. 生成处方PDF（A5纸张），嵌入系统印章
2. 将PDF转为Base64，创建签署记录
3. 如系统开启了CA签章功能，返回CA签署标识给前端，由医生完成手写签名
4. 如未开启CA，直接完成医生开方，推送处方创建事件

### 二、药师审核（audit）

药师审核处方时的校验逻辑（第163-215行）：

1. **状态校验**：处方必须是"医生已通过 + 药师待审核"状态，否则提示"当前处方不能被审核"（第168行）
2. **时效校验**：检查处方是否超时。如果处方过期时间已到，提示"处方单超时未审核已失效"（第179行）——处方有效期为配置的小时数
3. **身份校验**：当前登录用户必须是已启用的药师身份，非药师提示"非药师身份或者当前药师不可用, 不能审核处方"（第183行）
4. **专业校验**（第724-741行）：药师专业必须匹配处方类型：
   - 西药师（专业编码01）只能审西药处方
   - 中药师（专业编码02）可审中药处方、以及只含中成药的成药处方
   - 专业不匹配提示"您登记的药师专业不能审核当前类型的处方"
5. 药师审核通过后进入签署流程（同医生签署），不通过则驳回

### 三、处方完成（complete）

签署完成后确认处方状态（第438-507行）：

- **医生角色**：当前状态必须是"审核中"才能完成，否则提示"当前处方不能完成开方, 处方状态异常"
- **药师角色**：当前状态必须是"审核中"才能完成，否则提示"当前处方不能审核, 处方状态异常"
- 完成后推送处方创建事件或审核事件，触发后续流程（如推送订阅消息、发送IM通知）

### 四、处方撤回（revoke）

医生可以撤回自己开具的处方（第284-300行）：

- 如果处方已生成订单（患者已支付），撤回失败，提示"撤回失败，该处方已存在订单，请联系管理员取消订单后重新开方"
- 撤回后只撤销医生的签署记录，并发布处方撤回事件

### 五、处方删除（delete）

删除处方的条件（第303-315行）：

- 只能删除"草稿"或"已撤回"状态的处方，其他状态提示"非草稿、已撤回状态的处方不允许删除"
- 只有处方医生本人才能删除，他人删除提示"您无权限删除处方"

### 六、患者查看处方（viewByOutpatientNo）

患者查看处方时（第319-331行）：
- 首次查看会标记为已读，并推送"处方已查看"事件
- 重复查看不会重复标记

### 七、处方过期处理（expiration）

定时触发过期处理（第510-528行）：

- 超时未审核的处方，状态标记为"已过期"
- 撤销医生的签署记录
- 发布处方过期事件

### 八、诊断重置（resetDiagnosis）

处方诊断需要修改时（第539-596行）：
- 重置处方诊断和病情描述
- 删除当前处方的所有签名记录
- 重新生成处方并自动完成医生开方和药师审方（跳过CA签章）

## 状态流转

```
草稿 → 医生审核中 → 医生已通过 → 药师审核中 → 药师已通过（审方完成）
         ↓              ↓             ↓
       已删除         已过期        药师已驳回
```

## 异常处理

| 情况 | 处理方式 |
|------|---------|
| 处方状态不匹配 | 后端拦截，提示"当前处方不能被审核"或"处方状态异常" |
| 处方已超时 | 后端拦截，提示"处方单超时未审核已失效" |
| 非药师/药师不可用 | 后端拦截，提示"非药师身份或者当前药师不可用, 不能审核处方" |
| 药师专业不匹配 | 后端拦截，提示"您登记的药师专业不能审核当前类型的处方" |
| 处方已存在订单时撤回 | 后端拦截，提示"撤回失败，该处方已存在订单" |
| 非草稿/已撤回状态删除 | 后端拦截，提示"非草稿、已撤回状态的处方不允许删除" |
| 非本人删除处方 | 后端拦截，提示"您无权限删除处方" |
| 未找到处方患者信息 | 后端拦截，提示"未找到处方患者信息, 无法审核处方, 请联系管理员" |
| 病历签名未完成 | 后端拦截，提示"系统配置生成病历需要签名, 请前往写病历页面操作完成后再开处方" |
