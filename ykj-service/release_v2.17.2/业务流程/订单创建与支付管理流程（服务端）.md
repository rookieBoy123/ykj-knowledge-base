# 订单创建与支付管理流程（服务端）

> 文档版本 1.0 | 面向非技术读者 | 2026-07-02

## 概述

订单系统是整个平台的交易核心，负责处理所有类型的订单创建、支付、取消和退款。订单来源包括处方订单、问诊订单、会诊订单和预约订单四大类。系统通过事件机制与其他业务模块（药品、问诊、医保等）联动，确保各环节数据一致性。

## 触发条件

- 患者在前端发起支付（药品购买、问诊付费、会诊缴费、服务预约付款）
- 订单超时自动取消
- 用户或系统发起退款申请

## 核心流程

### 一、订单创建

订单创建时系统执行以下步骤：

**1. 生成订单号**：以"O"开头 + 日期（8位）+ 随机数（7位）生成唯一订单编号。

**2. 判断零元订单**：如果订单总金额为0：
- 处方订单：状态直接设为"已支付"
- 需发货的订单：状态设为"待发货"
- 其他类型：状态设为"已完成"，自动记录完成时间
- 零元订单不需要走支付流程

**3. 计算已付金额**：测试环境下，非零元且非医保订单，总金额减0.01元作为测试支付金额。

**4. 计算过期时间**：设置订单24小时有效期，超时未支付将自动取消。

**5. 写入数据库**：
- 订单主表（订单基本信息）
- 订单商品表（药品或服务项目明细）
- 收货地址表（如有配送需求）
- 订单费用项表（各项费用明细如药品费、诊金、运费等）

**6. 发布创建事件**：通知其他模块（如处方模块标记已下单、问诊模块更新状态等）。

### 二、订单取消

**取消条件**：订单必须处于可取消状态（待支付、已完成[仅问诊和预约类型]等）。

**取消分类**：

| 取消类型 | 说明 | 最终状态 |
|---------|------|---------|
| 普通取消 | 用户主动取消 | 已取消（CANCELED） |
| 超时取消 | 24小时未支付自动取消 | 已失效（EXPIRED） |
| 第三方取消 | 处方订单通知第三方系统取消 | 事件通知后不修改本地状态 |
| 退款取消 | 已支付订单取消并申请退款 | 退款中（REFUNDING） |

**退款流程**：取消已支付订单时，系统自动创建退款申请单，推送退款事件给退款处理模块。支持自动退款（autoRefund=true）直接触发退款。

### 三、支付准备

支付前系统执行以下校验：

1. **分布式锁**：防止同一订单重复支付
2. **订单状态检查**：必须处于待支付或支付中状态
3. **支付渠道选择**：根据订单配置和系统开关确定可用支付方式
4. **支付金额计算**：订金模式支付订金金额，全款模式支付总金额

**支付方式**：
- 微信支付（微信JSAPI支付）
- 支付宝支付
- 医保统筹支付
- 医保个账支付
- 医保微信混合支付
- 线下支付（仅标记，不走线上支付）

### 四、申请退款

已支付订单支持单独申请退款（不取消订单）：

1. 校验订单已成功付款
2. 查找支付流水记录
3. 退款金额不能大于实际支付金额
4. 支持微信退款和医保退款两种渠道

**医保退款流程**：
- 获取医保授权码
- 查询医保订单信息
- 医保未退则先退医保（统筹+个账），再退微信支付部分
- 医保已退则只退微信支付部分
- 退款成功后记录退款流水

### 五、自动签收

处方订单发货15天后，系统定时自动签收：
- 查询发货超过15天且未签收的处方订单
- 状态从"已发货待收货"改为"已完成"
- 记录签收时间
- 每次处理100条，避免大数据量问题

### 六、到店自提取货

支持通过取货码到店自提：
- 校验取货码是否正确
- 检查订单是否已取货
- 更新订单状态为已完成
- 记录取货人和取货时间

## 订单状态流转

```
待支付(PENDING_PAYMENT) ──零元──→ 已完成(COMPLETED)
       │                              ↑
       ├──支付成功──→ 已支付(PAYED) ──┤ (处方订单15天自动签收)
       │                   │          │
       ├──超时──→ 已失效(EXPIRED)     │
       │                   ↓          │
       └──取消──→ 已取消(CANCELED)  待发货(PENDING_SHIPMENT)
                           │          │
                    退款申请→ 退款中(REFUNDING)
                           │
                    退款完成→ 已退款(REFUNDED)
```

## 异常处理

| 情况 | 系统处理 | 提示信息 |
|------|---------|---------|
| 订单生成失败 | 抛出内部错误 | "订单生成失败，请联系管理员" |
| 订单取消时状态不允许 | 拒绝操作 | "订单取消失败，当前状态不可取消" |
| 已完成订单取消（非问诊/预约） | 拒绝操作 | "订单取消失败，当前状态不可取消" |
| 未付款订单申请退款 | 拒绝操作 | "订单未成功付款，不支持申请退还金额" |
| 未找到支付流水 | 拒绝操作 | "未找到订单支付信息，无法退款，请联系管理员" |
| 退款金额大于支付金额 | 拒绝操作 | "申请退款金额不能大于实际支付金额" |
| 订单已绑定其他会员 | 拒绝操作 | "订单已绑定会员" |
| 取货码错误 | 拒绝操作 | "取货码有误" |
| 订单已取货 | 拒绝操作 | "该订单已取货" |
| 分布式锁获取失败 | 等待5秒后重试 | 无提示（自动重试） |

## 关联模块

- **处方模块**（ykj-service-inquiry）：处方下单后更新处方状态
- **医保模块**（ykj-service-yb）：医保支付和退款
- **会员模块**（ykj-service-member）：订单绑定会员
- **物流模块**：订单发货和配送
- **退款模块**（ykj-service-order）：退款申请处理
- **开放平台**（ykj-service-open）：第三方订单通知
