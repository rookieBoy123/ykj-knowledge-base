# 医生排班管理与停诊处理流程

> 文档版本 2026-07-02 | 面向读者：产品经理、运营、客服

## 概述

描述医生排班的完整后端逻辑，包括管理后台手动排班、排班模板自动同步、停诊处理、号源占用与释放。排班决定了患者能否预约医生以及什么时候可以问诊。

## 触发条件

- **管理后台**：管理员手动为医生设置排班
- **定时任务**：系统每天自动从模板同步生成未来15天的排班
- **医生/管理员**：医生临时停诊
- **患者预约/取消时**：号源的占用和释放

## 步骤详解

### 第一步：手动保存排班

管理员为某医生某天设置排班：

1. **参数校验**：医生ID、班次列表不能为空
2. **班次名称冲突检查**：同一天内班次名称不能重复
3. **班次时间冲突检查**：同一天内不同班次的时间段不能重叠
4. **号段时间范围校验**：每个号段的时间必须在所属班次的时间范围内
5. **号段时间冲突检查**：同一班次内不同号段的时间不能重叠
6. **当天排班特殊校验**：若排班日期是今天，需额外校验：
   - 结束时间已过的号段不允许新增
   - 当前时间内号段不允许编辑号源数
   - 不允许修改已过期号段的启用状态
7. **生成唯一编号**：为每个班次和号段生成UUID编号
8. **保存策略**：先新增班次详情→再保存班次→最后删除旧班次数据（保证数据的原子性）

### 第二步：排班自动同步

定时任务每天自动执行：

1. 从模板列表中获取所有排班模板
2. 计算15天后的日期作为目标排班日期
3. 遍历所有需要同步的医生，检查是否已存在该日期的排班
4. 若不存在，用模板数据创建当天的排班（默认状态：未发布、未停诊）

### 第三步：排班发布

管理员可一键发布所有"未发布"状态的排班：

1. 筛选所有 `isRelease=false` 且 `isEnabled=true` 的排班
2. 批量将其 `isRelease` 设置为 `true`
3. 若没有待发布的排班，提示"当前无待发布排班"

### 第四步：医生停诊

医生或管理员可停用某个班次或整天停诊：

1. **全天停诊**：将该医生当天所有班次标记为停诊
2. **班次停诊**：仅停用指定的单个班次
3. **停诊原因**：若该班次已有人挂号（已用号源>0），必须填写停诊原因；原因不超过100字
4. **推送停诊通知**：停诊后通过事件机制通知已挂号的患者

### 第五步：号源使用与释放

患者预约/取消预约时操作号源：

**预约（使用号源）**：
1. 先校验排班状态：已启用、已发布、未停诊
2. 校验号源结束时间：预约不能超过号段结束时间前30分钟，否则提示"预约不能超过号段结束时间前30分钟"
3. 双重校验号源可用性：扣减前再次检查号源是否有剩余
4. 号源已用数+1，班次已用数+1

**取消预约（释放号源）**：
1. 取消时不校验停诊状态（允许已停诊的班次取消）
2. 号源已用数-1，班次已用数-1，班次已退数+1

### 第六步：患者端排班查询

患者查看医生排班时：

1. 仅返回 `已启用 + 已发布 + 未停诊` 的班次
2. 每个班次下展示号段的时间段和剩余号源数
3. 不返回空班次（无有效号段的班次）

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 班次名称重复 | 提示"班次名称存在冲突, 请检查" |
| 班次时间重叠 | 提示"班次时间存在冲突, 请检查" |
| 号段时间超出班次范围 | 提示"号段时间XX-XX, 不在XX班次时间内" |
| 号段时间冲突 | 提示"号段时间存在冲突, 请检查" |
| 无待发布排班 | 提示"当前无待发布排班" |
| 班次不存在 | 提示"班次不存在" |
| 号源已用完 | 提示"班次信息发生变更, 请刷新后重试" |
| 预约超时（离号段结束<30分钟） | 提示"预约不能超过号段结束时间前30分钟" |
| 排班信息变更 | 提示"预约信息发生变更, 请刷新后重试" |
| 停诊无原因（有挂号患者） | 提示"停诊的班次已有患者挂号, 请填写停诊原因" |
| 停诊原因超长（>100字） | 提示"停诊原因不能超过100字" |
| 当天已过号段编辑 | 提示"小于当前时间的排班不允许操作" |

## 关联功能

- 医生排班模板管理（后台设置模板→定时任务自动同步）
- 患者预约问诊（选择医生→查看排班→选号段→预约）
- 问诊流程（预约成功后进入问诊）
- 号源管理（预约占用/取消释放）
