# 诊疗科目配置管理流程（服务端）

> 文档版本 2026-07-05 | 面向运维/开发人员

## 概述

诊疗科目（MedicalSubjects）是互联网医院的基础配置数据，代表医院获得执业许可的诊疗科目。每个科目包含科目编码、科目名称和三个执业许可日期。该模块为前端"诊疗科目配置"管理页面提供数据支撑，同时为科室单位配置提供科目依赖关系。

## 数据模型

数据表：`t_medical_subjects`

| 字段 | 类型 | 说明 |
|------|------|------|
| id | Long | 主键 |
| code | String | 科目编码（不可重复） |
| name | String | 科目名称 |
| licenseIssuingDate | Date | 执业许可发证日期 |
| licenseEffectiveDate | Date | 执业许可生效日期 |
| licenseExpirationDate | Date | 执业许可到期日期 |
| description | String | 描述 |
| remark | String | 备注 |
| deleted | Integer | 逻辑删除标记（0=正常，1=已删除） |

## 接口说明

服务模块：`ykj-service-hospital`
接口基路径：`/mgmt/hospital/medical-subjects`

| 方法 | 路径 | 功能 | 权限 |
|------|------|------|------|
| POST | `/` | 新增科目 | `hosp:medical-subjects:write` |
| PUT | `/` | 修改科目 | `hosp:medical-subjects:write` |
| DELETE | `/{id}` | 删除科目 | 继承默认删除权限 |
| GET | `/page` | 分页查询 | 继承默认分页权限 |
| GET | `/{id}` | 按ID查询详情 | 继承默认查询权限 |

## 核心业务逻辑

### 1. 新增流程

```
接收请求 → 地区校验 → 编码重复检查 → 插入数据
```

- **地区校验**：仅云南地区（`yunnan`）强制校验三个执业许可日期。如果三个日期中任一为空，抛出异常："执业许可发证/生效/到期日期不能为空"
- **重名检查**：以科目编码（code）为准，先查询是否存在相同编码。如果已存在，抛出异常"请勿重复添加"

### 2. 修改流程

```
接收请求 → 地区校验 → 编码重复检查（排除自身） → 更新数据
```

- 地区校验规则与新增相同
- 重复检查排除当前记录自身的ID，确保修改时不因"与自己重复"而报错

### 3. 删除流程

```
接收请求 → 查询科目 → 检查关联科室 → 删除
```

- 删除前会查询该科目编码下是否有关联的科室（通过 `DepartmentAppService.findByMedicalCategoryCode`）
- 如果存在关联科室，抛出异常"请删除关联科室再删除科目！"，阻止删除

### 4. 查询流程

- **分页查询**：支持按科目名称模糊搜索（右匹配），不支持按编码搜索
- **详情查询**：按ID精确查询单条记录
- **全量查询**：提供 `findAll()` 方法返回所有未删除的科目

## 状态流转

诊疗科目无复杂状态机，只有正常/已删除两种状态（逻辑删除）。

## 异常处理

| 情况 | 异常类型 | 提示信息 |
|------|---------|---------|
| 新增时编码已存在 | BizException | "请勿重复添加" |
| 修改时编码冲突 | BizException | "请勿重复添加" |
| 删除有关联科室的科目 | BizException | "请删除关联科室再删除科目！" |
| 云南地区日期未填 | BizException | "执业许可发证日期不能为空" 等 |
| 请求参数缺失必填项 | Validation异常 | "科目编码不能为空" / "科目名称不能为空" |

## 关联模块

- **科室单位配置**（UnitConfiguration）：科室通过 `medicalCategoryCode` 关联到诊疗科目的 `code` 字段
- **数据字典**（Dictionary）：科目编码列表来源于 `ksbm` 分类的字典数据
- **医院配置**（HospitalConfig）：地域自适应逻辑决定哪些地区显示诊疗科目标签页
