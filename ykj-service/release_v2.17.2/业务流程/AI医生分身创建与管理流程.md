# AI医生分身创建与管理流程

> 文档版本 1.0 | 面向读者：运营、产品

## 概述

平台为医生创建AI数字分身，配置服务费和基本信息，生成专属二维码供患者扫码进入AI问诊。AI分身继承真实医生的基本信息（姓名、科室、职称、医院等），提供自动化图文问诊服务。

## 状态机

```
创建 → 启用(STATUS_ENABLE) ⇄ 禁用(STATUS_DISABLE)
```

## 核心流程

### 1. 创建AI分身

**前置校验**：
- 医生必须存在且未被禁用（status≠DISABLE）
- 医生必须有"医生"工作类型（workType=Doctor）
- 该医生尚未创建过AI分身（一人只能有一个AI分身）
- 服务费不能为负数，且不能超过配置的最大值

**创建步骤**：
1. 校验医生信息和费用
2. 如果未填写AI分身名称，自动生成"AI+医生姓名"
3. 初始状态设为"启用"（STATUS_ENABLE）
4. 自动为医生开通AI图文问诊服务（ServiceType=AiImageText），服务费按配置金额
5. 自动生成专属二维码

### 2. 编辑AI分身

- 可以修改服务费、名称等信息
- 同样需要校验医生状态和费用范围
- 编辑时会同步更新医生的AI图文问诊服务费用

### 3. 启用/禁用

- **启用**（enable）：AI分身状态变为启用
- **禁用**（disable）：AI分身状态变为禁用，前端不再展示AI分身入口

### 4. 查询AI分身

- **按医生ID查询**（findByDoctorId）：返回该医生的AI分身详情
- **查询可用分身**（findAvailable）：先检查系统开关（func.aiDoctorEnabled），再检查分身状态，两者都满足才返回
- **分页列表**（pageList）：后台管理使用，支持筛选条件，编号格式为"AI+医生ID"

### 5. AI分身信息组装

返回的AI分身数据包含：
- AI分身自身信息（名称、编号、状态、服务费、问诊次数、二维码）
- 医生基础信息（姓名、头像、科室、职称、医院、简介、是否名医）
- 专家团队信息（如果该医生是某专家团队的首席医生，则包含团队医生列表和头像）

### 6. 问诊次数统计

每次AI问诊完成后，系统自动更新askCount（问诊次数+1）。

## 异常处理

| 情况 | 后端处理 |
|------|----------|
| AI分身不存在 | 抛出"AI医生不存在！" |
| 医生已被禁用 | 抛出"该医生已被禁用，请联系管理员！" |
| 医生无医生工作类型 | 抛出"该医生无医生工作类型！" |
| 重复创建 | 抛出"该医生已存在AI医生！" |
| 服务费小于0 | 抛出"服务费不能小于0！" |
| 服务费超过上限 | 抛出"服务费不能超过X元！" |
| 二维码生成失败 | 抛出"生成AI医生二维码失败！" |
| 系统开关关闭 | 返回null，不展示AI分身功能 |
