# 后台管理端多方式登录流程（服务端）

> 文档版本：2026-07-03 | 面向读者：系统管理员、运维人员

## 概述

后台管理系统支持三种登录方式：**用户名密码登录**（管理后台）、**手机验证码登录**（线下药房）、**AccessKey登录**（程序化调用）。所有登录方式通过统一的认证服务处理，登录成功后返回访问令牌，后续请求携带令牌完成身份认证。

服务模块：`ykj-service-system`

## 触发条件

- 管理员打开管理后台登录页面，输入用户名密码
- 线下药房操作员打开药房管理端，通过手机号+验证码登录
- 第三方系统通过 AccessKey 程序化调用 API

## 步骤详解

### 一、管理后台用户名密码登录

**接口**：`POST /mgmt/login`

**请求参数**：
- `username`：登录用户名（必填，不可为空）
- `password`：登录密码（必填，不可为空）

**业务逻辑**：
1. 校验用户名和密码均不为空（接口层 `@Valid` 校验）
2. 调用 `AdminAuthenticationService.login(username, password, LoginClient.MANAGER)`
3. 认证通过后返回 `AdminLoginResult`，包含：
   - 访问令牌（accessToken）
   - 用户基本信息（用户名、姓名）
   - 用户角色列表
   - 当前使用的角色信息
   - 权限列表（permissions）

### 二、线下药房手机验证码登录

**接口**：`POST /mgmt/login/offlineLogin`

**请求参数**：
- `account`：手机号（必填）
- `checkCode`：短信验证码（必填）

**业务逻辑**：
1. 校验手机号和验证码均不为空
2. 调用 `AdminAuthenticationService.mobileLogin(account, verifyCode, LoginClient.OFFLINE)`
3. 返回 accessToken（仅返回令牌字符串，非完整的登录结果对象）

**获取验证码**：先调用 `GET /mgmt/login/getCheckCode?mobile=xxx`，系统向手机发送短信验证码。限制条件：
- 手机号必须是系统已注册用户
- 距离上一次获取验证码必须超过 10 分钟

### 三、AccessKey 程序化登录

**接口**：`GET /mgmt/login/accessKeyLogin`（另有 `accessKeyLoginV2` 作为备用入口）

**请求参数**：
- `accessKeyId`：访问密钥ID
- `accessKeySercet`：访问密钥Secret

**业务逻辑**：
1. 调用 `AdminAuthenticationService.accessKeyLogin(accessKeyId, accessKeySercet)`
2. 认证通过返回 `AccessKeyLoginResult`，包含访问令牌
3. 通常用于第三方系统对接，如开放平台自动下单

### 四、角色切换

**接口**：`POST /mgmt/user/changeRole`

当一个用户拥有多个角色时（如既是"运营专员"又是"财务专员"），可以在管理后台切换当前使用的角色，从而获得不同的菜单和权限：

1. 传入要切换到的角色编码（roleCode）
2. 系统校验该用户是否拥有该角色
3. 切换成功后返回新的 accessToken，前端用新令牌重新加载菜单

### 五、登出

**接口**：`POST /mgmt/login/logout`

调用 `loginService.logout()` 清除当前用户的认证信息，使令牌失效。

## 异常处理

| 情况 | 系统处理方式 |
|------|------------|
| 用户名或密码为空 | 接口层拦截，返回校验错误"用户名不能为空"或"密码不能为空" |
| 用户名密码错误 | 认证服务返回登录失败（401 未授权） |
| 用户被禁用 | 已禁用的用户在 Spring Security 层被拦截，无法完成登录 |
| 手机号未注册 | "手机号登录必须是系统注册用户" |
| 验证码发送频率过高 | "距离上一次接收短信时间必须大于10分钟" |
| 手机号或验证码为空 | 返回"用户账号/手机号码不能为空"或"验证码不能为空" |
| AccessKeyID/Secret 无效 | 认证服务返回认证失败 |
| 角色切换时角色码无效 | 用户无此角色，切换失败 |

## 关联功能

- **用户管理**：登录账号必须在用户管理中存在且状态为启用
- **角色管理**：登录后返回的角色列表来自角色管理模块
- **菜单管理**：登录后加载的菜单由角色关联的菜单权限决定
- **访问密钥管理**：AccessKey 登录所需的密钥在访问密钥管理页面创建
- **系统操作日志**：登录操作会记录审计日志
