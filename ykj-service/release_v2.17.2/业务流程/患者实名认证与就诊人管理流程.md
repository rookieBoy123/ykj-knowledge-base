# 患者实名认证与就诊人管理流程

> 文档版本 1.0 | 面向非技术读者 | 2026-07-01

## 概述

平台要求患者（就诊人）在使用问诊、开药等核心功能前必须完成实名认证。系统通过人脸识别（FaceID）校验姓名和身份证号是否匹配，并将就诊人绑定到当前会员账号下。本流程描述就诊人从新增到认证、从绑定到管理的完整后端逻辑。

## 触发条件

- 患者在小程序中添加新就诊人
- 患者编辑已有就诊人信息
- 患者从线下门诊同步就诊人数据

## 步骤详解

### 第一步：新增就诊人

当患者在小程序中填写就诊人信息并提交时，系统执行以下校验：

1. **身份证号格式校验**：根据证件类型分别校验——大陆身份证（18位格式）、护照、港澳通行证、台湾通行证各有独立的格式校验规则。
2. **非大陆证件补充信息**：如果证件类型不是大陆身份证，必须手动填写性别和出生日期（系统无法从证件号自动提取）。
3. **并发锁保护**：使用"真实姓名+身份证号"作为锁键，防止同一就诊人被重复创建。

### 第二步：实名认证

如果平台开启了身份校验开关（`identityVerifyEnabled`），系统会调用人脸识别服务进行实名认证：

- **新就诊人**：调用FaceID校验姓名+身份证号是否匹配 → 认证通过则标记为"已认证"
- **已有就诊人且未认证**：同样触发FaceID校验 → 认证通过后更新认证状态

如果平台**未开启**身份校验开关，则就诊人默认为"未认证"状态。

### 第三步：自动提取信息

通过身份证号自动计算并填充：

- **性别**：根据身份证第17位奇偶性判断男/女
- **出生日期**：从身份证第7-14位提取
- **姓名拼音首字母**：用于后管的拼音搜索功能
- **年龄**：根据出生日期自动计算

### 第四步：绑定会员关系

就诊人创建后，自动与当前会员账号建立绑定关系：

- **关系类型**：本人、配偶、子女、父母、其他等
- **一对一绑定规则**：同一会员+同一就诊人只能绑定一次
- **本人关系限制**：每个会员只能有一个"本人"关系的就诊人（防止一人多绑）

### 第五步：同步收货地址

如果新增就诊人时填写了省市区+详细地址，系统会**自动**将该地址同步保存为该会员的收货地址（方便后续药品配送）。

### 第六步：触发建档事件

就诊人创建成功后，系统发布"患者注册事件"，通知其他模块同步数据（如导诊系统、HIS系统）。

## 就诊人编辑流程

编辑已有就诊人时，系统会：

1. **信息合并**：如果编辑后的姓名+身份证号与系统中另一个就诊人匹配，系统自动合并两条记录，将旧的就诊人关系替换为新的。
2. **重新认证**：编辑姓名或身份证号后，系统重新触发实名认证（因为身份信息已变化）。
3. **地址更新**：编辑地址信息只更新字段，不触发重新认证。

## 就诊人解绑流程（短信验证）

患者可以通过短信验证码解绑就诊人：

1. 系统发送短信验证码到会员绑定手机号
2. 输入验证码后，删除会员与就诊人的绑定关系
3. 注意：解绑只删除关系，不删除就诊人数据本身

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 证件号码格式错误 | 提示"请输入正确的证件号码" |
| 非大陆证件未填性别/生日 | 提示"非大陆居民身份证请正确填写性别和生日" |
| 同一就诊人重复创建 | 并发锁保护，已存在则更新而非新建 |
| 本人关系已绑定 | 校验拦截，不能重复绑定"本人" |
| 一人绑定多会员 | 100600错误码：提示"同一就诊人只能绑定在一个手机号"，询问是否解绑 |
| 患者不存在或已解绑 | 提示"患者不存在或者已经被解绑" |
| 未实名认证 | 提示"患者未实名认证"，阻止使用医保等功能 |
| 请求频率过高 | 提示"请求频率过高，请稍后再试"（分布式锁保护） |

## 医保相关

- 查询患者慢病病种时，必须先通过实名认证
- 系统会从医保接口获取患者的险种信息（职工医保/居民医保/新农合）和慢病病种列表
- 未实名认证的患者无法使用医保支付

## 关联功能

- **医保慢病**：实名认证后查询患者慢病资格
- **收货地址**：新增就诊人时自动同步地址
- **HIS对接**：患者建档事件触发HIS数据同步
- **问诊下单**：下单时校验患者可用性（`findUsableById`）
