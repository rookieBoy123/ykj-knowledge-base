# 订单支付渠道管理与重复支付防护流程

> 文档版本 2026-07-02 | 面向读者：产品、运营

## 概述

订单创建后进入待支付状态，系统根据业务规则自动组合可用的支付渠道（微信支付、支付宝、医保支付），并内置多重防护机制防止重复支付和超时支付。

## 触发条件

- 患者在小程序端点击"去支付"
- 处方订单、问诊订单、服务包订单等各类订单进入待支付状态

## 步骤详解

### 第一步：获取可用支付渠道

系统根据订单类型自动判断哪些支付方式可用：

- **微信支付**（JSAPI）：所有订单默认支持，默认选中，排在第一位
- **微信扫码支付**（Native）：所有订单默认支持
- **支付宝支付**：所有订单默认支持
- **医保支付**（微信医保）：仅处方订单支持，需开启医保支付功能开关（medicalInsurancePay），且订单类型为处方且无订金

每个支付渠道都有独立的可用性校验结果（如医保支付会校验患者参保资格、问诊类型是否为慢病等）。

### 第二步：计算需支付金额

支付金额的计算规则：

- **首次支付**：需支付金额 = 订单总金额 - 已付金额（通常为 0）
- **订金支付**：需支付金额 = 订金金额（如有）
- **测试环境**：若开启测试支付，系统将订单总金额调整为最终只需支付 0.01 元，其余金额自动标记为已付

### 第三步：支付预处理（防重复校验）

在调用第三方支付接口前，系统执行三重防重复校验：

1. **状态校验**：检查订单是否处于"待支付"状态。若已支付则提示"当前订单已支付"；若非待支付状态则提示"订单已失效"
2. **历史渠道查单**：查询该订单之前发起过的所有预支付记录，对每个历史支付渠道（微信/支付宝）逐一向第三方查询真实支付状态：
   - 查到已支付 → 系统自动补单（将订单状态推进到已支付），然后拦截本次支付请求，提示"当前订单已支付"
   - 查单失败（如网络抖动）→ 按未支付放行，保证支付体验不受影响
   - 未查到 → 正常进入下一步
3. **预支付锁**：对订单号加分布式锁（5 秒等待、10 秒租约），若锁获取失败则提示"支付处理中，请稍后重试"，防止并发重复调用

### 第四步：调用第三方支付

通过支付策略工厂分发到对应的支付渠道（微信/支付宝/医保），各自完成预支付下单并返回支付参数（如微信的 prepay_id、支付宝的 trade_no）。

预支付前的清理动作：关闭该订单的旧预支付单（含第三方接口关闭 + 数据库标记），清除跨渠道的支付二维码缓存。

### 第五步：支付回调完成

收到微信/支付宝的支付成功回调后：

1. **再次加锁**（入账锁，5 秒等待、10 秒租约）
2. **幂等判断**：检查该笔支付流水（transactionId 或 payOrderNo）是否已经处理过，已处理则跳过
3. **重复支付检测**：若订单已不是"待支付"状态，说明这是重复支付（如回调延迟到达），系统自动发起退款
4. **保存流水**：记录收入流水
5. **推进订单状态**：
   - 咨询/预约/会诊类订单 → 直接完成（已完结）
   - 服务包订单 → 待发货
   - 处方订单 → 已付款（待药房发货）
6. **生成取货码**：处方订单/实物订单自动生成取货码
7. **修正已付金额**：若多次支付导致已付金额超过总金额，自动截断为总金额
8. **发布支付成功事件**：通知下游模块（如发票生成、积分发放）

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 订单已支付 | 提示"当前订单已支付！"，拦截后续支付操作 |
| 订单已失效（超时/取消） | 提示"订单已失效！" |
| 历史渠道查单发现已支付 | 自动补单后提示"当前订单已支付！" |
| 支付中（并发锁冲突） | 提示"支付处理中，请稍后重试" |
| 重复支付回调 | 自动原路退款（退款金额=重复支付的金额） |
| 已付金额超过总金额 | 系统自动将已付金额截断为订单总金额，避免财务异常 |
| 医保退款 | 先调医保退款接口（若未退），再调微信医保退款，成功后插入支出流水 |

## 支付渠道对照

| 支付方式 | 渠道标识 | 默认选中 | 适用场景 |
|---------|---------|---------|---------|
| 微信支付（JSAPI） | Wechat | ✓ | 小程序内支付，所有订单 |
| 微信扫码（Native） | QrWechat | ✗ | PC端/扫码支付 |
| 支付宝 | Alipay | ✗ | 所有订单 |
| 医保支付 | Insurance | ✗ | 仅处方订单，需开启医保功能 |

## 关联功能

- 订单生命周期管理（订单创建→待支付→已付款/已完成）
- 医保费用上传流程（医保支付前需先上传费用明细）
- 退款审核与处理流程（重复支付自动退款、申请退款）
- 处方预结算流程（结算后创建订单进入支付）
