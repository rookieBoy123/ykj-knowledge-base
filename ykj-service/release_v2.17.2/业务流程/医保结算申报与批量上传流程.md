# 医保结算申报与批量上传流程（服务端）

> 文档版本 v1.0 | 面向读者：后台开发/医保管理员

## 概述
医保结算申报是平台将已完成的医保订单数据上报给医保系统的关键流程。服务端负责管理结算单的完整生命周期——从订单支付完成自动生成结算草稿，到管理员审核后批量上传至医保局系统。结算单包含患者信息、病种信息、科室信息、支付信息等核心数据。

## 触发条件
- **数据来源**：患者通过医保支付完成订单后，系统自动生成医保结算记录
- **上传触发**：管理员在前台选择结算单后，调用批量上传接口
- **权限要求**：需要拥有"医保结算上传"权限（权限编码 `medins:declares:upload`）

## 核心步骤

### 1. 结算单生成
当患者通过医保支付（微信医保/医保个账/医保统筹）完成订单支付时，系统自动生成一条医保结算记录。结算单关联订单编号、患者身份证、病种信息、科室信息等。

### 2. 结算单查询与筛选
管理员可通过以下条件查询结算单：
- **患者姓名**：按姓名模糊匹配
- **结算时间**：按日期范围筛选
- **申报时间**：按申报日期范围筛选
- **人员编号**：医保系统的人员唯一编号
- **病种编码**：关联医保慢病病种
- **身份证号**：加密存储，查询时自动加密匹配
- **部门编号**：所属科室编号
- **申报状态**：0=草稿，1=待上传，2=已上传，3=上传失败，4=已取消

### 3. 批量上传
管理员选中一条或多条结算单后，指定申报日期，点击"批量上传"：
- 系统将选中的结算单状态从"草稿/待上传"更新为"已上传"
- 系统调用医保局接口，上传结算明细数据
- 上传成功后记录申报时间
- 上传失败则标记为"上传失败"状态，供管理员重试

### 4. 结算单详情查看
每条结算单支持查看详情，包含：
- 订单编号、患者姓名、身份证（脱敏显示）、年龄、性别
- 病种名称和编码
- 科室名称和编号
- 支付时间、申报时间
- 当前申报状态

**数据脱敏**：身份证号在返回前端时自动脱敏处理。年龄和性别从身份证号自动计算，不单独存储。

## 申报状态流转
```
草稿(0) → 待上传(1) → 已上传(2)
                     → 上传失败(3)
草稿(0) → 已取消(4)
待上传(1) → 已取消(4)
```

## 异常处理
| 情况 | 处理方式 |
|------|---------|
| 身份证号查询 | 前端传明文，后端自动加密后匹配数据库加密值 |
| 批量上传部分失败 | 不支持部分成功，整体事务回滚（待确认） |
| 申报日期未填写 | 接口接受空值，由业务逻辑层给出默认值（待确认） |
| 重复上传 | 已上传状态的结算单可被再次选中，但上传时会被校验拦截 |
| 权限不足 | 接口返回403，上传按钮对无权限用户隐藏 |

## 关联功能
- **医保慢病管理**：结算单中的病种来源于慢病病种配置
- **医保标准药品目录**：结算明细中的药品需与标准目录匹配
- **收款清单**：结算金额统计汇总
- **订单管理**：结算单与订单一一对应
