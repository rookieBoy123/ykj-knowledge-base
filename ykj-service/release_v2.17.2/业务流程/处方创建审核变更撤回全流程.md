# 处方创建审核变更撤回全流程

> 文档版本 2026-07-02 | 面向读者：产品经理、运营、客服

## 概述

描述医生开处方、药师审核、处方变更、撤回、删除、过期处理的完整后端逻辑。处方经过"草稿→提交→医生签章→药师审核→完成"的严格状态流转，涉及电子签名和CA认证。

## 触发条件

- **医生端**：开具处方（创建/保存草稿/变更）
- **药师端**：审核处方（通过/驳回）
- **医生/管理员**：撤回处方、删除处方
- **定时任务**：处方过期自动处理

## 步骤详解

### 第一步：创建处方草稿

1. **新增草稿**：若没有带门诊号，创建新处方草稿；若有门诊号，检查是新增还是更新已有草稿
2. **引用处方不走草稿更新**：引用已有处方时不更新原处方，走新增草稿的逻辑
3. **同时创建病历**：创建处方草稿时同步创建/更新关联的病历记录（含诊断信息、过敏史、既往病史等）
4. 草稿状态为"待提交"（DoctorApproval.Pending）

### 第二步：正式提交处方

1. **校验指令**：通过 CreatePrescriptionItemValidator 校验处方各字段的合法性
2. **从草稿提交**：若有门诊号则从草稿转为正式处方（引用草稿的处方内容）
3. **关联病历**：若尚未关联病历，创建病历并将诊断信息绑定到处方
4. **设置失效时间**：处方提交后自动计算失效时间（配置的失效小时数，通常24-48小时）
5. **生成电子处方PDF**：组装处方数据→生成HTML→转PDF（A5纸张）→加盖印章
6. **医生签章**：创建医生签署记录，判断是否需要CA签名：
   - **需要CA签名**：返回 `authSign=true` 和 `caUniqueId`，前端引导医生完成CA签章操作
   - **不需要CA签名**（CA功能关闭或禁用）：直接完成医生开方，推进到处方创建完成状态
7. **发送通知**：处方创建完成事件→通过IM消息推送处方给患者、发送订阅消息

### 第三步：药师审核处方

1. **状态校验**：必须是"医生已开方 + 药师待审核"状态才能审核
2. **时效校验**：若处方已超过失效时间，提示"处方单超时未审核已失效"
3. **药师身份校验**：获取当前用户的药师信息，校验"非药师身份或者当前药师不可用, 不能审核处方"
4. **药师专业匹配**：西药师(01)审核西药处方，中药师(02)审核中药处方/含中成药的成药处方。不匹配提示"您登记的药师专业不能审核当前类型的处方"
5. **审核通过**：药师签章→处方状态变为"已审核"→推送审核通过通知
6. **审核驳回**：记录驳回原因→推送驳回通知（PrescriptionDoctorAuditRejectEvent）

### 第四步：处方变更

医生修改已提交但未审核的处方：

1. **状态校验**：只能变更特定状态的处方（通过 ChangePrescriptionValidator 校验）
2. **患者不可变更**：处方变更时不改变患者信息
3. **重新设置失效时间**：变更后重新计算失效时间
4. **标记变更**：处方标记为"已变更"（isChange=true），推送变更事件
5. **若无病历则补充创建**：零售机构（药店）的处方变更时，若之前因病历签章问题未创建病历，此时补建
6. **已有病历则补充诊断**：在现有病历上追加新的诊断信息

### 第五步：处方撤回

医生或管理员撤回已提交的处方：

1. **已存在订单保护**：若处方已生成有效订单，提示"撤回失败，该处方已存在订单，请联系管理员取消订单后重新开方"
2. **撤销签章记录**：撤回医生签署记录
3. **处方状态更新**：处方状态变更为"已撤回"
4. **发布撤回事件**：通知相关系统处理

### 第六步：处方删除

1. **限制删除状态**：仅允许"草稿"和"已撤回"状态的处方删除
2. **医生权限校验**：只能删除自己开的处方（"您无权限删除处方"）

### 第七步：处方过期

定时任务或系统触发：

1. 将医生审核状态设为"已过期"（DoctorApproval.Expired）
2. 将药师审核状态设为"已过期"（PharmacistApproval.Expired）
3. 撤销医生签署记录
4. 发布处方过期事件（PrescriptionOverdueEvent）

### 第八步：患者查看处方

患者查看处方后：
1. 记录查看标记（viewFlag=true）
2. 推送"患者已查看处方"事件

### 第九步：诊断重置

系统功能：当需要重新生成处方时，可重置诊断信息：
1. 更新诊断和病情描述
2. 删除当前处方签章→重新提交医生签章→完成医生开方→重新提交药师签章→完成药师审方
3. 全程禁用CA（disableCa=true），不弹出签章页面

## 状态流转图

```
草稿(Pending) → 提交 → 审核中(Processing) → 医生签署完成(Pass) → 药师审核中(Processing) → 审核通过(Pass) / 审核驳回(Reject)
                       ↓                              ↓
                    可撤回                         处方过期(Expired)
                       ↓
                  已撤回(Revoked) → 可删除

医生驳回(仅零售机构草稿状态): Pending → DoctorApprovalReject
```

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 处方不能审核（状态不对） | 提示"当前处方不能被审核" |
| 处方超时未审核 | 提示"处方单超时未审核已失效" |
| 非药师身份审核 | 提示"非药师身份或者当前药师不可用, 不能审核处方" |
| 药师专业不匹配 | 提示"您登记的药师专业不能审核当前类型的处方" |
| 处方已存在有效订单（撤回时） | 提示"撤回失败，该处方已存在订单，请联系管理员取消订单后重新开方" |
| 非草稿/撤回状态删除 | 提示"非草稿、已撤回状态的处方不允许删除" |
| 非本人处方删除 | 提示"您无权限删除处方" |
| 病历签章未完成（开处方时） | 提示"系统配置生成病历需要签名, 请前往写病历页面操作完成后再开处方" |
| 找不到医生签章记录（药师审核时） | 提示"未找到医生签章记录或者签章未完成" |
| 处方已过期（审核时） | 提示"处方单超时未审核已失效" |
| 草稿状态医生驳回（仅零售机构） | 仅零售机构且草稿(Pending)状态允许，其他提示"该处方不允许医生驳回操作" |
| 患者信息不存在 | 提示"未找到处方患者信息, 无法审核处方, 请联系管理员" |

## 关联功能

- 病历管理（处方创建/变更时同步病历）
- 电子签名（CA签章 vs 普通签名）
- 问诊流程（从问诊室开具处方）
- 处方结算（审核通过后进入结算）
- 药师管理（药师专业与处方类型匹配）
- 订单管理（撤回前检查已有订单）
