# 服务预约生命周期管理流程

> 文档版本：2026-07-02 | 面向读者：产品经理、测试人员

## 概述

服务预约全生命周期管理，覆盖患者从小程序预约健康服务项目到核销完成的完整流程。包括预约单创建、订单生成、支付、取消和核销等环节。涉及服务模块 `ykj-service-reservation`，核心类是 `ServiceReservationsServiceImpl`（826行）。

## 状态流转

```
待支付(0) → [支付成功] → 待核销(4) → [核销] → 已核销(5)
   ↓
已取消(1)
```

## 核心环节

### 1. 创建预约订单
- 接口：`POST /api/serviceReservations/generateOrder`
- 患者选择服务项目后提交预约
- 系统验证用户必须已登录（memberId 不为空），否则返回"未授权"错误
- 生成预约单编号，状态为**待支付(0)**

### 2. 查看预约列表
- 接口：`GET /api/serviceReservations`
- 患者可查看自己的预约单分页列表
- 支持按状态筛选：待支付(0)、已取消(1)、待核销(4)、已核销(5)

### 3. 查看预约详情
- 接口：`GET /api/serviceReservations/{reservationNo}`
- 根据预约单编号查看详情，必须验证预约属于当前登录患者

### 4. 取消预约
- 接口：`PUT /api/serviceReservations/cancelOrder`
- 患者可以主动取消预约单
- 必须填写取消原因（cancelDesc）
- 状态从待支付(0)变为已取消(1)

### 5. 支付与核销
- 支付成功后状态变为**待核销(4)**
- 患者到院后出示预约信息，由工作人员核销
- 核销后状态变为**已核销(5)**

## 异常处理

| 情况 | 系统处理方式 |
|------|------------|
| 生成订单时未登录 | 返回"未授权"错误（后端拦截） |
| 查看详情但预约不属于当前用户 | 后端校验 memberId，不返回他人数据 |
| 取消预约未填原因 | cancelDesc 由前端控制必填 |

## 关联功能

- `ykj-service-reservation` 模块：服务预约核心逻辑
- 后台管理 MgmtServiceReservationsController：运营人员管理预约单
- 健康服务包：服务包订单也走预约核销流程
- 支付模块 (`ykj-service-order`)：预约订单的支付处理
