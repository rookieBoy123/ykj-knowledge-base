# 医生排班管理与号源控制流程

> 文档版本：v1.0 | 面向读者：运营、客服、产品

## 概述

管理后台为医生设置每周排班模板，系统自动同步生成每日排班和号源。患者预约时系统实时扣减号源，支持停诊、号源发布、预约上限控制等完整的班次管理能力。

## 触发条件

- **前置条件**：管理后台已配置排班模板
- **入口**：管理后台 → 医生管理 → 排班设置

## 步骤详解

### 第一步：保存医生排班

管理员为指定医生保存某一天的排班：

- **必填校验**：必须指定医生和班次，否则提示"请指定要保存排班的医生"
- **班次时间冲突检测**：校验新设置的班次时间是否与已有班次重叠，防止同一医生同一时段多个班次
- **当前时间冲突检测**：检查是否与当前正在进行的班次冲突（正在进行的不允许编辑）
- **自动分配班次编号**：系统按顺序为每个班次分配唯一编号
- **班次详情保存**：每个班次包含多个号段（如上午/下午），每个号段有独立的时间段
- **旧数据清理**：保存新班次后自动删除旧的班次和号段

### 第二步：号源发布

管理员可以将排班从"未发布"状态批量改为"已发布"：

- **发布条件**：查找所有「未发布」且「已启用」的排班
- **无待发布提示**：如果没有待发布的排班，提示"当前无待发布排班"
- **批量操作**：一键发布所有待发布排班

### 第三步：停诊处理

医生因故不能出诊时，管理员可以进行停诊操作：

- **停诊范围**：支持全天停诊（停止当天所有班次）或指定班次停诊
- **参数校验**：必须指定医生、日期；指定班次停诊时必须指定班次编号
- **影响范围**：停诊会同时标记班次和对应的所有号段
- **停诊原因**：必须填写停诊原因，原因长度不超过100字
- **事件通知**：停诊成功后系统发布「医生停诊」事件，通知已预约该医生号源的患者

### 第四步：号源使用与释放

患者预约时系统实时操作号源：

- **号源扣减（预约）**：
  - 检查班次是否已发布、已启用、未停诊
  - 号源结束时间 + 30分钟 > 当前时间，才允许预约（不能超过号段结束前）
  - 扣减前二次校验号源是否已用完（防止并发）
  - 扣减成功后：号段已用数+1，班次已用数+1
- **号源释放（取消）**：
  - 取消预约时：号段已用数-1，班次已用数-1，班次已退数+1

### 第五步：排班自动同步

系统定时任务自动为医生生成15天后的排班：

- **同步时机**：每天定时执行
- **医生范围**：仅同步已配置了排班模板的医生（`findSyncScheduleDoctor`）
- **跳过条件**：如果15天后该医生已有排班，跳过不处理
- **模板应用**：根据医生绑定的排班模板，自动生成当天的班次和号段

### 第六步：排班查询

- **管理后台查询**：按医生姓名、科室筛选，分页展示医生列表+排班详情
- **小程序端查询（患者）**：仅返回已发布、未停诊、已启用的班次（`isRelease=true & isStopReception=false & isEnabled=true`），未发布或已停诊的班次对患者不可见

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 未指定医生 | 提示"请指定要保存排班的医生" |
| 班次为空 | 提示"请指定要保存排班的医生" |
| 班次详情保存错误 | 提示"班次详情保存错误" |
| 班次保存错误 | 提示"班次保存错误" |
| 预约信息变更 | 提示"预约信息发生变更，请刷新后重试" |
| 预约超过号段结束时间 | 提示"预约不能超过号段结束时间前30分钟" |
| 号源已用完 | 提示"班次信息发生变更，请刷新后重试" |
| 停诊时班次不存在 | 提示"班次不存在" |
| 停诊失败 | 提示"停诊失败，请联系管理员" |

## 关联功能

- **服务预约流程**：患者预约时使用排班号源
- **预约问诊流程**：医生排班用于预约问诊的时间选择
- **管理后台排班管理**：排班模板的创建和管理
