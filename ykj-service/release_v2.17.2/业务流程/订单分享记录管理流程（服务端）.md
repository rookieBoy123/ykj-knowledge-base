# 订单分享记录管理流程（服务端）

> 文档版本：2026-07-04 | 面向读者：产品经理、后端开发
> 关联模块：`ykj-service-order`

## 概述

患者可以将待支付的订单分享给其他会员（如家人、朋友），由接收方代为支付。系统记录每一次分享行为，包括分享人、订单信息、以及所有接收过该分享的会员列表。

## 参与角色

| 角色 | 操作 | 限制 |
|------|------|------|
| 患者（分享人） | 创建订单分享记录 | 只能分享自己的订单；订单必须是待支付状态 |
| 接收人 | 接收分享并代付 | 被添加到分享记录的接收人列表中 |

## 流程详解

### 1. 创建分享记录（doAdd）

**接口**：`POST /api/order/share-record`（新增）

**前置条件校验**：
1. **订单必须存在**：根据订单编号查询订单，找不到则返回"找不到对应的订单信息"
2. **订单归属校验**：当前登录会员的ID必须等于订单的会员ID，不是自己的订单则返回"不属于当前用户的订单不允许分享"
3. **订单状态校验**：订单必须处于"待支付"状态（`PENDING_PAYMENT`），非待支付状态返回"订单非代付状态不允许分享"
4. **防重复创建**：如果该订单已经有分享记录（按创建时间取最新一条），直接返回已有记录，不重复创建

**校验通过后**：
- 创建分享记录，记录分享人的会员ID、订单编号
- 同时记录订单的来源类型（`sourceType`）和业务编号（`businessNo`）

### 2. 接收分享（doUpdate）

**接口**：`PUT /api/order/share-record/{id}`（更新）

**前置条件校验**：
1. **分享记录必须存在**：找不到返回"找不到对应的订单分享记录"
2. **去重判断**：接收人会员ID如果已经等于分享人ID，或者已经在接收人列表中，则不做任何操作直接返回
3. **订单状态再次校验**：订单仍必须为待支付状态，非待支付返回"订单非待付状态"

**校验通过后**：
- 将接收人的会员ID追加到接收人列表中（逗号分隔，自动去重）
- 同一订单可以有多个接收人

### 3. 数据模型

分享记录包含以下关键信息：
- **分享人会员ID**：谁发起的分享
- **订单编号**：被分享的订单
- **接收人会员ID列表**：所有接收过该分享的会员ID（逗号分隔，如"member001,member002"）
- **来源类型**：订单的来源类型（如小程序、管理后台等）
- **业务编号**：关联的业务编号

## 状态约束

```
订单状态   →  是否允许分享
待支付      →  ✅ 允许
已支付      →  ❌ 不允许（提示"订单非代付状态不允许分享"）
已取消      →  ❌ 不允许
已退款      →  ❌ 不允许
已完成      →  ❌ 不允许
```

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 分享的订单不存在 | 返回"找不到对应的订单信息" |
| 分享别人的订单 | 返回"不属于当前用户的订单不允许分享" |
| 订单不是待支付状态 | 返回"订单非代付状态不允许分享" |
| 同一订单重复分享 | 返回已有分享记录（幂等，不重复创建） |
| 接收人已接收过该分享 | 不做任何操作，直接返回（去重） |
| 更新不存在的分享记录 | 返回"找不到对应的订单分享记录" |
| 更新时订单状态已变化 | 返回"订单非待付状态" |

## 关联功能

- **订单创建与支付**：分享功能依赖订单状态为待支付
- **代付支付流程**：接收人通过分享记录关联的订单进行代付
- **会员体系**：分享人和接收人都需要是系统会员
