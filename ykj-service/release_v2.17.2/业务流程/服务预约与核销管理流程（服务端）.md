# 服务预约与核销管理流程（服务端）

> 文档版本 1.0 | 面向非技术读者 | 2026-07-02

## 概述

服务预约模块处理患者在平台上预约线下医疗服务的完整后端逻辑。患者在线选择服务项目、填写联系人信息、选择预约时间后创建预约单并支付，到院后通过核销码完成服务核销。核销时支持自动开具电子发票。

## 触发条件

- 患者在前端选择医疗服务项目并提交预约
- 患者到院出示核销码完成服务核销
- 后台管理人员查询和管理预约记录

## 核心流程

### 一、预约单查询

支持多维度筛选预约记录：
- **预约单号**：精确查询
- **联系人手机号**：加密存储，查询时自动加密后匹配
- **项目名称**：模糊搜索
- **服务方式**：筛选线上/线下服务
- **人员类型**：个人/企业
- **项目类型**：筛选服务类别
- **服务状态**：待核销/已核销/已取消/已退款等
- **预约时间**：按日期范围筛选

结果按创建时间倒序排列。

### 二、核销流程

核销是服务预约的核心环节，患者到院后出示8位核销码完成服务确认：

**1. 核销码查询**：
- 输入核销码查找预约记录
- 核销码不存在时提示"未查询到预约记录，核销码无效"
- 预约已核销时提示"此核销码已核销"

**2. 核销校验**：
- 预约记录必须存在
- 关联订单必须存在
- 预约状态必须是"待核销"
- 核销码必须匹配（不区分大小写）

**3. 核销执行**：
- 更新预约状态为"已核销"
- 记录核销时间
- 更新关联订单为"已完成"

**4. 自动开票**（条件触发）：
如果系统开启了自动开票功能（autoInvoice=true）且订单金额不为0：
- 自动推送开票事件
- 生成电子发票信息
- 包含票据业务标识、证件类型、卡类型、医疗类别等信息

### 三、预约详情查询

通过预约ID查询完整的预约信息，包括：
- **预约基本信息**：预约单号、预约时间、联系人信息
- **订单信息**：订单金额、支付状态、费用明细
- **服务项目信息**：项目名称、包含的服务内容
- **患者信息**：从患者服务查询就诊人详情
- **服务详情**：服务的具体描述和说明

### 四、患者信息补充

核销时可以补充患者信息（如HIS患者信息），用于关联线下就诊记录和电子发票开具。

## 预约状态流转

```
待核销(UNVERIFIED) ──核销成功──→ 已核销(VERIFIED)
       │
       ├──取消──→ 已取消(CANCEL)
       │
       ├──退款申请──→ 退款中(REFUND)
       │
       └──退款完成──→ 已退款(REFUNDED)
```

## 异常处理

| 情况 | 系统处理 | 提示信息 |
|------|---------|---------|
| 核销码不存在 | 拒绝操作 | "未查询到预约记录，核销码无效！" |
| 核销码已使用 | 拒绝操作 | "此核销码已核销" |
| 预约非待核销状态 | 拒绝操作 | "该订单不是待核销状态订单" |
| 核销码不匹配 | 拒绝操作 | "错误的核销码" |
| 关联订单不存在 | 拒绝操作 | 返回错误 |
| 预约记录不存在 | 拒绝操作 | 返回错误 |

## 关联模块

- **订单模块**（ykj-service-order）：创建预约订单、完成订单
- **患者模块**（ykj-service-hospital）：查询患者信息
- **HIS模块**（ykj-service-his）：同步线下患者信息
- **电子发票模块**：自动开具发票
- **会员模块**（ykj-service-member）：查询会员信息
- **消息通知模块**（ykj-service-system）：发送预约状态变更通知
