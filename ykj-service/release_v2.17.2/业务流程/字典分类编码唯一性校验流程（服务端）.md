# 字典分类编码唯一性校验流程（服务端）

> 文档版本 2026-07-04 | 面向开发/技术支持

## 概述
字典管理系统的核心约束是"分类编码 + 标准编码不能重复"。在管理后台新增或编辑字典分类、字典内容时，服务端提供独立的校验接口，前端在提交前调用校验，确保数据的唯一性。本文档详细说明这两个校验接口的工作机制。

> **与现有文档的关系**：本文档聚焦于数据校验机制。字典的基础 CRUD 操作可参考已有文档「数据字典分类与内容管理流程（服务端）.md」。

## 校验接口一：分类唯一性检查

### 接口信息
- 路径：`POST /mgmt/dictionary/_check-exists`
- 权限：`sys:dict:read`
- 入参：`categoryCode`（分类编码）、`standardCode`（标准编码），编辑时额外带 `id`

### 校验逻辑
1. 根据传入的 `categoryCode` 和 `standardCode`，查询数据库中是否已存在匹配的分类
2. 编辑场景下（传入 `id`）：排除自身后检查是否存在重复
3. 返回 `true`（已存在）或 `false`（不存在）

### 前端集成
在管理后台的字典集管理页面中：
- 新增分类时：填写完成后点击保存，前端先调用此接口检查唯一性
- 编辑分类时：仅当名称被修改时才调用校验（通过对比 `oldValidName` 与当前值判断）
- 如果校验返回"已存在"，前端提示"该编码及标准编码已存在目录中，请重新输入"，阻止提交

## 校验接口二：字典内容编码唯一性检查

### 接口信息
- 路径：`POST /mgmt/dictionary/content/_check-exists`
- 权限：`sys:dict:read`
- 入参：`categoryId`（所属分类ID）、`contentCode`（内容编码），编辑时额外带 `id`

### 校验逻辑
1. 根据传入的 `categoryId` 和 `contentCode`，查询该分类下是否已存在相同编码的内容
2. 编辑场景下：排除自身后检查
3. 返回 `true`（已存在）或 `false`（不存在）

### 前端集成
- 新增字典内容时：填写编码后点击保存，前端先调用此接口
- 编辑内容时：仅当编码被修改时才调用
- 如果校验返回"已存在"，前端提示"编码已存在，请重新输入"，阻止提交

## 分类接口汇总

| 接口 | 方法 | 路径 | 权限 | 说明 |
|------|------|------|------|------|
| 查询字典树 | GET | `/mgmt/dictionary/tree` | `sys:dict:read` | 返回树形结构的分类列表 |
| 查询分类内容 | GET | `/mgmt/dictionary/content` | 无权限限制 | 按分类ID查询字典内容（支持按内容编码筛选） |
| 校验分类存在 | POST | `/mgmt/dictionary/_check-exists` | `sys:dict:read` | 检查分类编码+标准编码是否重复 |
| 校验内容存在 | POST | `/mgmt/dictionary/content/_check-exists` | `sys:dict:read` | 检查同分类下内容编码是否重复 |
| 新增分类 | POST | `/mgmt/dictionary` | `sys:dict:write` | 创建新的字典分类 |
| 编辑分类 | PUT | `/mgmt/dictionary/{id}` | `sys:dict:write` | 更新分类信息 |
| 新增内容 | POST | `/mgmt/dictionary/content` | `sys:dict:read` | 在某分类下新增字典项（注意：写入操作但权限用 read） |
| 编辑内容 | PUT | `/mgmt/dictionary/content/{id}` | `sys:dict:write` | 更新字典项 |
| 删除内容 | DELETE | `/mgmt/dictionary/content/{id}` | `sys:dict:write` | 删除字典项 |
| 按编码查内容 | GET | `/mgmt/dictionary/content?categoryCode=` | 无权限限制 | 按分类编码查询字典内容列表（供前端下拉框使用） |

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 分类编码+标准编码重复 | `checkCategoryExists` 返回 `true`，前端提示"已存在" |
| 同分类下内容编码重复 | `checkContentExists` 返回 `true`，前端提示"编码已存在" |
| 查询的字典树为空 | 返回空 `HttpResult`（data 为 null），不报错 |
| 编辑时未修改名称 | 前端跳过校验，直接提交 |
| 新增内容的权限标注为 `sys:dict:read` | 这是代码中的权限定义，只要有读取权限即可新增内容 |

## 关联功能
- **管理后台 - 字典集管理**：前端调用这些接口实现完整的字典管理
- **其他模块的下拉选项**：通过 `GET /mgmt/dictionary/content?categoryCode=` 接口读取字典数据
- **系统分类树形管理**：字典分类树的查询接口也被其他模块复用（如科室诊断对照中的科室列表）
