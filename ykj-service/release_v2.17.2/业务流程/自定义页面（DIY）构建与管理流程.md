# 自定义页面（DIY）构建与管理流程

> 文档版本：2026-07-03 | 面向读者：产品、运营、管理员

## 概述

自定义页面（DIY）是平台提供的小程序页面可视化搭建工具。管理员可以在后台创建和编辑自定义页面（拖拽组件、配置样式），设置其中一个为"首页"或"医带患首页"，小程序端即可展示对应的自定义页面。每个自定义页面还支持生成预览二维码，方便在发布前扫码预览效果。

## 触发条件

- 管理员需要定制小程序首页或特色页面
- 运营人员需要搭建活动落地页
- 需要在正式发布前预览自定义页面效果

## 页面类型

自定义页面有三种类型：

| 类型 | 含义 | 说明 |
|------|------|------|
| 普通页面 | 待使用的页面 | 新创建的页面默认为普通页面；同类型只能有一个启用页面，其他自动降级为普通页面 |
| 首页 | 小程序用户端首页 | 设置后小程序用户看到的首页即为此页面 |
| 医带患首页 | 医生带患者模式首页 | 医生端特定场景下的首页 |

**互斥规则**：同一类型（首页/医带患首页）同时只能有一个页面处于使用状态。当把一个页面设为某类型时，之前该类型的页面会自动降为"普通页面"。

## 步骤详解

### 第一步：创建自定义页面

管理员在后台填写页面基本信息：
- **页面名称**：用于后台识别和管理
- **是否显示**：控制该页面在列表中是否可见
- **标题、标题颜色、标题背景颜色**：页面顶部标题区样式
- **顶部颜色、背景颜色**：页面全局配色
- **是否使用背景图片**：可选择上传背景图
- **是否显示返回按钮**：控制导航栏返回按钮
- **页面内容（value）**：JSON 格式的组件布局数据（由可视化编辑器生成）

创建时系统自动将页面类型设为"普通页面"，同时自动生成预览二维码。

### 第二步：编辑页面

修改页面的样式配置和组件布局。编辑时有以下限制：
- **内置页面不能编辑**（系统预制页面，标记为 builtIn=true）
- 编辑不会改变页面当前的使用状态（保持在"首页"/"医带患首页"/"普通页面"）
- 更新时自动处理素材域名替换

### 第三步：启用页面（设为首页）

选择一个"普通页面"，将其启用为"首页"或"医带患首页"。启用时：
1. 系统先将同类型的所有其他页面降级为"普通页面"
2. 再将当前页面类型改为目标类型
3. 小程序端通过 API 查询对应类型的页面即可展示

### 第四步：生成预览二维码

每个自定义页面保存时自动生成预览二维码（微信小程序码），扫描后可在手机端预览页面效果。如果二维码失效或需要更新，可点击"重置预览二维码"重新生成。

### 第五步：删除页面

删除时有保护规则：
- **内置页面不能删除**
- **正在使用中的页面**（类型为首页或医带患首页）**不能删除**，需先将其降级为普通页面

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 保存页面失败 | 提示"保存失败" |
| 编辑不存在的页面 | 提示"页面不存在" |
| 编辑内置页面 | 提示"内置页面不能删除"（系统会阻止编辑内置页面） |
| 删除内置页面 | 提示"内置页面不能删除" |
| 删除正在使用的页面 | 提示"页面正在使用中，无法删除" |
| 启用时页面不存在 | 提示"页面不存在" |
| 启用操作失败 | 提示"页面设置失败，请联系管理员" |
| 重置二维码时页面不存在 | 提示"页面不存在" |
| 重置二维码生成失败 | 提示"重置预览二维码失败" |
| 小程序端查询时类型为普通页面 | 提示"参数错误，请指定要查询的页面类型" |

## 小程序端展示

小程序端通过 API `/api/diy/page` 获取指定类型的自定义页面：
- 传 `type=1` 获取首页
- 传 `type=2` 获取医带患首页
- 传 `id` 获取指定页面详情（用于预览）

查询时系统会自动将页面内容中的初始化素材域名替换为实际的 OSS 域名，确保图片正常显示。

## 关联功能

- **底部分组配置**：自定义底部分组（底部导航栏）搭配自定义页面使用
- **素材管理**：页面中使用的图片从素材库中选取
- **小程序预览**：通过预览二维码在手机端查看效果
