# 外部机构对接问诊流程

> 文档版本：v1.0 | 面向读者：运营、产品、技术对接

## 概述

第三方医疗或体检机构通过开放平台对接本系统的问诊服务。机构可以发起图文问诊、上传患者资料、创建病历、生成订单、回填聊天记录，实现院外问诊数据的完整闭环。

## 触发条件

- **前置条件**：第三方机构已申请并获得了对接凭证（AppId）
- **入口**：机构通过API接口调用（非小程序用户直接操作）

## 步骤详解

### 第一步：发起问诊

机构通过接口发起一个问诊请求：

- **机构身份校验**：根据AppId查找机构信息，找不到则提示"找不到对应的机构信息"
- **问诊方式**：支持普通问诊和预约问诊两种方式。预约问诊必须提供预约开始和结束时间
- **患者信息必填**：患者姓名、性别、年龄为必填项，缺少任一则提示"参数缺失，请正确填写患者信息"
- **病史信息选填**：过敏史、既往史、其他病史为选填项
- **病历图片处理**：如果上传了病历图片，系统将其转存到平台存储中
- **扩展信息**：支持传递来源标识、额外扩展参数、是否禁用聊天室等

### 第二步：完整上传创建

机构需要上传完整的患者和问诊数据时，执行以下完整流程：

- **注册会员**：根据患者手机号注册或查找平台会员
- **保存患者信息**：创建或更新患者档案（默认关系为"其他"）
- **创建问诊单**：
  - 根据机构名称匹配医生（优先精确匹配，否则模糊匹配）
  - 生成问诊单号，设置服务类型为「图文问诊」
  - 问诊状态设为「进行中」
  - 生成IM群组ID，创建医生和患者的IM聊天群
- **创建病历**：根据上传的诊断信息生成病历记录
- **创建订单**：生成平台订单，记录支付金额和支付时间
- **回填聊天记录**：生成医生接诊消息和问诊内容
- **自动结束问诊**：完成数据上传后自动结束问诊，触发问诊结束事件

### 第三步：取消问诊

机构可以取消通过其发起的问诊：

- **机构权限校验**：确认问诊属于该机构，否则提示"问诊不属于请求机构，无权限取消"
- **状态限制**：仅「待接诊」和「接诊中」状态的问诊可以取消。其他状态提示"该问诊状态不允许取消"
- **退款控制**：支持选择是否退款（`noRefund`参数），以及是否推送通知

### 第四步：更新问诊信息

机构可以修改已创建的问诊数据：

- **修改病情描述**：更新问诊单和病历中的病情描述
- **补充病历图片**：上传新的病历图片，系统异步转存到平台存储
- **数据同步**：问诊单和病历同步更新，保持一致

### 第五步：查询问诊视频记录

机构可以查询某次问诊的视频通话记录：

- **权限校验**：确认问诊属于该机构
- **数据返回**：返回该问诊的所有视频回放地址列表

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 机构不存在 | 提示"找不到对应的机构信息" |
| 预约问诊缺少预约时间 | 提示"参数缺失，预约问诊需要提供预约开始、结束时间" |
| 患者信息不完整 | 提示"参数缺失，请正确填写患者信息" |
| 取消不属于本机构的问诊 | 提示"问诊不属于请求机构，无权限取消" |
| 取消非允许状态的问诊 | 提示"该问诊状态不允许取消" |
| 生成问诊单失败 | 提示"生成问诊单失败" |
| 获取医生IM账号失败 | 提示"获取医生im账号信息失败" |
| 获取患者IM账号失败 | 提示"获取患者im账号信息失败" |
| 修改时找不到问诊信息 | 提示"找不到对应的问诊信息" |
| 修改时找不到病历信息 | 提示"找不到对应的病历信息" |
