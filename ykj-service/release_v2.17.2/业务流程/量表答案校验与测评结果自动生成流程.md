# 量表答案校验与测评结果自动生成流程

> 文档版本 2026-07-02 | 面向读者：了解后端业务逻辑

## 概述

当患者或医生在小程序端完成量表测评答题时，每道题的答案提交到服务端进行校验和存储。当答完最后一题时，服务端自动计算总分和各维度得分，生成测评结果。本流程涵盖答案保存、答案修改、结果生成的完整后端逻辑。

## 触发条件

- **前置条件**：测评记录已创建（recordId），量表已存在（scaleId），题目已配置
- **入口**：小程序调用 `/api/scale/answer` 接口（POST 新增答案 / PUT 修改答案）

## 步骤详解

### 第一步：答案提交校验（新增/修改）

每次提交答案时，服务端执行三重校验：

**校验一：题目存在性**
- 根据 questionId 查找对应题目
- 如果找不到题目 → 抛出异常"找不到对应的量表题目！"

**校验二：题型与答案数量匹配**
- 单选题（type=0）：答案不能超过1个 → 否则抛出"单选题不可以有多个答案！"
- 多选题（type=1）：允许多个答案

**校验三：答案有效性**
- 提交的答案必须在题目的选项编号集合中
- 存在不在选项中的答案 → 抛出"该答案不存在于问题选项中！"

**校验四：测评完成保护**
- 查询是否已有该 recordId 的测评结果
- 已存在结果 → 抛出"已测评完成，不能再次修改答案！"

### 第二步：答案保存

校验通过后，将答案写入数据库：
- 新增模式（INSERT）：首次作答，记录答题内容、题目ID、测评记录ID
- 修改模式（UPDATE）：修改已提交的答案，按答案记录ID更新

保存失败 → 抛出内部错误。

### 第三步：最后一题判断与结果生成

保存成功后，判断是否为最后一题：
- 调用 `scaleQuestionService.judgeLastOne(questionId)` 判断
- 如果是最后一题 → 触发结果自动生成

**结果生成逻辑**：

1. **查询所有题目**：根据 scaleId 查询该量表的所有题目
2. **构建题目选项映射**：每个题目的选项解析为 `选项编号 → 分值` 的映射
3. **构建题目维度映射**：有维度信息的题目，建立 `题目ID → 维度名称` 的映射（如"躯体化"、"焦虑"）
4. **查询所有已提交答案**：根据 recordId 查询所有答题记录
5. **计算总分**：遍历所有答案，每题得分 = 选中选项的分值之和（多选累加），总分 = 所有题目得分之和
6. **计算维度分**：有维度信息的题目，累计到对应维度下，维度得分 = 该维度所有题目得分之和
7. **生成结果记录**：保存 totalScore（总分）、dimensionScores（维度得分JSON）、scaleTotalScore（量表总分），并关联 recordId 和 scaleId

### 第四步：结果返回给前端

生成的测评结果包含：
- 总分（totalScore）
- 量表总分（scaleTotalScore）
- 维度得分（dimensionScores，JSON格式，如 `{"躯体化": 15.5, "焦虑": 20.0, "抑郁": 18.0}`）
- 结果解读（content/illustrate）
- 建议等级等内容

前端根据角色不同展示不同颜色主题（患者端使用系统主题色，医生端使用蓝色#3B8FFF）。

## 状态流转

```
答题中 → （答完最后一题）→ 结果已生成 [不可再修改]
```

- 结果生成前：可以新增/修改/删除答案
- 结果生成后：所有修改操作拦截（"已测评完成，不能再次修改答案！"）

## 异常处理

| 情况 | 系统处理方式 |
|------|------------|
| 题目不存在 | 抛出异常"找不到对应的量表题目！"，前端提示错误 |
| 单选题答案超过1个 | 抛出异常"单选题不可以有多个答案！"，前端提示错误 |
| 答案不在选项范围内 | 抛出异常"该答案不存在于问题选项中！"，前端提示错误 |
| 测评已完成再次修改 | 抛出异常"已测评完成，不能再次修改答案！"，前端提示错误 |
| 保存答案数据库失败 | 抛出内部错误"保存量表答案操作失败！" |
| 修改答案数据库失败 | 抛出业务异常"修改量表答案操作失败！" |
| 查询答案时questionId为空 | 抛出异常"量表题目id不能为空！" |
| 查询答案时recordId为空 | 抛出异常"量表测评记录id不能为空！" |

## 关联功能

- 关联**量表管理**：量表题目和选项的配置在管理后台维护
- 关联**测评记录管理**：每次测评产生一条新记录（recordId）
- 关联**维度配置**：题目维度信息用于雷达图展示（需3个及以上维度）
- 关联**问诊关联**：测评结果可关联问诊，在结果页展示问诊医生和对话记录
