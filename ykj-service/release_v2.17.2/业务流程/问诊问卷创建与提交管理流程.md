# 问诊问卷创建与提交管理流程

> 文档版本 1.0 | 面向读者：产品、运营

## 概述

医生在问诊过程中向患者发送问卷，患者填写后提交，系统记录答案并通知医生。问卷与问诊单绑定，支持私有问卷权限控制。

## 触发条件

- 医生在问诊聊天室中点击"发送问卷"
- 或系统自动在问诊流程中推送问卷

## 核心流程

### 1. 创建问卷（从模板复制）

医生选择问卷模板后，系统创建一个与当前问诊单绑定的问卷实例：

**私有问卷权限校验**：
- 如果问卷是私有的（isPrivate=true），只能由创建该问卷的医生发送
- 非创建医生发送私有问卷→拒绝"该问卷不能被当前医生创建"

**创建数据**：
1. 创建问卷主记录（InquirySurvey），绑定问诊编号（inquiryNo）、医生ID，状态设为"未完成"
2. 复制问卷模板下的所有题目（InquirySurveyQuestion），每题关联问卷编号
3. 复制每题的所有选项（InquirySurveyAnswer），选项关联对应题目编号

**发送到聊天室**：
创建完成后，自动通过IM群消息服务发送问卷卡片到问诊聊天室，患者可在聊天界面中看到并点击填写。

### 2. 患者提交问卷

患者填写完问卷后提交：

**校验**：
- 提交的答案列表不能为空
- 问卷必须存在且处于"未完成"状态

**处理步骤**：
1. 更新问卷状态为"已完成"，记录提交时间
2. 找到问卷下所有题目和对应答案选项
3. 将患者提交的答案写入（选中/填写）
4. 更新答案数据
5. 发送提交完成消息通知医生

### 3. 查询问卷

- **按问卷ID查询**（findById）：返回完整问卷（含题目+选项+医生姓名）
- **按问诊编号查询**（findByInquiryNo）：返回该问诊单下所有问卷列表（含题目数量统计）
- 两种查询都会自动组装医生姓名信息

### 4. 推送问卷到问诊中

sendInquirySurvey方法是医生主动推送问卷的完整入口：

**校验**：
- 问卷模板必须存在
- 问诊单必须存在
- 问诊聊天室存在（groupId不能为空）
- 私有问卷权限检查（非创建医生不能发送）

**流程**：
1. 创建问卷实例（调用create）
2. 获取医生在IM中的账号信息
3. 生成问卷卡片消息
4. 通过IM群消息服务异步发送到聊天室

## 异常处理

| 情况 | 后端处理 |
|------|----------|
| 问卷模板不存在 | 抛出"问卷不存在" |
| 问诊单不存在 | 抛出"问诊单不存在" |
| 聊天室异常 | 抛出"问诊单聊天室异常" |
| 私有问卷非创建医生发送 | 抛出"您当前无法发送此问卷" |
| 提交答案为空 | 抛出"需要提交的答案不能为空" |
| 问诊问卷不存在 | 抛出"问诊问卷不存在" |
| 创建/更新数据库失败 | 抛出相应错误并记录日志 |

## 关联模块

- ykj-service-hospital（问卷模板服务 ApiSurveyService）
- ykj-service-im（IM即时通讯模块，群消息发送）
- ykj-service-inquiry（问诊模块，问诊单管理 InquiryAppointService）
- 卡消息组装（问卷卡片的IM消息格式生成）
