# 药品黑名单校验与处方拦截流程（服务端）

> 文档版本 | 面向读者：开发人员 / 产品经理

## 概述
药品黑名单是平台级的安全管控机制。当管理员将某个药品通用名加入黑名单后，医生在开具处方时系统会自动校验——如果处方中包含黑名单药品，将被拦截或标记。该机制同时影响在线开具和Excel导入两种场景。

## 触发条件
- **数据来源**：管理后台药品黑名单管理模块
- **影响范围**：医生端处方开具（小程序/PC端）
- **服务模块**：`ykj-service-medicine`

## 核心流程

### 1. 黑名单数据管理（管理员操作）

**新增黑名单**（`POST /mgmt/medicine/blacklists`）：
- 管理员提交药品通用名
- 系统校验是否已存在同名词条（重复→提示"药品重复"）
- 新增成功后，触发 `BlacklistManager` 更新缓存/索引，确保处方校验实时生效

**编辑黑名单**（`PUT /mgmt/medicine/blacklists/{id}`）：
- 同样校验名称重复（排除自身ID）
- 更新后同步刷新缓存

**删除黑名单**（`DELETE /mgmt/medicine/blacklists/{id}`、`POST /mgmt/medicine/blacklists/_batch_delete`）：
- 支持单条删除和批量删除
- 删除后处方校验不再拦截该药品

### 2. Excel批量导入（管理员操作）

**导入接口**（`POST /mgmt/medicine/blacklists/_import`）：
- 解析Excel文件，提取通用名列
- 自动去重：Excel内部重复和与已有黑名单的重复
- 仅导入新增的药品，已存在的标记为导入失败
- 导入成功后同步刷新 `BlacklistManager`

### 3. 处方药品校验（医生操作触发）

**校验接口**（`GET /mgmt/medicine/blacklists/_check_keyword`）：
- 医生在处方中添加药品时，前端调用此接口
- 传入药品名称，系统在黑名单中进行模糊匹配
- 如果命中黑名单，返回 `true`，前端进行拦截提示

**校验逻辑**（`MedicalBlacklistServiceImpl.checkDrugExist`）：
- 对输入的药品名称先清理空白字符
- 在黑名单中以 `LIKE` 方式模糊匹配
- 命中任何一条即视为"在黑名单中"

### 4. 状态流转图

```
管理员新增 ──→ 重名校验 ──→ 存入数据库 ──→ 刷新BlacklistManager缓存
     │              │
     │          重复→报错"药品重复"
     │
管理员删除 ──→ 从数据库移除 ──→ 不再拦截
     │
Excel导入 ──→ 逐行解析 ──→ 去重对比 ──→ 新数据入库 ──→ 重复数据→标记失败
     │
医生开处方 ──→ 搜索药品 ──→ 调用checkDrugExist ──→ 命中黑名单→拦截提示
                                               ──→ 未命中→正常使用
```

## 异常处理

| 情况 | 系统处理方式 | 异常来源 |
|------|-------------|---------|
| 新增/编辑时药品名称重复 | 返回错误"药品重复" | 后端拦截（BizException） |
| Excel导入数据全部重复 | 所有记录标记为导入失败 | 后端处理 |
| 处方校验命中黑名单 | 返回 true，前端显示拦截提示 | 后端返回结果 |

## 关联功能
- **管理后台药品黑名单管理**：管理员操作入口
- **医生端处方开具**：实时校验入口
- **BlacklistManager**：缓存/索引管理，确保校验性能

## 权限要求
- 查看：`med:blacklists:read`
- 新增/编辑/删除：`med:blacklists:write`
- Excel导入：`med:blacklists:import`
