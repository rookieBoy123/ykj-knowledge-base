# 量表发布校验与问诊中下发流程

> 文档版本：2026-07-02 | 面向读者：医生、运营管理员

## 概述

本流程描述医生如何将量表发布给患者并在问诊对话中发送量表卡片。覆盖量表从草稿到发布的数据校验、题目上一题/下一题关系自动生成、维度/总分评分标准的计算，以及量表通过IM消息下发到患者端的完整链路。

## 触发条件

- **管理员/医生**在后台或小程序中将草稿状态的量表发布
- **医生**在问诊室中选择量表发送给患者
- **管理员**禁用一个量表

## 步骤详解

### 一、量表发布（草稿→已发布）

#### 发布前数据完整性校验

1. **题目校验**：草稿状态的量表必须至少包含一道题目，否则提示"该量表缺少题目信息！"
2. **每道题目的逐项校验**：
   - 使用@InsertValid注解校验每道题目的必填字段
   - 缺失数据时提示"量表题目信息未完善！缺失数据：xxx"
3. **题目业务逻辑校验**：调用verifyScaleQuestion方法，检查题目选项等业务规则。
4. **评分标准校验**：
   - 必须有至少一条评分标准，否则提示"该量表缺少评分标准！"
   - 逐条校验评分标准的必填字段
   - 调用verifyScaleRating方法校验评分标准与量表的匹配关系

#### 发布时自动处理

5. **上一题/下一题关系**：系统按题目顺序自动设置每道题的previousId和nextId，构成题目的单向链表结构。第一题无上一题，最后一题无下一题。
6. **题目总分计算**：
   - 单选题：取所有选项中分值（value）最大的那个作为该题的最高分
   - 多选题：取所有选项分值之和作为该题的最高分
   - 所有题目最高分累加 → 量表总分（totalScore）
7. **维度评分计算**（仅维度评分类型）：
   - 按维度分组：相同维度的题目最高分累加 → 该维度的总最高分
   - 以JSON格式保存维度分数映射（dimensionScores）
8. **状态变更**：所有校验通过后，量表状态从「草稿」变更为「已发布」。
9. **发布幂等性**：如果量表已经是「已发布」状态，重复调用发布方法不会报错，直接返回。

### 二、量表禁用

1. **前提条件**：只能禁用「已发布」状态的量表，对草稿状态执行禁用提示"该量表不是已发布状态，不可禁用!"。
2. **状态变更为禁用**：设置为DISABLED_STATUS。
3. **已发布的测评记录**：禁用不影响已经发送给患者的测评记录，患者仍可完成已有的测评。

### 三、量表编辑（仅草稿状态）

1. **状态保护**：只有草稿状态的量表才能编辑基础信息，非草稿状态提示"非草稿状态的量表信息不能编辑！"。
2. **评分类型切换**：如果从「维度评分」切换为「总分评分」，系统自动删除所有题目的维度字段（dimension）和所有评分标准的维度信息。这是数据清理操作，防止维度类型残留数据。

### 四、量表发送给患者（问诊中）

1. **量表存在性检查**：先确认量表是否存在，不存在提示"找不到对应的量表信息！"
2. **创建测评记录**：保存一条ScaleEvaluateRecord，记录量表ID、患者ID、问诊编号。
3. **获取问诊信息**：通过inquiryNo查找对应的问诊记录，确定IM群组。
4. **发送系统消息**：在IM会话中发送一条系统消息，内容为"医生给您发了一个测评量表，请及时进入测评"。
5. **发送量表卡片**：
   - 卡片图标：量表专用图标（scaleCard.png）
   - 卡片标题："测评量表"
   - 卡片内容：量表名称
   - 操作按钮：「查看量表详情」（蓝色主按钮），携带recordId和scaleId参数跳转
   - 发送方式：异步发送（sendMsgAsync），不阻塞主流程

### 五、测评重新评估

1. **适用场景**：患者在测评中途退出后，医生或患者可以重新进入测评。
2. **限制条件**：已完成的测评（FINISHED状态）不能重新评估，提示"该量表测评已完成！"
3. **操作**：删除已有答案→重置测评时间为当前时间→返回第一道题目重新开始。

## 异常处理

| 情况 | 系统处理方式 |
|------|------------|
| 发布时量表无题目 | 提示"该量表缺少题目信息！" |
| 发布时题目必填字段缺失 | 提示"量表题目信息未完善！缺失数据：xxx" |
| 发布时无评分标准 | 提示"该量表缺少评分标准！" |
| 禁用时量表非已发布状态 | 提示"该量表不是已发布状态，不可禁用!" |
| 编辑时量表非草稿状态 | 提示"非草稿状态的量表信息不能编辑！" |
| 找不到量表信息 | 提示"找不到对应的量表信息！" |
| 找不到问诊信息 | 提示"找不到对应的问诊信息！" |
| 测评已完成仍尝试重评 | 提示"该量表测评已完成！" |
| 量表维度不存在 | 提示"量表不存在该维度！" |

## 关联功能

- **量表题目管理**：量表发布前必须配置题目和选项
- **量表评分标准管理**：支持总分评分和维度评分两种模式
- **问诊系统**：量表在问诊室中通过IM卡片发送给患者
- **患者端量表答题**：患者点击卡片进入答题流程
- **量表测评记录**：每次发送量表都会创建一条测评记录
