# 药房注册与管理流程

> 文档版本 2026-07-02 | 面向产品/开发

## 概述

药房是药品订单履约的核心主体。药房管理模块负责药房的注册、编辑、手机号绑定、服务配置以及关联用户管理。支持线上药房（在线发药）和线下药房（线下取药）两种类型。

## 触发条件

- 运营人员在管理后台新增/编辑药房
- 线下药房人员通过小程序注册/绑定

## 新增药房流程

### 第一步：基本信息校验
- **手机号校验**：手机号不能为空
- **名称唯一性**：药房名称不能与已有药房重复（"药房名称重复"）
- **手机号唯一性**：同一手机号只能绑定一个药房（"该手机号已绑定药房"）

### 第二步：可选第三方药房
- 如果启用第三方药房（enabledThird=true）→ 先创建第三方药房记录 → 记录关联ID

### 第三步：设置排序
- 新药房排在最后：order = 当前最大排序 + 1

### 第四步：插入数据
- 保存药房基本信息到数据库
- 如果失败 → "药房新增失败"

### 第五步：加工服务配置
- 如果药品类型包含饮片（PHARMACY_DRUGS_TYPE_SLICE）→ 保存药房的加工服务（代煎、制粉等工艺）

### 第六步：线下药房特殊处理
- 如果药房类型为"线下"（PHARMACY_TYPE_OFFLINE）→ 自动注册/变更药房管理员账号
  - 手机号未注册 → 自动注册新用户 → 授予"线下药房管理员"角色
  - 手机号已注册 → 直接授予管理员角色

## 编辑药房流程

### 基础校验
- 检查药房是否存在（"药房信息不存在"）
- 名称唯一性（编辑时排除自身ID）
- 手机号唯一性（编辑时排除自身ID）
- 不允许将已启用第三方的药房改为禁用（"已启用第三方药房的不允许编辑为禁用"）

### 第三方药房信息同步
- 已有第三方记录 → 编辑更新
- 新增第三方 → 创建新记录

### 线下药房手机号变更
- 修改了手机号 → 重新处理管理员注册/换绑

### 加工服务更新
- 如果药品类型包含饮片 → 重新保存加工服务信息

## 药房绑定/解绑

### 绑定（bind）
- 一个用户只能绑定一个药房（"当前用户已绑定别的药房"）
- 线下药房自动绑定对应的药房管理员
- 避免重复绑定（已存在绑定关系则跳过）

### 解绑（unbind）
- 检查用户是否还有其他线下药房角色
- 无其他角色 → 删除用户-药房关联
- 有其他角色 → 不删除（用户可能管理多个药房）

## 药房查询

### 按用户查询
- 查询当前用户绑定的药房
- 支持仅查询可用状态（available=1）的药房

### 分页/列表查询
- 系统默认药房模式（pharmacyProperties.defaultEnabled）→ 强制限制为指定默认药房
- 线下药房开关（pharmacyProperties.offlineEnabled）→ 关闭时只查线上药房
- 支持按绑定关系过滤（如医生的药房授权）

## 异常处理

| 情况 | 处理方式 |
|------|---------|
| 手机号为空 | "手机号不能为空" |
| 药房名称重复 | "药房名称重复" |
| 手机号已绑定其他药房 | "该手机号已绑定药房" |
| 编辑不存在的药房 | "药房信息不存在" |
| 禁用已启用的第三方药房 | "已启用第三方药房的不允许编辑为禁用" |
| 用户已绑定其他药房 | "当前用户已绑定别的药房" |
| 新增/编辑数据库失败 | 对应的失败提示 |

## 关联模块

| 模块 | 关联方式 |
|------|---------|
| 药品管理 | 药房管理药品库存和上下架 |
| 订单管理 | 订单关联药房作为发货方 |
| 系统用户 | 线下药房自动创建管理员账号 |
| 第三方对接 | 药房可对接外部药品系统 |
| 医生授权 | 医生通过药房授权才能开具该药房药品 |
