# 导诊记录创建与使用流程

> 文档版本 v1.0 | 面向读者：产品与运营人员  
> 服务模块：ykj-service-guidance

## 概述

当患者通过第三方渠道（如医院公众号、HIS系统等）进入导诊服务时，平台记录患者的导诊信息（主诉、病历资料、监护人信息等），并生成唯一导诊编号。医生接诊时可通过导诊编号直接获取患者档案，减少重复填写。

## 核心流程

### 一、导诊记录创建（guidanceSyn）

第三方系统传入患者信息和导诊数据：

1. **创建患者档案**：调用PatientArchivesService创建或更新患者健康档案
2. **处理病历资料**：将上传的病历图片（medicalRecordImages）通过OSS服务转存为平台内部资源ID
3. **保存导诊记录**：
   - 根据身份证号计算年龄
   - 生成唯一导诊编号（UUID）
   - 状态设为"未使用(Unused)"
   - 关联患者档案ID
4. **保存监护人信息**（如果有）：将监护人的姓名、身份证、与患者关系保存到监护人表
5. 返回唯一导诊编号（uniqueId）

### 二、医生使用导诊记录（useArchivesByUniqueId）

医生接诊时通过导诊编号获取患者信息：

1. 根据唯一编号查询导诊记录
2. 校验记录是否存在 → 不存在抛出"资源不存在"
3. 将状态更新为"已使用(Used)"，记录医生ID
4. 返回患者的档案信息（姓名、身份证、关系、病历资料等）

### 三、更新导诊患者信息（upRecordPatient）

患者信息不完整时，可补全患者档案：

1. 根据唯一编号查询导诊记录
2. 调用PatientArchivesService更新患者档案
3. 更新导诊记录关联的patientId
4. 返回更新后的患者信息

## 异常处理

| 情况 | 处理方式 |
|------|---------|
| 参数异常（uniqueId为空） | 抛出"参数异常" |
| 导诊记录不存在 | 抛出"资源不存在" |
| 同步导诊记录失败（保存失败） | 抛出"同步导诊记录失败" |
| 监护人新增失败 | 抛出"患者监护人新增失败" |
| 数据更新失败 | 抛出"数据更新失败" |

## 状态流转

```
第三方创建导诊
    ↓
未使用(Unused) —— 医生首次接诊使用 → 已使用(Used)
    ↓
患者信息补全（仍为已使用状态）
```

## 关键设计

- **唯一编号**：使用fastSimpleUUID生成全局唯一导诊编号，用于跨系统传递
- **患者档案复用**：通过PatientArchivesService管理患者档案，导诊记录仅关联patientId
- **病历图片处理**：外部图片URL通过OSS transfer转为内部资源ID，统一管理
- **年龄计算**：根据身份证号自动解析年龄，无需手动填写
- **监护人支持**：支持录入监护人信息（姓名、身份证、关系），适用于儿童/老人等需要监护人的场景
- **身份证加密**：监护人身份证使用Crypto加密存储

## 关联功能

- 患者档案管理（PatientArchivesService）
- OSS文件存储服务
- 开放平台导诊接口（OpenController/OpenHisController）
- 医生接诊流程
