# 处方预结算与下单流程

> 文档版本 v1.0 | 面向系统分析

## 概述

处方预结算是在患者下单购买处方药品之前，计算所有费用项的预览功能。结算时正式创建订单、扣减库存并生成支付信息。支持快递配送和到店自提两种取药方式，支持普通支付和医保支付两种付款渠道。

**对应服务模块**：ykj-service-inquiry

## 触发条件

- 患者在小程序端查看处方药品后点击"去下单"
- 患者切换配送方式（快递↔自提）时重新计算费用
- 患者提交订单进行最终结算

## 预结算流程

### 第一步：查处方并校验状态
1. 根据处方编号（outpatientNo）查询处方信息
2. 校验处方状态是否允许结算
3. 查询药房信息（名称、地址、服务费等）

### 第二步：组装药房信息
- 药房名称、地址
- 药房支持的配送方式

### 第三步：组装收货地址
- 患者已有的收货地址（快递模式）
- 药房地址作为自提地址（自提模式）

### 第四步：确认服务加工费
- 根据选择的加工技术（如中药代煎）计算加工费
- 更新处方的加工信息

### 第五步：计算费用项
费用项包括：
1. **药品费用**：所有处方药品的单价×数量之和
2. **服务加工费**：中药代煎等加工服务的费用
3. **运费**：根据收货地址匹配运费规则计算
4. **定金**（如有配置）：部分药房支持预付定金模式

最终预结算总金额 = 各项费用之和（如出现负数则置为0）

## 正式结算流程

### 第一步：校验
1. 检查该处方是否已存在有效订单（"已存在订单信息"）
2. 重新执行预结算（费用项不合并服务费，精确计算）

### 第二步：创建订单
1. 组装订单数据：处方信息、药品明细、收货地址、费用项
2. 支持自定义订单号（开放平台调用时）
3. 确定订单来源类型：处方结算（PRESCRIPTION）
4. 调用订单服务创建订单

### 第三步：支付渠道
- 普通支付：微信支付、支付宝
- 医保支付：仅当系统开启医保功能时可用
  - 需要医保授权码查询医保信息
  - 调用医保服务获取预结算结果
  - 支持微信医保小程序支付

### 第四步：生成处方PDF
1. 创建处方PDF文件（区分西药/中药处方）
2. 包含患者信息、诊断、药品明细、用法用量
3. 医生签名、医院印章
4. 上传PDF到OSS

## 医保结算特殊处理

处方结算关联医保时：
1. 查询患者的医保授权信息
2. 医保支付方式包括：统筹支付、个人账户支付、医保微信支付
3. 结算时会生成医保费用明细上传
4. 医保结算需要将处方费用拆分为医保支付 + 自费支付

## 异常处理

| 情况 | 后端提示 |
|------|---------|
| 已存在有效订单 | 已存在订单信息 |
| 处方状态不允许结算 | （处方状态校验异常） |
| 药房信息缺失 | （查询药房异常） |
| 收货地址不完整 | （地址校验异常） |
| 费用计算异常 | 预结算金额负数时自动置0 |

## 状态流转

```
处方待下单 → 预结算（展示费用预览）
           → 正式结算（创建订单）
           → 订单待支付 → 已支付 → 已发货 → 已完成
```

## 关联功能

- 订单服务（ykj-service-order）：创建和管理订单
- 药品服务（ykj-service-medicine）：药品信息和药房
- 运费服务（ykj-service-order）：运费计算
- 医保服务（ykj-service-yb）：医保结算
- 会员服务（ykj-service-member）：收货地址
- 病历服务（ykj-service-inquiry）：关联诊断
