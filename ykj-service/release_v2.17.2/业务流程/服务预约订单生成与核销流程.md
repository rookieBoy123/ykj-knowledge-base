# 服务预约订单生成与核销流程

> 文档版本：v1.0 | 面向读者：运营、客服、产品

## 概述

患者通过小程序预约线下医疗服务（如体检、核酸、健康证等），系统生成预约单和订单，支持线上支付和线下核销。预约包含完整的生命周期管理：待支付 → 待核销 → 已核销 / 已取消 / 退款中 / 已退款。

## 触发条件

- **前置条件**：患者已登录，选择了可预约的服务项目（体检/核酸/健康证/检查等）
- **入口**：患者在小程序服务详情页点击「立即预约」→ 填写信息 → 提交预约

## 步骤详解

### 第一步：生成预约单和订单

患者提交预约后，系统在同一个事务中执行以下操作：

- **校验可预约性**：检查该项目当前日期是否可预约。如果项目没有可预约日期，提示"当前项目暂无可预约时间"；如果选择的日期不在可预约列表内，提示"当前日期该项目暂无法预约"
- **计算费用**：根据服务项目和预约人数计算总费用和费用明细项
- **生成预约单号**：格式为固定前缀 + 年月日 + 7位随机数
- **状态判断**：
  - 总费用为0 → 状态直接设为「待核销」，同时生成8位随机核销码
  - 总费用大于0 → 状态设为「待支付」，等待患者付款
- **创建订单**：生成订单号，订单来源标记为"服务预约"，关联预约单号
- **创建发票信息**：如果患者选择了需要发票，自动创建发票记录（0元订单不能开票）
- **0元预约推送**：免费预约直接触发订阅消息通知

### 第二步：支付回调处理

患者支付成功后，系统接收支付平台回调：

- **更新预约状态**：从「待支付」变为「待核销」
- **生成核销码**：自动生成8位随机大写字母+数字组成的核销码
- **发送通知**：支付成功订阅消息推送

### 第三步：订单取消与退款

- **取消条件**：仅「待支付」和「待核销」状态的预约可以取消
- **取消状态映射**：小程序端取消状态包含三种后端状态：已取消、已退款、退款中
- **退款流程**：已付款的预约取消后触发退款申请

### 第四步：核销（线下使用）

患者到医院出示核销码完成服务：

- **核销码查询**：工作人员通过核销码查找预约记录
- **核销校验**：
  - 核销码必须存在 → 否则提示"未查询到预约记录，核销码无效"
  - 预约必须是「待核销」状态 → 否则提示"此核销码已核销"
- **执行核销**：更新状态为「已核销」，记录核销时间
- **订单完成**：同步将关联订单标记为「已完成」
- **自动开票**：如果系统配置了自动开票且订单金额大于0，核销时自动触发票据生成

### 第五步：预约详情查询

- **权限控制**：当前用户必须是预约本人或被分享的用户，否则提示"当前患者无权限查看该预约单信息"
- **信息聚合**：详情页同时展示预约信息、订单信息、费用明细、患者信息、发票信息、服务项目详情
- **患者信息**：从患者档案和HIS系统双重获取（姓名、性别、年龄、门诊号）

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 项目没有可预约日期 | 提示"当前项目暂无可预约时间" |
| 选择的日期不可预约 | 提示"当前日期该项目暂无法预约" |
| 项目不存在 | 返回"未找到" |
| 核销时核销码无效 | 提示"未查询到预约记录，核销码无效" |
| 核销时已核销 | 提示"此核销码已核销" |
| 核销时非待核销状态 | 提示"该订单不是待核销状态订单" |
| 核销时核销码不匹配 | 提示"错误的核销码" |
| 非本人查看预约详情 | 提示"当前患者无权限查看该预约单信息" |
| 会员ID为空 | 提示"未授权" |

## 关联功能

- **订单生成与支付处理流程**：预约订单的支付
- **退款审核与处理流程**：预约取消后的退款
- **发票管理流程**：电子发票的生成和管理
- **服务预约选与支付流程**（小程序）：患者端的操作入口
