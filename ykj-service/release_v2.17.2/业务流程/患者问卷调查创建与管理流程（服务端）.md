# 患者问卷调查创建与管理流程（服务端）

> 文档版本 v1.0 | 面向读者：后台开发/运营管理员

## 概述
问卷调查是平台收集患者反馈、评估健康状况的重要工具。服务端支持调查问卷的完整生命周期管理——从创建问卷、配置题目、关联医生、下发给患者，到患者填写提交、数据统计。问卷调查独立于随访模块的随访问卷，更侧重于患者端自主评价和调查。

## 触发条件
- **问卷创建**：管理员在后台创建新的调查问卷
- **问卷下发**：关联到指定医生，患者在问诊后可见
- **患者填写**：患者通过小程序端进入问卷页面作答
- **权限要求**：管理端操作需对应权限

## 核心步骤

### 1. 问卷模板创建
管理员创建调查问卷，配置以下信息：
- 问卷标题和说明
- 题目列表（支持单选、多选、填空等题型）
- 问卷适用范围（全平台/指定医生/指定科室）

### 2. 关联医生配置
问卷需要关联到具体医生才生效。医生的问卷配置包含：
- 是否启用问卷调查
- 关联哪些问卷模板
- 问卷下发时机（问诊结束后自动推送 / 手动推送）

### 3. 患者端填写
- 患者在问诊结束后，系统根据医生配置自动推送问卷
- 进入问卷页面查看题目列表
- 逐题作答后提交
- 提交后系统记录答卷数据和提交时间

### 4. 结果统计
- 管理员可在后台查看每份问卷的统计结果
- 支持按医生、按时间范围聚合统计
- 可查看单个患者的答卷详情

## 关键约束
| 约束项 | 说明 |
|--------|------|
| 问卷关联 | 问卷必须关联到医生，患者只能看到关联医生的问卷 |
| 填写次数 | 同一患者对同一问卷可多次填写（取决于配置） |
| 匿名/实名 | 问卷可配置为匿名或实名模式 |
| 题目必答 | 可设置题目为必答，未答完不可提交 |

## 异常处理
| 情况 | 处理方式 |
|------|---------|
| 医生未配置问卷 | 患者问诊结束后不会收到问卷推送 |
| 问卷已停用 | 患者端不可见，但历史答卷数据保留 |
| 患者多次填写 | 根据问卷配置决定是否允许 |
| 提交时网络异常 | 前端暂存答案，网络恢复后重新提交 |

## 关联功能
- **问诊模块**：问卷下发与问诊流程关联
- **医生管理**：问卷配置与医生设置关联
- **随访问卷**：独立于随访模块的问卷体系
