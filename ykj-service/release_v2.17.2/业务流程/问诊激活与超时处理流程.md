# 问诊激活与超时处理流程

> 文档版本 2026-07-02 | 面向产品运营、客服

## 概述

问诊预约服务负责任务问诊从创建到结束的完整管理，包括问诊激活（开启聊天、发送开始通知）、重复问诊校验、问诊超时自动结束等后端逻辑。确保患者在正确的时间进入问诊，避免重复下单，以及在超时后自动流转状态。

## 触发条件

- 患者支付完成后激活问诊
- 系统定时任务扫描超时问诊单
- 所属模块：`ykj-service-inquiry`

## 步骤详解

### 第一步：重复问诊校验

患者下单前，系统自动检查是否已有相同条件下的进行中问诊，避免重复支付：

**校验维度**：
- 同一会员、同一患者、同一医生、同一服务类型
- 问诊状态为「待支付」或「问诊中」

**特殊场景**：
- **商城问诊**：仅校验是否有相同用药需求编号的问诊
- **预约挂号**：校验同一患者在同一天同一医生是否有号段
- **快速问诊**：只校验快速问诊类型的重复

**检查结果**：
- 已有进行中问诊 → 拒绝创建，提示用户已有问诊在进行
- 无进行中问诊 → 允许创建

**额外校验**：
- 医生不能问诊自己账号下的患者（防止刷单）

### 第二步：问诊激活

支付完成后激活问诊单，进入正式问诊阶段：

**激活流程**：
1. 校验问诊单是否存在
2. 启用问诊聊天功能（enableInquiry）
3. 发送问诊开始通知（系统消息推送到IM群聊）
4. 可选：自动接诊（reception=true时主动调起医生接诊）

**问诊可用性检查**：
- 未支付状态 → 不可进入问诊
- 聊天室被禁用 → 不可进入问诊
- 群聊ID不存在 → 不可进入问诊

### 第三步：问诊角色管理

系统管理问诊中的不同角色：

**主治医生**（DoctorFlag=Main）：
- 负责主要诊疗的医生
- 问诊结束时修改为「完成」状态

**服务医生**（DoctorFlag=Service）：
- 协助问诊的辅助医生
- 可能包含医助或转诊医生

**患者**：
- 问诊的发起方

所有角色都有对应的IM账号，用于群聊通信。

### 第四步：超时自动处理

系统通过定时任务自动处理超时的问诊：

**自动结束超时问诊**：
- 扫描状态为「草稿」或「问诊中」的问诊单
- 匹配指定的问诊ID和医生状态
- 系统配置 autoEnd 开关控制是否启用

**有服务医生的超时场景**：
1. **主治医生正在接诊中**（状态=MainUnderway）：
   - 自动转诊给服务医生
2. **医助正在接待中**（状态=AssistantUnderway）：
   - 自动取消问诊，原因："超过接诊时间，系统自动取消"
   - 取消类型标记为「医生拒绝接诊」

**无服务医生的超时场景**：
- 直接结束问诊

### 第五步：患者端问诊列表查询

患者查看自己的问诊列表，按类型区分：

- **已预约**（wait）：问诊中但未签到的预约挂号
- **问诊中**（underway）：正在进行的问诊（排除未签到预约），按创建时间升序
- **已完成**（finish）：医生标记为完成的问诊，按创建时间倒序
- **待支付**（draft）：草稿状态的问诊

消息列表中的问诊：显示进行中、已完成、已取消的问诊，按创建时间倒序和状态排序。

## 异常处理

| 情况 | 系统处理方式 |
|-----|------------|
| 问诊单不存在 | 提示"问诊单不存在, 无法激活问诊" |
| 已有进行中问诊 | 提示"系统不支持问诊自己账号下医生"或拒绝创建 |
| 会员信息不存在 | 提示"会员信息不存在" |
| 非医生角色 | 提示"当前用户非医生角色" |
| 参数错误 | 提示"参数错误" |
| 问诊自动结束开关关闭 | 跳过定时任务处理，问诊保持当前状态 |

## 关联功能

- 问诊激活 → IM群聊创建 → 消息推送
- 问诊激活 → 医生接诊
- 问诊超时 → 自动转诊 → 服务医生接诊
- 问诊超时 → 自动取消 → 退款处理
- 问诊超时 → 自动结束 → 病历归档
