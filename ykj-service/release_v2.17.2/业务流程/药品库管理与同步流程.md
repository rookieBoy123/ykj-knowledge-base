# 药品库管理与同步流程

> 文档版本：2026-07-01 | 面向读者：产品经理、技术支持

## 概述

平台维护一个统一的药品库，支持中药和成药两种类型。药品数据可手动录入、批量导入，也可从第三方药房系统（阿康）同步获取。

## 药品分类

系统将药品分为两大类：

| 类型 | 说明 | 核心字段 |
|------|------|---------|
| 中药 | 中药材/中药饮片 | 药品别名、药用部位、性味归经、功能主治、用法用量 |
| 成药 | 中成药/西药 | 通用名、商品名、规格、批准文号、生产厂家 |

每种药品都关联到具体的药房（Pharmacy），不同药房的药品独立管理。

## 步骤详解

### 第一步：手动录入药品

**录入中药：**
1. 管理员填写中药信息（药品别名必填）
2. 系统检查同一药房下是否已存在同名中药（按drugsAlias匹配）
3. 如已存在，提示"药品已存在"
4. 如不存在，创建新的中药记录

**录入成药：**
1. 管理员填写成药信息
2. 系统通过多字段组合（通用名+商品名+规格+厂家+批准文号）判断药品是否已存在
3. 如已存在，提示"药品已存在"
4. 如不存在，创建新的成药记录

### 第二步：批量导入药品

系统支持通过Excel模板批量导入药品：

1. 下载导入模板（中药模板 / 成药模板）
2. 按模板格式填写药品数据
3. 上传Excel文件
4. 系统逐行处理：已存在的药品自动覆盖更新（导入优先），不存在的创建新记录
5. 导入失败的记录返回错误原因（如数据格式错误等）

### 第三步：从第三方药房同步药品

平台对接了阿康（AKANG）药房系统，可以拉取药品数据：

1. 配置第三方药房的接口信息和药房映射
2. 系统调用阿康的药品列表接口（每页100条）
3. 拉取到的药品数据自动入库，并关联到对应的平台药房
4. 可单独获取药品图片信息（通过药品编码查询）
5. 连接超时阈值设为7000毫秒，超时则同步失败

### 第四步：药品搜索与匹配

系统支持多种药品搜索方式：

- **中药搜索**：支持中文名称和拼音首字母搜索。输入拼音字母时，系统使用字符串相似度算法对结果排序
- **成药搜索**：搜索前先检查是否命中精麻毒管制药品黑名单。如果搜索关键词在黑名单中，直接提示"该药品为精麻毒等管制类药品，按规定不可线上开方"
- **按药房搜索**：可筛选特定药房的药品
- **按医生搜索**：根据医生身份自动关联其有权限的药房

### 第五步：药品校验

开方时系统对药品进行完整性校验：

- 检查药品ID是否存在，不存在返回禁用状态（status=DISABLED）
- 检查药品库信息是否完整，不完整抛出"药品库信息不存在"错误
- 检查药品的库存单位和换算比例（用于处方剂量计算）

### 药品状态管理

每种药品有关联状态：
- **启用**：正常可用，医生可以搜索和在处方中使用
- **禁用**：不可用，医生搜索不到该药品
- **黑名单**（精麻毒管制类）：系统层面禁止线上开方，搜索该药品时直接拦截

## 异常处理

| 情况 | 系统处理方式 |
|------|------------|
| 重复录入药品 | 手动录入时返回"药品已存在"；批量导入时覆盖更新已有药品 |
| 搜索管制药品 | 命中黑名单时返回"该药品为精麻毒等管制类药品，按规定不可线上开方" |
| 药品库信息缺失 | 返回"药品库信息不存在"的内部错误 |
| 第三方同步超时 | 超过7秒连接超时，同步中断，需重试 |
| 药房未配置 | 如果医生无关联药房，可查询的药房列表为空 |

## 关联功能

- **处方开具**：医生在小程序端搜索药品时，从药品库中查询符合权限的药品
- **价格公示**：药品价格数据可供患者端的价格公示页面使用
- **药房管理**：药品库与药房系统紧密关联，不同药房的药品独立管理
- **库存管理**：药品库存状态影响医生开方时的可用性提示
