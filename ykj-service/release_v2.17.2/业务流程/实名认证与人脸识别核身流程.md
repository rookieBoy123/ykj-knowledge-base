# 实名认证与人脸识别核身流程

> 文档版本：2026-07-03 第5轮 | 面向读者：产品经理、测试人员

## 概述

实名认证是用户在平台进行问诊、开方、购药等操作的前置条件。系统支持腾讯云和阿里云两家服务商的人脸核身能力，提供"身份证二要素校验→人脸活体检测→视频对比"三层安全验证，确保用户身份真实有效。

## 触发条件

- **前置条件**：用户已注册并有实名认证需求
- **入口**：小程序的"个人中心→实名认证"入口，或在问诊/购药流程中被拦截要求先认证
- **调用方**：问诊模块、会员模块

## 步骤详解

### 第一步：身份证二要素校验

用户提交姓名和身份证号后，系统先进行"姓名+身份证号"是否一致的校验：

- **腾讯云模式**：调用腾讯云FaceID的 `IdCardVerification` 接口，验证姓名和身份证号是否匹配。如果账户欠费（错误码 UnauthorizedOperation.Arrears），系统提示"实名信息异常（QF），请联系客服 [个人中心-联系客服]"
- **阿里云模式**：调用阿里云CloudAuth的 `Id2MetaVerify` 接口。阿里云采用双区域容灾策略——先尝试上海节点（cn-shanghai），如果返回500服务端错误则自动切换到北京节点（cn-beijing）重试。两个节点都失败则校验不通过

二要素校验不通过，直接返回失败，不进入后续步骤。

### 第二步：获取人脸核身启动URL

校验通过后，系统生成一个唯一的订单号（orderNo），调用人脸识别服务获取启动人脸核身的H5页面URL：

- **腾讯云模式**（getStartH5FaceUrl）：
  1. 获取AccessToken（缓存20分钟，过期自动刷新）
  2. 获取SignTicket（与AccessToken同步刷新）
  3. 上送用户身份信息（姓名、身份证号、用户ID、订单号）获取 `h5faceId`
  4. 获取NonceTicket
  5. 生成带签名的启动URL：包含 `webankAppId`、`nonce`（32位随机串）、`orderNo`、`h5faceId`、回跳地址

- **阿里云模式**（getStartH5FaceUrl）：
  1. 需要必传 `redirectUrl` 和 `metaInfo`（客户端环境信息），任一为空抛出异常
  2. 调用 `InitFaceVerify` 接口设置：场景ID、活体检测模式（LIVENESS）、身份证信息、回跳地址
  3. 同样采用双区域容灾策略（上海→北京自动切换）
  4. 返回 `certifyId`（认证标识）和 `certifyUrl`（认证页面URL）

### 第三步：用户完成人脸识别

用户在小程序中打开第二步返回的认证URL，按照页面引导完成点头、眨眼、张嘴等活体检测动作。活体检测默认为静默模式（SILENT），用户无需做特定动作。

### 第四步：查询认证结果

用户完成人脸识别后，系统查询认证结果：

- **腾讯云模式**（getFaceResult）：调用查询接口，传入应用ID、订单号、nonce、SignTicket签名。返回code为"0"表示通过
- **阿里云模式**（getFaceResult）：调用 `DescribeFaceVerify` 查询，返回code为"200"且pass标记为"T"表示通过

### 第五步：视频活体对比（增强模式）

除H5页面模式外，系统还支持直接上传视频进行活体检测：

- **腾讯云模式**（videoFaceVerify）：上传身份证视频（Base64编码），调用 `LivenessRecognition` 接口，对比视频中的人脸与身份证照片。成功返回Result为"Success"
- **阿里云模式**（videoFaceVerify）：上传视频文件流，调用 `ContrastFaceVerify` 接口，进行人脸对比。同样支持双区域容灾

视频上传后会立即删除本地临时文件。

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 腾讯云账户欠费 | 后端拦截：提示"实名信息异常（QF），请联系客服 [个人中心-联系客服]" |
| 阿里云上海节点返回500 | 自动切换到北京节点重试（双区域容灾），两个节点都失败则返回失败 |
| redirectUrl未传 | 后端拦截：阿里云直接抛出异常"redirectUrl and metaInfo must not be null" |
| 身份证二要素不匹配 | 返回认证失败，腾讯云返回描述信息 |
| AccessToken缓存命中 | 20分钟内不重新请求腾讯云API，直接使用缓存的Token |
| 请求Ticket返回空 | 记录错误日志，返回null |
| 视频文件不存在 | 标记转储失败且设置重试次数为上限值（3次），不再重试 |
| 身份信息上送失败 | 记录错误日志，返回null |

## 技术特点

- **双服务商支持**：通过 `FaceBaseService` 接口统一腾讯云（type="tx"）和阿里云（type="ali"）两种实现
- **三层缓存策略**：Redis缓存 → 本地加密库 → 第三方API调用，减少外部请求
- **双区域容灾**：阿里云自动在上海和北京节点间切换，规避单点故障
- **签名安全**：腾讯云所有请求都需要通过SignTicket或NonceTicket生成签名，防止篡改

## 关联功能

- 会员实名认证（ykj-service-member）：调用人脸核身完成实名
- 问诊下单（ykj-service-inquiry）：要求用户已实名认证
- 电子签章（ykj-service-ca）：部分签章流程需要先完成人脸核身
