# 开放平台问诊创建、取消与信息更新流程（服务端）

> 文档版本 2026-07-06 | 面向：产品经理、运营人员

## 概述

外部合作机构通过开放平台创建图文问诊单，系统自动匹配医生、生成问诊单、创建IM群聊、保存病历和订单，并支持后续的信息更新和取消操作。

## 触发条件

- **前置条件**：合作机构已注册并获得AppId，且机构下有关联医生
- **入口**：外部机构系统调用开放平台API

## 步骤详解

### 一、问诊发起（initiate）

外部机构传入AppId、医生ID、病情描述、患者信息（姓名/性别/年龄）、可选的病历图片和病史信息。

系统处理：
1. 根据AppId查找机构信息，找不到则直接返回"找不到对应的机构信息"（代码位置：OpenInquiryServiceImpl.initiate() 第91-93行）
2. 如果是预约问诊，必须提供预约开始和结束时间，缺少任一参数会提示"参数缺失, 预约问诊需要提供预约开始、结束时间"（第100-102行）
3. 患者信息（姓名/性别/年龄）为必填项，任一缺失会提示"参数缺失, 请正确填写患者信息"（第113-114行）
4. 病历图片由外部URL自动转存到系统OSS

### 二、问诊创建与上传（upload/create）

这是更完整的问诊创建流程，包含患者注册、问诊生成、IM创建、病历保存和订单生成：

1. **患者注册**：通过手机号自动注册会员，默认关系为"其他"
2. **医生匹配**（第605-616行）：按医生姓名和身份证号查找机构下的医生。如果是零售机构且未匹配到，系统会从机构下随机分配一位医生；如果机构下完全没有医生，返回"该机构下没有关联医生！"
3. **生成问诊单**：设置服务类型为图文问诊、问诊状态为进行中、主医生接诊中，生成唯一问诊单号。如果生成失败提示"生成问诊单失败"（第285-287行）
4. **创建IM群聊**：自动创建腾讯IM群组（群名："医生名工作室"），将医生和患者加入群聊
5. **创建病历**：诊断信息必须先在系统诊断库存在，验证不通过会提示"诊断信息不存在，诊断：XXX, 编码：XXX"（第732-733行）
6. **创建订单**：生成问诊费用订单，费用>0元时自动生成支付流水（支持微信支付和线下支付两种渠道）
7. **发送问诊结束事件**：标记问诊完成

### 三、问诊取消（cancel）

外部机构可以取消自己创建的已接诊问诊（第146-163行）：

- 首先验证机构身份（找不到机构提示"找不到对应的机构信息"）
- 再验证问诊存在（找不到提示"找不到对应的问诊信息"）
- 检查问诊是否属于该机构（不属于提示"问诊不属于请求机构,无权限取消"）
- 只有在"待接诊"或"接诊中"状态的问诊才能取消，其他状态提示"该问诊状态不允许取消"

### 四、问诊信息更新（updateBase）

支持更新病情描述、病历图片、患者病史、诊断信息（第364-493行）：

- 通过问诊单号查找问诊和关联病历，任一不存在即退出
- 更新病情描述、病历图片、患者病史（过敏史/既往史/家族史等）和诊断
- 如果问诊尚未创建订单（如初次上传未提供金额），系统会根据更新信息自动补创建订单
- 病历图片异步转存，不阻塞主流程

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 找不到机构信息 | 后端拦截，提示"找不到对应的机构信息" |
| 预约问诊缺少时间参数 | 后端拦截，提示"参数缺失, 预约问诊需要提供预约开始、结束时间" |
| 患者信息不完整 | 后端拦截，提示"参数缺失, 请正确填写患者信息" |
| 机构下无关联医生 | 后端拦截，提示"该机构下没有关联医生！" |
| 问诊不属于该机构 | 后端拦截，提示"问诊不属于请求机构,无权限取消" |
| 问诊状态不允许取消 | 后端拦截，提示"该问诊状态不允许取消" |
| 诊断编码不存在 | 后端拦截，提示"诊断信息不存在" |
| IM账号获取失败 | 后端拦截，提示"获取医生/患者im账号信息失败" |
| 病历图片转存失败 | 异步处理，失败不影响主流程但会记录日志 |

## 关联功能

- 开放平台订单全生命周期（处方下单→结算→发货→取消）
- 会员注册与管理（自动注册外部患者）
- IM即时通讯（自动创建群聊）
- 病历管理（自动创建和更新病历）
