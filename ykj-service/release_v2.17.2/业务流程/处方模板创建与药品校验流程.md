# 处方模板创建与药品校验流程

> 文档版本 1.0 | 面向读者：产品、运营

## 概述

医生创建个人处方模板，保存常用处方以便快速复用。模板分为中药模板和成药模板两种，创建时系统自动校验诊断、药品有效性、名称唯一性。

## 触发条件

- 医生端 → 我的药房 → 处方模板 → 创建模板
- 或从开方页面保存为模板

## 核心流程

### 1. 创建处方模板

**前置校验**：
- 模板名称不能与该医生的同类型模板重复
- 药品详情列表不能为空
- 必须包含诊断信息（成药模板强制要求）

**编号生成**：
每个模板自动生成唯一的门诊编号（outpatientNo），格式为"TR+年月日+7位随机数"（如TR202607020012345）。

**诊断保存**：
- 每个模板可关联多个诊断（按诊断类型区分：诊断/辨病/辨证）
- 诊断批量写入诊断关联表

### 2. 药品信息校验

根据处方类型不同，校验方式不同：

**中药模板（type=Herbal）**：
- 通过药品通用名（drugsAlias）从药品库查询
- 校验通用名不能为空
- 校验所有药材在药品库中都存在，否则提示具体失效药品名称
- 自动补充：药品名称、别名、单位、药品库ID

**成药模板（type=Patent）**：
- 通过监管ID（supervisionId）从药品库查询
- 校验监管ID不能为空
- 校验所有药品在药品库中都存在，否则提示具体失效药品名称
- 自动补充：药品名称、别名、单位、监管ID、药品库ID

### 3. 诊断校验

- 医嘱诊断（diagnosisType=DISEASE）的编码必须在诊断库中存在
- 调用诊断服务（diagnosisService.checkDiagnosis）验证

### 4. 编辑模板

- 同样校验名称唯一性（排除当前模板自身）
- 更新时先删除旧的诊断和药品关联，再重新插入

### 5. 模板数据构成

每个处方模板包含三部分数据：
- **模板主信息**（PrescriptionTemplate）：标题、类型、医生ID、创建时间
- **诊断关联**（PrescriptionTemplateDiagnose）：多条诊断记录
- **药品明细**（PrescriptionTemplateItem）：多条药品记录

## 异常处理

| 情况 | 后端处理 |
|------|----------|
| 模板名称重复 | 抛出"模板名称重复" |
| 药品列表为空 | 抛出"处方模板药品详情不能为空" |
| 诊断信息为空（成药） | 抛出"处方模板诊断不能为空" |
| 中药通用名为空 | 抛出"通用名不能为空" |
| 中药通用名有误 | 抛出"通用名有误" |
| 存在失效药品 | 抛出"设置XX处方模板失败, 存在失效药品: XXX" |
| 成药的监管ID为空 | 抛出"监管ID不能为空" |
| 成药的药品ID有误 | 抛出"药品ID有误" |
| 创建失败 | 抛出"处方模板创建失败" |
