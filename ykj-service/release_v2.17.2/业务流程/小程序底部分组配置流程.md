# 小程序底部分组配置流程

> 文档版本：2026-07-03 | 面向读者：产品、运营、管理员

## 概述

底部分组是小程序底部导航栏（Tab Bar）的自定义配置功能。管理员可以在后台配置底部导航栏的图标（选中/未选中两种状态图标）、跳转页面等信息。每个小程序只有一个底部分组（编码固定为 `BOTTOM`），修改后小程序端实时生效。

## 触发条件

- 需要修改小程序底部导航栏的图标样式
- 需要调整底部导航的跳转目标
- 需要启用/禁用底部导航栏

## 数据模型

底部分组包含两级数据：
- **分组（DiyGroup）**：顶层的底部分组配置，包含是否启用（enabled）、编码（固定为 `BOTTOM`）、名称（固定为"底部分组"）
- **分组数据（DiyGroupData）**：每个底部导航项的详细配置，包含排序（sort）、显示值（value，JSON 格式，含 checked 图标、unchecked 图标、跳转路径等）

## 步骤详解

### 第一步：首次创建底部分组

系统自动设置：
- 编码固定为 `BOTTOM`
- 名称固定为"底部分组"
- 如果编码重复（已存在），系统提示"存在相同编码的分组，请重新输入"

### 第二步：配置导航项

在底部分组中添加多个导航项（分组数据），每个导航项配置：
- **选中状态图标（checked）**：导航项被选中时显示的图标
- **未选中状态图标（unchecked）**：导航项未被选中时显示的图标
- **跳转路径**：点击后跳转到的小程序页面路径
- **排序（sort）**：导航项在底部栏的显示顺序，从 0 开始递增

图标来源于素材库，保存时系统自动处理素材域名替换。

### 第三步：更新底部分组

更新流程：
1. 先根据编码查询现有底部分组
2. 如果分组不存在，提示"分组不存在，请重新输入"
3. 更新分组的启用状态
4. **删除所有旧的导航项数据**，重新保存新的导航项列表
5. 如果新的导航项列表为空，只更新启用状态，清空所有导航项

### 第四步：启用/禁用

通过分组的 `enabled` 字段控制底部导航栏是否在小程序中显示。禁用后小程序端将不显示底部导航栏。

## 小程序端展示

小程序端通过 API `/api/diy/group/bottom` 获取底部分组配置，查询时系统自动将图标路径中的初始化素材域名替换为实际的 OSS 域名，确保图标正常显示。

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 创建时编码重复 | 提示"存在相同编码的分组，请重新输入" |
| 保存失败 | 提示"保存失败，请稍后重试" |
| 更新时分组不存在 | 提示"分组不存在，请重新输入" |
| 更新失败 | 提示"更新失败，请稍后重试" |
| 查询时分组未创建 | 返回空（null），小程序端按默认配置展示 |

## 关联功能

- **自定义页面（DIY）**：自定义页面与底部分组配合使用，构成完整的小程序自定义首页
- **素材管理**：导航项图标从素材库选取
