# 会诊签章签名权限校验与类型识别流程（服务端深挖）

> 文档版本：2026-07-05 | 面向读者：产品经理、测试人员、技术支持

## 概述

会诊签章系统在医生进行签名操作时，需要校验该医生是否有权签署对应类型的签名。系统定义了五种签名类型（申请签名、审核签名、受邀签名、意见签名、报告签名），每种类型对应不同的权限校验规则。只有通过校验的医生才能完成签名操作。

## 触发条件

- 前置条件：会诊已创建，会诊编号已生成
- 入口：医生在小程序/PC端会诊详情页点击签名按钮

## 签名类型与权限规则

### 五种签名类型

| 签名类型 | 英文标识 | 谁可以签 | 触发时机 |
|---------|---------|---------|---------|
| 会诊申请签名 | APPLICATION | 会诊主治医师（创建会诊的医生） | 提交会诊申请时 |
| 审核签名 | REVIEW | 审核医生 | 审核通过会诊时 |
| 受邀签名 | INVITATION | 被邀请参加会诊的医生 | 确认参加会诊时 |
| 意见签名 | OPINION | 提交会诊意见的医生 | 提交会诊意见时 |
| 报告签名 | REPORT | 会诊主治医师 | 发布会诊报告时 |

### 权限校验规则

1. **会诊申请签名**：只有会诊的创建医生（主治医师）才能签。如果当前医生ID与创建者不同，提示"非主治医生无法签字"。

2. **受邀签名**：只有被邀请的医生列表中的医生才能签。系统会遍历会诊的所有受邀医生，如果当前医生不在列表中，提示"非受邀医生无法签字"。

3. **意见签名**：与受邀签名规则相同，只有受邀医生才能提交意见签名。如果当前医生不在受邀列表中，提示"非医生意见提交无法签字"。

4. **报告签名**：与会诊申请签名规则相同，只有会诊主治医师才能签。如果非主治医师操作，提示"非主治医生无法签字"。

## 步骤详解

### 第一步：签名发起

医生点击签名按钮，系统收到签名请求，包含：会诊编号、医生ID、签名类型、签名图片（或CA签章数据）。

### 第二步：权限校验

系统查找会诊的完整信息，然后调用校验方法检查医生是否有权签署该类型：

- 查询会诊基本信息（状态、创建医生等）
- 查询会诊受邀医生列表
- 根据签名类型逐一匹配权限

### 第三步：旧签名撤销

如果医生之前已经发起过同类型的签名（状态为"待签"或"已接受"），系统会先撤销旧签名记录，避免重复生成多条签名数据。

### 第四步：CA与非CA分支

- **启用CA签章**：系统创建CA签章对象，向CA服务平台发起签章请求，生成带数字证书的PDF报告
- **未启用CA签章**：系统直接将手写签名图片嵌入PDF报告，标记为普通签名

### 第五步：签名完成后的状态流转

签名完成后，根据不同类型触发不同的后续动作：

- **申请签名完成**：会诊状态流转为"申请中"，发送系统消息通知医生
- **审核签名完成**：会诊状态流转为"审核通过"，发送审核结果通知，邀请受邀医生加入
- **所有受邀医生签署完成**：会诊状态流转为"待患者确认"，发送会诊卡片消息和订阅通知给患者
- **报告签名完成**：会诊状态流转为"已完成"，计算医生收益，发布会诊完成事件

## 异常处理

| 情况 | 系统处理方式 |
|------|------------|
| 非主治医生签申请 | 前端提示"非主治医生无法签字" |
| 非受邀医生签受邀 | 前端提示"非受邀医生无法签字" |
| 非受邀医生签意见 | 前端提示"非医生意见提交无法签字" |
| 签名类型未知 | 后端抛出异常"未知的会诊签名类型" |
| CA签章创建失败 | 前端提示"创建签署记录失败，请联系管理员" |
| 签章记录已撤销 | 前端提示"签章记录已撤销，请刷新重试" |
| PDF生成失败 | 前端提示"电子图片创建失败" |
| 未找到签名记录 | 前端提示"未找到会诊签名记录记录" |

## 关联功能

- **CA签章系统**：提供数字证书签名服务
- **会诊审核流程**：签名完成后触发审核状态流转
- **系统消息推送**：签名完成后向医生/患者发送通知
- **订阅消息**：签名完成后向患者发送微信订阅消息
- **医生收益计算**：会诊报告签名完成后计算医生收益
