# 受邀机构医生管理流程（服务端）

> 文档版本 2026-07-04 | 面向系统对接人员

## 概述

受邀机构（远程会诊合作医院）的管理员登录后，系统自动识别其所属机构，所有医生管理操作自动限定在该机构范围内。这是机构数据隔离机制的核心——每个受邀机构只能管理自己的医生数据。

## 关键设计：机构自动识别

受邀机构管理员登录系统后，服务端通过其登录账号查找 `Inst`（机构）表，获取当前管理员所属的机构ID。后续所有数据操作（查询、新增、编辑、删除）都会自动带上这个机构ID，无需前端手动传递。

如果找不到对应的机构记录，服务端会抛出"非法请求"异常拒绝所有操作。

## 接口清单

| 接口 | 方法 | 路径 | 权限 | 说明 |
|------|------|------|------|------|
| 分页查询 | GET | `/mgmt/consultation/invited-inst-doctors` | `consul:invited-inst-doctors:read` | 查询当前机构下医生列表 |
| 下拉列表 | GET | `/mgmt/consultation/invited-inst-doctors/list` | `consul:invited-inst-doctors:read` | 医生集合查询（支持科室/职称筛选） |
| 新增 | POST | `/mgmt/consultation/invited-inst-doctors` | `consul:invited-inst-doctors:write` | 新增医生 |
| 编辑 | PUT | `/mgmt/consultation/invited-inst-doctors/{id}` | `consul:invited-inst-doctors:write` | 编辑医生 |
| 查询详情 | GET | `/mgmt/consultation/invited-inst-doctors/{id}` | `consul:invited-inst-doctors:read` | 查询单个医生 |
| 删除 | DELETE | `/mgmt/consultation/invited-inst-doctors/{id}` | `consul:invited-inst-doctors:write` | 删除医生 |

## 医生数据结构

| 字段 | 类型 | 说明 |
|------|------|------|
| name | String | 医生姓名 |
| departmentNo | String | 科室编号（关联数据字典 `ksbm`） |
| positionCategory | String | 职称（关联数据字典 `yszyjszwdm`） |
| positionGrade | String | 职称等级（关联数据字典 `zyjszwlbdm`） |
| phone | String | 联系电话 |
| introduction | String | 简介（最大500字符） |
| status | Boolean | 状态：true=启用，false=禁用 |

## 删除校验：会诊关联保护

删除医生时，服务端会先查询该医生是否已被关联到会诊记录中（`InstConsultationDoctorEntity` 表）。如果存在关联，拒绝删除并提示："医生【姓名】存在被申请或安排会诊记录，不可删除！"

这是为了防止删除已参与会诊的医生导致会诊数据出现医生信息缺失的异常。

## 下拉列表筛选逻辑

下拉列表查询（`/list`）支持通过职称等级要求进行过滤，但有一个特殊处理：
- 当 `jobTitle` 参数为 `3`（"不涉及"）时，不做职称等级过滤，返回所有医生
- 当 `jobTitle` 为其他值（1=正高，2=副高）时，该值会映射到 `positionGrade` 字段进行精确匹配

下拉列表只返回状态为"启用"的医生（`status=true`）。

## 关联模块

- **ykj-service-consultation**：会诊模块
- **InstService**：机构管理服务（用于根据用户名查找机构）
- **InstConsultationDoctorService**：会诊-医生关联服务（用于删除校验）
