# 病历签署CA同步与PDF生成流程

> 文档版本 2026-07-02 | 面向读者：产品、运营

## 概述

医生在问诊过程中为患者创建或编辑病历后，需要通过电子签名完成病历签署。系统支持两种签署模式：普通签名（在PDF末尾追加文字签名）和CA认证签名（同步到第三方CA机构生成具有法律效力的电子签章）。

## 触发条件

- 医生在问诊室中点击"写病历"并保存
- 医生编辑已有病历并保存
- 管理员在后台对未签署病历发起签署

## 步骤详解

### 第一步：创建病历

医生填写病历信息后点击保存，系统执行以下操作：

1. **组装医生信息**：查询医生详情，自动填充医生姓名、医院名称、科室编码/名称、职称类别、执业类别（西医/中医）
2. **组装患者信息**：若指定了就诊人，查询患者详情自动填充姓名、性别、身份证、生日、年龄/月龄、监护人信息
3. **病史信息策略**：优先使用医生填写的病史（过敏史/既往史），若医生未填则取患者的已有病史信息
4. **初始状态**：病历签名状态置为"未签名"
5. **保存诊断**：保存中西医诊断列表，校验诊断名称不能为空，校验疾病诊断编码是否存在

### 第二步：创建签署记录与CA判断

每次病历创建/编辑后，系统自动创建一条签署记录（状态="签署中"），然后判断是否使用CA签名：

- **CA启用条件**：必须同时满足两个开关——CA基础服务开关（enableCA）+ 病历CA功能开关（medicalRecordCAEnabled）
- **主动禁用CA**：若调用方传入 disableCa=true，则强制使用普通签名模式

### 第三步：CA同步（CA模式）

若使用CA签名，系统执行CA同步：

1. 根据医生信息查询CA账号（caUserNo）
2. 生成CA签署请求（含签名坐标、签署方关键词等参数）
3. 同步到第三方CA机构（医网信/放心签）
4. 返回 caUniqueId 给前端，前端据此调起CA签名页面
5. 用户（医生/患者）在CA页面完成实名签名

### 第四步：完成签署

签署完成后系统回调完成签署流程：

**CA模式**：
1. 查询CA签署记录，获取签署后的 PDF 文件（Base64格式）
2. 若有CA账号的签名图片，替换为CA签名图片
3. 取CA签署完成后的 signedData

**普通模式**：
1. 根据病历数据生成 PDF（含患者信息、就诊信息、病史、中西医诊断）
2. 在 PDF 末尾追加"医生签名：XXX"文字

**共同步骤**：
- 上传签署后的 PDF 到对象存储
- 更新病历状态：签名状态="已签名"，保存签署文件地址和签名数据
- 更新签署记录状态为"已签名"

### 第五步：病历编辑与重新签署

编辑病历后，医生签名状态重置为"未签名"，需要重新签署。编辑时的特殊处理：

1. **诊断差异计算**：对比原有诊断和更新后的诊断，找到需删除的和需新增的
2. **关联保护**：不允许删除已经被处方等关联的诊断，提示"不允许删除已经被关联的诊断。诊断名称：【XXX】"
3. **例外**：若调用方传入 disableValidDelBindingDiagnosis=true，则跳过关联保护检查

### 第六步：病历撤销

撤销条件：所有实现 MedicalRecordRevoke 接口的断言通过（如验证病历是否已被处方引用等）

撤销操作：逻辑删除（标记删除原因），不物理删除数据。

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 病历不存在 | 提示"查询病历失败，病历不存在" |
| 诊断名称为空 | 提示"诊断名称不能为空" |
| 医生不存在 | 提示"医生不存在" |
| 病历已签署 | 提示"病历已签署，不需要重复签署" |
| 非"签署中"状态完成签署 | 提示"非签署中病历，无法完成签署" |
| 删除已关联的诊断 | 提示"不允许删除已经被关联的诊断。诊断名称：【XXX】" |
| 病历已绑定会员时换绑 | 提示"该病历已绑定会员，不允许重新绑定" |
| 参数错误（未找到签署记录） | 提示"参数错误，未找到对应的病历签署记录" |
| 会员不存在时绑定 | 提示"变更病历患者失败，会员不存在" |

## 签署模式对比

| 对比项 | CA 签名 | 普通签名 |
|--------|---------|---------|
| 法律效力 | 等同于手写签名 | 电子标记 |
| 操作方式 | 跳转第三方CA页面实名认证 | 系统自动完成 |
| 签名图片 | CA账号签名图片 | 医生姓名文字 |
| 前置条件 | CA开关 + 病历CA开关均开启 | 无需额外配置 |

## 关联功能

- 病历创建与签章（小程序端/后台端）
- CA签章用户管理
- 处方开具（依赖病历中的诊断信息）
- 对象存储（签章后PDF文件存储）
