# 微信医保支付回调与结算流程

> 文档版本 2026-07-03 | 面向产品/运营/客服

## 概述

患者在微信小程序中使用医保支付后，微信支付平台会主动回调通知系统支付结果。系统接收回调后进行支付状态同步和业务处理，确保医保结算数据准确。

## 触发条件

- 患者在微信小程序中选择"医保支付"方式完成支付
- 微信支付平台异步推送支付结果通知
- 系统无需用户主动触发，由微信平台自动发起

## 步骤详解

### 第一步：微信平台发起支付回调

患者在小程序完成医保支付后，微信支付平台向系统发送支付结果通知。通知以XML格式传输，包含支付流水号、支付金额、支付状态等关键信息。

### 第二步：系统接收并校验回调

系统接收回调数据后进行解析和校验：
- 校验支付信息的完整性和合法性
- 解析支付流水号、医保结算单号
- 提取支付金额、支付时间
- 识别支付渠道（医保个账、医保统筹、自费部分）

### 第三步：更新订单支付状态

根据回调结果更新对应订单的支付状态：
- 如支付成功：将订单状态从"待支付"更新为"已支付"
- 标记支付渠道和金额明细
- 记录支付完成时间

### 第四步：同步医保结算数据

支付成功后，将结算数据同步到医保结算记录中：
- 生成医保结算记录
- 关联问诊单号、处方单号
- 记录医保各项支付明细（统筹支付、个账支付、自费金额）

### 第五步：触发后续业务流程

- 通知医生端：患者已完成支付，可进行后续诊疗
- 触发处方审核流程（如有处方）
- 如涉及药品配送，触发备货通知

## 异常处理

| 情况 | 系统处理方式 |
|------|------------|
| 回调数据格式异常 | 返回失败响应，微信平台会按策略重试 |
| 支付金额与订单金额不匹配 | 记录异常日志，标记订单为异常状态，需人工核查 |
| 订单不存在或已处理 | 记录日志并返回成功（幂等处理，避免重复回调） |
| 医保结算数据保存失败 | 事务回滚，返回失败响应，等待微信重试 |
| 网络超时未收到回调 | 提供主动查询接口，可手动查询微信支付结果 |

## 关联功能

- **订单管理**：订单支付状态更新（ykj-service-order）
- **问诊模块**：支付完成后触发问诊流程（ykj-service-inquiry）
- **处方审核**：支付成功后进入药师审方环节（ykj-service-inquiry）
- **医保结算管理**：管理后台可查看上传状态（管理后台-医保结算管理）
- **微信小程序**：患者端支付成功页面跳转
