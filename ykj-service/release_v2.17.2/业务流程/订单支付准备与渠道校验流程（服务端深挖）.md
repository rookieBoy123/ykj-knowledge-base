# 订单支付准备与渠道校验流程（服务端深挖）

> 文档版本：v1.0 | 面向读者：产品经理、测试工程师、后台开发

## 概述

用户点击"支付"按钮后，服务端在拉起支付页面前会执行一套完整的支付准备逻辑，包括：订单可支付状态校验、支付渠道动态组装、金额计算、有效期计算、历史渠道反查防止重复支付等

## 校验流程

### 第一步：订单可支付状态校验

系统按以下顺序逐一检查：

1. **已支付检查**：如果订单已处于"已支付"状态，直接返回错误码 `40001`，提示"当前订单已支付"
2. **非待支付状态检查**：如果订单不是"待支付"状态（如已取消/已完成），返回错误码 `40002`，提示"订单已失效"
3. **有效期检查**：如果订单已超过过期时间（创建时设置，默认24小时），也返回"订单已失效"
4. **历史渠道反查**：查找该订单历史上所有 callbackFlag=0（回调等待中）的预支付单，向微信/支付宝查单——如果查到已支付，则自动补单并拦截本次支付（防止重复支付）

### 第二步：支付渠道动态组装

校验通过后，系统根据配置和订单类型动态组装可用的支付渠道：

| 支付渠道 | 条件 | 默认选中 |
|----------|------|----------|
| 微信支付（JSAPI） | 始终支持 | ✅ 是 |
| 微信扫码支付（Native） | 始终支持 | 否 |
| 支付宝 | 需配置 `payment.aliPay.enabled=true` | 否 |
| 医保支付 | 需满足三个条件：非定金订单、医保开关已开启、订单来源类型支持医保 | 否 |

其中医保支付还需要额外校验（如患者的医保资质、问诊类型是否支持医保结算）

### 第三步：支付金额计算

- 如果没有已付金额（首付），支付金额 = 订单总金额
- 如果已有部分支付（如订金），支付金额 = 总金额 - 已支付金额
- 测试环境：非医保、非0元订单，系统会将订单金额改为0.01元方便测试

### 第四步：返回支付信息

最终返回：
- 订单编号
- 待支付金额
- 支付有效期（过期时间 - 当前时间，或创建时间计算的过期时间）
- 可用支付渠道列表

## 状态机

```
订单创建（PENDING_PAYMENT）
    ↓
支付准备（校验通过 → 返回支付渠道）
    ↓
用户选择渠道 → 预支付（加分布式锁 → 调第三方 → 生成支付单）
    ↓
支付回调（成功 → 已支付 / 失败 → 保持待支付）
    ↓
已支付（PAYED）

订单过期（超过24h）→ EXPIRED
取消（PENDING_PAYMENT 或特定类型 COMPLETED）→ CANCELED / REFUNDING
```

## 异常处理

| 情况 | 系统处理方式 | 错误码 |
|------|-------------|--------|
| 订单已支付 | 前端提示"当前订单已支付" | 40001 |
| 订单已过期或非待支付 | 前端提示"订单已失效" | 40002 |
| 预支付锁获取失败（并发） | 前端提示"支付处理中，请稍后重试" | 40003 |
| 历史渠道查到已支付 | 自动补单后提示"当前订单已支付" | 40001 |
| 医保资质不满足 | 支付渠道列表不显示医保选项 | — |

## 关联功能

- **订单创建**：`OrderCommandServiceImpl.createOrder()` — 创建时计算过期时间
- **订单取消**：`OrderCommandServiceImpl.cancelOrder()` — 取消时使用乐观锁保护
- **支付回调**：`PaymentWxPayController` / `PaymentAliPayController` — 回调后推进订单状态
- **自动收货**：`OrderCommandServiceImpl.automaticReceiptFor15Days()` — 发货15天后自动签收
