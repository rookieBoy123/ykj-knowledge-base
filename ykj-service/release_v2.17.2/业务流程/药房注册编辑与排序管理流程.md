# 药房注册、编辑与排序管理流程

> 文档版本：2026-07-02 | 面向读者：产品/运营/实施

## 概述

药房是系统中最核心的资源配置之一，医生开方必须从授权药房中选择药品。管理员在后台管理药房的基础信息（名称、类型、负责人）、启用状态、库存管理开关、药品类型（成药/中药）、加工服务配置、第三方药房对接，以及药房的显示排序。本流程覆盖 ykj-service-medicine 模块的 PharmacyServiceImpl 后端核心逻辑。

## 步骤详解

### 一、新增药房

**前置校验**（三重）：
1. **手机号必填**：`"手机号不能为空"`
2. **名称唯一**：药房名称不能与已有药房重复 → `"药房名称重复"`
3. **手机号唯一**：一个手机号只能绑定一个药房 → `"该手机号已绑定药房"`

**创建逻辑**：
- 启用第三方药房时：先创建第三方药房记录（`thirdPharmacyService.create`），再创建药房记录
- 药房显示序号：自动设为当前最大值 + 1（排在最后）
- **药品类型含中药饮片**：自动保存切片加工服务（`pharProcessingServicesService`）
- **线下药房**：自动注册/绑定药房管理员账号（给手机号注册为系统用户，授予"线下药房管理员"角色 `ROLE_CODE_OFFLINE_ADMIN`），并绑定用户与药房关系

### 二、编辑药房

**前置校验**（四重）：
1. 药房是否存在（根据ID查找）
2. 药房名称排己重复校验（排除自身ID）
3. 手机号排己绑定校验
4. 已启用第三方药房的不允许改为禁用 → `"已启用第三方药房的不允许编辑为禁用"`

**编辑逻辑**：
- 启用第三方药房且之前未启用：新增第三方药房记录
- 已启用第三方药房：更新第三方药房信息
- 线下药房修改手机号：解绑旧号码关联 → 注册/绑定新号码
- 药品类型含中药饮片：保存或更新加工服务

### 三、药房绑定管理

**医生-药房绑定**（`checkBinding`）：校验指定医生是否有指定药房的权限。医生只能看到被授权药房的药品。

**用户-药房绑定**（`bind`）：
- 一个用户只能绑定一个药房（不支持多药房切换）
- 已绑定其他药房时提示 → `"当前用户已绑定别的药房"`
- 线下药房自动绑定管理员

**用户-药房解绑**（`unbind`）：
- 仅当用户没有其他线下药房角色时才删除关联关系
- 避免误删还有药房管理权限的用户

### 四、药房查询策略

药房查询时受系统配置影响：

| 配置项 | 影响 |
|--------|------|
| `pharmacyProperties.defaultEnabled=true` | 强制返回默认药房（忽略查其他） |
| `pharmacyProperties.offlineEnabled=false` | 只查询线上药房，不返回线下药房 |
| `bindings`（授权列表） | 只返回医生已授权的药房 |
| `pharmacyType` 参数 | 可按药房类型筛选 |

### 五、药房药品类型查询

查询医生可用的药品类型（`findEnableDrugsType`）：
- 遍历医生授权药房的药品类型（成药/中药饮片）
- 去重汇总返回
- 用于前端判断是否显示中药/成药Tab

### 六、药房状态管理

**启用**（`enable`）：状态设为启用（1），药房出现在医生选择列表中

**禁用**（`disable`）：状态设为禁用（0），药房从医生选择列表中移除

### 七、药房排序管理

四个排序操作方法：

| 操作 | 方法 | 边界保护 |
|------|------|---------|
| 排至最前 | `moveTop` | 已最前 → `"已经排在最前啦"` |
| 排至最后 | `moveBottom` | 已最后 → `"已经排在最后啦"` |
| 上移 | `moveUp` | 已最前 → `"已经排在最前啦"` |
| 下移 | `moveDown` | 已最后 → `"已经排在最后啦"` |

排序下限和上限通过 `minOrder()` 和 `maxOrder()` 实时查询。

### 八、药品数量统计

当药品新增或删除时（`updatePharmacyDrugsCount`）：
- 分别统计成药数量（`westernDrugsCount`）和中药数量（`chineseDrugsCount`）
- 更新到药房记录的统计字段

## 异常处理

| 错误提示 | 触发条件 |
|---------|---------|
| "手机号不能为空" | 新增药房未填手机号 |
| "药房名称重复" | 新增时名称冲突 |
| "该手机号已绑定药房" | 新增时手机号已被占用 |
| "药房信息不存在，id=XXX" | 编辑时ID不存在 |
| "已启用第三方药房的不允许编辑为禁用" | 编辑第三方药房状态 |
| "传参错误，没找到对应药房" | 排序/启禁用时ID不存在 |
| "已经排在最前啦" | 排至最前/上移边界 |
| "已经排在最后啦" | 排至最后/下移边界 |
| "当前用户已绑定别的药房" | 用户已绑定其他药房 |
| "ids不能为空" | 批量查询ID为空 |
| "药房新增失败" | 数据库插入失败 |
| "药房修改失败" | 数据库更新失败 |
| "药房名称为空" | 名称校验时传空 |
| "手机号码为空" | 手机号校验时传空 |
| "启用失败" / "禁用失败" | 启禁用操作数据库失败 |

## 关联功能

- **药品管理**（DrugsServiceImpl/MedicalDrugsServiceImpl）：药品的增删改查
- **医生药房授权**：医生只能看到授权药房
- **第三方药房对接**（ThirdPharmacyService）：启用后定时同步药品和库存
- **库存管理**：库存扣减受药房库存开关控制
- **处方开立**：从药房选择药品开方
- **线下药房管理**：独立的药房端管理后台
