# 患者随访管理流程

> 文档版本：v1.0 | 最后更新：2026-07-02 | 面向读者：产品经理、运营人员

## 概述

医生可为问诊过的患者创建随访计划，按计划向患者推送随访任务（如问卷、患教内容、提醒等），跟踪患者康复情况。随访计划有完整的生命周期（进行中→已完成/已取消），每个计划包含多个随访任务。

## 模块信息

- **服务模块**：`ykj-service-followup`
- **代码来源**：FollowupServiceImpl.java (125行)、ApiFollowupController.java (90行)、ApiFollowupTaskController.java (97行)

## 触发条件

- **前置条件**：医生与患者存在问诊记录（无问诊记录无法创建随访）
- **入口**：医生端小程序 → 患者管理 → 添加随访

## 核心概念

| 概念 | 说明 |
|------|------|
| 随访计划 | 医生为一个患者创建的康复跟踪计划，包含名称、患者、医生等基本信息 |
| 随访任务 | 随访计划下的具体执行项，按时间节点发送给患者，包含问卷/患教/提醒等类型 |
| 随访模板 | 预设的随访方案模板，医生可基于模板快速创建随访计划 |

## 随访计划状态流转

```
进行中(0) → 已完成(1) → (可添加总结)
     ↓
  已取消(2)
```

| 状态 | 状态码 | 说明 |
|------|--------|------|
| 进行中 | 0 | 随访计划正在执行中 |
| 已完成 | 1 | 随访计划已执行完毕，可添加总结 |
| 已取消 | 2 | 随访计划被取消（同时取消所有待随访任务） |

## 随访任务状态流转

```
待随访(0) → 已随访(1) → 已完成(2)
     ↓
  已取消(3)
```

| 状态 | 状态码 | 说明 |
|------|--------|------|
| 待随访 | 0 | 任务已创建但未到随访时间 |
| 已随访 | 1 | 任务已推送给患者 |
| 已完成 | 2 | 患者已完成任务（如填写问卷、阅读患教内容） |
| 已取消 | 3 | 任务被取消（通常因随访计划取消而连带取消） |

## 接口详解

### 一、随访计划管理

| 操作 | 接口 | 说明 |
|------|------|------|
| 列表查询 | GET /api/followups | 分页查询随访计划，可按名称、创建人、患者、医生、状态筛选 |
| 添加随访 | POST /api/followups | 新增随访计划，需校验该医生与患者存在问诊记录 |
| 添加校验 | POST /api/followups/_save-verify | 校验是否存在同名、同患者且进行中的重复随访计划 |
| 随访详情 | GET /api/followups/{id} | 查看随访计划详情 |
| 总结添加 | POST /api/followups/_summary | 为已完成的随访计划添加总结 |
| 取消随访 | POST /api/followups/cancel | 取消随访计划，同时取消所有待随访状态的任务 |

### 二、随访任务管理

| 操作 | 接口 | 说明 |
|------|------|------|
| 任务列表 | GET /api/followup/tasks | 分页查询随访任务（小程序端仅可查"已随访"和"已完成"状态） |
| 任务详情 | GET /api/followup/tasks/{id} | 查看任务详情 |
| 任务小结 | POST /api/followup/tasks/_summary | 医生为任务添加小结 |
| 内容状态编辑 | POST /api/followup/tasks/_edit-item-status | 标记患教已读、提醒状态等 |
| 问卷答题 | POST /api/followup/tasks/_answer-sheet | 患者提交随访问卷答案 |
| 问卷详情 | POST /api/followup/tasks/_item-questionnaire-details | 查看随访任务中的问卷详情 |
| 患教详情 | POST /api/followup/tasks/_item-course-details | 查看任务中的患教内容详情 |
| 提醒详情 | POST /api/followup/tasks/_item-remind-details | 查看任务中的提醒详情 |

## 业务规则

### 1. 添加随访的前置校验
- **必须存在问诊记录**：添加随访时，校验患者与该医生的问诊记录，不存在则提示"未有问诊记录无法添加随访"
- **防重复校验**：同一医生对同一患者，不能创建两个名称相同且状态为"进行中"的随访计划

### 2. 添加总结的限制
- 只有状态为"已完成"的随访计划才能添加总结
- 否则系统提示"已完成的计划才能添加总结"

### 3. 取消随访的联动处理
- 取消计划时，自动将该计划下所有"待随访"状态的任务批量更新为"已取消"
- 已随访和已完成的任务不受影响

### 4. 任务状态查询限制
- 小程序端查询任务列表时，仅允许查询"已随访"和"已完成"状态
- 传入其他状态值会返回"非法状态查询"错误
- 不传状态值时默认查询以上两种状态

## 异常处理

| 情况 | 处理方式 |
|------|---------|
| 无问诊记录添加随访 | 后端拦截，返回"未有问诊记录无法添加随访" |
| 重复创建进行中的随访 | save-verify 校验拦截 |
| 非"已完成"计划添加总结 | 后端拦截，返回"已完成的计划才能添加总结" |
| 小程序查询非法状态 | 后端拦截，返回"非法状态查询" |
| 取消计划时关联任务处理 | 自动将待随访任务批量标记为已取消 |

## 关联功能

- **问诊管理**：随访依赖问诊记录，无问诊不可创建随访
- **患者管理**：随访与患者绑定
- **问卷管理**：随访任务可包含问卷内容
- **患教内容管理**：随访任务可包含患者教育内容
