# 问诊单激活与超时自动处理流程

> 文档版本 2026-07-02 | 面向读者：产品、运营

## 概述

问诊单创建后需要经过激活、绑定会员、超时处理等多个环节。系统通过定时任务自动处理超时未接诊的问诊单，确保问诊流程不会长时间挂起。

## 触发条件

- 患者支付成功后，系统自动激活问诊单
- 开放平台将问诊单绑定给指定会员
- 定时任务扫描超时未处理的问诊单

## 步骤详解

### 第一步：问诊单激活

问诊支付完成后，系统自动激活问诊单：

1. 校验问诊单是否存在
2. 启用问诊聊天室（IM群组）
3. 发送"问诊开始"通知给患者和医生
4. 若配置了自动接诊，系统主动调起医生接诊流程

### 第二步：开放平台绑定会员

当外部机构患者通过开放平台对接时，需要将问诊单绑定给平台会员：

1. 校验机构是否存在（通过 appId）
2. 校验问诊单存在且属于该机构
3. 校验问诊状态：只允许"草稿"或"问诊中"状态的问诊单绑定
4. 校验会员不能是问诊单中的参与医生（防止医生给自己看病）
5. 首次绑定时设置会员 ID 并激活问诊
6. 已被其他会员绑定的问诊单提示"问诊单已被领取！"

### 第三步：重复问诊检测

患者发起新问诊时，系统检查是否已有进行中的同类型问诊：

- **商城问诊**：相同用药需求编号不允许重复
- **快速问诊**：同一服务类型 + 同一会员 + 同一患者 + 同一医生 → 已有待支付或问诊中的记录时不允许再发起
- **预约挂号**：同一会员 + 同一医生 + 同一日期（挂号日期） → 不允许重复挂号
- **普通问诊**：同一服务类型 + 同一医生 + 同一会员 + 同一患者 → 不允许重复发起
- **不允许自己问诊自己**：若当前会员是医生账号下的医生，提示"系统不支持问诊自己账号下医生"

### 第四步：问诊状态与患者端展示

患者端按四种选项卡分类展示问诊列表：

| 选项卡 | 筛选逻辑 |
|--------|---------|
| 已预约 | 问诊中状态 + 挂号类型 + 未签到 |
| 问诊中 | 问诊中状态 + 排除已预约但未签到的记录 |
| 已完成 | 医生标记为"完成"状态 |
| 待支付 | 草稿状态（支付前） |

"已预约"和"问诊中"虽然都是 flowStatus=20，但通过是否签到（isCheckIn）区分：未签到→已预约，已签到→问诊中。

### 第五步：超时自动处理（定时任务）

系统配置了自动结束超时问诊的定时任务（需开启 autoEnd 开关）。规则如下：

1. **任务参数**：问诊单ID + 医生标记状态
2. **可处理状态**：仅"草稿"和"问诊中"状态的问诊单
3. **有服务医生时**：
   - 主治医生问诊中（MainUnderway）→ 执行**转诊**操作，将问诊转给服务医生
   - 助理医生问诊中（AssistantUnderway）→ 自动**取消问诊**，原因为"超过接诊时间，系统自动取消"
4. **无服务医生时**：直接**结束问诊**

### 第六步：患者列表显示规则

患者端消息列表中的问诊筛选逻辑：

- 医生列表：按"待接诊"或"已预约"类型筛选
- 患者列表：按"wait/underway/finish/draft"四种类型筛选
- 消息列表：展示已结束、已取消、问诊中三种状态的问诊

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 问诊单不存在 | 提示"问诊单不存在，无法激活问诊" |
| 机构不存在 | 提示"机构不存在" |
| 问诊已结束或已取消时绑定 | 提示"问诊已结束或已取消！" |
| 医生绑定为自己的患者 | 提示"问诊单参与医生不能作为问诊单患者" |
| 问诊单已被其他人领取 | 提示"问诊单已被领取！" |
| 问诊自己账号下的医生 | 提示"系统不支持问诊自己账号下医生" |
| 当前用户非医生角色 | 提示"当前用户非医生角色" |
| 参数错误（type 为空） | 提示"参数错误" |

## 关联功能

- 问诊生命周期管理（创建→支付→接诊→问诊中→结束）
- 问诊预约锁定与签到管理
- IM群组管理（聊天室创建/禁用）
- 自动结束超时问诊（定时任务）
