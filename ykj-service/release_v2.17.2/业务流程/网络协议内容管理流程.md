# 网络协议内容管理流程

> 文档版本：2026-07-03 | 面向读者：产品、运营、管理员

## 概述

网络协议是指小程序和管理后台中需要展示给用户的各类法律协议、隐私政策、用户协议等文本内容。例如《用户服务协议》《隐私政策》《知情同意书》等。管理员可以在后台编辑这些协议的内容（支持富文本），编辑后小程序端和 App 端实时生效。

## 触发条件

- 需要更新《用户服务协议》或《隐私政策》的条款内容
- 新增一种协议类型（如新增《数据授权协议》）
- 法务要求修改现有协议文本

## 数据模型

每个网络协议包含以下字段：
- **协议编码（code）**：唯一标识，小程序端通过 code 获取对应协议（如 `USER_AGREEMENT`、`PRIVACY_POLICY`）
- **协议内容（content）**：富文本内容，支持 HTML 格式
- **是否在 App 端显示（appShow）**：控制该协议是否在小程序/App 端公开可见

## 步骤详解

### 第一步：查询协议列表

管理员在后台可查看所有已配置的网络协议，支持分页和条件筛选。

### 第二步：编辑协议内容

管理员选择一条协议记录，修改其内容：
- 编辑时**协议编码不可修改**（code 字段更新时被置为 null）
- 协议内容使用富文本编辑器
- 内容中可使用 `${hospital}` 占位符，查询时自动替换为系统配置的医院名称

### 第三步：小程序端获取协议

小程序端通过编码（code）获取协议内容：
- 请求 `/api/cms/network-protocols/{code}`，如 `/api/cms/network-protocols/USER_AGREEMENT`
- 系统自动将内容中的 `${hospital}` 替换为实际的互联网医院名称
- 如果 code 对应的协议不存在，返回空

### 第四步：小程序端协议列表

小程序端也可以分页查询所有标记为"App 端显示"的协议列表，用于协议列表页展示。

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 查询不存在的协议编码 | 返回空（null），小程序端展示空白或默认文案 |
| 协议内容中的占位符 | 自动替换为系统配置的医院名称（通过 SystemProperties.title） |
| 编辑时传入的 code | 被忽略（code 不允许修改），更新时不改变协议编码 |

## 关联功能

- **系统配置管理**：`${hospital}` 占位符的替换值来自系统配置中的互联网医院名称
- **小程序协议页**：小程序端通过此 API 获取协议内容并渲染展示
- **用户协议管理（管理后台）**：后台的协议文档管理与本功能相邻，都是内容管理模块的组成部分
