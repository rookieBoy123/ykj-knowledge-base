# 会诊受邀机构管理流程（服务端）

> 文档版本 2026-07-04 | 面向技术/产品/运营

## 概述
管理远程会诊体系中的受邀机构（合作医院）。每个受邀机构对应一个独立的登录账号，拥有自己的医生和服务资源。系统在创建机构时自动注册系统用户并分配"受邀机构管理员"角色。

## 涉及服务模块
- `ykj-service-consultation`（会诊模块）：机构CRUD、医生管理、服务管理
- `ykj-service-system`（系统模块）：用户注册、角色授权、启用/禁用

## 核心接口

| 接口 | 方法 | 路径 | 权限码 |
|------|------|------|--------|
| 分页查询 | GET | `/mgmt/consultation/insts` | `consul:insts:read` |
| 下拉列表 | GET | `/mgmt/consultation/insts/list` | `consul:insts:read` |
| 新增 | POST | `/mgmt/consultation/insts` | `consul:insts:write` |
| 编辑 | PUT | `/mgmt/consultation/insts/{id}` | `consul:insts:write` |
| 详情 | GET | `/mgmt/consultation/insts/{id}` | `consul:insts:read` |
| 删除 | DELETE | `/mgmt/consultation/insts/{id}` | `consul:insts:write` |

## 核心业务规则

### 1. 新增机构的完整流程

```
接收请求 → 校验机构名称唯一性 → 校验账号名是否重复 → 校验手机号是否已注册
→ 注册系统用户（SysUser）→ 授予"受邀机构管理员"角色 → 保存机构记录
```

**状态图**：无状态流转，机构创建后即为"启用"或"禁用"状态。

**详细步骤**：

1. **机构名称唯一性校验**：查询是否已有同名机构，存在则抛出 `机构名称已存在!`
2. **系统用户注册**：使用机构账号、联系人、手机号、密码注册系统用户（`UserRegisterCommand`）
3. **账号名重复校验**：如果账号名已被其他用户占用，抛出 `账户名重复！`
4. **手机号重复校验**：如果手机号已被其他用户注册，抛出 `手机号已被注册！`
5. **角色授予**：注册成功后，自动授予 `invitedOrgManager`（受邀机构管理员）角色

### 2. 编辑机构

- **名称唯一性校验**：检查新名称是否与其他机构（排除自身）重复，重复则抛出 `机构名称已存在!`
- **同步更新系统用户**：
  - 根据启用/禁用状态，对应调用 `userService.enable()` 或 `userService.disable()`
  - 更新用户手机号
  - 校验账户是否存在（不存在则抛出 `账户不存在！`）
  - 校验手机号是否被其他用户占用（存在且不是当前用户，则抛出 `手机号已被注册！`）

### 3. 删除机构的级联检查

删除前会检查两个关联关系：

| 检查项 | 条件 | 拦截提示 |
|--------|------|----------|
| 机构下是否有医生 | `instDoctorMapper` 查询该机构ID | 请先删除受邀机构【X】下的服务和医生，否则无法删除! |
| 机构下是否有服务 | `instServeMapper` 查询该机构ID | 同上 |

两个条件任一满足即拦截删除。删除成功后，系统会收回该机构账号的 `invitedOrgManager` 角色。

### 4. 分页查询

- 支持按机构名称模糊搜索（`like` 查询）
- 按创建时间倒序排列
- 下拉列表接口（`/list`）只返回启用状态的机构

## 异常处理

| 情况 | 系统处理方式 | 提示位置 |
|------|------------|---------|
| 机构名称已存在（新增） | 抛出 `机构名称已存在!` | 后端拦截 |
| 机构名称已存在（编辑） | 抛出 `机构名称已存在!` | 后端拦截 |
| 账号名重复 | 抛出 `账户名重复！` | 后端拦截 |
| 手机号已被注册（新增） | 抛出 `手机号已被注册！` | 后端拦截 |
| 手机号被其他用户占用（编辑） | 抛出 `手机号已被注册！` | 后端拦截 |
| 编辑时账户不存在 | 抛出 `账户不存在！` | 后端拦截 |
| 删除时机构下有医生/服务 | 抛出 `请先删除受邀机构【X】下的服务和医生，否则无法删除!` | 后端拦截 |

## 关联功能
- **受邀机构医生管理**：机构创建后，可在其下添加参与会诊的医生（`MgmtInvitedInstDoctorController`）
- **受邀机构服务管理**：机构创建后，可配置其提供的会诊服务
- **远程会诊申请**：受邀机构出现在会诊申请的发邀请方选择列表中
