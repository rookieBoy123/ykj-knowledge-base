# 服务预约全生命周期流程

> 文档版本：2026-07-02 | 面向读者：产品经理、运营人员、客服

## 概述

服务预约是医院提供的线下体检、健康证、检查等服务的在线预约功能。患者通过小程序选择服务项目、预约时间后生成预约单，到院后出示核销码完成服务。整个生命周期涵盖预约创建→支付→核销→取消→退款→自动开票等完整链路。

## 触发条件

- 入口路径：小程序「服务预约」→ 选择服务项目 → 确认预约信息 → 提交预约
- 前置条件：患者已完成实名认证、已添加就诊人

## 步骤详解

### 第一步：选择服务项目与预约时间

患者在小程序浏览可预约的服务项目列表，系统根据项目类型展示不同服务：
- **个人预约（0元）**：检查、健康证等免费项目
- **个人预约（收费）**：体检等收费项目
- **团队/企业预约**：企业团体体检，仅收定金

患者选择服务后，系统展示该服务的可预约日期列表。只有配置了有效排班的日期才可选。

### 第二步：填写预约信息并提交

患者填写以下信息后提交：
- 联系人与联系方式
- 就诊人选择
- 预约日期和时间段
- 服务方式（上门服务 vs 自行到院）
- 是否需要发票

系统在不同情况下生成不同状态：
- **金额为 0 元**：预约单状态直接设为「待核销」，同时生成 8 位大写随机核销码
- **金额大于 0 元**：预约单状态设为「待付款」，进入支付流程

### 第三步：支付（仅收费项目）

对于收费项目，系统计算费用明细：
- **个人预约**：基础服务费 + 可选上门服务费 + 非工作时间段附加费
- **团队预约**：仅收定金

系统创建订单（订单来源标记为「服务预约」），进入收银台完成支付。

### 第四步：支付成功 → 生成核销码

支付成功后，系统自动将预约单状态改为「待核销」并生成 8 位大写字母随机核销码，同时推送订阅消息通知患者预约成功。0 元项目在提交时直接生成核销码，无需等待支付。

### 第五步：到院核销

患者到院后出示核销码，医务人员在管理后台或扫码设备输入核销码进行核销。核销成功后：
- 预约单状态变为「已核销」
- 关联订单完成
- 如开通了自动开票且订单金额不为 0，系统自动发起电子发票开具

### 第六步：取消与退款

患者可在两种状态下取消预约：

**情况一：待支付状态（或 0 元待核销状态）**
系统直接取消预约单和订单，推送取消通知。

**情况二：已支付的待核销状态（金额 > 0）**
取消后预约单状态变为「退款中」，系统发起退款申请。

### 第七步：超时自动取消

对于超时未支付的收费预约单，订单系统自动取消订单后，预约模块监听到取消事件，将预约单状态同步改为「已取消」。

## 异常处理

| 情况 | 系统处理方式 |
|------|-------------|
| 项目暂无可预约时间 | 提交时提示"当前项目暂无可预约时间"，阻止提交 |
| 选择的日期不在可预约范围内 | 提示"当前日期该项目暂无法预约" |
| 核销码输入错误 | 提示"错误的核销码"，核销失败 |
| 核销码已被使用 | 提示"此核销码已核销" |
| 预约单不是待核销状态 | 提示"该订单不是待核销状态订单" |
| 非当前预约人查看 | 检查是否为订单分享记录中的被分享人，否则提示"当前患者无权限查看该预约单信息" |
| 退款时找不到预约单 | 提示"传参错误，找不到对应预约单/订单" |
| 取消时状态不符 | 提示"该预约单不可取消，仅待支付和待核销状态预约订单可取消" |

## 状态流转

```
提交预约 → 待支付（收费项目）→ 支付成功 → 待核销 → 核销完成 → 已核销
提交预约 → 待核销（免费项目）→ 核销完成 → 已核销
待支付 → 超时/主动取消 → 已取消
待核销 → 主动取消（已付费）→ 退款中 → 退款完成 → 已退款
```

## 关联功能

- 订单系统：订单创建、支付回调、取消同步
- 订阅消息：预约成功/取消通知推送
- 电子发票：核销后自动开票（需开通自动开票功能）
- HIS 系统：患者信息查询（获取 hisPatId 用于发票）
- 订单分享：被分享人可查看预约详情
