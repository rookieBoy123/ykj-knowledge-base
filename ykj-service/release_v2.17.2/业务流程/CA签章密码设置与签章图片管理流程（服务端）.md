# CA签章密码设置与签章图片管理流程（服务端）

> 文档版本：2026-07-05 | 面向读者：后台开发/产品

## 概述
服务端管理医生的CA签章密码和签章图片。医生设置签章密码后，系统自动生成一个默认的文字签章（将医生姓名转为图片）；修改签章图片时需要二次验证密码。支持医网信平台和源康健PaaS双平台。

## 触发条件
- 医生完成CA账号创建后，在医网信插件中设置签章密码
- 医生在手写签名页面修改签章图片
- 管理员或系统触发手机号变更

## 核心流程

### 1. 签章密码首次设置（passwordSetting）

**接口**：`PUT /api/ca/users/{userNo}/pwd-setting`

**前置校验**：
- 验证当前CA平台是否允许用户自行修改配置（`allowUpdateSetting`），不支持则返回「暂不支持用户修改配置」
- 查找用户平台绑定记录，不存在则记录日志并跳过

**自动生成默认签章**：如果用户还没有签章图片，系统自动将医生姓名转为文字图片作为默认签章（`getDefaultStampImage` → `ImageUtils.textToBase64Image`）

**密码存储**：密码以MD5加密存储，不保存明文。

### 2. 签章图片修改（stampImageSetting）

**接口**：`PUT /api/ca/users/{userNo}/stampImage-setting`

**二次密码验证**：修改签章图片前必须先验证签章密码，验证不通过返回「您的密码验证不通过, 请重新输入」

**签章更新**：密码验证通过后，将新的签章图片Base64数据更新到用户平台记录。

### 3. 密码验证（verifyPassword）

单独提供密码验证接口，用于签章图片修改前的二次确认。验证方式：将用户输入的密码进行MD5加密后与数据库中存储的密文比对（忽略大小写）。

### 4. 手机号变更联动（changePhone）

当医生变更手机号时：
1. 更新本地CA用户表的手机号（加密存储）
2. 同步更新到医网信平台（调用平台的sync接口）
3. 更新后认证状态重置为「认证中」（AuthStatus.Authing），需要医生重新完成证书下载

## 认证状态流转

| 状态 | 枚举值 | 说明 |
|------|--------|------|
| 未认证 | NotAuth | 尚未开始认证流程 |
| 认证中 | Authing | 正在认证/手机号变更后重新认证 |
| 认证成功 | Succeed | 认证通过，可正常使用签章 |
| 认证失败 | Failure | 认证被拒绝，含失败原因 |

## 异常处理

| 情况 | 处理方式 |
|------|---------|
| 平台不允许修改配置 | 后端直接拦截，返回「暂不支持用户修改配置」 |
| 签章密码验证不通过 | 返回「您的密码验证不通过, 请重新输入」 |
| 用户不存在 | 记录日志并跳过（不抛异常） |
| CA平台策略未找到 | 抛出内部错误「CA服务配置异常」 |
| 手机号同步失败 | 抛出异常「变更手机号码失败，请稍后重试」 |
| CA账号同步失败 | 返回null，后续流程中断 |

## 数据安全

- 身份证号：使用加密存储（`crypto.encrypt`）
- 手机号：使用加密存储
- 签章密码：MD5加密存储，不保存明文
- 用户账号编号：SecureRandom生成19字节随机数，Base64编码为25位字符串，去掉特殊字符

## 关联功能
- **CA账号创建**：医生认证时自动创建CA账号并同步到医网信平台
- **CA认证回调**：医网信认证完成后回调 `handlerAuthed` 更新状态
- **处方签章**：调用签章服务时先验证密码
