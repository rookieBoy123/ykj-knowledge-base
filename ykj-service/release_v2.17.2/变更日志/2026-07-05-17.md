# 变更日志 2026-07-05-17

## 新增业务流程（4篇）

1. **订单过期与倒计时取消流程（服务端深挖）** — 描述 `POST /api/orders/{orderNo}/expire` 新接口的完整处理逻辑：幂等保护、状态校验、前端倒计时归零时的二次确认机制、与 cancelOrder 的内部复用关系
   - 代码来源：`ApiOrderController.expire()`，`OrderCommandServiceImpl.cancelOrder()` 第210-293行

2. **订单退款审核拒绝后状态恢复与退款完成流程（服务端深挖）** — 描述 originFlowStatus 字段的快照设计：退款申请时保存原始状态→审核通过→已退款(90)→审核拒绝→恢复原始状态。说明为何 restoreOrderFlowStatus 不使用乐观锁
   - 代码来源：`OrderCommandServiceImpl.restoreOrderFlowStatus()` 第1144-1161行，`completeRefund()`，Order Entity `originFlowStatus` 字段

3. **订单完成事件与跨服务状态联动流程（服务端深挖）** — 描述 OrderCompletedEvent 事件的设计和触发场景（completeOrder/confirmReceipt/自动收货），以及0元订单的状态分派规则（按订单来源类型映射不同状态）
   - 代码来源：`OrderCommandServiceImpl.completeOrder()`，`OrderCompletedEvent`，`PAYED_STATUS` 常量 Map

4. **支付渠道切换与二维码作废流程（服务端深挖）** — 描述 cancelPayment 接口的设计目的、前端调用时机、后端关闭预支付单的流程、支付宝开关从 FuncProperties 到 PaymentProperties 的配置迁移
   - 代码来源：`ApiOrderController.cancelPayment()`，`OrderCommandServiceImpl.cancelPayment()`，`PaymentProperties.AlipayProperties.enabled`
