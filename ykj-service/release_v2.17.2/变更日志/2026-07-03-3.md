## 变更明细

### 1. 退款申请新增"自编号"字段
- **类型**：功能增强（数据库变更）
- **描述**：退款申请表 `t_order_refund_apply` 新增 `custom_no`（自编号）字段，允许在退款申请中关联自定义编号，方便商户侧对退款单据进行内部编号管理。
- **影响范围**：退款申请数据存储
- **影响角色**：运营人员（后台可查看/使用自编号）
- **代码来源**：
  - `V2_17_2_14__ddl.sql` — 新增 `custom_no` 列
  - commit: e59246faaf

### 2. 源康健PaaS平台CA签章配置自动修复
- **类型**：Bug修复
- **描述**：修复了使用源康健PaaS电子签章平台时，系统配置接口无法正确返回签章客户端ID（clientId）的问题。现在当检测到签章平台为PaaS（platform=3）时，系统会自动注入正确的 clientId 和平台标识，确保签章功能正常工作。
- **影响范围**：使用源康健PaaS平台的医院/商户的电子签章功能
- **影响角色**：医生（处方签章、病历签章）、患者（签署确认）
- **代码来源**：
  - `ApiSysSettingController.java` — 新增 PaaS CA clientId 自动注入逻辑，从 ykj-service-system 迁至 ykj-service-server
  - `pom.xml` — server 模块新增 ca-api 依赖
  - commit: 079c795487

### 3. 订单分成比例字段清理
- **类型**：代码重构（无业务影响）
- **描述**：从订单模型（Order）、订单明细模型（OrderItem）、订单DTO 及对应的 MyBatis Mapper XML 中移除了不再使用的 `inqCommRatio`（问诊费分成比例）、`commRatio`（药品分成比例）及 `businessId`（商务ID）字段。属于代码清理，不影响现有业务功能。
- **影响范围**：无业务影响
- **代码来源**：
  - `Order.java`、`OrderDto.java`、`OrderMapper.xml` — 移除 inqCommRatio 和 businessId
  - `OrderItem.java`、`OrderItemDto.java`、`OrderItemMapper.xml` — 移除 commRatio
  - commits: 7e0febc255, c9368b248a, 03e0be2840
